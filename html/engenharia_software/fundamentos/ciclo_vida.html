Ciclo de Vida do Software (SDLC)

O caminho que todo software percorre, da ideia à aposentadoria

O Software Development Life Cycle (SDLC) descreve as fases pelas quais todo software passa desde sua concepção até que seja desativado. O que diferencia os modelos de processo não são as fases em si — são a ordem, a duração e, acima de tudo, quando o feedback chega ao time.

Fases Típicas do SDLC

As fases a seguir aparecem em praticamente todos os modelos de processo — o que varia é se elas são sequenciais, sobrepostas ou repetidas em ciclos curtos.

Fase Atividades Principais Artefatos de Saída
Planejamento Definição de escopo, viabilidade, riscos iniciais, cronograma e orçamento Plano de projeto, análise de viabilidade
Levantamento de Requisitos Entrevistas, workshops, análise de stakeholders, priorização Documento de requisitos (SRS), backlog, user stories
Design Arquitetura do sistema, modelo de dados, design de interfaces, protótipos Diagrama de arquitetura, ERD, wireframes, ADRs
Implementação Codificação, revisões de código, testes unitários, integração contínua Código-fonte, relatórios de cobertura, builds
Testes Testes de integração, sistema, aceite, carga e segurança Relatórios de teste, lista de defeitos, critérios de aceite validados
Implantação Deploy em produção, configuração de monitoramento, treinamento Software em produção, runbook, dashboards de observabilidade
Manutenção Correção de bugs, melhorias incrementais, gestão de dívida técnica Patches, novas versões, documentação atualizada

Modelos de Ciclo de Vida

Cada modelo organiza as fases de forma diferente e é mais adequado a determinados contextos de projeto. Na prática, times híbridos combinam elementos de vários modelos.

Cascata (Waterfall)

Fases sequenciais e bem documentadas. Uma fase só começa quando a anterior está completa. Feedback chega tarde — adequado para projetos com requisitos extremamente estáveis (governo, hardware embarcado).

Iterativo e Incremental

O sistema é desenvolvido em partes (incrementos), cada um passando por todas as fases. Permite validação parcial antecipada. Base de modelos como RUP (Rational Unified Process).

Ágil

Ciclos curtos (sprints de 1–4 semanas), entrega frequente de valor, feedback contínuo de usuário e negócio. Scrum, Kanban e XP são frameworks ágeis. Requer colaboração intensa e baixa burocracia.

Espiral

Cada ciclo inclui análise de riscos explícita antes de avançar. Indicado para projetos de alto risco e grande escala (ex: sistemas críticos, defesa). Combina prototipagem com planejamento rigoroso.

DevOps / Continuous Delivery

Elimina a barreira entre desenvolvimento e operação. Deploy contínuo, infraestrutura como código, monitoramento em produção retroalimentando o backlog. O ciclo de vida vira um loop contínuo.
