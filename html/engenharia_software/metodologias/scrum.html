Scrum

Framework para entregar incrementos com qualidade em ciclos curtos

Scrum não é uma metodologia completa — é um framework leve que define papéis, eventos e artefatos para gerar feedback regular e incrementos prontos. O objetivo é melhoria da previsibilidade, não da velocidade de cerimônias.

Papéis e responsabilidades

Scrum define três papéis com responsabilidades distintas. Anti-padrões claros ajudam a identificar quando os papéis estão sendo assumidos incorretamente.

PapelResponsabilidade centralAnti-padrão comum
Product OwnerPriorizar backlog por valor, definir objetivo do produtoPO ausente, backlog sem prioridade real
Scrum MasterRemover impedimentos, proteger o processo, facilitar melhoriaSM como gerente de status e controlador
DevelopersEntregar incremento pronto com qualidade dentro da sprintAceitar mais trabalho do que conseguem terminar

Eventos Scrum

Cada evento tem um objetivo específico. Torná-los rituais vazios é o principal sintoma de teatro ágil.

EventoObjetivoSinal de disfunção
Sprint PlanningDefinir objetivo e plano da sprintObjetivo vago, time aceita trabalho sem DoD
Daily ScrumSincronizar e expor bloqueiosStatus report para gerente, sem desbloqueio
Sprint ReviewDemonstrar incremento e coletar feedback realDemo sem stakeholders, sem perguntas reais
RetrospectiveMelhorar o sistema de trabalho com açõesDesabafo sem prioridade de ação

Definition of Done — exemplo

DoD é o contrato de qualidade do incremento. Sem DoD claro, "pronto" significa coisas diferentes para pessoas diferentes.

Definition of Done — time de produto web:

Código
  [ ] todos os testes automatizados passam (unit + integration)
  [ ] cobertura de linhas acima do threshold do projeto
  [ ] code review aprovado por >= 1 dev
  [ ] linting e formatação aplicados (CI verde)

Deploy
  [ ] merge na branch principal sem conflito
  [ ] deploy bem-sucedido em staging
  [ ] smoke test manual em staging OK

Documentação
  [ ] README atualizado se comportamento mudou
  [ ] ADR criado se houve decisão arquitetural relevante
  [ ] critérios de aceite do item marcados como atendidos

Produto
  [ ] demonstrado ao PO ou stakeholder durante sprint review
  [ ] métricas de observabilidade configuradas

Combine Scrum com: Kanban (fluxo) e XP (práticas técnicas).
