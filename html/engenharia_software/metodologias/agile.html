Agile

Valores e princípios para lidar com incerteza e mudança

Agile não é um processo: é um conjunto de valores para construir software em ambientes incertos. O centro é entregar valor cedo, coletar feedback real e adaptar o plano antes que erros fiquem caros.

Os 4 valores do Manifesto Ágil

O Manifesto Ágil (2001) não rejeita a coluna da direita — apenas prioriza a esquerda quando há conflito.

Práticas que materializam Agile

Agile se manifesta em práticas concretas. Sem elas, vira só vocabulário.

Backlog priorizado por valor
  -- PO responsável pela ordem, criada com o time

Entregas frequentes (sprints ou releases contínuos)
  -- ciclos curtos revelam problemas cedo

Code review + CI
  -- qualidade embutida no fluxo, não fase final

Testes automatizados
  -- confiança para mudar sem medo de regressão

Retrospectiva com ação
  -- melhoria do sistema, não desabafo semanal

O que Agile não é

Muitos problemas vêm de mal-entendidos sobre o que Agile propõe de fato.

Não é "sem documentação"

É documentação útil e enxuta. ADRs, README e critérios de aceite têm alto ROI.

Não é "sem planejamento"

É planejamento contínuo e adaptativo. Roadmaps evoluem; o objetivo permanece.

Não é pressa

Qualidade ruim vira dívida técnica. Código apressado compromete velocidade futura.

Não é framework fixo

Scrum e Kanban são implementações. Agile é o conjunto de valores por trás deles.
