Domínio isolado por portas e adaptadores

Arquitetura Hexagonal

Ports and Adapters — Alistair Cockburn

A Arquitetura Hexagonal protege o núcleo (regras e casos de uso) de detalhes externos. O domínio define portas (interfaces) e a infraestrutura fornece adaptadores (implementações concretas). Isso permite testar regras de negócio sem banco, framework ou HTTP.

Portas e adaptadores

O hexágono divide o sistema em interior (domínio) e exterior (infraestrutura):

Tipo Definição Exemplos
Porta de entrada Interface que o domínio expõe para ser acionado (driven port). CriarPedidoUseCase, AutenticarUsuario
Porta de saída Interface que o domínio usa para acessar detalhes externos (driving port). PedidoRepositorio, NotificacaoGateway
Adaptador primário Implementa a comunicação de fora para dentro (ativa o domínio). HTTP Controller, Message Consumer, CLI
Adaptador secundário Implementa as portas de saída (infra a serviço do domínio). PedidoRepositorioSQL, EmailAdapter, KafkaPublisher

Exemplo — porta e adaptador

O domínio define a interface; a infraestrutura implementa sem o domínio saber os detalhes:

// PORTA DE SAÍDA (definida pelo domínio)
interface PedidoRepositorio {
  buscar(id: string): Promise<Pedido>;
  salvar(pedido: Pedido): Promise<void>;
  listarPorCliente(clienteId: string): Promise<Pedido[]>;
}

// ADAPTADOR SECUNDÁRIO (infraestrutura)
class PedidoRepositorioPostgres implements PedidoRepositorio {
  async buscar(id: string): Promise<Pedido> {
    const row = await this.db.query('SELECT * FROM pedidos WHERE id = $1', [id]);
    return PedidoMapper.fromRow(row);
  }

  async salvar(pedido: Pedido): Promise<void> {
    await this.db.query('INSERT INTO pedidos...', [pedido.id, ...]);
  }

  async listarPorCliente(clienteId: string): Promise<Pedido[]> {
    const rows = await this.db.query('SELECT * FROM pedidos WHERE cliente_id = $1', [clienteId]);
    return rows.map(PedidoMapper.fromRow);
  }
}

// CASO DE USO (domínio — não sabe que existe Postgres)
class ConfirmarPedidoUseCase {
  constructor(private repo: PedidoRepositorio) {}

  async executar(pedidoId: string): Promise<void> {
    const pedido = await this.repo.buscar(pedidoId);
    pedido.confirmar();
    await this.repo.salvar(pedido);
  }
}

Benefícios e armadilhas

A arquitetura hexagonal tem benefícios reais mas requer disciplina para não virar burocracia:

Testabilidade

Testes unitários de casos de uso rodal sem banco, servidor ou rede usando fakes das portas.

Substituição de infraestrutura

Trocar PostgreSQL por MongoDB, REST por Queue — o domínio não precisa mudar.

Armadilha: interfaces para tudo

Criar interface sem necessidade real de substituição é over-engineering (viola YAGNI).

Armadilha: domínio com framework

Annotations de ORM dentro de entidades, imports do Express no caso de uso — viola a fronteira.

Hexagonal é a base prática de Clean Architecture. Combine com DIP para maximizar testabilidade.

"O domínio define o que precisa; a infraestrutura entrega como consegue."
