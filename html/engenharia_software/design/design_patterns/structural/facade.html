Interface simplificada para subsistema complexo

Facade

Estrutural — GoF

Facade fornece uma interface simplificada para um conjunto de interfaces num subsistema. Reduz a complexidade visível para o cliente sem remover a capacidade de acessar o subsistema diretamente quando necessário. Um dos padrões mais comuns e práticos.

Facade vs Adapter vs Mediator

Padrão Objetivo Esconde impl.?
Facade Simplificar acesso a subsistema complexo. Sim, mas acesso direto ainda possível.
Adapter Converter interface incompativel para a esperada. Sim, protege o domínio da interface externa.
Mediator Coordenar interação entre objetos que se conhecem. Não — coordena, não simplifica interface.

Exemplo — facade de envio de pedido

// SUBSISTEMA — classes complexas e detalhadas
class ValidadorPedido {
  validar(pedido: Pedido): ValidationResult { /* ... */ return { ok: true }; }
}

class CalculadorFrete {
  calcular(cep: string, itens: Item[]): number { /* ... */ return 15; }
}

class ProcessadorPagamento {
  processar(valor: number, cartao: Cartao): PaymentResult { /* ... */ return { sucesso: true }; }
}

class NotificadorEmail {
  enviarConfirmacao(email: string, pedidoId: string): void { /* ... */ }
}

// FACADE — orquestra o subsistema com interface simples
class PedidoFacade {
  constructor(
    private validador: ValidadorPedido,
    private frete: CalculadorFrete,
    private pagamento: ProcessadorPagamento,
    private notificador: NotificadorEmail
  ) {}

  async finalizar(pedido: Pedido, cartao: Cartao, emailCliente: string): Promise<string> {
    const validacao = this.validador.validar(pedido);
    if (!validacao.ok) throw new Error('Pedido inválido');

    const valorFrete = this.frete.calcular(pedido.cep, pedido.itens);
    const total = pedido.subtotal + valorFrete;

    const pagto = this.pagamento.processar(total, cartao);
    if (!pagto.sucesso) throw new Error('Pagamento recusado');

    this.notificador.enviarConfirmacao(emailCliente, pedido.id);
    return pedido.id;
  }
}

// Cliente usa apenas a facade — simples e legível
await facadePedido.finalizar(pedido, cartao, 'cliente@email.com');

Quando usar

Use quando

Subsistema complexo com várias classes que o cliente precisa coordenar. Wrapper de SDK externo complexo.

Não bloqueia acesso

Facade não encapsula completamente — quem precisa de controle granular ainda acessa as classes do subsistema.

Ponto único de entrada

Centraliza a orquestração. Mudanças no subsistema ficam encapsuladas nela, sem afetar o cliente.

Evite quando

O subsistema já é simples. Uma facade sobre uma única classe é indireção gratuita.

Facade é uma das melhores ferramentas para simplificar onboarding em sistemas legados. Veja Adapter para converter interfaces incompatíveis.

"A facade não é uma muralha — é uma porta principal. O subsistema permanece acessível; a facade simplifica o caminho comum."
