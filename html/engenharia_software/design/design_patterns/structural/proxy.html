Substituto ou representante de outro objeto

Proxy

Estrutural — GoF

Proxy fornece um substituto que controla o acesso a outro objeto. Pode fazer isso para lazy loading, caching, controle de acesso, logging ou interseito de chamadas remotas. O cliente usa a mesma interface do objeto real.

Tipos de Proxy

Tipo Objetivo Exemplo
Virtual Proxy Adia instanciação cara até a primeira chamada. Imagem lazy-loaded, relatório sob demanda
Cache Proxy Armazena resultado de chamadas custosas. Cache de leitura de banco, memoização
Protection Proxy Verifica permissões antes de delegar. Autorização de recurso, ACL
Remote Proxy Representa objeto em outro espaço de memória/rede. RPC stub, gRPC client stub

Exemplo — Cache Proxy

interface ServicoTaxa {
  calcularTaxa(produto: string): Promise<number>;
}

class ServicoTaxaExterno implements ServicoTaxa {
  async calcularTaxa(produto: string): Promise<number> {
    // chamada real a API externa (lenta)
    return 0.12;
  }
}

class ServicoTaxaComCache implements ServicoTaxa {
  private cache = new Map<string, { taxa: number; expira: number }>();
  private TTL_MS = 5 * 60 * 1000; // 5 minutos

  constructor(private real: ServicoTaxa) {}

  async calcularTaxa(produto: string): Promise<number> {
    const agora = Date.now();
    const cached = this.cache.get(produto);

    if (cached && cached.expira > agora) {
      return cached.taxa; // hit de cache
    }

    const taxa = await this.real.calcularTaxa(produto);
    this.cache.set(produto, { taxa, expira: agora + this.TTL_MS });
    return taxa;
  }
}

// Uso — cliente não sabe que há um proxy
const taxas: ServicoTaxa = new ServicoTaxaComCache(new ServicoTaxaExterno());
await taxas.calcularTaxa('PROD-001'); // miss — chama real
await taxas.calcularTaxa('PROD-001'); // hit — retorna do cache

Quando usar

Use quando

Precisa adicionar controle, cache, lazy loading ou logging sem alterar o objeto real ou os clientes.

vs Decorator

Ambos envolvem um objeto com a mesma interface. Decorator adiciona comportamento; Proxy controla acesso. A fronteira é sutil.

Transparente

O cliente não sabe que está usando um proxy. Isso é uma vantagem: adicionar proxy não requer mudança no cliente.

Evite quando

O objeto não tem custo real de acesso. Proxy de caching sem medir o ganho real é prematura otimização.

Proxy e Decorator são estruturalmente similares. A diferença está na intenção: Proxy controla acesso; Decorator enriquece comportamento.

"Proxy coloca-se no caminho entre cliente e objeto real — sem que o cliente saiba ou precise mudar."
