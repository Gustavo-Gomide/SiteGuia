Separar abstração de implementação para variar ambas

Bridge

Estrutural — GoF

Bridge separa uma abstração da sua implementação, permitindo que as duas variem de forma independente. Evita a explosão de subclasses quando há duas dimensões de variação. Em vez de herança, usa composição.

O problema que Bridge resolve

Sem Bridge (herança) Com Bridge (composição)
FormaCirculo, FormaQuadrado, FormaDark, FormaLight... Forma (abstrato) + Tema (impl.) combinam independentemente
N formas × M implementações = N×M classes N + M classes
Adicionar nova impl. = N novas subclasses Adicionar nova impl. = 1 nova classe

Exemplo — formas com renderizadores

// IMPLEMENTAÇÃO (varia independentemente)
interface Renderizador {
  renderizarCirculo(raio: number): string;
  renderizarQuadrado(lado: number): string;
}

class RenderizadorSVG implements Renderizador {
  renderizarCirculo(raio: number) { return `<circle r="${raio}"/>`; }
  renderizarQuadrado(lado: number) { return `<rect width="${lado}" height="${lado}"/>`; }
}

class RenderizadorCanvas implements Renderizador {
  renderizarCirculo(raio: number) { return `ctx.arc(0,0,${raio},0,2*Math.PI)`; }
  renderizarQuadrado(lado: number) { return `ctx.rect(0,0,${lado},${lado})`; }
}

// ABSTRAÇÃO (delega render ao impl. injetado)
abstract class Forma {
  constructor(protected render: Renderizador) {}
  abstract desenhar(): string;
}

class Circulo extends Forma {
  constructor(private raio: number, render: Renderizador) { super(render); }
  desenhar() { return this.render.renderizarCirculo(this.raio); }
}

class Quadrado extends Forma {
  constructor(private lado: number, render: Renderizador) { super(render); }
  desenhar() { return this.render.renderizarQuadrado(this.lado); }
}

// Combinam livremente:
new Circulo(10, new RenderizadorSVG()).desenhar();
new Circulo(10, new RenderizadorCanvas()).desenhar();
new Quadrado(20, new RenderizadorSVG()).desenhar();

Quando usar

Use quando

Há duas dimensões independentes de variação que combinadas causariam explosão de subclasses.

Composição sobre herança

Injetar o renderizador (implem.) em vez de herdar garante mais flexibilidade e testabilidade.

Runtime switch

A implementação pode ser trocada em runtime. A abstração permanece a mesma interface para o cliente.

Evite quando

Há apenas uma dimensão de variação. Strategy já resolve sem a estrutura de abstração/impl.

Bridge é conceitualmente similar a Strategy mas aplicado à separação estrutural de abstração/implementação. Veja Decorator para adicionar comportamento sem mudar estrutura.

"Bridge troca N×M subclasses por N+M classes — composição vence herança quando há múltiplos eixos de variação."
