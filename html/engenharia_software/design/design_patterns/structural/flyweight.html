Compartilhar estado intrínseco para reduzir memória

Flyweight

Estrutural — GoF — otimização de memória

Flyweight usa compartilhamento para suportar grandes quantidades de objetos granulares de forma eficiente. Divide o estado do objeto em intrínseco (compartilhado, imutável) e extrínseco (passado na chamada, único por instância).

Estado intrínseco vs extrínseco

Tipo Definição Exemplo (jogo 2D)
Intrínseco Compartilhado entre todas as instâncias. Imutável. Textura do sprite, cor da árvore, fonte do caracter
Extrínseco Passado na chamada. Único por contexto/instância. Posição X/Y, escala, opacidade

Exemplo — renderizador de árvores

// FLYWEIGHT — estado intrínseco compartilhado
class TipoArvore {
  constructor(
    readonly nome: string,
    readonly cor: string,
    readonly textura: ImageBitmap // pesado, compartilhado
  ) {}

  renderizar(x: number, y: number, escala: number): void {
    // estado extrínseco passado como parêmetro
    console.log(`Renderizar ${this.nome} em (${x},${y}) escala ${escala}`);
  }
}

// FLYWEIGHT FACTORY — garante compartilhamento
class FabricaTipoArvore {
  private pool = new Map<string, TipoArvore>();

  obter(nome: string, cor: string, textura: ImageBitmap): TipoArvore {
    const chave = `${nome}_${cor}`;
    if (!this.pool.has(chave)) {
      this.pool.set(chave, new TipoArvore(nome, cor, textura));
    }
    return this.pool.get(chave)!;
  }
}

// CLIENTE — cada árvore no mapa guarda só posição + ref ao flyweight
class Arvore {
  constructor(
    private x: number,
    private y: number,
    private tipo: TipoArvore // ref compartilhada!
  ) {}

  draw() { this.tipo.renderizar(this.x, this.y, 1.0); }
}

// 10.000 árvores, mas apenas 3 TipoArvore na memória

Quando usar

Use quando

Grande número de objetos similares consome memória excessiva. Jogos, editores de texto (caracteres), servidores de renderização.

Imutabilidade obrigatória

O estado intrínseco DEVE ser imutável. Se um flyweight mutar, todos que o compartilham são afetados.

Invisível para o cliente

Flyweight é uma otimização de implementação. A factory esconde o compartilhamento; o cliente não precisa saber.

Evite quando

Poucos objetos ou objetos sem estado compartilhável. A complexidade de separar intrínseco/extrínseco não compensa.

Flyweight é usado internamente em Java String.intern(), Python string interning e editores de texto. Veja Prototype para cópias ao invés de compartilhamento.

"Flyweight troca memória por complexidade de idéntica. Só otimize quando o custo de memória for comprovado."
