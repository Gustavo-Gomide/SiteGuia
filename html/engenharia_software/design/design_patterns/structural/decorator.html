Adicionar comportamento em runtime sem modificar a classe

Decorator

Estrutural — GoF

Decorator envolve um objeto com outro que tem a mesma interface, acrescentando comportamento antes ou depois de delegar a chamada. Permite compor comportamentos como camadas sem subclasses e sem modificar o objeto original.

Decorator vs herança

Aspecto Herança Decorator
Momento Compilação (estático). Runtime (dinâmico).
Combinação Explosão de subclasses para cada combinação. Decorators se encadeiam livremente.
OCP Adicionar comportamento = modificar hierarquia. Adicionar comportamento = nova classe decorator.

Exemplo — logging e caching em repositório

interface PedidoRepo {
  buscar(id: string): Promise<Pedido>;
  salvar(pedido: Pedido): Promise<void>;
}

// Impl. base
class PedidoRepoPostgres implements PedidoRepo {
  async buscar(id: string) { /* query */ return {} as Pedido; }
  async salvar(pedido: Pedido) { /* insert/update */ }
}

// DECORATOR: adiciona logging
class PedidoRepoComLog implements PedidoRepo {
  constructor(private repo: PedidoRepo, private log: Logger) {}

  async buscar(id: string): Promise<Pedido> {
    this.log.info(`Buscando pedido ${id}`);
    const pedido = await this.repo.buscar(id); // delega
    this.log.info(`Pedido ${id} encontrado`);
    return pedido;
  }

  async salvar(pedido: Pedido): Promise<void> {
    this.log.info(`Salvando pedido ${pedido.id}`);
    await this.repo.salvar(pedido);
  }
}

// DECORATOR: adiciona cache
class PedidoRepoComCache implements PedidoRepo {
  private cache = new Map<string, Pedido>();
  constructor(private repo: PedidoRepo) {}

  async buscar(id: string): Promise<Pedido> {
    if (this.cache.has(id)) return this.cache.get(id)!;
    const pedido = await this.repo.buscar(id);
    this.cache.set(id, pedido);
    return pedido;
  }

  async salvar(p: Pedido) { this.cache.delete(p.id); await this.repo.salvar(p); }
}

// Composicião: cache + log + postgres
const repo = new PedidoRepoComCache(
  new PedidoRepoComLog(
    new PedidoRepoPostgres(), logger
  )
);

Quando usar

Use quando

Comportamentos são combinaveis: logging, cache, retry, circuit breaker, auditoria sobre qualquer repositório.

Encadeamento

Decorators se encadeiam na ordem do wrapping. A ordem importa — Cache antes de Log vs Log antes de Cache são diferentes.

Annotations/TypeScript

O síntax de decorator (@Cache, @Log) é Decorator pattern implementado como ação em meta-nível sobre classes.

Evite quando

Apenas um comportamento e sem composição. Subclasse ou função wrapper simples pode ser mais legível.

Decorator é base da cadeia de middlewares e AOP. Veja Chain of Responsibility para sequências onde cada handler decide parar ou passar.

"Decorator é composição de comportamentos — cada camada adiciona uma responsabilidade sem saber das outras."
