Tratar objetos individuais e compostos da mesma forma

Composite

Estrutural — GoF

Composite compõe objetos em árvores e permite que o cliente trate tanto folhas (objetos simples) quanto galhos (compostos) de forma uniforme. A interface é a mesma — o cliente não distingue entre um arquivo e uma pasta.

Estrutura da árvore

Tipo Função Exemplo
Component Interface comum entre folhas e compostos. interface ItemArvorquivo
Leaf Objeto sem filhos. Implementa a operação diretamente. Arquivo com seus bytes
Composite Contão de Components. Delega a operação a cada filho. Pasta com lista de filhos

Exemplo — sistema de arquivos

interface ItemArquivo {
  nome: string;
  tamanho(): number;
  exibir(indent?: string): void;
}

class Arquivo implements ItemArquivo {
  constructor(public nome: string, private bytes: number) {}
  tamanho() { return this.bytes; }
  exibir(indent = '') { console.log(`${indent}[arquivo] ${this.nome} (${this.bytes}B)`); }
}

class Pasta implements ItemArquivo {
  private filhos: ItemArquivo[] = [];

  constructor(public nome: string) {}

  adicionar(item: ItemArquivo): void { this.filhos.push(item); }

  tamanho(): number {
    return this.filhos.reduce((s, f) => s + f.tamanho(), 0);
  }

  exibir(indent = ''): void {
    console.log(`${indent}[pasta] ${this.nome}/`);
    this.filhos.forEach(f => f.exibir(indent + '  '));
  }
}

// Uso — cliente trata Arquivo e Pasta da mesma forma
const raiz = new Pasta('src');
raiz.adicionar(new Arquivo('main.ts', 1200));
const utils = new Pasta('utils');
utils.adicionar(new Arquivo('helper.ts', 400));
utils.adicionar(new Arquivo('validator.ts', 600));
raiz.adicionar(utils);

raiz.exibir();
// [pasta] src/
//   [arquivo] main.ts (1200B)
//   [pasta] utils/
//     [arquivo] helper.ts (400B)
console.log(raiz.tamanho()); // 2200

Quando usar

Use quando

Estrutura hierárquica (menus, árvores de categoria, filesystems, CPDs de UI) onde folhas e compostos devem ser tratados uniformemente.

Recursividade

Operações como tamanho(), render(), exportar() propagam naturalmente pela árvore via delegação recursiva.

Combine com Visitor

Visitor percorre a árvore Composite e aplica operações sem modificar as classes dos nós.

Evite quando

A estrutura não é hierárquica. Uma lista simples e função recursiva pode ser mais legível.

Composite combina bem com Visitor (percorrer árvore) e Iterator (iterar uniforme sobre qualquer subárvore).

"Composite torna a diferença entre uno e múltiplo irrelevante para o cliente — a interface é a mesma."
