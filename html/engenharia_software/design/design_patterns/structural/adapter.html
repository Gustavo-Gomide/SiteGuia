Traduzir a interface de um objeto para a que o cliente espera

Adapter

Estrutural — GoF — o mais usado para integrações

Adapter converte a interface de uma classe em outra que o cliente espera. É a solução clássica para integrar APIs externas ou legadas sem poluir o domínio com detalhes da implementação. Na Arquitetura Hexagonal, os adaptadores são exatamente isso.

Participantes

Participante Função Exemplo
Target Interface que o cliente usa. EmailGateway { enviar(para, assunto, corpo) }
Adaptee Classe com interface incompativel (API externa, legada). SendGrid { sendEmail(config) }
Adapter Impl. Target; traduz chamadas para o Adaptee. SendGridAdapter implements EmailGateway

Exemplo — adapter de email externo

// TARGET — interface do domínio
interface EmailGateway {
  enviar(para: string, assunto: string, corpo: string): Promise<void>;
}

// ADAPTEE — SDK externo (interface que não controlamos)
class SendGridClient {
  async sendEmail(config: {
    to: string;
    subject: string;
    htmlBody: string;
    from: string;
    apiKey: string;
  }): Promise<void> {
    // chamada real ao SendGrid
  }
}

// ADAPTER — traduz o domínio para o SDK
class SendGridAdapter implements EmailGateway {
  constructor(
    private client: SendGridClient,
    private apiKey: string,
    private remetente: string
  ) {}

  async enviar(para: string, assunto: string, corpo: string): Promise<void> {
    await this.client.sendEmail({
      to: para,
      subject: assunto,
      htmlBody: corpo,
      from: this.remetente,
      apiKey: this.apiKey,
    });
  }
}

// Domínio só conhece EmailGateway — trocar SendGrid por outro = nova classe Adapter

Quando usar

Use quando

Integraçêo de API externa, SDK legado, ou qualquer código com interface incompatível. Forma a base da Arquitetura Hexagonal.

Testabilidade

Com Adapter, o domínio depende de interface. Testes usam um fake da interface, não da API real.

Isolamento de mudança

Mudança na API externa afeta apenas o Adapter. O domínio e os testes ficam estáveis.

vs Facade

Facade simplifica uma interface complexa. Adapter converte uma interface incompativel. Às vezes se combinam.

Adapter é o padrão central da Arquitetura Hexagonal. Veja também Facade para simplificação de subsistemas complexos.

"Adapter protege o domínio de detalhes externos. O domínio fala sua língua — o adaptador traduz para a língua da infraestrutura."
