Iterator

Comportamental — GoF

Iterator define um protocolo para percorrer uma coleção (hasNext(), next()) sem expor se ela é uma lista, árvore, grafo ou qualquer outra estrutura. O cliente sempre usa a mesma interface independente da estrutura interna.

Protocolo e variantes

Tipo Descrição Exemplo
Iterator simples hasNext() + next() linear. Lista, fila, pilha
Iterator reverso Percurso de trás para frente. Histórico de undo
Iterator de árvore BFS ou DFS configurável. Navegação em AST, filesystems
Iterator lazy Gera elementos sob demanda (generator). Streams, resultados de banco paginado

Exemplo — iterator de árvore

class TreeNode {
  constructor(
    public value: number,
    public left: TreeNode | null = null,
    public right: TreeNode | null = null
  ) {}
}

class InOrderIterator {
  private stack: TreeNode[] = [];

  constructor(root: TreeNode | null) {
    this.pushLeft(root);
  }

  private pushLeft(node: TreeNode | null): void {
    while (node) {
      this.stack.push(node);
      node = node.left;
    }
  }

  hasNext(): boolean {
    return this.stack.length > 0;
  }

  next(): number {
    const node = this.stack.pop()!;
    this.pushLeft(node.right);
    return node.value;
  }
}

// Uso — cliente não sabe que é uma árvore
const it = new InOrderIterator(root);
while (it.hasNext()) {
  console.log(it.next()); // imprime em ordem crescente
}

Quando usar

Múltiplas estruturas

Quando diferentes classes precisam ser percorríveis de forma uniforme: listas, árvores, grafos, resultados de query.

Múltiplas ordens

BFS, DFS, in-order, reverse — cada um é um Iterator diferente sobre a mesma coleção.

Já embutido

Na maioria das linguagens modernas, Iterator é nativo (JavaScript Symbol.iterator, Java Iterable, Python __iter__).

Evite quando

A coleção é simples e o for nativo resolve. Só crie um Iterator customizado quando há estrutura especial.

Iterator é a base dos generators em JavaScript/Python. Veja Composite para estruturas em árvore que combinam bem com iterators customizados.

"Iterator desacopla o algoritmo de percurso da estrutura de dados — o cliente não precisa saber o que está percorrendo."
