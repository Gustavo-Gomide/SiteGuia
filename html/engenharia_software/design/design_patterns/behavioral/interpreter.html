Interpreter

Comportamental — GoF

Interpreter representa uma gramática de linguagem como uma hierarquia de classes. Cada expressão da linguagem é um objeto que sabe se interpretar. É usado em expressões regulares, buscas estruturadas, regras de negócio configuráveis e configurações declarátivas.

Estrutura

Tipo Função Exemplo
Terminal Expressão folha — interpreta um valor concreto. Número literal, variável, constante
NonTerminal Expressão composta — agrega outros nós e os interpreta. Soma, E, Ou, Negação
Contexto Informação global para a interpretação (variáveis, estado). Map de variáveis para valores

Exemplo — regra de elegibilidade

interface Expressao {
  interpretar(ctx: Record<string, boolean>): boolean;
}

class Variavel implements Expressao {
  constructor(private nome: string) {}
  interpretar(ctx: Record<string, boolean>) {
    return ctx[this.nome] ?? false;
  }
}

class E implements Expressao {
  constructor(private esq: Expressao, private dir: Expressao) {}
  interpretar(ctx: Record<string, boolean>) {
    return this.esq.interpretar(ctx) && this.dir.interpretar(ctx);
  }
}

class Ou implements Expressao {
  constructor(private esq: Expressao, private dir: Expressao) {}
  interpretar(ctx: Record<string, boolean>) {
    return this.esq.interpretar(ctx) || this.dir.interpretar(ctx);
  }
}

// Regra: (ehVip OU ehBulk) E pagamentoConfirmado
const regra = new E(
  new Ou(new Variavel('ehVip'), new Variavel('ehBulk')),
  new Variavel('pagamentoConfirmado')
);

console.log(regra.interpretar({ ehVip: true, pagamentoConfirmado: true })); // true
console.log(regra.interpretar({ ehBulk: false, pagamentoConfirmado: true })); // false

Quando usar e quando evitar

Use quando

A gramática é simples e estável. Regras de negócio precisam ser compostas dinamicamente (ex: permissões configuráveis).

Evite quando

A gramática é complexa. Parsers reais (ANTLR, PEG.js) são mais robústo para linguagens não triviais.

Na prática

SQL parsers, expressões regulares, rule engines, filtros de query são casos clássicos de Interpreter.

Alternativa

Para gramáticas grandes, use Strategy com parâmetros ou um visitor sobre uma árvore gerada por um parser externo.

Interpreter é menos comum que outros padrões comportamentais. Para fluxos mais simples, Strategy ou Chain of Responsibility geralmente são suficientes.

"Interpreter brilha quando as regras mudam com mais frequência do que o código — configuração como código."
