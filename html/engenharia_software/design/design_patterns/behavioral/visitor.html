Adicionar operações a estruturas sem modificar suas classes

Visitor

Comportamental — GoF

Visitor separa algoritmos da estrutura do objeto. Cada elemento aceita um visitor; o visitor decide o que fazer com cada tipo de elemento. Isso permite adicionar novas operações sem modificar as classes dos elementos — aplica OCP para atuações sobre estruturas.

Protocolo de double dispatch

Participante Função Exemplo
Element Interface com accept(visitor). Nó de AST, componente UI
ConcreteElement Chama visitor.visitTipo(this) no accept. NumeroNode, SomaNode, ProdutoNode
Visitor Interface com um método visit por tipo de elemento. AvaliarVisitor, ImprimirVisitor

Exemplo — AST de expressões

interface Visitor {
  visitNumero(n: NumeroNode): number;
  visitSoma(n: SomaNode): number;
  visitProduto(n: ProdutoNode): number;
}

interface NodeExpr {
  accept(v: Visitor): number;
}

class NumeroNode implements NodeExpr {
  constructor(readonly valor: number) {}
  accept(v: Visitor) { return v.visitNumero(this); }
}

class SomaNode implements NodeExpr {
  constructor(readonly esq: NodeExpr, readonly dir: NodeExpr) {}
  accept(v: Visitor) { return v.visitSoma(this); }
}

class ProdutoNode implements NodeExpr {
  constructor(readonly esq: NodeExpr, readonly dir: NodeExpr) {}
  accept(v: Visitor) { return v.visitProduto(this); }
}

// Visitor concreto: avaliação
class AvaliarVisitor implements Visitor {
  visitNumero(n: NumeroNode)   { return n.valor; }
  visitSoma(n: SomaNode)       { return n.esq.accept(this) + n.dir.accept(this); }
  visitProduto(n: ProdutoNode) { return n.esq.accept(this) * n.dir.accept(this); }
}

// (2 + 3) * 4
const ast = new ProdutoNode(
  new SomaNode(new NumeroNode(2), new NumeroNode(3)),
  new NumeroNode(4)
);
console.log(ast.accept(new AvaliarVisitor())); // 20

Quando usar e quando evitar

Use quando

A hierarquia de tipos é estável mas as operações mudam. Compiladores, ASTs, relatores sobre estruturas de dados.

Evite quando

O conjunto de tipos muda frequentemente — adicionar um novo tipo exige atualizar todos os visitors existentes.

Double dispatch

O despacho duplo (accept no elemento + visit no visitor) garante que o visitor certo processa o tipo certo.

vs Interpreter

Interpreter define interpretação dentro do objeto. Visitor externaliza a operação. Para operações múltiplas, Visitor escala melhor.

Visitor é muito usado em compiladores e transformadores de AST. Veja Composite — estruturas em árvore que Visitor frequentemente percorre.

"Visitor resolve o dilema entre tipos estáveis e operações dinâmicas — adicione comportamentos sem tocar nos elementos."
