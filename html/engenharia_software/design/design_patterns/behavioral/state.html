Comportamento muda conforme o estado interno

State

Comportamental — GoF

State permite que um objeto altere seu comportamento quando o estado interno muda, fazendo parecer que a classe mudou. Substitui if/switch de estado espalhados por classes de estado que encapsulam transições e comportamentos.

Exemplo — máquina de estado de pedido

Cada estado sabe quais transições são válidas a partir dele:

Estado confirmar() cancelar() entregar()
Rascunho → Confirmado → Cancelado Erro
Confirmado Erro → Cancelado → Entregue
Entregue Erro Erro Erro
Cancelado Erro Erro Erro

Implementação

interface EstadoPedido {
  confirmar(pedido: Pedido): void;
  cancelar(pedido: Pedido): void;
  entregar(pedido: Pedido): void;
}

class EstadoRascunho implements EstadoPedido {
  confirmar(pedido: Pedido): void {
    pedido.estado = new EstadoConfirmado();
  }
  cancelar(pedido: Pedido): void {
    pedido.estado = new EstadoCancelado();
  }
  entregar(): void {
    throw new Error('Pedido em rascunho não pode ser entregue.');
  }
}

class EstadoConfirmado implements EstadoPedido {
  confirmar(): void { throw new Error('Já confirmado.'); }
  cancelar(pedido: Pedido): void { pedido.estado = new EstadoCancelado(); }
  entregar(pedido: Pedido): void { pedido.estado = new EstadoEntregue(); }
}

class Pedido {
  estado: EstadoPedido = new EstadoRascunho();

  confirmar() { this.estado.confirmar(this); }
  cancelar()  { this.estado.cancelar(this); }
  entregar()  { this.estado.entregar(this); }
}

Quando usar

Use quando

O comportamento varia por estado e o número de estados é conhecido. Pedidos, conexões, workflow, players de áudio/vídeo.

Elimina if/switch

Em vez de if (status === 'confirmado') {...} espalhado, cada comportamento fica encapsulado na classe de estado.

vs Strategy

Strategy troca algoritmo externamente. State troca comportamento internamente baseado em transições de estado.

Evite quando

Poucos estados simples — um enum + switch pode ser mais legível. State vale a pena quando as transições são complexas.

State é muito usado em jogos (estado do personagem) e workflows. Veja Strategy para troca de algoritmo externo sem máquina de estado.

"State distribui a máquina de estados entre classes — cada estado sabe para onde pode ir, não o objeto central."
