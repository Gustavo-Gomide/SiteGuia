Handlers em cadeia — cada um decide parar ou passar

Chain of Responsibility

Comportamental — GoF

Chain of Responsibility encadeia handlers. Cada handler recebe a requisição, decide se trata ou passa ao próximo. Isso elimina if/else longos e torna a cadeia configurável sem alterar os handlers existentes.

Estrutura e contexto

Cada handler tem uma responsabilidade e redireciona para o próximo quando não trata:

Parte Função Exemplo
Handler (interface) Define handle(req) e setNext(). Middleware, Validador
Handler concreto Trata se capaz, caso contrário delega ao próximo. AuthMiddleware, RateLimitMiddleware
Cliente Monta a cadeia e dispara a requisição no primeiro handler. App.use(auth).use(rateLimit)...

Exemplo — pipeline de middleware

Cada middleware decide processar ou rejeitar antes de passar adiante:

interface Handler {
  setNext(handler: Handler): Handler;
  handle(req: Request): Response | null;
}

class AuthMiddleware implements Handler {
  private next: Handler | null = null;

  setNext(handler: Handler): Handler {
    this.next = handler;
    return handler;
  }

  handle(req: Request): Response | null {
    if (!req.headers.authorization) {
      return { status: 401, body: 'Não autorizado' };
    }
    // Passou a autenticação, passa ao próximo
    return this.next?.handle(req) ?? null;
  }
}

class RateLimitMiddleware implements Handler {
  handle(req: Request): Response | null {
    if (this.excedeuLimite(req.ip)) {
      return { status: 429, body: 'Muitas requisições' };
    }
    return this.next?.handle(req) ?? null;
  }
}

// Montagem da cadeia
const auth = new AuthMiddleware();
auth.setNext(new RateLimitMiddleware()).setNext(new LogMiddleware());
const resultado = auth.handle(request);

Quando usar e quando não usar

Use quando

Há múltiplos handlers possíveis e a ordem importa. Middlewares HTTP, pipelines de validação, filtros de log.

Configurabilidade

A cadeia pode ser montada dinamicamente (em runtime) sem alterar os handlers já escritos.

Cuidado

Se a requisição chegar ao final sem ser tratada, o comportamento deve estar definido (null, exception, default).

Evite quando

Apenas um handler vai tratar — nesse caso, Strategy simples resolve sem a estrutura de cadeia.

Chain of Responsibility é a base dos middlewares em Express, Django e .NET. Veja também Command para encapsular operações como objetos.

"A cadeia é flexível quanto à composição, não quanto ao fluxo. Defina bem o que acontece quando nenhum handler trata."
