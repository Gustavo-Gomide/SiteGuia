Mediator

Comportamental — GoF

Mediator coloca as regras de interação entre componentes em um objeto mediador. Os componentes (colegas) não se conhecem diretamente — só notificam o mediador. Isso reduz acoplamento em sistemas com muitas interações complexas entre componentes.

Antes e depois

Cada componente depende de vários outros Lógica centralizada no mediador

Exemplo — formulário com validação

interface Mediator {
  notify(sender: Componente, evento: string): void;
}

abstract class Componente {
  constructor(protected mediator: Mediator) {}

  mudar(): void {
    this.mediator.notify(this, 'changed');
  }
}

class CampoEmail extends Componente {}
class CampoSenha extends Componente {}
class BotaoLogin extends Componente {
  habilitado = false;

  habilitar(v: boolean) { this.habilitado = v; }
}

class FormularioLoginMediator implements Mediator {
  constructor(
    private email: CampoEmail,
    private senha: CampoSenha,
    private botao: BotaoLogin,
    private getValor: (c: Componente) => string
  ) {}

  notify(sender: Componente, evento: string): void {
    if (evento === 'changed') {
      const emailValido = this.getValor(this.email).includes('@');
      const senhaValida = this.getValor(this.senha).length >= 6;
      // Só o mediador sabe a lógica de habilitar o botão
      this.botao.habilitar(emailValido && senhaValida);
    }
  }
}

Quando usar e cuidados

Use quando

Múltiplos componentes interagem e o acoplamento está crescendo. Workflows, formulários complexos, UIs de estados.

Centralização de lógica

O mediador concentra a lógica de coordenação. Isso facilita o teste do fluxo completo em um único lugar.

Mediador gordo

Se o mediador crescer demais, ele vira um God Object. Divida por contexto se a responsabilidade crescer.

vs Observer

Observer é muitos-para-muitos distribuído. Mediator é muitos-para-um central. Use cada um no contexto certo.

Mediator é conceitualmente semelhante a um event bus centralizado. Veja Observer para distribuição descentralizada de eventos.

"O mediador assume a responsabilidade de orquestrar interacões — os colegas ficam livres de saber uns dos outros."
