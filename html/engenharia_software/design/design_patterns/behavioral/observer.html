Inscritos notificados quando o estado muda

Observer

Comportamental — GoF — base de sistemas reativos e event-driven

Observer define uma relação um-para-muitos: quando o Subject muda de estado, todos os Observers inscritos são notificados automaticamente. É a base de event listeners, sistemas reativos (RxJS, MobX) e event-driven architecture.

Subject e Observers

Participante Função Exemplo
Subject Mantém lista de observers, notifica na mudança. EventEmitter, Store, Sensor
Observer Recebe update() quando o subject muda. UI Component, Logger, Alert
Evento/Dado Informação enviada na notificação (push) ou consultada (pull). novo valor, tipo de evento

Exemplo — EventEmitter simples

type Listener<T> = (data: T) => void;

class EventEmitter<Events extends Record<string, unknown>> {
  private listeners: Partial<Record<keyof Events, Listener<unknown>[]>> = {};

  on<K extends keyof Events>(evento: K, fn: Listener<Events[K]>): void {
    if (!this.listeners[evento]) this.listeners[evento] = [];
    (this.listeners[evento] as Listener<unknown>[]).push(fn as Listener<unknown>);
  }

  off<K extends keyof Events>(evento: K, fn: Listener<Events[K]>): void {
    this.listeners[evento] = (this.listeners[evento] ?? []).filter(l => l !== fn) as Listener<unknown>[];
  }

  emit<K extends keyof Events>(evento: K, data: Events[K]): void {
    (this.listeners[evento] ?? []).forEach(fn => fn(data));
  }
}

// Uso tipado
type PedidoEvents = { confirmado: { pedidoId: string }; cancelado: { pedidoId: string; motivo: string } };

const emitter = new EventEmitter<PedidoEvents>();
emitter.on('confirmado', ({ pedidoId }) => console.log(`Pedido ${pedidoId} confirmado`));
emitter.emit('confirmado', { pedidoId: '123' });

Quando usar e cuidados

Use quando

Um objeto muda e outros precisam reagir sem acoplamento direto. Event listeners, state management, notificações.

Memory leak

Observers que não fazem off() ficam retidos. Sempre cancele subscriptions quando o componente for destruído.

Ordem de notificação

A ordem em que os observers são notificados geralmente não deve importar. Se importar, use Mediator.

Debugging

Fluxos com muitos observers podem ser difíceis de rastrear. Use nomes de eventos descritivos e logging estruturado.

Observer é o padrão por trás de React state, RxJS Observables e event-driven architecture. Veja Event-Driven Architecture para a versão distribuída.

"Observer inverte o controle: o producer não precisa saber quem vai reagir — quem quer reagir se inscreve."
