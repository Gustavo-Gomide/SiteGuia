Esqueleto do algoritmo na classe base, detalhes nas subclasses

Template Method

Comportamental — GoF

Template Method define a estrutura de um algoritmo na classe base, delega etapas variáveis para subclasses. A ordem e o esqueleto são fixos; apenas os pontos de variação precisam ser sobrescritos. Prefira Strategy quando possível, pois Template Method depende de herança.

Template Method vs Strategy

Aspecto Template Method Strategy
Mecanismo Herança — subclasse sobrescreve métodos. Composição — injeta objeto de estratégia.
Acoplamento Forte — subclasse depende da classe base. Fraco — contexto depende apenas da interface.
Runtime troca Não — tipo fixado em compilação. Sim — estrategia pode ser trocada em runtime.
Preferência Use quando herança já faz sentido no domínio. Prefira quando flex. de troca é necessária.

Exemplo — gerador de relatórios

abstract class GeradorRelatorio {
  // Template Method — esqueleto fixo
  gerar(): string {
    const cabecalho = this.gerarCabecalho();
    const corpo    = this.gerarCorpo();
    const rodape   = this.gerarRodape();
    return `${cabecalho}\n${corpo}\n${rodape}`;
  }

  protected abstract gerarCabecalho(): string;
  protected abstract gerarCorpo(): string;

  // Hook — opcional, subclasse pode sobrescrever
  protected gerarRodape(): string {
    return `--- Gerado em ${new Date().toISOString()} ---`;
  }
}

class RelatorioHTML extends GeradorRelatorio {
  protected gerarCabecalho() { return '<h1>Relatório</h1>'; }
  protected gerarCorpo()    { return '<table>...</table>'; }
}

class RelatorioCSV extends GeradorRelatorio {
  protected gerarCabecalho() { return 'nome,valor,data'; }
  protected gerarCorpo()    { return '...linhas CSV...'; }
  protected gerarRodape()   { return ''; } // hook sobrescrito
}

Quando usar

Use quando

A estrutura do algoritmo é estável. Variações são bem delimitadas. Frameworks com pontos de extensão (hooks).

Hooks opcionais

Etapas não-abstratas (com implementação padrão) são hooks — subclasse sobrescreve apenas se precisar.

Cuidado com herança

Não adicione estado à subclasse que não faz sentido na base. Herança profunda = acoplamento forte.

Prefira Strategy

Se a variação pode ser extraida como objeto, Strategy via composição é mais flexível e testável.

Template Method é base de muitos frameworks (JUnit, Hibernate, Spring). Para variáveis externas, prefira Strategy.

"Defina o esqueleto uma vez, deixe os detalhes para quem especializa. O framework controla o fluxo — você preenche os espaços."
