Algoritmos intercambiáveis encapsulados em classes

Strategy

Comportamental — GoF — um dos mais usados na prática

Strategy define uma família de algoritmos, encapsula cada um e os torna intercambiáveis. O cliente escolhe (ou injeta) a estratégia; o contexto a executa sem saber os detalhes. Elimina if/else de variação e aplica diretamente o OCP (aberto para extensão, fechado para modificação).

Antes e depois

Sem Strategy Com Strategy
if (tipo === 'vip') ... else if (tipo === 'bulk') ... Cada tipo é uma classe que implementa DescontoStrategy
Adicionar novo tipo = mexer na função existente Adicionar novo tipo = nova classe isolada
Difícil de testar cada variação isoladamente Cada estratégia testada em unit test separado

Exemplo — estratégias de desconto

interface DescontoStrategy {
  calcular(valorBruto: number): number;
}

class DescontoVip implements DescontoStrategy {
  calcular(valor: number) { return valor * 0.2; }
}

class DescontoBulk implements DescontoStrategy {
  constructor(private quantidade: number) {}
  calcular(valor: number) {
    return this.quantidade >= 10 ? valor * 0.15 : valor * 0.05;
  }
}

class SemDesconto implements DescontoStrategy {
  calcular() { return 0; }
}

// Contexto usa a estratégia injetada
class Pedido {
  constructor(
    private items: { preco: number }[],
    private desconto: DescontoStrategy
  ) {}

  calcularTotal(): number {
    const bruto = this.items.reduce((s, i) => s + i.preco, 0);
    return bruto - this.desconto.calcular(bruto);
  }
}

// Uso — troca de estratégia sem alterar Pedido
const pedido1 = new Pedido(items, new DescontoVip());
const pedido2 = new Pedido(items, new DescontoBulk(15));
const pedido3 = new Pedido(items, new SemDesconto());

Quando usar

Use quando

Há variação real de algoritmo (cálculo, ordenação, validação, pagamento). As variantes são intercambiáveis na mesma interface.

Testabilidade

Cada estratégia é uma classe isolada e testavel. O Contexto também é testável com qualquer fake strategy.

vs Template Method

Strategy usa composição (injeta objeto). Template Method usa herança (sobrescreve métodos). Prefira Strategy.

Evite quando

Apenas uma implementação existe. Uma função de alta ordem (sort((a,b) => a-b)) já resolve sem criar interface.

Strategy é o padrão comportamental mais usado. Combine com SOLID (OCP + DIP) para maximizar extensibilidade.

"Strategy transforma variação de algoritmo em ponto de extensão explícito — sem if/else, sem modificação."
