Criar objetos clonando protótipos existentes

Prototype

Criacional — GoF

Prototype cria novos objetos copiando (clonando) um objeto existente (protótipo). É útil quando a criação direta é cara (configuração pesada), quando a classe exata não é conhecida em runtime, ou quando se precisa de cópias independentes com pequenas variações.

Cópia rasa vs profunda

Tipo O que copia Risco
Shallow clone Propriedades primitivas copiadas; objetos internos compartilhados. Mutar sub-objeto afeta original e clone.
Deep clone Toda a árvore de objetos é duplicada. Mais caro; cuidado com ciclos e referências externas.
Clone semântico Clone manual que decide o que copiar/compartilhar por domínio. Mais controle; requer implementação explícita.

Exemplo — clonagem de configurações

interface Clonavel<T> {
  clonar(): T;
}

class ConfiguracaoRelatorio implements Clonavel<ConfiguracaoRelatorio> {
  constructor(
    public titulo: string,
    public colunas: string[],
    public filtros: Record<string, string>
  ) {}

  clonar(): ConfiguracaoRelatorio {
    return new ConfiguracaoRelatorio(
      this.titulo,
      [...this.colunas],            // copia do array (deep)
      { ...this.filtros }           // copia do objeto (shallow nos valores)
    );
  }
}

// Protótipo base
const relatorioBase = new ConfiguracaoRelatorio(
  'Vendas',
  ['data', 'produto', 'valor'],
  { status: 'ativo' }
);

// Clone com variação — sem afetar o original
const relatorioMes = relatorioBase.clonar();
relatorioMes.titulo = 'Vendas de Janeiro';
relatorioMes.filtros['mes'] = '01/2025';

console.log(relatorioBase.titulo);  // 'Vendas' — não afetado
console.log(relatorioMes.titulo);   // 'Vendas de Janeiro'

Quando usar

Use quando

Criar objetos do zero é caro (I/O, configuração pesada). Precisa de cópias independentes com pequenas variações.

Test Data Builder

Em testes, Prototype permite criar um objeto base válido e clonar com variações por teste.

Cuidado com deep clone

Cópia rasa de objetos com referências aninhadas pode gerar compartilhamento inesperado de estado.

Evite quando

O objeto é simples o suficiente para criar diretamente com new ou literal. Clonagem adiciona complexidade sem benefício.

Prototype é nativo em JavaScript (Object.assign, spread, structuredClone). Veja Builder quando a construção por etapas for mais clara.

"Prototype externaliza a criação por cópia — o protótipo carrega o estado padrão, o clone tem sua própria vida."
