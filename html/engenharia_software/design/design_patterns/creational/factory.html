Subclasse decide qual objeto concreto criar

Factory Method

Criacional — GoF

Factory Method define uma interface para criar um objeto, mas delega às subclasses a decisão de qual classe concreta instanciar. Isso desacopla a criação de objetos do código que os usa. Na prática do dia a dia, funções fábrica simples resolvem bem sem precisar de herança.

Variações práticas

Variação Mecanismo Quando usar
Função fábrica simples Função que retorna instância baseado em parâmetro. Criação simples, sem hier. de subclasses.
Factory Method (GoF) Método na classe base sobrescrito nas subclasses. Frameworks — base define fluxo, sub decide produto.
Static factory Método estático com nome descritivo (fromEmail()). Construtores com múltiplas entradas ou nomes semânticos.

Exemplo — fábrica de notificações

interface Notificador {
  enviar(destinatario: string, mensagem: string): void;
}

class NotificadorEmail implements Notificador {
  enviar(dest: string, msg: string) {
    console.log(`Email para ${dest}: ${msg}`);
  }
}

class NotificadorSMS implements Notificador {
  enviar(dest: string, msg: string) {
    console.log(`SMS para ${dest}: ${msg}`);
  }
}

class NotificadorPush implements Notificador {
  enviar(dest: string, msg: string) {
    console.log(`Push para ${dest}: ${msg}`);
  }
}

// Função fábrica simples (prática mais comum)
function criarNotificador(tipo: 'email' | 'sms' | 'push'): Notificador {
  switch (tipo) {
    case 'email': return new NotificadorEmail();
    case 'sms':   return new NotificadorSMS();
    case 'push':  return new NotificadorPush();
  }
}

// Static factory com nome semântico
class Usuario {
  static fromEmail(email: string, nome: string): Usuario {
    const u = new Usuario();
    u.email = email;
    u.nome = nome;
    return u;
  }

  static convidado(): Usuario {
    return new Usuario(); // defaults
  }
}

Quando usar

Use quando

O tipo exato do objeto depende de configuração, parâmetro ou contexto em runtime.

Static factory

Preferido sobre construtores quando o nome semântico importa: Distancia.fromKm(10) vs new Distancia(10).

Extensão

Adicionar novo tipo = nova classe + caso no switch. O código cliente não muda.

Evite quando

Apenas um tipo é criado. Inversão de dependência (injetar interface) pode ser mais simples que uma fábrica.

Para famílias de produtos compatíveis, veja Abstract Factory. Para construção passo a passo, veja Builder.

"Fábrica centraliza a decisão de qual classe instanciar — o cliente fala o que quer, não como criar."
