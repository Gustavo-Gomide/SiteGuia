Construção passo a passo de objetos complexos

Builder

Criacional — GoF

Builder separa a construção de um objeto complexo da sua representação final. Permite criar diferentes configurações do mesmo objeto passo a passo, sem construtores com muitos parâmetros ou objetos parcialmente inicializados.

Problemas que Builder resolve

Problema Sem Builder Com Builder
Muitos parâmetros new Query(table, cols, null, null, limit, null, sort) new QueryBuilder().from('pedidos').limit(10).build()
Configurações opcionais Vários construtores sobrecarregados ou parâmetros opcionais confusos. Cada método do builder é opcional; build() valida no final.
Imutabilidade Objeto mutavel com setters chamados em sequência. build() retorna objeto imutável e válido.

Exemplo — SQL Query Builder

class QueryBuilder {
  private tabela = '';
  private colunas: string[] = ['*'];
  private condicoes: string[] = [];
  private _limit: number | null = null;
  private _orderBy: string | null = null;

  from(tabela: string): this {
    this.tabela = tabela;
    return this;
  }

  select(...cols: string[]): this {
    this.colunas = cols;
    return this;
  }

  where(condicao: string): this {
    this.condicoes.push(condicao);
    return this;
  }

  limit(n: number): this {
    this._limit = n;
    return this;
  }

  orderBy(col: string): this {
    this._orderBy = col;
    return this;
  }

  build(): string {
    if (!this.tabela) throw new Error('Tabela obrigatória.');
    let sql = `SELECT ${this.colunas.join(', ')} FROM ${this.tabela}`;
    if (this.condicoes.length) sql += ` WHERE ${this.condicoes.join(' AND ')}`;
    if (this._orderBy) sql += ` ORDER BY ${this._orderBy}`;
    if (this._limit) sql += ` LIMIT ${this._limit}`;
    return sql;
  }
}

const query = new QueryBuilder()
  .from('pedidos')
  .select('id', 'total', 'status')
  .where('status = \'ativo\'')
  .orderBy('criado_em DESC')
  .limit(20)
  .build();
// SELECT id, total, status FROM pedidos WHERE status = 'ativo' ORDER BY criado_em DESC LIMIT 20

Quando usar

Use quando

Objetos com muitos parâmetros opcionais. Construção em etapas com validação no final.

Fluent interface

Devolver this em cada método permite encadeamento (method chaining) legível.

Validação centralizada

O build() valida o estado completo antes de criar — evita objetos inválidos.

Evite quando

O objeto tem poucos parâmetros bóvios. Um construtor simples ou objeto literal resolve sem o overhead do Builder.

Builder é muito usado em testes (Test Data Builder) e ORMs (QueryBuilder). Veja Factory Method para criação sem etapas.

"Builder orquestra a construção complexa — o objeto final só existe quando está completo e válido."
