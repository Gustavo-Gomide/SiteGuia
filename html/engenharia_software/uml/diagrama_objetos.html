Diagrama de Objetos

Instâncias concretas de classes em um momento específico — o "fotografia do domínio em execução".

O diagrama de objetos é o complemento prático do diagrama de classes: enquanto classes definem o tipo, objetos mostram um cenário real com valores concretos. É a melhor forma de validar se o modelo faz sentido.

Quando usar

Por ser mais simples que o diagrama de classes, é especialmente útil para comunicação com pessoas que não conhecem o código.

Situação Para que usar o diagrama de objetos Exemplo de pergunta respondida
Validar modelo de classes Instanciar com dados reais para checar se as relações fazem sentido "Um pedido pode existir sem cliente?"
Onboarding no domínio Mostrar um exemplo concreto para quem está aprendendo o negócio "Como fica um pedido com 2 itens na prática?"
Depurar entendimentos Resolver divergências entre documentação e realidade do banco "O status no banco bate com o estado modelado?"
Criar casos de teste Definir estado inicial e esperado de entidades em testes "Como montar o fixture para esse cenário de teste?"

Exemplo: pedido com dois itens

Armadilhas comuns

O diagrama de objetos é muito simples — e por isso é fácil errar o propósito e incluir detalhes que atrapalham.

Listar todos os atributos

O poder do diagrama de objetos é o foco. Mostrar 30 campos vira planilha. Inclua só os atributos que explicam o cenário.

Esquecer os links

A relação entre instâncias é o ponto central do diagrama. Objetos sem links são apenas listas de dados desconexas.

Confundir com diagrama de classes

Classe define o tipo; objeto é uma instância. Misturar os dois (colocar classe e instância no mesmo diagrama) gera confusão.

Não indicar o momento

O diagrama de objetos é um snapshot. Indicar quando foi capturado (antes/depois de um evento) torna-o muito mais informativo.

Combine com Diagrama de Classes para validar o modelo, e com a Introdução à UML para escolher o diagrama certo.

"Se o modelo de classes não consegue representar um exemplo real, o modelo está errado — não o exemplo."
