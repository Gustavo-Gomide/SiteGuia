Argo CD

GitOps no Kubernetes

Estado desejado no Git e reconciliacao continua no cluster.

Argo CD aplica o modelo pull: o cluster sincroniza a partir do repositorio, reduzindo drift operacional.

Conceitos-Chave

Application

Objeto que aponta para repositorio, path e destino de deploy.

Sync

Sincronizacao manual ou automatica entre Git e cluster.

Health

Status operacional dos recursos para auditoria rapida.

Fluxo GitOps

Desenvolvedor altera manifesto no Git, pipeline valida e Argo CD aplica no cluster conforme politica de sincronizacao.

Esse fluxo centraliza historico de mudancas e facilita rollback.
