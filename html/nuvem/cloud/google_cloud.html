A nuvem de dados, IA e ecossistema cloud-native.

Google Cloud: introducao

O GCP roda sobre a mesma base de infraestrutura usada por servicos globais do Google e e referencia em BigQuery, IA e Kubernetes.

A organizacao por projetos e a experiencia de plataforma costumam agradar times orientados a produto e dados.

Tudo em projetos

A unidade principal e o Projeto: recursos, APIs, permissoes e faturamento ficam agrupados por contexto de negocio.

Rede global privada

Grande parte do trafego entre regioes pode seguir pela backbone privada do Google, reduzindo latencia e exposicao.

Compute Engine (GCE)

Custom Machine Types

E possivel ajustar CPU e RAM de forma independente, otimizando custo para cargas que nao cabem bem em tamanhos fixos.

Live Migration

Em classes suportadas, a VM pode ser migrada em manutencao sem desligamento, reduzindo indisponibilidade operacional.

Economia

  • Spot VMs: grande desconto com risco de interrupcao.
  • Sustained Use Discounts: desconto automatico conforme uso continuo.

Cloud Functions

Cloud Functions executa funcoes orientadas a eventos sem gerenciamento de servidor, com escala automatica.

1st Gen vs 2nd Gen

A 2nd Gen, baseada em Cloud Run, amplia concorrencia, limites de tempo e controle de trafego entre versoes.

Triggers comuns

  • HTTP/Webhook
  • Cloud Storage
  • Pub/Sub
  • Firestore

Exemplo (Python)

Recebendo uma requisicao HTTP simples.

import functions_framework

@functions_framework.http
def hello_http(request):
    request_json = request.get_json(silent=True)
    request_args = request.args

    if request_json and 'name' in request_json:
        name = request_json['name']
    elif request_args and 'name' in request_args:
        name = request_args['name']
    else:
        name = 'World'

    return f'Hello {name}!'
