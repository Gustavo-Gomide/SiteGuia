P vs NP

A maior questão em aberto da Ciência da Computação Teórica.

P é a classe dos problemas que conseguimos resolver rapidamente. NP é a classe dos problemas cuja solução conseguimos verificar rapidamente. Toda solução rápida pode ser verificada rápida — portanto P ⊆ NP. Mas o contrário vale? A pergunta P = NP? permanece sem resposta desde 1971.

Definições Formais

P = ∪k≥1 TIME(nk)

Uma linguagem L ∈ P se existe uma MT determinística M e uma constante k tal que M decide L em O(nk) passos.

NP = ∪k≥1 NTIME(nk)

Equivalentemente: L ∈ NP se existe um verificador polinomial V e um polinômio p tal que: w ∈ L ⟺ ∃c, |c| ≤ p(|w|), V(w,c) aceita. O certificado c é uma "prova" de que w ∈ L.

Ser P ⊆ NP é trivial: um decisor determinístico já é um verificador que ignora o certificado.

Teorema de Cook-Levin

Em 1971, Stephen Cook e (independentemente) Leonid Levin provaram que SAT é NP-completo — o primeiro resultado desse tipo. A prova mostra como simular qualquer NDTM polinomial via uma fórmula booleana em CNF.

SAT = { <φ> | φ é uma fórmula booleana satisfazível }

Dado qualquer problema L ∈ NP, existe uma redução polinomial de L para SAT. Portanto: se SAT ∈ P então P = NP.

A partir de Cook-Levin, centenas de outros problemas foram provados NP-completos por reduções de SAT (ou de variantes como 3-SAT). Cada redução herda a NP-dureza.

Impacto Prático

Embora NP-completude seja um resultado do pior caso, ela moldou toda a prática computacional:

  • Criptografia: RSA, criptografia de curva elíptica e hashing se baseiam em funções conjecturadas como difíceis (fatoração, ECDLP). Se P = NP, toda criptografia moderna colapsa.
  • Algoritmos de aproximação: Para TSP, MAX-CUT, VERTEX COVER — garantias de razão de aproximação provadas via redução e inviabilidade de algoritmos exatos.
  • Heurísticas e ILP solvers: SAT solvers modernos (CDCL), algoritmos genéticos, programação linear inteira — usados em prática mesmo sem garantias polinomiais no pior caso.
  • Machine Learning: Aprender uma rede booleana mínima, treinar SVMs em kernels genéricos, minimizar redes neurais — problemas NP-difíceis sob modelos computacionais padrão.

A questão P vs NP é um dos 7 Problemas do Milênio (Clay Institute), com prêmio de US$1.000.000. O consenso da comunidade é que P ≠ NP, mas nenhuma prova existe.
