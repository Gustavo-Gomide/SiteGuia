Fundamentos de Linguagens Formais

Linguagens Formais

A matemática que define precisamente o que é uma linguagem — e a base de compiladores, verificadores e autômatos.

Uma linguagem formal é simplesmente um conjunto de cadeias sobre um alfabeto. Essa definição minimalista está por trás de expressões regulares, descrição de linguagens de programação, gramáticas e modelos de autômatos.

Alfabeto (Σ)

Conjunto finito e não-vazio de símbolos. Exemplos:

  • Σ = {0, 1} — binário
  • Σ = {a, b, c, ..., z} — ASCII minúsculo
  • Σ = {(, ), +, *, id} — tokens de expressão aritmética

Cadeia (string)

Sequência finita de símbolos de Σ. A cadeia vazia é ε (epsilon). Comprimento de w é denotado |w|.

  • w = abba → |w| = 4
  • ε → |ε| = 0
  • Σ* = todos as cadeias possíveis sobre Σ (incluindo ε)
  • Σ⁺ = Σ* \ {ε} (cadeias não-vazias)

Linguagem (L)

Qualquer subconjunto de Σ*. Pode ser finita ou infinita.

  • L = {ε, a, aa, aaa, ...} = {aⁿ | n ≥ 0}
  • L = {w ∈ {a,b}* | w tem igual nº de a's e b's}
  • L = ∅ (linguagem vazia — nenhuma cadeia)
  • L = Σ* (linguagem universal)

Concatenação não é comutativa

L · M ≠ M · L em geral. {ab}·{c} = {abc}, mas {c}·{ab} = {cab}. Isso distingue linguagens formais de conjuntos numéricos.

Hierarquia de Chomsky

Noam Chomsky (1956) propôs uma classificação de quatro tipos de linguagens formais, definidas pelo poder expressivo de suas gramáticas e pelo tipo de autômato que as reconhece. A hierarquia é de inclusão estrita.

TipoNomeRestrição da produçãoReconhecedorFechada sob
3 Regular A → aB | a (produções lineares) AFD / AFN ∪, ·, *, ∩, complemento
2 Livre de Contexto (LLC) A → α (A não-terminal, α qualquer) Autômato de Pilha (AP) ∪, · — não ∩, não complemento
1 Sensível ao Contexto (LSC) αAβ → αγβ, |γ| ≥ 1 MT Linearmente Limitada ∪, ∩, complemento, ·
0 Recursivamente Enumerável α → β (irrestrito) Máquina de Turing ∪, ·, * — não complemento

Inclusão estrita

Regular ⊊ LLC ⊊ LSC ⊊ RE. Existem linguagens LLC que não são regulares (ex: {aⁿbⁿ | n≥0}), linguagens LSC que não são LLC (ex: {aⁿbⁿcⁿ}), e linguagens RE que não são decidíveis (ex: o Problema da Parada).

Exemplos Canônicos por Tipo

Tipo 3 — Regular

L = {w ∈ {a,b}* | w contém "ab" como subcadeia}

Regex: (a|b)*ab(a|b)*

Reconhecível por AFD com 3 estados.

Tipo 2 — Livre de Contexto

L = {aⁿbⁿ | n ≥ 1}

Gramática: S → aSb | ab

Requer pilha — não pode ser expressa por AFD.

Tipo 1 — Sensível ao Contexto

L = {aⁿbⁿcⁿ | n ≥ 1}

Não é LLC (Pumping Lemma para LLC). Pode ser reconhecida por MT com fita limitada pelo tamanho da entrada.

Tipo 0 — Rec. Enumerável (não decidível)

L_halt = {⟨M, w⟩ | MT M para com entrada w}

O Problema da Parada. Nenhum algoritmo (MT) pode decidir isso para todos os casos.

Lema do Bombeamento

O Lema do Bombeamento é a ferramenta padrão para provar que uma linguagem NÃO é regular.

Enunciado (versão para regulares)

Se L é regular, então existe uma constante p (comprimento de bombeamento) tal que toda cadeia w ∈ L com |w| ≥ p pode ser escrita como w = xyz onde:

  1. |y| ≥ 1
  2. |xy| ≤ p
  3. Para todo i ≥ 0: xyⁱz ∈ L

Como usar: para provar que L não é regular, assume-se que é, escolhe-se uma cadeia w com |w| ≥ p, e mostra-se que nenhuma decomposição xyz satisfaz a condição 3 para todo i. Contradicão.

Exemplo: {aⁿbⁿ} não é regular

Seja p o comprimento de bombeamento. Tome w = aᵖbᵖ. Qualquer decomposição xyz com |xy| ≤ p implica y = aᵏ (k≥1). Então xy²z = aᵖ⁺ᵏbᵖ, que tem mais a's que b's — não pertence à linguagem. Contradição. Logo {aⁿbⁿ} não é regular.
