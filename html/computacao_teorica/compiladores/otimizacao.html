Introdução

Otimização

Fazendo mais com menos — sem alterar o resultado do programa.

A fase de otimização transforma o código intermediário para que o programa resultante seja mais rápido, menor ou consume menos energia — sem mudar o comportamento observável. É o “coelho de cartola” dos compiladores: magia com garantia formal.

Passes Locais

Passes locais atuam dentro de um bloco básico (sequência de instruções sem desvios):

  • Constant Folding: x = 2 * 3 → x = 6 em tempo de compilação.
  • Constant Propagation: substitui variáveis de valor constante pelos valores diretamente.
  • Dead Code Elimination: remove código cujos resultados nunca são usados.
  • Common Subexpression Elimination (CSE): a*b + c + a*b → computa a*b uma vez.
  • Peephole: substitui sequências curtas de instruções por versões mais eficientes (x*2 → x<<1).
// Antes: dead code + constant folding
int a = 2 * 3;      // a = 6 (constant folding)
int b = a + 0;      // b = 6 (additive identity)
bool c = false;     // c nunca usado
return a;           // b e c: dead code

// Depois:
return 6;

Passes Globais e de Loop

Passes globais analisam o grafo de fluxo de controle (CFG) completo:

  • Inlining: substitui chamadas de função pelo corpo da função — elimina overhead de chamada.
  • Loop Invariant Code Motion (LICM): move cálculos que não dependem do loop para fora dele.
  • Loop Unrolling: replica o corpo do loop N vezes, reduzindo overhead de desvio.
  • Strength Reduction: x = i * 4 no loop → x += 4 (adição em vez de multiplicação).
  • Vectorização (Auto-SIMD): converte loops em instruções SIMD (ex.: AVX2, NEON).

O back-edge no CFG identifica loops. LICM move instruções invariantes para o pré-header.

Análise de Fluxo de Dados

Muitas otimizações dependem de análises sobre o grafo de fluxo de controle:

  • Reaching Definitions: “quais definições de x podem chegar a este ponto?” — base para constant propagation.
  • Liveness Analysis: “x ainda será usado após este ponto?” — base para dead-code e alocação de registradores.
  • Dominators: bloco A domina B se todo caminho de entrada passa por A — base para detecção de loops e SSA.
