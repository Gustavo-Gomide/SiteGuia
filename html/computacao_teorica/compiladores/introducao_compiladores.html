Compiladores

Do código-fonte ao binário — cada fase do pipeline explicada em detalhe.

Um compilador é um programa que traduz código escrito em uma linguagem de alto nível para uma representação equivalente de mais baixo nível — geralmente código de máquina ou bytecode. Esse processo é dividido em fases bem definidas, e compreendê-las é fundamental para construir linguagens, otimizadores e ferramentas de análise estática.

O Que é um Compilador?

Um compilador recebe um programa-fonte (texto) e produz um programa-alvo (binário, bytecode ou outra representação). O processo envolve análise e síntese:

  • Front-end (análise): lê o fonte, verifica correção e constrói representações intermediárias.
  • Middle-end (otimização): aplica transformações independentes de máquina.
  • Back-end (síntese): gera código para a arquitetura-alvo.

Por que Python é chamada de “linguagem interpretada”?

Quando alguém diz que Python é interpretada, está usando uma simplificação popular. Na realidade, o interpretador CPython compila o arquivo .py para bytecode (armazenado em .pyc) e depois uma VM interna executa esse bytecode instrução por instrução. O que diferencia Python de linguagens como C não é a ausência de compilação, mas sim:

  • O passo de compilação é automático e transparente — você executa python script.py e não precisa invocar um compilador manualmente.
  • O bytecode gerado não é código de máquina nativo, mas um formato portável executado pela VM.
  • O termo “interpretada” popularizou-se porque do ponto de vista do usuário o comportamento é o de um interpretador: edite o .py e execute imediatamente, sem etapa de build.

Pipeline de Compilação

O pipeline clássico (baseado no livro de Aho, Lam, Sethi e Ullman — “Dragon Book”) divide a compilação em seis fases sequenciais:

  1. Análise Léxica — transforma o fluxo de caracteres em tokens (unidades mínimas de significado).
  2. Análise Sintática — constrói a AST a partir dos tokens.
  3. Análise Semântica — verifica tipos, escopo e significado usando a AST e a tabela de símbolos.
  4. Geração de IR — converte a AST em formato como TAC ou SSA.
  5. Otimização — aplica transformações na IR para melhorar desempenho ou reduzir tamanho sem alterar o comportamento.
  6. Geração de Código — produz código de máquina ou bytecode para a arquitetura-alvo a partir da IR otimizada.

A Tabela de Símbolos

Todas as fases compartilham uma estrutura de dados central: a tabela de símbolos. Ela mapeia identificadores (variáveis, funções, tipos) para seus atributos (tipo, escopo, endereço).

  • A análise léxica insere novos identificadores.
  • A análise semântica verifica e anota tipos.
  • A geração de código consulta endereços e convenções de chamada.
// Exemplo de entrada da tabela de símbolos
{ nome: "x", tipo: "int", escopo: "funcao:main", offset: 4 }
{ nome: "soma", tipo: "fn(int,int)->int", escopo: "global", addr: 0x1004 }

Breve História

  • 1952 — Grace Hopper desenvolve o A-0, considerado o primeiro compilador.
  • 1957 — FORTRAN I (IBM, John Backus): primeiro compilador amplamente adotado; prova que código gerado automaticamente pode ser eficiente.
  • 1960 — ALGOL 60 introduz estrutura de blocos, recursão e escopo léxico, moldando a teoria de compiladores por décadas.
  • 1975 — "Principles of Compiler Design" (Aho e Ullman) sistematiza a área.
  • 1986 — Segunda edição ("Dragon Book", Aho, Sethi, Ullman) torna-se referência canônica.
  • 2003 — LLVM (Chris Lattner) introduz IR modular e reusável, revolucionando compiladores.
