Introdução

Análise Léxica

Da sequência de caracteres para tokens — a primeira fase do pipeline.

A análise léxica (ou scanning) é a fase do compilador que lê o código-fonte carácter a carácter e o agrupa em unidades significativas chamadas tokens. É formalmente equivalente ao reconhecimento de linguagens regulares por autômatos finitos.

Token, Lexema e Padrão

Três conceitos formam a base da análise léxica:

ConceitoDefiniçãoExemplo (x = 3.14)
PadrãoRegra (expressão regular) que descreve a forma dos lexemas de um tipo[a-z][a-z0-9]* para identificadores
LexemaSequência exata de caracteres no fonte que casa com o padrãox
TokenPar (tipo, valor) que o scanner entrega ao parser<ID, "x">, <ASSIGN, "=">, <NUM, 3.14>

DFA e o Scanner

Cada tipo de token é descrito por uma expressão regular que é convertida em um DFA (autômato finito determinístico). O scanner simula todos esses DFAs em paralelo (ou um DFA combinado) para reconhecer tokens.

Abaixo: DFA mínimo para identificadores ([a-zA-Z][a-zA-Z0-9]*)

O estado q1 é de aceitação (duplo círculo). q0 → q1 em qualquer letra; q1 → q1 em letra ou dígito (self-loop). Dígito em q0 é erro — começa um número.

Maximal Munch e Prioridade

Regra do Maior Token

O scanner usa a estratégia maximal munch: sempre avança enquanto o DFA aceitar, e só emite o token quando o próximo carácter causaria rejeição. Isso garante que != seja um único token (operador), não dois (! e =).

Prioridade entre Padrões

Quando o mesmo lexema pode casar com vários padrões (ex.: if é tanto identificador quanto palavra-chave), a prioridade define o vencedor — geralmente palavras-chave têm prioridade sobre identificadores.

// Exemplo de entrada e saída do scanner para: if (x != 0) return x;
Tokens emitidos:
  <KW,    "if">
  <LPAREN,"(">
  <ID,    "x">
  <NEQ,   "!=">
  <NUM,   "0">
  <RPAREN,")">
  <KW,    "return">
  <ID,    "x">
  <SEMI,  ";">

Exemplo Flex

/* lexer.l */
%%
[a-zA-Z][a-zA-Z0-9]*  { return IDENT; }  /* identificador */
[0-9]+                 { return NUM;   }  /* inteiro */
"if"                   { return KW_IF; }  /* palavra-chave */
[ \t\n]+               { /* ignorar */ }
.                      { return UNKNOWN; }
%%

Erros Léxicos e Recuperação

Um erro léxico ocorre quando nenhum padrão casa com o prefixo atual. Estratégias de recuperação:

  • Deletação: remove o carácter inválido e continua.
  • Substituição: troca o carácter inválido por um válido.
  • Modo pânico: avança até um delimitador conhecido (ponto-e-vírgula, chave).

Registrar todos os erros e só interromper a compilação no final permite diagnosticar múltiplos problemas de uma vez.
