Representação Intermediária (IR)

O “meio-campo” entre AST e instruções de máquina.

A IR (​Intermediate Representation) é um formato interno do compilador que representa o programa de forma independente tanto da linguagem-fonte quanto da arquitetura-alvo. Ela é o ponto central onde otimizações são aplicadas e onde frontend e backend se desacoplam.

O Problema M × N

Sem IR, cada par (linguagem-fonte, arquitetura-alvo) exigiria um compilador separado: M linguagens × N arquiteturas = M×N implementações. Com IR, o custo cai para M + N:

Cada linguagem implementa apenas um frontend (fonte → IR) e cada arquitetura implementa apenas um backend (IR → código). M+N em vez de M×N.

Three-Address Code (TAC)

TAC (código de três endereços) é a IR mais simples estudada em cursos. Cada instrução tem no máximo uma operação e três operandos (x = y op z). Considerações complexas são partidas em temporários:

// Código-fonte: a = b * c + d;
// TAC:
t1 = b * c
t2 = t1 + d
a  = t2

// Código-fonte: if (x > 0) y = x; else y = -x;
// TAC:
    t1 = x > 0
    ifFalse t1 goto L1
    y = x
    goto L2
L1: y = -x
L2: ...

Static Single Assignment (SSA)

Na forma SSA, cada variável é atribuída exatamente uma vez. Atribuições múltiplas geram novas versões:
x = 1; x = x + 1; → x⊂1; = 1; x⊂2; = x⊂1; + 1;

Quando dois caminhos de fluxo convergem, SSA usa funções φ (phi) para selecionar a versão correta:

if (cond) x1 = a; else x2 = b;
// Após o if: x3 = phi(x1, x2)  // seleciona conforme o caminho tomado

Por que SSA é usado em compiladores modernos?

  • Simplifica análises: definicaão única facilita rastrear uso/def.
  • Muitas otimizações (constant propagation, dead-code) são triviais em SSA.
  • LLVM IR, GCC GIMPLE e Java Bytecode usam formas baseadas em SSA.

LLVM IR: Exemplo Real

; Trecho de LLVM IR para: int soma(int a, int b) { return a + b; }
define i32 @soma(i32 %a, i32 %b) {
entry:
  %result = add i32 %a, %b   ; SSA: %result definido uma vez
  ret i32 %result
}

Características da LLVM IR: tipoção explícita, SSA rigoroso, controle de fluxo via basic blocks, e portabilidade entre múltiplos backends (x86, ARM, WASM).
