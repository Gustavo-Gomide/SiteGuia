Introdução

Interpretadores

Executando programas sem gerar binário “de uma vez”.

Um interpretador executa o programa diretamente de uma representação de alto nível (AST ou bytecode) sem necessariamente gerar código nativo prévio. É a escolha de Python, Ruby, Lua e de REPLs de linguagens funcionais. A linha com compiladores é gradual: a maioria dos interpretadores modernos usa alguma forma de compilação para bytecode.

Tree-Walk Interpreter

A abordagem mais simples: percorre a AST recursivamente e executa cada nó diretamente.

// Interpreter tree-walk para BinaryOp
fn eval(node) {
    match node {
        BinaryOp('+', left, right) =>
            eval(left) + eval(right),
        Literal(v) => v,
        Var(name)  => env.lookup(name),
        // ...
    }
}
  • Vantagens: simples de implementar, fácil de debugar, ótimo para linguagens experimentais.
  • Desvantagens: muito mais lento que bytecode (overhead por nó visitado).
  • Usos: Ruby 1.8 (YARV antes era tree-walk), primeiros Python, muitos DSLs.

Bytecode + Máquina Virtual

Compila para bytecode (instruções de pilha ou registrador) e uma VM executa-as. É o modelo de CPython, Java (JVM), Lua e .NET CLR.

O ciclo dispatch — FETCH (ler bytecode) → DECODE (identificar opcode) → EXECUTE (→ loop) — é o coração de toda VM baseada em bytecode.

# CPython bytecode para: x = a + b
import dis
dis.dis(lambda a, b: a + b)
# LOAD_FAST  0 (a)
# LOAD_FAST  1 (b)
# BINARY_OP  0 (+)
# RETURN_VALUE

REPL

O REPL (Read–Eval–Print Loop) permite execução interativa de trechos de código. Funciona compilando (ou interpretando) cada expressão individualmente e imprimindo o resultado antes de ler a próxima:

loop:
  input  = read()      # Read
  ast    = parse(input)
  result = eval(ast)   # Eval
  print(result)        # Print
  goto loop            # Loop

REPLs ricas (como as de Haskell, Clojure, Common Lisp) suportam recarga de funções em tempo real, inspeção de estado e hot-patching.

JIT Compilation

O JIT (Just-In-Time) compila funções quentes (muito executadas) para código nativo em tempo de execução, combinando a flexibilidade do interpretador com a velocidade do compilador:

  • Tracing JIT (LuaJIT, PyPy): registra trechos de execução quentes e compila o trace.
  • Method JIT (JVM HotSpot, V8 Turbofan): compila métodos/funções quentes inteiras.
  • Tieréd JIT (V8, .NET RyuJIT): vários níveis de otimização progressiva.
