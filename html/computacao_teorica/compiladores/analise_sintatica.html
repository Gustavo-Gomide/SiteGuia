Análise Sintática

De tokens para árvore — validando estrutura e construindo a AST.

A análise sintática (ou parsing) recebe a sequência de tokens produzida pelo scanner e verifica se eles formam construções válidas segundo a gramática da linguagem. O produto é uma árvore sintática abstrata (AST) usada pelas fases seguintes.

Gramáticas Livres de Contexto (CFG)

A sintaxe de linguagens de programação é especificada por gramáticas livres de contexto (CFG), escritas em BNF ou EBNF:

expr   ::= expr '+' term | term
term   ::= term '*' factor | factor
factor ::= '(' expr ')' | ID | NUM

A CFG define as derivações possíveis. O parser encontra a derivação que corresponde ao programa.

Parse Tree vs AST

  • Parse tree: reflete cada derivação exata (inclui nós de gramática internos).
  • AST: versão compacta que omite nós redundantes etérminos de sintaxe pura.

Descida Recursiva

A técnica mais intuitiva de parser LL: cada não-terminal vira uma função recursiva.

// Parser de descida recursiva para expr = term ('+' term)*
fn parse_expr(tokens) {
    let left = parse_term(tokens);
    while tokens.peek() == '+' {
        tokens.consume();
        let right = parse_term(tokens);
        left = ASTNode::Add(left, right);
    }
    left
}

Simples de escrever, mas gramáticas com recursão à esquerda precisam de transformação prévia.

Erros Sintáticos e Recuperação

  • Modo pânico: descarta tokens até encontrar um ponto de sincronização (ex.: ; ou }).
  • Produções de erro: gramática inclui regras especiais para construções incorretas conhecidas.
  • Reparação global: encontra a menor sequência de inserções/deleções que corrije o programa (custoso).
