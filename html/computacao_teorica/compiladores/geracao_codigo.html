Geração de Código

Transformando IR em instruções reais — a última milha do compilador.

A geração de código recebe a IR (já otimizada) e produz instruções para uma arquitetura-alvo específica (x86-64, ARM, RISC-V, WASM). Envolve três problemas NP-hard: seleção de instruções, alocação de registradores e escalonamento.

Seleção de Instruções

Traduz operações da IR para instruções da ISA. Abordagens:

  • Casamento de árvore (tree matching): representa IR como árvore e aplica padrões de instrução — maximize cobertura de subarvores.
  • Cobertura por tiles: cada instrução da ISA é um “tile” que cobre um subgráfico da IR.
  • IBURG / BURG: gera seletores de instrução automáticos a partir de especificações de custo.
// IR: t = a + b * c
// Seleção para x86-64:
  movq  a(%rip), %rax    ; carrega a
  imulq b(%rip), %rcx    ; rcx = b * c
  addq  %rcx, %rax       ; rax = a + (b*c)  <-- usa LEA ou MUL conforme custo

Alocação de Registradores

A IR pode ter ilimitados temprários; a CPU tem poucos registradores físicos. O problema é equivalente a coloração de grafo:

  • Cada temporário é um nó.
  • Dois temporários que estão vivos ao mesmo tempo têm uma aresta (interferem).
  • Colorir o grafo com K cores = alocar K registradores.
  • Se um nó não puder ser colorido, seu valor vaza para memória (spill).

t1 e t4 não interferem → podem compartilhar %rax. t2 usa %rbx, t3 usa %rcx.
