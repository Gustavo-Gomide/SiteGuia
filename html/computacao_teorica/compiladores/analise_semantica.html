Análise Semântica

Verificando significado — tipos, escopo e consistência de uso.

Após o parser construir a AST, a análise semântica verifica regras que a gramática não captura: tipos compatíveis, variáveis declaradas antes do uso, chamadas de função com argumentos corretos. Esta fase anota a AST com informações de tipo e escopo que guiarão a geração de código.

Escopo e Pilha de Escopos

Cada bloco ({...}) abre um novo escopo (empilhado). Na saída do bloco o escopo é descartado. Isso implementa as regras de visibilidade de linguagens com bloco-escopo (C, Java, Rust).

int x = 1;           // escopo global
void f() {
    int x = 2;       // novo escopo: oculta o global
    {
        int x = 3;   // escopo mais interno
        print(x);    // usa x=3
    }
    print(x);        // usa x=2
}
print(x);            // usa x=1 (global)

Checagem de Tipos

O verificador de tipos garante que operações são aplicadas a operandos compatíveis. Sistemas de tipos diferem em estatic vs. dinâmico e forte vs. fraco:

DimensãoEstáticoDinâmico
Quando checaEm tempo de compilaçãoEm tempo de execução
Erros capturadosAntes de rodarApenas ao executar o caminho errado
ExemplosC, Rust, Java, GoPython, JavaScript (sem TypeScript)

Inferência de Tipos

Compiladores modernos (Rust, Haskell, Swift) usam inferência de tipos (Hindley-Milner) para deduzir o tipo de expressões automaticamente sem anotações explícitas.

// Rust: tipo inferido como i32
let x = 42;         // sem anotacao, compilador infere i32
let y: f64 = 3.14;  // anotacao explicita
let z = x + 1;      // infere i32 porque x eh i32

Gramáticas de Atributos e Decoração da AST

A análise semântica decora a AST: anota cada nó com informações como tipo, endereço de memória e referência para a tabela de símbolos. Dois tipos de atributos:

  • Herdados: fluem de pai para filho (ex.: contexto de tipo esperado).
  • Sintetizados: fluem de filho para pai (ex.: tipo de uma subexpressão).
// No nó BinaryOp('+', e1, e2):
// - sintetizado: tipo = unify(e1.tipo, e2.tipo)
// - herdado: contexto_tipo_esperado

Erros Semânticos

  • Uso de variável não declarada: identificador não encontrado na tabela de símbolos.
  • Tipo incompatível: 3 + true em linguagens fortemente tipadas.
  • Número errado de argumentos: chamada de função com aridade incorreta.
  • Retorno ausente: função não-void sem return em todos os caminhos.
  • Redeclação: mesmo identificador declarado duas vezes no mesmo escopo.
