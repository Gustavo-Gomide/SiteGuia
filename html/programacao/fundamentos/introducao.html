Fundamentos de Programação

Os alicerces que todo programador usa, independente da linguagem

Antes de aprender qualquer linguagem, você precisa dominar a lógica por trás delas. Esta seção cobre os fundamentos universais — do pensamento computacional à modularização — que todo código compartilha.

O que é programação?

Programação é o processo de dar instruções precisas e ordenadas a um computador para que ele resolva um problema. O computador não tem intuição — ele executa exatamente o que você escreve, na ordem em que foi escrito.

Algoritmo

Uma sequência finita de passos bem definidos para resolver um problema. É a base de qualquer programa.

Uma receita de bolo é um algoritmo. Um GPS calculando rotas é um algoritmo.

Código-fonte

O texto que você escreve em uma linguagem de programação (Python, JavaScript, C++). O computador não entende texto diretamente — ele precisa de um intermediário para traduzir para instruções de máquina.

Execução

O momento em que o computador lê e processa suas instruções, produzindo um resultado. Cada instrução é executada na sequência em que foi escrita, salvo quando a lógica desvia o fluxo.

Compilado vs Interpretado

Todo código precisa ser traduzido para a linguagem que o processador entende (binário). Existem dois modelos principais:

Compilado

O código inteiro é traduzido de uma vez antes de rodar, gerando um executável otimizado.

Mais rápido em execução. Exemplos: C, C++, Go, Rust.

Interpretado

O código é lido e executado linha por linha em tempo real por um interpretador.

Mais flexível e fácil de testar. Exemplos: Python, Ruby, PHP.

Híbrido (JIT)

Compila para um formato intermediário (bytecode) e traduz para a máquina em tempo de execução.

Exemplos: Java (JVM), JavaScript (V8), C# (.NET).

O que você aprenderá aqui

Os fundamentos cobertos nesta seção formam a base de qualquer linguagem. Domine esses conceitos e aprender novas linguagens fica muito mais rápido.

Pensamento Computacional

Como estruturar o raciocínio para resolver problemas antes de escrever código: decomposição, reconhecimento de padrões, abstração e algoritmos.

Lógica de Programação

Variáveis, tipos de dados, operadores, condicionais, loops, funções e recursão — os blocos de construção de qualquer programa.

Pseudocódigo e Fluxogramas

Ferramentas para planejar e comunicar a lógica de um algoritmo sem se prender à sintaxe de uma linguagem específica.

Modularização

Dividir um sistema em partes independentes e reutilizáveis. A diferença entre código que escala e código impossível de manter.

Tratamento de Exceções

Como lidar com erros e situações inesperadas sem travar o programa. Try, catch, finally e boas práticas de robustez.

Antes de começar

Não pule esta seção. Muitos iniciantes travam não porque não sabem a linguagem, mas porque não sabem estruturar o pensamento para resolver o problema. Aprender a sintaxe é fácil — aprender a lógica é o desafio real.
