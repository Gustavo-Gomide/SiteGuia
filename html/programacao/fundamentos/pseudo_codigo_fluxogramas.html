Pseudocódigo e Fluxogramas

A planta baixa do software — planeje antes de codificar

Em entrevistas de emprego — especialmente em empresas como Google e Amazon — você terá que resolver problemas no quadro branco. Eles não querem saber se decorou a sintaxe do Java; querem ver seu pensamento lógico. Pseudocódigo e fluxogramas são as ferramentas para isso.

1. O que é Modelagem?

É a "Planta Baixa" do software. Existem duas formas principais de fazer isso:

Pseudocódigo

Escrever a lógica em linguagem nativa, focando na estrutura e ignorando detalhes técnicos (ponto-e-vírgula, chaves).

  • Mais rápido de escrever.
  • Próximo da linguagem final.

Fluxograma

Representação visual usando formas geométricas padronizadas (ISO 5807). Mostra o caminho que os dados percorrem.

  • Melhor para visualizar decisões complexas.
  • Fácil de explicar para leigos.

2. Simbologia Padrão (ISO 5807)

Não invente formas. Use o padrão internacional para que qualquer programador entenda seu fluxograma.

Terminal

Marca onde o algoritmo começa e termina. Obrigatório.

Processamento

Cálculos, somas ou atribuição de variáveis.

Decisão

Uma pergunta de Sim/Não. Divide o fluxo em dois caminhos.

Entrada / Saída

Interação com o usuário (Teclado ou Tela).

3. Estudo de Caso: Sistema de Login

Vamos criar a lógica de um login que bloqueia após 3 tentativas erradas. Esse é um algoritmo clássico que envolve Variáveis, Loops e Condicionais.

A. Pseudocódigo

Note como usamos indentação (espaço à esquerda) para mostrar o que está dentro do Loop.

algoritmo_login.portugol
// 1. Declaração de Variáveis CONSTANTE SENHA_CORRETA = "1234" VAR tentativas : INTEIRO = 0 VAR senha_digitada : TEXTO VAR logado : BOOLEANO = FALSO INICIO // 2. Loop: Repete enquanto tentativas menor que 3 E não estiver logado ENQUANTO (tentativas < 3 E logado == FALSO) FACA ESCREVA "Digite a senha: " LEIA senha_digitada // 3. Verificação SE (senha_digitada == SENHA_CORRETA) ENTAO logado = VERDADEIRO ESCREVA "Acesso Permitido!" SENAO tentativas = tentativas + 1 // Incrementa erro ESCREVA "Senha Incorreta." FIM_SE FIM_ENQUANTO // 4. Resultado Final SE (logado == FALSO) ENTAO ESCREVA "Conta Bloqueada." FIM_SE FIM

Acompanhe as setas. Veja como o fluxo "NÃO" da senha volta para o início para tentar de novo.

C. Teste de Mesa (Trace Table)

É assim que você testa se o código funciona sem usar o computador. Simule a memória linha a linha.

Linha Var: tentativas Var: logado Input Condição Saída Tela
1-5 0 FALSO - - (Início)
8 0 FALSO - VERDADEIRO -
11 0 FALSO "teste" - "Digite..."
18 1 FALSO - - "Incorreta"
8 1 FALSO - VERDADEIRO (Retorno)
11 1 FALSO "1234" - "Digite..."
15 1 VERDADEIRO - - "Sucesso!"
