Polimorfismo

Polimorfismo

Uma Interface, Muitos Comportamentos

Poly = muitas · Morph = formas. O mesmo comando, efeitos diferentes.

Polimorfismo é a capacidade de um mesmo nome (método, operador, função) ter comportamentos diferentes dependendo do tipo ou da classe do objeto que o executa. É o mecanismo que permite escrever código genérico e extensível: você escreve forma.desenhar() e o Python descobre qual implementação chamar em tempo de execução — sem ifs ou switches.

O que é Polimorfismo?

O termo vem do grego polýmorphos — "de muitas formas". Em POO isso se manifesta quando diferentes objetos respondem à mesma mensagem (chamada de método) cada um à sua maneira. O chamador não precisa saber qual tipo concreto está usando — basta que ele cumpra o contrato esperado.

Polimorfismo é o resultado natural de dois outros pilares: herança (subclasses que sobrescrevem métodos da superclasse) e interfaces/protocolos (objetos que assinam o mesmo contrato sem necessariamente compartilhar ancestral). Ambos chegam ao mesmo efeito: código que funciona com qualquer implementação, presente ou futura.

Polimorfismo por Subtipagem (Override)

A forma mais clássica: subclasses sobrescrevem (override) métodos da superclasse. O código que chama o método não precisa saber o tipo concreto — o Python resolve em tempo de execução qual implementação usar (dynamic dispatch).

polimorfismo_subtipagem.py
from abc import ABC, abstractmethod
import math

class Forma(ABC):
    @abstractmethod
    def desenhar(self) -> None: ...

    @abstractmethod
    def area(self) -> float: ...


class Circulo(Forma):
    def __init__(self, raio: float):
        self.raio = raio
    def desenhar(self) -> None:
        print(f"⬤  Círculo  raio={self.raio}")
    def area(self) -> float:
        return math.pi * self.raio ** 2


class Retangulo(Forma):
    def __init__(self, l: float, a: float):
        self.l, self.a = l, a
    def desenhar(self) -> None:
        print(f"▬  Retângulo {self.l}×{self.a}")
    def area(self) -> float:
        return self.l * self.a


class Triangulo(Forma):
    def __init__(self, base: float, alt: float):
        self.base, self.alt = base, alt
    def desenhar(self) -> None:
        print(f"▲  Triângulo base={self.base} h={self.alt}")
    def area(self) -> float:
        return (self.base * self.alt) / 2


# Código genérico — funciona com QUALQUER subclasse presente ou futura
formas: list[Forma] = [Circulo(5), Retangulo(4, 6), Triangulo(3, 8)]
for f in formas:
    f.desenhar()
    print(f"  área = {f.area():.2f}")

O loop for f in formas: f.desenhar() é um exemplo de código aberto para extensão, fechado para modificação (Princípio Aberto/Fechado — OCP): adicionar um novo tipo de forma não requer alterar uma linha do loop.

Duck Typing — Polimorfismo sem Hierarquia

Em Python você não precisa herdar de nenhuma superclasse comum para ser polimórfico. SE um objeto tem o método esperado, ele funciona. Esta ideia tem nome: duck typing — se anda como pato e grasna como pato, é um pato. O tipo nominal não importa; o comportamento sim.

duck_typing.py — polimorfismo sem herança
# Nenhuma das 3 classes herda de uma base comum

class Arquivo:
    def salvar(self, dados: str) -> None:
        print(f"[ARQUIVO] Salvando: {dados}")


class BancoDeDados:
    def salvar(self, dados: str) -> None:
        print(f"[DB] INSERT '{dados}'")


class Nuvem:
    def salvar(self, dados: str) -> None:
        print(f"[NUVEM] Upload: {dados}")


def persistir(repositorio, dados: str) -> None:
    # Não verifica tipo — apenas chama .salvar()
    repositorio.salvar(dados)


persistir(Arquivo(),       "relatório")  # [ARQUIVO] Salvando: relatório
persistir(BancoDeDados(), "relatório")  # [DB] INSERT 'relatório'
persistir(Nuvem(),         "relatório")  # [NUVEM] Upload: relatório

Duck typing é extremamente flexível mas sacrifica a verificação em tempo de compilação. Para obter o melhor dos dois mundos — flexibilidade e segurança — use Protocol com anotações de tipo e um verificador estático como mypy.

Os Tipos de Polimorfismo

Subtipagem (Override)

Subclasse sobrescreve método da superclasse. O dispatch é dinâmico: o Python escolhe a implementação correta em tempo de execução com base no tipo real do objeto.

Duck Typing

Sem herança. Qualquer objeto que implemente a interface esperada funciona. Padrão em Python — poderoso mas requer disciplina com anotações de tipo para manter a segurança.

Paramétrico (Genéricos)

Uma função ou classe funciona com qualquer tipo passado como parâmetro. Em Python: list[T], TypeVar. Verificado estaticamente, não em runtime.

Ad-hoc (Sobrecarga)

Uma função se comporta diferente dependendo do tipo do argumento. Python não tem sobrecarga nativa de funções mas oferece functools.singledispatch para isso.

Avançado: singledispatch e Genéricos

functools.singledispatch — sobrecarga por tipo

Permite registrar implementações diferentes de uma função para tipos diferentes, sem hierarquia de classes. Ideal para funções que processam dados heterogêneos mantendo o código organizado e sem isinstance em cascata.

singledispatch_exemplo.py
from functools import singledispatch

@singledispatch
def processar(valor) -> str:
    return f"Tipo não suportado: {type(valor).__name__}"

@processar.register(int)
def _(valor: int) -> str:
    return f"Inteiro ao quadrado: {valor ** 2}"

@processar.register(str)
def _(valor: str) -> str:
    return f"String maiúscula: {valor.upper()}"

@processar.register(list)
def _(valor: list) -> str:
    return f"Lista de {len(valor)} elemento(s)"

print(processar(7))          # Inteiro ao quadrado: 49
print(processar("python"))  # String maiúscula: PYTHON
print(processar([1, 2, 3])) # Lista de 3 elemento(s)
print(processar(3.14))      # Tipo não suportado: float

Genéricos com TypeVar

TypeVar permite escrever funções e classes que operam sobre qualquer tipo mantendo a coerência de tipos verificável estaticamente. É polimorfismo paramétrico: o tipo concreto é substituído no momento do uso.

genericos.py
from typing import TypeVar, Generic

T = TypeVar("T")

class Pilha(Generic[T]):
    """Pilha genérica — funciona com int, str, qualquer tipo."""
    def __init__(self) -> None:
        self._dados: list[T] = []

    def push(self, item: T) -> None:
        self._dados.append(item)

    def pop(self) -> T:
        return self._dados.pop()

    def topo(self) -> T:
        return self._dados[-1]


pilha_int: Pilha[int] = Pilha()
pilha_int.push(10)
pilha_int.push(20)
print(pilha_int.pop())     # 20

pilha_str: Pilha[str] = Pilha()
pilha_str.push("alpha")
print(pilha_str.topo())   # alpha

Evite isinstance() em cascata. Se você tem um if isinstance(x, A): ... elif isinstance(x, B): ... isso é o sinal de que falta polimorfismo. Mova o comportamento para dentro das classes com um método comum e deixe o dispatch dinâmico trabalhar.
