SQL Linguagem de Banco de Dados

SQL: Linguagem de Consulta Estruturada

A linguagem padrão para criar, consultar e manter bancos de dados relacionais.

SQL (Structured Query Language) é a linguagem que permite conversar com um banco de dados relacional. Criada na IBM nos anos 1970, tornou-se padrão ISO e é utilizada em todos os SGBDs relacionais modernos, como MySQL, PostgreSQL, SQL Server, Oracle e SQLite.

Famílias de Comandos SQL

Os comandos SQL são agrupados em 5 famílias, cada uma com um propósito distinto:

DQL Data Query Language

Consultar/ler dados. Só leitura, não altera o banco.

SELECT

DML Data Manipulation Language

Manipular linhas de dados: inserir, alterar e remover.

INSERT  UPDATE  DELETE

DDL Data Definition Language

Definir a estrutura do banco: criar, alterar e destruir objetos.

CREATE  ALTER  DROP  TRUNCATE

DCL Data Control Language

Controlar permissões de acesso aos dados.

GRANT  REVOKE

TCL Transaction Control Language

Gerenciar transações: confirmar ou desfazer operações.

START TRANSACTION  COMMIT  ROLLBACK  SAVEPOINT

Sintaxe Básica

Case-insensitive

SQL ignora maiúsculas/minúsculas em palavras-chave. Por convenção, escrevemos as palavras-chave em MAIÚSCULAS para legibilidade.

-- Tudo equivalente:
select * from usuarios
SELECT * FROM usuarios
Select * From Usuarios

Ponto-e-vírgula

Termina uma instrução SQL. Em scripts com vários comandos, é obrigatório para separá-los. Strings usam aspas simples.

SELECT * FROM usuarios;
INSERT INTO produtos (nome) VALUES ('Mouse');
-- Strings: aspas SIMPLES, nunca duplas

DQL: Consultando Dados

O SELECT é com certeza o comando mais usado em SQL. Ele permite filtrar, ordenar e transformar dados sem alterar nada.

SELECT básico

-- Tudo (evitar em produção — trafica dados desnecessários)
SELECT * FROM produtos;

-- Colunas específicas (preferível)
SELECT nome, preco FROM produtos;

-- Eliminar duplicatas
SELECT DISTINCT categoria FROM produtos;

-- Ordenar resultados
SELECT nome, preco FROM produtos ORDER BY preco DESC;
SELECT nome FROM clientes  ORDER BY nome ASC;   -- ASC é padrão

Filtrando com WHERE

-- Operadores de comparação
SELECT * FROM produtos WHERE preco > 100;
SELECT * FROM produtos WHERE categoria = 'Eletrônicos';
SELECT * FROM produtos WHERE preco <> 50;     -- diferente

-- LIKE: busca por padrão em texto (% = qualquer trecho, _ = um char)
SELECT * FROM clientes WHERE nome LIKE 'Ana%';   -- começa com Ana
SELECT * FROM clientes WHERE nome LIKE '%silva';  -- termina com silva

-- IN: lista de valores possíveis
SELECT * FROM pedidos WHERE status IN ('pendente', 'processando');

-- BETWEEN: intervalo inclusivo (data, número)
SELECT * FROM produtos WHERE preco BETWEEN 50 AND 150;

-- IS NULL / IS NOT NULL
SELECT * FROM clientes WHERE email IS NULL;

Limitar resultados

-- MySQL / PostgreSQL
SELECT nome FROM produtos ORDER BY preco DESC LIMIT 10;

-- SQL Server / Oracle
SELECT TOP 10 nome FROM produtos ORDER BY preco DESC;

Funções de Agregação

Funções de agregação calculam um resultado a partir de um conjunto de linhas essenciais para relatórios e estatísticas.

-- COUNT, SUM, AVG, MIN, MAX
SELECT COUNT(*)              AS total_pedidos   FROM pedidos;
SELECT COUNT(email)          AS com_email       FROM clientes;  -- ignora NULL
SELECT SUM(valor_total)      AS receita_total   FROM pedidos;
SELECT AVG(preco)            AS preco_medio     FROM produtos;
SELECT MIN(preco), MAX(preco) AS min_max        FROM produtos;

-- GROUP BY: agrupa linhas com o mesmo valor
SELECT categoria, COUNT(*) AS qtd
FROM produtos
GROUP BY categoria;

-- HAVING: filtra os grupos (WHERE não funciona em agregações)
SELECT categoria, AVG(preco) AS media
FROM produtos
GROUP BY categoria
HAVING AVG(preco) > 200;  -- só categorias com média acima de R$ 200

Ordem de Execução Lógica

O SQL não é processado na ordem em que você escreve. A ordem real de execução é:

-- 1. FROM        qual tabela usar
-- 2. WHERE       filtrar linhas brutas
-- 3. GROUP BY    agrupar
-- 4. HAVING      filtrar grupos
-- 5. SELECT      escolher/calcular colunas
-- 6. ORDER BY    ordenar resultado
-- 7. LIMIT       recortar quantidade
Regra prática: você não pode usar alias criado no SELECT dentro do WHERE — o WHERE é executado antes. Use o HAVING (ou CTE) para isso.

JOINs: Unindo Tabelas

JOINs combinam linhas de duas ou mais tabelas com base em uma condição de ligação (geralmente PK FK).

INNER JOIN

Somente registros que têm correspondência em ambos os lados.

SELECT c.nome, p.total
FROM clientes c
INNER JOIN pedidos p ON c.id = p.cli_id;

LEFT JOIN

Todos da esquerda + correspondência da direita (NULL se não houver). Útil para "clientes que nunca compraram".

SELECT c.nome, p.total
FROM clientes c
LEFT JOIN pedidos p ON c.id = p.cli_id
WHERE p.id IS NULL;  -- sem pedidos

RIGHT JOIN

Todos da direita + correspondência da esquerda. Menos usado prefira LEFT JOIN invertendo a ordem.

CROSS JOIN

Produto cartesiano combina toda linha de A com toda linha de B. Cuidado: gera MN linhas.

SELECT c.nome, p.nome
FROM cores c CROSS JOIN produtos p;

SELF JOIN

JOIN da tabela com ela mesma para hierarquias (funcionário gerente).

SELECT e.nome, g.nome AS gerente
FROM funcionarios e
JOIN funcionarios g ON e.gerente_id = g.id;

Subconsultas

Uma subconsulta (subquery) é um SELECT dentro de outro SELECT. Permite usar o resultado de uma consulta como valor ou filtro.

-- Escalar: retorna um único valor
SELECT nome FROM produtos
WHERE preco > (SELECT AVG(preco) FROM produtos);

-- IN: retorna uma lista
SELECT nome FROM clientes
WHERE id IN (SELECT cli_id FROM pedidos WHERE status = 'pendente');

-- Tabela derivada no FROM
SELECT cat, media FROM (
  SELECT categoria AS cat, AVG(preco) AS media FROM produtos GROUP BY categoria
) AS resumo
WHERE media > 100;

-- Muitas subconsultas IN podem ser reescritas como JOIN (mais eficiente)
SELECT DISTINCT c.nome
FROM clientes c
JOIN pedidos p ON c.id = p.cli_id
WHERE p.status = 'pendente';

Índices

Um índice é uma estrutura auxiliar (B-Tree por padrão) que acelera buscas como o índice de um livro. O banco cria um para a PK automaticamente.

-- Criar índice simples
CREATE INDEX idx_prod_categoria ON produtos (categoria);

-- Índice único (também valida unicidade)
CREATE UNIQUE INDEX idx_cli_email ON clientes (email);

-- Remover índice
DROP INDEX idx_prod_categoria ON produtos;  -- MySQL
DROP INDEX idx_prod_categoria;              -- PostgreSQL

Quando índice ajuda

  • Coluna usada frequentemente em WHERE.
  • Coluna usada em JOIN ON.
  • Coluna usada em ORDER BY.
  • Alta cardinalidade (muitos valores distintos).

Quando índice prejudica

  • Tabelas muito pequenas (varredura total é mais rápida).
  • Colunas com poucas variações (sexo M/F baixa cardinalidade).
  • Tabelas com muitos INSERTs/UPDATEs (índice precisa ser mantido).
  • Índices demais = escrita lenta e RAM desperdiçada.

DDL: Definindo Estrutura

DDL define os objetos do banco tabelas, colunas, índices, constraints. Comandos DDL têm COMMIT implícito; não dão ROLLBACK.

-- CREATE TABLE com constraints
CREATE TABLE produtos (
  id          INT           AUTO_INCREMENT PRIMARY KEY,
  nome        VARCHAR(100)  NOT NULL,
  preco       DECIMAL(10,2) NOT NULL CHECK (preco >= 0),
  categoria   VARCHAR(50),
  estoque     INT           DEFAULT 0,
  criado_em   DATETIME      DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT  uq_prod_nome  UNIQUE (nome)
);

-- ALTER TABLE: modificar estrutura existente
ALTER TABLE produtos ADD COLUMN desconto DECIMAL(5,2);
ALTER TABLE produtos MODIFY COLUMN nome VARCHAR(150) NOT NULL;
ALTER TABLE produtos DROP COLUMN desconto;

-- DROP TABLE: remove tabela e dados (irreversível!)
DROP TABLE produtos;

-- TRUNCATE: remove todos os dados, mantém a estrutura
TRUNCATE TABLE logs_acesso;
Cuidado: DROP e TRUNCATE são operações sem ROLLBACK na maioria dos SGBDs. Use com confirmação em produção.

DML: Manipulando Dados

-- INSERT: uma linha
INSERT INTO clientes (nome, email) VALUES ('Ana Silva', 'ana@email.com');

-- INSERT: múltiplas linhas (mais eficiente que vários INSERTs)
INSERT INTO produtos (nome, preco, categoria) VALUES
  ('Mouse',    89.90, 'Periférico'),
  ('Teclado', 149.90, 'Periférico'),
  ('Monitor', 899.00, 'Display');

-- UPDATE: atualizar dados
UPDATE produtos SET preco = preco * 0.9 WHERE categoria = 'Periférico';

-- DELETE: remover linhas
DELETE FROM pedidos WHERE status = 'cancelado' AND criado_em < '2023-01-01';
UPDATE e DELETE sem WHERE afetam todas as linhas da tabela! Sempre verifique com um SELECT antes de executar alterações em massa.

DCL: Controlando Permissões

DCL define quem pode fazer o quê. Essencial para segurança em ambientes com múltiplos usuários.

-- Criar usuário
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'senha_forte';

-- GRANT: conceder permissões
GRANT SELECT, INSERT ON loja.* TO 'app_user'@'localhost';
GRANT ALL PRIVILEGES ON loja.* TO 'admin'@'localhost';

-- Ver permissões
SHOW GRANTS FOR 'app_user'@'localhost';

-- REVOKE: revogar permissões
REVOKE INSERT ON loja.* FROM 'app_user'@'localhost';
Princípio do Menor Privilégio: dê ao usuário apenas as permissões que ele precisa, e nada mais.

TCL: Controlando Transações

TCL trata de transações grupos de operações que devem ser executadas como um bloco atômico.

-- Transação básica
START TRANSACTION;

  UPDATE conta SET saldo = saldo - 100 WHERE id = 1;
  UPDATE conta SET saldo = saldo + 100 WHERE id = 2;

COMMIT;    -- confirma tudo
-- ou: ROLLBACK;  para desfazer

-- SAVEPOINT: ponto de retorno parcial
START TRANSACTION;
  INSERT INTO pedido ...;
  SAVEPOINT sp_pedido;
  INSERT INTO item_pedido ...;   -- falhou

ROLLBACK TO SAVEPOINT sp_pedido; -- volta só até aqui
COMMIT;                          -- salva o pedido sem os itens

O padrão da maioria dos SGBDs é Read Committed. O MySQL/InnoDB usa Repeatable Read.

Views

Uma View é uma consulta salva que se comporta como uma tabela virtual. Cada vez que é acessada, executa o SELECT original por baixo. 

-- Criar view
CREATE VIEW pedidos_ativos AS
  SELECT p.id, c.nome AS cliente, p.total, p.status
  FROM pedidos p
  JOIN clientes c ON p.cli_id = c.id
  WHERE p.status IN ('pendente', 'processando');

-- Usar como tabela
SELECT * FROM pedidos_ativos WHERE total > 500;

-- Atualizar definição
CREATE OR REPLACE VIEW pedidos_ativos AS ...;

-- Remover
DROP VIEW pedidos_ativos;

Usos comuns

  • Segurança: expor apenas certas colunas para um usuário.
  • Simplificação: esconder JOINs complexos dos devs.
  • Legado: manter compatibilidade quando tabela é reestruturada.

Limitações

  • Views com JOINs não são diretamente atualizáveis (UPDATE).
  • Não armazenam dados cada acesso executa o SELECT completo.
  • Para performance em leitura pesada, use Materialized Views (PostgreSQL).

Stored Procedures

Uma Stored Procedure é um bloco de SQL pré-compilado armazenado no servidor. Aceita parâmetros, tem lógica condicional e loops.

-- MySQL: criar procedure com parâmetro de entrada
DELIMITER //
CREATE PROCEDURE aplicar_desconto(
  IN p_categoria VARCHAR(50),
  IN p_desconto  DECIMAL(5,2)
)
BEGIN
  IF p_desconto <= 0 OR p_desconto > 0.9 THEN
    SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Desconto inválido';
  END IF;

  UPDATE produtos
  SET preco = preco * (1 - p_desconto)
  WHERE categoria = p_categoria;

  SELECT ROW_COUNT() AS linhas_afetadas;
END //
DELIMITER ;

-- Chamar
CALL aplicar_desconto('Periférico', 0.10);

Triggers

Um Trigger é executado automaticamente pelo SGBD quando ocorre um evento em uma tabela (INSERT, UPDATE ou DELETE). É acionado BEFORE (antes) ou AFTER (depois) do evento. 

-- Trigger de auditoria: log de toda alteração de preço
CREATE TRIGGER after_preco_update
AFTER UPDATE ON produtos
FOR EACH ROW
BEGIN
  IF OLD.preco <> NEW.preco THEN
    INSERT INTO log_precos (produto_id, preco_antigo, preco_novo, alterado_em)
    VALUES (OLD.id, OLD.preco, NEW.preco, NOW());
  END IF;
END;

-- Trigger BEFORE: validação antes de inserir
CREATE TRIGGER before_pedido_insert
BEFORE INSERT ON pedidos
FOR EACH ROW
BEGIN
  IF NEW.total <= 0 THEN
    SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Total deve ser positivo';
  END IF;
END;
Use triggers com moderação. Eles executam silenciosamente, dificultando debug. Para lógica de negócio complexa, prefira Stored Procedures chamadas explicitamente.

ACID no SQL

As propriedades ACID são garantidas pelos SGBDs transacionais através de TCL. Veja na prática:

-- Transferência bancária demonstrando ACID
START TRANSACTION;

UPDATE contas SET saldo = saldo - 500 WHERE id = 101;   -- débito

-- Verificação de consistência
SELECT saldo INTO @saldo FROM contas WHERE id = 101;
IF @saldo < 0 THEN
  ROLLBACK;   -- Atomicidade: desfaz tudo — saldo não pode ser negativo
ELSE
  UPDATE contas SET saldo = saldo + 500 WHERE id = 202;
  COMMIT;     -- Durabilidade: persiste no disco; Isolamento: outras transações
END IF;       --              não vêem os dados intermediários

Referência Rápida

Template de SELECT Completo

SELECT [DISTINCT] colunas | expressões | funções
FROM tabela_principal
  [JOIN outra ON cond]
  [LEFT JOIN outra2 ON cond]
WHERE condição_linha
GROUP BY coluna_agrupamento
HAVING condição_grupo
ORDER BY coluna [ASC|DESC]
LIMIT n OFFSET m;

Funções de Texto

CONCAT(a, b)             -- une strings
LENGTH(str)              -- comprimento em bytes
CHAR_LENGTH(str)         -- comprimento em caracteres
UPPER(str) / LOWER(str)  -- maiúsculo / minúsculo
TRIM(str)                -- remove espaços nas pontas
LTRIM / RTRIM(str)       -- remove espaço esquerda / direita
SUBSTRING(str, pos, len) -- extrai trecho
REPLACE(str, old, new)   -- substitui
CONCAT_WS(sep, a, b)     -- une com separador
LPAD(str, n, pad)        -- preenche à esquerda

Funções de Data

NOW()                    -- data e hora atual
CURDATE() / CURTIME()   -- só data / só hora
YEAR(d) / MONTH(d) / DAY(d) -- extrai partes
DATEDIFF(d1, d2)         -- diferença em dias
DATE_ADD(d, INTERVAL n DAY|MONTH|YEAR)
DATE_FORMAT(d, '%d/%m/%Y %H:%i') -- formatar

Funções Numéricas

ABS(n)                   -- valor absoluto
CEIL(n) / FLOOR(n)      -- teto / chão

MOD(a, b)                -- resto da divisão (a % b)
POWER(base, exp)         -- potenciação
SQRT(n)                  -- raiz quadrada

Condicionais e Nulos

COALESCE(a, b, c)       -- retorna o primeiro não-NULL
NULLIF(a, b)             -- retorna NULL se a = b
IFNULL(a, default)       -- retorna default se a é NULL

-- CASE WHEN (equivalente a if/else)
SELECT nome,
  CASE
    WHEN preco < 50     THEN 'Barato'
    WHEN preco < 200    THEN 'Médio'
    ELSE                     'Caro'
  END AS faixa_preco
FROM produtos;

Operações de Conjunto

-- UNION: une resultados (remove duplicatas)
SELECT nome FROM clientes UNION SELECT nome FROM fornecedores;

-- UNION ALL: une sem remover duplicatas (mais rápido)
SELECT email FROM lista_a UNION ALL SELECT email FROM lista_b;

-- INTERSECT: somente o que existe nos dois (PostgreSQL / SQL Server)
SELECT id FROM tabela_a INTERSECT SELECT id FROM tabela_b;

-- EXCEPT / MINUS: o que existe em A mas não em B
SELECT id FROM tabela_a EXCEPT SELECT id FROM tabela_b;

-- EXISTS: verifica se subquery retorna linhas
SELECT nome FROM clientes c
WHERE EXISTS (
  SELECT 1 FROM pedidos p WHERE p.cli_id = c.id AND p.status = 'pendente'
);
