Oracle Database — O Porta-Aviões dos Bancos de Dados

O SGBD corporativo mais robusto do planeta, escolhido por bancos, governos e multinacionais.

O Oracle Database é o SGBD proprietário lder do segmento enterprise: projetado para jamais falhar, lidar com petabytes e garantir segurança e conformidade máximas.

O que é Oracle Database?

O Oracle Database é um sistema gerenciador de banco de dados relacional proprietário desenvolvido pela Oracle Corporation desde 1977. É o SGBD mais usado em ambientes corporativos de missão crítica: bancos globais, companhias aéreas, governos e operadoras de telecomunicações dependem dele diariamente.

Pense no Oracle como a opção quando falhar não é uma opção: operações bancárias em tempo real, controle de tráfego aéreo, registros médicos hospitalares. O preço disso é licenciamento caro e necessidade de DBAs certificados para operar.

A Oracle foi fundada em 1977 por Larry Ellison após ler um artigo de Edgar Codd sobre bancos relacionais — o mesmo Codd que criou o modelo relacional que todo SGBD usa até hoje.

2023 Oracle 23c traz suporte nativo a JSON Relational Duality Views, unindo o melhor dos mundos relacional e documentos.

Arquitetura Interna — SGA, PGA e Background Processes

O Oracle Database tem uma das arquiteturas internas mais complexas entre os SGBDs. Compreendê-la é fundamental para qualquer DBA Oracle.

SGA — System Global Area

A SGA é uma área de memória compartilhada entre todos os processos do Oracle. Contém:

  • Buffer Cache: caches dos blocos de dados lidos do disco. Evita leituras físicas repetidas.
  • Shared Pool: cache do código SQL parsed (Library Cache) e do dicionário de dados (Data Dictionary Cache).
  • Redo Log Buffer: buffer de todas as transações, antes de serem escritas no redo log em disco.
  • Large Pool: memória para operações grandes como backups (RMAN) e I/O paralelo.

PGA — Program Global Area

A PGA é memória privada de cada processo/sessão. Usada para sort areas, hash joins e variáveis de sessão. Diferente da SGA, não é compartilhada.

Processos de Background Essenciais

ProcessoFunção
SMONSystem Monitor — faz crash recovery na inicialização e limpa segmentos temporários.
PMONProcess Monitor — limpa sessões de usuários que morreram abruptamente.
DBWnDatabase Writer — escreve blocos "sujos" do Buffer Cache para os datafiles no disco.
LGWRLog Writer — grava o Redo Log Buffer nos Redo Log files em disco. Crítico para durabilidade.
ARCHArchiver — copia os redo logs cheios para archive logs. Essencial para recuperação point-in-time.
CKPTCheckpoint — sinaliza todos os DBWn para gravar dados ao disco e atualiza os control files.

PL/SQL — A Linguagem Procedural da Oracle

O PL/SQL (Procedural Language / SQL) é a extensão proprietária da Oracle ao SQL padrão. Permite escrever lógica de programação complexa diretamente no banco: procedimentos, funções, pacotes, triggers e cursores.

Bloco Anônimo — Estrutura Básica

PL/SQL
-- Todo bloco PL/SQL tem 3 seções: DECLARE, BEGIN, EXCEPTION
DECLARE
  v_salario  NUMBER(10,2);
  v_bonus    NUMBER(10,2) := 0;
  v_nome     VARCHAR2(100);
BEGIN
  -- Selecionar dados para variáveis
  SELECT nome, salario
  INTO   v_nome, v_salario
  FROM   funcionarios
  WHERE  id = 101;

  -- Lógica condicional
  IF v_salario > 10000 THEN
    v_bonus := v_salario * 0.05; -- 5% de bônus
  ELSIF v_salario > 5000 THEN
    v_bonus := v_salario * 0.10; -- 10% de bônus
  ELSE
    v_bonus := 1000;             -- Bônus fixo
  END IF;

  UPDATE funcionarios SET bonus = v_bonus WHERE id = 101;
  COMMIT;

  DBMS_OUTPUT.PUT_LINE('Bônus de ' || v_nome || ': R$ ' || v_bonus);

EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('Funcionário não encontrado!');
    ROLLBACK;
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Erro inesperado: ' || SQLERRM);
    ROLLBACK;
END;
/

Stored Procedure com Parâmetros

PL/SQL
-- Criar uma procedure reutilizável
CREATE OR REPLACE PROCEDURE conceder_aumento(
  p_funcionario_id IN  NUMBER,
  p_percentual     IN  NUMBER,
  p_mensagem       OUT VARCHAR2
) AS
  v_salario_atual NUMBER(10,2);
  v_novo_salario  NUMBER(10,2);
BEGIN
  SELECT salario INTO v_salario_atual
  FROM funcionarios WHERE id = p_funcionario_id;

  v_novo_salario := v_salario_atual * (1 + p_percentual / 100);

  UPDATE funcionarios
  SET salario = v_novo_salario,
      data_reajuste = SYSDATE
  WHERE id = p_funcionario_id;

  COMMIT;
  p_mensagem := 'Salário atualizado de ' || v_salario_atual
                || ' para ' || v_novo_salario;
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    p_mensagem := 'Funcionário ' || p_funcionario_id || ' não encontrado.';
    ROLLBACK;
END conceder_aumento;
/

-- Chamar a procedure
DECLARE
  v_msg VARCHAR2(200);
BEGIN
  conceder_aumento(101, 15, v_msg);
  DBMS_OUTPUT.PUT_LINE(v_msg);
END;
/

Cursor Explícito — Iterar Sobre Resultados

PL/SQL
DECLARE
  CURSOR cur_funcionarios IS
    SELECT id, nome, salario
    FROM funcionarios
    WHERE departamento_id = 10
    ORDER BY salario DESC;

  v_total_salarios NUMBER := 0;
BEGIN
  FOR reg IN cur_funcionarios LOOP
    v_total_salarios := v_total_salarios + reg.salario;
    DBMS_OUTPUT.PUT_LINE(reg.nome || ' — R$ ' || reg.salario);
  END LOOP;

  DBMS_OUTPUT.PUT_LINE('Total: R$ ' || v_total_salarios);
END;
/

Package — Organização de Código

Os packages são o equivalente a módulos/namespaces em linguagens de programação — agrupam funções e procedures relacionadas.

PL/SQL
-- Especificação do package (interface pública)
CREATE OR REPLACE PACKAGE pkg_funcionarios AS
  PROCEDURE listar_por_depto(p_depto_id NUMBER);
  FUNCTION  calcular_bonus(p_salario NUMBER) RETURN NUMBER;
END pkg_funcionarios;
/

-- Corpo do package (implementação)
CREATE OR REPLACE PACKAGE BODY pkg_funcionarios AS

  PROCEDURE listar_por_depto(p_depto_id NUMBER) AS
  BEGIN
    FOR r IN (SELECT nome, salario FROM funcionarios
              WHERE departamento_id = p_depto_id) LOOP
      DBMS_OUTPUT.PUT_LINE(r.nome || ' | ' || r.salario);
    END LOOP;
  END listar_por_depto;

  FUNCTION calcular_bonus(p_salario NUMBER) RETURN NUMBER AS
  BEGIN
    RETURN CASE
      WHEN p_salario > 15000 THEN p_salario * 0.05
      WHEN p_salario > 7000  THEN p_salario * 0.08
      ELSE p_salario * 0.12
    END;
  END calcular_bonus;

END pkg_funcionarios;
/

-- Usar o package
EXEC pkg_funcionarios.listar_por_depto(20);
SELECT pkg_funcionarios.calcular_bonus(8000) FROM dual;

RAC — Real Application Clusters

O Oracle RAC é a solução de alta disponibilidade mais avançada entre os SGBDs. Permite que vários servidores (nós) acessem simultaneamente o mesmo armazenamento compartilhado. Se um nó falhar, os outros continuam operando — o usuário não percebe a interrupção.

Diferença: RAC vs Replicação Simples

AspectoOracle RACReplicação Master-Replica
Escrita Todos os nós escrevem Apenas o master escreve
Armazenamento Compartilhado (ASM/SAN) Independente por servidor
Consistência Imediata entre todos os nós Eventual (delay de replicação)
Failover Automático, transparente Semi-automático (precisamos eleger novo master)
Custo Altíssimo (HW + Licença RAC) Moderado

Cache Fusion: O mecanismo interno do RAC que permite que os nós compartilhem blocos de dados na memória (buffer cache) via rede de alta velocidade (Interconnect), sem precisar ir ao disco toda vez. É o segredo da performance do RAC.

Uma CPU moderna com muitos cores pode custar centenas de milhares de dólares para licenciar. Algumas funcionalidades como Partitioning e RAC custam o mesmo que a licença base. Sempre envolva advogados e consultores Oracle antes de fechar contrato.

Recursos Únicos e Diferenciais

Flashback Technology

Voltar no tempo: consulte dados como estavam em qualquer ponto no passado, desfaça erros de DELETE/UPDATE sem restore de backup. SELECT * FROM tabela AS OF TIMESTAMP ...

Partitioning

Dividir tabelas gigantes em partições físicas (por data, por range, por lista, por hash). Consultas em tabelas com bilhões de linhas ficam rápidas ao eliminar partições irrelevantes.

Advanced Security

Transparent Data Encryption (TDE) — criptografia em repouso sem mudar código da aplicação. Data Masking — mascara dados sensíveis em ambientes de desenvolvimento.

Multitenant (CDB/PDB)

Um Container Database (CDB) pode hospedar múltiplos Pluggable Databases (PDB). Ideal para consolidação de servidores — rodar dezenas de bancos num servidor com custo de administração reduzido.

In-Memory Column Store

Armazenar tabelas em formato colunar na SGA. Consultas analíticas ficam até 100x mais rápidas sem mover dados para um sistema separado.

Data Guard

Sincronização automática com um banco standby em outro datacenter. Em caso de desastre, o failover para o standby é executado em segundos.

Quando Usar Oracle Database

Use Oracle quando...

  • Você é um banco, seguradora ou empresa financeira com regulamentação pesada.
  • Precisa de 99,999% de disponibilidade (RAC + Data Guard).
  • Tem grandes volumes de dados históricos que requerem particionamento inteligente.
  • O orçamento de TI permite investimento enterprise.
  • Precisa de suporte oficial garantido para auditoria e compliance.

Evite Oracle quando...

  • É uma startup ou empresa de médio porte sem budget milionário para licenças.
  • Sua equipe não tem DBAs Oracle certificados disponíveis.
  • Precisa de agilidade e deploys rápidos (o Oracle é complexo de administrar).
  • Quer migrar para cloud sem vendor lock-in com a Oracle.

Veredicto: O Oracle Database é tecnicamente impressionante — difícil encontrar um sistema mais robusto. Mas seu modelo de licenciamento o torna inadequado para a maioria das empresas. Para novos projetos, o PostgreSQL oferece 90% dos recursos enterprise do Oracle gratuitamente. Reserve o Oracle para sistemas legados ou onde SLA contratual exige o portfólio Oracle.
