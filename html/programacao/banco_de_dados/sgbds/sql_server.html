SQL Server — A Potência da Microsoft

O SGBD da Microsoft para o ecossistema Windows, .NET e Azure.

O Microsoft SQL Server é o SGBD padrão do ecossistema Microsoft: profundamente integrado ao Windows, .NET e Azure, com edicões gratuitas para dev e licenças enterprise para produção crítica.

O que é SQL Server?

O Microsoft SQL Server é um SGBD relacional desenvolvido desde 1989 pela Microsoft. Originalmente uma parceria com a Sybase, o SQL Server cresceu para se tornar o banco preferido em empresas que operam no ecossistema Windows e Azure. Em 2017 passou a rodar nativamente no Linux e Docker.

Sua linguagem procedural, o T-SQL (Transact-SQL), é um superconjunto do SQL padrão com extensões proprietárias para controle de fluxo, variáveis e tratamento de erros — semelhante ao PL/SQL do Oracle e ao PL/pgSQL do PostgreSQL.

As edições Developer e Express são gratuitas e suficientes para aprendizado e projetos pequenos — sem nenhuma restrição de funcionalidades na edição Developer.

T-SQL — Transact-SQL

Assim como o Oracle tem PL/SQL, a Microsoft tem o T-SQL um superconjunto do SQL padrão ANSI com extensões proprietárias para variáveis, controle de fluxo, tratamento de erros e operações de alto desempenho.

T-SQL
-- TOP ao invés de LIMIT (diferença do padrão ANSI)
SELECT TOP 5 nome, salario
FROM funcionarios
ORDER BY salario DESC;

-- Variáveis T-SQL começam com @
DECLARE @Bonus INT;
SET @Bonus = 1000;

UPDATE funcionarios
SET salario = salario + @Bonus
WHERE ativo = 1;

-- CTE com ROW_NUMBER  top 3 por departamento
WITH RankedFuncionarios AS (
  SELECT nome, salario, departamento,
         ROW_NUMBER() OVER (PARTITION BY departamento ORDER BY salario DESC) AS rn
  FROM funcionarios
)
SELECT nome, salario, departamento
FROM RankedFuncionarios
WHERE rn <= 3;

-- MERGE (Upsert)  nativo no T-SQL
MERGE INTO clientes AS target
USING clientes_atualizados AS source
  ON target.id = source.id
WHEN MATCHED THEN
  UPDATE SET target.email = source.email, target.nome = source.nome
WHEN NOT MATCHED THEN
  INSERT (id, nome, email)
  VALUES (source.id, source.nome, source.email);

Diferenças do padrão SQL: paginação usa OFFSET n ROWS FETCH NEXT n ROWS ONLY (não LIMIT). Strings usam aspas simples apenas. Data/hora usa GETDATE() em vez de NOW().

Stored Procedures

O T-SQL permite criar rotinas armazenadas diretamente no banco com controle de fluxo completo, tratamento de exceções e transações compiladas e cacheadas pelo servidor para execuções repetidas eficientes.

T-SQL
CREATE OR ALTER PROCEDURE TransferirSaldo
    @ContaOrigem  INT,
    @ContaDestino INT,
    @Valor        DECIMAL(15,2)
AS
BEGIN
    SET NOCOUNT ON;
    DECLARE @SaldoOrigem DECIMAL(15,2);

    BEGIN TRY
        BEGIN TRANSACTION;

        SELECT @SaldoOrigem = saldo
        FROM contas
        WHERE id = @ContaOrigem
        WITH (UPDLOCK, ROWLOCK);  -- Lock evita race condition

        IF @SaldoOrigem < @Valor
            THROW 50001, 'Saldo insuficiente para transferência.', 1;

        UPDATE contas SET saldo = saldo - @Valor WHERE id = @ContaOrigem;
        UPDATE contas SET saldo = saldo + @Valor WHERE id = @ContaDestino;

        INSERT INTO historico_transacoes (origem, destino, valor, realizado_em)
        VALUES (@ContaOrigem, @ContaDestino, @Valor, GETDATE());

        COMMIT TRANSACTION;
    END TRY
    BEGIN CATCH
        IF @@TRANCOUNT > 0
            ROLLBACK TRANSACTION;
        THROW;  -- Re-lança exceção para o caller
    END CATCH;
END;

-- Executar
EXEC TransferirSaldo @ContaOrigem = 101, @ContaDestino = 205, @Valor = 350.00;

Ferramentas

SSMS — SQL Server Management Studio

A ferramenta visual oficial gratuita, considerada por muitos DBAs a melhor IDE de banco de dados jamais criada. Assistentes visuais, diagramas ER, depuração de stored procedures, plano de execução visual e monitoramento de performance. Disponível apenas no Windows.

Azure Data Studio

A alternativa multiplataforma (Windows, macOS, Linux) baseada no VS Code. Mais leve que o SSMS, com suporte a notebooks SQL, extensões da comunidade e integração nativa com Azure. Preferida para desenvolvimento fora do Windows.

SQL Server Profiler

Ferramenta de rastreamento que captura eventos do servidor em tempo real queries executadas, logins, erros, deadlocks. Essencial para diagnóstico de problemas de performance em produção.

Database Engine Tuning Advisor

Analisa uma carga de trabalho e sugere automaticamente índices, estatísticas e views indexadas que melhorariam a performance. Gera scripts de implementação prontos para revisão e aplicação.

Edições e Custo

Diferente do MySQL (totalmente gratuito), o SQL Server tem um modelo com diferentes edições para diferentes cenários.

Developer

Gratuita. Inclui todos os recursos da versão Enterprise, mas licenciada exclusivamente para desenvolvimento e testes não pode ser usada em produção. A escolha para desenvolvedores que querem testar recursos avançados sem nenhum custo.

Express

Gratuita. Pode ser usada em produção, mas com limitações: máximo de 10 GB de dados por banco e restrições de CPU/RAM. Ideal para aplicações pequenas, departamentais e protótipos com baixo volume de dados.

Standard

Paga. Sem os limites do Express. Inclui replicação e recursos de nível médio. Para PMEs que precisam de banco estável e performático sem o custo da edição Enterprise.

Enterprise

Paga ($$$$). Sem nenhuma limitação. Inclui Always On Availability Groups, Columnstore Indexes, criptografia avançada, BI integrado e suporte oficial da Microsoft. Para grandes corporações com sistemas críticos de missão.

Azure SQL Database: A versão totalmente gerenciada na nuvem da Microsoft elimina a necessidade de instalar e manter servidor. Pagamento por uso, escalabilidade automática e backups incluídos. Muito popular em projetos que já operam no ecossistema Azure.

SQL Server no Linux e Docker

Desde o SQL Server 2017, a Microsoft portou o banco para Linux e containers Docker uma mudança histórica que refletiu a nova filosofia open-source adotada sob Satya Nadella.

Shell / T-SQL
# Rodar SQL Server 2022 em Docker
docker run -e "ACCEPT_EULA=Y" -e "MSSQL_SA_PASSWORD=SenhaForte!123" \
  -p 1433:1433 --name sqlserver2022 \
  -d mcr.microsoft.com/mssql/server:2022-latest

# Conectar com sqlcmd de dentro do container
docker exec -it sqlserver2022 /opt/mssql-tools18/bin/sqlcmd \
  -S localhost -U SA -P "SenhaForte!123" -No

-- Verificar versão
SELECT @@VERSION;
GO

-- Criar banco e tabela de teste
CREATE DATABASE MeuBanco;
GO
USE MeuBanco;
CREATE TABLE teste (id INT, nome VARCHAR(50));
INSERT INTO teste VALUES (1, 'Olá SQL Server no Linux!');
SELECT * FROM teste;
GO

Segurança: nunca use senhas hardcoded em produção. Use variáveis de ambiente ou secrets managers (Azure Key Vault, HashiCorp Vault). A conta SA (System Administrator) é o superusuário — desabilite-a em produção e crie contas com os privilégios mínimos necessários.

Recursos Avançados

Always On Availability Groups

Alta disponibilidade empresarial sem shared storage. Um grupo de réplicas sincronizadas que permite failover automático em segundos. Leituras podem ser distribuídas entre réplicas secundárias para escalar horizontalmente.

Columnstore Indexes

Índices orientados a coluna que comprimem dados em até 10 e aceleram queries analíticas (OLAP) em 10 a 100. Perfeitos para data warehouses e relatórios sobre grandes volumes históricos sem mover dados para outro sistema.

In-Memory OLTP (Hekaton)

Tabelas e stored procedures compiladas nativamente que residem em memória. Podem atingir 30 mais throughput que tabelas em disco para cargas OLTP de escrita intensiva, como sistemas de pedidos em pico de carga.

Always Encrypted

Criptografia fim-a-fim onde os dados permanecem criptografados mesmo durante o processamento nem o DBA consegue ver o valor em texto plano. As chaves de criptografia ficam somente no cliente, nunca no servidor.
