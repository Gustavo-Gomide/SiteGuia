Conjuntos

Conjuntos na Programação

Coleções sem duplicatas e com pertinência em tempo constante — onde a Teoria dos Conjuntos de Cantor encontra as hash tables modernas.

Um conjunto (set) é uma coleção de elementos únicos e sem ordem definida. Operações como verificar se um elemento pertence ao conjunto, calcular interseção, união e diferença são a linguagem natural dos conjuntos — e nas linguagens modernas, essas operações são executadas em tempo $O(1)$ ou $O(n)$ graças a hash tables. Compreender conjuntos é essencial para eliminar duplicatas, comparar coleções e modelar relações entre grupos de dados.

O que são Conjuntos

Na matemática, um conjunto é uma coleção bem definida de objetos distintos, chamados elementos ou membros. A Teoria dos Conjuntos, formalizada por Georg Cantor entre 1874 e 1897, é a base sobre a qual grande parte da matemática moderna foi construída. Cantor introduziu os conceitos de cardinalidade, conjuntos infinitos e a notação que usamos até hoje: $\{1, 2, 3\}$, $\emptyset$ (conjunto vazio), $\in$ (pertence a), $\subseteq$ (subconjunto de).

Na programação, os conjuntos herdaram essas propriedades matemáticas e as implementam com eficiência computacional. As duas garantias fundamentais de um conjunto são:

Unicidade

Cada elemento aparece no máximo uma vez. Tentar inserir um elemento já existente é uma operação silenciosa — o conjunto simplesmente permanece inalterado.

unicidade.py
s = {1, 2, 3, 2, 1}
print(s)  # {1, 2, 3} — duplicatas removidas

s.add(4)  # Adiciona 4
s.add(2)  # Já existe, nada muda
print(s)  # {1, 2, 3, 4}

Ausência de Ordem

Os elementos não têm posição fixa. Não é possível acessar s[0]. A cada impressão, a ordem pode variar (depende da implementação de hash interna).

sem_indice.py
s = {"banana", "maçã", "laranja"}

# Isso gera TypeError!
# print(s[0])

# Para iterar, use for
for fruta in s:
    print(fruta)  # ordem arbitrária

Por que conjuntos são rápidos? A maioria das linguagens implementa conjuntos com hash tables. Ao inserir um elemento, um hash é calculado e determina em qual "balde" (bucket) o elemento vai. Para verificar se algo pertence ao conjunto, calcula-se o hash do elemento e verifica-se apenas aquele bucket — operação $O(1)$ em média, independente do tamanho do conjunto.

Raízes Teóricas e Implementação

A teoria dos conjuntos de Cantor foi axiomatizada por Ernst Zermelo (1908) e depois estendida por Abraham Fraenkel — o sistema ZF (Zermelo-Fraenkel) é a mais amplamente aceita fundação da matemática. O axioma da extensionalidade nesse sistema afirma que dois conjuntos são iguais se e somente se possuem exatamente os mesmos elementos — exatamente a semântica de igualdade implementada em linguagens como Python.

Na computação, a implementação eficiente de conjuntos se deve à hash table, estrutura de dados descrita formalmente por Hans Peter Luhn (IBM, 1953) e amplamente desenvolvida por Donald Knuth no TAOCP. Em Python, o tipo set é implementado como uma hash table aberta com resolução de colisão por endereçamento aberto, garantindo $O(1)$ amortizado para inserção, remoção e pertinência.

Criando Conjuntos em Python

Python oferece o tipo nativo set (mutável) e frozenset (imutável). Existem diversas formas de criar um conjunto:

criacao.py
# Literal com chaves {}
a = {1, 2, 3}

# Construtor set() a partir de iterável
b = set([4, 5, 6, 6, 5])  # {4, 5, 6}
c = set("banana")          # {'b', 'a', 'n'}

# Conjunto vazio — DEVE usar set(), não {}
vazio = set()   # Correto
# vazio = {}    # ERRADO — isso cria um dict!

# Set comprehension
quadrados = {x**2 for x in range(6)}  # {0, 1, 4, 9, 16, 25}

# frozenset — imutável, pode ser usado como chave de dict
fs = frozenset({1, 2, 3})
d = {fs: "valor"}  # Válido! frozenset é hashable

Use frozenset quando precisar de um conjunto que nunca muda — como constante global, chave de dicionário ou elemento de outro conjunto.

Operações de Conjuntos

As quatro operações fundamentais da teoria dos conjuntos estão disponíveis diretamente em Python, tanto como operadores simbólicos quanto como métodos. Os diagramas de Venn abaixo mostram visualmente o resultado de cada operação.

União — A ∪ B

Todos os elementos que pertencem a A, a B, ou a ambos.

uniao.py
A = {1, 2, 3, 4}
B = {3, 4, 5, 6}

# Operador |
print(A | B)  # {1, 2, 3, 4, 5, 6}

# Método
print(A.union(B))

Interseção — A ∩ B

Apenas os elementos que pertencem a ambos A e B simultaneamente.

intersecao.py
A = {1, 2, 3, 4}
B = {3, 4, 5, 6}

# Operador &
print(A & B)  # {3, 4}

# Método
print(A.intersection(B))

Diferença — A − B

Elementos que pertencem a A mas não pertencem a B. A operação não é comutativa: A − B ≠ B − A.

diferenca.py
A = {1, 2, 3, 4}
B = {3, 4, 5, 6}

# Operador -
print(A - B)  # {1, 2}  — só em A
print(B - A)  # {5, 6}  — só em B

# Método
print(A.difference(B))

Diferença Simétrica — A △ B

Elementos que estão em A ou B, mas não em ambos. Equivale a (A ∪ B) − (A ∩ B).

dif_simetrica.py
A = {1, 2, 3, 4}
B = {3, 4, 5, 6}

# Operador ^
print(A ^ B)  # {1, 2, 5, 6}

# Método
print(A.symmetric_difference(B))

Relações entre Conjuntos

Além das operações que produzem novos conjuntos, é possível verificar relações entre dois conjuntos: subconjunto, superconjunto e disjunção.

relacoes.py
A = {1, 2, 3}
B = {1, 2, 3, 4, 5}
C = {10, 20}

# Subconjunto: A ⊆ B (todo elem. de A está em B)
print(A <= B)            # True
print(A.issubset(B))     # True

# Subconjunto próprio: A ⊂ B (A ⊆ B e A ≠ B)
print(A < B)             # True
print(A < A)             # False — não é próprio

# Superconjunto: B ⊇ A
print(B >= A)            # True
print(B.issuperset(A))   # True

# Disjuntos: sem elementos em comum
print(A.isdisjoint(C))   # True — A ∩ C = ∅
print(A.isdisjoint(B))   # False — têm 1,2,3 em comum

# Igualdade: mesmos elementos
print({1,2,3} == {3,1,2})  # True — ordem irrelevante

Pertinência: o operador in

A principal vantagem de conjuntos sobre listas é o teste de pertinência em $O(1)$.

pertinencia.py
import time

dados = list(range(1_000_000))
dados_set = set(dados)

alvo = 999_999

# Busca em lista — O(n), percorre do início
t = time.perf_counter()
alvo in dados
t_lista = time.perf_counter() - t

# Busca em set — O(1), direto via hash
t = time.perf_counter()
alvo in dados_set
t_set = time.perf_counter() - t

# Resultado típico: lista ~10ms, set ~0.0001ms
print(f"Lista: {t_lista:.6f}s")
print(f"Set:   {t_set:.6f}s")

Métodos do set em Python

Além dos operadores simbólicos, set oferece métodos para modificação in-place e variantes que aceitam qualquer iterável (não apenas sets).

Métodos de Modificação (in-place)

metodos_mutacao.py
s = {1, 2, 3}

# add() — adiciona um elemento
s.add(4)          # {1, 2, 3, 4}

# remove() — remove; lança KeyError se não existir
s.remove(4)       # {1, 2, 3}
# s.remove(99)   # KeyError!

# discard() — remove; silencioso se não existir
s.discard(99)     # Nenhum erro

# pop() — remove e retorna elemento arbitrário
x = s.pop()       # Remove algum elemento

# clear() — esvazia o conjunto
s.clear()         # set()

# update() — adiciona vários elementos de iterável
s.update([5, 6, 7])

Operações In-Place (|=, &=, -=, ^=)

inplace.py
A = {1, 2, 3}
B = {3, 4, 5}

# Modificam A diretamente (sem criar novo set)
A |= B  # União in-place: {1,2,3,4,5}

A = {1, 2, 3}
A &= B  # Interseção in-place: {3}

A = {1, 2, 3}
A -= B  # Diferença in-place: {1, 2}

A = {1, 2, 3}
A ^= B  # Dif. simétrica in-place: {1,2,4,5}

# Métodos equivalentes aceitam qualquer iterável:
A.update([6, 7])              # |= com lista
A.intersection_update([1,6]) # &= com lista
A.difference_update([1])     # -= com lista

Casos de Uso Práticos

Conjuntos resolvem de forma elegante e eficiente uma categoria específica de problemas. Reconhecer quando usar um conjunto versus uma lista ou dicionário é uma marca do programador experiente.

Remover Duplicatas

A forma mais rápida de eliminar elementos repetidos de uma coleção.

dedup.py
emails = [
    "a@x.com", "b@x.com",
    "a@x.com", "c@x.com",
    "b@x.com"
]

unicos = list(set(emails))
print(unicos)
# ['a@x.com', 'b@x.com', 'c@x.com']
# (ordem pode variar)

Teste de Pertinência Frequente

Quando você precisa verificar muitas vezes se algo está em uma coleção grande.

blacklist.py
# IPs banidos — consulta milhões de vezes por dia
ips_banidos = set([
    "192.168.1.1",
    "10.0.0.5",
    # ... milhares de IPs
])

def verificar_acesso(ip: str) -> bool:
    return ip not in ips_banidos  # O(1)

Encontrar Elementos Comuns

Descobrir quais elementos dois grupos compartilham.

comum.py
seguidores_ana  = {"alice", "bob", "carol"}
seguidores_beto = {"bob", "dave", "carol"}

# Amigos em comum
comuns = seguidores_ana & seguidores_beto
print(comuns)  # {'bob', 'carol'}

# Quem segue apenas Ana
so_ana = seguidores_ana - seguidores_beto
print(so_ana)  # {'alice'}

Rastrear Itens Visitados

Em algoritmos de busca (BFS, DFS), rastrear nós já visitados para evitar ciclos.

bfs.py
from collections import deque

def bfs(grafo, inicio):
    visitados = set()       # O(1) para checar
    fila = deque([inicio])

    while fila:
        no = fila.popleft()
        if no in visitados:
            continue
        visitados.add(no)     # Marca como visto
        fila.extend(grafo[no])

    return visitados

Contagem de Elementos Únicos

Contar quantos valores distintos existem em uma coleção.

unicos.py
notas_alunos = [7, 8, 7, 9, 6, 8, 10, 6]

print(len(set(notas_alunos)))  # 5 notas distintas

palavras = "o gato e o rato e o pato".split()
vocab = set(palavras)
print(len(vocab))  # 5 palavras únicas

Validação de Permissões

Verificar se um usuário tem todas as permissões necessárias para uma ação.

permissoes.py
permissoes_usuario = {"ler", "escrever", "deletar"}
permissoes_admin   = {"ler", "escrever", "admin"}

def pode_acessar(usuario, necessarias):
    # issubset: todas necessárias estão em usuario?
    return necessarias.issubset(usuario)

requer = {"ler", "escrever"}
print(pode_acessar(permissoes_usuario, requer)) # True
print(pode_acessar(permissoes_usuario, {"admin"})) # False

Boas Práticas

Quando Usar Conjuntos

  1. Use set quando precisar de pertinência frequente: Se você vai executar x in colecao muitas vezes, converta para set. A diferença de $O(n)$ para $O(1)$ é drástica em grandes coleções.
  2. Use set para eliminar duplicatas: list(set(colecao)) é a forma mais idiomática e eficiente. Atenção: a ordem original não é preservada.
  3. Use frozenset como chave de dicionário ou elemento de set: Quando precisar de um conjunto imutável que sirva como identificador único de um grupo.
  4. Prefira operadores simbólicos por legibilidade: A | B é mais claro que A.union(B) quando ambos os operandos já são sets. Use os métodos quando o segundo operando pode ser qualquer iterável.
  5. Não use set se a ordem importa: Para coleções onde a sequência importa (histórico, log, ranking), use listas. Desde Python 3.7, dicionários preservam ordem de inserção e podem ajudar em alguns casos.

Elementos hashable: Sets só aceitam elementos hashable — aqueles que implementam __hash__() e __eq__(). Tipos imutáveis (int, str, tuple, frozenset) são hashable. Tipos mutáveis (list, dict, set) não são. Tentar adicionar uma lista a um set gera TypeError: unhashable type.

Complexidade das Operações

A eficiência dos conjuntos se deve à hash table subjacente. Compreender a complexidade de cada operação orienta escolhas de design corretas:

OperaçãoNotaçãoComplexidade MédiaComplexidade Pior Caso
x in sPertinência$O(1)$$O(n)$
s.add(x)Inserção$O(1)$$O(n)$
s.remove(x)Remoção$O(1)$$O(n)$
len(s)Cardinalidade$O(1)$$O(1)$
s | tUnião$O(|s|+|t|)$$O(|s| \cdot |t|)$

O pior caso $O(n)$ ocorre apenas com muitas colisões de hash, situação praticamente impossível com tipos nativos Python.

Erros Comuns

Conjunto vazio com {}

Colchetes vazios criam um dicionário, não um conjunto.

vazio.py
# ERRADO
s = {}
print(type(s))  # <class 'dict'> !

# CORRETO
s = set()
print(type(s))  # <class 'set'>

Elemento não-hashable

Listas e dicionários não podem ser elementos de um set.

unhashable.py
# TypeError — lista não é hashable
s = {[1, 2], [3, 4]}  # Erro!

# Use tuplas (imutáveis, hashable)
s = {(1, 2), (3, 4)}  # OK

# Ou frozenset de frozensets
s = {frozenset({1,2}), frozenset({3,4})}  # OK

Assumir ordem de iteração

A ordem de iteração de um set é determinística (dada a mesma execução), mas não garante que seja a ordem de inserção.

ordem.py
s = {3, 1, 4, 1, 5, 9}

# ERRADO — não assuma ordem
primeiro = list(s)[0]  # Pode ser qualquer um!

# Se precisar de ordem, converta e ordene
ordenado = sorted(s)
print(ordenado)  # [1, 3, 4, 5, 9] — lista ordenada

Modificar set durante iteração

Assim como listas, modificar um set durante iteração causa RuntimeError.

iter_mod.py
s = {1, 2, 3, 4, 5}

# ERRADO — RuntimeError
for x in s:
    if x % 2 == 0:
        s.remove(x)  # Erro!

# CORRETO — itere sobre cópia
pares = {x for x in s if x % 2 == 0}
s -= pares

Tópicos Avançados

Conjuntos com objetos personalizados

Para que instâncias de classes personalizadas possam ser inseridas em sets, é necessário implementar __hash__() e __eq__(). Dois objetos considerados iguais por __eq__ devem ter o mesmo hash.

custom_hashable.py
class Ponto:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, outro):
        return self.x == outro.x and self.y == outro.y

    def __hash__(self):
        return hash((self.x, self.y))  # Baseado nos atributos

    def __repr__(self):
        return f"Ponto({self.x}, {self.y})"

pontos = {Ponto(0, 0), Ponto(1, 2), Ponto(0, 0)}
print(pontos)  # {Ponto(0, 0), Ponto(1, 2)} — duplicata removida

Multiconjuntos com Counter

Um multiconjunto (ou bag) permite duplicatas, mantendo a contagem de ocorrências. Python oferece collections.Counter para esse propósito.

counter.py
from collections import Counter

texto = "o rato roeu a roupa do rei de roma"
contagem = Counter(texto.split())

print(contagem.most_common(3))
# [('o', 2), ('de', 2), ('rato', 1)]

# Operações similares a conjuntos
a = Counter({"maçã": 3, "banana": 2})
b = Counter({"maçã": 1, "laranja": 4})

print(a + b)  # maçã:4, banana:2, laranja:4
print(a & b)  # maçã:1 (mínimo)
print(a | b)  # maçã:3 (máximo)

Bloom Filters — conjuntos probabilísticos

Para conjuntos extremamente grandes onde memória é crítica, Bloom Filters (Burton Howard Bloom, 1970) oferecem teste de pertinência probabilístico: nunca produzem falsos negativos, mas podem ter falsos positivos com probabilidade controlável. Usados em sistemas de cache (Chrome Safe Browsing, Cassandra, Redis) e bioinformática.

Um Bloom filter com 1% de taxa de falso positivo usa apenas ~9.6 bits por elemento — ordens de magnitude mais eficiente que uma hash table completa. A biblioteca pybloom-live oferece implementação pronta para Python.
