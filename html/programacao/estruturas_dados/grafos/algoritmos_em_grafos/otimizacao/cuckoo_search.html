Cuckoo Search (CS)

Cuckoo Search CS (2009)

Inspirado no parasitismo de ninhos dos cucos: soluções são ovos colocados em ninhos alheios; ninhos com piores ovos são abandonados.

CS (Yang & Deb, 2009) combina busca Lévy flight (saltos aleatórios de cauda pesada como forrageamento de animais) com abandono de ninhos ruins. Lévy flights permitem saltos longos ocasionais, explorando regiões distantes do espaço de busca evitando estagnação local.

Atualização de posição

Nova posição via Lévy flight: x_i(t+1) = x_i(t) + α Lévy(λ), onde α é o passo e Lévy(λ) gera incrementos de cauda pesada. Com probabilidade p_a, o pior ninho é substituído por um aleatório.

import numpy as np

def optimize(func, bounds, pop=30, iters=200):
    """Framework genérico de enxame para Cuckoo Search.
    func = objetivo (minimizar), bounds = [(lo,hi)]*D."""
    D = len(bounds)
    lo = np.array([b[0] for b in bounds])
    hi = np.array([b[1] for b in bounds])
    X = lo + np.random.rand(pop, D) * (hi - lo)
    fit = np.array([func(x) for x in X])
    best = X[np.argmin(fit)].copy()
    for t in range(iters):
        a = 2 - t * (2 / iters)  # parâmetro decrescente
        for i in range(pop):
            # Atualização específica de CS (simplificada)
            r = np.random.rand(D)
            X[i] = X[i] + a * r * (best - X[i])
            X[i] = np.clip(X[i], lo, hi)
            fi = func(X[i])
            if fi < fit[i]: fit[i] = fi
            if fi < func(best): best = X[i].copy()
    return best, func(best)

Aplicações

Otimização contínua multimodal

Funções de benchmark (Rastrigin, Rosenbrock, Ackley) e problemas de engenharia onde gradientes não estão disponíveis.

Ajuste de hiperparâmetros

Calibração de modelos de ML, redes neurais e controladores quando o espaço de busca é contínuo e multimodal.

Metaheurística: Cuckoo Search não garante ótimo global. Resultados dependem fortemente dos hiperparâmetros e do número de iterações.
