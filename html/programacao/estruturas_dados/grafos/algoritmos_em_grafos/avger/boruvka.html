Borůvka: MST por componentes

O algoritmo MST mais antigo e naturalmente paralelo: cada componente encontra sua aresta mais barata simultaneamente.

O algoritmo de Borůvka (1926) é o precursor de Kruskal e Prim. A ideia: em cada fase, cada componente conectado seleciona sua aresta de menor peso que cruza o corte (sai do componente). Todas as arestas selecionadas são adicionadas ao MST fazendo o número de componentes cair pela metade a cada fase. Após O(log V) fases: O(E log V).

Vantagem: paralelismo natural

Em cada fase, todos os componentes podem encontrar sua aresta mínima independentemente e em paralelo. Isso torna Borůvka a base de algoritmos MST paralelos e distribuídos.

Implementação em Python

boruvka.py
def boruvka(n, arestas):
    """n = número de vértices (0..n-1).
    arestas = [(u, v, w)]. O(E log V)."""
    pai = list(range(n))

    def find(x):
        while pai[x] != x: pai[x] = pai[pai[x]]; x = pai[x]
        return x

    def union(a, b):
        pai[find(a)] = find(b)

    mst, custo, componentes = [], 0, n

    while componentes > 1:
        barata = [None] * n   # aresta mais barata de cada componente
        for u, v, w in arestas:
            cu, cv = find(u), find(v)
            if cu == cv: continue
            if barata[cu] is None or w < barata[cu][2]:
                barata[cu] = (u, v, w)
            if barata[cv] is None or w < barata[cv][2]:
                barata[cv] = (u, v, w)

        for aresta in barata:
            if aresta is None: continue
            u, v, w = aresta
            if find(u) != find(v):
                union(u, v); mst.append(aresta); custo += w; componentes -= 1

    return mst, custo

# Exemplo
arestas = [(0,1,10),(0,2,6),(0,3,5),(1,3,15),(2,3,4)]
mst, total = boruvka(4, arestas)
print(total)  # 19

Aplicações

MST paralela e distribuída

Borůvka é a base de algoritmos MST em sistemas distribuídos cada nó processa seu componente de forma independente.

Algoritmos lineares esperados

Combinado com amostragem aleatória, Borůvka é parte de algoritmos MST esperados O(E) como o de Karger-Klein-Tarjan.

Atenção: Para simplificar, a implementação acima pode adicionar duplicatas se houver peso empatado use um set de arestas já adicionadas ou verify find() antes de union() para evitar isso.
