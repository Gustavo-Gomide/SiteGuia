Prim: MST crescendo de um vértice

A árvore geradora mínima construída vértice a vértice sempre adicionando o vizinho mais barato à fronteira atual.

Prim constrói a MST partindo de um vértice e expandindo a árvore sempre pela aresta de menor peso que cruza o corte entre vértices já incluídos e vértices ainda fora. Usando um heap de mínimo: O((V + E) log V). Muito similar ao Dijkstra a diferença é que Prim armazena o custo da aresta mais barata para entrar na árvore, não a distância desde a origem.

Prim vs Kruskal

Prim cresce localmente a partir de um vértice. Bom para grafos densos (lida bem com E grande).
Kruskal considera todas as arestas globalmente. Bom para grafos esparsos (é dominado pelo sort de E arestas).

Implementação em Python

prim.py
import heapq

def prim(grafo, inicio):
    """grafo = {v: [(viz, peso)]}. O((V+E) log V).
    Retorna lista de arestas MST e custo total."""
    visitado = {inicio}
    # heap: (peso, origem, destino)
    heap = [(w, inicio, v) for v, w in grafo[inicio]]
    heapq.heapify(heap)
    mst, custo = [], 0

    while heap and len(visitado) < len(grafo):
        w, u, v = heapq.heappop(heap)
        if v in visitado: continue
        visitado.add(v); mst.append((u, v, w)); custo += w
        for viz, ww in grafo[v]:
            if viz not in visitado:
                heapq.heappush(heap, (ww, v, viz))
    return mst, custo

# Exemplo
g = {'A':[('B',1),('C',4)], 'B':[('A',1),('C',3),('D',2)],
     'C':[('A',4),('B',3),('D',5)], 'D':[('B',2),('C',5)]}
mst, total = prim(g, 'A')
print(mst)    # [('A','B',1), ('B','D',2), ('B','C',3)]
print(total)  # 6

Aplicações

Redes densas

Para grafos com muitas arestas (E V²), Prim com array simples O(V²) é comparável ou melhor que Kruskal O(E log E).

Design de redes

Planejamento de redes de distribuição (água, energia, dados) onde cada extensão tem custo.

Aproximação TSP

A MST de Prim é usada como base para a aproximação 2x do Problema do Caixeiro Viajante.

Atenção: Prim, como Kruskal, só garante a MST em grafos não-direcionados e conectados. Certifique-se de que o grafo é conexo antes de executar; caso contrário, a árvore gerada não cobrirá todos os vértices.
