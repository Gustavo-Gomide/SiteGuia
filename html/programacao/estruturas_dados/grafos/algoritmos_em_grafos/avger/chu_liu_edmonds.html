Chu-Liu / Edmonds

Arborescência mínima: MST em grafos direcionados

O algoritmo de Chu-Liu/Edmonds encontra a árvore geradora mínima em grafos direcionados um problema qualitativamente mais difícil que o caso não-direcionado.

Em grafos não-direcionados, três algoritmos resolvem MST em O(E log V). Em grafos direcionados, pedir uma arborescência (árvore enraizada onde todo vértice é alcançável pela raiz) é diferente. O algoritmo de Chu-Liu/Edmonds (1965/1967) garante a solução ótima em O(EV) e, com filas de prioridade de Fibonacci, em O(E log V).

Arborescência vs MST

Uma arborescência com raiz r é uma árvore geradora direcionada onde todos os vértices são alcançáveis a partir de r seguindo as arestas. Não basta que o grafo seja conexo: a direção das arestas importa.

Ideia do algoritmo

Para cada vértice (exceto a raiz), escolha a aresta de entrada de menor peso. Se as arestas escolhidas formam uma arborescência: fim. Se formam um ciclo: contraia o ciclo em um super-nó, ajuste os pesos das arestas de entrada (removendo o peso da aresta que seria substituída) e recurse. Após expandir recursivamente, montamos a arborescência ótima.

Implementação em Python

chu_liu_edmonds.py
def chu_liu_edmonds(n, raiz, arestas):
    """n = vértices, raiz = índice, arestas = [(u,v,w)].
    Retorna (custo, lista de arestas da arborescência)."""
    INF = float('inf')

    while True:
        # Aresta de menor peso entrando em cada vértice
        min_in = [(None, INF)] * n
        for u, v, w in arestas:
            if v != raiz and w < min_in[v][1]:
                min_in[v] = (u, w)

        # Detectar ciclos com as arestas escolhidas
        visitado = [-1] * n
        ciclo_id = [-1] * n
        num_ciclos = 0
        for v in range(n):
            if v == raiz: continue
            u = v
            while u != raiz and visitado[u] == -1:
                visitado[u] = v; u = min_in[u][0]
            if u != raiz and visitado[u] == v:
                # Encontrou ciclo: marcá-lo
                c = u
                while ciclo_id[c] == -1:
                    ciclo_id[c] = num_ciclos; c = min_in[c][0]
                num_ciclos += 1

        if num_ciclos == 0:
            return sum(w for _,w in min_in if w < INF)

        # Contrair ciclos em super-nós
        novo_id = [ciclo_id[v] if ciclo_id[v] != -1
                   else num_ciclos + v for v in range(n)]
        novo_n = num_ciclos + n
        nova_raiz = novo_id[raiz]
        novas_arestas = []
        for u, v, w in arestas:
            nu, nv = novo_id[u], novo_id[v]
            if nu != nv:
                adj = w - (min_in[v][1] if ciclo_id[v] >= 0 else 0)
                novas_arestas.append((nu, nv, adj))
        n, raiz, arestas = novo_n, nova_raiz, novas_arestas

Aplicações

Redes de transmissão direcionadas

Encontrar a rede de custo mínimo para distribuir informação de uma fonte para todos os nós e.g., multicast direcionado.

Análise de dependências

Em grafos de dependência (pacotes, tarefas), a arborescência mínima representa a forma mais barata de satisfazer todas as dependências a partir de um ponto raiz.

Diferença crítica: Kruskal e Prim não funcionam em grafos direcionados para este problema. A presença de ciclos direcionados exige a contração iterativa de Chu-Liu/Edmonds.

Enquanto MST em grafos não-direcionados tem três algoritmos clássicos eficientes, a versão direcionada demandou 40 anos a mais para ser resolvida e o algoritmo resultante é uma das pérolas algorítmicas do século XX.

"Contraia o ciclo, ajuste os pesos, recurse. A elegância de Chu-Liu/Edmonds está na redução recursiva de um problema difícil a uma instância menor de si mesmo." Edmonds, 1967
