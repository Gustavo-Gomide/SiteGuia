Edmonds Blossom: matching em grafos gerais

O algoritmo que resolveu o emparelhamento máximo para grafos gerais a contração de flores elimina ciclos ímpares que confundem outros métodos.

Hopcroft-Karp funciona apenas para grafos bipartidos. Em grafos gerais, ciclos de tamanho ímpar (flores) impedem que BFS simples encontre caminhos aumentantes corretamente. O algoritmo de Edmonds (1965) resolve isso contraindo flores em super-vértices e aplicando busca no grafo contraído. Complexidade: O(V³); com melhorias: O(VE).

O que é uma flor (blossom)?

Uma flor é um ciclo de comprimento ímpar em que exatamente um vértice (a "raiz" ou "base") é alcançável por um caminho alternante a partir de um vértice livre. Ao contrair toda a flor em um único vértice, o caminho aumentante pode ser encontrado normalmente e depois "expandido" para recuperar o caminho original.

def blossom_matching(n, adj):
    """n = vértices, adj = lista de adjacência.
    Retorna tamanho do emparelhamento máximo."""
    match = [-1] * n  # match[v] = parceiro de v (-1 se livre)
    resultado = 0

    for s in range(n):
        if match[s] != -1: continue
        # Tenta encontrar caminho aumentante a partir de s
        if _augment(s, n, adj, match):
            resultado += 1  # encontrou caminho aumentante
    return resultado

Aplicações

Emparelhamento em redes sociais

Quando o grafo não é bipartido (usuários podem se conectar com qualquer outro), Blossom é o único algoritmo polinomial correto.

Heurísticas para TSP

O algoritmo de Christofides para TSP usa emparelhamento de peso mínimo em grafos gerais (subgrafo de grau ímpar).

Complexidade de implementação: Blossom é notoriamente difícil de implementar corretamente. Em produção, use networkx.max_weight_matching ou a biblioteca blossom5.
