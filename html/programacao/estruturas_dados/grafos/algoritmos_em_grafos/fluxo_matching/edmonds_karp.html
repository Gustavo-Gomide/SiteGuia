Edmonds-Karp: Ford-Fulkerson com BFS

Uma modificação simples usar BFS em vez de DFS transforma Ford-Fulkerson em um algoritmo com garantia polinomial robusta: O(VE²).

Edmonds-Karp (1972) é Ford-Fulkerson com a restrição de que o caminho aumentante seja sempre o mais curto no grafo residual (BFS). Essa escolha garante que o comprimento dos caminhos aumentantes nunca decresce, o que limita o número total de aumentos a O(VE), resultando em O(VE²) no total.

Por que BFS funciona melhor

DFS pode selecionar caminhos longos e desnecessariamente desfazer fluxo bom. BFS seleciona o caminho mais curto em número de arestas o que pode parecer subótimo por aresta, mas garante que cada aresta se torna saturada no máximo O(V) vezes, limitando as iterações.

Implementação em Python

edmonds_karp.py
from collections import deque, defaultdict

class EdmondsKarp:
    def __init__(self, n):
        self.n = n
        self.cap = [[0] * n for _ in range(n)]

    def add_edge(self, u, v, c):
        self.cap[u][v] += c

    def _bfs(self, s, t, pai):
        visitado = [False] * self.n
        visitado[s] = True
        fila = deque([s])
        while fila:
            u = fila.popleft()
            for v in range(self.n):
                if not visitado[v] and self.cap[u][v] > 0:
                    visitado[v] = True; pai[v] = u; fila.append(v)
                    if v == t: return True
        return False

    def max_flow(self, s, t):
        total = 0
        pai = [-1] * self.n
        while self._bfs(s, t, pai):
            f = float('inf')
            v = t
            while v != s:
                u = pai[v]; f = min(f, self.cap[u][v]); v = u
            v = t
            while v != s:
                u = pai[v]; self.cap[u][v] -= f; self.cap[v][u] += f; v = u
            total += f; pai = [-1] * self.n
        return total

# Exemplo
g = EdmondsKarp(6)
g.add_edge(0,1,16); g.add_edge(0,2,13)
g.add_edge(1,3,12); g.add_edge(2,4,14)
g.add_edge(3,5,20); g.add_edge(4,5,4)
g.add_edge(2,1,4); g.add_edge(3,2,9); g.add_edge(4,3,7)
print(g.max_flow(0, 5))  # 23

Aplicações

Planejamento de redes

Calcular a capacidade máxima de transporte em redes logísticas, de telecomunicações ou elétricas.

Emparelhamento bipartido

Edmonds-Karp é a escolha comum para resolver matching bipartido via redução a fluxo máximo.

Limitação: Para grafos muito densos ou grandes, Dinic (O(V²E)) ou push-relabel (O(V²E)) são mais eficientes na prática.

Uma modificação de uma linha BFS em vez de DFS é suficiente para transformar um método sem garantia de convergência em um algoritmo polinomial. Pequenas escolhas algorítmicas têm grandes consequências.

"Use sempre o caminho mais curto e você garante que nenhum trabalho será repetido demais." Edmonds & Karp, 1972
