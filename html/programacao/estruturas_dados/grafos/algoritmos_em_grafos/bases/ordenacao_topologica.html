Ordenação Topológica: a ordem das dependências

Dado um DAG, encontrar uma ordem linear dos vértices que respeite todas as dependências a espinha dorsal de pipelines, sistemas de build e makefiles.

A ordenação topológica de um DAG (Directed Acyclic Graph) é uma sequência de vértices onde, para toda aresta (u v), u aparece antes de v. É a formalização de "faça X antes de Y". Só existe em grafos sem ciclos a presença de um ciclo significa que as dependências são circulares e nenhuma ordem é válida.

Exemplo: compilação com dependências

Uma ordenação topológica válida: A, B, C, D, E (ou A, B, D, C, E múltiplas ordens podem ser válidas).

Toda ordenação topológica válida começa pelos vértices sem predecessores (grau de entrada 0) e termina nos vértices sem sucessores. O algoritmo de Kahn processa exatamente nessa ordem.

Implementação: Algoritmo de Kahn

toposort_kahn.py
from collections import deque, defaultdict

def topo_kahn(grafo, vertices):
    """Kahn's algorithm  O(V+E). Detecta ciclos se resultado incompleto."""
    grau_entrada = defaultdict(lambda: 0, {v: 0 for v in vertices})
    for u in grafo:
        for v in grafo[u]:
            grau_entrada[v] += 1

    fila = deque(v for v in vertices if grau_entrada[v] == 0)
    ordem = []

    while fila:
        u = fila.popleft()
        ordem.append(u)
        for v in grafo.get(u, []):
            grau_entrada[v] -= 1
            if grau_entrada[v] == 0:
                fila.append(v)

    if len(ordem) != len(vertices):
        raise ValueError("Grafo tem ciclo  ordenação topológica impossível")
    return ordem

# Exemplo
g = {'A':['C','D'], 'B':['D'], 'C':['E'], 'D':['E'], 'E':[]}
print(topo_kahn(g, ['A','B','C','D','E']))
# ['A', 'B', 'C', 'D', 'E']

Aplicações

Sistemas de build

make, cmake e Gradle calculam a ordem de compilação via ordenação topológica das dependências de módulos.

Gerenciadores de pacotes

npm, pip e apt determinam a ordem de instalação de pacotes com dependências via toposort.

Escalonamento de tarefas

Pipelines de CI/CD e workflows de ETL precisam garantir que tarefas predecessoras terminem antes das sucessoras.

Atenção: Ordenação topológica exige um DAG grafo direcionado acíclico. Se o grafo tiver ciclos (dependência circular), não existe solução. Detectar e reportar esse ciclo ao usuário é tão importante quanto o algoritmo em si.
