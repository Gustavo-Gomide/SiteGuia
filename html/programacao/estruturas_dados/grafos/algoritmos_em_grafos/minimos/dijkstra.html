Dijkstra: caminho mínimo de fonte única

O algoritmo de referência para caminhos mais curtos em grafos com pesos não-negativos base do GPS, roteamento IP e muito mais.

Dijkstra resolve o problema do caminho mínimo de fonte única (SSSP): dado um vértice de origem, encontrar o caminho de menor custo até todos os outros vértices. Funciona para grafos ponderados não-negativos, direcionados ou não. Com heap de mínimo, atinge O((V + E) log V).

Ideia: relaxamento ganancioso

Dijkstra processa vértices em ordem crescente de distância estimada. Para cada vértice processado, relaxa suas arestas se a distância via esse vértice for menor que a distância atual do vizinho, atualiza.

Invariante: quando um vértice é retirado do heap, sua distância é definitiva nenhum caminho mais curto pode existir (garantia válida apenas para pesos 0).

Implementação em Python

dijkstra.py
import heapq

def dijkstra(grafo, origem):
    """SSSP em O((V+E) log V). grafo = {v: [(viz, peso), ...]}.
    Retorna dist (distâncias mínimas) e pai (caminho)."""
    dist = {v: float('inf') for v in grafo}
    dist[origem] = 0
    pai = {origem: None}
    heap = [(0, origem)]

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue   # entrada obsoleta
        for v, w in grafo[u]:
            nd = dist[u] + w
            if nd < dist[v]:
                dist[v] = nd; pai[v] = u
                heapq.heappush(heap, (nd, v))
    return dist, pai

def caminho(pai, destino):
    p, v = [], destino
    while v is not None: p.append(v); v = pai.get(v)
    return p[::-1]

# Exemplo
g = {'A':[('B',4),('C',2)], 'B':[('D',3)],
     'C':[('B',1),('D',5)], 'D':[]}
dist, pai = dijkstra(g, 'A')
print(dist)   # {'A':0, 'B':3, 'C':2, 'D':6}
print(caminho(pai, 'D'))  # ['A','C','B','D']

Quando usar

GPS e mapas

Encontrar a rota mais rápida/curta entre dois pontos numa rede viária com pesos (distância ou tempo).

Roteamento de redes

OSPF (Open Shortest Path First) usa Dijkstra para calcular tabelas de roteamento em redes IP.

Jogos

Pathfinding em mapas de tiles com custos variados de terreno (barro=custo alto, estrada=custo baixo).

Atenção: Dijkstra produz resultados incorretos com arestas de peso negativo. Para grafos com pesos negativos (sem ciclos negativos), use Bellman-Ford. Com ciclos negativos, o problema não tem solução.
