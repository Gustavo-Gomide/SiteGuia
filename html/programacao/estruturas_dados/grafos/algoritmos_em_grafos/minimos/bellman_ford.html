Bellman-Ford: caminhos mínimos com pesos negativos

O algoritmo que Dijkstra não consegue fazer: tratar pesos negativos e detectar ciclos negativos ao custo de uma complexidade maior.

Bellman-Ford resolve o SSSP (Single Source Shortest Path) para grafos com pesos negativos. Enquanto Dijkstra necessita de pesos não-negativos, Bellman-Ford funciona desde que não exista ciclo negativo acessível a partir da origem. Se houver, o algoritmo detecta e avisa o caminho mínimo seria menos infinito.

Ideia: V-1 rodadas de relaxamento

Em um grafo com V vértices, o caminho mais curto sem ciclos tem no máximo V-1 arestas. Bellman-Ford relaxa todas as arestas V-1 vezes, garantindo que o caminho ótimo de qualquer tamanho seja encontrado. Após a V-ésima rodada, se ainda houver relaxamentos possíveis, existe um ciclo negativo.

Implementação em Python

bellman_ford.py
def bellman_ford(arestas, vertices, origem):
    """arestas = [(u, v, peso), ...]. O(V*E).
    Retorna dist ou levanta erro se ciclo negativo."""
    dist = {v: float('inf') for v in vertices}
    dist[origem] = 0
    pai = {origem: None}

    # V-1 rodadas de relaxamento
    for _ in range(len(vertices) - 1):
        for u, v, w in arestas:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                pai[v] = u

    # Detectar ciclo negativo
    for u, v, w in arestas:
        if dist[u] + w < dist[v]:
            raise ValueError("Ciclo negativo detectado!")

    return dist, pai

# Exemplo com peso negativo
arestas = [('A','B',4),('A','C',5),
           ('B','C',-3),('C','D',2)]
dist, _ = bellman_ford(arestas, ['A','B','C','D'], 'A')
print(dist)  # {'A':0, 'B':4, 'C':1, 'D':3}

Aplicações

Finanças / arbitragem

Detectar oportunidades de arbitragem em câmbio de moedas um ciclo negativo no log dos preços indica lucro garantido.

Roteamento BGP

O protocolo BGP usa variantes de Bellman-Ford (distance-vector) para calcular rotas entre sistemas autônomos.

Validação de restrições

Verificar consistência de sistemas de restrições de diferença (difference constraints) base de análise de programas.

Atenção: Para grafos esparsos e sem pesos negativos, Dijkstra é sempre preferível O(VE) pode ser O(V³) no pior caso. Use Bellman-Ford apenas quando pesos negativos forem inevitáveis ou quando detectar ciclos negativos for um requisito.
