Algoritmo A* (A-Star)

Algoritmo A*

A*: Dijkstra com bússola

Caminho mínimo guiado por heurística encontra o caminho ótimo explorando muito menos nós que Dijkstra quando a heurística é boa.

A* combina o custo real acumulado g(n) com uma estimativa heurística h(n) até o destino: f(n) = g(n) + h(n). Com uma heurística admissível (que nunca superestima), A* garante o caminho ótimo como Dijkstra mas direciona a busca, examinando muito menos vértices.

f(n) = g(n) + h(n)

g(n): custo real do caminho da origem até n (igual ao Dijkstra).
h(n): estimativa (heurística) de n até o destino deve ser admissível (h custo real).
f(n): estimativa do custo total da rota passando por n. A* sempre expande o nó com menor f(n).

  • Com h=0: A* se torna idêntico ao Dijkstra.
  • Com heurística perfeita: A* expande apenas o caminho ótimo.
  • Com heurística inadmissível: mais rápido mas sem garantia de optimalidade.

Implementação em Python

a_estrela.py
import heapq

def a_estrela(grafo, inicio, alvo, h):
    """grafo = {no: [(viz, peso)]}, h = heurística admissível.
    Retorna caminho ótimo de inicio a alvo."""
    g = {inicio: 0}
    heap = [(h(inicio, alvo), inicio)]
    pai = {inicio: None}

    while heap:
        _, atual = heapq.heappop(heap)
        if atual == alvo:
            caminho, v = [], alvo
            while v is not None: caminho.append(v); v = pai[v]
            return caminho[::-1]
        for viz, w in grafo.get(atual, []):
            g_tent = g[atual] + w
            if g_tent < g.get(viz, float('inf')):
                g[viz] = g_tent
                heapq.heappush(heap, (g_tent + h(viz, alvo), viz))
                pai[viz] = atual
    return []

Aplicações

Jogos e pathfinding

NPCs e personagens em jogos 2D/3D A* é o padrão da indústria para navegar por mapas de tiles com obstáculos.

Robótica

Planejamento de trajetória com obstáculos conhecidos a heurística euclidiana guia o robô diretamente ao destino.

GPS com informação geográfica

Quando a posição dos nós é conhecida, a distância direta ao destino é uma heurística excelente.

Atenção: A heurística deve ser admissível (nunca superestima) para garantir o caminho ótimo. Uma heurística inadmissível pode tornar A* mais rápido, mas você perde a garantia de optimalidade solução aceitável para jogos, problemática para sistemas críticos.
