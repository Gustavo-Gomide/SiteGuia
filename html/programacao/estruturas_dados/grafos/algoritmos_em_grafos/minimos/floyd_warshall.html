Floyd-Warshall: todos os pares de caminhos mínimos

Três linhas de código que resolvem todos os pares usando programação dinâmica sobre vértices intermediários.

Floyd-Warshall calcula o caminho mínimo entre todos os pares de vértices em O(V³). Usa programação dinâmica com a ideia: o caminho mínimo de i a j usando apenas vértices {0..k} como intermediários pode ser melhorado ao incluir o vértice k+1. Aceita pesos negativos mas não ciclos negativos.

Recorrência

dp[i][j][k] = mínimo entre dist(ij sem passar por k) e dist(ik) + dist(kj).
Simplificando in-place: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]) para cada vértice intermediário k.

Implementação em Python

floyd_warshall.py
def floyd_warshall(n, arestas):
    """n = número de vértices (0..n-1).
    arestas = [(i, j, peso)]. O(V³)."""
    INF = float('inf')
    dist = [[INF]*n for _ in range(n)]
    for i in range(n): dist[i][i] = 0
    for u, v, w in arestas: dist[u][v] = w

    for k in range(n):
        for i in range(n):
            for j in range(n):
                dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

    # Detectar ciclo negativo: diagonal negativa
    if any(dist[i][i] < 0 for i in range(n)):
        raise ValueError("Ciclo negativo detectado")

    return dist

# Exemplo: 4 vértices
arestas = [(0,1,3),(0,3,7),(1,0,8),(1,2,2),
           (2,3,1),(3,0,2)]
d = floyd_warshall(4, arestas)
for row in d: print(row)

Aplicações

Fechamento transitivo

Adaptando para operações lógicas, resolve se existe caminho entre qualquer par (alcançabilidade).

Grafos pequenos

Para V 500 e necessidade de todos os pares, Floyd-Warshall é simples e suficiente.

Análise de redes

Calcular centralidade de intermediação (betweenness) e outras métricas de rede que precisam de todos os caminhos.

Atenção: O(V³) e O(V²) de espaço tornam Floyd-Warshall impraticável para grafos grandes. Para V > 1000, prefira executar Dijkstra (ou Bellman-Ford) a partir de cada vértice ou use o algoritmo de Johnson para grafos esparsos com pesos negativos.
