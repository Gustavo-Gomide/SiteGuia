Viterbi: caminho mais provável no grafo treliça

Encontrar a sequência de estados ocultos mais provável em um HMM é equivalente a encontrar o caminho mais curto numa estrutura de grafo chamada treliça.

O algoritmo de Viterbi (1967) resolve a decodificação em Modelos Ocultos de Markov (HMM): dada uma sequência de observações, encontrar a sequência de estados ocultos mais provável. Trata-se de programação dinâmica no grafo treliça onde os nós são (estado, tempo) e as arestas codificam probabilidades de transição e emissão.

HMM como grafo

Um HMM define um grafo treliça com T N nós (T = comprimento da sequência, N = estados). Cada aresta (i,t)(j,t+1) tem peso log P(j|i) + log P(obs_{t+1}|j). Viterbi encontra o caminho de maior peso nesse DAG equivalente ao caminho mais curto com pesos negativos.

Implementação em Python

viterbi.py
import numpy as np

def viterbi(obs, A, B, pi):
    """obs = sequência de observações (índices).
    A = matriz de transição NN.
    B = matriz de emissão NM.
    pi = probabilidades iniciais N.
    Retorna (prob, sequência de estados)."""
    T, N = len(obs), len(pi)
    dp  = np.full((T, N), -np.inf)
    ptr = np.zeros((T, N), dtype=int)

    dp[0] = np.log(pi) + np.log(B[:, obs[0]])

    for t in range(1, T):
        for j in range(N):
            candidatos = dp[t-1] + np.log(A[:, j])
            ptr[t, j] = np.argmax(candidatos)
            dp[t, j] = candidatos[ptr[t, j]] + np.log(B[j, obs[t]])

    # Backtracking
    estados = [int(np.argmax(dp[T-1]))]
    for t in range(T-1, 0, -1):
        estados.append(int(ptr[t, estados[-1]]))
    estados.reverse()
    return np.max(dp[T-1]), estados

# Exemplo: HMM de clima (Sol/Chuva)  atividade (Caminhada/Compras/Limpeza)
A  = np.array([[0.7, 0.3], [0.4, 0.6]])  # transição
B  = np.array([[0.5, 0.4, 0.1], [0.1, 0.3, 0.6]])  # emissão
pi = np.array([0.6, 0.4])
obs = [0, 1, 2]  # Caminhada, Compras, Limpeza
prob, seq = viterbi(obs, A, B, pi)
print(seq)  # ex: [0, 0, 1]  Sol, Sol, Chuva

Aplicações

Reconhecimento de fala

Decodificar a sequência de fonemas mais provável dado o sinal de áudio aplicação histórica central de Viterbi.

NLP (PoS tagging, NER)

Tagging de partes do discurso e reconhecimento de entidades nomeadas com HMM usam Viterbi como algoritmo de inferência.

Comunicações digitais

Decodificação de códigos convolucionais em canais com ruído a aplicação original para a qual Viterbi foi inventado.

Underflow: Trabalhe sempre em log-probabilidades para evitar underflow numérico ao multiplicar muitas probabilidades pequenas.
