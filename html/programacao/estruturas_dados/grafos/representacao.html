Representação de Grafos

Como armazenar um grafo na memória

Matriz de adjacência ou lista de adjacência escolher a representação certa define a eficiência de todos os algoritmos que virão.

A forma como armazenamos um grafo na memória não é detalhe ela determina a complexidade de operações básicas como "verificar se existe aresta entre A e B" ou "listar todos os vizinhos de um vértice". As duas representações mais comuns são a matriz de adjacência e a lista de adjacência, cada uma com trade-offs claros entre memória e velocidade de acesso.

Matriz de adjacência

Uma matriz VV onde M[i][j] = 1 (ou o peso) se existe aresta de i para j, e 0 caso contrário.

matriz_adjacencia.py
# Grafo A-B-C-D não-direcionado
V = ['A', 'B', 'C', 'D']
n = len(V)
mat = [[0]*n for _ in range(n)]

def add_edge(u, v):
    mat[V.index(u)][V.index(v)] = 1
    mat[V.index(v)][V.index(u)] = 1   # remover para dígrafo

add_edge('A', 'B'); add_edge('A', 'C')
add_edge('B', 'D'); add_edge('C', 'D')

Lista de adjacência

Um dicionário (ou array de listas) onde cada vértice mapeia para a lista de seus vizinhos diretos. Para o mesmo grafo A-B-C-D com arestas A-B, A-C, B-D, C-D:

lista_adjacencia.py
from collections import defaultdict

grafo = defaultdict(list)

def add_edge(u, v, peso=1):
    grafo[u].append((v, peso))
    grafo[v].append((u, peso))   # omitir para dígrafo

add_edge('A', 'B'); add_edge('A', 'C')
add_edge('B', 'D'); add_edge('C', 'D')

# grafo = {'A': [('B',1),('C',1)], 'B': [('A',1),('D',1)], ...}

Ideal paraGrafos densos (E V²)Grafos esparsos (E << V²)

Atenção: A maioria dos grafos do mundo real é esparsa (pense em redes sociais ou mapas de cidades). Usar matriz de adjacência nesse contexto desperdiça O(V²) de memória e torna algoritmos como BFS e DFS mais lentos prefira lista de adjacência como padrão.
