Programação Dinâmica

Memorizar para Não Recalcular

Subproblemas sobrepostos resolvidos uma única vez e armazenados numa tabela.

Programação Dinâmica transforma recursion exponencial em solução polinomial ao identificar subproblemas que se repetem e guardar seus resultados. A subestrutura ótima garante que a solução global pode ser construída a partir das soluções ótimas dos subproblemas.

O que é?

Programação Dinâmica (DP) é uma técnica que resolve problemas complexos quebrando-os em subproblemas mais simples. A diferença fundamental em relação a Divisão e Conquista é que na DP os subproblemas se sobrepõem: os mesmos cálculos aparecem múltiplas vezes em ramos diferentes da recursão.

A DP foi introduzida por Richard Bellman (1957) como uma técnica de otimização multietápica. Dois ingredientes necessários: subestrutura ótima (solução global pode ser construída a partir das ótimas locais) e subproblemas sobrepostos (os mesmos subproblemas aparecem várias vezes).

DP não é recursividade com cache — é uma estratégia de projeto que explora subestrutura ótima + subproblemas sobrepostos. Quando apenas um desses ingredientes está presente, use Divisão e Conquista (sem sobreposição) ou algoritmos gulosos (sem DP de múltiplos estados).

Os Dois Pilares

  • Subproblemas Sobrepostos: o problema pode ser decomposto em partes menores que aparecem repetidamente em diferentes ramos da recursão. Exemplo: F(3) é necessário tanto para computar F(4) quanto para F(5).
  • Subestrutura Ótima: a solução ótima do problema pode ser construída a partir das soluções ótimas dos subproblemas. DP combina os dois para garantir corretude e eficiência.

Top-Down vs Bottom-Up

Estratégia Como funciona
Memoização (Top-Down) Recursivo. Calcula sob demanda e guarda no cache. Fácil de escrever.
Tabulação (Bottom-Up) Iterativo. Preenche a tabela do caso-base até o resultado. Sem overhead de pilha.

Implementação — Fibonacci e Knapsack

dp_fibonacci.py
# ── Top-Down: Memoização ──────────────────────────────────
from functools import lru_cache

@lru_cache(maxsize=None)
def fib_memo(n: int) -> int:
    if n <= 1: return n
    return fib_memo(n - 1) + fib_memo(n - 2)

# ── Bottom-Up: Tabulação ──────────────────────────────────
def fib_tab(n: int) -> int:
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

print(fib_memo(10))   # 55
print(fib_tab(10))    # 55

# ── Knapsack 0/1 (Bottom-Up) ──────────────────────────────
def knapsack_dp(pesos: list, valores: list, cap: int) -> int:
    """Mochila 0/1: retorna valor máximo. O(N·W)."""
    n  = len(pesos)
    dp = [[0] * (cap + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for w in range(cap + 1):
            dp[i][w] = dp[i-1][w]              # excluir item i
            if pesos[i-1] <= w:
                dp[i][w] = max(dp[i][w],
                    dp[i-1][w - pesos[i-1]] + valores[i-1])  # incluir
    return dp[n][cap]

print(knapsack_dp([2,3,4,5], [3,4,5,7], 10))  # 12

Impacto na Complexidade

Abordagem Tempo Espaço F(50)
Recursão simples (sem cache) O(2ⁿ) — exponencial O(n) pilha Trava / demora anos
DP com memoização O(n) — linear O(n) cache Instantâneo (<1 ms)
DP tabulação otimizada O(n) — linear O(1) — 2 variáveis Instantâneo (<1 ms)
