Backtracking

Retrocesso Sistemático na Árvore de Estados

Explorar, detectar conflito, desfazer a escolha e tentar o próximo caminho.

Backtracking é uma técnica de busca que constrói a solução incrementalmente. Ao detectar que o caminho atual viola alguma restrição do problema, o algoritmo desfaz a última decisão tomada ("volta atrás") e tenta a próxima alternativa — podando ramos inviáveis antes de expandi-los.

O que é?

Backtracking (retrocesso) é uma versão sistemática de Força Bruta com poda: em vez de gerar todos os candidatos e filtrá-los depois, o algoritmo testa a validade de cada escolha antes de continuar descendo na árvore de recursão.

Segundo Knuth (2011), backtracking pode ser visto como uma busca em profundidade (DFS) numa árvore de possibilidades onde galhos são cortados assim que uma inconsistência é detectada.

A diferença entre Backtracking e Força Bruta pura é a poda: detectar inconsistências cedo, antes de gerar os descendentes do nó ruim. Quanto mais restritivas as regras, mais eficiente é o corte.

O Template Recursivo

Quase todo algoritmo de backtracking segue a mesma estrutura de três passos dentro de uma função recursiva: escolher → explorar → desfazer.

  1. Escolher (Choose): Faça uma escolha incremental — coloque a rainha na coluna X, marque a célula como visitada, inclua o item na mochila.
  2. Explorar (Explore): Chame recursivamente para tentar completar a solução com essa escolha feita. Se o estado for inválido, a poda acontece aqui (antes da chamada recursiva).
  3. Desfazer (Un-choose): Após a recursão retornar (encontrou solução ou beco), remova a escolha para que o loop possa tentar a alternativa seguinte.

Poda (Pruning)

A poda é o que diferencia Backtracking de Força Bruta. Ao gerar a sequência [A, B, ...] e a regra diz "B não pode seguir A", o algoritmo para imediatamente — sem gerar [A, B, C], [A, B, D], etc. A árvore inteira abaixo de [A, B] é descartada sem ser explorada.

Implementação — N-Queens

O Problema das N-Rainhas: posicionar N rainhas num tabuleiro N×N de modo que nenhuma duas se ataquem (mesma coluna, linha ou diagonal).

n_queens.py
def n_queens(n: int) -> list[list[int]]:
    """Retorna todas as soluções para o problema das N-Rainhas."""
    solucoes: list[list[int]] = []
    tabuleiro: list[int] = []  # tabuleiro[i] = coluna da rainha na linha i

    def eh_valido(linha: int, col: int) -> bool:
        for l in range(linha):
            c = tabuleiro[l]
            if c == col or abs(c - col) == abs(l - linha):
                return False  # mesma coluna ou mesma diagonal
        return True

    def backtrack(linha: int) -> None:
        if linha == n:
            solucoes.append(tabuleiro[:])  # solução completa encontrada
            return
        for col in range(n):
            if eh_valido(linha, col):          # PODA: só continua se for válido
                tabuleiro.append(col)          # A. escolher
                backtrack(linha + 1)           # B. explorar
                tabuleiro.pop()                # C. desfazer (backtrack)

    backtrack(0)
    return solucoes

print(len(n_queens(8)))  # 92 soluções para o tabuleiro 8×8

Sem poda: O(N!) = 40 320 para N=8. Com backtracking: apenas ~2 000 nós visitados — uma redução de ~95% no espaço de busca explorado.
