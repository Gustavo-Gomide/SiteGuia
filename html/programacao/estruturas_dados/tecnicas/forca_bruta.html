Força Bruta

Enumeração Exaustiva de Candidatos

Testar todas as possibilidades até encontrar a solução — sem atalhos, sem heurísticas.

Força Bruta é a estratégia mais direta de resolução de problemas: enumerar sistematicamente todos os candidatos possíveis e verificar cada um. Ineficiente em escala, mas matematicamente garantida — e fundamental como linha de base para validar qualquer algoritmo mais inteligente.

O que é?

Uma técnica que enumera todos os candidatos sistematicamente e verifica se cada um satisfaz a especificação do problema. Não há inteligência na escolha — o espaço de busca é percorrido de forma completa e sequencial.

A analogia clássica: uma fechadura de 3 dígitos decimais tem 10³ = 1 000 combinações. Força Bruta testa 000, 001, 002… até encontrar a correta. No pior caso verifica todas.

No pior caso (111), todas as 2³ = 8 combinações são testadas. Para N dígitos com base B o custo é O(Bᴺ) — crescimento exponencial.

Força Bruta é sempre correta e completa: se existe solução no espaço de busca, ela será encontrada. A questão nunca é sobre corretude, mas se o tempo disponível suporta a complexidade do espaço.

3 dígitos decimais = 10³ = 1 000 tentativas. 6 dígitos alfanuméricos (a–z, 0–9) = 36⁶ ≈ 2,2 bilhões. A complexidade exponencial torna Força Bruta impraticável para entradas moderadas.

Quando Usar?

Use quando…

  • Entradas pequenas: N ≤ 20 para O(2ᴺ); N ≤ 10 para O(N!).
  • Baseline de testes: gera resposta correta para validar algoritmos otimizados em casos de teste.
  • Sem estrutura explorável: nenhuma propriedade ordenada ou subestrutura ótima identificável.
  • Custo de desenvolvimento importa mais: implementação trivial compensa o custo de execução.

Evite quando…

  • N cresce além de dezenas (TSP com 20 cidades = 20! ≈ 10¹⁸ rotas).
  • O problema tem subestrutura ótima → use Programação Dinâmica.
  • Há propriedade de escolha gulosa → use Algoritmo Guloso.
  • O espaço de busca é estruturado → Backtracking + poda reduz dramaticamente o custo.

Implementação

O padrão de Força Bruta é sempre o mesmo: gerar todos os candidatos e testar cada um. Abaixo, o TSP (Caixeiro Viajante) pela abordagem exaustiva:

tsp_forca_bruta.py
from itertools import permutations

def tsp_forca_bruta(dist: list[list[float]]) -> tuple[list[int], float]:
    """Encontra a rota de menor custo visitando todas as cidades. O(N!)."""
    n = len(dist)
    melhor_custo = float('inf')
    melhor_rota  = []

    for perm in permutations(range(1, n)):   # cidade 0 é ponto de partida fixo
        rota  = [0] + list(perm) + [0]          # começa e termina em 0
        custo = sum(dist[rota[i]][rota[i+1]] for i in range(len(rota)-1))
        if custo < melhor_custo:
            melhor_custo, melhor_rota = custo, rota

    return melhor_rota, melhor_custo

# Exemplo — 4 cidades, matriz de distâncias simétrica
d = [
    [0, 2, 9, 10],
    [1, 0, 6,  4],
    [15,7, 0,  8],
    [6, 3, 12, 0],
]
print(tsp_forca_bruta(d))  # ([0, 1, 3, 2, 0], 21.0)  — 3! = 6 permutações testadas

Para N = 4: 3! = 6 permutações. Para N = 15: 14! ≈ 87 bilhões — impraticável sem cortes. O algoritmo de Held–Karp resolve TSP em O(2ᴺ · N) usando Programação Dinâmica.

Complexidade e Comparação

Problema Força Bruta Técnica Otimizada Redução
Busca em lista desordenada O(N) O(log N) — Busca Binária Logarítmica
Ordenação O(N²) — Selection Sort O(N log N) — Merge Sort Quasilinear
Caminho mínimo (grafo) O(V!) — todas as rotas O(E log V) — Dijkstra Polinomial
TSP O(N!) O(2ᴺ · N) — Held–Karp Exponencial menor
Subsequência comum máxima O(2ᴺ) — todas as subsequências O(N · M) — Prog. Dinâmica Polinomial
