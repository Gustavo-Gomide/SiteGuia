Branch and Bound

Otimização com Poda por Limite Inferior

Ramificar o espaço de busca, calcular bounds e descartar ramos sem potencial.

Branch and Bound é uma busca exaustiva inteligente para problemas de otimização combinatória. Para cada ramo calcula-se um lower bound; se esse limite já supera a melhor solução conhecida, o ramo inteiro é descartado sem explorar nenhum descendente.

O que é?

B&B é uma enumeração implícita: em vez de gerar todos os candidatos (Força Bruta), o algoritmo mantém a variável global melhor_solução e poda qualquer ramo cujo lower bound já seja ≥ esse valor.

Formalizados por Lawler e Wood (1966) e Land e Doig (1960), os algoritmos de B&B são hoje a base de solvers de otimização inteira como CPLEX e Gurobi. Aplicações clássicas: Caixeiro Viajante (TSP), Mochila 0/1 (Knapsack) e Alocação de Tarefas.

B&B não enuncia soluções explicitamente: o espaço de busca é podado de forma implícita. Quanto mais apertado o lower bound, mais ramos são cortados e mais eficiente é o algoritmo na prática.

Visualização — A Árvore com Poda

Objetivo: minimizar o custo. A melhor solução atual serve como Upper Bound (UB). Qualquer ramo com Lower Bound (LB) ≥ UB é podado imediatamente.

O ramo "s/ A" tinha LB = 30. Como 30 ≥ UB = 25, todo o subconjunto que não inclui o item A é descartado sem ser enumerado.

Mecânica do Algoritmo

  1. Ramificar (Branch): cada decisão binária (incluir/excluir, girar esquerda/direita) cria dois filhos na árvore de estados.
  2. Limitar (Bound): calcule um lower bound otimista — "qual o melhor valor alcançável a partir daqui?". Tipicamente usa-se uma relaxação do problema (ex.: mochila fracionária para o 0/1 Knapsack).
  3. Podar (Prune): se LB ≥ melhor_solução, descarte o nó e todos os seus descendentes.
  4. Estratégia de Busca: Best-First Search com fila de prioridade — expande primeiro os nós com menor bound (mais promissores).

Implementação — Knapsack 0/1

O Problema da Mochila 0/1: selecionar itens com peso e valor para maximizar o valor total sem exceder a capacidade. O upper_bound usa a relaxação fracionária como teto otimista.

knapsack_bb.py
import heapq

def knapsack_bb(pesos, valores, cap):
    """Knapsack 0/1 via Branch and Bound com Best-First Search."""
    n      = len(pesos)
    melhor = [0]   # melhor valor encontrado até agora

    def upper_bound(nivel, peso, valor):
        """Relaxação fracionária: limite superior otimista."""
        p, v = peso, valor
        for i in range(nivel, n):
            if p + pesos[i] <= cap:
                p += pesos[i]; v += valores[i]
            else:
                v += (cap - p) / pesos[i] * valores[i]; break
        return v

    # Fila de prioridade: (-ub, nivel, peso, valor)
    fila = [(-upper_bound(0, 0, 0), 0, 0, 0)]

    while fila:
        neg_ub, nivel, peso, valor = heapq.heappop(fila)
        if -neg_ub <= melhor[0]: continue   # PODA
        if nivel == n:
            melhor[0] = max(melhor[0], valor); continue

        # Ramificação 1 — INCLUIR item `nivel`
        np, nv = peso + pesos[nivel], valor + valores[nivel]
        if np <= cap:
            melhor[0] = max(melhor[0], nv)
            ub1 = upper_bound(nivel + 1, np, nv)
            if ub1 > melhor[0]: heapq.heappush(fila, (-ub1, nivel+1, np, nv))

        # Ramificação 2 — EXCLUIR item `nivel`
        ub2 = upper_bound(nivel + 1, peso, valor)
        if ub2 > melhor[0]: heapq.heappush(fila, (-ub2, nivel+1, peso, valor))

    return melhor[0]

# pesos [2,3,4,5], valores [3,4,5,7], capacidade 10 → 12
print(knapsack_bb([2,3,4,5], [3,4,5,7], 10))  # 12
