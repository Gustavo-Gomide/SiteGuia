Heaps: acesso instantâneo ao máximo (ou mínimo)

Uma árvore binária completa que mantém o maior (ou menor) elemento sempre acessível na raiz em O(log n) para inserção e remoção.

Agendadores de sistema operacional, algoritmos de caminho mínimo (Dijkstra), compressão Huffman e o algoritmo de ordenação Heap Sort compartilham uma necessidade: extrair o elemento de maior (ou menor) prioridade de forma eficiente. O heap é a estrutura que resolve exatamente esse problema.

O que é um Heap?

Um heap é uma árvore binária completa que satisfaz a propriedade de heap: em um max-heap, o valor de cada nó é maior ou igual ao valor de seus filhos; em um min-heap, é menor ou igual.

Árvore binária completa significa que todos os níveis estão preenchidos da esquerda para a direita, sem lacunas isso permite armazenar o heap em um array sem desperdício de espaço, usando aritmética de índices para navegar entre pai e filhos.

Segundo Cormen et al. (2022), para um array A que representa um heap, o pai do elemento no índice i está em i/2, o filho esquerdo em 2i e o filho direito em 2i+1.

A propriedade de heap garante acesso O(1) ao máximo (no max-heap) ou ao mínimo (no min-heap). Isso é tudo que uma fila de prioridade precisa para funcionar eficientemente.

Como Funciona

O diagrama mostra um max-heap e sua representação equivalente em array. O valor 90 está na raiz e portanto no índice 1 do array (usando índice 1-based).

Inserção Sift Up

Insere o novo elemento no final do array (última posição do último nível). Depois compara com o pai e sobe enquanto violar a propriedade. Custo: O(log n).

Extração do Máximo Sift Down

Remove a raiz (máximo), coloca o último elemento na raiz, e desce comparando com os filhos enquanto violar a propriedade. Custo: O(log n).

Build Heap

Constrói um heap a partir de um array não-ordenado aplicando sift down a todos os nós internos de baixo para cima. Custo: O(n) surpreendentemente linear.

Implementação

heap.pseudo
// A[1..n]: array 1-based. pai(i)=i/2, filhoEsq(i)=2i, filhoDir(i)=2i+1

FUNCAO siftDown(A, i, n) {
    maior = i
    esq  = 2 * i
    dir  = 2 * i + 1
    SE esq <= n E A[esq] > A[maior]  maior = esq
    SE dir <= n E A[dir] > A[maior]  maior = dir
    SE maior != i {
        trocar(A[i], A[maior])
        siftDown(A, maior, n)
    }
}

FUNCAO buildHeap(A, n) {
    PARA i = n/2 ATE 1 (decrescente) {
        siftDown(A, i, n)
    }
}

FUNCAO extrairMaximo(A, n) {
    max = A[1]
    A[1] = A[n]
    n = n - 1
    siftDown(A, 1, n)
    RETORNE max
}

Heaps não suportam busca eficiente por valor arbitrário isso é O(n). Se precisar encontrar e remover um elemento específico que não seja o máximo/mínimo, considere uma estrutura adicional de índice.

Casos de Uso

Fila de Prioridade

Implementação natural: inserir = push com sift up; extrairMáximo = pop com sift down. Base do escalonamento de processos em SOs.

Heap Sort

Algoritmo de ordenação O(n log n) in-place: constrói o heap e extrai o máximo repetidamente. Não precisa de memória adicional além do próprio array.

Algoritmo de Dijkstra

Um min-heap armazena os vértices a explorar, priorizando o de menor distância acumulada. Reduz a complexidade de O(V²) para O((V+E) log V).

Compressão Huffman

Um min-heap seleciona os dois símbolos de menor frequência a cada iteração para construir a árvore de codificação ótima.

Complexidade

Operação Tempo Observação
Inserção (push) O(log n) Sift up percorre no máximo a altura da árvore.
Extrair máximo/mínimo O(log n) Sift down percorre no máximo a altura da árvore.
Ver máximo/mínimo (peek) O(1) Sempre na raiz índice 1 do array.
Build Heap O(n) Embora pareça O(n log n), a análise amortizada prova O(n).
Busca por valor O(n) Sem garantia de posição pode estar em qualquer lugar.
