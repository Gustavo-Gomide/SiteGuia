Árvores de Busca (BST)

Árvores de Busca (BST)

BST: busca eficiente pela estrutura

Uma árvore binária cuja regra de ordenação permite encontrar qualquer elemento em O(log n) sem arrays ordenados ou hashing.

A Árvore Binária de Busca é a estrutura que transforma a busca binária em uma estrutura de dados dinâmica: mantém os elementos ordenados durante inserções e remoções, permitindo busca logarítmica a qualquer momento.

O que é uma BST?

Uma Árvore Binária de Busca (BST) é uma árvore binária que satisfaz a propriedade de BST: para cada nó x, todos os nós na subárvore esquerda têm valor menor que x, e todos os nós na subárvore direita têm valor maior.

Segundo Cormen et al. (2022), se y é um nó na subárvore esquerda de x, então y.chave x.chave; se y é na subárvore direita, então y.chave x.chave.

A travessia em-ordem de uma BST produz os elementos em ordem crescente. Essa propriedade é diretamente derivada da regra esquerdo-menor-direito-maior.

Como Funciona

O diagrama mostra uma BST com raiz 50. Para buscar o valor 35, seguimos o caminho destacado: 50 30 40 35.

Busca

Compara o valor buscado com o nó atual. Se menor, vai para a esquerda; se maior, vai para a direita. Repete até encontrar o valor ou chegar a um nó nulo.

Inserção

Segue o mesmo caminho da busca. Quando chega a um ponteiro nulo, cria um novo nó naquele lugar. A posição é determinada pela propriedade de BST.

Remoção

Três casos: nó folha (remove diretamente), nó com um filho (substitui pelo filho) e nó com dois filhos (substitui pelo sucessor em-ordem mínimo da subárvore direita).

Implementação

bst.pseudo
FUNCAO buscar(no, valor) {
    SE no == NULO OU no.valor == valor
        RETORNE no
    SE valor < no.valor
        RETORNE buscar(no.esquerdo, valor)
    RETORNE buscar(no.direito, valor)
}

FUNCAO inserir(no, valor) {
    SE no == NULO
        RETORNE NovoNo(valor)
    SE valor < no.valor
        no.esquerdo = inserir(no.esquerdo, valor)
    SENAO SE valor > no.valor
        no.direito = inserir(no.direito, valor)
    RETORNE no
}

FUNCAO minimo(no) {
    ENQUANTO no.esquerdo != NULO
        no = no.esquerdo
    RETORNE no
}

Inserir elementos já ordenados (1, 2, 3, 4) numa BST gera uma árvore degenerada essencialmente uma lista encadeada com altura O(n) e busca O(n). Para dados do mundo real sem controle de ordem, prefira AVL ou Rubro-Negra.

Complexidade

Operação Caso médio Pior caso
Busca O(log n) O(n)
Inserção O(log n) O(n)
Remoção O(log n) O(n)
Mínimo / Máximo O(log n) O(n)
Espaço O(n) O(n)

O pior caso ocorre quando a árvore se degenera em lista (altura n). Isso é prevenido pelas variantes auto-balanceadas AVL e Rubro-Negra.
