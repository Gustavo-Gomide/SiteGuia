Tries: busca em strings pelo prefixo

A árvore onde cada nível representa um caractere busca, inserção e correspondência de prefixo em tempo proporcional ao comprimento da chave.

Quando você digita as primeiras letras de uma palavra no teclado do celular e ele sugere completamente a palavra, ou quando um mecanismo de busca prevê o que você vai digitar, a estrutura subjacente é quase sempre uma Trie. Busca em O(k) onde k é o comprimento da string independentemente de quantas palavras estão armazenadas.

O que é uma Trie?

Uma Trie (pronuncia-se "trai", do inglês retrieval) é uma árvore n-ária na qual cada aresta representa um caractere. Um nó na profundidade d representa o prefixo formado pelos caracteres das arestas no caminho da raiz até ele. As palavras completas são marcadas com uma flag especial no nó final.

Segundo Knuth (1998), as Tries (ou árvores de busca digital) foram introduzidas por Fredkin em 1960 e são especialmente eficientes quando o universo de chaves é um conjunto de strings sobre um alfabeto fixo.

A complexidade de busca em uma Trie é O(k), onde k é o comprimento da string buscada. Isso é independente do número de strings armazenadas n ao contrário de uma tabela hash, onde a função hash processa a string inteira mas o tempo de cálculo não varia com n.

Como Funciona

O diagrama mostra uma Trie com as palavras "CA", "CAO", "CASA" e "GATO". Os nós marcados com * indicam o final de uma palavra válida.

Inserção

Percorre a trie caractere por caractere. Se o nó do próximo caractere não existe, cria-o. Ao inserir o último caractere, marca o nó como fim de palavra. Custo: O(k).

Busca exata

Navega caractere por caractere. Se em algum ponto o nó filho não existe, a palavra não está na trie. Se chegou ao fim e o nó está marcado como palavra, encontrou. Custo: O(k).

Busca por prefixo

Navega até o nó correspondente ao último caractere do prefixo. A partir daí, toda a subárvore representa palavras que começam com esse prefixo. Base do autocomplete.

Compartilhamento de prefixo

"CARRO" e "CASA" compartilham os nós C e A. O espaço por palavra depende apenas dos caracteres únicos após o prefixo comum mais eficiente que armazenar strings individualmente quando há muitos prefixos compartilhados.

Implementação

trie.pseudo
ESTRUTURA NoTrie {
    filhos    = mapa_vazio       // char  NoTrie
    ehPalavra = FALSO
}

ESTRUTURA Trie {
    raiz = NoTrie()

    FUNCAO inserir(palavra) {
        atual = raiz
        PARA CADA char EM palavra {
            SE char NAO ESTA EM atual.filhos
                atual.filhos[char] = NoTrie()
            atual = atual.filhos[char]
        }
        atual.ehPalavra = VERDADEIRO
    }

    FUNCAO buscar(palavra) {
        atual = raiz
        PARA CADA char EM palavra {
            SE char NAO ESTA EM atual.filhos
                RETORNE FALSO
            atual = atual.filhos[char]
        }
        RETORNE atual.ehPalavra
    }

    FUNCAO buscaPrefixo(prefixo) {
        atual = raiz
        PARA CADA char EM prefixo {
            SE char NAO ESTA EM atual.filhos
                RETORNE FALSO
            atual = atual.filhos[char]
        }
        RETORNE VERDADEIRO   // prefixo existe
    }
}

Uma trie no pior caso pode usar muito mais memória que uma tabela hash equivalente. Uma trie com k palavras de comprimento médio m e alfabeto de tamanho σ pode ter até kmσ ponteiros. Para grandes alfabetos (Unicode completo), use estruturas de trie comprimidas como a Patricia Trie ou DAWG.

Casos de Uso

Autocomplete

Digitando "pro", a trie retorna todas as palavras no dicionário com esse prefixo em O(k + n_resultados), sem varrer toda a coleção.

Verificação ortográfica

Verificar se uma palavra existe no dicionário é O(k) mais rápido que qualquer busca em lista ou árvore BST que compara strings inteiras.

Roteamento IP

Roteadores usam Tries binárias (bit-a-bit) para encontrar a correspondência de prefixo mais longa entre um endereço IP e as rotas na tabela de roteamento.

Compressão de texto

Algoritmos como LZW e Huffman adaptativo usam tries para rastrear sequências já vistas e substituí-las por códigos mais curtos.

Complexidade

Operação Tempo Observação
Busca exata O(k) k = comprimento da string. Independe de n.
Inserção O(k) Cria no máximo k novos nós.
Busca de prefixo O(k) Localiza o nó do prefixo em O(k).
Listagem por prefixo O(k + m) m = número total de caracteres nos resultados.
Espaço O(nkσ) σ = tamanho do alfabeto. Pode ser alto para Unicode.
