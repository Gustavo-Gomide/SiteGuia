Grafos: relacionamentos sem hierarquia

A estrutura que modela redes sociais, rotas de GPS, dependências de software e qualquer sistema de conexões complexas.

Quando o GPS calcula a rota mais rápida entre dois pontos, quando o algoritmo de recomendação do YouTube sugere o próximo vídeo ou quando o compilador resolve dependências entre módulos, o que está sendo manipulado é um grafo. Grafos são a estrutura de dados mais geral e poderosa da computação.

O que é um Grafo?

Um grafo G é um par ordenado G = (V, E), onde V é um conjunto de vértices (ou nós) e E é um conjunto de arestas (ou arcos) que conectam pares de vértices. Diferentemente das árvores, grafos não possuem raiz, não exigem hierarquia e podem ter ciclos.

Segundo Cormen et al. (2022), algoritmos em grafos são fundamentais para o campo da computação: praticamente todo problema computacional de interesse pode ser modelado como um problema de grafo.

Uma árvore é, formalmente, um grafo acíclico conectado. Todo par de vértices está ligado por exatamente um caminho. Portanto, toda árvore é um grafo, mas nem todo grafo é uma árvore.

Tipos de Grafo

Não-direcionado

As arestas não têm sentido: se A conecta a B, B também conecta a A. Modela relações simétricas como amizades em redes sociais.

Direcionado (dígrafo)

Cada aresta tem uma direção: (A B) não implica (B A). Modela dependências, links da web, fluxo de tráfego.

Ponderado

Cada aresta possui um peso (distância, custo, capacidade). Necessário para algoritmos de caminho mínimo como Dijkstra e Bellman-Ford.

Conexo vs Desconexo

Um grafo é conexo se existe caminho entre todo par de vértices. Em um grafo desconexo, existem componentes isoladas.

Cíclico vs Acíclico

Um grafo acíclico não possui ciclos. Um DAG (Grafo Acíclico Dirigido) é amplamente usado para modelar dependências de tarefas.

Representações

Dois formatos dominam a representação de grafos em código. A escolha depende da densidade de arestas e das operações necessárias:

representacoes.pseudo
// LISTA DE ADJACÊNCIA  recomendada para grafos esparsos
// grafo[v] = lista de vizinhos de v
grafo = {
    A: [B, C],
    B: [A, D],
    C: [A, D],
    D: [B, C]
}

// MATRIZ DE ADJACÊNCIA  recomendada para grafos densos
// matriz[i][j] = 1 se existe aresta entre i e j
//   A  B  C  D
// A[0, 1, 1, 0]
// B[1, 0, 0, 1]
// C[1, 0, 0, 1]
// D[0, 1, 1, 0]

Algoritmos Fundamentais

BFS Busca em Largura

Explora os vizinhos de um vértice antes de avançar. Usa uma fila. Encontra o menor caminho (em número de arestas) em grafos não-ponderados. Complexidade: O(V + E).

DFS Busca em Profundidade

Avança tão longe quanto possível antes de recuar. Usa uma pilha (ou recursão). Base para detecção de ciclos, ordenação topológica e componentes fortemente conexas. Complexidade: O(V + E).

Dijkstra

Caminho mais curto de uma origem para todos os vértices em grafos ponderados com pesos não-negativos. Usa um min-heap. Complexidade: O((V+E) log V).

Bellman-Ford

Caminho mais curto com pesos negativos. Detecta ciclos de peso negativo. Mais lento que Dijkstra: O(VE).

Kruskal / Prim

Encontram a Árvore Geradora Mínima o subconjunto de arestas que conecta todos os vértices com custo mínimo. Basis para projetos de redes físicas.

Ordenação Topológica

Ordena os vértices de um DAG tal que para cada aresta (uv), u vem antes de v. Base para sistemas de build, compiladores e resolução de dependências.

A escolha do algoritmo certo depende fortemente das propriedades do grafo: pesos negativos inviabilizam Dijkstra; grafos desconexos exigem repetição de BFS/DFS para cada componente; ciclos tornam impossível a ordenação topológica.

Grafos são a linguagem universal para modelar relações complexas. Dominar seus algoritmos fundamentais é requisito para resolver os problemas mais desafiadores da computação.

"Os grafos constituem o modelo matemático mais importante em computação. Quase todo problema pode ser descrito em termos de grafos, e a resolução eficiente de problemas em grafos é o coração de inúmeras aplicações práticas." Sedgewick e Wayne, Algorithms
