Árvores B+

Árvores B+: dados nas folhas, índices acima

A variante da B-Tree que separa navegação de armazenamento e habilita varreduras sequenciais eficientes.

Quando um banco de dados executa SELECT * FROM pedidos WHERE data BETWEEN '2024-01-01' AND '2024-12-31', ele navega pela B+ para achar o primeiro registro e então percorre as folhas encadeadas em ordem. Essa varredura de intervalo é impossível de forma eficiente em B-Trees comuns.

O que é uma Árvore B+?

Uma árvore B+ é uma variante da B-Tree com duas regras adicionais fundamentais:

  1. Todos os dados reais ficam somente nas folhas. Os nós internos contêm apenas chaves de roteamento (cópias das chaves das folhas), usadas para direcionar a navegação.
  2. As folhas são ligadas em lista duplamente encadeada em ordem crescente de chave, permitindo varreduras sequenciais sem subir pela árvore.

Segundo Ramakrishnan e Gehrke (2003), a B+-Tree é a estrutura de índice dominante em sistemas de bancos de dados por combinar buscas pontuais O(log n) com varreduras de intervalo O(k + log n), onde k é o número de registros retornados.

Estrutura Visual

Nós internos

Contêm apenas cópias das chaves utilizadas para roteamento. Não armazenam dados reais. Isso permite maior fator de ramificação mais filhos por nó.

Nós folha

Armazenam todas as chaves reais com os dados (ou ponteiros para os dados). Estão todos no mesmo nível e ligados em lista encadeada.

Varredura de intervalo

Para buscar um intervalo, basta encontrar o limite inferior pela árvore e então percorrer as folhas encadeadas até o limite superior sem backtrack.

B-Tree vs B+-Tree

Critério B-Tree B+-Tree
Localização dos dados Em qualquer nó Somente nas folhas
Varredura sequencial O(n log n) requer traversal O(n) percorre lista de folhas
Fator de ramificação Menor (nós internos guardam dados) Maior (nós internos só com chaves)
Busca pontual Pode terminar em nó interno Sempre termina na folha
Uso predominante Sistemas de arquivos gerais Índices de bancos de dados

Na B+-Tree, a chave de um nó interno pode ser duplicada na folha correspondente. Isso aumenta levemente o espaço total, mas é o preço do encadeamento eficiente de folhas.

Complexidade

Operação Tempo Observação
Busca pontual O(log n) Sempre percorre até a folha.
Varredura de intervalo [a,b] O(log n + k) k = número de registros no intervalo.
Inserção O(log n) Split pode propagar até a raiz.
Remoção O(log n) Merge/redistribuição nas folhas.
Espaço O(n) Chaves podem ser duplicadas em nós internos.
