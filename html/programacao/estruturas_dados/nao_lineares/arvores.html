Árvores: hierarquia e recursividade

A estrutura não-linear que modela hierarquias, decisões e relacionamentos de todo tipo.

Sistemas de arquivos, documentos HTML, organogramas corporativos, motores de banco de dados e compiladores têm algo em comum: todos organizam seus dados em árvores. Entender árvores é entender como o computador representa o mundo hierárquico.

O que é uma Árvore?

Uma árvore é uma estrutura de dados não-linear composta por nós conectados por arestas, onde existe um nó especial chamado raiz e nenhum nó tem mais de um pai o que impede ciclos. Formalmente, é um grafo acíclico conectado e direcionado.

Pense em um livro com capítulos, seções e subseções. O livro é a raiz; cada capítulo é filho do livro; cada seção é filha do capítulo. Nunca uma subseção pertence a dois capítulos ao mesmo tempo. Essa hierarquia sem duplicidade é a essência de uma árvore.

Segundo Cormen et al. (2022), uma árvore com raiz é um conjunto finito de nós no qual um nó é designado raiz e os demais são divididos em subconjuntos disjuntos, cada um também uma árvore com raiz.

A diferença principal entre árvore e lista encadeada: na lista cada nó tem exatamente um sucessor; na árvore cada nó pode ter vários filhos, mas somente um pai.

Terminologia Essencial

Antes de explorar variantes, é necessário dominar o vocabulário que toda literatura técnica usa ao descrever árvores:

Nó (Node)

Unidade básica. Armazena um valor e referências para seus filhos. Qualquer elemento da árvore é um nó, inclusive a raiz.

Raiz (Root)

O único nó sem pai. É o ponto de entrada da árvore. Toda travessia começa na raiz.

Folha (Leaf)

Nó sem filhos. Está sempre no limite inferior da hierarquia. Também chamado de nó terminal.

Altura

Número de arestas no caminho mais longo da raiz até uma folha. Uma árvore com apenas a raiz tem altura 0.

Profundidade

Número de arestas do caminho da raiz até um nó específico. A raiz tem profundidade 0; seus filhos, profundidade 1.

Grau

Número de filhos de um nó. Uma árvore binária tem grau máximo 2. Uma árvore n-ária permite grau até n.

Principais Tipos de Árvore

O conceito de árvore é uma abstração ampla. Diferentes restrições impostas sobre a estrutura geram variantes especializadas, cada uma com propriedades e aplicações distintas:

Árvore Binária

Cada nó tem no máximo dois filhos: esquerdo e direito. Base para BSTs, heaps e AVL.

Árvore Binária de Busca (BST)

Árvore binária onde esq < nó < dir. Permite busca em O(log n) quando balanceada.

Árvore AVL

BST com auto-balanceamento via fator de equilíbrio. Garante altura O(log n) sempre.

Árvore Rubro-Negra

BST auto-balanceada com regras de coloração. Usada no kernel Linux e no Java TreeMap.

Árvore B e B+

Árvores multigrau balanceadas, otimizadas para acesso em disco. Base de índices de bancos de dados.

Trie

Árvore de prefixos para strings. Cada nível representa um caractere, permitindo buscas por prefixo em O(k).

Heap

Árvore binária completa que satisfaz a propriedade de heap: pai filhos (max-heap) ou pai filhos (min-heap).

Travessias

Percorrer todos os nós de uma árvore é chamado de travessia. A ordem de visita determina o resultado e define três estratégias clássicas, todas de custo O(n):

travessias.pseudo
// Pré-ordem: visita nó ANTES dos filhos
FUNCAO preOrdem(no) {
    SE no == NULO RETORNE
    visitar(no)
    preOrdem(no.esquerdo)
    preOrdem(no.direito)
}

// Em-ordem: visita nó ENTRE os filhos (resulta em ordem crescente para BST)
FUNCAO emOrdem(no) {
    SE no == NULO RETORNE
    emOrdem(no.esquerdo)
    visitar(no)
    emOrdem(no.direito)
}

// Pós-ordem: visita nó DEPOIS dos filhos
FUNCAO posOrdem(no) {
    SE no == NULO RETORNE
    posOrdem(no.esquerdo)
    posOrdem(no.direito)
    visitar(no)
}

Todas as travessias recursivas usam a pilha de chamadas implicitamente. Para árvores muito profundas, a recursão pode causar estouro de pilha nesses casos, implemente iterativamente usando uma pilha explícita.

Complexidade Geral

Operação Árvore geral BST balanceada
Travessia completa O(n) O(n)
Busca O(n) O(log n)
Inserção O(1)* O(log n)
Altura máxima O(n) O(log n)
Espaço O(n) O(n)

* com ponteiro direto para o pai onde inserir.
