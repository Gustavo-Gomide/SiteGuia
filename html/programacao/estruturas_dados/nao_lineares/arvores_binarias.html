Árvores Binárias

Árvores Binárias: dois filhos, infinitas possibilidades

A restrição de no máximo dois filhos por nó é o fundamento de BSTs, heaps, AVL e da maioria dos algoritmos em árvore.

A árvore binária é o tipo de árvore mais estudado e implementado em ciência da computação. Sua simplicidade geométrica esquerdo e direito permite algoritmos recursivos elegantes e estruturas especializadas de alto desempenho.

O que é uma Árvore Binária?

Uma árvore binária é uma árvore na qual cada nó possui no máximo dois filhos, convencionalmente chamados de filho esquerdo e filho direito. Qualquer dos dois (ou ambos) pode estar ausente.

Segundo Knuth (1997), uma árvore binária é uma estrutura finita de nós que é vazia ou consiste de um nó raiz com duas subárvores binárias disjuntas: a subárvore binária esquerda e a subárvore binária direita.

A ausência de um filho é representada por um ponteiro nulo. A distinção entre filho esquerdo e direito mesmo quando só existe um é semanticamente significativa: uma árvore com filho único à esquerda é diferente de uma com filho único à direita.

Tipos de Árvore Binária

Completa

Todos os níveis estão cheios exceto possivelmente o último, que é preenchido da esquerda para a direita. Heaps são geralmente armazenadas como árvores binárias completas em arrays.

Cheia (Strict / Proper)

Cada nó tem exatamente 0 ou 2 filhos nunca apenas um. Importante em análise de algoritmos de divisão e conquista.

Perfeita

Todos os nós internos têm 2 filhos e todas as folhas estão no mesmo nível. Uma árvore perfeita de altura h possui exatamente 2^(h+1) 1 nós.

Degenerada (Patológica)

Cada nó interno tem apenas um filho. Equivale a uma lista encadeada e degrada a complexidade de busca para O(n).

Implementação

Um nó de árvore binária armazena um valor e dois ponteiros: um para o filho esquerdo e um para o direito. O ponto de entrada da árvore é o ponteiro para a raiz.

arvore_binaria.pseudo
ESTRUTURA No {
    valor
    esquerdo = NULO
    direito  = NULO
}

ESTRUTURA ArvoreBinaria {
    raiz = NULO

    FUNCAO altura(no) {
        SE no == NULO RETORNE -1
        altEsq = altura(no.esquerdo)
        altDir = altura(no.direito)
        RETORNE 1 + max(altEsq, altDir)
    }

    FUNCAO contarNos(no) {
        SE no == NULO RETORNE 0
        RETORNE 1 + contarNos(no.esquerdo) + contarNos(no.direito)
    }
}

As funções altura e contarNos ambas percorrem toda a árvore, custando O(n). O cálculo de altura compara recursivamente as alturas das duas subárvores e retorna a maior mais um.

Complexidade

Operação Tempo Observação
Busca (geral) O(n) Sem restrição de ordenação, pode exigir visitar todos os nós.
Travessia O(n) Todo nó é visitado exatamente uma vez.
Altura (balanceada) O(log n) Árvore perfeita de n nós tem altura log n.
Altura (degenerada) O(n) Lista linear disfarçada de árvore.
Espaço O(n) Um nó por elemento armazenado.
