Árvores AVL: equilíbrio garantido

A primeira estrutura de dados auto-balanceada da história O(log n) garantido em toda operação.

Proposta por Adelson-Velsky e Landis em 1962, a árvore AVL resolve o problema de degeneração das BSTs comuns. A cada inserção ou remoção, o balanceamento é verificado e corrigido automaticamente via rotações, mantendo a altura sempre logarítmica.

O que é uma Árvore AVL?

Uma árvore AVL é uma BST onde cada nó armazena um fator de balanceamento (FB): a diferença entre a altura da subárvore direita e a subárvore esquerda. A invariante da AVL é que este fator pertença ao conjunto {1, 0, +1} para todos os nós.

Segundo Adelson-Velsky e Landis (1962), essa restrição de diferença de altura garantida garante que a árvore com n nós tenha altura no máximo 1,44 log(n+2) ou seja, O(log n).

O fator de balanceamento é mantido automaticamente. Quando uma inserção ou remoção viola a invariante (FB fora de {1, 0, +1}), a árvore executa uma rotação para restaurar o equilíbrio.

Rotações

Existem quatro tipos de desequilíbrio, cada um corrigido por um tipo de rotação:

Rotação Simples Direita (LL)

Desequilíbrio à esquerda do filho esquerdo. O filho esquerdo sobe e torna-se a nova raiz do subconjunto. O nó original desce para a direita.

Rotação Simples Esquerda (RR)

Espelho do caso LL. Desequilíbrio à direita do filho direito. O filho direito sobe e o nó original desce para a esquerda.

Rotação Dupla Esquerda-Direita (LR)

Desequilíbrio à direita do filho esquerdo. Primeiro rotate-esquerda no filho, depois rotate-direita na raiz. Dois passos.

Rotação Dupla Direita-Esquerda (RL)

Desequilíbrio à esquerda do filho direito. Primeiro rotate-direita no filho, depois rotate-esquerda na raiz. Dois passos.

Implementação

Cada nó armazena a altura da subárvore. O fator de balanceamento é calculado como altura(direito) altura(esquerdo). Após cada inserção, atualiza-se a altura e verifica-se o balanceamento subindo pela árvore.

avl.pseudo
FUNCAO fatorBalanceamento(no) {
    RETORNE altura(no.direito) - altura(no.esquerdo)
}

FUNCAO rebalancear(no) {
    fb = fatorBalanceamento(no)
    SE fb > 1 {                           // desequilíbrio direita
        SE fatorBalanceamento(no.direito) < 0
            no.direito = rotateDireita(no.direito)  // RL
        RETORNE rotateEsquerda(no)
    }
    SE fb < -1 {                          // desequilíbrio esquerda
        SE fatorBalanceamento(no.esquerdo) > 0
            no.esquerdo = rotateEsquerda(no.esquerdo) // LR
        RETORNE rotateDireita(no)
    }
    RETORNE no
}

FUNCAO inserir(no, valor) {
    no = inserirBST(no, valor)    // inserção normal de BST
    atualizarAltura(no)
    RETORNE rebalancear(no)
}

Cada rotação custa O(1), mas a inserção pode exigir até O(log n) atualizações de altura ao subir pela árvore. Na remoção podem ser necessárias múltiplas rotações ao longo do caminho até a raiz.

Complexidade

Operação Tempo Observação
Busca O(log n) Garantido pelo balanceamento. Sem pior caso degenerado.
Inserção O(log n) Inclui no máximo uma rotação simples ou dupla.
Remoção O(log n) Pode exigir O(log n) rotações no caminho até a raiz.
Rotação O(1) Apenas redirecionamento de ponteiros.
Espaço extra por nó O(1) Apenas o campo de altura (ou fator de balanceamento).

A AVL é a solução clássica para o desequilíbrio de BSTs. Para cargas com mais inserções que buscas, a Rubro-Negra pode ser preferível.

"Uma árvore AVL é uma árvore de busca binária que é balanceada por altura: para cada nó p, as alturas das subárvores esquerda e direita de p diferem em no máximo 1." Goodrich e Tamassia, Estruturas de Dados e Algoritmos em Java
