Tabelas Hash

Tabelas Hash: O(1) na média para tudo

Buscas, inserções e remoções em tempo constante amortizado à custa de uma boa função de dispersão.

Toda vez que Python executa valor = dicionario["chave"], o JavaScript acessa uma propriedade de objeto, ou um sistema verifica uma senha em cache uma tabela hash está trabalhando. É a estrutura de dados com melhor relação custo-benefício para a maioria dos problemas de dicionário.

O que é uma Tabela Hash?

Uma tabela hash é uma estrutura de dados que mapeia chaves para valores usando uma função de dispersão (hash function) para calcular o índice de um array onde o valor deve ser armazenado. O resultado é acesso O(1) amortizado em vez de O(n) de uma lista ou O(log n) de uma árvore.

Segundo Cormen et al. (2022), uma tabela hash é uma estrutura de dados eficiente para implementar dicionários. Embora a busca em um elemento possa levar O(n) no pior caso, na prática as tabelas hash têm desempenho excelente.

Tabela hash e dicionário são conceitos distintos: dicionário é a interface (mapeamento chave-valor); tabela hash é uma implementação possível dessa interface. Outras implementações incluem BSTs, AVLs e listas de associação.

Como Funciona

Função de Dispersão

Transforma a chave em um índice inteiro. Uma boa função distribui as chaves uniformemente pelos buckets, minimizando colisões. Ex: hash(k) = k mod m.

Fator de Carga

Razão entre o número de elementos inseridos n e o número de buckets m: α = n/m. Quando α cresce, as colisões aumentam. A tabela é tipicamente redimensionada quando α ultrapassa 0,7 a 0,75.

Redimensionamento

Quando o fator de carga ultrapassa o limite, a tabela dobra de tamanho e reinsere todos os elementos. Embora O(n) por evento, o custo amortizado por inserção continua O(1).

Tratamento de Colisões

Uma colisão ocorre quando duas chaves distintas produzem o mesmo índice. Duas estratégias dominam:

Encadeamento Separado

Cada bucket armazena uma lista encadeada. Colisões são resolvidas inserindo na lista do bucket correspondente. Simples, mas usa memória extra para os ponteiros.

Endereçamento Aberto

Todos os elementos ficam no próprio array. Ao colidir, procura-se o próximo slot livre pela estratégia de probing: linear (+1), quadrático (+i²) ou duplo hashing.

Com uma função hash ruim que concentra todas as chaves no mesmo bucket, a tabela hash degenera para uma lista encadeada com complexidade O(n) em todas as operações. A escolha da função hash é tão importante quanto a estrutura em si.

Implementação

tabela_hash.pseudo
ESTRUTURA TabelaHash {
    m       = 16          // número de buckets
    buckets = Array(m, listas_vazias)

    FUNCAO hash(chave) {
        RETORNE chave.codigoHash() % m
    }

    FUNCAO inserir(chave, valor) {
        idx = hash(chave)
        buckets[idx].adicionarOuSubstituir(chave, valor)
    }

    FUNCAO buscar(chave) {
        idx = hash(chave)
        RETORNE buckets[idx].encontrar(chave)
    }

    FUNCAO remover(chave) {
        idx = hash(chave)
        buckets[idx].remover(chave)
    }
}

Complexidade

Operação Caso médio Pior caso
Busca O(1) O(n)
Inserção O(1) O(n)
Remoção O(1) O(n)
Redimensionamento O(n) por evento O(1) amortizado
Espaço O(n) O(n)

O pior caso O(n) ocorre somente com uma função hash patológica ou ataques deliberados de colisão. Implementações modernas usam hashing aleatório para mitigar o problema.
