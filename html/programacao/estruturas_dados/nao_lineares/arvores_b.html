Árvores B (B-Trees)

Árvores B: índices que vivem no disco

A estrutura projetada para minimizar leituras de disco ao armazenar múltiplas chaves por nó e manter a árvore com poucos níveis.

Toda vez que você faz uma consulta em um banco de dados relacional, uma árvore B (ou B+) está trabalhando nos bastidores. Projetada nos anos 1970, a árvore B nasceu de uma necessidade concreta: os discos rígidos são ordens de magnitude mais lentos que a memória RAM, e cada nível de árvore percorrido é um acesso a disco.

O que é uma Árvore B?

Uma árvore B de ordem m é uma árvore de busca balanceada generalizada onde cada nó pode armazenar múltiplas chaves e ter múltiplos filhos. A ordem m define os limites: cada nó (exceto a raiz) tem entre m/2 1 e m 1 chaves e entre m/2 e m filhos.

Segundo Bayer e McCreight (1972), a B-Tree foi projetada para sistemas de armazenamento secundário: ao aumentar o fator de ramificação (ordem m), a altura da árvore decresce drasticamente, reduzindo o número de acessos ao disco.

Um banco de dados real usa ordem m na casa das centenas a milhares. Uma árvore B de ordem 1000 com um bilhão de registros terá altura 3 ou seja, no máximo 3 leituras de disco para qualquer busca.

Estrutura de um Nó

Propriedade de ordenação

As chaves dentro de cada nó estão em ordem crescente. Para um nó com chaves k < k, todos os valores no filho central estão entre k e k.

Divisão de nós (split)

Quando um nó atinge a capacidade máxima (m1 chaves), ele é dividido ao meio. A chave central sobe para o nó pai. Essa operação propaga o crescimento para cima.

Fusão de nós (merge)

Na remoção, se um nó fica com menos que m/21 chaves, ele se funde com um vizinho ou redistribui chaves. A árvore nunca fica desequilibrada verticalmente.

Todas as folhas no mesmo nível

Diferente de BSTs, todas as folhas de uma árvore B estão sempre na mesma profundidade. Isso garante O(log n) mesmo sem rotações.

Casos de Uso

Bancos de dados relacionais

MySQL InnoDB, PostgreSQL e Oracle usam B-Trees (ou B+) para índices primários e secundários. Uma consulta com WHERE id = X percorre uma B-Tree.

Sistemas de arquivos

NTFS, HFS+, ext4 e Btrfs usam B-Trees para estruturar o diretório de arquivos, tornando a busca por arquivo name em O(log n).

Armazenamento em SSD

Controladores de SSD usam variantes de B-Tree para mapear endereços lógicos em físicos, gerenciando o desgaste das células de forma eficiente.

Complexidade

Operação Tempo Acessos a disco
Busca O(log n) O(log_m n)
Inserção O(log n) O(log_m n)
Remoção O(log n) O(log_m n)
Espaço O(n)

A base do logaritmo é m (a ordem da árvore). Quanto maior m, menor o número de acessos a disco daí a importância de dimensionar a ordem para corresponder ao tamanho de uma página de disco.
