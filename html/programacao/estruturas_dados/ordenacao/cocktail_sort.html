Cocktail Shaker Sort: Bubble Sort para os dois lados

Variante bidirecional do Bubble Sort — percorre o array em ambas as direções alternadamente, como uma coqueteleira.

O Bubble Sort clássico tem um problema conhecido: elementos pequenos no final da lista (chamados "tartarugas") demoram muito para chegar ao início — cada um avança apenas uma posição por passagem. O Cocktail Shaker Sort resolve isso fazendo também a varredura da direita para a esquerda a cada ciclo, trazendo as tartarugas para frente muito mais rápido. Na prática, converge em cerca de metade das passagens do Bubble Sort puro.

O que é o Cocktail Shaker Sort?

Também chamado de Bidirectional Bubble Sort, Cocktail Sort ou Shaker Sort. Knuth no TAOCP vol. 3 demonstrou que o número de passagens é aproximadamente metade do Bubble Sort para dados aleatórios.

O conceito de "tartaruga" foi formalizado: um elemento muito pequeno perto do fim se move apenas 1 posição por passagem no Bubble Sort. A fase ← do Cocktail resolve esse caso específico.

Analogia: Uma coqueteleira é agitada para um lado depois para o outro. Cada ciclo completo (→ + ←) garante dois elementos fixos: o maior no fim e o menor no início.

Como Funciona: Passo a Passo

Cada ciclo tem duas fases. A fase → borbulha o maior ao fim; a fase ← borbulha o menor ao início.

Fase → (esquerda para direita)

Igual ao Bubble Sort: compara pares adjacentes e borbulha o maior ao fim. Ao final, o maior elemento restante está na posição final correta.

Fase ← (direita para esquerda)

Percorre de volta do fim ao início fazendo o processo invertido: borbulha o menor elemento para o início. Isso é o que resolve as "tartarugas" do Bubble Sort.

Implementação em Python

cocktail_sort.py
def cocktail_sort(arr: list) -> list:
    n = len(arr)
    inicio, fim = 0, n - 1
    trocou = True

    while trocou:
        trocou = False

        # Fase →: borbulha o maior elemento ao fim
        for i in range(inicio, fim):
            if arr[i] > arr[i + 1]:
                arr[i], arr[i + 1] = arr[i + 1], arr[i]
                trocou = True

        if not trocou:
            break

        fim -= 1       # maior elemento fixo no fim
        trocou = False

        # Fase ←: borbulha o menor elemento ao início
        for i in range(fim, inicio, -1):
            if arr[i] < arr[i - 1]:
                arr[i], arr[i - 1] = arr[i - 1], arr[i]
                trocou = True

        inicio += 1    # menor elemento fixo no início

    return arr


if __name__ == "__main__":
    dados = [5, 3, 8, 1, 9, 2]
    print(cocktail_sort(dados))  # [1, 2, 3, 5, 8, 9]
  • inicio e fim se aproximam a cada ciclo, reduzindo a região ativa.
  • A flag trocou é verificada após a fase → — se não houver trocas, paramos antes da fase ←.
  • Cada ciclo completo fixa dois elementos: o maior no fim e o menor no início.

Estabilidade: Estável — trocas ocorrem apenas com comparações estritas, elementos iguais não trocam de posição.

Quando Usar

Vantagem sobre Bubble Sort

Resolve eficientemente o problema de tartarugas. Para arrays quase-ordenados pode ser 2x mais rápido que o Bubble Sort puro.

Limitações

Ainda é O(n²) no caso médio. Para n > 1000 prefira Merge Sort, Quick Sort ou Tim Sort. É principalmente um algoritmo didático e histórico.

Produção: Cocktail Sort ainda é O(n²) no caso médio — para arrays com mais de algumas dezenas de elementos prefira Insertion Sort, Merge Sort ou Tim Sort. Seu diferencial real é apenas para arrays com muitas "tartarugas" (pequenos valores no final).
