Gnome Sort

Gnome Sort: o gnomo que organiza flores

Algoritmo com um único ponteiro que avança quando está certo e recua trocando quando está errado — como um gnomo reorganizando vasos de flores.

Proposto por Hamid Sarbazi-Azad em 2000 e popularizado por Dick Grune, o Gnome Sort é provavelmente o algoritmo de ordenação com a implementação mais simples: usa apenas um ponteiro e uma condição — se os vizinhos estão na ordem certa, avança; se não estão, troca e recua. É funcionalmente equivalente ao Insertion Sort mas com uma lógica de movimentação diferente.

O que é o Gnome Sort?

O algoritmo mantém um índice pos que funciona como o gnomo. Em cada passo:

  • Se pos == 0 ou arr[pos] >= arr[pos-1]: o par está correto, avança (pos++).
  • Senão: troca arr[pos] com arr[pos-1] e recua (pos--).

O gnomo carrega o elemento menor para trás, da mesma forma que o Insertion Sort insere um elemento na posição correta na parte já ordenada.

Analogia: Um gnomo de jardim precisa deixar os vasos de flores em ordem de tamanho. Ele olha para o vaso atual e o anterior: se estiver na ordem certa, vai para frente. Se não estiver, troca os dois vasos e dá um passo para trás para verificar de novo.

Como Funciona: Passo a Passo

Ordenando [5, 3, 1, 4]. O marcador (▼) indica a posição do ponteiro pos.

O ponteiro avança continuamente até encontrar um elemento fora de lugar, então recua carregando-o para a posição correta — exatamente como o Insertion Sort funciona internamente.

Implementação em Python

gnome_sort.py
def gnome_sort(arr: list) -> list:
    pos = 0
    n = len(arr)

    while pos < n:
        if pos == 0 or arr[pos] >= arr[pos - 1]:
            # Par correto ou no início: avança
            pos += 1
        else:
            # Fora de ordem: troca e recua
            arr[pos], arr[pos - 1] = arr[pos - 1], arr[pos]
            pos -= 1

    return arr


if __name__ == "__main__":
    dados = [5, 3, 1, 4]
    print(gnome_sort(dados))  # [1, 3, 4, 5]

O Gnome Sort tem a implementação mais curta de todos os algoritmos de ordenação de comparação — apenas 6 linhas efetivas. É funcionalmente idêntico ao Insertion Sort em número de comparações e trocas, mas com uma lógica de controle de fluxo diferente.

Análise de Complexidade

Estabilidade: Estável — a condição de troca usa < estrito, elementos iguais não trocam de posição.

Quando Usar

Vantagens

Implementação extremamente simples — ideal para demonstrar conceitos básicos de algoritmos em uma ou duas linhas. Sem loops aninhados explícitos, apenas um while com uma condição.

Limitações

Performance idêntica ao Insertion Sort mas com overhead de movimentação. Para produção sempre prefira Insertion Sort (arrays pequenos) ou Tim Sort (geral).

Produção: Gnome Sort executa O(n²) comparações e movimentos no caso médio, sem vantagem sobre o Insertion Sort — que tem a mesma complexidade mas menos overhead. Use Insertion Sort se precisar de um algoritmo simples e in-place para pequenos arrays.
