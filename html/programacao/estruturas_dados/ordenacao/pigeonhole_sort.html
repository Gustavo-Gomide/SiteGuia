Pigeonhole Sort

Pigeonhole Sort: um buraco por pombo

Algoritmo de ordenação por endereçamento direto. Cada valor possível tem exatamente um "buraco" e cada elemento vai diretamente para o seu.

O Pigeonhole Sort é análogo ao Counting Sort, mas ao invés de contar ocorrências e reconstruir, ele literalmente distribui os elementos em listas (buracos) indexadas pelo valor e depois recolhe em ordem. É ideal quando o número de valores distintos k é próximo de n.

O que é o Pigeonhole Sort?

O nome vem do "princípio das gaivinhas" (pigeonhole principle). A ideia é: se você tem n pombos e n buracos, cada pombo vai para seu buraco. Da mesma forma, cada elemento vai para um slot indexado pelo seu valor.

Diferente do Counting Sort que guarda apenas contagem, o Pigeonhole Sort guarda os próprios elementos nas listas, então é estável — se dois elementos têm o mesmo valor, a ordem relativa é preservada.

Quando usar: Funciona bem quando o intervalo de valores k ≈ n. Se k for muito maior que n (ex: ordenar 100 números entre 0 e 10.000.000), o desperdício de memória e tempo para criar os buracos supera qualquer benefício.

Como Funciona: Passo a Passo

Para [8, 3, 2, 7, 4, 6, 8]: range = [2..8], 7 buracos.

Implementação em Python

pigeonhole_sort.py
def pigeonhole_sort(arr: list) -> list:
    if not arr:
        return arr

    minVal = min(arr)
    maxVal = max(arr)
    tamanho = maxVal - minVal + 1

    # Cria os "buracos" (listas vazias, uma por valor possível)
    buracos = [[] for _ in range(tamanho)]

    # Fase 1: distribui cada elemento em seu buraco
    for elem in arr:
        buracos[elem - minVal].append(elem)

    # Fase 2: recolhe os buracos em ordem
    idx = 0
    for buraco in buracos:
        for elem in buraco:
            arr[idx] = elem
            idx += 1

    return arr


if __name__ == "__main__":
    dados = [8, 3, 2, 7, 4, 6, 8]
    print(pigeonhole_sort(dados))  # [2, 3, 4, 6, 7, 8, 8]

Pontos importantes do código:

  • tamanho = maxVal - minVal + 1 cria exatamente um buraco por valor possível no intervalo — sem desperdício, sem colissão.
  • buracos[elem - minVal].append(elem) normaliza o índice para começar em 0, mesmo que os valores reais comecem em outro número (ex: 50, 51...).
  • A fase 2 não cria um array novo — reescreve o próprio arr in-place, economizando alocação de memória.
  • Buracos vazios são listas vazias — o for elem in buraco simplesmente não itera, sem nenhuma condição especial.

Análise de Complexidade

CasoTempoEspaço
TodosO(n + k)O(n + k)

Onde k = maxVal - minVal + 1 (tamanho do intervalo). Se k ≈ n, a complexidade efetiva é O(n) — linear! Se k ≫ n, o custo de criar e percorrer os buracos domina.

Estabilidade: Estável — elementos iguais são adicionados às listas preservando a ordem original.

Quando Usar

Intervalo pequeno de inteiros

Ordenar notas de alunos (0–100), idades (0–120), dias do mês (1–31). O intervalo k é fixo e pequeno, tornando a ordenação linear.

vs. Counting Sort

Pigeonhole usa listas nos buracos (permite objetos complexos); Counting Sort usa contagem inteira (mais eficiente para dados simples). Ambos têm mesma complexidade assintótica.

Cuidado com o intervalo: Se os dados tiverem valor máximo muito distante do mínimo (ex: [1, 1.000.000]), o algoritmo cria 1.000.000 de buracos — desperdiçando memória e tempo mesmo para um array de apenas 2 elementos. Verifique sempre se k ≈ n antes de usar.
