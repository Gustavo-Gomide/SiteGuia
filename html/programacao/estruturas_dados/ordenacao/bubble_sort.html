Bubble Sort

Bubble Sort: bolhas que sobem

O algoritmo introdutório clássico. Elementos maiores "flutuam" para o final como bolhas em um líquido.

O Bubble Sort é o ponto de partida de todo estudo de ordenação. Sua lógica cabe em duas frases: compare dois vizinhos, troque se estiverem na ordem errada, repita. Em cada passagem completa pelo array, o maior elemento não-ordenado garante sua posição final como uma bolha encontrando seu nível em um líquido.

O que é o Bubble Sort?

O Bubble Sort (em português: Ordenação por Bolha) é um algoritmo de ordenação por comparação que percorre repetidamente uma lista, compara elementos adjacentes e os troca quando estão na ordem errada. O processo se repete até que nenhuma troca seja necessária o que significa que a lista está ordenada.

O algoritmo foi publicado formalmente em 1956 e o nome "bubble sort" foi cunhado em artigos de 1962. Apesar de ser um dos primeiros algoritmos ensinados em CS, é raramente usado em produção: como observa Knuth (1998), o Bubble Sort não tem nenhuma qualidade que o recomende em relação ao Insertion Sort, exceto ser mais fácil de memorizar.

Analogia: Imagine ordenar uma fila de pessoas por altura. Você percorre a fila do início ao fim comparando pares: se a pessoa à esquerda for mais alta, elas trocam de lugar. Após uma passagem completa, a pessoa mais alta está garantidamente no final. Você repete o processo ignorando a última posição (já ordenada) até que nenhuma troca ocorra.

Como Funciona: Passo a Passo

Observe o padrão: após cada passagem, um elemento a mais fica fixo no final (marcado em verde). Isso permite que o loop interno diminua 1 a cada rodada, economizando comparações.

Otimização com flag

Se uma passagem completa terminar sem nenhuma troca, o array já está ordenado. Podemos parar imediatamente. Isso torna o melhor caso O(n) uma única passagem verificando um array já ordenado.

Invariante do loop externo

Após a i-ésima passagem, os i maiores elementos estão em suas posições finais corretas. Essa propriedade garante a corretude do algoritmo por indução.

Implementação em Python

bubble_sort.py
def bubble_sort(arr: list) -> list:
    n = len(arr)

    for i in range(n):
        trocou = False

        # Loop interno: cada passagem garante o maior elemento restante
        # na posição n-1-i (já ordenados ficam fora do loop)
        for j in range(0, n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                trocou = True

        # Otimização: sem trocas = array já ordenado
        if not trocou:
            break

    return arr


# Teste
if __name__ == "__main__":
    dados = [64, 34, 25, 12, 22, 11, 90]
    resultado = bubble_sort(dados.copy())
    print(resultado)  # [11, 12, 22, 25, 34, 64, 90]

Pontos importantes do código:

  • O loop externo (for i in range(n)) controla o número de passagens. No máximo n-1 passagens são necessárias.
  • O loop interno vai até n - 1 - i porque os últimos i elementos já estão nos lugares corretos não precisamos revisitá-los.
  • A flag trocou implementa a otimização: se nenhuma troca ocorreu em uma passagem, o array está ordenado e saímos cedo com break.
  • A troca em Python é elegante: a, b = b, a sem variável temporária.

Análise de Complexidade

CasoTempoEspaçoSituação
MelhorO(n)O(1)Array já ordenado flag detecta e para na 1ª passagem.
MédioO(n²)O(1)Dados aleatórios aproximadamente n²/4 comparações.
PiorO(n²)O(1)Array em ordem inversa todas as n(n-1)/2 trocas necessárias.

Por que O(n²)? No pior caso, o loop externo roda n vezes e o loop interno roda n-1, n-2, ... vezes. A soma é (n-1) + (n-2) + ... + 1 = n(n-1)/2, que é O(n²).

Estabilidade: Estável a troca só ocorre quando arr[j] > arr[j+1] (estritamente maior), então elementos iguais nunca são trocados e mantêm sua ordem relativa original.

Quando Usar (e Quando Não Usar)

Use quando...

O array é muito pequeno (menos de 10-20 elementos), quando a simplicidade da implementação é mais importante que a performance, ou quando você precisa verificar se um array quase-ordenado está de fato ordenado (melhor caso O(n)).

Evite quando...

O array tem mais de algumas dezenas de elementos. Para n=1.000, são ~500.000 comparações. Para n=1.000.000, são ~500 bilhões. O Insertion Sort é sempre preferível ao Bubble Sort nas mesmas condições mesma complexidade, menos overhead.

Uso educacional

O Bubble Sort é excelente para ensinar os conceitos de: invariante de loop, análise de complexidade, estabilidade, otimização com flag de saída antecipada, e por que O(n²) é inaceitável para grandes entradas.

Nota histórica: Em 2000, o ex-presidente dos EUA Barack Obama, quando entrevistado no Google, foi perguntado: "Qual é a forma mais eficiente de ordenar um milhão de inteiros?" Ele respondeu: "Eu acho que o Bubble Sort seria a escolha errada." demonstrando que mesmo fora da área técnica, o Bubble Sort ficou famoso como exemplo do que não fazer.
