Insertion Sort: inserir no lugar certo, um a um

Mantém uma "mão" de cartas sempre ordenada. A cada passo, pega a próxima carta não-ordenada e a encaixa na posição correta dentro da mão já ordenada.

O Insertion Sort Ordenação por Inserção é o algoritmo que qualquer jogador de baralho já executou intuitivamente. Ele constrói uma sequência crescente inserindo cada novo elemento no lugar certo entre os já posicionados. É o algoritmo de escolha quando o array está quase ordenado: suas O(n) iterações no melhor caso batem qualquer algoritmo O(n log n) nessa situação. Por isso o Tim Sort (padrão de Python e Java) usa Insertion Sort para sub-arrays pequenos.

O que é o Insertion Sort?

O Insertion Sort divide logicamente o array em duas partes: uma parte ordenada (à esquerda) que cresce a cada passo, e uma parte não-ordenada (à direita) que encolhe. Para cada elemento da parte não-ordenada, o algoritmo vai "inserindo" esse elemento na posição correta dentro da parte ordenada, deslocando os elementos maiores uma posição para a direita.

Uma propriedade importante: o Insertion Sort é um algoritmo online ele pode ordenar uma lista à medida que os dados chegam, sem precisar ver todos de uma vez. Também é adaptativo: aproveitase de ordenação já presente nos dados. Um array quase-ordenado (cada elemento no máximo k posições fora do lugar) é ordenado em O(nk) tempo.

Analogia das cartas de baralho: Ao comprar cartas de um baralho, você mantém a mão ordenada. Quando recebe uma nova carta, a insere na posição correta: move cartas maiores para a direita até encontrar onde ela se encaixa. Ao fim da compra, a mão inteira está ordenada isso é exatamente o Insertion Sort.

Algoritmo adaptativo

O número de deslocamentos no loop interno depende de quantas posições o elemento precisa "voltar". Se o array já está ordenado, zero deslocamentos são feitos O(n) comparações no total e O(1) escritas por elemento.

Estabilidade garantida

O Insertion Sort é estável: o loop interno para quando encontra um elemento menor ou igual (arr[j] > key). Elementos iguais não são trocados, preservando sua ordem relativa original.

Implementação em Python

insertion_sort.py
def insertion_sort(arr: list) -> list:
    for i in range(1, len(arr)):
        key = arr[i]       # elemento a ser inserido
        j = i - 1

        # Desloca elementos maiores que key uma posição à direita
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1

        arr[j + 1] = key  # insere key na posição correta

    return arr


# Teste
if __name__ == "__main__":
    dados = [5, 2, 4, 6, 1, 3]
    resultado = insertion_sort(dados.copy())
    print(resultado)  # [1, 2, 3, 4, 5, 6]

Pontos importantes do código:

  • O laço começa em i = 1 porque um array de um elemento (índice 0) já está "ordenado" trivialmente.
  • key = arr[i] guarda o valor antes de ser sobrescrito pelos deslocamentos.
  • O while move cada elemento maior uma posição para a direita isso equivale a "abrir espaço" para a inserção.
  • Ao fim do while, j+1 é a posição correta: ou chegamos no início, ou encontramos o primeiro elemento menor ou igual a key.

Análise de Complexidade

CasoTempoEspaçoObservação
MelhorO(n)O(1)Array já ordenado nenhum deslocamento
MédioO(n²)O(1)Cada elemento percorre em média n/4 posições
PiorO(n²)O(1)Array em ordem reversa máximo de deslocamentos

Melhor caso O(n): Se o array já está ordenado, o loop interno nunca entra apenas a condição arr[j] > key é avaliada uma vez por elemento. Isso resulta em exatamente n-1 comparações e nenhuma escrita extra.

Estabilidade: Estável. O loop interno usa arr[j] > key (estrito), portanto nunca move elementos iguais.

Quando Usar

Arrays quase-ordenados

Se os dados já chegam em ordem ou com poucas inversões, o Insertion Sort é extremamente eficiente. Streams de dados de sensores (geralmente ordenados por tempo) são um exemplo clássico.

Sub-arrays pequenos em algoritmos híbridos

O Tim Sort (Python, Java) usa Insertion Sort para runs menores que 64 elementos. O Quick Sort também pode delegar partições pequenas ao Insertion Sort para evitar overhead de recursão.

Grandes conjuntos de dados aleatórios

Para n > 1.000 em dados aleatórios, O(n²) é proibitivo. Use Merge Sort (garantido O(n log n)) ou Tim Sort.

No Python nativo: A função list.sort() e sorted() usam o Tim Sort, que internamente usa Insertion Sort para sub-listas pequenas. Portanto, ao chamar sorted() em Python você indiretamente usa Insertion Sort toda vez.
