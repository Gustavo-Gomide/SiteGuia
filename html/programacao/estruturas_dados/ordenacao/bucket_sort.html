Bucket Sort: distribuir em baldes, ordenar cada balde

Divide o intervalo de valores em n baldes, distribui cada elemento no balde correspondente, ordena cada balde individualmente e concatena. O(n) esperado para distribuições uniformes.

O Bucket Sort é o algoritmo de escolha quando os dados estão uniformemente distribuídos em um intervalo conhecido (como números reais entre 0 e 1, ou porcentagens). Ao distribuir n elementos em n baldes de igual largura, a expectativa é que cada balde tenha O(1) elementos tornando a ordenação de cada balde O(1) amortizado. O resultado é O(n) esperado, quebrando o limite O(n log n) para esse caso específico.

O que é o Bucket Sort?

O Bucket Sort (Ordenação por Baldes) é um algoritmo de ordenação por distribuição. Ele particiona o espaço de valores em intervalos iguais (baldes), cada elemento é inserido no balde correspondente ao seu valor, cada balde é ordenado separadamente (geralmente com Insertion Sort), e os baldes são concatenados.

Funciona melhor quando a distribuição dos dados de entrada é uniforme: se os n elementos estão uniformemente distribuídos em n baldes, a análise probabilística mostra que o custo esperado de ordenar todos os baldes é O(n). Se a distribuição for assimétrica ou todos os elementos caírem em um único balde, o desempenho degrada para o do algoritmo usado em cada balde (O(n²) para Insertion Sort).

Analogia do caixa de correspondência: Distribuir cartas pelos andares de um prédio: cada andar tem uma caixa (balde). A entregadora lê o número do andar (dígito relevante) e coloca na caixa correta. Depois, dentro de cada caixa, as poucas cartas são facilmente ordenadas pelo número da porta.

Número de baldes

Usar n baldes para n elementos maximiza a chance de cada balde ter O(1) elementos. Menos baldes reduz memória mas aumenta o tamanho médio de cada balde há um trade-off entre uso de memória e velocidade.

Algoritmo interno por balde

Baldes pequenos são melhor ordenados com Insertion Sort (rápido para listas pequenas). Para baldes maiores, Merge Sort ou Quick Sort. Usar o próprio Bucket Sort recursivamente também é possível.

Implementação em Python

bucket_sort.py
def bucket_sort(arr: list, n_baldes: int = None) -> list:
    if len(arr) <= 1:
        return arr

    n = len(arr)
    if n_baldes is None:
        n_baldes = n

    minv, maxv = min(arr), max(arr)
    intervalo  = (maxv - minv) / n_baldes

    # Cria n_baldes listas vazias
    baldes = [[] for _ in range(n_baldes)]

    # Distribui cada elemento no balde correto
    for val in arr:
        idx = int((val - minv) / intervalo)
        if idx == n_baldes:   # val == maxv cai no último balde
            idx -= 1
        baldes[idx].append(val)

    # Ordena cada balde individualmente
    resultado = []
    for balde in baldes:
        resultado.extend(sorted(balde))  # usa Tim Sort (Python builtin)

    return resultado


# Teste
if __name__ == "__main__":
    dados = [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21]
    resultado = bucket_sort(dados)
    print(resultado)  # [0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94]

Pontos importantes do código:

  • intervalo = (maxv - minv) / n_baldes calcula a largura de cada balde, normalizando para qualquer intervalo [minv, maxv].
  • A verificação if idx == n_baldes: idx -= 1 trata o elemento máximo, que produziria índice fora dos limites pela divisão inteira.
  • Usar sorted(balde) delega ao Tim Sort do Python ótimo para baldes pequenos de dados quasi-ordenados.
  • Para dados inteiros, adaptar o cálculo de índice para garantir que cada balde cobre exatamente os valores corretos.

Análise do caso médio: Se os n elementos são uniformemente distribuídos em n baldes, o valor esperado do número de elementos por balde é 1. Ordenar um balde de k elementos com Insertion Sort custa O(k²). A soma esperada de k² para todos os baldes é O(n). Com n baldes, concatenar é O(n). Total: O(n).

Estabilidade: Estável se o algoritmo interno de cada balde for estável.

Quando Usar

Dados uniformemente distribuídos

Scores normalizados (0.0 a 1.0), porcentagens, dados gerados por distribuição uniforme. Se a hipótese de uniformidade se sustenta, Bucket Sort bate algoritmos O(n log n) em prática.

Dados de ponto flutuante

Enquanto Counting Sort e Radix Sort funcionam com inteiros, Bucket Sort se adapta naturalmente a floats normalizados basta definir os baldes no intervalo [0.0, 1.0) ou normalizar os dados.

Distribuições não-uniformes

Distribuições com caudas pesadas, bimodais, ou com muitos valores repetidos degradam o desempenho. Se a distribuição é desconhecida, prefira Quick Sort ou Tim Sort.
