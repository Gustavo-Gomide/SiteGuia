Cycle Sort: o mínimo de escritas possível

Algoritmo ótimo em número de escritas na memória — cada elemento é movido no máximo uma vez para sua posição final correta.

Proposto por B.R. Sampson em 1989, o Cycle Sort é o único algoritmo de ordenação por comparação que realiza o número mínimo teórico de escritas. Enquanto outros algoritmos podem escrever o mesmo elemento múltiplas vezes durante trocas intermediárias, o Cycle Sort identifica a posição final correta de cada elemento e o leva diretamente para lá — cada elemento é escrito exatamente uma vez. Isso o torna ideal para memórias com vida útil limitada por número de ciclos de escrita, como EEPROM e flash NOR.

O que é o Cycle Sort?

A ideia se baseia na teoria de decomposição de permutações em ciclos. Qualquer permutação pode ser decomposta em ciclos disjuntos: um ciclo (a → b → c → a) significa que a deve ir para onde está b, b para onde está c, e c para onde está a.

O Cycle Sort encontra esses ciclos um por um. Para cada elemento arr[i], conta quantos elementos do array são menores que ele — isso determina sua posição correta. Em seguida, percorre o ciclo colocando cada elemento em seu lugar definitivo.

Por que mínimo de escritas? Cada elemento participa de exatamente um ciclo. Uma vez que você inicia um ciclo, cada elemento nele é escrito exatamente uma vez — diretamente na sua posição final. Isso dá o mínimo teórico de n−c escritas, onde c é o número de ciclos.

Como Funciona: Passo a Passo

Para [4, 3, 2, 1]: há dois ciclos — (4↔1) e (3↔2) — cada um exigindo 1 escrita.

Implementação em Python

cycle_sort.py
def cycle_sort(arr: list) -> list:
    n = len(arr)
    escritas = 0

    for inicio_ciclo in range(n - 1):
        item = arr[inicio_ciclo]

        # Encontra a posição correta de 'item'
        pos = inicio_ciclo
        for i in range(inicio_ciclo + 1, n):
            if arr[i] < item:
                pos += 1

        if pos == inicio_ciclo:
            continue    # elemento já está no lugar

        # Pula duplicatas
        while item == arr[pos]:
            pos += 1

        # Coloca item na posição correta
        arr[pos], item = item, arr[pos]
        escritas += 1

        # Rotaciona o restante do ciclo
        while pos != inicio_ciclo:
            pos = inicio_ciclo
            for i in range(inicio_ciclo + 1, n):
                if arr[i] < item:
                    pos += 1
            while item == arr[pos]:
                pos += 1
            arr[pos], item = item, arr[pos]
            escritas += 1

    return arr


if __name__ == "__main__":
    dados = [4, 3, 2, 1]
    print(cycle_sort(dados))  # [1, 2, 3, 4]
  • Para cada elemento, conta quantos menores existem à direita — isso determina sua posição correta.
  • A variável escritas conta o total de escritas — será sempre o mínimo teórico (n−número_de_ciclos).
  • O tratamento de duplicatas garante que elementos iguais sejam tratados corretamente sem loops infinitos.

Mínimo de escritas: O número de escritas é exatamente n − c, onde c é o número de ciclos na permutação. Este é o mínimo teórico possível para qualquer algoritmo de ordenação por comparação in-place.

Estabilidade: Não estável — o ciclo pode mover elementos iguais de forma não relativa.

Quando Usar

EEPROM e Flash

Memórias com ciclos de escrita limitados (EEPROM, Flash NOR) degradam a cada escrita. O Cycle Sort minimiza o número de escritas, estendendo a vida útil da memória em aplicações embarcadas.

Limitações

O(n²) em tempo — o loop de contagem de posição é quadrático. Para dados em RAM onde escritas não custam mais que leituras, algoritmos O(n log n) são sempre melhores escolhas.

Caso de uso real: Cycle Sort minimiza o número de escritas — o que é crítico em memórias flash com vida útil limitada por ciclos de escrita. Fora desse cenário específico, é O(n²) e geralmente não é a melhor escolha.
