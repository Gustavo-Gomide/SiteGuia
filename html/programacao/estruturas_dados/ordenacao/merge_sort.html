Merge Sort: dividir, ordenar, mesclar

Divide o problema ao meio recursivamente até ter sub-problemas triviais (arrays de tamanho 1), depois mescla as partes ordenadas de volta garantindo O(n log n) em qualquer caso.

O Merge Sort é o primeiro algoritmo que demonstra o poder do paradigma dividir e conquistar aplicado à ordenação. Ao contrário do Bubble Sort e do Selection Sort, ele nunca degrada para O(n²): o pior caso é idêntico ao melhor caso O(n log n). Essa previsibilidade o torna o algoritmo de escolha para sistemas onde garantias de tempo de resposta são críticas. Em troca, ele requer O(n) de memória auxiliar.

O que é o Merge Sort?

O Merge Sort foi concebido por John von Neumann em 1945, durante o desenvolvimento do computador EDVAC. É um dos algoritmos mais antigos ainda em uso diário o Python usa uma variante dele (Tim Sort) como mecanismo de ordenação padrão. A operação central é o merge (mesclagem): dadas duas listas ordenadas, produzir uma única lista ordenada em tempo O(n).

O algoritmo é fundamentalmente recursivo: divide o array ao meio, ordena cada metade (chamando a si mesmo), e mescla os dois resultados. A base da recursão é um array de tamanho 0 ou 1, que já está trivialmente ordenado.

Analogia dos baralhos: Imagine duas pilhas de cartas, cada uma já ordenada. Para mesclá-las, você olha o topo de cada pilha, pega o menor, coloca na pilha resultado, e repete. Ao fim, a pilha resultado está completamente ordenada. O Merge Sort faz exatamente isso, escalonando recursivamente.

A operação de merge

Dados dois arrays ordenados de tamanho p e q, o merge produz um array de tamanho p+q em O(p+q) comparações. Ele compara os elementos frontais de cada array e vai movendo o menor para o resultado. Quando um array esgota, os restantes do outro são copiados diretamente.

Por que O(n log n)?

A recursão cria log(n) níveis (dividindo ao meio a cada nível). Em cada nível, o trabalho total de merge é O(n) (todos os elementos são tocados uma vez). Logo: log(n) níveis O(n) por nível = O(n log n).

Implementação em Python

merge_sort.py
def merge_sort(arr: list) -> list:
    if len(arr) <= 1:
        return arr  # caso base: já ordenado

    meio = len(arr) // 2
    esq  = merge_sort(arr[:meio])
    dir  = merge_sort(arr[meio:])

    return _merge(esq, dir)


def _merge(esq: list, dir: list) -> list:
    resultado = []
    i = j = 0

    # Compara frentes de cada metade e insere o menor
    while i < len(esq) and j < len(dir):
        if esq[i] <= dir[j]:
            resultado.append(esq[i])
            i += 1
        else:
            resultado.append(dir[j])
            j += 1

    # Copia os elementos restantes (um dos lados já esgotou)
    resultado.extend(esq[i:])
    resultado.extend(dir[j:])
    return resultado


# Teste
if __name__ == "__main__":
    dados = [38, 27, 43, 9, 82, 10]
    resultado = merge_sort(dados)
    print(resultado)  # [9, 10, 27, 38, 43, 82]

Pontos importantes do código:

  • O caso base len(arr) <= 1 retorna imediatamente um array de 0 ou 1 elemento está sempre ordenado.
  • A divisão arr[:meio] e arr[meio:] cria cópias em Python isso é parte do custo O(n) de espaço.
  • A condição esq[i] <= dir[j] usa "menor ou igual", garantindo a estabilidade: elementos iguais da metade esquerda são inseridos antes dos da direita.
  • resultado.extend(esq[i:]) copia os elementos restantes em batch quando um dos lados esgota, o outro já está ordenado e é simplesmente copiado.

PiorO(n log n)O(n)Garantido não há degradação

Espaço O(n): A principal desvantagem do Merge Sort é que ele precisa de memória auxiliar proporcional ao tamanho do input para armazenar os sub-arrays durante o merge. Existem versões in-place, mas com complexidade de tempo maior e implementação mais complexa.

Estabilidade: Estável. A condição <= no merge garante que elementos iguais da metade esquerda precedem os da direita, mantendo a ordem de ocorrência original.

Quando Usar

Garantia de tempo de resposta

Quando o pior caso importa sistemas de tempo real, cobrança, saúde o Merge Sort é preferível ao Quick Sort (que pode degradar para O(n²) no pior caso sem bom pivot).

Ordenação de listas ligadas

O Merge Sort é naturalmente eficiente em listas ligadas (linked lists): não precisa de acesso aleatório. Já o Quick Sort é muito mais trabalhoso em listas ligadas.

Restrição de memória

O(n) de espaço auxiliar pode ser proibitivo em sistemas embarcados ou com grandes volumes de dados. Nesse caso, prefira Heap Sort (O(1) de espaço extra, O(n log n) garantido).
