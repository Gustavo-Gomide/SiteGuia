Heap Sort

Heap Sort: construir o monte, extrair o máximo

Transforma o array em um max-heap (onde cada pai é maior que seus filhos), depois extrai o maior elemento repetidamente ordenando in-place com garantia O(n log n) em qualquer caso.

O Heap Sort combina as melhores propriedades do Selection Sort e do Merge Sort: como o Selection Sort, é in-place (O(1) de memória auxiliar); como o Merge Sort, garante O(n log n) mesmo no pior caso. Ele usa a estrutura de dados heap (montículo) implicitamente dentro do próprio array, sem alocar espaço extra. É o algoritmo ideal quando memória é escassa e garantias de tempo são necessárias.

O que é o Heap Sort?

O Heap Sort foi introduzido por J.W.J. Williams em 1964 no mesmo artigo em que ele definiu a estrutura de dados heap. Robert Floyd aprimorou o algoritmo posteriormente com a função heapify (ou sift-down), tornando a fase de construção do heap O(n) em vez de O(n log n).

Um max-heap é um array em que para qualquer índice i, o elemento arr[i] é maior ou igual a arr[2i+1] (filho esquerdo) e arr[2i+2] (filho direito). A raiz (índice 0) é sempre o maior elemento. O Heap Sort explora isso: extrai a raiz (máximo), coloca no fim do array, diminui o heap em 1, e restaura a propriedade heap.

Analogia do torneio: Imagine um torneio de eliminação onde o vencedor é sempre o maior. O heap é a tabela do torneio armazenada no array. O Heap Sort tira o campeão (raiz), coloca no pódio (fim do array), e organiza um novo torneio sem ele repetidamente.

Como Funciona: Passo a Passo

O algoritmo tem duas fases:

Fase 1: Build Heap O(n)

Aplica heapify de baixo para cima, começando pelo último nó interno (n//2 - 1) até a raiz. Apesar de parecer O(n log n), a análise amortizada comprova que é O(n) nós mais fundos fazem menos trabalho.

Fase 2: Extração O(n log n)

n extrações de O(log n) cada. A raiz (máximo) troca com o último elemento, o heap encolhe em 1, e sift-down restaura a propriedade heap. Após n extrações, o array está ordenado de fora para dentro.

Implementação em Python

heap_sort.py
def heap_sort(arr: list) -> list:
    n = len(arr)

    # Fase 1: construir max-heap (de baixo para cima)
    for i in range(n // 2 - 1, -1, -1):
        _heapify(arr, n, i)

    # Fase 2: extrair máximo um a um
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # move max para o fim
        _heapify(arr, i, 0)              # restaura heap de tamanho i

    return arr


def _heapify(arr: list, tamanho: int, raiz: int) -> None:
    """Sift-down: garante que arr[raiz] seja maior que seus filhos."""
    maior = raiz
    esq   = 2 * raiz + 1
    dir   = 2 * raiz + 2

    if esq < tamanho and arr[esq] > arr[maior]:
        maior = esq
    if dir < tamanho and arr[dir] > arr[maior]:
        maior = dir

    if maior != raiz:
        arr[raiz], arr[maior] = arr[maior], arr[raiz]
        _heapify(arr, tamanho, maior)  # propaga para baixo


# Teste
if __name__ == "__main__":
    dados = [4, 10, 3, 5, 1]
    resultado = heap_sort(dados.copy())
    print(resultado)  # [1, 3, 4, 5, 10]

Pontos importantes do código:

  • O loop de build heap começa em n//2 - 1 porque nós folha (índices n//2) não têm filhos e já satisfazem a propriedade heap trivialmente.
  • _heapify implementa sift-down: compara a raiz com seus dois filhos e troca com o maior deles se necessário, propagando recursivamente até que a propriedade esteja satisfeita.
  • A troca arr[0], arr[i] = arr[i], arr[0] coloca o máximo na posição final correta a mesma lógica do Selection Sort, mas escolhendo o máximo em O(log n) em vez de O(n).
  • O algoritmo é in-place: nenhuma memória adicional proporcional ao input é usada.

Por que mais lento que Quick Sort na prática? O Heap Sort tem péssimo comportamento de cache: o sift-down acessa índices espalhados pelo array (pai filho esquerdo filho direito), sem localidade de referência. O Quick Sort acessa memória sequencialmente durante o particionamento, aproveitando linhas de cache. Por isso, em benchmarks reais, Quick Sort geralmente supera Heap Sort por 2-3x.

Estabilidade: Instável. As trocas durante sift-down podem inverter a ordem de elementos iguais.

Quando Usar

Garantia de pior caso + in-place

Quando se precisa de O(n log n) garantido E O(1) de memória extra o Heap Sort é o único algoritmo de comparação que oferece ambas as propriedades simultaneamente.

IntroSort (fallback do Quick Sort)

A implementação padrão de ordenação em C++ (std::sort) usa IntroSort: começa com Quick Sort e muda para Heap Sort quando a profundidade de recursão passa de 2log(n) evitando o pior caso do Quick Sort.

Quando cache importa

A falta de localidade de referência do Heap Sort o penaliza em CPUs modernas com hierarquias de cache. Para dados em memória RAM, Quick Sort ou Tim Sort serão mais rápidos na prática.
