Pancake Sort

Pancake Sort: ordem com uma espátula

Algoritmo que só pode realizar uma operação: inverter um prefixo do array. Como virar uma pilha de panquecas com uma espátula.

O Pancake Sorting é único: a única operação permitida é o flip — inverter todos os elementos de arr[0] até arr[k]. Foi proposto em 1979 por um estudante de Harvard chamado Bill Gates em seu único artigo acadêmico publicado, quando ainda era estudante antes de fundar a Microsoft. O problema pergunta: qual o número mínimo de flips para ordenar n elementos? Gates e Papadimitriou provaram um limite superior de (5n+5)/3.

O que é o Pancake Sort?

Imagine uma pilha de panquecas de tamanhos diferentes. A única ferramenta disponível é uma espátula que você pode inserir em qualquer ponto e virar todas as panquecas acima dela. O objetivo é ordenar a pilha da menor (topo) para a maior (base).

A estratégia clássica é iterativa: encontre a maior panqueca não ordenada, faça dois flips — um para trazer ela ao topo, outro para colocá-la na posição final correta.

Conexão acadêmica: O artigo de Gates de 1979 com Christos Papadimitriou é "Bounds for Sorting by Prefix Reversal". Em 2011, estudantes del UT Dallas melhoraram o limite superior de Gates usando algoritmos genéticos. O problema do número mínimo exato de flips ainda é em aberto para muitos valores de n.

Como Funciona: Passo a Passo

Para [3, 1, 4, 2]: a estratégia é colocar o maior no fim, depois o segundo maior, e assim por diante.

Implementação em Python

pancake_sort.py
def flip(arr, k):
    """Inverte arr[0..k] (inclusive)."""
    arr[:k + 1] = arr[:k + 1][::-1]


def pancake_sort(arr: list) -> list:
    n = len(arr)

    for tamanho in range(n, 1, -1):
        # Encontra o índice do maior elemento em arr[0..tamanho-1]
        idx_max = arr[:tamanho].index(max(arr[:tamanho]))

        if idx_max == tamanho - 1:
            continue   # já está no lugar

        # Flip 1: se não está no topo, traz ao topo
        if idx_max != 0:
            flip(arr, idx_max)

        # Flip 2: coloca na posição final correta (fim da região ativa)
        flip(arr, tamanho - 1)

    return arr


if __name__ == "__main__":
    dados = [3, 1, 4, 2]
    print(pancake_sort(dados))  # [1, 2, 3, 4]

Pontos importantes do código:

  • O método arr[:k+1].index(max(...)) encontra o índice do maior em O(n) — simples e sem estrutura auxiliar.
  • O if idx_max == tamanho - 1: continue evita flips desnecessários quando o maior já está na posição correta.
  • O if idx_max != 0 evita um flip trivial (flip do próprio topo) quando o maior já está no índice 0.
  • O slice reverso [::-1] é O(k) — eficiente e legível em Python.

O número máximo de flips com esta estratégia é 2(n-1) — dois flips por elemento exceto o primeiro. Gates e Papadimitriou provaram que qualquer sequência pode ser ordenada em no máximo (5n+5)/3 flips.

Estabilidade: Não estável — flips podem reordenar elementos iguais.

Curiosidades

O artigo de Bill Gates

"Bounds for Sorting by Prefix Reversal" (1979) é o único artigo científico publicado por Gates. Ele e Papadimitriou melhoraram o limite conhecido de 2n flips para (5n+5)/3. Gates era estudante de Harvard quando o escreveu.

Problema em aberto

Determinar o número mínimo exato de flips para ordenar qualquer permutação de n elementos é um problema NP-hard para panquecas "queimadas" (onde um lado deve ficar para baixo). Para o caso simples ainda há questões em aberto para grandes n.

Uso prático: O Pancake Sort tem interesse teórico e histórico, mas O(n²) e o overhead dos flips o tornam impraticável em produção. Seu maior valor é em problemas de rearranja com operações restritas, como roteação de genome (pancake sorting biparído) em bioinformática.
