Odd-Even Sort

Odd-Even Sort: Bubble Sort com fases paralelas

Variante do Bubble Sort que alterna entre comparar pares nos índices ímpares e nos pares — permitindo paralelismo em hardware dedicado.

Também chamado de Odd-Even Transposition Sort ou Brick Sort, proposto por Ken Batcher em 1968. A ideia é dividir cada passagem em duas fases: na fase ímpar compara os pares (arr[1],arr[2]), (arr[3],arr[4])...; na fase par compara (arr[0],arr[1]), (arr[2],arr[3])... Cada fase pode ser executada completamente em paralelo — todas as comparações são independentes entre si, tornando-o eficiente em processadores SIMD e redes de ordenação.

O que é o Odd-Even Sort?

O Odd-Even Sort alterna entre duas fases a cada rodada:

  • Fase ímpar: compara e troca pares nos índices (1,2), (3,4), (5,6)... — ou seja, arr[i] e arr[i+1] para i ímpar.
  • Fase par: compara e troca pares nos índices (0,1), (2,3), (4,5)... — ou seja, arr[i] e arr[i+1] para i par.

Após n rodadas (n fases cada), o array está garantidamente ordenado. É provado que n rodadas são suficientes para qualquer permutação.

Conexão com hardware: Em um processador com n/2 núcleos, cada fase leva O(1) — todas as comparações rodam em paralelo. O sorting acontece em O(n) tempo paralelo, comparado a O(n log n) de algoritmos sequenciais.

Como Funciona: Passo a Passo

Para [5, 3, 1, 4, 2]. Em cada fase, os pares comparados são indicados em vermelho.

Implementação em Python

odd_even_sort.py
def odd_even_sort(arr: list) -> list:
    n = len(arr)
    ordenado = False

    while not ordenado:
        ordenado = True

        # Fase ímpar: compara pares (1,2), (3,4), (5,6)...
        for i in range(1, n - 1, 2):
            if arr[i] > arr[i + 1]:
                arr[i], arr[i + 1] = arr[i + 1], arr[i]
                ordenado = False

        # Fase par: compara pares (0,1), (2,3), (4,5)...
        for i in range(0, n - 1, 2):
            if arr[i] > arr[i + 1]:
                arr[i], arr[i + 1] = arr[i + 1], arr[i]
                ordenado = False

    return arr


if __name__ == "__main__":
    dados = [5, 3, 1, 4, 2]
    print(odd_even_sort(dados))  # [1, 2, 3, 4, 5]

Quando Usar

Hardware paralelo

Em sistemas SIMD, processadores vetoriais ou FPGAs onde cada fase pode ser executada em um único ciclo de clock, o Odd-Even Sort é O(n) — linear no número de elementos.

Sequencialmente

Equivalente ao Bubble Sort em performance. Não há vantagem sequencial sobre Insertion Sort ou Quick Sort. Use apenas para fins didáticos ou em contextos de hardware paralelo.

Paralelismo: Odd-Even Sort só vale a pena em hardware paralelo (como processadores vetoriais ou redes de interconexão). Em Python sequencial, é O(n²) igual ao Bubble Sort — prefira Tim Sort para uso geral.
