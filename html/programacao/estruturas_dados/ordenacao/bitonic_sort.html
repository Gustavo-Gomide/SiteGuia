Bitonic Sort: ordenação por rede paralela

Algoritmo de ordenação baseado em rede de comparadores — cada comparação é independente das outras e pode ser executada em paralelo.

Proposto por Ken Batcher em 1968, o Bitonic Sort é um sorting network: uma sequência fixa de comparadores (cada um conecta dois fios e troca se necessário) que ordena qualquer entrada. A grande vantagem é que o padrão de comparações é completamente determinístico e independente dos dados — o que permite execução massivelmente paralela em GPUs e FPGAs, onde o Bitonic Sort é amplamente usado até hoje.

O que é uma sequência Bitônica?

Uma sequência bitônica é uma que primeiro sobe e depois desce (ou vice-versa): ex. [1, 4, 7, 5, 2] sobe até 7 e depois desce. Um array de n=2ᵏ elementos pode ser construído como uma sequência bitônica e depois mesclado para produzir a sequência ordenada.

O algoritmo constrói repetidamente sequências bitônicas: começa criando pares onde um número está em ordem ascendente e o vizinho em ordem descendente. Então mescla, criando sequências bitônicas maiores, até o array inteiro estar ordenado.

Restrição importante: O Bitonic Sort requer que n seja uma potência de 2. Para outros tamanhos, preenche-se com valores sentinela (−∞ ou +∞) até a próxima potência de 2.

Como Funciona: Rede de Comparadores

Para n=8, existem 6 estágios de comparações. Cada coluna vertical mostra um estágio; linhas conectadas = comparadores (troca se o menor não está em cima).

Cada estágio (S1, S2, S3...) pode ser executado completamente em paralelo — todos os comparadores de um estágio são independentes entre si. Isso é o que torna o Bitonic Sort ideal para hardware paralelo.

Implementação em Python

bitonic_sort.py
def comparar_trocar(arr, i, j, ascendente):
    """Troca arr[i] e arr[j] se não estiverem na ordem desejada."""
    if (arr[i] > arr[j]) == ascendente:
        arr[i], arr[j] = arr[j], arr[i]


def mescla_bitonica(arr, inicio, n, ascendente):
    """Mescla uma sequência bitônica em arr[inicio:inicio+n]."""
    if n > 1:
        meio = n // 2
        for i in range(inicio, inicio + meio):
            comparar_trocar(arr, i, i + meio, ascendente)
        mescla_bitonica(arr, inicio, meio, ascendente)
        mescla_bitonica(arr, inicio + meio, meio, ascendente)


def bitonic_sort_rec(arr, inicio, n, ascendente):
    """Ordena arr[inicio:inicio+n] na direção dada."""
    if n > 1:
        meio = n // 2
        # Primeira metade: ascendente; segunda: descendente → forma bitonico
        bitonic_sort_rec(arr, inicio, meio, True)
        bitonic_sort_rec(arr, inicio + meio, meio, False)
        mescla_bitonica(arr, inicio, n, ascendente)


def bitonic_sort(arr: list) -> list:
    # n deve ser potência de 2
    n = len(arr)
    assert n & (n - 1) == 0, "n deve ser potência de 2"
    bitonic_sort_rec(arr, 0, n, True)
    return arr


if __name__ == "__main__":
    dados = [3, 7, 4, 8, 6, 2, 1, 5]
    print(bitonic_sort(dados))  # [1, 2, 3, 4, 5, 6, 7, 8]

Análise de Complexidade

ModoTempoParaleloEspaço
SequencialO(n log² n)—O(log² n)
ParaleloO(log² n)O(n log² n / log² n)O(log² n)

Com n processadores (um por comparador), cada estágio executa em O(1), dando O(log² n) total de etapas paralelas. Para n=1024 são apenas ~100 etapas paralelas.

Estabilidade: Não estável.

Quando Usar

GPU e FPGA

O padrão fixo de comparações é ideal para hardware paralelo. Amplamente usado em computação gráfica, física de partículas e simulações onde arrays grandes precisam ser ordenados em GPU.

Limitações

Requer n = potência de 2. Não estável. Sequencialmente é O(n log² n) que é pior que O(n log n) de Quick/Merge Sort. O valor está no paralelismo, não na eficiência sequencial.

Hardware paralelo: Bitonic Sort brilha em GPUs e FPGAs onde todos os comparadores operam simultaneamente. Em software sequencial, é O(n log²n) — pior que Merge Sort. Use apenas quando o paralelismo físico estiver disponível.
