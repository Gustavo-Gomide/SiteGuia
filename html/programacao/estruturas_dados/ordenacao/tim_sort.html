Tim Sort: o padrão de Python e Java

Algoritmo híbrido que une Insertion Sort e Merge Sort. Criado por Tim Peters em 2002, é o algoritmo padrão de list.sort() em Python e Arrays.sort() em Java.

O Tim Sort detecta runs — subsequências naturalmente ordenadas (crescentes ou decrescentes) — e as usa como base para o Merge Sort. Para sequências muito curtas usa Insertion Sort, que tem baixa constante. O resultado é um algoritmo com O(n) no melhor caso (já ordenado), O(n log n) no pior, estável e extremamente rápido para dados reais.

O que é o Tim Sort?

Tim Peters criou o Tim Sort em 2002 para uso em Python. A intuição é que dados reais raramente são aleatórios — frequentemente contêm regiões já ordenadas (parcialmente ou totalmente). Tim Sort explora essas regiões como "runs".

Um run mínimo (RUN) é tipicamente entre 32 e 64 elementos. Runs menores que esse tamanho são estendidos com Insertion Sort (que é ótimo para pequenos arrays). Depois, todos os runs são mesclados com Merge Sort bottom-up.

Adotado amplamente: Python (list.sort e sorted() desde 2002), Java (Arrays.sort para objetos desde Java 7), Android (runtime), Swift (stdlib desde Swift 5). É o algoritmo de uso geral mais adotado no mundo.

Implementação Simplificada em Python

tim_sort.py
# Implementação simplificada para fins didáticos
# Na prática, use sorted() ou list.sort() que usam o TimSort nativo do CPython

RUN = 32


def insertion_sort(arr: list, esq: int, dir: int) -> None:
    for i in range(esq + 1, dir + 1):
        temp = arr[i]
        j = i - 1
        while j >= esq and arr[j] > temp:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = temp


def merge(arr: list, esq: int, meio: int, dir: int) -> None:
    esq_arr = arr[esq:meio + 1]
    dir_arr = arr[meio + 1:dir + 1]

    i = j = 0
    k = esq

    while i < len(esq_arr) and j < len(dir_arr):
        if esq_arr[i] <= dir_arr[j]:
            arr[k] = esq_arr[i]; i += 1
        else:
            arr[k] = dir_arr[j]; j += 1
        k += 1

    while i < len(esq_arr):
        arr[k] = esq_arr[i]; i += 1; k += 1

    while j < len(dir_arr):
        arr[k] = dir_arr[j]; j += 1; k += 1


def tim_sort(arr: list) -> list:
    n = len(arr)

    # Fase 1: Insertion Sort em blocos de tamanho RUN
    for i in range(0, n, RUN):
        insertion_sort(arr, i, min(i + RUN - 1, n - 1))

    # Fase 2: Merge bottom-up dos runs
    tamanho = RUN
    while tamanho < n:
        for esq in range(0, n, 2 * tamanho):
            meio = min(esq + tamanho - 1, n - 1)
            dir  = min(esq + 2 * tamanho - 1, n - 1)
            if meio < dir:
                merge(arr, esq, meio, dir)
        tamanho *= 2

    return arr


if __name__ == "__main__":
    dados = [5, 3, 1, 2, 4, 9, 7, 8, 6]
    print(tim_sort(dados))  # [1, 2, 3, 4, 5, 6, 7, 8, 9]

Pontos importantes do código:

  • A Fase 1 percorre o array em blocos de tamanho RUN e aplica Insertion Sort em cada um — o min(i + RUN - 1, n - 1) garante que o último bloco (possivelmente menor) seja tratado corretamente.
  • A Fase 2 dobra o tamanho do merge a cada iteração external — começando com tamanho RUN, depois 2×RUN, 4×RUN, até cobrir o array inteiro.
  • O if meio < dir evita tentar fazer merge quando um dos lados está vazio (ocorre no último bloco ímpar).
  • O Tim Sort real no CPython tem otimizações adicionais: detecção de runs naturais decrescentes, galloping mode para merges com muitos elementos iguais, e uma pilha de runs com invariantes estritas.

Estabilidade: Estável — tanto o Insertion Sort interno quanto o Merge Sort preservam a ordem relativa de elementos iguais.

O espaço O(n) vem dos arrays auxiliares no merge. O Tim Sort real em CPython usa otimizações adicionais (galloping mode) que reduzem o número de comparações em merges com muitos elementos iguais.

Por que é o Padrão?

Dados reais têm runs

Em aplicações práticas, arrays parcialmente ordenados são extremamente comuns — logs por timestamp, dados de banco já parcialmente ordenados, listas com inserções recentes no final. Tim Sort é linear nesses casos.

Constantes pequenas

Insertion Sort tem overhead mínimo para arrays pequenos (cache-friendly, poucos ponteiros). Usar ele nos runs pequenos e Merge Sort nos grandes dá o melhor dos dois mundos.

Estável e confiável

Estabilidade é essencial em bancos de dados e UI (ordenar por múltiplos critérios). Tim Sort garante estabilidade com O(n log n) pior caso — nenhum algoritmo in-place faz os dois ao mesmo tempo.

Use o nativo: Em Python, sorted() e list.sort() já usam Tim Sort nativo em C — ordens de magnitude mais rápido que qualquer implementação Python pura. Esta implementação é exclusivamente para fins didáticos.
