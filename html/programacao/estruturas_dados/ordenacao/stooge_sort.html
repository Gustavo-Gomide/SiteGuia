Stooge Sort: pior que bolha, mas correto

Algoritmo recursivo que ordena dividindo o array em três partes de 2/3. Aparece no CLRS como exemplo de recursão ineficiente.

O Stooge Sort tem complexidade O(nlog 3 / log 1.5) ≈ O(n2.71) — pior que o Bubble Sort (O(n²)). Mesmo assim, o algoritmo é correto. Aparece no livro "Introduction to Algorithms" (CLRS) como exercício para mostrar que recursão mal estruturada pode ser mais cara que abordagens iterativas simples.

O que é o Stooge Sort?

A lógica recursiva é a seguinte: se o primeiro elemento for maior que o último, troque-os. Depois, recursivamente ordene os primeiros 2/3 do array, depois os últimos 2/3, e por fim os primeiros 2/3 novamente.

Três recursões sobre 2/3 do array → T(n) = 3T(2n/3) + O(1). Pelo Teorema Mestre: a = 3, b = 3/2, log_b(a) = log(3)/log(1.5) ≈ 2.71. Portanto T(n) = Θ(n^2.71).

Por que existe? CLRS usa o Stooge Sort como problema no capítulo de divisão e conquista para mostrar que o número de subproblemas e tamanho importam muito. É um exemplo didático perfeito de "correto, mas péssimo".

Como Funciona: Divisão em 2/3

Para [3, 1, 4, 2] (n=4): ceil(4*2/3) = 3.

Implementação em Python

stooge_sort.py
import math


def stooge_sort(arr: list, inicio: int = 0, fim: int = -1) -> list:
    if fim == -1:
        fim = len(arr) - 1

    # Caso base: 1 elemento
    if inicio >= fim:
        return arr

    # Se o primeiro é maior que o último, troca
    if arr[inicio] > arr[fim]:
        arr[inicio], arr[fim] = arr[fim], arr[inicio]

    # Se há mais de 2 elementos, ordena recursivamente em 3 passos
    if fim - inicio + 1 > 2:
        t = (fim - inicio + 1) // 3

        # Passo 1: primeiros 2/3
        stooge_sort(arr, inicio, fim - t)

        # Passo 2: últimos 2/3
        stooge_sort(arr, inicio + t, fim)

        # Passo 3: primeiros 2/3 de novo
        stooge_sort(arr, inicio, fim - t)

    return arr


if __name__ == "__main__":
    dados = [3, 1, 4, 2]
    print(stooge_sort(dados))  # [1, 2, 3, 4]

Pontos importantes do código:

  • O parâmetro fim=-1 é uma convenção para detectar a chamada inicial e calcular o índice correto do último elemento automaticamente.
  • O valor de t = (fim - inicio + 1) // 3 é o tamanho de 1/3 da região ativa — usamos divisão inteira, o que pode introduzir pequenas variações mas não afeta a correção.
  • O import math não é necessário nesta versão — a divisão inteira com // substitui math.ceil.

Análise de Complexidade

CasoTempoEspaço (pilha)
TodosΘ(n2.71)O(log n)

A recorrência é T(n) = 3T(2n/3) + O(1). Pelo Teorema Mestre com a=3, b=3/2:

$$\log_{3/2}(3) = \frac{\log 3}{\log 1.5} \approx 2.709$$

Portanto T(n) = Θ(n^2.709) — assintoticamente pior que Bubble Sort O(n²).

Estabilidade: Não estável. Espaço: O(log n) na pilha de recursão.

Contexto de Uso

Fins didáticos apenas

O Stooge Sort aparece no CLRS (problema 2-3) como exercício. É útil para estudar o Teorema Mestre e entender como a divisão escolhida impacta a complexidade final.

Comparativo didático

Se você substituir as 3 chamadas em 2/3 por 2 chamadas em 1/2 (como o Merge Sort), a complexidade cai para O(n log n). Isso ilustra por que a escolha de como dividir é fundamental.

Aviso de uso: Nunca use Stooge Sort em código de produção. Ele é apresentado aqui exclusivamente como estudo de caso de divisão e conquista mal-dimensionada. Para qualquer tarefa real, use sorted() do Python (Tim Sort) ou Merge Sort.
