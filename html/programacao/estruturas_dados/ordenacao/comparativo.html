Como Escolher o Algoritmo Certo

Uso geral (qualquer dado)

Tim Sort É o padrão do Python e Java por um motivo. O(n log n) garantido, estável, e O(n) para dados quase ordenados.

Máxima velocidade, sem estabilidade

Quick Sort (com pivô aleatório) melhores constantes na prática. Ou Introsort (Quick + Heap + Insertion) como em C++ std::sort.

Inteiros com intervalo limitado

Counting Sort (k pequeno) ou Radix Sort (inteiros de largura fixa) quebram a barreira O(n log n).

Memória mínima (O(1) auxiliar)

Heap Sort O(n log n) garantido e O(1) espaço, porém cache-unfriendly. Alternativa: Shell Sort para n moderado.

Array quase ordenado

Insertion Sort (n < 100) ou Tim Sort (qualquer n) ambos detectam runs naturais e aproximam O(n).

Minimizar escritas na memória

Cycle Sort único algoritmo com O(n) escritas totais. Ideal para EEPROM/Flash com limite de ciclos de escrita.
