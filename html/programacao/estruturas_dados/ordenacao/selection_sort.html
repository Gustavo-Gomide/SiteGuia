Selection Sort

Selection Sort: selecionar o mínimo, repetidamente

Divide o array em uma parte ordenada (cresce da esquerda) e uma não-ordenada (encolhe da direita). Sempre seleciona o menor da parte não-ordenada e o coloca no fim da parte ordenada.

O Selection Sort resolve o problema de forma intuitiva: encontre o menor elemento da lista não-ordenada, coloque-o na primeira posição livre. Repita para o segundo menor, terceiro menor, e assim por diante. Após n-1 seleções, o array inteiro está ordenado. Sua grande vantagem é minimizar escritas na memória faz no máximo n-1 trocas.

O que é o Selection Sort?

O Selection Sort (Ordenação por Seleção) mantém dois "mundos" no array: uma partição ordenada, que começa vazia e cresce pela esquerda, e uma partição não-ordenada, que começa com todo o array e encolhe pela direita. A cada iteração, o menor elemento da parte não-ordenada é selecionado e movido para a fronteira entre os dois mundos.

Segundo Cormen et al. (2022), o Selection Sort é útil para entender a distinção entre o número de comparações (sempre O(n²)) e o número de trocas (sempre exatamente n-1 no pior caso) sendo o algoritmo com menor número de escritas entre os O(n²).

Analogia: Imagine organizar cartas de baralho espalhadas na mesa. Você varre a mesa inteira para encontrar o menor valor, coloca-o na primeira posição do monte ordenado. Depois varre novamente para encontrar o segundo menor, coloca na segunda posição. E assim por diante cada varredura garante uma carta em seu lugar definitivo.

Como Funciona: Passo a Passo

Exatamente n-1 trocas

Em cada round, exatamente uma troca acontece (ou zero, se o mínimo já está no lugar). Com n elementos, são no máximo n-1 trocas muito menos que o Bubble Sort, que pode fazer O(n²) trocas no pior caso.

Instabilidade

O Selection Sort é instável. A troca pode separar elementos iguais que estavam juntos. Exemplo: ordenar [5a, 3, 5b, 1] por valor pode resultar em [1, 3, 5b, 5a] a ordem relativa dos dois 5s foi invertida.

Implementação em Python

selection_sort.py
def selection_sort(arr: list) -> list:
    n = len(arr)

    for i in range(n - 1):
        # Encontra o índice do menor elemento em arr[i..n-1]
        idx_min = i
        for j in range(i + 1, n):
            if arr[j] < arr[idx_min]:
                idx_min = j

        # Troca o mínimo encontrado com arr[i] (se necessário)
        if idx_min != i:
            arr[i], arr[idx_min] = arr[idx_min], arr[i]

    return arr


# Teste
if __name__ == "__main__":
    dados = [64, 25, 12, 22, 11]
    resultado = selection_sort(dados.copy())
    print(resultado)  # [11, 12, 22, 25, 64]

Pontos importantes do código:

  • O loop externo vai até n-1 porque o último elemento se posiciona automaticamente.
  • idx_min = i: começamos assumindo que o mínimo é o próprio elemento i, depois verificamos se existe algo menor à direita.
  • A verificação if idx_min != i evita uma troca desnecessária quando o mínimo já está na posição correta.
  • A troca simultânea arr[i], arr[idx_min] = arr[idx_min], arr[i] é a única escrita no array por iteração o que torna o Selection Sort ideal quando escritas são caras.

Análise de Complexidade

CasoComparaçõesTrocasEspaço
MelhorO(n²)O(n)O(1)
MédioO(n²)O(n)O(1)
PiorO(n²)O(n)O(1)

Por que sempre O(n²)? O loop externo roda n-1 vezes. O loop interno sempre percorre a parte não-ordenada inteira para encontrar o mínimo (n-1) + (n-2) + ... + 1 = n(n-1)/2 comparações. Não há saída antecipada possível: mesmo que o array já esteja ordenado, o algoritmo precisa confirmar que o elemento em i é de fato o mínimo de tudo à direita.

Estabilidade: Instável. A troca direta pode inverter a ordem relativa de elementos iguais.

Quando Usar

Memória flash / EEPROM

Memórias flash têm um número limitado de ciclos de escrita. O Selection Sort faz no máximo n-1 escritas qualquer outro algoritmo O(n²) faz mais. É a escolha correta quando cada escrita "custa" fisicamente.

Arrays muito pequenos

Para n 20, a baixa constante e a ausência de overhead tornam o Selection Sort competitivo às vezes mais rápido que Quick Sort em prática por não ter chamadas de função recursivas.

Dados quase-ordenados

Diferente do Insertion Sort (O(n) no melhor caso), o Selection Sort é sempre O(n²), mesmo para arrays já ordenados. Não consegue se beneficiar de ordenação prévia.
