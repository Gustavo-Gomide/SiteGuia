Shell Sort: Insertion Sort com saltos

Generaliza o Insertion Sort aplicando-o inicialmente a elementos distantes (gap grande), depois diminuindo o gap progressivamente até 1. Elementos "viajam longas distâncias" rapidamente nas primeiras passagens.

A fraqueza do Insertion Sort é mover elementos lentamente um elemento na posição errada de longe desloca apenas uma posição por vez. O Shell Sort resolve isso: nas primeiras iterações, compara e troca elementos separados por um grande gap, permitindo correções a longa distância em poucas operações. Quando o gap chega a 1, o array está quase-ordenado e o Insertion Sort final corre em tempo quase-linear.

O que é o Shell Sort?

O Shell Sort foi proposto por Donald Shell em 1959 no artigo "A High-Speed Sorting Procedure" publicado no Communications of the ACM. Foi o primeiro algoritmo de ordenação a quebrar a barreira O(n²) sem usar recursão ou memória extra décadas antes do Quick Sort e Merge Sort se popularizarem.

O algoritmo é uma generalização do Insertion Sort: em vez de comparar elementos adjacentes (gap=1), usa um gap (salto) variável. Para cada valor de gap, executa um Insertion Sort "diluído" comparando e trocando apenas elementos distanciados por gap posições. O gap é reduzido progressivamente segundo uma sequência de gaps até chegar a 1.

A sequência de gaps importa muito: A complexidade do Shell Sort depende diretamente da sequência de gaps escolhida. A sequência original de Shell (n/2, n/4, ..., 1) dá O(n²). A sequência de Knuth (1, 4, 13, 40, ...) dá O(n^(3/2)). A sequência de Ciura (empiricamente otimizada) é considerada a melhor para uso prático mas a complexidade exata ainda é um problema em aberto na Ciência da Computação.

Sequência de Knuth: 1, 4, 13, 40, 121...

Gerada pela fórmula h = 3h + 1, começando em h=1. Donald Knuth a analisou em The Art of Computer Programming e mostrou complexidade O(n^(3/2)) significativamente melhor que O(n²).

Sequência de Ciura: 1, 4, 10, 23, 57, 132...

Determinada empiricamente por Marcin Ciura em 2001 por benchmarks extensivos. É considerada a melhor sequência para uso prático mas não tem fórmula fechada conhecida e sua complexidade exata é um problema em aberto.

Implementação em Python

shell_sort.py
def shell_sort(arr: list) -> list:
    n = len(arr)

    # Sequência de Knuth: 1, 4, 13, 40, 121, ...
    gap = 1
    while gap < n // 3:
        gap = gap * 3 + 1

    # Insertion Sort com gap decrescente
    while gap >= 1:
        for i in range(gap, n):
            key = arr[i]
            j   = i - gap
            while j >= 0 and arr[j] > key:
                arr[j + gap] = arr[j]
                j -= gap
            arr[j + gap] = key
        gap //= 3

    return arr


# Teste
if __name__ == "__main__":
    dados = [8, 5, 3, 1, 9, 6, 0, 7]
    resultado = shell_sort(dados.copy())
    print(resultado)  # [0, 1, 3, 5, 6, 7, 8, 9]

Pontos importantes do código:

  • O loop while gap < n // 3: gap = gap * 3 + 1 calcula o maior valor da sequência de Knuth menor que n garantindo que o primeiro gap seja adequado ao tamanho do array.
  • O loop interno é idêntico ao Insertion Sort, exceto que usa gap como passo em vez de 1.
  • j -= gap move gap posições para trás comparando e deslocando apenas elementos separados por gap.
  • Quando gap = 1, a última iteração é um Insertion Sort padrão mas o array já está quase-ordenado, então roda rapidamente.

Análise de Complexidade

Ciura (1, 4, 10, 23, 57...)? (melhor empírico)O(1)

Complexidade em aberto: A complexidade ótima do Shell Sort para uma sequência de gaps arbitrária permanece um problema em aberto. Sabe-se que o limite inferior é Ω(n log²n) para qualquer sequência de gaps, mas a sequência ótima não é conhecida. É um dos poucos problemas fundamentais da área de algoritmos ainda sem solução.

Estabilidade: Instável. Trocas com gap > 1 podem inverter a ordem de elementos iguais.

Quando Usar

Sistemas embarcados

O Shell Sort é in-place, simples de implementar e sem recursão. Em sistemas embarcados com RAM limitada e sem biblioteca padrão, é frequentemente a melhor opção acima de O(n²).

Dados médios (~10k elementos)

Para arrays de poucos milhares de elementos, Shell Sort com sequência de Ciura é competitivo com Quick Sort na prática sem a complexidade de implementação do Quick Sort e sem risco de PIOR caso.

Grandes volumes com estabilidade

Para n > 100k ou quando estabilidade é necessária (ordenação de objetos complexos), prefira Tim Sort ou Merge Sort.
