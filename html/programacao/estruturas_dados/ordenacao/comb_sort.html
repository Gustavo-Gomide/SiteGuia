Comb Sort: Bubble Sort com pente

Evolução do Bubble Sort que usa um "gap" decrescente para comparar elementos distantes antes de refinar com gap=1.

O Bubble Sort compara sempre vizinhos imediatos (gap=1), o que causa o problema das tartarugas — elementos pequenos longe do início demoram muitas passagens para chegar na posição certa. O Comb Sort, proposto por Włodzimierz Dobosiewicz em 1980 e popularizado por Stephen Lacey e Richard Box em 1991, resolve isso iniciando com um gap grande e encolhendo-o por um fator de ~1.3 a cada passagem, até chegar em 1 e finalizar como um Bubble Sort normal.

O que é o Comb Sort?

A ideia é inspirada no Shell Sort: em vez de comparar apenas elementos adjacentes, o Comb Sort compara pares separados por um intervalo (gap) que começa grande e diminui progressivamente. O fator de encolhimento ótimo é aproximadamente 1.3 — valores entre 1.2 e 1.5 funcionam bem, mas 1.3 foi determinado empiricamente como o melhor.

O gap especial de 9 ou 10 deve ser substituído por 11 para evitar um padrão de pior caso — esse detalhe foi identificado por Lacey e Box e é chamado de combinação mágica 11.

Analogia: Imagine um pente com dentes espaçados. Primeiro você passa o pente com dentes largos (gap grande) organizando grosseiramente, depois com dentes cada vez mais juntos, até o pente de dentes adjacentes (gap=1) que finaliza a ordenação.

Como Funciona: Passo a Passo

Para [8, 4, 1, 6, 2, 7, 3, 5] com n=8, o gap começa em 6 (8/1.3≈6), depois 4, 3, 2, 1.

Note como o gap grande elimina rapidamente os grandes deslocamentos. Quando o gap chega em 1, quase nenhuma troca é necessária — diferente do Bubble puro que precisa de muitas passagens com gap=1.

Implementação em Python

comb_sort.py
def comb_sort(arr: list) -> list:
    n = len(arr)
    gap = n
    fator = 1.3
    ordenado = False

    while not ordenado:
        # Encolhe o gap pelo fator ~1.3
        gap = int(gap / fator)
        if gap <= 1:
            gap = 1
            ordenado = True   # após gap=1 sem trocas, termina

        trocou = False
        for i in range(n - gap):
            if arr[i] > arr[i + gap]:
                arr[i], arr[i + gap] = arr[i + gap], arr[i]
                trocou = True

        if trocou:
            ordenado = False  # houve troca: mais uma passagem

    return arr


if __name__ == "__main__":
    dados = [8, 4, 1, 6, 2, 7, 3, 5]
    print(comb_sort(dados))  # [1, 2, 3, 4, 5, 6, 7, 8]
  • O gap começa em n e é dividido por 1.3 a cada passagem.
  • Flag ordenado só será True quando gap=1 e nenhuma troca ocorrer na última passagem.
  • Quando gap cai para 9 ou 10, o algoritmo tende a ter pior desempenho — algumas implementações forçam gap=11 nesse intervalo (regra dos "11").

Quando Usar

Vantagens

Simples como o Bubble Sort mas significativamente mais rápido na prática. Boa escolha quando se quer algo melhor que Bubble sem a complexidade do Quick ou Merge Sort.

Limitações

Complexidade de pior caso ainda é O(n²). Não estável. Para produção, Tim Sort e Introsort são superiores.

Produção: O Comb Sort é uma curiosidade histórica. Em produção, Shell Sort e Tim Sort superam o Comb Sort em praticamente todos os cenários. O fator de redução 1.3 foi determinado empiricamente — não há prova analítica de que seja ótimo.

O Comb Sort demonstra uma ideia fundamental: comparações de longa distância destroem eficientemente as inversões grandes antes de tratar as pequenas — ideia que também está no Shell Sort e no Merge Sort.
