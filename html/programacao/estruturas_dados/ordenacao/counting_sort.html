Counting Sort

Counting Sort: contar, acumular, distribuir

Não compara elementos conta quantas vezes cada valor aparece, calcula posições acumuladas, e distribui os elementos diretamente em suas posições finais. O(n+k) quando o intervalo k é conhecido.

O Counting Sort quebra o limite teórico de O(n log n) para ordenação por comparação. Ele consegue isso porque não compara: ele conta. Para dados com intervalo pequeno e conhecido (como notas de 0 a 100, idades, bytes), o Counting Sort ordena em tempo linear O(n+k). É a base do Radix Sort e componente essencial do Tim Sort em casos especiais.

O que é o Counting Sort?

O Counting Sort é um algoritmo de ordenação não-comparativo ele nunca pergunta "a < b?". Em vez disso, usa o valor dos elementos diretamente como índices de um array auxiliar de contagem. O algoritmo foi descrito por Harold H. Seward em 1954 em sua tese de mestrado no MIT.

O algoritmo tem três fases: (1) contar quantas vezes cada valor aparece; (2) acumular as contagens para calcular as posições iniciais de cada valor no output; (3) distribuir os elementos do input para o output usando as posições calculadas.

Analogia da urna eleitoral: Para contar votos em uma eleição com 5 candidatos, não é preciso comparar votos entre si basta ter 5 caixinhas e jogar cada voto na caixinha correspondente. Ao final, conta-se cada caixinha. O Counting Sort faz exatamente isso.

A fase de acumulação garante estabilidade

Ao iterar o input de trás para frente na fase de distribuição, e decrementar o índice acumulado após cada colocação, a ordem relativa de elementos iguais é mantida tornando o Counting Sort estável.

Limitação: k deve ser pequeno

O array de contagem tem tamanho k (máximo dos valores). Se k >> n, o algoritmo desperdiça memória e tempo. Para notas (0-100), k=101 é ótimo. Para CPFs (9 dígitos), k = 1 bilhão é inviável.

Implementação em Python

counting_sort.py
def counting_sort(arr: list) -> list:
    if not arr:
        return arr

    k     = max(arr)
    count = [0] * (k + 1)

    # Fase 1: Contar ocorrências
    for val in arr:
        count[val] += 1

    # Fase 2: Acumular (prefix sum)  count[i] vira posição inicial em output
    for i in range(1, k + 1):
        count[i] += count[i - 1]

    # Fase 3: Distribuir de trás para frente (garantir estabilidade)
    output = [0] * len(arr)
    for val in reversed(arr):
        count[val] -= 1
        output[count[val]] = val

    return output


# Teste
if __name__ == "__main__":
    dados = [4, 2, 2, 8, 3, 3, 1]
    resultado = counting_sort(dados)
    print(resultado)  # [1, 2, 2, 3, 3, 4, 8]

Pontos importantes do código:

  • O array count tem tamanho k+1 para indexar diretamente pelo valor do elemento.
  • A fase de acumulação (prefix sum) transforma contagens em posições: count[i] passa a indicar quantos elementos são i, ou seja, a posição logo após onde o último elemento de valor i deve ser colocado.
  • Iterar reversed(arr) na distribuição preserva a ordem relativa de iguais garantindo estabilidade.
  • O algoritmo assume valores não-negativos. Para valores negativos ou com offset, é necessário deslocar os índices: min_val = min(arr) e usar count[val - min_val].

Quebrando o limite Ω(n log n): O teorema do limite inferior de comparação diz que qualquer algoritmo que ordena por comparação precisa de no mínimo n log n operações. O Counting Sort não usa comparações usa os valores como índices e por isso contorna esse limite.

Estabilidade: Estável (desde que a fase de distribuição percorra o input de trás para frente).

Quando Usar

k pequeno e conhecido

Notas (0-100), faixas etárias, bytes (0-255), classificações (1-5 estrelas). Qualquer dado com intervalo bem delimitado e pequeno relativo a n é candidato ideal.

Base do Radix Sort

O Radix Sort usa Counting Sort como subrotina para ordenar dígito a dígito. Cada passagem do Radix Sort é um Counting Sort com k=10 (ou k=256 para bytes).

Dados contínuos ou grande intervalo

Números de ponto flutuante, CPFs, timestamps de alta resolução qualquer valor com intervalo na casa de milhões torna o array de contagem inutilizável pela memória consumida.
