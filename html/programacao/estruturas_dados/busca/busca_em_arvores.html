Busca em Árvores Binárias (BST)

Busca em Árvores Binárias

BST: navegação inteligente por propriedade de ordem

Cada comparação elimina metade da árvore O(log n) quando balanceada, mesmo princípio da busca binária mas em estrutura dinâmica.

Uma Árvore Binária de Busca (BST) organiza dados com uma regra simples: para cada nó, todos os valores à esquerda são menores e todos à direita são maiores. Essa propriedade transforma a busca em um processo de decisão binária a cada nível ir à esquerda ou à direita sem precisar verificar metade dos nós.

O que é busca em BST?

Diferente de busca em arrays, a busca em BST navega por uma estrutura de nós ligados. A propriedade de ordem da BST garante que, ao comparar o alvo com o valor do nó atual, sabemos exatamente qual subárvore pode conter o alvo e qual pode ser descartada completamente. O processo termina quando o alvo é encontrado ou quando chegamos a um filho nulo.

Propriedade BST: Para qualquer nó N: todos os valores na subárvore esquerda de N são menores que N, e todos na subárvore direita são maiores. Essa invariante é mantida em inserção e remoção.

Implementação em Python

busca_bst.py
class No:
    def __init__(self, valor):
        self.valor = valor
        self.esq = None
        self.dir = None


def busca_bst(no, alvo):
    """Busca recursiva em BST. Retorna o nó encontrado ou None."""
    # Caso base: nó nulo (não encontrado) ou encontrado
    if no is None:
        return None
    if no.valor == alvo:
        return no

    # Propriedade BST: decide qual subárvore explorar
    if alvo < no.valor:
        return busca_bst(no.esq, alvo)   # vai à esquerda
    else:
        return busca_bst(no.dir, alvo)   # vai à direita


# --- Montagem manual da BST do exemplo ---
raiz = No(50)
raiz.esq = No(25); raiz.dir = No(75)
raiz.esq.esq = No(10); raiz.esq.dir = No(30)
raiz.dir.esq = No(60); raiz.dir.dir = No(90)

resultado = busca_bst(raiz, 60)
print(resultado.valor if resultado else "Não encontrado")  # 60

Pontos importantes:

  • A cada chamada recursiva, o problema é reduzido à metade da árvore restante análogo à busca binária.
  • Retornar o nó (não só um booleano) permite ao chamador acessar dados adicionais armazenados no nó.
  • Para evitar recursão em árvores muito grandes, a versão iterativa usa um loop while no: avançando no = no.esq ou no = no.dir.
  • A eficiência depende criticamente do balanceamento da árvore.

Quando usar

Dados dinâmicos e ordenados

Quando há frequentes inserções e remoções junto com buscas, BSTs AutoBalanceadas (AVL, Red-Black) oferecem O(log n) garantido para todas as operações.

Predicados de intervalo

"Todos os valores entre X e Y" uma travessia in-order da BST retorna os elementos em ordem, facilitando consultas de intervalo.

Dicionários ordenados

Bancos de dados relacionais usam B-Trees (generalizações de BST) como estrutura de índice primária para buscas e varreduras de intervalo.

Atenção: Inserir elementos já ordenados (ex: 1, 2, 3, 4, 5) em uma BST simples cria uma árvore degenerada equivalente a uma lista ligada com busca O(n). Use árvores autobalanceadas (AVL, Red-Black Tree) ou heaps para garantir O(log n) independentemente da ordem de inserção.
