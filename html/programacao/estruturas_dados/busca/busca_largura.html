Busca em Largura (BFS)

BFS: expansão em ondas concêntricas

Explora camada por camada a partir da origem e garante o menor número de saltos para atingir qualquer nó.

A Busca em Largura (Breadth-First Search) visita primeiro todos os vizinhos diretos de um nó antes de avançar para os vizinhos dos vizinhos. Essa estratégia de "ondas" garante uma propriedade fundamental: quando o alvo é alcançado pela primeira vez, o caminho percorrido é o mais curto possível em número de arestas.

O que é BFS?

BFS usa uma fila (Queue) estrutura FIFO (First In, First Out) para controlar a ordem de visita. Um nó é adicionado à fila quando é descoberto e removido quando é processado. Um conjunto visitados evita que o mesmo nó seja processado mais de uma vez, protegendo contra ciclos. O resultado é uma travessia por distância crescente a partir do nó inicial.

Analogia: Imagine jogar uma pedra em um lago calmo. As ondas se expandem em círculos concêntricos o anel mais próximo da pedra representa os vizinhos imediatos, o segundo anel os vizinhos dos vizinhos, e assim por diante. O BFS percorre o grafo exatamente assim.

Implementação em Python

bfs.py
from collections import deque

def bfs(grafo, inicio):
    """Percorre o grafo em largura a partir de 'inicio'."""
    visitados = set()
    fila = deque([inicio])     # deque é eficiente para popleft O(1)
    visitados.add(inicio)
    ordem = []

    while fila:
        no = fila.popleft()    # FIFO: remove do início
        ordem.append(no)

        for vizinho in grafo[no]:
            if vizinho not in visitados:
                visitados.add(vizinho)
                fila.append(vizinho)

    return ordem


# --- Exemplo de uso ---
grafo = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [], 'E': [], 'F': []
}

print(bfs(grafo, 'A'))  # ['A', 'B', 'C', 'D', 'E', 'F']

Pontos importantes:

  • deque do módulo collections é preferido a list porque popleft() é O(1); em uma lista comum seria O(n).
  • O nó inicial é marcado como visitado antes de entrar no loop, garantindo que não seja enfileirado novamente.
  • Marcar como visitado no momento de enfileirar (não de processar) evita duplicatas na fila.
  • Para encontrar o caminho (não só a ordem), mantenha um dicionário pai e reconstrua de volta do alvo até a origem.

Quando usar

Caminho mais curto (não ponderado)

BFS garante que o primeiro caminho encontrado até o alvo é o de menor número de arestas. Usado em redes sociais (menor grau de separação) e jogos em grade.

Nível / distância em grafos

Para encontrar todos os nós a exatamente k saltos da origem útil em análise de redes e propagação de informação.

Verificação de conectividade

Determinar se dois nós estão no mesmo componente conexo ou se o grafo inteiro é conectado.

Atenção: BFS assume que todas as arestas têm o mesmo custo (grafos não ponderados). Em grafos com pesos diferentes, BFS pode encontrar um caminho com menos saltos mas custo total muito maior. Para grafos ponderados, use Dijkstra ou A*.
