Busca Linear: varredura elemento a elemento

O algoritmo mais simples e universal verifica cada item até encontrar o alvo ou esgotar a lista.

A busca linear (ou busca sequencial) percorre a coleção da esquerda para a direita, comparando cada elemento com o alvo. Não exige ordenação nem estrutura especial qualquer lista funciona. Sua simplicidade tem um custo: no pior caso, é necessário verificar todos os n elementos.

O que é a Busca Linear?

A busca linear é o algoritmo de busca mais intuitivo: examine o primeiro elemento; se for o alvo, retorne sua posição; caso contrário, avance para o próximo. Repita até encontrar o alvo ou chegar ao final da lista. Por não fazer suposições sobre a ordem dos dados, é o único algoritmo de busca aplicável a coleções completamente desordenadas.

Analogia: Imagine procurar suas chaves em uma gaveta bagunçada. Você pega cada item, um por um, verifica se é a chave e coloca de volta. Não há atalho o único jeito é olhar todos os itens até achar ou concluir que não estão ali.

Implementação em Python

busca_linear.py
def busca_linear(arr, alvo):
    """Retorna o índice do alvo ou -1 se não encontrado."""
    for i in range(len(arr)):
        if arr[i] == alvo:
            return i          # encontrado: retorna posição
    return -1              # não encontrado


# --- Exemplo de uso ---
numeros = [10, 50, 42, 15]
resultado = busca_linear(numeros, 42)

if resultado != -1:
    print(f"Encontrado no índice {resultado}")  # Encontrado no índice 2
else:
    print("Não encontrado")

Pontos importantes:

  • A função itera com range(len(arr)), garantindo acesso a cada índice.
  • Retorna o índice (não o valor) para que o chamador saiba onde o elemento está.
  • O valor sentinela -1 é a convenção padrão em Python para "não encontrado".
  • Funciona com qualquer tipo comparável números, strings, objetos com __eq__.

Quando usar

Lista pequena ou não ordenada

Quando a coleção tem poucos elementos ou não está ordenada, a busca linear é a escolha mais simples e adequada.

Busca única em coleção temporária

Se a busca ocorre apenas uma vez, o custo de ordenar para usar busca binária é maior que percorrer linearmente.

Tipos sem ordem definida

Buscar por critérios complexos (ex: primeiro objeto com ativo=True) onde não existe chave de ordenação natural.

Atenção: Em listas grandes (n > 1000) com buscas frequentes, a busca linear pode se tornar um gargalo de desempenho. Considere ordenar o array e usar busca binária, ou utilizar uma estrutura como set ou dicionário para buscas O(1).
