DFS em Grafos: a pergunta que evita o loop infinito

Ao contrário de árvores, grafos têm ciclos. O DFS em grafos responde a cada passo: "Eu já estive aqui?"

Aplicar DFS em grafos introduz um desafio que não existe em árvores: ciclos. Sem controle, o algoritmo percorreria o mesmo ciclo para sempre. A solução é um conjunto visitados um "muro" que impede revisitar nós já processados. Com ele, DFS em grafos se torna uma ferramenta poderosa para detecção de ciclos, componentes conexos e ordenação topológica.

O "muro" do visitado

Em grafos, uma aresta de saída pode apontar para um nó que já foi visitado anteriormente no caminho atual (criando um ciclo) ou em outro ramo da busca (um "atalho" que já foi explorado). Em ambos os casos, o conjunto visitados detecta isso e o algoritmo simplesmente ignora o vizinho já visto.

Regra: Antes de visitar qualquer nó, verifique: está em visitados? Se sim, retorne. Se não, adicione-o, processe e explore os vizinhos. Essa verificação transforma DFS de potencialmente infinito para O(V + E).

Implementação em Python

dfs_grafos.py
def dfs_grafo(grafo, inicio):
    """DFS iterativo em grafo com detecção de ciclos via visitados."""
    visitados = set()
    pilha = [inicio]
    ordem = []

    while pilha:
        no = pilha.pop()
        if no in visitados:
            continue            # já processado  ignora
        visitados.add(no)
        ordem.append(no)

        for vizinho in reversed(grafo[no]):
            if vizinho not in visitados:
                pilha.append(vizinho)

    return ordem


def detectar_ciclo(grafo):
    """Detecta se um grafo dirigido tem ciclo usando DFS.
    Usa dois conjuntos: visitados (todos) e na_pilha (caminho atual)."""
    visitados = set()
    na_pilha = set()

    def _dfs(no):
        visitados.add(no)
        na_pilha.add(no)
        for vizinho in grafo.get(no, []):
            if vizinho not in visitados:
                if _dfs(vizinho):
                    return True
            elif vizinho in na_pilha:
                return True   # aresta de retorno = ciclo
        na_pilha.remove(no)
        return False

    return any(_dfs(v) for v in grafo if v not in visitados)


# --- Exemplos ---
grafo_com_ciclo = {'A': ['B'], 'B': ['C'], 'C': ['A']}
grafo_sem_ciclo  = {'A': ['B'], 'B': ['C'], 'C': []}
print(detectar_ciclo(grafo_com_ciclo))  # True
print(detectar_ciclo(grafo_sem_ciclo))   # False

Pontos importantes:

  • A detecção de ciclos em grafos dirigidos requer dois conjuntos: visitados (todos os nós já processados) e na_pilha (nós no caminho atual desde a origem).
  • Uma "aresta de retorno" que aponta para um nó que ainda está na pilha de recursão confirma um ciclo.
  • Em grafos não dirigidos, qualquer aresta para um nó já visitado (exceto o pai imediato) indica ciclo.
  • DFS completo requer iniciar de cada nó não visitado para cobrir grafos desconexos (componentes separados).

Espaço (recursivo)O(V)Pilha de chamadas cresce com a profundidade do caminho mais longo.

Quando usar

Detecção de ciclos

Em grafo dirigido: aresta de retorno (back edge) indica ciclo. Essencial em compiladores para detectar dependências circulares.

Componentes conexos

Executar DFS de cada nó não visitado identifica "ilhas" separadas no grafo útil em análise de redes e clustering.

Ordenação topológica

Adicionar nós ao resultado após explorar todos os vizinhos (pós-ordem) produz uma ordenação topológica válida de um DAG.

Atenção: DFS em grafos com ciclos sem o conjunto visitados causa loop infinito imediato. Ao contrário de árvores, onde a estrutura hierárquica evita naturalmente revisitas, grafos exigem esse controle explícito.
