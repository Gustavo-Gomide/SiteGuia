Algoritmo de Dijkstra

Dijkstra: o GPS do mundo real

Encontra o caminho de menor custo total em grafos ponderados a base de roteamento de redes e navegadores GPS.

O Algoritmo de Dijkstra resolve o problema do caminho mais curto em grafos com pesos não negativos. Ao contrário da BFS (que conta saltos), Dijkstra acumula o custo das arestas. A cada passo, ele é "ganancioso" sempre processa o nó não visitado com menor custo acumulado, garantindo que, quando um nó é visitado, seu custo já é definitivamente o mínimo.

O conceito de relaxamento

Dijkstra começa assumindo que todos os destinos estão a distância infinita. Conforme explora o grafo, descobre atalhos e "relaxa" (atualiza para menor) as distâncias conhecidas. O processo de relaxamento é: se o custo de chegar ao nó V passando pelo nó U é menor do que o custo atual registrado para V, atualize.

Relaxamento: Para cada vizinho V de U: se dist[U] + peso(U,V) < dist[V], então dist[V] = dist[U] + peso(U,V). Essa simples verificação, repetida sistematicamente, garante as distâncias mínimas.

Implementação em Python

dijkstra.py
import heapq

def dijkstra(grafo, inicio):
    """Retorna dict com menor distância de 'inicio' até cada nó.
    grafo = { no: [(vizinho, peso), ...], ... }"""
    dist = {no: float('inf') for no in grafo}
    dist[inicio] = 0
    heap = [(0, inicio)]    # (custo_acumulado, nó)

    while heap:
        custo, u = heapq.heappop(heap)   # menor custo primeiro

        if custo > dist[u]:
            continue                 # versão desatualizada, pula

        for v, peso in grafo[u]:
            novo_custo = dist[u] + peso
            if novo_custo < dist[v]:
                dist[v] = novo_custo     # relaxamento
                heapq.heappush(heap, (novo_custo, v))

    return dist


# --- Exemplo de uso ---
grafo = {
    'A': [('B', 10), ('C', 2)],
    'B': [('D', 1)],
    'C': [('B', 3), ('D', 8)],
    'D': []
}
print(dijkstra(grafo, 'A'))
# {'A': 0, 'B': 5, 'C': 2, 'D': 6}

Pontos importantes:

  • heapq do Python implementa um min-heap sempre retira o elemento com menor valor primeiro, servindo como fila de prioridade.
  • A verificação if custo > dist[u]: continue descarta entradas antigas na fila que foram substituídas por caminhos melhores técnica "lazy deletion".
  • Inicializar todas as distâncias com float('inf') garante que qualquer caminho real seja menor e cause relaxamento.
  • Dijkstra não funciona com pesos negativos use Bellman-Ford nesses casos.

Quando usar

Roteamento em redes

O protocolo OSPF (Open Shortest Path First) usado na internet usa Dijkstra para calcular rotas de menor custo entre roteadores.

GPS e navegação

Base dos algoritmos de navegação GPS encontrar o caminho de menor distância (ou tempo) entre dois pontos em um mapa ponderado.

Jogos e IA

Pathfinding de NPCs em jogos com terrenos de custo variável (estrada vs floresta vs montanha) cada tipo de terreno tem um peso diferente.

Atenção: Dijkstra falha com pesos negativos. A suposição central do algoritmo é que adicionar uma aresta nunca diminui o custo total do caminho. Com pesos negativos, essa propriedade é violada e Dijkstra pode declarar um caminho ótimo prematuro antes de descobrir um atalho "negativo". Use Bellman-Ford para grafos com pesos negativos.
