Algoritmo A* (A-Star)

Algoritmo A*

A*: Dijkstra com bússola

O algoritmo que move personagens em jogos combina custo real com uma estimativa inteligente para priorizar os melhores caminhos.

O A* (pronunciado "A estrela") é a evolução inteligente do Dijkstra. Enquanto Dijkstra explora em todas as direções igualmente até tropeçar no alvo, o A* usa uma heurística uma estimativa da distância até o objetivo para priorizar caminhos que parecem mais promissores. Com uma boa heurística, A* examina muito menos nós que Dijkstra e ainda garante o caminho ótimo.

A fórmula central

Para cada nó candidato, A* calcula um escore total f(n) que combina o custo já pago e o custo estimado ainda a pagar:

f(n) = g(n) + h(n) onde g(n) é o custo real do caminho da origem até n, e h(n) é a estimativa (heurística) de n até o destino. A* sempre expande o nó com menor f(n).

A qualidade da heurística h(n) determina a eficiência do A*:

  • h(n) = 0: sem heurística A* se torna idêntico ao Dijkstra.
  • h(n) custo real: heurística admissível A* garante caminho ótimo.
  • h(n) > custo real: heurística inadmissível A* é mais rápido mas pode não ser ótimo.

Implementação em Python

a_estrela.py
import heapq

def a_estrela(grafo, inicio, alvo, heuristica):
    """A* com heurística admissível. Retorna caminho ótimo.
    grafo = { no: [(vizinho, peso), ...] }
    heuristica = função h(no, alvo) -> float"""
    # g: custo real da origem até o nó
    g = {inicio: 0}
    # f = g + h
    f_inicio = heuristica(inicio, alvo)
    heap = [(f_inicio, inicio)]
    pai = {inicio: None}

    while heap:
        _, atual = heapq.heappop(heap)

        if atual == alvo:
            return _reconstruir(pai, inicio, alvo)

        for vizinho, peso in grafo.get(atual, []):
            g_tent = g[atual] + peso
            if g_tent < g.get(vizinho, float('inf')):
                g[vizinho] = g_tent
                f = g_tent + heuristica(vizinho, alvo)
                heapq.heappush(heap, (f, vizinho))
                pai[vizinho] = atual

    return []   # caminho não existe


def _reconstruir(pai, inicio, alvo):
    caminho, atual = [], alvo
    while atual is not None:
        caminho.append(atual); atual = pai[atual]
    return caminho[::-1]


# Heurística Manhattan para grade 2D
def manhattan(a, b):
    return abs(a[0] - b[0]) + abs(a[1] - b[1])

Pontos importantes:

  • A heurística deve ser admissível (nunca superestima o custo real) para que A* garanta o caminho ótimo.
  • g.get(vizinho, float('inf')) trata nós não vistos como tendo distância infinita.
  • O heap contém tuplas (f, nó) Python ordena pelo primeiro elemento (f), garantindo que o nó de menor f saia primeiro.
  • Para mapas com obstáculos, adicione verificação de células bloqueadas antes de enfileirar vizinhos.

Heurísticas comuns

NomeFórmulaQuando usar
Distância Manhattan |x1-x2| + |y1-y2| Grade com movimentos em 4 direções (cima, baixo, esq, dir).
Distância Euclidiana ((x1-x2)² + (y1-y2)²) Espaço contínuo com movimento em qualquer ângulo.
Distância Chebyshev max(|x1-x2|, |y1-y2|) Grade com movimentos em 8 direções (inclui diagonais).
Zero (sem heurística) h(n) = 0 Equivalente ao Dijkstra sem direcionamento.

Quando usar

Pathfinding em jogos

NPCs em jogos 2D e 3D A* é o algoritmo de pathfinding mais usado na indústria de jogos, especialmente em grids com obstáculos.

Navegação com mapa

Quando há informação sobre a posição geográfica dos nós, a distância direta ao destino é uma heurística admissível e muito eficiente.

Robótica e autonomia

Planejamento de trajetória para robôs A* encontra o caminho de menor custo evitando obstáculos em espaços de configuração.

Atenção: Uma heurística inadmissível (que superestima o custo) torna A* mais rápido mas perde a garantia de optimalidade pode retornar um caminho que parece bom mas não é o melhor. Sempre verifique se sua heurística é admissível antes de confiar nos resultados como ótimos.
