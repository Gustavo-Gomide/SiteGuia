BFS em Grafos: garantindo o caminho de menos saltos

A BFS aplicada a grafos com ciclos garante o caminho mais curto mas só em grafos não ponderados.

Enquanto a BFS básica visita nós em ordem de distância crescente, a aplicação em grafos com ciclos exige o conjunto visitados para evitar revisitas. A propriedade mágica permanece: o primeiro momento em que você alcança qualquer nó, o caminho percorrido é o de menor número de saltos a partir da origem.

Expansão em camadas

BFS processa o grafo em "anéis" de distância a partir da origem. Todos os nós a distância 1 são visitados antes dos de distância 2, que por sua vez precedem os de distância 3. Essa ordem de visita é a chave para garantir que, ao encontrar o alvo, o caminho acumulado é o mais curto.

Propriedade fundamental: Quando um nó é retirado da fila BFS pela primeira vez, a distância registrada até ele é definitiva não existe caminho mais curto que aquele. Isso distingue BFS do DFS, que pode chegar ao mesmo nó por caminhos mais longos primeiro.

Implementação com rastreamento de caminho

bfs_caminho.py
from collections import deque

def bfs_caminho(grafo, inicio, alvo):
    """Retorna o caminho mais curto (em saltos) de inicio até alvo."""
    visitados = {inicio}
    fila = deque([inicio])
    pai = {inicio: None}   # rastreia de onde viemos

    while fila:
        no = fila.popleft()

        if no == alvo:
            return _reconstruir(pai, inicio, alvo)

        for vizinho in grafo[no]:
            if vizinho not in visitados:
                visitados.add(vizinho)
                pai[vizinho] = no
                fila.append(vizinho)

    return []   # caminho não existe


def _reconstruir(pai, inicio, alvo):
    """Reconstrói o caminho do alvo até o início e inverte."""
    caminho = []
    atual = alvo
    while atual is not None:
        caminho.append(atual)
        atual = pai[atual]
    caminho.reverse()
    return caminho


# --- Exemplo de uso ---
grafo = {
    'I': ['A', 'B', 'C'],
    'A': ['D', 'E'],
    'B': [],
    'C': ['T'],
    'D': [], 'E': [], 'T': []
}
print(bfs_caminho(grafo, 'I', 'T'))  # ['I', 'C', 'T']

Pontos importantes:

  • O dicionário pai registra "de onde viemos" para cada nó permite reconstruir o caminho ao final.
  • Marcar como visitado no momento de enfileirar (não de processar) evita que o mesmo nó seja enfileirado múltiplas vezes.
  • A reconstrução percorre o dicionário pai do alvo até a origem e inverte o resultado.
  • BFS não funciona com grafos ponderados para esses, use Dijkstra.

Quando usar

Menor caminho não ponderado

GPS simplificado (sem distâncias), graus de separação em redes sociais, menor número de movimentos em puzzles de grade.

Verificação de alcançabilidade

Determinar se existe qualquer caminho entre dois nós, ou quais nós são alcançáveis a partir de uma origem.

Nível em grafos

Calcular a distância em saltos de todos os nós a partir de uma origem útil em análise de difusão e epidemiologia.

Limitação crucial: BFS assume que 1 salto = 1 custo. Em um grafo onde uma aresta custa 1 e outra custa 1000, BFS pode escolher o caminho com 2 saltos e custo 1000 em vez do caminho com 3 saltos e custo 3. Para grafos com pesos, utilize Dijkstra.
