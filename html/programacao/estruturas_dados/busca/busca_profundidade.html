Busca em Profundidade (DFS)

DFS: mergulhar fundo antes de voltar

Explora um caminho até o fim antes de recuar e tentar a próxima alternativa a base do backtracking.

A Busca em Profundidade (Depth-First Search) segue um caminho tão longe quanto possível antes de retroceder. Usa uma pilha (implícita na recursão ou explícita com uma lista) para lembrar os pontos de retorno. Não garante o caminho mais curto, mas é essencial para explorar estruturas completas, detectar ciclos e resolver problemas de ordenação topológica.

O que é DFS?

DFS parte de um nó inicial e avança sempre para o primeiro vizinho não visitado, repetindo o processo recursivamente. Quando chega a um nó sem vizinhos não visitados (beco sem saída), retrocede (backtrack) para o nó anterior e tenta o próximo vizinho disponível. Continua até ter visitado todos os nós alcançáveis.

Analogia: Explorar um labirinto seguindo sempre a parede da esquerda até não conseguir avançar, então voltando ao último cruzamento e tentando outro corredor. DFS é exatamente esse método sistemático de exploração.

Implementação em Python

dfs.py
# Versão recursiva
def dfs_recursivo(grafo, no, visitados=None):
    if visitados is None:
        visitados = set()
    visitados.add(no)
    print(no, end=" ")
    for vizinho in grafo[no]:
        if vizinho not in visitados:
            dfs_recursivo(grafo, vizinho, visitados)
    return visitados


# Versão iterativa (pilha explícita)
def dfs_iterativo(grafo, inicio):
    visitados = set()
    pilha = [inicio]
    ordem = []

    while pilha:
        no = pilha.pop()        # LIFO: remove do topo
        if no not in visitados:
            visitados.add(no)
            ordem.append(no)
            for vizinho in reversed(grafo[no]):
                if vizinho not in visitados:
                    pilha.append(vizinho)

    return ordem


# --- Exemplo de uso ---
grafo = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [], 'E': [], 'F': []
}
print(dfs_iterativo(grafo, 'A'))  # ['A', 'B', 'D', 'E', 'C', 'F']

Pontos importantes:

  • A versão recursiva é mais legível mas pode atingir o limite de recursão do Python em grafos muito profundos (sys.setrecursionlimit).
  • Na versão iterativa, reversed(grafo[no]) garante que os vizinhos sejam visitados na mesma ordem que na versão recursiva (o primeiro vizinho fica no topo da pilha).
  • O conjunto visitados é essencial em grafos com ciclos; sem ele, o algoritmo entraria em loop infinito.
  • DFS não garante o caminho mais curto apenas que todos os nós alcançáveis são visitados.

Atenção: Em grafos com ciclos, omitir o conjunto visitados causa loops infinitos. Em grafos muito profundos, a versão recursiva pode exceder o limite de profundidade de recursão do Python (padrão: 1000 chamadas).
