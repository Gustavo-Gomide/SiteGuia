Filas: a estrutura FIFO

Uma estrutura linear onde o primeiro a entrar é o primeiro a sair — assim como uma fila de banco.

Filas controlam o fluxo quando há mais tarefas do que capacidade de processamento. Estão por toda parte: em servidores web, impressoras, mensageiros e algoritmos de busca em grafos. Entender como funcionam é entender como sistemas gerenciam concorrência e ordem.

O que é uma Fila?

Uma fila é uma estrutura de dados linear que segue o princípio FIFO — o primeiro elemento inserido é o primeiro a ser removido. A inserção acontece sempre no final da fila e a remoção sempre no início.

Pense em uma fila de banco. O primeiro cliente a chegar é o primeiro a ser atendido. Novos clientes se posicionam ao final. Ninguém pula na frente — a ordem chegada determina a ordem de atendimento.

Segundo Cormen et al. (2022), a fila implementa uma política first-in, first-out: o elemento deletado é sempre aquele que já estava no conjunto por mais tempo.

Filas diferem de pilhas exatamente na direção do fluxo: na pilha, entrada e saída são no mesmo lado (topo); na fila, entrada é no final e saída é no início.

Como Funciona

O diagrama abaixo ilustra o fluxo FIFO: novos elementos entram pelo lado direito (enqueue) e saem pelo lado esquerdo (dequeue).

Enqueue — Enfileirar

Adiciona um elemento na parte de trás da fila. Se a fila tiver capacidade máxima e estiver cheia, ocorre Overflow.

Dequeue — Desenfileirar

Remove e retorna o elemento da frente. Se a fila estiver vazia, ocorre Underflow.

Frente / Peek

Lê o próximo elemento a ser atendido sem removê-lo. Útil para inspecionar o estado da fila sem altera-la.

Implementação

Filas podem ser construídas sobre arrays ou listas encadeadas. Com lista encadeada nenhum elemento precisa ser deslocado: o enqueue adiciona ao fim e o dequeue remove do início, ambos com ponteiro direto.

fila.pseudo
CLASSE Fila {
    lista = []   // array ou lista encadeada

    FUNCAO enqueue(item) {
        lista.adicionarNoFim(item)   // O(1)
    }

    FUNCAO dequeue() {
        SE (lista.vazia()) RETORNE Erro
        RETORNE lista.removerDoInicio()   // O(1)
    }

    FUNCAO frente() {
        SE (lista.vazia()) RETORNE Erro
        RETORNE lista[0]
    }

    FUNCAO vazia() {
        RETORNE lista.tamanho() == 0
    }
}

Em arrays com tamanho fixo, o dequeue desloca todos os elementos restantes — custo O(n). Use o padrão de fila circular (dois índices: frente e trás) para manter O(1) mesmo em arrays estáticos.

Casos de Uso

A ordem FIFO é a política justa e previsível por excelência. Qualquer sistema que precise garantir que o mais antigo seja processado primeiro usa uma fila.

Spool de impressão

Quando várias pessoas enviam documentos para a mesma impressora, os trabalhos são enfileirados. O primeiro enviado é o primeiro impresso.

Servidores web

Requisições HTTP que chegam mais rápido do que o servidor consegue processar são colocadas em fila aguardando atendimento.

Sistemática de mensagens

Mensageiros como RabbitMQ e Kafka organizam mensagens em filas. Consumidores processam na mesma ordem em que os produtores enviaram.

Algoritmo BFS

A busca em largura em grafos usa uma fila para visitar nós camada por camada — primeiro os vizinhos imediatos, depois os vizinhos dos vizinhos.

Escalonamento de processos

Sistemas operacionais colocam processos prontos em uma fila. O escalonador retira o primeiro, executa por uma fatia de tempo e, se necessário, reinfileira ao final.

Buffer de transmissão

Em redes e streams de áudio/vídeo, pacotes ou quadros são armazenados em fila para garantir que sejam processados na ordem correta de chegada.

Complexidade das Operações

Com lista encadeada todas as operações de borda são O(1). A busca por valor é O(n) porque viola a semântica da fila.

Operação Tempo Motivo
enqueue O(1) Adiciona ao fim com ponteiro direto.
dequeue O(1) Remove do início com ponteiro direto.
frente O(1) Lê o primeiro elemento sem deslocar nada.
Busca por valor O(n) Exige percorrer a fila inteira.
Espaço O(n) Proporcional ao número de elementos armazenados.
