Deques: a fila de duas pontas

A estrutura linear mais flexível — permite inserir e remover elementos em qualquer uma das extremidades.

O deque (“déck”, do nome Fila Duplamente Terminada) é uma generalização de pilhas e filas. Ele absorve os dois padrões — LIFO e FIFO — em uma única estrutura, e abre casos de uso impossíveis com qualquer das duas isoladamente.

O que é um Deque?

Um deque (Fila Duplamente Terminada) é uma estrutura de dados linear que permite inserções e remoções tanto no início quanto no final, sem restrições de direção. Não há uma regra LIFO nem FIFO obrigatória — a aplicação escolhe qual extremidade usar em cada operação.

Pense em um baralho na mão. Você pode comprar uma carta do topo, do fundo, e descartar em qualquer uma das duas pontas. Nenhuma posição é privilegiada — as duas extremidades são igualmente acessíveis com custo constante.

Segundo Goodrich e Tamassia (2013), um deque é uma coleção de elementos lineares onde inserções e remoções podem acontecer em ambas as extremidades da sequência.

Um deque pode simular uma pilha (usando apenas as operações de uma ponta) ou uma fila (inserindo em uma ponta e removendo da outra). Ele é mais genérico que as duas.

Como Funciona

O diagrama abaixo mostra as quatro operações possíveis: inserção e remoção em cada uma das duas extremidades.

Inserir na frente

Adiciona um elemento no início. O novo elemento passa a ser a primeira posição do deque.

Remover da frente

Remove e retorna o primeiro elemento. O próximo torna-se o novo início.

Inserir no fundo

Adiciona um elemento ao final. O novo elemento torna-se a última posição do deque.

Remover do fundo

Remove e retorna o último elemento. O anterior torna-se o novo final.

Implementação

O deque é geralmente implementado com uma lista duplamente encadeada: cada nó tem ponteiro para o próximo e para o anterior, tornando a inserção e remoção em ambas as pontas O(1) sem ambiguíade.

deque.pseudo
CLASSE Deque {
    lista = ListaDuplamenteEncadeada()

    FUNCAO inserirFrente(item) {
        lista.inserirNoInicio(item)   // O(1)
    }

    FUNCAO inserirFundo(item) {
        lista.inserirNoFim(item)   // O(1)
    }

    FUNCAO removerFrente() {
        SE (lista.vazia()) RETORNE Erro
        RETORNE lista.removerDoInicio()   // O(1)
    }

    FUNCAO removerFundo() {
        SE (lista.vazia()) RETORNE Erro
        RETORNE lista.removerDoFim()   // O(1)
    }
}

Em arrays com índice único, inserir na frente custa O(n) pois desloca todos os elementos. Para manter O(1) em arrays, use o padrão de buffer circular com dois índices (frente e fundo) e aritmética modular.

Casos de Uso

A liberdade nas duas pontas torna o deque ideal para problemas onde a ordem de processo pode mudar dinamicamente.

Verificação de palíndromo

Insira os caracteres da palavra no deque e compare removendo simultaneamente da frente e do fundo. Se todos os pares corresponderem, é um palíndromo.

Histórico com limite

Editores que limitam o desfazer a N ações: nova ação entra no fundo; se o limite for atingido, a ação mais antiga sai da frente.

Roubo de tarefas

Em sistemas multiúcleos, um núcleo ocioso pode roubar tarefas do fundo do deque de outro núcleo enquanto o dono processa da frente — minimizando travações.

Máximo deslizante

Algoritmo clássico em janelas de dados: usa um deque para manter os índices dos candidatos a máximo, removendo entradas obsoletas da frente e as menores do fundo.

Complexidade das Operações

Com lista duplamente encadeada, todas as operações de borda são O(1). A busca por valor no interior viola a semântica da estrutura e custa O(n).

Operação Tempo Motivo
Inserir na frente O(1) Ponteiro direto para o início.
Inserir no fundo O(1) Ponteiro direto para o fim.
Remover da frente O(1) Atualiza o ponteiro de cabeça.
Remover do fundo O(1) Atualiza o ponteiro de cauda.
Busca por valor O(n) Exige varredura linear.
Espaço O(n) Proporcional ao número de elementos.
