Conjuntos (Sets)

Conjuntos — Unicidade como Regra

A estrutura de dados que implementa a Teoria dos Conjuntos: sem duplicatas, com busca O(1) e operações matemáticas nativas.

Um conjunto (set) é uma coleção de elementos onde cada valor aparece no máximo uma vez. Diferente de um array, a pergunta central não é "qual elemento está na posição X?" mas sim "este elemento pertence ao grupo?". Essa mudança de perspectiva define tudo: a implementação, o custo das operações e o tipo de problema que um set resolve com elegância.

O que é um Conjunto

Na matemática, um conjunto é uma coleção bem definida de objetos distintos — formalizados por Georg Cantor no século XIX. Na computação, um Set é o TAD que implementa essa abstração: garante unicidade dos elementos e expõe operações de pertencimento, inserção e remoção.

Segundo Cormen et al. (Introduction to Algorithms, 2022), conjuntos dinâmicos são a categoria de estrutura sobre a qual operações como insert, delete e search operam em tempo sublinear — desde que a implementação seja escolhida corretamente. Sedgewick e Wayne (Algorithms, 2011) complementam: o set é o melhor ponto de partida quando o problema é de pertencimento, não de ordenação ou mapeamento.

Analogia prática: pense em uma lista de convidados numa festa. Você não se importa com a ordem — apenas se o nome está ou não está na lista. Essa é exatamente a pergunta que um set responde em O(1).

A Regra da Unicidade

Ao tentar inserir um elemento já presente, o conjunto simplesmente ignora a operação — sem erro, sem duplicata. Internamente, antes de inserir, o set verifica se o elemento já existe. Se sim, descarta. Se não, insere.

O código abaixo ilustra o comportamento em Python e JavaScript, as duas linguagens com sets mais usados no dia a dia:

Python

exemplo.py
convidados = {"Ana", "Marcos"}

convidados.add("Bia")   # entra
convidados.add("Ana")   # ignorado

print(convidados)
# {"Ana", "Marcos", "Bia"}

# Pertencimento — O(1)
"Marcos" in convidados   # True
"Joao"   in convidados   # False

JavaScript

exemplo.js
const convidados = new Set(["Ana", "Marcos"]);

convidados.add("Bia");   // entra
convidados.add("Ana");   // ignorado

console.log(convidados.size); // 3

// Pertencimento — O(1)
convidados.has("Marcos"); // true
convidados.has("Joao");   // false

Operações de Teoria dos Conjuntos

O grande diferencial de um set em relação a outras coleções são as operações algébricas herdadas diretamente da matemática. Elas operam sobre dois conjuntos inteiros e retornam um novo conjunto.

União (A ∪ B)

Todos os elementos que pertencem a A, a B ou a ambos. Sem duplicatas no resultado.

Python
A | B
A.union(B)

Interseção (A ∩ B)

Apenas os elementos que existem em A e em B simultaneamente.

Python
A & B
A.intersection(B)

Diferença (A − B)

O que está em A mas não está em B. Útil para "quem cancelou?" ou "o que mudou?".

Python
A - B
A.difference(B)

Diferença Simétrica (A △ B)

O que está em um ou no outro, mas não em ambos. Equivale a XOR lógico entre os grupos.

Python
A ^ B
A.symmetric_difference(B)

Exemplo concreto: análise de mudanças

Um caso clássico: detectar o que foi adicionado ou removido entre duas versões de uma lista.

comparacao.py
ontem = {"Ana", "Marcos", "Bia"}
hoje  = {"Marcos", "Bia", "Carlos"}

removidos    = ontem - hoje       # {"Ana"}
adicionados  = hoje  - ontem      # {"Carlos"}
permaneceram = ontem & hoje       # {"Marcos", "Bia"}

Hash Set vs Tree Set

O TAD "conjunto" pode ser implementado de formas distintas. A escolha define o custo das operações e se os elementos ficam ou não em ordem.

Hash Set

Usa uma tabela hash internamente. Cada elemento é passado por uma função hash que calcula a posição de armazenamento. Resultado: todas as operações principais são O(1) em média.

Desvantagem: não mantém ordem. Iterar o set produz elementos em ordem imprevisível. No pior caso (muitas colisões), o desempenho degrada para O(n).

Python / JavaScript
# Python: set() usa hash
s = {3, 1, 4}
print(s)  # {1, 3, 4} ou outra ordem

// JS: Set usa hash
new Set([3, 1, 4])

Tree Set

Usa uma árvore balanceada (geralmente Red-Black Tree) internamente. Cada operação é O(log n), mais lento que o hash set — mas os elementos ficam sempre ordenados.

Vantagem: permite iterar em ordem, encontrar o menor/maior elemento e fazer range queries. Use quando a ordem importa.

Java
// Java: TreeSet mantém ordem
Set<Integer> ts = new TreeSet<>();
ts.add(3); ts.add(1); ts.add(4);
// Iteração: 1, 3, 4 (sempre)

// HashSet: sem garantia de ordem
Set<Integer> hs = new HashSet<>();

Análise de Complexidade (Big O)

Hash Set vs Tree Set vs Array (comparação)

Operação Hash Set Tree Set Array (sem ordem)
Inserção O(1) médio O(log n) O(n) — deve checar duplicata
Remoção O(1) médio O(log n) O(n) — encontra + desloca
Pertencimento O(1) médio O(log n) O(n) — busca linear
Iteração O(n) O(n) ordenado O(n)
Menor/Maior O(n) O(log n) O(n)
Pior caso (busca) O(n) colisões O(log n) garantido O(n)

Cuidado com o pior caso do Hash Set: se todos os elementos colidirem no mesmo bucket, o desempenho cai para O(n). Isso raramente ocorre com funções hash bem projetadas e dados reais, mas é um vetor de ataque em inputs adversariais. Linguagens modernas mitigam isso com hash randomization.

Quando Usar Conjuntos

Use quando

  • A pergunta é "este elemento está presente?" — pertencimento O(1).
  • Precisa eliminar duplicatas de uma coleção.
  • Vai realizar operações de união, interseção ou diferença entre grupos.
  • Mantém uma lista de IDs já processados (cache de visitados em BFS/DFS, por exemplo).
  • Valida unicidade em tempo real — e-mails cadastrados, usernames, tokens.

Evite quando

  • Precisa de acesso por índice ou posição — use array.
  • Os dados têm chave-valor associados — use hash map (dicionário).
  • Precisa de duplicatas intencionais — use lista ou multiset.
  • A ordem de inserção importa e deve ser preservada — use lista ou LinkedHashSet.
  • O tipo dos elementos não é hashável (mutável, como listas em Python).
