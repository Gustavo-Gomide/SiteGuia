Listas Encadeadas

Listas Encadeadas — Flexibilidade Total de Memória

Nós conectados por ponteiros: inserção e remoção em O(1), sem deslocar nenhum elemento.

Uma lista encadeada é uma sequência de elementos onde cada um guarda, além do seu dado, um ponteiro indicando onde o próximo elemento está na memória. Diferente do array, os elementos não precisam ser contíguos — podem estar espalhados pela RAM. Essa característica elimina o custo de deslocamento em inserções e remoções, mas remove o acesso direto por índice.

O que é uma Lista Encadeada

Enquanto o array é como uma fileira de cadeiras numeradas e fixas no cinema — vizinhos físicos — a lista encadeada é como uma caça ao tesouro: cada pista leva à próxima. Os dados podem estar em qualquer endereço da memória; o que os conecta são os ponteiros.

Segundo Knuth (The Art of Computer Programming, 1968), listas encadeadas são a estrutura fundamental para alocação dinâmica de memória — foram a base dos primeiros alocadores de memória e ainda hoje são o mecanismo interno de filas, pilhas e grafos. Cormen et al. (Introduction to Algorithms, 2022) formalizam: uma lista encadeada é um conjunto de objetos (nós) dispostos em ordem linear, onde a ordem é determinada por ponteiros armazenados em cada nó, não por posição física.

Ponto central: listas encadeadas trocam acesso por índice O(1) por inserção e remoção O(1). Essa troca é vantajosa quando você modifica a estrutura com frequência e raramente precisa acessar elementos por posição.

Anatomia de um Nó

A unidade básica da lista é o nó. Cada nó tem duas partes: o dado (a informação armazenada) e o ponteiro (o endereço de memória do próximo nó). O último nó aponta para null.

O Dado

A informação real armazenada: um número, uma string, um objeto qualquer. O nó não impõe restrições ao tipo do dado.

O Ponteiro

Um endereço de memória que aponta para o próximo nó. O último nó da lista tem ponteiro null — indica o fim da cadeia.

estrutura do nó
// Pseudocódigo — estrutura de um nó
classe No {
    dado    // o valor armazenado
    proximo  // referência para o próximo nó (ou null)
}

// Construindo uma lista: 10 → 25 → 37 → null
no1 = No(10)
no2 = No(25)
no3 = No(37)
no1.proximo = no2
no2.proximo = no3
no3.proximo = null
cabeca = no1  // a lista inteira é acessada pela cabeça

Tipos de Lista Encadeada

A forma como os ponteiros são organizados determina quais operações são possíveis e a que custo. Existem três variantes principais.

Simplesmente Encadeada

Cada nó aponta apenas para o próximo. Percurso somente do início para o fim. Sem como "voltar atrás" sem reiniciar da cabeça.

estrutura
cabeca → [10|•] → [25|•] → [37|null]

Duplamente Encadeada

Cada nó aponta para o próximo e para o anterior. Permite percurso nos dois sentidos e remoção O(1) sem precisar do nó anterior. Custo: dois ponteiros por nó.

estrutura
null ← [10|•|•] ⇄ [25|•|•] ⇄ [37|•|null]

Circular

O último nó aponta de volta para a cabeça em vez de null. Útil para estruturas que precisam percorrer em loop contínuo — escalonadores de processo, buffers circulares.

estrutura
cabeca → [10|•] → [25|•] → [37|•]
                              ↑___________________|

Inserção e Remoção — O(1) na Prática

O grande trunfo da lista encadeada é que inserir ou remover um nó, dado um ponteiro para a posição, exige apenas redirecionar ponteiros. Nenhum elemento precisa ser copiado ou deslocado.

Inserção no início — O(1)

Cria o novo nó, faz ele apontar para a cabeça atual e atualiza a cabeça. Nenhum percurso necessário.

pseudocódigo
novoNo = No(valor)
novoNo.proximo = cabeca
cabeca = novoNo  // O(1)

Inserção no meio — O(n) busca + O(1) inserção

É preciso percorrer a lista até o ponto desejado (O(n)) e, então, redirecionar dois ponteiros (O(1)).

pseudocódigo
atual = cabeca
enquanto atual.proximo ≠ posicao:
    atual = atual.proximo   // O(n)
novoNo.proximo = atual.proximo
atual.proximo = novoNo      // O(1)

Remoção — O(1) dado o anterior

Basta fazer o nó anterior apontar para o nó seguinte ao removido. O nó removido fica sem referência (coletado pelo GC ou liberado manualmente).

pseudocódigo
// anterior aponta para o nó a remover
anterior.proximo = anterior.proximo.proximo
// nó antigo perde toda referência → O(1)

Array versus Lista Encadeada

Não existe "melhor" — existe o mais adequado para cada padrão de uso. Conhecer a diferença permite escolher sem tentar e errar.

Comparativo de complexidade

Operação Array Lista (simples) Lista (dupla)
Acesso por índice O(1) O(n) O(n)
Busca por valor O(n) O(n) O(n)
Inserção no início O(n) O(1) O(1)
Inserção no fim O(1)* O(n)** O(1)***
Inserção no meio O(n) O(n) busca + O(1) O(n) busca + O(1)
Remoção (dado o nó) O(n) O(n) busca anterior O(1)
Uso de memória n × T n × (T + 1 ptr) n × (T + 2 ptrs)
Localidade de cache Excelente Ruim Ruim

* Amortizado. ** O(1) com ponteiro para cauda. *** Com ponteiro para cauda.

Cache miss: em listas encadeadas, cada nó pode estar em qualquer lugar da RAM. O processador não consegue pré-carregar os nós seguintes no cache, gerando um cache miss a cada acesso. Em benchmarks práticos, um array com O(n) busca linear pode ser mais rápido que uma lista com O(1) inserção quando os elementos são pequenos e a memória está quente no cache.

Quando Usar Listas Encadeadas

Use quando

  • Insere e remove elementos com frequência, especialmente no início.
  • O tamanho da coleção varia muito e memória contígua é escassa.
  • Está implementando pilha, fila ou deque com operações de ponta.
  • Precisa de remoção O(1) dado o ponteiro do nó (lista dupla).
  • Está construindo estruturas como grafos ou listas de adjacência.

Evite quando

  • Precisa de acesso por índice com frequência — use array.
  • Performance de cache é crítica — dados compactos e contíguos são superiores.
  • Os elementos são muito pequenos — o overhead do ponteiro pode dobrar o uso de memória.
  • Precisa de busca binária — só funciona em estruturas com acesso direto.
