Pilhas: a estrutura LIFO

Uma estrutura linear onde somente o topo é acessível — o último a entrar é o primeiro a sair.

Pilhas estão em toda parte: no histórico do navegador, no desfazer de editores de texto, na execução de funções no processador. Entender como funcionam é entender como o computador organiza e rastreia a sua própria execução.

O que é uma Pilha?

Uma pilha é uma estrutura de dados linear de acesso restrito: todas as inserções e remoções acontecem exclusivamente por uma extremidade chamada topo. O princípio que governa essa ordem é o LIFO — o último elemento inserido é o primeiro a ser removido.

Pense em uma pilha de pratos na pia. Você coloca novos pratos sempre no topo e, quando precisa de um, pega do topo também. Nunca é possível retirar um prato do meio sem antes remover todos que estão acima dele. A pilha de dados funciona exatamente assim.

Segundo Cormen et al. (2022), uma pilha implementa uma política de acesso last-in, first-out: o elemento removido de um conjunto é o inserido mais recentemente.

Pilhas diferem de arrays e listas exatamente nessa restrição de acesso. Você não lê ou escreve no meio — somente no topo. Isso garante que as operações principais sejam sempre O(1).

Como Funciona

A pilha possui três operações fundamentais. O diagrama abaixo mostra visualmente a dinâmica do topo:

Push — Empilhar

Adiciona um elemento acima do topo atual. O novo elemento passa a ser o topo. Se a pilha tiver capacidade máxima e estiver cheia, ocorre Stack Overflow.

Pop — Desempilhar

Remove e retorna o elemento do topo. O elemento abaixo passa a ser o novo topo. Se a pilha estiver vazia, ocorre Stack Underflow.

Peek — Espiar

Lê o elemento do topo sem removê-lo. Útil para tomar decisões com base no topo sem alterar o estado da pilha.

Implementação

Uma pilha pode ser construída sobre um array ou sobre uma lista encadeada. A versão com array é a mais simples: o topo corresponde ao último índice ocupado.

pilha.pseudo
CLASSE Pilha {
    items = []   // array interno

    FUNCAO push(elemento) {
        items.adicionarNoFim(elemento)
    }

    FUNCAO pop() {
        SE (items.vazia()) RETORNE Erro
        RETORNE items.removerDoFim()
    }

    FUNCAO peek() {
        SE (items.vazia()) RETORNE Erro
        RETORNE items[ultimo_indice]
    }

    FUNCAO vazia() {
        RETORNE items.tamanho() == 0
    }
}

Como todas as operações tocam apenas o fim do array (push adiciona no último índice; pop remove do último), o custo é constante — O(1) — sem necessidade de mover nenhum outro elemento.

Em linguagens com arrays de tamanho fixo (como C), é obrigatório verificar se a pilha está cheia antes de cada push. Use uma lista encadeada quando o tamanho máximo não for conhecido antecipadamente.

Casos de Uso

A restrição LIFO, que parece limitante, é exatamente o que torna a pilha indispensável em situações onde a ordem de reversão importa.

Desfazer (Ctrl+Z)

Editores registram cada ação do usuário em uma pilha. Ao pressionar Ctrl+Z, a última ação é retirada do topo e revertida — um pop com efeito colateral.

Pilha de chamadas (Call Stack)

O processador mantém uma pilha de contextos de função. Quando uma função chama outra, o estado atual é empilhado. Quando retorna, é desempilhado e a execução continua de onde parou.

Histórico do navegador

Cada URL visitada é empilhada. O botão "Voltar" executa um pop e exibe a página anterior.

Verificação de parênteses

Compiladores usam pilhas para checar balanceamento: ao encontrar um símbolo de abertura, empilha; ao encontrar um de fechamento, desempilha e verifica se corresponde.

Avaliação de expressões

Calculadoras e interpretadores usam duas pilhas para avaliar expressões aritméticas — uma para operandos e outra para operadores — seguindo a notação polonesa reversa.

Busca em profundidade (DFS)

A busca em profundidade em grafos e árvores usa uma pilha — explicitamente ou via recursão (que usa a call stack) — para rastrear os nós ainda a visitar.

Complexidade das Operações

Todas as operações que tocam somente o topo são O(1). O único caso de custo linear é buscar um elemento arbitrário — o que viola a semântica da pilha e, na prática, indica uso equivocado da estrutura.

Operação Tempo Motivo
push O(1) Insere no topo — sem percorrer nenhum outro elemento.
pop O(1) Remove do topo — sem deslocar nenhum outro elemento.
peek O(1) Lê o topo diretamente pelo índice — sem percorrer nada.
Busca por valor O(n) Exige desempilhar até encontrar — viola a semântica LIFO.
Espaço O(n) Proporcional ao número de elementos armazenados.
