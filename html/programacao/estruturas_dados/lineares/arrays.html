Arrays — A Estrutura Fundamental

Memória contígua, acesso O(1) por índice e a base de quase todas as outras estruturas de dados.

Um array é uma coleção de elementos armazenados em posições consecutivas de memória. Essa continuidade é o que permite calcular o endereço de qualquer posição em tempo constante O(1), mas também é a razão pela qual inserir ou remover no meio é custoso.

O que é um Array

Um array (ou vetor) é a estrutura de dados mais primitiva e universal da computação. Presente em toda linguagem — de Assembly a Python — ele mapeia um conjunto de elementos para blocos contíguos de memória RAM.

Segundo Knuth (The Art of Computer Programming, 1968), sequências lineares com acesso por índice são a fundação sobre a qual todas as estruturas mais complexas são construídas. Cormen et al. (Introduction to Algorithms, 2022) formalizam: dado um array A de n elementos, o elemento na posição i ocupa o endereço base + i × sizeof(elemento).

Diferença fundamental: um array não procura seus elementos — ele calcula o endereço diretamente. É por isso que o acesso é sempre O(1), independente do tamanho.

Anatomia e Acesso Direto

Cada elemento de um array ocupa exatamente o mesmo espaço em bytes. O índice funciona como um multiplicador de deslocamento a partir do endereço base.

A Matemática do Acesso

O computador não procura o índice — ele calcula o endereço em uma operação.

Array começa em 1000, cada inteiro ocupa 4 bytes. Onde está o índice 3?

cálculo
1000 + (3 × 4) = 1012  →  O(1)

Sintaxe

exemplo
int[] numeros = {15, 7, 42, 9, 33};

// Leitura — O(1)
print(numeros[2]); // 42

// Escrita — O(1)
numeros[0] = 99;

Inserção e Remoção

Arrays são rápidos para leitura, mas custosos para modificar a estrutura. Inserir ou remover no meio exige deslocar todos os elementos subsequentes (shifting).

Inserir no Fim

Se há espaço alocado, apenas grava na próxima posição livre.

O(1)*
[A, B, C, _]  →  append(D)
[A, B, C, D]

Inserir no Meio

Cada elemento após a posição alvo desloca uma posição à direita.

O(n)
[A, B, C]  →  insert(1, X)
[A, X, B, C]

Inserir no Início

Pior caso: 100% dos elementos precisam ser deslocados.

O(n)
[A, B, C]  →  insert(0, X)
[X, A, B, C]

Remover no Meio

Remove e fecha o buraco deslocando os seguintes à esquerda.

O(n)
[A, B, C, D]  →  remove(1)
[A, C, D]

* Em arrays dinâmicos, inserir no fim pode ser O(n) quando cheio e precisa realocar — mas o custo amortizado continua O(1).

Array Estático vs Dinâmico

Há duas variantes fundamentais, dependendo de como o tamanho é gerenciado em memória.

Array Estático

Tamanho fixo definido na criação. Alocado na stack ou heap com tamanho imutável. Sem overhead de gerenciamento.

C / Java
// C — compile-time
int nums[5] = {1, 2, 3, 4, 5};

// Java — runtime fixo
int[] arr = new int[5];

Array Dinâmico

Cresce automaticamente. Quando a capacidade esgota, aloca um bloco maior e copia tudo. Custo de cópia O(n), mas amortizado O(1) por inserção.

Python / Java
# Python — list é dinâmica
nums = []
nums.append(1)  # O(1) amortizado

// Java
List<Integer> arr = new ArrayList<>();

Estratégia de crescimento (doubling)

Arrays dinâmicos geralmente dobram a capacidade quando cheios. O custo de cópia se dilui ao longo de várias inserções:

amortização
cap=4  → cheio → realoca cap=8   (copia 4 elementos)
cap=8  → cheio → realoca cap=16  (copia 8 elementos)
cap=16 → cheio → realoca cap=32  (copia 16 elementos)
─────────────────────────────────────────────────────
Total de cópias para n inserções: 1+2+4+…+n ≈ 2n
→ O(1) amortizado por inserção

Arrays Multidimensionais (Matrizes)

Arrays dentro de arrays. O mais comum é a matriz 2D — uma grade de linhas e colunas.

Visualização

Pense em uma planilha ou tabuleiro: cada célula é acessada por linha e coluna.

exemplo
int[][] matrix = {
  {1, 2, 3}, // Linha 0
  {4, 5, 6}, // Linha 1
  {7, 8, 9}  // Linha 2
};

print(matrix[1][2]); // 6

Onde é usado

  • Processamento de imagens — cada pixel é matrix[y][x].
  • Jogos de tabuleiro e mapas tile-based.
  • Álgebra linear — multiplicação de matrizes, transformações 3D.
  • Machine learning — pesos de redes neurais são tensores (arrays n-D).

Row-major vs Column-major: em C, Python e Java os elementos de uma linha ficam contíguos na memória. Iterar linha a linha é cache-friendly. Em Fortran e MATLAB é column-major — iterar por coluna é mais eficiente nesses casos.

Quando Usar Arrays

Use quando

  • Precisa de acesso por índice O(1) com frequência.
  • Está iterando sequencialmente — cache-friendly.
  • O tamanho é conhecido ou relativamente estável.
  • Está implementando pilha, fila ou heap sobre um array.
  • Performance de localidade de cache é crítica.

Evite quando

  • Insere ou remove no meio com frequência — prefira lista encadeada.
  • O tamanho varia drasticamente e memória é restrita.
  • Precisa buscar por chave — prefira hash map.
  • Os dados têm hierarquia ou relações arbitrárias — prefira árvore ou grafo.
