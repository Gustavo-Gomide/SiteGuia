Dicionários: a estrutura chave-valor

Mapeamento associativo que permite recuperar qualquer valor em tempo constante a partir de uma chave única.

Dicionários estão entre as estruturas mais usadas na programação. Eles aparecem como dict em Python, HashMap em Java, object em JavaScript. Entender como funcionam por dentro — a função hash, colisões e as garantias de desempenho — é essencial para usá-los corretamente.

O que é um Dicionário?

Um dicionário (também chamado de mapa ou tabela hash) é uma estrutura de dados que armazena pares chave → valor. Dado uma chave, a estrutura retorna o valor associado em tempo médio O(1) — independentemente de quantos pares estão armazenados.

Pense em um dicionário de português. Você não lê da página 1 até achar a palavra: você vai direto na letra correspondente. O dicionário de dados funciona da mesma forma — a chave é a palavra e o valor é a definição.

Segundo Cormen et al. (2022), um dicionário dinâmico dá suporte a operações de inserção, exclusão e busca, podendo resolver cada uma em tempo esperado O(1) com uma boa função hash.

Arrays acessam elementos por índice numérico (0, 1, 2…). Dicionários acessam por chaves semânticas arbitrárias (“nome”, “idades”, “admin”). Isso elimina a necessidade de saber em que posição o dado foi guardado.

Como Funciona: a Função Hash

Por baixo de todo dicionário existe um array interno. Quando você armazena o par “nome” → “Carlos”, a estrutura não guarda a string “nome” diretamente — ela a converte em um índice numérico por meio de uma função hash.

A função hash transforma a chave em um número inteiro — o índice. O valor é armazenado nessa posição do array interno. Na busca, a mesma função é aplicada à chave para encontrar o índice diretamente, sem varredura.

mapa.pseudo
CLASSE Dicionario {
    array = [null] * 128   // array interno de tamanho fixo

    FUNCAO inserir(chave, valor) {
        indice = hash(chave) % array.tamanho   // O(1) esperado
        array[indice] = valor
    }

    FUNCAO buscar(chave) {
        indice = hash(chave) % array.tamanho   // mesma função
        RETORNE array[indice]
    }

    FUNCAO remover(chave) {
        indice = hash(chave) % array.tamanho
        array[indice] = null
    }
}

Colisões

Uma colisão ocorre quando duas chaves diferentes produzem o mesmo índice hash. Toda implementação real precisa tratar colisões, pois elas são inevitáveis em arrays de tamanho finito.

Encadeamento separado

Cada posição do array armazena uma lista encadeada. Se duas chaves colidirem, ambas são adicionadas à lista naquele índice. A busca percorre a lista até encontrar a chave exata.

Endereçamento aberto

Quando o índice já está ocupado, o algoritmo procura a próxima posição livre (sondagem linear, quadrática ou hash duplo). Toda a estrutura permanece no array original.

Uma função hash ruim que coloca todas as chaves no mesmo índice degrada o dicionário para O(n) em todas as operações — tão lento quanto uma lista encadeada. A qualidade da função hash determina o desempenho real.

Casos de Uso

Qualquer problema que exija associar um identificador a um dado e recuperar esse dado rapidamente se beneficia de um dicionário.

Cache e memoização

Armazena resultados já calculados. A chave é os parâmetros da função; o valor é o resultado. Se a mesma entrada vier de novo, o resultado é recuperado em O(1) sem recalcular.

Contagem de frequência

Para contar palavras em um texto, a chave é a palavra e o valor é o contador. Cada nova ocorrência incrementa o contador diretamente sem precisar buscar na lista.

Inversão de índice

Mecanismos de busca mapeiam cada palavra para a lista de documentos que a contêm. Dado o termo de busca, o resultado é recuperado instantaneamente.

Configurações e metadados

Dados como “usuário”: “Carlos” e “nivel”: “admin” são naturalmente representados como pares chave-valor, sem precisar de índices ou colunas fixas.

Deduplicar elementos

Inserir itens como chaves de um dicionário e verificar se a chave já existe é a forma mais eficiente de eliminar duplicatas de um conjunto de dados.

Roteamento de mensagens

Em brokers e sistemas distribuídos, dicionários mapeiam identificadores de tópicos ou filas para os handlers responsáveis pelo processamento.

Complexidade das Operações

O desempenho depende da qualidade da função hash. Com boa distribuição, todas as operações principais são O(1) esperado. No pior caso (todas as chaves colidirem), degradão para O(n).

Operação Média Pior caso
Buscar (get) O(1) O(n) — todas as chaves no mesmo bucket.
Inserir (put) O(1) O(n) — todas as chaves no mesmo bucket.
Remover (delete) O(1) O(n) — todas as chaves no mesmo bucket.
Iterar sobre pares O(n) O(n) — percorre todos os elementos.
Espaço O(n) O(n) — proporcional ao número de pares.
