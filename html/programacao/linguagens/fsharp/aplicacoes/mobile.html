F# - Mobile

Aplicacoes Mobile com F#

F# no mobile aparece principalmente via .NET MAUI, Fabulous (MVU) e backend compartilhado com o restante do ecossistema .NET.

No mobile, o desafio central e estado de tela + sincronizacao de dados. F# ajuda a deixar essas transicoes explicitas e testaveis.

Padrao MVU em Mobile

Modelo de tela com comandos async
type Model = {
    Carregando: bool
    Itens: string list
    Erro: string option
}

type Msg =
    | Carregar
    | Carregado of string list
    | Falhou of string

let update msg model =
    match msg with
    | Carregar -> { model with Carregando = true; Erro = None }
    | Carregado itens -> { model with Carregando = false; Itens = itens }
    | Falhou e -> { model with Carregando = false; Erro = Some e }

Esse formato simplifica estados classicos de app mobile: loading, sucesso, erro, refresh e offline.

Offline-First e Sincronizacao

Cache local

Salve dados essenciais para abertura rapida e operacao sem rede.

Fila de comandos

Acumule acoes locais e envie quando a conectividade voltar.

Conflitos

Defina politica de merge por dominio, nao por conveniencia tecnica.

Offline-first nao e apenas cache. Exige modelagem de conflito e idempotencia no backend.

Quando F# e uma Boa Escolha no Mobile

Use F# quando

  • Voce quer compartilhar regras com backend .NET.
  • Equipe ja domina MVU e programacao funcional.
  • Produto exige regras complexas de validacao e sincronizacao.

Reavalie quando

  • Equipe depende de exemplos e plugins focados em C#.
  • Prazo exige onboarding imediato sem curva funcional.
  • UI nativa profunda for prioridade absoluta.
