F# - Embedded e IoT

F# para Edge, Gateway e IoT

F# raramente roda no firmware de microcontroladores pequenos, mas e excelente no software de borda que coleta, valida e envia dados de sensores para a nuvem.

Pense em IoT como uma cadeia: sensor -> gateway -> nuvem. F# costuma brilhar no gateway, onde regras de negocio, resiliencia e tipagem fazem diferenca.

Arquitetura Recomendada

  • Firmware em C/C++/Rust para dispositivos restritos.
  • Gateway local em .NET + F# para normalizar telemetria.
  • Broker MQTT para desacoplamento de produtores e consumidores.
  • Envio para cloud (Azure IoT, AWS IoT, Kafka) com retentativa.

Mensageria MQTT com F#

Publicar telemetria com MQTTnet
open MQTTnet
open MQTTnet.Client
open System.Text

let payload = "{\"temperatura\": 23.7, \"umidade\": 52}"

task {
    let factory = MqttFactory()
    let client = factory.CreateMqttClient()

    let options =
        MqttClientOptionsBuilder()
            .WithTcpServer("broker.local", 1883)
            .Build()

    do! client.ConnectAsync(options)

    let msg =
        MqttApplicationMessageBuilder()
            .WithTopic("fabrica/sensor/linha-1")
            .WithPayload(Encoding.UTF8.GetBytes(payload))
            .WithAtLeastOnceQoS()
            .Build()

    do! client.PublishAsync(msg)
}

Leitura de Hardware no Edge

Em Linux embarcado (como Raspberry Pi), F# pode usar System.Device.Gpio para leitura de pinos e atuar em relays, LEDs e sensores simples.

Leitura de pino digital
open System.Device.Gpio

use controller = new GpioController()
let pin = 17

controller.OpenPin(pin, PinMode.Input)
let estado = controller.Read(pin)
printfn "Estado do pino 17: %A" estado

Boas Praticas em IoT com F#

Idempotencia

Mensagens podem chegar duplicadas. Use chave de evento para deduplicar no consumidor.

Backpressure

Limite taxa de envio quando a nuvem estiver lenta para evitar perda e travamento local.

Store-and-forward

Persistencia local temporaria para operar offline e reenviar quando houver conectividade.

Nao confunda edge robusto com firmware de tempo real. F# e excelente para orquestracao e regras, nao para hard real-time de micros pequenos.
