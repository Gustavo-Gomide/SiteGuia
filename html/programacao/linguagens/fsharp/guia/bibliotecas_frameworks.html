F# - Bibliotecas e Frameworks

Bibliotecas e Frameworks de F#

F# herda todo o ecossistema .NET e ainda adiciona bibliotecas funcionais que simplificam modelagem de dominio, pipelines e tratamento de erros.

A regra pratica e simples: se uma biblioteca C# e solida, ela tambem e usavel em F#. O ganho de F# aparece ao combinar essas bibliotecas com tipos discriminados, imutabilidade e composicao de funcoes.

Frameworks Web

Composicao de rotas com Giraffe
open Giraffe

let health : HttpHandler =
    text "ok"

let app =
    choose [
        GET >=> route "/health" >=> health
        RequestErrors.notFound (text "rota nao encontrada")
    ]

Saturn sobe o nivel de abstracao para organizacao estilo controller/router. Suave, por outro lado, e util quando voce quer servidor funcional enxuto sem depender do pipeline completo do ASP.NET Core.

Evite escolher framework apenas por popularidade. Em F#, prefira o que se encaixa no modelo funcional da equipe e na necessidade de observabilidade.

Dados, Type Providers e Analise

FSharp.Data com CSV tipado em tempo de compilacao
open FSharp.Data

type Vendas = CsvProvider<"./dados/vendas.csv">

let linhas = Vendas.Load("./dados/vendas.csv").Rows
let total = linhas |> Seq.sumBy (fun r -> r.Valor)
printfn "Total: %M" total

Esse estilo reduz erro de schema em runtime. Se o arquivo mudar de formato, voce descobre no build, nao em producao.

Testes e Qualidade de Codigo

FsCheck

Property-based testing para validar invariantes em muitos casos gerados automaticamente.

Expecto

Framework de testes idiomatico para F#, com boa ergonomia para suites funcionais.

xUnit + FluentAssertions

Alternativa amplamente adotada no mundo .NET com facil integracao em CI.

Boas praticas

  • Teste regras puras sem I/O primeiro.
  • Isole acesso externo com interfaces ou funcoes injetadas.
  • Use testes de propriedade para validadores, parsers e transformacoes.

Comparacao Rapida: F# vs C# no Uso de Bibliotecas

Em F#

  • Composicao com pipeline torna fluxos lineares e legiveis.
  • Modelagem com DU + record elimina estados invalidos.
  • Result/Option reduz uso excessivo de excecoes.

Atencoes

  • Algumas libs C# exigem wrappers para API mais idiomatica.
  • Exemplos oficiais sao, em geral, escritos em C#.
  • Ferramentas com source generators costumam focar C#.
