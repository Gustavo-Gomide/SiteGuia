F# — Avançado

F# avançado cobre computation expressions para programação monádica (async, result, seq), Type Providers para acesso tipado a dados externos, Railway Oriented Programming para tratamento de erros composável, interoperabilidade bidirecional com C# e o sistema completo de OOP quando necessário.

Railway Oriented Programming (ROP) é um padrão F# para encadear operações que podem falhar. Em vez de exceções ou ifs aninhados, cada função retorna Result<'ok, 'erro> e o operador bind conecta trilhos — desvio automático para o trilho de erro caso qualquer etapa falhe, sem if aninhado.

Computation Expressions — async e task

async workflow e task computation expression
open System.Net.Http
open System.Threading.Tasks

// async { } — computation expression do F#
// (baseado em F# Async, diferente de Task C#)
let buscarTexto (url: string) : Async<string> =
    async {
        use client = new HttpClient()
        let! conteudo = client.GetStringAsync(url) |> Async.AwaitTask
        return conteudo
    }

// Compor asyncs
let buscarMultiplos urls : Async<string list> =
    urls
    |> List.map buscarTexto
    |> Async.Parallel
    |> Async.map Array.toList

// Executar async
let resultado = buscarTexto "https://example.com" |> Async.RunSynchronously

// task { } — F# 6+, compatível com C# Task diretamente
let buscarTask (url: string) : Task<string> =
    task {
        use client = new HttpClient()
        let! resp = client.GetStringAsync(url)
        return resp
    }

// seq { } — sequence expression (lazy)
let infinita =
    seq {
        let mutable i = 0
        while true do
            yield i
            i <- i + 1
    }

let primeiros10 = infinita |> Seq.take 10 |> Seq.toList

Result e Railway Oriented Programming

Encadear operações falíveis com Result e bind
type Erro =
    | EntradaInvalida of string
    | BancoDeDadosErro of string
    | NaoEncontrado

// Cada função retorna Result — composição via bind
let validarEmail (email: string) : Result<string, Erro> =
    if email.Contains("@") then Ok email
    else Error (EntradaInvalida "Email inválido")

let validarIdade (idade: int) : Result<int, Erro> =
    if idade > 0 && idade < 150 then Ok idade
    else Error (EntradaInvalida "Idade inválida")

let buscarNoBanco (email: string) : Result<Pessoa, Erro> =
    // simulado
    if email = "teste@ex.com" then
        Ok { Nome = "Teste"; Idade = 25; Email = email }
    else Error NaoEncontrado

// result computation expression (com FsToolkit.ErrorHandling)
let processarCadastro email idade =
    result {
        let! emailV = validarEmail email
        let! idadeV = validarIdade idade
        let! pessoa  = buscarNoBanco emailV
        return { pessoa with Idade = idadeV }
    }

// Sem computation expression — pipeline manual com bind
let processarManual email idade =
    validarEmail email
    |> Result.bind (fun emailV ->
        validarIdade idade
        |> Result.map (fun _ -> emailV))
    |> Result.bind buscarNoBanco

// Tratar resultado final
match processarCadastro "teste@ex.com" 25 with
| Ok pessoa -> printfn "Processado: %s" pessoa.Nome
| Error (EntradaInvalida msg) -> printfn "Erro de entrada: %s" msg
| Error NaoEncontrado         -> printfn "Não encontrado"
| Error (BancoDeDadosErro e)  -> printfn "Banco: %s" e

Type Providers

FSharp.Data — CSV, JSON e HTML type providers
// Instalar: dotnet add package FSharp.Data
open FSharp.Data

// CSV Type Provider — tipos gerados a partir de CSV real
type Populacao = CsvProvider<"populacao.csv">
// populacao.csv: País,Ano,Populacao
//                Brasil,2020,212000000

let dados = Populacao.Load("populacao.csv")
for row in dados.Rows do
    printfn "%s (%d): %d" row.País row.Ano row.Populacao   // tipado!

let total = dados.Rows |> Seq.sumBy (fun r -> r.Populacao)

// JSON Type Provider
type GitHubApi = JsonProvider<"https://api.github.com/repos/dotnet/fsharp">
let repo = GitHubApi.GetSample()
printfn "Stars: %d" repo.StargazersCount   // completamente tipado!

// HTML Type Provider
type TableauHtml = HtmlProvider<"https://en.wikipedia.org/wiki/F_Sharp_(programming_language)">
let pagina = TableauHtml.GetSample()
for tabela in pagina.Tables do
    printfn "Tabela: %s" tabela.Name

Interoperabilidade com C#

Usar bibliotecas C# de F# e exportar para C#
open System.Collections.Generic
open Microsoft.Extensions.Logging

// Usar classes C# normalmente
let lista = List<int>()      // System.Collections.Generic.List
lista.Add(1); lista.Add(2)
lista.Count                  // 2

// Implementar interfaces C#
type MeuComparador() =
    interface System.IComparable<int> with
        member _.CompareTo(other) = 0   // F# é estrito: todos os membros

// Classes F# para consumo em C#
type MinhaClasse(nome: string) =
    let mutable _contador = 0

    member _.Nome = nome                     // property
    member _.Contador = _contador
    member _.Incrementar() = _contador <- _contador + 1

    static member Criar nome = MinhaClasse(nome)

// Usar NuGet C# (ex: Newtonsoft.Json)
open Newtonsoft.Json

type Configuracao = { Host: string; Porta: int }
let config = { Host = "localhost"; Porta = 5432 }
let json    = JsonConvert.SerializeObject(config)
let config2 = JsonConvert.DeserializeObject<Configuracao>(json)

// Nullable interop — C# pode retornar null
let buscarEnv chave =
    System.Environment.GetEnvironmentVariable(chave)
    |> Option.ofObj   // null → None, valor → Some

buscarEnv "PATH" |> Option.defaultValue "não definido"

Você tem IntelliSense, completação automática e verificação de tipo para dados externos — um erro de nome de coluna ou campo virar um erro de compilação, não um crash em produção.
