F# — Introdução

F# — Functional-First .NET

F# é uma linguagem de programação funcional-first, fortemente tipada, de código aberto que roda na plataforma .NET. Criada por Don Syme na Microsoft Research em 2005, F# combina a expressividade da programação funcional (inspirada em ML/OCaml) com acesso total ao ecossistema .NET — incluindo todas as bibliotecas C#, NuGet e interoperabilidade bidirecional.

F# usa inferência de tipos Hindley-Milner: o compilador deduz os tipos da grande maioria das expressões sem anotações. Diferente de C# (orientado a objetos com features funcionais adicionadas), F# é funcional por padrão — imutabilidade é o padrão, efeitos colaterais são explícitos, e o sistema de tipos previne classes inteiras de bugs em tempo de compilação.

Primeiros Programas

Hello World e conceitos imediatos
// F# — indentação significativa (como Python, mas tipado)

// printfn = printf + newline; %s string, %d int, %f float
printfn "Olá, Mundo!"
printfn "F# versão %s" "latest"

// let — imutável por padrão
let nome = "F#"               // string inferida
let versao = 8                // int inferido
let pi = 3.14159265358979    // float inferido

// Função — let com argumentos
let somar a b = a + b        // (int -> int -> int)
let resultado = somar 3 4   // 7 — aplicação parcial!

// Aplicação parcial
let somar5 = somar 5         // (int -> int)
let dez = somar5 5

printfn "Resultado: %d" resultado
printfn "Dez: %d" dez

Características Essenciais

Imutabilidade por Padrão

let x = 5 cria um binding imutável. Para mutabilidade, let mutable x = 5 — mas é desempenhado. Records e DUs são imutáveis por default; cópias com mudança via with.

Discriminated Unions

Tipos soma com pattern matching exaustivo — modelam estados de domínio com precisão. type Resultado = Ok of int | Erro of string. O compilador verifica se todos os casos foram tratados.

Type Providers

Magia única do F#: gera tipos em tempo de compilação a partir de fontes externas (CSV, JSON, SQL, OpenAPI). Acesso tipado a dados externos sem escrever modelos manualmente.

Computation Expressions

Syntactic sugar para monads — async { }, seq { }, result { }. Permite escrever código assíncrono ou de tratamento de erros com aparência imperativa mas semântica funcional.

Interop Total com .NET

Usa exatamente as mesmas DLLs que C#. Pode chamar qualquer biblioteca NuGet, usar ASP.NET, Entity Framework, SignalR — tudo disponível de F# sem adaptações.

Pipeline Operator |>

valor |> funcao1 |> funcao2 |> funcao3 — composição legível de funções. Inspirado em Unix pipes, fundamental para o estilo F# de processamento de dados.

F# vs C# — Quando Usar Cada Um

Prefira F# quando…

  • Modelagem de domínio complexo (DDD com DUs em vez de classes)
  • Análise e transformação de dados com pipelines funcionais
  • Ciência de dados, ML, processamento estatístico
  • Código com muitas regras de negócios a validar
  • Sistemas financeiros: cálculos, derivativos, risk
  • Você quer concorrência sem estado mutável compartilhado

Prefira C# quando…

  • Aplicações CRUD padrão com ORM e controllers
  • Time não familiar com paradigma funcional
  • GUIs desktop (WPF, WinForms — mais natural em C#)
  • Muito código legado C# para manter
  • Scaffolding automático (Visual Studio gera C#)

Um sistema F# bem escrito torna estados inválidos irrepresentáveis — bug de lógica de negócio vira erro de compilação.
