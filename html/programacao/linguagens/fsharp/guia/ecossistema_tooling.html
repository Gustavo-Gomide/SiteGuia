F# — Ecossistema e Tooling

F# usa o ecossistema .NET inteiro: dotnet CLI para projetos, NuGet/Paket para pacotes, Fantomas para formatação automática, Ionide no VS Code para IDE de nível profissional, FAKE para builds funcionais, FsCheck para property-based testing e Fable para compilar F# em JavaScript.

Fable compila F# para JavaScript idiomatic — não é uma transpilação mecânica, mas gera código JS legível que integra diretamente com o ecossistema npm. Você pode usar React com Feliz (DSL F# para JSX), escrever testes com Jest e fazer bundle com Webpack ou Vite, tudo a partir de código F# puro.

FSI — F# Interactive (Scripts)

script.fsx — automação, análise de dados, glue code
// Arquivo: analise.fsx
// Executar: dotnet fsi analise.fsx

// Referência a pacotes NuGet direto no script (F# 5+)
#r "nuget: FSharp.Data, 6.4.0"
#r "nuget: Plotly.NET, 4.0.0"
#r "nuget: FsToolkit.ErrorHandling, 4.0.0"

open FSharp.Data
open Plotly.NET

let args = fsi.CommandLineArgs   // argumentos da linha de comando
let arquivo = if args.Length > 1 then args.[1] else "dados.csv"

type Vendas = CsvProvider<"vendas.csv">
let dados = Vendas.Load(arquivo)

// Processar
let totalPorMes =
    dados.Rows
    |> Seq.groupBy (fun r -> r.Mes)
    |> Seq.map (fun (mes, rows) -> mes, rows |> Seq.sumBy (fun r -> r.Valor))
    |> Seq.sortBy fst

for (mes, total) in totalPorMes do
    printfn "Mês %d: R$ %.2f" mes total

// Gerar gráfico HTML
let meses, valores = totalPorMes |> Seq.toArray |> Array.unzip
Chart.Bar(valores, meses)
|> Chart.withTitle "Vendas por Mês"
|> Chart.show

Ionide — IDE F# no VS Code

FsAutoComplete

LSP server que alimenta o Ionide: completação, hover docs, peek definition, find references, rename, go to definition em projetos .NET completos.

F# Solution Explorer

Gerencia arquivos .fsproj, ordena <Compile> (ordem importa em F#!), cria módulos, namespaces e projetos diretamente na UI.

FSI Integrado

Selecione código e pressione Alt+Enter para enviar ao FSI REPL embutido. Ideal para análise de dados exploratory e verificação rápida de funções.

Diagnósticos em tempo real

Erros de tipo, warnings de pattern matching incompleto, subrecomendações de Fantomas e sugestões de refatoração aparecem inline enquanto você digita.

FsCheck — Property-Based Testing

Testar propriedades matemáticas em vez de exemplos
// dotnet add package FsCheck.Xunit
open FsCheck
open FsCheck.Xunit
open Xunit

// Property: reverter duas vezes = original
[<Property>]
let reverterDuasVezes (lista: int list) =
    List.rev (List.rev lista) = lista   // FsCheck gera centenas de listas

// Property com pré-condição
[<Property>]
let somaComutativa (a: int) (b: int) =
    a + b = b + a

// Custom generator para tipos de domínio
type Email = Email of string
let emailGen =
    Arb.generate<string>
    |> Gen.map (fun s -> Email (s + "@test.com"))
    |> Arb.fromGen

type Generators =
    static member Email() = emailGen

[<Property(Arbitrary = [| typeof<Generators> |])>]
let emailSempreValido (Email e) =
    e.Contains("@")

// Rodar FsCheck standalone (sem framework de teste)
Check.Quick reverterDuasVezes          // roda 100 casos
Check.Verbose reverterDuasVezes        // mostra cada caso gerado

FAKE — Build System Funcional

build.fsx — targets, dependências, CI/CD
// build.fsx
#r "nuget: Fake.Core.Target"
#r "nuget: Fake.DotNet.Cli"
#r "nuget: Fake.IO.FileSystem"

open Fake.Core
open Fake.DotNet
open Fake.IO

// Inicializar contexto FAKE
Target.initEnvironment()

Target.create "Clean" (fun _ ->
    Shell.cleanDir "./artifacts"
)

Target.create "Restore" (fun _ ->
    DotNet.restore id "."
)

Target.create "Build" (fun _ ->
    DotNet.build (fun opts ->
        { opts with Configuration = DotNet.BuildConfiguration.Release }) "."
)

Target.create "Test" (fun _ ->
    DotNet.test (fun opts ->
        { opts with NoBuild = true }) "."
)

Target.create "Publish" (fun _ ->
    DotNet.publish (fun opts ->
        { opts with OutputPath = Some "./artifacts"
                    Configuration = DotNet.BuildConfiguration.Release }) "."
)

// Cadeia de dependências
open Fake.Core.TargetOperators
"Clean" ==> "Restore" ==> "Build" ==> "Test" ==> "Publish"

[<EntryPoint>]
let main args = Target.runOrDefault "Test" args

Fable — F# para JavaScript

F# compilado para JS com Feliz (React DSL)
// Criar projeto Fable
// dotnet new fable -lang F# -o FableApp
// cd FableApp ; npm install ; dotnet fable watch src --run webpack serve

// Componente React com Feliz (F# React DSL)
// dotnet add package Feliz
open Feliz
open Feliz.Hooks

[<ReactComponent>]
let Contador () =
    let (count, setCount) = React.useState 0
    Html.div [
        prop.className "contador"
        prop.children [
            Html.h1 (sprintf "Contagem: %d" count)
            Html.button [
                prop.onClick (fun _ -> setCount (count + 1))
                prop.text "Incrementar"
            ]
            Html.button [
                prop.onClick (fun _ -> setCount (count - 1))
                prop.text "Decrementar"
            ]
        ]
    ]

// Chamadas JS (bindings)
open Fable.Core.JsInterop

[<Import("useState", from = "react")>]
let useState<'T> (init: 'T) : 'T * ('T -> unit) = jsNative

// window.alert via Fable
open Browser.Dom
window.alert("olá do F#")

// Fetch API
Fetch.fetch "/api/dados" []
|> Promise.bind (fun resp -> resp.json<MeuTipo>())
|> Promise.map (fun dados -> printfn "%A" dados)

Paket vs NuGet

Use Paket quando

  • Múltiplos projetos na solution com dependências centralizadas
  • Quiser paket.lock preciso com hashes de segurança
  • Precisar de dependências de fontes alternativas (GitHub, HTTP)
  • Quiser grupos de dependências (Main, Tests, Build)
  • Projetos F# mais antigos ou scripts FSI com #load

Use NuGet (padrão) quando

  • Projeto novo e simples — menos fricção de setup
  • Integração direta com dotnet add package
  • Ferramentas de CI/CD que esperam packages.lock.json
  • Solução .NET mista F# + C# com Visual Studio
  • Não precisar de features avançadas do Paket
