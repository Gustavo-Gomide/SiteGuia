OCaml — Ecossistema & Tooling

opam, dune, utop, Merlin, ocamlformat e tudo que você precisa para um workflow profissional em OCaml.

O ecossistema OCaml passou por uma revolução entre 2013 e 2020: opam tornou o gerenciamento de pacotes trivial, dune substituiu o caótico sistema de Makefiles, e ocaml-lsp trouxe autocompletar de primeira classe para VS Code e Emacs. Hoje, iniciar um projeto OCaml profissional leva menos de 5 minutos.

opam — Gerenciador de Pacotes

opam (OCaml Package Manager) gerencia compiladores, switches (ambientes isolados) e pacotes de terceiros. É o equivalente ao rustup + cargo do mundo OCaml.

bash — Comandos opam essenciais
# ── Switches — ambientes isolados ──────────────────────────────────
opam switch list            # listar switches disponíveis
opam switch create 5.2.0    # criar switch com OCaml 5.2.0
opam switch create 4.14.2   # LTS — mais estável para produção
opam switch set 5.2.0       # ativar switch
eval $(opam env)            # aplicar variáveis ao shell atual

# Switch local para o projeto (recomendado)
cd meu_projeto
opam switch create . 5.2.0 --no-install
eval $(opam env)

# ── Instalar pacotes ────────────────────────────────────────────────
opam install dune utop ocamlformat merlin ocaml-lsp-server
opam install lwt dream yojson cohttp-lwt-unix
opam install alcotest ppx_inline_test

# ── Gerenciar pacotes ───────────────────────────────────────────────
opam list                   # listar instalados
opam list --available       # listar todos disponíveis no repositório
opam search json            # buscar pacotes
opam info yojson            # detalhes de um pacote
opam update                 # atualizar índice de pacotes
opam upgrade                # atualizar pacotes instalados
opam remove pacote          # remover pacote

# ── Arquivo opam — declarar dependências do projeto ─────────────────
# meu_projeto.opam (gerado pelo dune)
# opam install . --deps-only   ← instala apenas as deps do projeto
opam pin add . --no-action  # "pinnar" versão local do projeto

dune — Build System

dune é o build system padrão de OCaml. Usa arquivos declarativos (dune) para descrever o projeto. Gerencia compilação, testes, geração de documentação e empacotamento com um único comando.

Estrutura de projeto com dune
meu_projeto/
├── dune-project          # raiz do projeto — obrigatório
├── meu_projeto.opam      # gerado por dune (dune build @install)
├── lib/
│   ├── dune              # biblioteca pública
│   ├── util.ml
│   └── util.mli          # interface (opcional — assinatura explícita)
├── bin/
│   ├── dune              # executável
│   └── main.ml
└── test/
    ├── dune              # testes
    └── test_util.ml
dune-project — arquivo raiz
(lang dune 3.14)

(name meu_projeto)

(generate_opam_files true)

(source
 (github usuario/meu_projeto))

(package
 (name meu_projeto)
 (synopsis "Descrição curta do projeto")
 (description "Descrição longa")
 (depends
  (ocaml (>= 4.14))
  dune
  lwt
  yojson
  alcotest))
lib/dune — biblioteca
(library
 (name meu_projeto)       ; nome do módulo principal
 (public_name meu_projeto) ; nome no opam (para libs instaláveis)
 (libraries lwt yojson)   ; dependências
 (preprocess              ; preprocessors (ppx)
  (pps lwt_ppx ppx_deriving.show)))
bin/dune — executável
(executable
 (name main)
 (public_name meu-projeto) ; nome do binário instalado
 (libraries meu_projeto lwt lwt.unix)
 (preprocess (pps lwt_ppx)))
bash — Comandos dune
dune build              # compilar tudo
dune exec bin/main.exe  # compilar e executar
dune exec bin/main.exe -- --arg1 val  # com argumentos
dune test               # rodar todos os testes
dune clean              # limpar artefatos de build (_build/)
dune build @doc         # gerar documentação odoc
dune build @install     # gerar arquivo .opam
dune install            # instalar binário/lib no switch opam

# Modo watch — recompila ao salvar
dune build --watch
dune exec --watch bin/main.exe

utop — REPL Avançado

utop (Universal Top-Level) é um REPL interativo para OCaml com autocompletar, histórico, syntax highlighting e integração com dune. É a forma mais rápida de experimentar código OCaml.

utop — sessão interativa
# Iniciar utop
utop

(* Expressões terminam com ;; no toplevel *)
# 1 + 1;;
- : int = 2

(* Definições sem ;; funcionam com utop moderno *)
# let x = 42;;
val x : int = 42

(* Carregar biblioteca *)
# #require "yojson";;
# open Yojson.Basic;;
# from_string {|{"nome": "OCaml", "ano": 1996}|};;
- : t = `Assoc [("nome", `String "OCaml"); ("ano", `Int 1996)]

(* Carregar arquivo .ml *)
# #use "meu_arquivo.ml";;

(* Usar projeto dune no utop *)
dune utop lib  # ← no terminal, carrega a lib do projeto

(* Dentro do utop após dune utop lib: *)
# open Meu_projeto;;
# Util.soma 2 3;;
- : int = 5
~/.ocamlinit — configuração inicial do utop
(* ~/.ocamlinit — executado ao iniciar utop *)
#use "topfind";;
#thread;;
#require "str";;
let pp = Format.printf;;  (* alias útil *)

Merlin e ocaml-lsp-server

Merlin é o servidor de análise semântica para OCaml. ocaml-lsp-server é o wrapper LSP que expõe Merlin para editores. Juntos, oferecem autocompletar, navegação, type hints e erros em tempo real nos editores mais populares.

bash — Instalar suporte a editores
# Instalar servidor LSP (funciona com VS Code, Neovim, Emacs, etc.)
opam install merlin ocaml-lsp-server

# VS Code: instalar extensão "OCaml Platform" (by OCaml Labs)
# ID: ocamllabs.ocaml-platform

# A extensão detecta automaticamente o switch opam ativo
# Para projetos com switch local:
cd meu_projeto
eval $(opam env)
code .

# Neovim com nvim-lspconfig:
# require('lspconfig').ocamllsp.setup{}

# Emacs: instalar tuareg-mode + merlin via MELPA
Autocompletar

Merlin analisa tipos em tempo real. List. + Tab mostra todos os valores do módulo com seus tipos. O autocompletar é baseado em semântica, não em texto.

Type-on-hover

Passe o mouse sobre qualquer expressão para ver seu tipo inferido. Indispensável ao aprender OCaml — o compilador é o seu guia.

Erros em tempo real

Erros de tipo são sublinhados antes de salvar. Como OCaml tem inferência total, erros geralmente indicam o local exato da discrepância de tipos.

Jump to definition

Navegue até a definição de qualquer função, tipo ou módulo — incluindo a biblioteca padrão. Funciona mesmo através de fronteiras de módulo.

ocamlformat — Formatação Automática

bash + .ocamlformat — formatador de código
# Instalar
opam install ocamlformat

# Formatar um arquivo
ocamlformat --inplace meu_arquivo.ml

# Formatar todos os .ml e .mli do projeto
dune fmt

# Verificar se o código está formatado (CI)
dune fmt --check
.ocamlformat — arquivo de configuração (raiz do projeto)
version = 0.26.2
profile = default    # default | compact | conventional | ocamlformat
line-length = 100
break-string-literals = auto
doc-comments = before

Testes com Alcotest e OUnit2

Alcotest é o framework de testes mais moderno para OCaml: saída colorida, execução paralela e integração nativa com dune. OUnit2 é mais antigo mas amplamente usado em projetos legados.

test/test_util.ml — Alcotest
(* opam install alcotest *)

open Alcotest

(* Testes unitários — verificações simples *)
let test_soma () =
  check int "soma básica" 5 (2 + 3);
  check int "soma com zero" 7 (7 + 0)

let test_strings () =
  check string "concatenação" "hello world" ("hello" ^ " world");
  check (option int) "parse int" (Some 42) (int_of_string_opt "42");
  check (option int) "parse inválido" None (int_of_string_opt "abc")

let test_lista () =
  check (list int) "filter pares"
    [2; 4]
    (List.filter (fun x -> x mod 2 = 0) [1; 2; 3; 4; 5])

(* Testar exceções *)
let test_excecao () =
  check_raises "divisão por zero" Division_by_zero
    (fun () -> ignore (1 / 0))

(* Agrupar em suites *)
let () =
  run "Meu Projeto" [
    "aritmética", [
      test_case "Soma básica"   `Quick test_soma;
      test_case "Divisão zero"  `Quick test_excecao;
    ];
    "strings", [
      test_case "Operações"     `Quick test_strings;
    ];
    "listas", [
      test_case "Filtragem"     `Quick test_lista;
    ];
  ]
test/dune — configuração de testes
(test
 (name test_util)
 (libraries meu_projeto alcotest))
ppx_inline_test — testes inline no código fonte
(* No dune: (preprocess (pps ppx_inline_test)) + (inline_tests) *)

let fatorial n =
  let rec go n acc = if n <= 0 then acc else go (n-1) (n*acc)
  in go n 1

let%test "fatorial 0" = fatorial 0 = 1
let%test "fatorial 5" = fatorial 5 = 120
let%test "fatorial 10" = fatorial 10 = 3628800

(* Executar: dune test --force *)

odoc — Documentação

odoc gera documentação HTML a partir de comentários especiais (** ... *) nos arquivos .mli. A documentação é vinculada ao sistema de tipos — links entre módulos, tipos e funções são gerados automaticamente.

lib/util.mli — Documentação com odoc
(** Módulo de utilitários matemáticos.

    Este módulo fornece funções para operações aritméticas comuns.
    
    {b Exemplo de uso:}
    {[
      let resultado = Util.soma 2 3
      (* resultado = 5 *)
    ]}
*)

(** [soma a b] retorna a soma de [a] e [b].
    
    @param a primeiro operando
    @param b segundo operando
    @return a + b
    @raise Invalid_argument se algum argumento for negativo
*)
val soma : int -> int -> int

(** [media xs] calcula a média aritmética de uma lista.
    
    @return [None] se a lista for vazia
*)
val media : float list -> float option
bash — Gerar e visualizar documentação
# Gerar documentação HTML
dune build @doc

# Documentação gerada em:
# _build/default/_doc/_html/meu_projeto/index.html

# Servir localmente
cd _build/default/_doc/_html
python3 -m http.server 8080
# Acesse http://localhost:8080

# Publicar no ocaml.org (via opam-publish)
opam install opam-publish
opam publish

ocamlfind — Gerenciamento de Bibliotecas

ocamlfind (findlib) é a camada de descoberta de bibliotecas OCaml. Permite compilar arquivos individuais com -package nome sem precisar especificar caminhos manualmente.

bash — ocamlfind para compilação manual
# Compilar com dependências via ocamlfind
ocamlfind ocamlopt \
  -package yojson,lwt,lwt.unix \
  -linkpkg \
  main.ml -o main

# Listar todas as bibliotecas instaladas
ocamlfind list

# Ver metadados de uma biblioteca
ocamlfind query yojson
# /home/user/.opam/5.2.0/lib/yojson

# Verificar dependências de uma biblioteca
ocamlfind query -r -p-format yojson

Alcotest Framework de testes unitários pytest (Python), JUnit (Java)
ocamlfind Descoberta e compilação com libs pkg-config (C), classpath (Java)
