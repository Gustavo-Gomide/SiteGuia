OCaml — Introdução

OCaml

Uma linguagem funcional, imperativa e orientada a objetos com um dos sistemas de tipos mais expressivos da indústria — inferência automática, pattern matching algébrico e zero custo de abstração.

OCaml é a linguagem que prova que segurança de tipos e desempenho não são excludentes. O compilador nativo gera código comparável a C/C++ enquanto garante ausência de erros de null pointer, buffer overflow e data races em tempo de compilação.

História e Origem

OCaml (Objective Categorical Abstract Machine Language) surgiu no INRIA (Institut National de Recherche en Informatique et en Automatique) na França. Sua genealogia remonta diretamente à família ML (Meta Language), criada por Robin Milner em 1973 no Edinburgh para o sistema de prova de teoremas LCF.

A linha do tempo detalhada é:

1973 — ML

Robin Milner cria ML como metalinguagem para o provador LCF. Introduz polimorfismo paramétrico e inferência de tipos (algoritmo W de Hindley-Milner).

1984 — Caml

Gilles Kahn e Guy Cousineau no INRIA criam Caml (Categorical Abstract Machine Language), implementação eficiente de ML para a CAM.

1990 — Caml Light

Xavier Leroy reimplementa do zero, criando um compilador bytecode portável e muito mais rápido. Adota garbage collector mais eficiente.

1996 — OCaml

Didier Rémy e Jérôme Vouillon adicionam o sistema de objetos (O de Objective). Xavier Leroy integra o compilador nativo de alta performance.

2012 — Módulos de Primeira Classe

Módulos passam a ser valores de primeira classe, permitindo abstração ainda mais poderosa.

2022 — OCaml 5 (Multicore)

Effects e suporte nativo a multicore com domains. Um dos projetos mais aguardados da história da linguagem, desenvolvido por KC Sivaramakrishnan e equipe.

Xavier Leroy é o principal arquiteto do OCaml moderno. Também é autor do CompCert — um compilador C verificado formalmente em Coq — demonstrando que OCaml é a ferramenta de escolha para software de máxima confiabilidade.

Características Fundamentais

Inferência de Tipos Total

O compilador deduz o tipo de cada expressão sem anotações obrigatórias. A função let soma a b = a + b tem tipo inferido int -> int -> int automaticamente.

Tipos Algébricos

Enumerações com dados associados (variants) modelam estruturas como árvores, resultados e erros sem null pointers. O compilador garante que todos os casos sejam tratados no pattern matching.

Multiparadigma

Funcional (funções de primeira classe, closures, imutabilidade por padrão), imperativo (mutabilidade explícita com ref, loops) e orientado a objetos (classes, herança, polimorfismo estrutural).

Sistema de Módulos

O sistema de módulos de OCaml é o mais expressivo das linguagens de produção. Módulos são unidades de abstração com interface (signature) separada da implementação.

Compilador Nativo

ocamlopt gera código de máquina nativo comparável a C. Aplicações OCaml têm desempenho em produção muito acima de linguagens interpretadas.

Garbage Collector Incremental

GC geracional com zona jovem minúscula (256 KB por padrão) coletada quase sem pausa. Vital para aplicações de baixa latência como trading e roteadores.

Casos de Uso em Produção

Jane Street Capital

A maior usuária de OCaml no mundo. Toda a infraestrutura de trading quantitativo — bilhões de dólares em transações diárias — é escrita em OCaml. Jane Street contribuiu com Core, Async e dezenas de bibliotecas abertas.

Compiladores e Ferramentas

O próprio compilador OCaml, o compilador Hack (PHP tipado do Facebook/Meta), o Coq (assistente de prova), o Why3 (verificação formal) e o compilador Rust em seus primórdios foram escritos em OCaml.

Verificação Formal

CompCert (compilador C verificado), seL4 (microkernel verificado em Isabelle), e projetos da área de segurança crítica usam OCaml como linguagem de implementação ou extração.

Sistemas Distribuídos

MirageOS é um unikernel escrito 100% em OCaml. Servidores DNS, firewalls e aplicações cloud rodam diretamente sobre hardware sem OS intermediário.

Análise Estática

O Facebook Infer (analisador estático para Android/iOS) é implementado em OCaml. Detecta bugs antes da execução em código Java, C, C++ e Objective-C.

Blockchain

Tezos, uma das maiores blockchains PoS, é implementada em OCaml. A escolha foi motivada pela capacidade de verificação formal dos contratos inteligentes.

Instalação e Configuração

O gerenciador de pacotes oficial do OCaml é o opam (OCaml Package Manager). Ele instala o compilador, gerencia múltiplas versões e instala bibliotecas de terceiros.

Instalação no Linux/macOS

bash — instalação via opam
# 1. Instalar opam via gerenciador do sistema
# Ubuntu/Debian:
sudo apt install opam

# macOS via Homebrew:
brew install opam

# 2. Inicializar opam (cria ~/.opam)
opam init

# 3. Aplicar as variáveis de ambiente
eval $(opam env)

# 4. Instalar a versão estável mais recente do OCaml
opam switch create 5.2.0   # ou 4.14.2 para LTS
eval $(opam env)

# 5. Verificar versão
ocaml --version
# The OCaml toplevel, version 5.2.0

# 6. Instalar ferramentas essenciais
opam install dune utop ocamlformat merlin ocaml-lsp-server

Instalação no Windows

PowerShell/WSL — Windows
# Opção 1: WSL2 (recomendada)
# Instale o Ubuntu via WSL2 e siga os passos do Linux acima

# Opção 2: opam nativo para Windows (experimental)
winget install OCaml.opam

# Opção 3: DkML (instalador all-in-one para Windows)

O REPL: utop

utop — Universal Top-level para OCaml
# Iniciar utop
utop

# Dentro do utop — expressões terminam com ;;
# 1 + 2;;
(* - : int = 3 *)

(* Definir função *)
let dobro x = x * 2;;
(* val dobro : int -> int = <fun> *)

dobro 21;;
(* - : int = 42 *)

(* Carregar um arquivo .ml *)
#use "meu_programa.ml";;

(* Instalar pacote e usar no utop *)
#require "str";;  (* bibliotecas findlib *)

Hello World e Primeiras Expressões

Todo programa OCaml é uma sequência de definições (com let) e expressões. Ao contrário de C/Java, não existe uma função main especial — a execução começa no nível superior do módulo.

hello.ml — Hello World em OCaml
(* Comentários em OCaml usam (* *) — podem ser aninhados *)

(* print_string: saída sem newline automático *)
let () = print_string "Hello, World!\n"

(* print_endline: adiciona '\n' automaticamente *)
let () = print_endline "Hello, OCaml!"

(* Printf.printf para saída formatada *)
let nome = "Mundo"
let () = Printf.printf "Olá, %s! 2 + 2 = %d\n" nome (2 + 2)

(* Format.printf — módulo de formatação tipada (preferido) *)
let () = Format.printf "Pi ≈ %.4f@\n" 3.14159

(* let () = expr é o idioma OCaml para efeitos colaterais
   O tipo unit (()) é equivalente ao void de C *)
Compilar e executar
# Compilar para bytecode (portável)
ocamlfind ocamlc -package str -linkpkg hello.ml -o hello
./hello

# Compilar para código nativo (mais rápido)
ocamlfind ocamlopt -package str -linkpkg hello.ml -o hello
./hello

# Com dune (recomendado para projetos reais):
# Crie dune-project e dune, depois:
dune exec ./hello.exe

# Executar diretamente pelo interpretador
ocaml hello.ml
Primeiras expressões — tipos inferidos
(* Tudo é uma expressão — cada expressão tem um tipo *)
let x = 42              (* val x : int = 42 *)
let y = 3.14            (* val y : float = 3.14 *)
let nome = "OCaml"      (* val nome : string = "OCaml" *)
let ativo = true        (* val ativo : bool = true *)
let letra = 'a'         (* val letra : char = 'a' *)

(* Operadores: int e float NÃO se misturam em OCaml *)
let soma_int   = 10 + 5      (* int: + *)
let soma_float = 1.5 +. 2.5  (* float: +. *)
let pot        = 2.0 ** 10.0 (* float: ** *)

(* Conversão explícita *)
let x_float = float_of_int x    (* 42.0 *)
let y_int   = int_of_float y    (* 3 — trunca *)

(* String operations *)
let saudacao = "Olá" ^ ", " ^ nome   (* ^ concatena strings *)
let tam = String.length nome           (* 5 *)
let upper = String.uppercase_ascii nome (* "OCAML" *)
