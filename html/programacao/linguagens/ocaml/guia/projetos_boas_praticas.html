OCaml - Projetos e Boas Praticas

Projetos OCaml em nivel profissional

Estrutura de repositorio, qualidade de codigo e operacao continua para equipes que querem evoluir software com seguranca.

Boas praticas em OCaml giram em torno de fronteiras claras de modulos, automacao no dune e disciplina de testes para manter invariantes de tipos ao longo do tempo.

Estrutura de projeto recomendada

Layout base
meu_sistema/
├── dune-project
├── meu_sistema.opam
├── lib/
│   ├── dominio/
│   ├── aplicacao/
│   ├── infraestrutura/
│   └── dune
├── bin/
│   ├── main.ml
│   └── dune
├── test/
│   ├── unit/
│   ├── integracao/
│   └── dune
└── .github/workflows/ci.yml

Separar dominio, aplicacao e infraestrutura evita que detalhes de I/O contaminem logica de negocio e facilita testes deterministas.

Dependencias, reproducibilidade e upgrades

Use switch local e registre versoes minimas no arquivo .opam. Em pipelines, execute opam lock para builds reproduziveis quando o projeto exigir previsibilidade forte.

Comandos essenciais
opam switch create . 5.2.0 --no-install
opam install . --deps-only --with-test
opam lock
opam exec -- dune build @all
opam exec -- dune test

Testes e qualidade de codigo

Unitarios

Alcotest para funcao pura e regras de dominio.

Propriedade

QCheck para validar invariantes em entradas aleatorias.

Integracao

Testar caminho completo de API + banco com fixtures isoladas.

Estilo

ocamlformat e ocaml-lsp no CI para consistencia.

Padroes avancados para manutencao longa

  • Use interfaces .mli para estabilizar contratos publicos.
  • Prefira tipos de dominio explicitos a strings "soltas".
  • Trate erro com result no nucleo e converta para HTTP/CLI na borda.
  • Versione APIs com modulos (Api_v1, Api_v2) para migracoes graduais.

Antipadrao frequente: esconder excecoes em camadas profundas. Em sistemas grandes, isso degrada observabilidade e encarece incidentes.

CI/CD e release

Trecho de pipeline CI
name: ci
on: [push, pull_request]
jobs:
  build-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: ocaml/setup-ocaml@v3
        with:
          ocaml-compiler: 5.2.0
      - run: opam install . --deps-only --with-test --yes
      - run: opam exec -- dune build @all
      - run: opam exec -- dune test

Com isso, todo commit valida build, interface de modulos e regressao funcional antes de chegar em producao.
