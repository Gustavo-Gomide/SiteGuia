OCaml - Aplicacoes em Embedded e IoT

OCaml em dispositivos e borda

Uso pratico de OCaml em gateways IoT, sistemas de telemetria e componentes de alto requisito de confiabilidade.

No IoT real, OCaml costuma atuar melhor no edge inteligente e no plano de controle, onde tipos fortes e concorrencia segura reduzem falhas operacionais.

Escopo realista para embedded com OCaml

Para microcontroladores ultrarrestritos (MCUs sem MMU), C/Rust ainda predominam. Ja em Linux embarcado (Raspberry Pi, gateways ARM), OCaml e altamente viavel para servicos de orquestracao, coleta e validacao.

Bom encaixe

Gateways MQTT, regras de negocio, edge analytics, sincronizacao offline.

Menor encaixe

Firmware bare-metal com hard realtime estrito e memoria minima.

Arquitetura tipica de edge gateway

Exemplo: validacao e publicacao MQTT

gateway.ml
type leitura = { sensor_id : string; temp_c : float; ts : int }

let valida l =
  l.temp_c > -. 40.0 && l.temp_c < 125.0

let normaliza l =
  { l with temp_c = Float.round (l.temp_c *. 100.0) /. 100.0 }

let publicar _cliente l =
  if valida l then
    let payload =
      Printf.sprintf {|{"sensor":"%s","temp":%.2f,"ts":%d}|}
        l.sensor_id l.temp_c l.ts in
    ignore payload
  else
    Printf.eprintf "Leitura invalida: %s\n" l.sensor_id

Mesmo sem mostrar o cliente MQTT completo, o exemplo ilustra o ponto central: invariantes de dominio capturados antes da transmissao.

Topicos avancados para confiabilidade

  • Store-and-forward local com disco para tolerar quedas de conectividade.
  • Supervisao de processo via systemd e health checks periodicos.
  • Cross-compilation controlada para ARM e imagens minimas de deploy.
  • Uso de MirageOS para appliances de rede com superficie de ataque reduzida.

Em IoT, o erro mais caro e ignorar comportamento offline. Planeje fila local, retry com backoff e idempotencia desde o inicio.
