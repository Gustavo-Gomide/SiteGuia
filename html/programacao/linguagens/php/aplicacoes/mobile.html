PHP como Backend para Aplicativos Móveis

REST API, JWT, OAuth2, push notifications FCM e Laravel Sanctum para apps React Native e Flutter

PHP não é usado para desenvolver aplicativos móveis nativos (Android/iOS). Seu papel em mobile é o de backend: fornecer APIs consumidas por React Native, Flutter, Swift ou Kotlin. Laravel é a escolha mais comum por oferecer autenticação, autorização, filas e cache prontos para uso.

Arquitetura PHP + App Móvel

Autenticação com Laravel Sanctum para Mobile

Apps móveis usam tokens (Bearer tokens) — não cookies de sessão. Laravel Sanctum emite tokens com escopos e TTL, que são enviados no header Authorization: Bearer <token>.

php — endpoints de autenticação mobile
// routes/api.php
Route::post('/auth/login', [MobileAuthController::class, 'login']);
Route::post('/auth/registro', [MobileAuthController::class, 'registro']);
Route::post('/auth/refresh', [MobileAuthController::class, 'refresh']);

Route::middleware('auth:sanctum')->group(function() {
    Route::delete('/auth/logout', [MobileAuthController::class, 'logout']);
    Route::get('/me', fn(Request $r) => new UserResource($r->user()));
});
php — MobileAuthController
class MobileAuthController extends Controller
{
    public function login(Request $req): JsonResponse
    {
        $req->validate([
            'email'       => 'required|email',
            'password'    => 'required|string',
            'device_name' => 'required|string|max:100', // identifica o dispositivo
        ]);

        $user = User::where('email', $req->email)->first();

        if (!$user || !Hash::check($req->password, $user->password)) {
            return response()->json(['message' => 'Credenciais inválidas'], 401);
        }

        // Um token por dispositivo — fácil revogar individualmente
        $token = $user->createToken($req->device_name)->plainTextToken;

        return response()->json([
            'token' => $token,
            'user'  => new UserResource($user),
        ]);
    }

    public function logout(Request $req): JsonResponse
    {
        $req->user()->currentAccessToken()->delete(); // revoga só este dispositivo
        return response()->json(['message' => 'Desconectado']);
    }
}

Push Notifications com FCM

O Firebase Cloud Messaging (FCM) é a solução do Google para push notifications em Android e iOS. PHP envia mensagens para o servidor FCM via HTTP v1 API, que as entrega ao dispositivo.

bash — instalar SDK Firebase Admin
composer require kreait/firebase-php
php — enviar push notification via Laravel Job
use Kreait\Firebase\Factory;
use Kreait\Firebase\Messaging\CloudMessage;
use Kreait\Firebase\Messaging\Notification;

class EnviarPushNotificationJob implements ShouldQueue
{
    use Dispatchable, Queueable, SerializesModels;

    public function __construct(
        public readonly string $deviceToken,
        public readonly string $titulo,
        public readonly string $corpo,
        public readonly array  $dados = [],
    ) {}

    public function handle(): void
    {
        $messaging = (new Factory())
            ->withServiceAccount(config('firebase.credentials'))
            ->createMessaging();

        $message = CloudMessage::withTarget('token', $this->deviceToken)
            ->withNotification(Notification::create($this->titulo, $this->corpo))
            ->withData($this->dados);

        $messaging->send($message);
    }
}

// Despachar ao confirmar pedido
EnviarPushNotificationJob::dispatch(
    $user->device_token,
    'Pedido Confirmado!',
    "Seu pedido #{$pedido->numero} está sendo preparado.",
    ['pedido_id' => (string) $pedido->id],
);

Segurança em APIs para Mobile

Checklist de segurança para APIs mobile

  • Rate limiting por usuário e por IP — use throttle:60,1 como mínimo, ajuste por endpoint sensível.
  • HTTPS obrigatório — nunca permita HTTP em produção. Configure HSTS no servidor.
  • Tokens com TTL — tokens de app mobile devem expirar. Use refresh tokens para renovação silenciosa.
  • Validação rigorosa — use Form Requests com validated(), nunca all() diretamente em fill().
  • Não retornar dados sensíveis — use API Resources para controlar quais campos são expostos. Nunca $user->toArray() diretamente.
  • Certificate pinning — no lado do app, fixe o certificado TLS para resistir a ataques man-in-the-middle.
