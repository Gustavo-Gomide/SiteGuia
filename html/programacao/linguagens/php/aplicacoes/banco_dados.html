PHP com Banco de Dados

PDO seguro, Eloquent ORM, migrations, seeders, transações e read/write splitting

Interagir com bancos de dados é a operação mais comum em aplicações PHP. Existem dois níveis: PDO, a camada de abstração nativa que deve ser usada com prepared statements obrigatórios para prevenir SQL injection, e Eloquent, o ORM do Laravel que mapeia tabelas a classes com sintaxe fluente e suporte completo a relacionamentos.

PDO — PHP Data Objects

PDO é a extensão oficial do PHP para acesso a bancos de dados. Fornece uma interface unificada para MySQL, PostgreSQL, SQLite, SQL Server e outros. A maior vantagem é o suporte a prepared statements, que eliminam completamente a possibilidade de SQL injection quando usados corretamente.

Nunca interpole variáveis diretamente em queries SQL. Use sempre prepared statements com ? ou :nome.

php — PDO seguro com prepared statements
try {
    $pdo = new PDO(
        'mysql:host=localhost;dbname=loja;charset=utf8mb4',
        $_ENV['DB_USER'],
        $_ENV['DB_PASS'],
        [
            PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
            PDO::ATTR_EMULATE_PREPARES   => false, // prepares reais no servidor
        ]
    );
} catch (PDOException $e) {
    // Não exponha detalhes de conexão ao usuário
    error_log($e->getMessage());
    throw new RuntimeException('Erro ao conectar ao banco de dados');
}

// SELECT com prepared statement (seguro contra SQL Injection)
$stmt = $pdo->prepare('SELECT * FROM usuarios WHERE email = :email AND ativo = :ativo');
$stmt->execute(['email' => $email, 'ativo' => true]);
$usuario = $stmt->fetch();

// INSERT com prepared statement
$insert = $pdo->prepare(
    'INSERT INTO produtos (nome, preco, estoque) VALUES (:nome, :preco, :estoque)');
$insert->execute(['nome' => $nome, 'preco' => $preco, 'estoque' => $estoque]);
$novoId = $pdo->lastInsertId();

Transações PDO

Transações garantem que um conjunto de operações seja executado de forma atômica — se uma falha, todas são revertidas.

php — transação PDO
$pdo->beginTransaction();
try {
    // Debita do estoque
    $pdo->prepare('UPDATE produtos SET estoque = estoque - ? WHERE id = ?')
       ->execute([$quantidade, $produtoId]);

    // Cria o pedido
    $pdo->prepare('INSERT INTO pedidos (usuario_id, total) VALUES (?, ?)')
       ->execute([$usuarioId, $total]);

    $pdo->commit();
} catch (PDOException $e) {
    $pdo->rollBack();
    throw $e;
}

Eloquent ORM

Eloquent é o ORM do Laravel. Cada tabela do banco tem um Model correspondente. O Eloquent gera SQL automaticamente e suporta eager loading, scopes, mutators, casts e eventos de modelo.

php — Model Eloquent completo
class Produto extends Model
{
    use HasFactory, SoftDeletes;

    protected $table = 'produtos';    // opcional se seguir convenção

    protected $fillable = ['nome', 'preco', 'estoque', 'categoria_id'];

    protected $casts = [
        'preco'      => 'decimal:2',
        'ativo'      => 'boolean',
        'metadados'  => 'array',       // JSON ↔ PHP array automaticamente
        'criado_em'  => 'datetime',
    ];

    // Relacionamentos
    public function categoria(): BelongsTo
    {
        return $this->belongsTo(Categoria::class);
    }

    public function pedidos(): BelongsToMany
    {
        return $this->belongsToMany(Pedido::class)
                    ->withPivot('quantidade', 'preco_unitario');
    }

    // Local scope reutilizável
    public function scopeAtivos(Builder $q): Builder
    {
        return $q->where('ativo', true)->where('estoque', '>', 0);
    }

    // Mutator (setter)
    public function setNomeAttribute(string $valor): void
    {
        $this->attributes['nome'] = trim(ucwords($valor));
    }
}

Queries com Eloquent

php — queries Eloquent
// Busca básica
$produto = Produto::find(42);
$produto = Produto::findOrFail(42); // lança 404 se não encontrado

// Eager loading — evita N+1 queries
$produtos = Produto::with(['categoria', 'imagens' => fn($q) => $q->limit(3)])
    ->ativos()                         // scope local
    ->where('preco', '<=', 1000)
    ->orderBy('nome')
    ->paginate(20);

// upsert
Produto::updateOrCreate(
    ['sku' => 'ABC123'],
    ['nome' => 'Produto X', 'preco' => 99.90]
);

Migrations e Seeders

Migrations versionam o schema do banco de dados como código-fonte. Cada migration é uma classe com métodos up() e down(), permitindo aplicar e reverter alterações de forma controlada.

php — migration
return new class extends Migration
{
    public function up(): void
    {
        Schema::create('produtos', function (Blueprint $table) {
            $table->id();
            $table->string('nome', 120);
            $table->string('sku', 50)->unique();
            $table->decimal('preco', 10, 2);
            $table->unsignedInteger('estoque')->default(0);
            $table->boolean('ativo')->default(true);
            $table->foreignId('categoria_id')->constrained()->cascadeOnDelete();
            $table->timestamps();
            $table->softDeletes();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('produtos');
    }
};

Read/Write Splitting e Connection Pooling

Em aplicações de alta carga, é comum usar uma instância de banco de dados para escrita (primária) e uma ou mais réplicas de leitura. Laravel suporta isso nativamente.

php — config/database.php (read/write split)
'mysql' => [
    'read' => [
        'host' => ['192.168.1.2', '192.168.1.3'], // réplicas
    ],
    'write' => [
        'host' => ['192.168.1.1'], // primária
    ],
    'sticky'   => true, // usa primária pra leitura na mesma request pós-write
    'driver'   => 'mysql',
    'database' => env('DB_DATABASE'),
    /* ... */
],
