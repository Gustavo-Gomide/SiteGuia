PHP — Terminal e CLI

PHP no Terminal e CLI

Scripts PHP-CLI, Symfony Console, cron, deploy e otimizações para execuções longas

PHP não vive apenas no servidor web. Via php-cli pode ser usado para scripts de automação, ferramentas administrativas, pipelines de ETL, migrações de dados e muito mais — com total acesso ao sistema de arquivos, stdin/stdout e execução de processos. O Symfony Console eleva isso ao nível profissional com suporte a subcomandos, flags, barras de progresso e formatação colorida.

PHP-CLI Básico — argv, stdin e stdout

Quando executado via linha de comando, o PHP recebe os argumentos em $argv (array) e $argc (contagem). A entrada padrão é acessível via fgets(STDIN) e a saída via echo ou fwrite(STDOUT, ...).

php — script CLI: saudacao.php
#!/usr/bin/env php
<?php

// $argv[0] = nome do script, $argv[1..n] = argumentos
if ($argc < 2) {
    fprintf(STDERR, "Uso: %s <nome>\n", $argv[0]);
    exit(1); // código de saída não-zero = erro
}

$nome = $argv[1];
echo "Olá, {$nome}!\n";

// Leitura interativa do stdin
echo "Digite sua cidade: ";
$cidade = trim(fgets(STDIN));
echo "Você é de {$cidade}.\n";
bash — executar o script
chmod +x saudacao.php
./saudacao.php Maria
# ou
php saudacao.php Maria

Symfony Console — CLI Profissional

O componente Symfony Console é o padrão ouro para CLIs em PHP. Usado internamente pelo Laravel (Artisan), Composer e dezenas de ferramentas PHP. Oferece subcomandos nomeados, argumentos e opções tipadas, barra de progresso, saída colorida e tabelas.

bash — instalar standalone
composer require symfony/console
php — comando com barra de progresso e table
use Symfony\Component\Console\Command\Command;
use Symfony\Component\Console\Input\InputInterface;
use Symfony\Component\Console\Input\InputArgument;
use Symfony\Component\Console\Input\InputOption;
use Symfony\Component\Console\Output\OutputInterface;
use Symfony\Component\Console\Helper\ProgressBar;
use Symfony\Component\Console\Helper\Table;

class ImportarProdutosCommand extends Command
{
    protected static $defaultName = 'produto:importar';

    protected function configure(): void
    {
        $this
            ->setDescription('Importa produtos de um arquivo CSV')
            ->addArgument('arquivo', InputArgument::REQUIRED, 'Caminho para o CSV')
            ->addOption('dry-run', null, InputOption::VALUE_NONE, 'Simular sem salvar')
            ->addOption('batch', 'b', InputOption::VALUE_OPTIONAL, 'Tamanho do lote', 100);
    }

    protected function execute(InputInterface $input, OutputInterface $output): int
    {
        $arquivo = $input->getArgument('arquivo');
        $dryRun  = $input->getOption('dry-run');
        $batch   = (int) $input->getOption('batch');

        if (!file_exists($arquivo)) {
            $output->writeln("<error>Arquivo não encontrado: {$arquivo}</error>");
            return Command::FAILURE;
        }

        $linhas  = count(file($arquivo)) - 1; // descontar header
        $progress = new ProgressBar($output, $linhas);
        $progress->start();

        $resumo = []; $ok = 0; $erros = 0;

        foreach (extrairCsvChunks($arquivo, $batch) as $lote) {
            foreach ($lote as $linha) {
                if (!$dryRun) salvarProduto($linha);
                $ok++;
                $progress->advance();
            }
        }

        $progress->finish();
        $output->writeln('');

        // Tabela de resumo
        (new Table($output))
            ->setHeaders(['Métrica', 'Valor'])
            ->setRows([
                ['Importados', $ok],
                ['Erros',      $erros],
                ['Dry Run',    $dryRun ? 'Sim' : 'Não'],
            ])
            ->render();

        return Command::SUCCESS;
    }
}
php — registrar e executar o comando
use Symfony\Component\Console\Application;

$app = new Application('MinhaFerramenta', '1.0.0');
$app->add(new ImportarProdutosCommand());
$app->run();
bash — usar o comando
php ferramenta.php produto:importar dados.csv --batch=500
php ferramenta.php produto:importar dados.csv --dry-run
php ferramenta.php produto:importar --help

Artisan Commands no Laravel

Laravel usa o Symfony Console internamente. Comandos Artisan customizados são a forma idiomática de escrever CLIs em projetos Laravel — com acesso completo ao container de dependências.

bash — criar novo comando Artisan
php artisan make:command LimparCacheExpiradoCommand
php — comando Artisan com agendamento
class LimparCacheExpiradoCommand extends Command
{
    protected $signature   = 'cache:limpar-expirado {--force : Não pedir confirmação}';
    protected $description = 'Remove entradas de cache expiradas do banco de dados';

    public function handle(CacheService $cache): int
    {
        if (!$this->option('force') && !$this->confirm('Confirmar limpeza?')) {
            $this->info('Operação cancelada.');
            return self::SUCCESS;
        }

        $removidos = $cache->limparExpirados();
        $this->info("{$removidos} entradas removidas.");
        return self::SUCCESS;
    }
}
php — agendamento em app/Console/Kernel.php
protected function schedule(Schedule $schedule): void
{
    $schedule->command('cache:limpar-expirado --force')->daily()->at('03:00');
    $schedule->command('queue:restart')->hourly();
    $schedule->command('backup:run')->weekly()->sundays()->at('01:00');
}

Adicione um único cron job que chama o agendador do Laravel a cada minuto:
* * * * * cd /var/www && php artisan schedule:run >> /dev/null 2>&1

Otimizações para Scripts de Longa Duração

Scripts CLI que processam grandes volumes de dados ou rodam por horas precisam de cuidado especial com consumo de memória e ciclos de garbage collection.

php — processamento em lotes sem estourar memória
// Configurar antes de iniciar
ini_set('memory_limit', '512M');

// Processar em chunks — não carregar tudo para memória
Produto::query()
    ->where('processado', false)
    ->chunkById(200, function(Collection $produtos) {
        foreach ($produtos as $produto) {
            processarProduto($produto);
        }

        // Forçar GC a cada lote para liberar memória de ciclos
        gc_collect_cycles();
    });

// Para processos muito longos (daemons), monitorar memória
if (memory_get_usage(true) > 400 * 1024 * 1024) { // > 400 MB
    error_log('Aviso: consumo de memória alto, reiniciando worker...');
    exit(0); // supervisor/systemd reinicia automaticamente
}

Boas práticas para scripts CLI de longa duração

  • Use chunkById() ou cursor() em vez de get() para grandes queries.
  • Evite acumular modelos Eloquent em arrays — desreferencie (unset()) após processar.
  • Use gc_collect_cycles() periodicamente para liberar referências circulares.
  • Monitore memory_get_peak_usage() no final para ajustar o limit.
  • Para daemons, use Supervisor ou systemd para reiniciar o processo se ele encerrar.
  • Feche conexões PDO/Redis explicitamente se o script não precisar mais delas.
