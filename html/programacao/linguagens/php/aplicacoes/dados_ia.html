PHP — Dados e IA

PHP para Dados e Inteligência Artificial

PHP-ML, integração com Python, FFI para bibliotecas nativas e consumo de APIs de IA

PHP não é a linguagem primária para ciência de dados ou machine learning — Python domina esse domínio. No entanto, PHP tem papel importante como orquestrador: consome APIs de modelos de IA, executa pipelines ETL, integra com serviços Python via chamadas de processo, e usa FFI para bindings nativos. Para ML simples, a biblioteca PHP-ML oferece algoritmos sem dependências externas.

PHP-ML — Machine Learning em PHP Puro

A biblioteca PHP-ML implementa algoritmos clássicos de machine learning em PHP puro, sem dependências externas. Adequada para modelos pequenos, prototipagem e ambientes onde não se pode instalar Python.

php — classificação com PHP-ML
use Phpml\Classification\KNearestNeighbors;
use Phpml\Pipeline;
use Phpml\Preprocessing\Normalizer;

// Dados de treinamento: [feature1, feature2] → label
$samples = [[1, 3], [1, 4], [2, 4], [3, 1], [4, 1], [4, 2]];
$labels  = ['a', 'a', 'a', 'b', 'b', 'b'];

// Pipeline: normalização → classificador KNN
$pipeline = new Pipeline([
    new Normalizer(),
    new KNearestNeighbors(k: 3),
]);

$pipeline->train($samples, $labels);
$predicao = $pipeline->predict([[3, 2]]);
echo $predicao[0]; // 'b'

Integração PHP + Python

Para modelos complexos (redes neurais, análise de grandes volumes de dados), a abordagem correta é delegar ao Python. PHP pode chamar scripts Python via proc_open ou se comunicar via APIs REST internas.

Nunca passe input do usuário diretamente via exec() ou shell_exec(). Use proc_open() com array de argumentos para evitar injeção de comandos.

php — chamada segura ao Python via proc_open
function chamarPython(string $scriptPath, array $payload): array
{
    // Usar array descriptor evita injeção de shell
    $proc = proc_open(
        ['python3', $scriptPath],
        [
            0 => ['pipe', 'r'],  // stdin
            1 => ['pipe', 'w'],  // stdout
            2 => ['pipe', 'w'],  // stderr
        ],
        $pipes
    );

    if (!is_resource($proc)) {
        throw new RuntimeException('Falha ao iniciar processo Python');
    }

    fwrite($pipes[0], json_encode($payload));
    fclose($pipes[0]);

    $saida = stream_get_contents($pipes[1]);
    $erro  = stream_get_contents($pipes[2]);
    fclose($pipes[1]);
    fclose($pipes[2]);
    proc_close($proc);

    if ($erro) error_log("Python error: {$erro}");
    return json_decode($saida, true) ?? [];
}

// Uso
$resultado = chamarPython('/app/scripts/analisar.py', ['texto' => $texto]);

Consumindo a API da OpenAI

A forma mais comum de IA em projetos PHP modernos é consumir APIs de modelos como GPT-4, DALL-E e Embeddings da OpenAI. O cliente oficial para PHP é mantido pela própria OpenAI.

bash — instalar cliente OpenAI
composer require openai-php/client
php — chat completion com caching de resposta
use OpenAI;
use Illuminate\Support\Facades\Cache;

$client = OpenAI::client(config('services.openai.key'));

function resumirTexto(string $texto): string
{
    $cacheKey = 'resumo.' . md5($texto);

    return Cache::remember($cacheKey, 86400, function() use ($texto) {
        global $client;
        $response = $client->chat()->create([
            'model'    => 'gpt-4o-mini',
            'messages' => [
                ['role' => 'system', 'content' => 'Resuma o seguinte texto em 3 linhas em português.'],
                ['role' => 'user',   'content' => $texto],
            ],
            'max_tokens' => 200,
        ]);
        return $response->choices[0]->message->content;
    });
}

PHP como Motor de ETL

PHP é usado em pipelines de ETL por sua disponibilidade em servidores web, facilidade com streams de arquivo e boas bibliotecas para CSV, XML e JSON.

php — pipeline ETL com generator
function extrairCsv(string $path): Generator
{
    $fh = fopen($path, 'r');
    $header = fgetcsv($fh);
    while (($row = fgetcsv($fh)) !== false) {
        yield array_combine($header, $row);
    }
    fclose($fh);
}

function transformar(Generator $linhas): Generator
{
    foreach ($linhas as $linha) {
        yield [
            'nome'  => trim($linha['name']),
            'preco' => (float) str_replace(',', '.', $linha['price']),
            'sku'   => strtoupper(trim($linha['sku'])),
        ];
    }
}

function carregar(Generator $registros, PDO $pdo): int
{
    $stmt = $pdo->prepare('INSERT INTO produtos (nome, preco, sku) VALUES (:nome, :preco, :sku) ON DUPLICATE KEY UPDATE preco = :preco');
    $count = 0;
    foreach ($registros as $r) {
        $stmt->execute($r);
        $count++;
    }
    return $count;
}

// Pipeline completo — nenhuma linha fica inteiramente em memória
$processados = carregar(transformar(extrairCsv('catalogo.csv')), $pdo);

FFI — Foreign Function Interface (PHP 7.4+)

FFI permite chamar funções de bibliotecas C compiladas diretamente de PHP, sem escrever extensões em C. Usado para integrar com bibliotecas de ML como libsvm, liblinear ou qualquer biblioteca nativa.

php — FFI com biblioteca C
// Exemplo: chamar função de biblioteca C de hash rápido
$ffi = FFI::cdef(
    "uint32_t murmur3_32(const char *key, int len, uint32_t seed);",
    "libhash.so"
);

$texto     = "hello world";
$hashValue = $ffi->murmur3_32($texto, strlen($texto), 42);
echo sprintf("Hash: %u\n", $hashValue);

FFI exige que a extensão esteja habilitada em php.ini (ffi.enable=true). Em produção, use com cautela — erros em código C podem derrubar o processo PHP.
