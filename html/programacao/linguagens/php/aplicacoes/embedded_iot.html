PHP para Embedded e IoT

PHP-CLI em Raspberry Pi, GPIO seguro, automação de hardware e gateway MQTT+HTTP

PHP não foi projetado para sistemas embarcados — mas pode rodar em dispositivos Linux como Raspberry Pi via PHP-CLI, controlando hardware, executando agendamentos e servindo como gateway HTTP/MQTT. Compreender onde PHP é viável (e onde não é) evita erros de arquitetura.

Limitações Fundamentais do PHP em IoT

PHP não possui acesso nativo a registros de hardware, não é tempo-real, não compila para microcontroladores (Arduino, ESP32) e tem overhead de memória e inicialização inadequado para sistemas com < 512 MB de RAM.

Tarefa IoT PHP Alternativa Ideal Observação
Firmware de microcontrolador Impossível C, C++, MicroPython PHP não compila para bare metal
Controle GPIO no Raspberry Pi Possível via exec Python (RPi.GPIO) Latência ~10–100 ms
Servidor HTTP para dispositivos IoT Adequado Node.js, Go PHP-FPM + Nginx funciona bem
Gateway MQTT → HTTP → banco Adequado Node.js Usando Mosquitto + PHP scripts
Dashboard de monitoramento Adequado Laravel, Livewire Excelente para dashboards web

PHP-CLI no Raspberry Pi

Qualquer Raspberry Pi rodando Raspberry Pi OS (Debian-based) pode executar PHP CLI. É uma boa escolha para scripts de coleta de dados, automações agendadas via cron e backends web leves.

bash — instalar PHP no Raspberry Pi OS
# Raspberry Pi OS (Bookworm)
sudo apt update && sudo apt install -y php-cli php-mbstring php-json

# Verificar
php --version
php -r "echo PHP_INT_SIZE === 8 ? 'ARM64 OK' : 'ARM32';"
php — ler sensor DHT22 via Python bridging
function lerSensorDHT22(int $pino): array
{
    // Python lida com GPIO; PHP processa o resultado — sem injeção de shell
    $proc = proc_open(
        ['python3', '/opt/sensors/dht22_read.py', (string)$pino],
        [1 => ['pipe', 'w'], 2 => ['pipe', 'w']],
        $pipes
    );
    $saida = stream_get_contents($pipes[1]);
    fclose($pipes[1]); fclose($pipes[2]);
    proc_close($proc);

    $dados = json_decode($saida, true);
    if (!isset($dados['temperatura'], $dados['umidade'])) {
        throw new RuntimeException('Leitura do sensor falhou');
    }
    return $dados;
}

$leitura = lerSensorDHT22(4); // GPIO pino 4
printf("Temp: %.1f°C  Umidade: %.1f%%\n", $leitura['temperatura'], $leitura['umidade']);

PHP como Gateway MQTT + HTTP

Uma arquitetura comum em IoT é: sensores publicam mensagens no broker MQTT (Mosquitto), e um serviço PHP consome essas mensagens, persiste no banco e expõe via API REST para dashboards e aplicativos.

bash — instalar phpMQTT
composer require bluerhinos/phpmqtt
php — script MQTT subscriber (daemon CLI)
require 'vendor/autoload.php';

$mqtt = new Bluerhinos\phpMQTT('localhost', 1883, 'php-gateway-' . uniqid());

if (!$mqtt->connect()) exit(1);

$topicos = [
    'sensores/+/temperatura' => ['qos' => 1, 'function' => function($topic, $msg) {
        $partes = explode('/', $topic);
        $sensor = $partes[1];
        $dados  = json_decode($msg, true);

        // persistir no banco de dados
        persistirLeitura($sensor, $dados['valor'], 'temperatura');
        printf("[%s] Sensor %s: %.1f°C\n", date('H:i:s'), $sensor, $dados['valor']);
    }],
];

$mqtt->subscribe($topicos, 0);

// Loop de leitura contínua
while ($mqtt->proc()) {
    usleep(100000); // 100ms
}

Automação com Cron Jobs

PHP-CLI é adequado para tarefas agendadas em Raspberry Pi — coleta de dados de sensores, relatórios, alertas por email ou Telegram.

bash — crontab para script PHP
# Coletar temperatura a cada 5 minutos
*/5 * * * * /usr/bin/php /opt/iot/coletar_temperatura.php >> /var/log/iot/temperatura.log 2>&1

# Relatório diário às 08h
0 8 * * * /usr/bin/php /opt/iot/relatorio_diario.php
