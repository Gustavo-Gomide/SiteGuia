PHP para GUI e Desktop

PHP-GTK, NativePHP, Electron + PHP backend — e quando preferir outra linguagem

PHP não é a escolha convencional para interfaces gráficas nativas, mas existem caminhos viáveis: PHP-GTK para Linux/Windows via GTK2, NativePHP que empacota aplicações Laravel em janelas de desktop e Electron + PHP como backend de uma interface web local. Cada abordagem tem trade-offs significativos.

Panorama: Opções de GUI com PHP

Abordagem Tecnologia Plataformas Maturidade Tamanho do Pacote

PHP-GTK — Uma Visão Histórica

PHP-GTK foi uma extensão criada por Andrei Zmievski em 2001 que permitia criar janelas nativas usando GTK2. Nunca foi compatível com PHP 7+ de forma estável e hoje é considerado historicamente interessante, mas não recomendado para novos projetos.

php — janela "Hello World" com PHP-GTK2 (histórico)
// PHP-GTK2 — requer extensão gtk2 instalada (PHP 5.x)
$window = new GtkWindow();
$window->set_title('Hello PHP-GTK');
$window->set_default_size(300, 100);

$label = new GtkLabel('Olá, mundo!');
$window->add($label);
$window->connect_simple('destroy', ['Gtk', 'main_quit']);
$window->show_all();

Gtk::main();

PHP-GTK não tem suporte a PHP 8. Não use para novos projetos. Para interfaces nativas modernas em PHP, use NativePHP.

NativePHP — Desktop com Laravel

NativePHP é um projeto de código aberto criado por Simon Hamp (com suporte da equipe Laravel) que permite empacotar aplicações Laravel como apps desktop usando Electron ou Tauri como shell. A app Laravel roda localmente no computador do usuário, servida por um servidor embutido.

bash — criar projeto NativePHP com Laravel
composer create-project laravel/laravel meu-app
cd meu-app
composer require nativephp/electron
php artisan native:install
php artisan native:serve   # inicia em modo dev com janela Electron
php — configuração da janela principal (NativePHP)
use Native\Laravel\Facades\Window;

class NativeAppServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        Window::open()
            ->width(1200)
            ->height(800)
            ->minWidth(600)
            ->title('Meu App Desktop')
            ->url(route('dashboard'));
    }
}

O que NativePHP oferece

Sistema de arquivos

Acesso a pastas do usuário (Downloads, Documentos, Desktop) via API de PHP.

Notificações nativas

Notificações do sistema operacional (macOS, Windows, Linux).

Menus nativos

Configuração de menus da barra de títulos e ícone na bandeja de sistema.

Auto-update

Distribuição de atualizações automaticamente via GitHub Releases.

Quando NÃO Usar PHP para Desktop

Avaliação honesta: para apps desktop de alto desempenho, jogos, ferramentas de baixo nível ou apps que precisam integração profunda com o SO, existem alternativas muito melhores.

Prefira outro para...

  • Apps de alta performance → Rust, C++
  • Apps cross-platform modernas → Flutter, .NET MAUI
  • Ferramentas de sistema → Go, Rust
  • Jogos nativos → C++ (Unreal), C# (Unity)
  • Apps com acesso a câmera/sensores avançados → Swift, Kotlin

PHP é razoável para...

  • Ferramentas administrativas internas
  • Apps que são basicamente um site local
  • Projects onde o time só conhece PHP/Laravel
  • Prototipagem rápida
  • Apps com lógica de negócio web reutilizável
