PHP para Desenvolvimento Web

O domínio natural do PHP: Laravel full-stack, REST APIs, autenticação, filas e cache com Redis

Web é o habitat natural do PHP. Com Laravel como framework principal, é possível construir desde um blog simples até uma plataforma SaaS com autenticação OAuth2, autorização granular com Policies, filas de tarefas assíncronas, broadcasting de eventos em tempo real e APIs consumidas por SPAs e aplicativos móveis — tudo com a mesma linguagem e o mesmo framework.

Rotas e Middleware

O sistema de roteamento do Laravel é expressivo e suporta nomeação, agrupamento, rate limiting e middleware em cadeia. Middleware são camadas que filtram e transformam requisições HTTP antes de chegarem ao controller.

php — routes/api.php
use App\Http\Controllers\Api\ProdutoController;
use App\Http\Controllers\Api\AuthController;

// Rotas públicas
Route::post('/login', [AuthController::class, 'login']);
Route::post('/register', [AuthController::class, 'register']);

// Grupo protegido com Sanctum + rate limiting
Route::middleware(['auth:sanctum', 'throttle:60,1'])->group(function() {
    Route::apiResource('produtos', ProdutoController::class);
    Route::get('/perfil', fn() => auth()->user());
});
php — middleware customizado
class EnsureAccountActive
{
    public function handle(Request $req, Closure $next): Response
    {
        if (auth()->check() && !auth()->user()->ativo) {
            return response()->json(['error' => 'Conta suspensa'], 403);
        }
        return $next($req);
    }
}

Autenticação com Laravel Sanctum

Laravel Sanctum oferece dois modos: token-based para SPAs e mobile apps, e session-based via cookie para SPAs no mesmo domínio. Tokens são hasheados no banco de dados e podem ter habilidades (escopos) específicas.

php — AuthController com Sanctum
class AuthController extends Controller
{
    public function login(LoginRequest $request): JsonResponse
    {
        $request->authenticate(); // Form Request valida credenciais

        $user  = $request->user();
        $token = $user->createToken('app-token', ['produtos:ler', 'pedidos:criar']);

        return response()->json([
            'token' => $token->plainTextToken,
            'user'  => new UserResource($user),
        ]);
    }

    public function logout(Request $request): JsonResponse
    {
        $request->user()->currentAccessToken()->delete();
        return response()->json(['message' => 'Desconectado']);
    }
}

Autorização com Gates e Policies

php — ProdutoPolicy completa
class ProdutoPolicy
{
    // before() intercepta antes de qualquer método — admin bypassa tudo
    public function before(User $user): ?bool
    {
        return $user->hasRole('admin') ? true : null;
    }

    public function viewAny(User $user): bool { return true; }

    public function update(User $user, Produto $produto): bool
    {
        return $user->id === $produto->user_id;
    }

    public function delete(User $user, Produto $produto): bool
    {
        return $user->id === $produto->user_id;
    }
}

// No Controller — automaticamente lança AuthorizationException
public function update(Request $req, Produto $produto): JsonResponse
{
    $this->authorize('update', $produto);
    $produto->update($req->validated());
    return response()->json(new ProdutoResource($produto));
}

Jobs e Filas Assíncronas

Filas permitem processar tarefas lentas (envio de email, processamento de imagens, chamadas a APIs externas) de forma assíncrona, sem bloquear a resposta HTTP.

php — Job e despacho
class ProcessarPedidoJob implements ShouldQueue
{
    use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

    public int $tries = 3;           // máx. tentativas
    public int $backoff = 60;         // segundos entre tentativas

    public function __construct(public readonly Pedido $pedido) {}

    public function handle(PagamentoService $pagamento): void
    {
        $pagamento->processar($this->pedido);
        $this->pedido->marcarComoConcluido();
    }

    public function failed(Throwable $e): void
    {
        $this->pedido->marcarComoFalho($e->getMessage());
    }
}

// Despacho na fila 'alta-prioridade' com delay
ProcessarPedidoJob::dispatch($pedido)
    ->onQueue('alta-prioridade')
    ->delay(now()->addSeconds(10));

Cache com Redis

php — cache com Laravel + Redis
use Illuminate\Support\Facades\Cache;

// Lembrar resultado de query por 30 minutos
$produtos = Cache::remember('produtos.ativos', 1800, function() {
    return Produto::ativos()->with('categoria')->get();
});

// Cache por usuário (cache tag)
$perfil = Cache::tags(['usuario', "usuario:{$userId}"])
    ->remember("perfil.{$userId}", 3600, fn() => User::findOrFail($userId));

// Invalidar cache de um usuário específico
Cache::tags(["usuario:{$userId}"])->flush();

// Locks distribuídos para evitar race conditions
$lock = Cache::lock("pedido:{$id}", 10);
if ($lock->get()) {
    try { processarPedido(); }
    finally { $lock->release(); }
}

API Resources e CORS

php — ProdutoResource
class ProdutoResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id'       => $this->id,
            'nome'     => $this->nome,
            'preco'    => (float) $this->preco,
            'ativo'    => (bool) $this->ativo,
            'categoria' => new CategoriaResource($this->whenLoaded('categoria')),
            'criado_em' => $this->created_at?->toIso8601String(),
            // Mostrar somente para admins
            'custo'    => $this->when(auth()->user()?->isAdmin(), $this->custo),
        ];
    }
}
