PHP — Introdução

PHP

A linguagem que construiu a web — do blog pessoal às plataformas que movem bilhões de usuários

PHP (PHP: Hypertext Preprocessor) é a linguagem de script server-side mais difundida da web, presente em mais de 75% dos servidores com linguagem detectável. Criada em 1994 por Rasmus Lerdorf como um conjunto de scripts CGI, evoluiu para uma linguagem madura, tipada e de alta performance — da versão 8.x com compilação JIT à programação assíncrona com Fibers.

História

A trajetória do PHP é um dos casos mais relevantes de evolução orgânica em linguagens de programação. Tudo começou com uma necessidade prática: Rasmus Lerdorf precisava rastrear quantas pessoas liam seu currículo online.

PHP/FI — 1994–1997

Rasmus Lerdorf criou um conjunto de scripts em C denominado Personal Home Page Tools (PHP Tools) em 1994. Em 1995, foi lançado o PHP/FI (Form Interpreter), que permitia processar formulários HTML e conectar-se a bancos de dados. Era um conjunto de macros CGI, não uma linguagem completa.

PHP 3 — 1998

Andi Gutmans e Zeev Suraski reescreveram o interpretador do zero. O PHP 3 introduziu uma gramática mais consistente, suporte a múltiplos bancos de dados e extensibilidade via módulos. Foi quando o idioma deixou de ser um conjunto de macros e se tornou uma linguagem de programação de fato. O acrônimo foi redefinido como PHP: Hypertext Preprocessor.

PHP 4 — 2000

Introduziu o Zend Engine 1.0, motor que acelerou execução drasticamente. Suporte a sessões ($_SESSION), buffers de saída e referências. Foi o primeiro PHP amplamente adotado comercialmente — impulsionou WordPress, Drupal e Joomla.

PHP 5 — 2004

Revolução orientada a objetos: Zend Engine 2, modelo OOP completo (visibilidade, interfaces, classes abstratas, construtores uniformes), PDO para bancos de dados, SPL, SimpleXML e exceções. Esta versão estabeleceu o PHP como plataforma séria para aplicações empresariais.

PHP 7 — 2015

Salto de desempenho de 2× a 3× em relação ao PHP 5. Introduziu scalar type hints, return types, o operador spaceship (<=>), null coalescing (??), tipos de retorno anuláveis, classes anônimas e declarações de tipo escalares. PHP 6 foi abandonado por problemas com Unicode; o próximo major foi diretamente o 7.

PHP 8 — 2020–presente

A versão mais avançada: JIT compiler, named arguments, match expression, nullsafe operator (?->), atributos nativos (#[Attribute]), union types, constructor property promotion. PHP 8.1 adicionou Enums, Fibers, readonly properties e tipos de interseção. PHP 8.2 trouxe readonly classes e PHP 8.3 consolidou o sistema de tipos.

Características da Linguagem

PHP possui um conjunto de características que o tornaram ideal para desenvolvimento web server-side. Entendê-las é fundamental para usá-lo bem.

Interpretada e compilada para opcodes

PHP é executado por um interpretador (o Zend Engine), que compila o código-fonte para opcodes (bytecode) em memória a cada requisição. Com OPcache ativado, esse bytecode é cacheado em memória compartilhada e reutilizado entre requisições — eliminando a fase de compilação repetida.

Tipagem dinâmica com declaração opcional

PHP usa tipagem dinâmica por padrão: o tipo de uma variável é determinado em tempo de execução. A partir do PHP 7, é possível declarar tipos em parâmetros, propriedades e retornos. Com declare(strict_types=1), conversões implícitas são proibidas, transformando-o em fortemente tipado dentro do arquivo.

Orientado à web por design

PHP tem suporte nativo a HTTP: superglobais como $_GET, $_POST, $_SESSION e $_COOKIE estão disponíveis sem importação. O modelo de execução padrão (shared-nothing) cria um processo isolado por requisição, simplificando o raciocínio sobre estado.

Multiparadigma

PHP suporta programação procedural, orientada a objetos e elementos de programação funcional (funções de primeira classe, closures, funções de array de alta ordem). Permite misturar paradigmas conforme a necessidade do contexto.

Ecossistema vasto

Packagist, o repositório oficial de pacotes PHP, contém mais de 400.000 pacotes. O gerenciador Composer resolve dependências, faz autoloading PSR-4 e executa scripts de ciclo de vida. Frameworks como Laravel, Symfony e WordPress dominam segmentos de mercado distintos.

Modelo de execução compartilhado-nada

Cada requisição HTTP inicia um processo/thread PHP do zero (sem estado compartilhado entre requisições por padrão). Isso simplifica concorrência mas exige estratégias externas (Redis, banco de dados) para persistência de estado entre requisições. Servidores como Swoole e RoadRunner alternam para modelo de workers persistentes.

PHP 8.x — Novidades Principais

O PHP 8 é a versão mais expressiva da linguagem. Abaixo estão os recursos mais impactantes com exemplos de código.

JIT — Just-In-Time Compilation

O JIT compila hot paths do bytecode de opcodes para código nativo de máquina durante a execução, eliminando a interpretação repetida. Mais impactante em código CPU-intensivo (processamento matemático, geração de imagens), menos perceptível em aplicações web com muitas I/O.

php.ini — ativar JIT
; Ativar OPcache (pré-requisito do JIT)
opcache.enable=1
opcache.jit_buffer_size=100M
; tracing = JIT baseado em rastreamento de execução (mais agressivo)
opcache.jit=tracing

Named Arguments

Permite passar argumentos pelo nome, tornando chamadas com muitos parâmetros legíveis e independentes da ordem dos parâmetros opcionais.

php — named arguments (PHP 8.0)
// Antes: ordem importava, parâmetros intermediários obrigatórios
array_slice($array, 0, 10, true);

// Agora: o último argumento pode ser nomeado sem precisar preencher os anteriores
array_slice($array, offset: 0, length: 10, preserve_keys: true);

Match Expression

Similar ao switch, mas sem fall-through, com comparação estrita (===) e retornando um valor diretamente.

php — match expression (PHP 8.0)
$status = 404;

$mensagem = match($status) {
    200, 201 => 'Sucesso',
    301, 302 => 'Redirecionamento',
    404      => 'Não encontrado',
    500      => 'Erro interno',
    default  => 'Status desconhecido',
};

echo $mensagem; // Não encontrado

Enums (PHP 8.1)

Enumerações nativas com suporte a backed enums (string ou int), implementação de interfaces e métodos.

php — enums (PHP 8.1)
enum Status: string
{
    case Ativo   = 'ativo';
    case Inativo = 'inativo';
    case Suspenso= 'suspenso';

    public function label(): string
    {
        return match($this) {
            Status::Ativo    => 'Ativo e operando',
            Status::Inativo  => 'Desativado',
            Status::Suspenso => 'Suspenso por violação',
        };
    }
}

$s = Status::Ativo;
echo $s->label(); // Ativo e operando
echo $s->value;   // ativo

Fibers (PHP 8.1)

Fibers são corrotinas de baixo nível que permitem pausar e retomar a execução de funções, base para async/await em nível de framework.

php — fibers (PHP 8.1)
$fiber = new Fiber(function(): void {
    $valor = Fiber::suspend('primeiro yield');
    echo "Retomado com: {$valor}\n";
});

$resultado = $fiber->start();         // 'primeiro yield'
$fiber->resume('dados externos');    // Retomado com: dados externos

Configurando o Ambiente

Existem diversas formas de executar PHP localmente. As mais comuns para desenvolvimento são descritas abaixo.

Instalar o PHP

bash — instalação
# Debian/Ubuntu
sudo apt install php8.3 php8.3-cli php8.3-mbstring php8.3-xml php8.3-curl

# macOS com Homebrew
brew install php

# Windows — recomendado via XAMPP ou scoop
scoop install php

Servidor embutido (php -S)

Para desenvolvimento rápido, PHP tem um servidor HTTP embutido que dispensa Apache/Nginx:

bash — servidor embutido
# Serve o diretório atual na porta 8000
php -S localhost:8000

# Definir document root específico
php -S localhost:8000 -t public/

# Verificar a versão instalada
php --version

# Ver extensões carregadas
php -m

O servidor embutido é single-threaded e não deve ser usado em produção. Ele serve apenas uma requisição por vez.

phpinfo()

A função phpinfo() gera uma página HTML com todas as informações de configuração do PHP instalado: versão, extensões, diretivas do php.ini e variáveis de ambiente.

php — info.php
<?php
phpinfo();

Nunca exponha phpinfo() em produção. Ele revela caminhos internos, versões e dados sensíveis do servidor.

Primeiros Programas

PHP pode ser executado como script web (embutido em HTML) ou como script CLI puro. Ambos os modos têm sintaxe idêntica — a diferença está na saída.

hello.php
<?php
// Tags de abertura obrigatórias em arquivos puramente PHP
echo "Hello, World!\n";

// Alternativa com print (retorna 1, portanto é uma expressão)
print("Hello again!\n");
pagina.php — PHP embutido em HTML
<!DOCTYPE html>
<html>
<body>
<?php
    $nome = 'Mundo';
    echo "<h1>Olá, {$nome}!</h1>";
?>
</body>
</html>

A convenção moderna é manter PHP completamente separado do HTML, usando frameworks ou templates (Blade, Twig) que separam lógica de apresentação.
