PHP — Bibliotecas e Frameworks

Bibliotecas e Frameworks PHP

Laravel, Symfony, Slim, WordPress, Composer e PHPUnit — o ecossistema que torna PHP produtivo

PHP tem um dos ecossistemas mais ricos de frameworks e bibliotecas do mundo server-side. Laravel lidera o desenvolvimento web moderno, Symfony fornece componentes prontos para produção, Slim é a escolha para microserviços e APIs, WordPress domina 43% da web, e Composer une tudo com gerenciamento de dependências elegante.

Composer — Gerenciador de Dependências

Composer é o gerenciador de dependências padrão do PHP. Ele resolve dependências, gera autoloading PSR-4/PSR-0 e executa scripts de ciclo de vida. Sem Composer, desenvolvimento PHP moderno seria impraticável.

bash — instalação e uso básico
# Instalar globalmente (Linux/macOS)
curl -sS https://getcomposer.org/installer | php
mv composer.phar /usr/local/bin/composer

# Criar novo projeto
composer init

# Adicionar dependência
composer require monolog/monolog

# Dependência de desenvolvimento
composer require --dev phpunit/phpunit

# Instalar de composer.lock
composer install

# Atualizar dependências
composer update
json — composer.json completo
{
    "name": "minha-empresa/meu-app",
    "description": "Aplicação PHP moderna",
    "require": {
        "php": "^8.2",

        "monolog/monolog": "^3.0"

            "@php artisan key:generate",
            "@php artisan migrate --force"
        ]
    }
}

Laravel

Laravel é o framework PHP mais popular do mundo. Segue o padrão MVC, inclui ORM (Eloquent), sistema de templates (Blade), CLI (Artisan), filas, broadcasting, autenticação, autorização e muito mais — tudo integrado e com convenções que minimizam configuração.

Rotas e Controllers

php — routes/web.php
use App\Http\Controllers\ProdutoController;

Route::get('/produtos', [ProdutoController::class, 'index']);
Route::post('/produtos', [ProdutoController::class, 'store']);
Route::get('/produtos/{produto}', [ProdutoController::class, 'show']);

// Route Model Binding — injeta o model automaticamente
Route::delete('/produtos/{produto}', [ProdutoController::class, 'destroy'])
    ->middleware('auth');

Eloquent ORM

php — model Eloquent
class Produto extends Model
{
    protected $fillable = ['nome', 'preco', 'estoque'];
    protected $casts = [
        'preco'   => 'decimal:2',
        'ativo'   => 'boolean',
    ];

    public function categoria(): BelongsTo
    {
        return $this->belongsTo(Categoria::class);
    }

    // Local scope
    public function scopeAtivos(Builder $query): Builder
    {
        return $query->where('ativo', true);
    }
}

Gates e Policies — Autorização

php — Policy
class ProdutoPolicy
{
    // Usuário só pode editar seus próprios produtos
    public function update(User $user, Produto $produto): bool
    {
        return $user->id === $produto->user_id;
    }

    // Admin pode deletar qualquer produto
    public function delete(User $user, Produto $produto): bool
    {
        return $user->hasRole('admin');
    }
}

// Uso no Controller
$this->authorize('update', $produto); // lança 403 se não autorizado

Symfony

Symfony é um conjunto de componentes PHP reutilizáveis e um framework web full-stack. Enquanto Laravel prioriza produtividade com convenções, Symfony prioriza flexibilidade e arquitetura explícita. É amplamente adotado em sistemas corporativos e fintech.

Componentes Independentes

Os componentes Symfony podem ser usados isoladamente — o próprio Laravel usa vários deles:

  • symfony/console — CLI commands
  • symfony/http-foundation — Request/Response
  • symfony/routing — roteamento
  • symfony/event-dispatcher — sistema de eventos
  • symfony/validator — validação
  • symfony/serializer — serialização/normalização
  • symfony/cache — PSR-6/PSR-16

Dependency Injection Container

O DI Container do Symfony usa autowiring por tipo para resolver dependências automaticamente.

yaml — services.yaml
services:
    _defaults:
        autowire: true
        autoconfigure: true

    App\:
        resource: '../src/'
        exclude: '../src/Entity/*'

Slim Framework — Microframework PSR-7/15

Slim é um microframework minimalista que implementa interfaces PSR-7 (HTTP messages) e PSR-15 (middleware). Ideal para APIs REST leves e microserviços onde um framework completo seria overengineering.

php — API com Slim 4
use Slim\Factory\AppFactory;
use Psr\Http\Message\ResponseInterface as Response;
use Psr\Http\Message\ServerRequestInterface as Request;

$app = AppFactory::create();
$app->addRoutingMiddleware();
$app->addErrorMiddleware(true, true, true);

$app->get('/usuarios/{id}', function (Request $req, Response $res, array $args): Response {
    $data = ['id' => (int) $args['id'], 'nome' => 'Alice'];
    $res->getBody()->write(json_encode($data));
    return $res->withHeader('Content-Type', 'application/json');
});

$app->run();

WordPress como Plataforma PHP

WordPress é mais que um CMS — é uma plataforma de desenvolvimento PHP com uma API de plugins e temas extremamente madura. Compreende 43% de todos os sites da web. Desenvolvedor PHP que conhece WordPress internals pode construir produtos para um mercado enorme.

Hooks — Actions e Filters

php — plugin WordPress
// Action: executado em um ponto específico
add_action('wp_head', function() {
    echo '<meta name="my-plugin" content="1">';
});

// Filter: modifica um valor
add_filter('the_title', function(string $title): string {
    return strtoupper($title);
});

WP_Query e REST API

php — WP_Query
$query = new WP_Query([
    'post_type'      => 'produto',
    'posts_per_page' => 10,
    'meta_query'     => [[
        'key'     => 'preco',
        'value'   => 100,
        'compare' => '>=',
        'type'    => 'NUMERIC',
    ]],
]);
while ($query->have_posts()) {
    $query->the_post();
    the_title('<h2>', '</h2>');
}

PHPUnit — Testes Unitários

PHPUnit é o framework de testes padrão do PHP. Segue o padrão xUnit e integra com Composer, CI/CD e ferramentas de análise estática.

php — teste unitário com PHPUnit
use PHPUnit\Framework\TestCase;

class CalculadoraTest extends TestCase
{
    private Calculadora $calc;

    protected function setUp(): void
    {
        $this->calc = new Calculadora();
    }

    public function testSomar(): void
    {
        $this->assertSame(5, $this->calc->somar(2, 3));
    }

    public function testDivisaoPorZero(): void
    {
        $this->expectException(DivisionByZeroError::class);
        $this->calc->dividir(10, 0);
    }

    /** @dataProvider provedorMultiplicacao */
    public function testMultiplicar(int $a, int $b, int $esperado): void
    {
        $this->assertSame($esperado, $this->calc->multiplicar($a, $b));
    }

    public static function provedorMultiplicacao(): array
    {
        return [[2, 3, 6], [0, 5, 0], [-2, 4, -8]];
    }
}
