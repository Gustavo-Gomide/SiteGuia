Orientação a Objetos Completa

php — class, herança, visibilidade
abstract class Animal
{
    public function __construct(
        protected readonly string $nome,  // constructor promotion + readonly (PHP 8.1)
        private int $idade,
    ) {}

    abstract public function som(): string;

    public function apresentar(): string
    {
        return "{$this->nome} diz: {$this->som()}";
    }
}

class Cachorro extends Animal
{
    public function som(): string { return 'Au au!'; }
}

$d = new Cachorro('Rex', 3);
echo $d->apresentar(); // Rex diz: Au au!

Traits — Composição de Comportamento

Traits resolvem o problema da herança múltipla sem o diamante de herança. São unidades de comportamento reutilizáveis que podem ser inseridas em qualquer classe.

php — trait
trait Timestamps
{
    private ?DateTime $criadoEm = null;
    private ?DateTime $atualizadoEm = null;

    public function tocarTimestamp(): void
    {
        $this->atualizadoEm = new DateTime();
        $this->criadoEm ??= $this->atualizadoEm;
    }
}

class Produto
{
    use Timestamps;

    public function __construct(public string $nome) {
        $this->tocarTimestamp();
    }
}

Namespaces e Autoloading (PSR-4)

Namespaces evitam colisão de nomes e organizam o código em hierarquias lógicas. O PSR-4 mapeia namespaces a caminhos de diretório e é o padrão usado pelo Composer.

php — namespaces
<?php
namespace App\Services;

use App\Models\Usuario;
use App\Repositories\UsuarioRepository;
use InvalidArgumentException; // namespace global

class AutenticacaoService
{
    public function __construct(
        private readonly UsuarioRepository $repo
    ) {}

    public function autenticar(string $email, string $senha): Usuario
    {
        $usuario = $this->repo->findByEmail($email)
            ?? throw new InvalidArgumentException('Usuário não encontrado');

        if (!password_verify($senha, $usuario->senha_hash)) {
            throw new InvalidArgumentException('Senha incorreta');
        }

        return $usuario;
    }
}
composer.json — PSR-4 autoloading
{
    "autoload": {
        "psr-4": {
            "App\\": "src/"
        }
    }
}

Com essa configuração, App\Services\AutenticacaoService mapeia automaticamente para src/Services/AutenticacaoService.php.

Generators e yield

Generators permitem criar sequências de valores de forma lazy (sob demanda), sem precisar construir todo o array em memória. São funções que usam yield para pausar e retornar um valor, retomando na próxima chamada a next().

php — generator para leitura de arquivo grande
function lerLinhas(string $arquivo): Generator
{
    $fh = fopen($arquivo, 'r');
    try {
        while (($linha = fgets($fh)) !== false) {
            yield trim($linha); // pausa e retorna a linha
        }
    } finally {
        fclose($fh);
    }
}

// Consumo: O arquivo inteiro NUNCA é carregado em memória
foreach (lerLinhas('dados.csv') as $linha) {
    processarLinha($linha);
}

// yield from — delegar para outro generator
function concatenar(Generator ...$gens): Generator
{
    foreach ($gens as $gen) {
        yield from $gen;
    }
}

Fibers — Concorrência Cooperativa (PHP 8.1)

Fibers são corrotinas de baixo nível. Ao contrário de generators, Fibers podem ser suspensas de qualquer ponto dentro de sua pilha de chamadas — não apenas no nível superior. Isso as torna a primitiva sobre a qual bibliotecas assíncronas (ReactPHP, Amp v3) constroem seus loops de eventos.

php — fibers detalhado
$fiber = new Fiber(function(): string {
    echo "[fiber] iniciou\n";
    $dados = Fiber::suspend('aguardando dados externos');
    echo "[fiber] recebeu: {$dados}\n";
    return "processamento concluído";
});

$suspenso = $fiber->start();     // [fiber] iniciou → 'aguardando dados externos'
echo "[main] fiber suspensa com: {$suspenso}\n";
$retorno = $fiber->resume('payload JSON'); // [fiber] recebeu: payload JSON
echo "[main] fiber retornou: {$fiber->getReturn()}\n";

Fibers não são threads. Elas rodam no mesmo thread do PHP. A concorrência é cooperativa: apenas uma Fiber executa por vez, e a troca ocorre nos pontos de suspend.

Tratamento de Erros e Exceções

php — try/catch/finally e hierarquia de exceções
// Hierarquia: Throwable < Error | Exception
//   Exception < RuntimeException, InvalidArgumentException, etc.
//   Error < TypeError, ParseError, ArithmeticError, etc.

function dividir(float $a, float $b): float
{
    if ($b === 0.0) {
        throw new DivisionByZeroError('Divisão por zero não permitida');
    }
    return $a / $b;
}

try {
    $resultado = dividir(10, 0);
} catch (DivisionByZeroError $e) {
    echo "Erro: " . $e->getMessage() . "\n";
} catch (Throwable $e) {
    // captura qualquer Error ou Exception
    error_log($e->getMessage());
} finally {
    // sempre executado, mesmo se houver return no try
    echo "Bloco finally executado.\n";
}

Atributos Nativos — #[Attribute] (PHP 8.0)

Atributos são metadados estruturados que podem ser aplicados a classes, funções, parâmetros, propriedades e constantes. Substituem as antigas anotações em docblock (@Route, @ORM\Column) com sintaxe interpretável pelo PHP em tempo de execução via Reflection.

php — definindo e usando Attributes
// 1. Definir o attribute
#[Attribute(Attribute::TARGET_METHOD)]
class Route
{
    public function __construct(
        public readonly string $path,
        public readonly string $method = 'GET',
    ) {}
}

// 2. Aplicar ao método
class UsuarioController
{
    #[Route('/usuarios', 'GET')]
    public function listar(): array { return []; }

    #[Route('/usuarios', 'POST')]
    public function criar(): array { return []; }
}

// 3. Ler via Reflection (framework faz isso automaticamente)
$reflection = new ReflectionClass(UsuarioController::class);
foreach ($reflection->getMethods() as $method) {
    $attrs = $method->getAttributes(Route::class);
    foreach ($attrs as $attr) {
        $route = $attr->newInstance();
        echo "{$route->method} {$route->path} → {$method->name}()\n";
    }
}

SPL — Standard PHP Library

A SPL é uma coleção de interfaces e classes nativas do PHP para estruturas de dados, iteradores e exceções especializadas. Usar a SPL em vez de implementações manuais garante performance e interoperabilidade com o ecossistema.

Estruturas de Dados

  • SplStack — pilha LIFO
  • SplQueue — fila FIFO
  • SplMinHeap / SplMaxHeap — heaps
  • SplDoublyLinkedList — lista duplamente ligada
  • SplFixedArray — array de tamanho fixo, mais eficiente em memória

Iteradores

  • DirectoryIterator — iteração sobre diretório
  • RecursiveDirectoryIterator — iteração recursiva
  • ArrayIterator — wrapper iterável para array
  • FilterIterator — filtragem lazy
  • LimitIterator — limitar iteração

Exceções especializadas

  • InvalidArgumentException
  • OutOfRangeException
  • OverflowException
  • UnexpectedValueException
  • DomainException
php — SplStack e SplMinHeap
$pilha = new SplStack();
$pilha->push('A');
$pilha->push('B');
$pilha->push('C');
echo $pilha->pop(); // C (LIFO)

$heap = new SplMinHeap();
$heap->insert(10);
$heap->insert(3);
$heap->insert(7);
echo $heap->extract(); // 3 (mínimo primeiro)

API de Reflexão

A API de Reflexão do PHP permite inspecionar classes, métodos, propriedades e parâmetros em tempo de execução. É a base para injeção de dependências automática, serialização/deserialização e frameworks de testes.

php — Reflection API
class Servico
{
    public function processar(string $entrada, int $limite = 100): array
    {
        return [];
    }
}

$refClass = new ReflectionClass(Servico::class);
$refMethod = $refClass->getMethod('processar');

foreach ($refMethod->getParameters() as $param) {
    echo $param->getType() . ' $' . $param->getName();
    if ($param->isOptional()) {
        echo ' = ' . $param->getDefaultValue();
    }
    echo "\n";
}
// string $entrada
// int $limite = 100
