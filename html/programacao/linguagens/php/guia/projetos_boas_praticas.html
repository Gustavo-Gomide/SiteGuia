PSR Standards — PHP-FIG

O PHP-FIG define as PSRs, padrões que garantem interoperabilidade entre frameworks e bibliotecas. Seguir PSRs é o mínimo para qualquer biblioteca distribuída via Packagist.

PSRTemaResumo
PSR-1Basic Coding StandardArquivos PHP puro com <?php, UTF-8, CamelCase para classes, UPPER_CASE para constantes
PSR-2Coding StyleIndentação com 4 espaços, chaves de classe na linha seguinte, etc. (obsoleto — use PSR-12)
PSR-4AutoloadingMapeia namespace a diretório: Vendor\Pkg\Cls → src/Pkg/Cls.php
PSR-6Cache InterfaceInterface padrão para pools de cache (CacheItemPoolInterface)
PSR-7HTTP Message InterfaceInterfaces imutáveis para Request, Response, URI, Stream
PSR-11Container InterfaceInterface mínima para DI containers (ContainerInterface)
PSR-12Extended Coding StyleExtensão do PSR-2 para PHP 7+: tipos, propriedades tipadas, construtores promovidos
PSR-15HTTP Server MiddlewareInterface para middleware PSR-7-compatível
PSR-16Simple CacheInterface simplificada de cache (CacheInterface)

Princípios SOLID em PHP

SOLID é o conjunto de cinco princípios de design orientado a objetos formulados por Robert C. Martin que tornam o código mais fácil de manter, testar e estender.

S — Single Responsibility Principle

Uma classe tem apenas um motivo para mudar. Não misture lógica de banco de dados com lógica de negócio na mesma classe.

php — SRP
// Violação: UserService faz autenticação, envio de email E acesso ao banco
class UserService { /* ... */ }

// Correto: responsabilidades separadas
class UserRepository { /* só acesso ao banco */ }
class AuthService     { /* só autenticação */ }
class MailService     { /* só envio de email */ }

O — Open/Closed Principle

Classes abertas para extensão, fechadas para modificação. Use interfaces e herança para adicionar comportamentos sem alterar código existente.

php — OCP com Strategy
interface DescontoStrategy
{
    public function calcular(float $preco): float;
}

class DescontoVIP implements DescontoStrategy
{
    public function calcular(float $preco): float { return $preco * 0.8; }
}
class DescontoBlackFriday implements DescontoStrategy
{
    public function calcular(float $preco): float { return $preco * 0.5; }
}

// Adicionar novo tipo de desconto = nova classe, nenhuma modificação existente

D — Dependency Inversion Principle

Classes de alto nível não dependem de classes de baixo nível. Ambas dependem de abstrações (interfaces).

php — DIP com interface
interface UserRepositoryInterface
{
    public function find(int $id): ?User;
}

class UserService
{
    // Depende da INTERFACE, não da implementação concreta
    public function __construct(
        private readonly UserRepositoryInterface $repo
    ) {}
}

DI Containers e Autowiring

Um DI (Dependency Injection) container automatiza a resolução de dependências. Em vez de instanciar objetos manualmente, o container analisa os tipos declarados no construtor e injeta as implementações corretas.

php — Laravel IoC Container
// Binding de interface → implementação concreta (AppServiceProvider)
$this->app->bind(UserRepositoryInterface::class, EloquentUserRepository::class);

// Singleton: mesma instância reutilizada em toda a requisição
$this->app->singleton(CacheService::class, function($app) {
    return new CacheService($app->make('cache.store'));
});

// Resolução automática — $repo injetado pelo container
class UserController
{
    public function __construct(private UserRepositoryInterface $repo) {}
}

Repository Pattern e Value Objects

php — Repository
interface ProdutoRepositoryInterface
{
    public function findById(int $id): ?Produto;
    public function findAtivos(): Collection;
    public function save(Produto $produto): void;
}

class EloquentProdutoRepository implements ProdutoRepositoryInterface
{
    public function findById(int $id): ?Produto
    {
        return Produto::find($id);
    }
    public function findAtivos(): Collection
    {
        return Produto::ativos()->get();
    }
    public function save(Produto $produto): void
    {
        $produto->save();
    }
}
php — Value Object (imutável)
final readonly class Dinheiro  // readonly class (PHP 8.2)
{
    public function __construct(
        public readonly int $centavos,
        public readonly string $moeda,
    ) {
        if ($this->centavos < 0) {
            throw new DomainException('Valor não pode ser negativo');
        }
    }

    public function somar(Dinheiro $outro): self
    {
        if ($this->moeda !== $outro->moeda) {
            throw new DomainException('Moedas diferentes');
        }
        return new self($this->centavos + $outro->centavos, $this->moeda);
    }
}

Estratégia de Testes

Testes Unitários

Testam uma unidade de lógica isolada (uma classe, um método). Dependências são substituídas por mocks.

php — PHPUnit com mock
$mockRepo = $this->createMock(UserRepositoryInterface::class);
$mockRepo->expects($this->once())
    ->method('find')
    ->with(42)
    ->willReturn(new User(id: 42, nome: 'Alice'));

$service = new UserService($mockRepo);
$user = $service->buscar(42);
$this->assertSame('Alice', $user->nome);

Testes de Feature (Laravel)

Testam fluxos completos HTTP dentro do framework, sem um servidor real.

php — feature test Laravel
class ProdutoApiTest extends TestCase
{
    use RefreshDatabase;

    public function testCriarProduto(): void
    {
        $user = User::factory()->create();

        $this->actingAs($user)
            ->postJson('/api/produtos', [
                'nome'  => 'Notebook',
                'preco' => 3500.00,
            ])
            ->assertCreated()
            ->assertJsonPath('nome', 'Notebook');
    }
}

CI/CD com GitHub Actions

yaml — .github/workflows/ci.yml
name: CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup PHP
        uses: shivammathur/setup-php@v2
        with:
          php-version: '8.3'
          extensions: pdo, mbstring, opcache
          coverage: xdebug

      - name: Install dependencies
        run: composer install --prefer-dist --no-progress

      - name: Run PHPStan
        run: ./vendor/bin/phpstan analyse src --level=8

      - name: Run PHPUnit
        run: ./vendor/bin/phpunit --coverage-clover coverage.xml

      - name: Upload coverage
        uses: codecov/codecov-action@v4
        with: { files: coverage.xml }

Deploy e Monitoramento

Deploy script com zero-downtime

bash — deploy.sh
#!/usr/bin/env bash
set -euo pipefail

echo "==> Instalando dependências de produção"
composer install --no-dev --optimize-autoloader --no-interaction

echo "==> Cacheando configuração"
php artisan config:cache
php artisan route:cache
php artisan view:cache

echo "==> Executando migrations (sem interação)"
php artisan migrate --force

echo "==> Reiniciando workers de fila"
php artisan queue:restart

echo "==> Deploy concluído"

Monitoramento

Laravel Telescope

Painel de debug para desenvolvimento: inspecione requests, jobs, queries, mails, notificações e exceções em tempo real.

Sentry

Monitoramento de erros em produção com contexto completo: stack trace, usuário, variáveis de escopo, breadcrumbs e agrupamento automático de erros similares.

php — Sentry init
\Sentry\init(['dsn' => env('SENTRY_DSN')]);
\Sentry\captureException(new Exception('test'));
