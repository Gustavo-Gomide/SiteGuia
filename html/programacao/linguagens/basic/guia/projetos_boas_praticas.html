BASIC — Projetos e Boas Práticas

O maior inimigo de programas BASIC é o estilo "spaghetti code" que marcou os anos 80: GOTOs em cascata, variáveis de 1-2 letras, sem funções documentadas. O BASIC moderno permite estrutura tão clara quanto qualquer linguagem — mas exige disciplina e boas práticas intencionais.

A regra mais importante: banir GOTO de código novo. Cada uso de GOTO pode ser substituído por IF/THEN, DO/LOOP, FOR/NEXT, SUB ou FUNCTION. Programas sem GOTO são mais legíveis, testáveis e fáceis de depurar.

Estrutura de Projeto FreeBASIC

Organização de arquivos — projeto médio
' Estrutura recomendada para projeto FreeBASIC:
'
' projeto/
'   main.bas           ← ponto de entrada
'   config.bi          ← constantes e configurações globais (include header)
'   types.bi           ← declarações de UDTs compartilhados
'   utils.bas          ← funções utilitárias genéricas
'   utils.bi           ← declarações públicas de utils.bas
'   modelo/
'     usuario.bas      ← lógica de negócio: usuários
'     usuario.bi       ← declarações públicas
'   ui/
'     tela_menu.bas
'   Makefile

' ── config.bi ─────────────────────────────────────────
' #pragma once — evitar inclusão dupla (FreeBASIC 1.x)
' (ou usar guard manual: #ifndef CONFIG_BI / #define CONFIG_BI / #endif)

Const APP_VERSION   = "1.0.0"
Const MAX_USUARIOS  = 500
Const DB_PATH       = "./data/app.db"
Const LOG_PATH      = "./logs/app.log"

' ── types.bi ──────────────────────────────────────────
Type TUsuario
    id      As Long
    nome    As String * 50
    email   As String * 100
    ativo   As Boolean
End Type

' ── main.bas ──────────────────────────────────────────
#include "config.bi"
#include "types.bi"
#include "utils.bi"
#include "modelo/usuario.bi"

Dim usuarios(MAX_USUARIOS) As TUsuario
Dim total As Integer = 0

CarregarUsuarios(DB_PATH, usuarios(), total)
Print "Total de usuários: " & total

Convenções de Nomenclatura

Nomenclatura recomendada em BASIC moderno
' ── Constantes: SCREAMING_SNAKE_CASE ─────────────────
Const MAX_TENTATIVAS = 3
Const PI = 3.14159265358979

' ── Variáveis locais: camelCase ──────────────────────
Dim nomeUsuario   As String
Dim totalItens    As Integer
Dim precoUnitario As Double

' ── UDTs (structs): PascalCase com prefixo T ─────────
Type TProduto
    id       As Long
    descricao As String
    preco    As Double
End Type

' ── SUBs: PascalCase verbo+substantivo ───────────────
Sub CalcularDesconto(ByRef preco As Double, percentual As Double)
    preco = preco * (1.0 - percentual / 100.0)
End Sub

Substituindo GOTO — Código Estruturado

Antes (spaghetti) vs Depois (estruturado)
' ── ANTES (estilo antigo com GOTO) ───────────────────
100 Input "Número positivo: ", n
110 If n <= 0 Then GoTo 100
120 resultado = 1
130 If n = 0 Then GoTo 160
140 resultado = resultado * n
150 n = n - 1 : GoTo 130
160 Print resultado

' ── DEPOIS (estruturado) ──────────────────────────────
Function Fatorial(n As Long) As Long
    Dim resultado As Long = 1
    Dim i As Long
    For i = 2 To n
        resultado = resultado * i
    Next i
    Fatorial = resultado
End Function

Sub Main()
    Dim n As Long
    Do
        Input "Número positivo: ", n
    Loop Until n > 0
    Print "Fatorial: " & Fatorial(n)
End Sub

Main()

Makefile para FreeBASIC

Makefile — compilar projeto multi-arquivo
FBC     = fbc
FBCFLAGS= -lang fb -O 2
TARGET  = meu_programa
SRCS    = main.bas utils.bas modelo/usuario.bas ui/tela_menu.bas

# Compilar todos os .bas com -c (apenas compilar, não linkar)
OBJS    = $(SRCS:.bas=.o)

all: $(TARGET)

$(TARGET): $(OBJS)
	$(FBC) $(FBCFLAGS) -x $(TARGET) $(OBJS)

%.o: %.bas
	$(FBC) $(FBCFLAGS) -c $< -o $@

clean:
	rm -f $(OBJS) $(TARGET)

run: all
	./$(TARGET)

.PHONY: all clean run
