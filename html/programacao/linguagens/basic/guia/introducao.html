BASIC — Introdução

BASIC (Beginner's All-purpose Symbolic Instruction Code) foi criado em 1964 por John G. Kemeny e Thomas E. Kurtz no Dartmouth College com um objetivo revolucionário para a época: tornar a programação acessível a estudantes de qualquer área, não apenas aos especialistas em matemática ou ciência da computação. Em computadores pessoais dos anos 1970–80 — Apple II, Commodore 64, IBM PC — o BASIC era frequentemente o único ambiente de programação disponível, carregado diretamente na ROM.

Longe de ser obsoleto, o BASIC sobrevive em dialetos modernos ativamente mantidos: FreeBASIC (compilado, compatível com C), QB64 (retrocompatível com QBasic/QuickBASIC), Visual Basic .NET (plataforma .NET completa), VBA (automação de Office), e Liberty BASIC / Just BASIC (ensino). A linguagem definiu o que seria "fácil de aprender" nas décadas seguintes.

História e Evolução

Linha do Tempo do BASIC

AnoMarco
1964Kemeny e Kurtz criam Dartmouth BASIC; executa em time-sharing no GE-225
1975Bill Gates e Paul Allen adaptam BASIC para o Altair 8800 — nasce a Microsoft
1977Applesoft BASIC na ROM do Apple II; Commodore BASIC V2 no PET
1981IBM PC carrega GW-BASIC (Microsoft); disseminação massiva
1985QuickBASIC da Microsoft: compilador, estruturas de controle modernas, sem LINE NUMBER obrigatório
1991Visual Basic 1.0: programação orientada a eventos com GUI drag-and-drop no Windows
1993QBasic incluído no MS-DOS 5+ — principal porta de entrada de programadores nos anos 90
2001Visual Basic .NET: atualização para .NET CLR, quebrando compatibilidade com VB6
2005FreeBASIC 0.15 lançado — compilador open-source, 32/64-bit, integração com C
2012QB64: emula DOS QBasic com suporte a gráficos e áudio modernos, compila para C++

Dialetos Modernos

FreeBASIC

Compilador open-source (GCC back-end). Suporta sintaxe do QuickBASIC e extensões modernas: ponteiros, tipos User-Defined, inline Assembly, ligação com bibliotecas C. Multi-plataforma: Windows, Linux, FreeBSD.

QB64

Emulador/compilador do QBasic original. Converte código QB para C++, compila nativamente. Mantém SCREEN, SOUND e DRAW da era DOS. Ideal para resgatar programas antigos ou ensinar BASIC clássico.

Visual Basic .NET

Parte do ecossistema .NET — acessa todas as bibliotecas .NET, suporta POO completa, LINQ, async/await. Compilado para IL (Intermediate Language). Muito usado em aplicações corporativas Windows.

VBA (Excel/Access/Word)

Dialeto embutido na suíte Microsoft Office. Permite automação completa de planilhas, documentos e bancos de dados. Base de milhões de macros corporativas. Interpretado (não compilado).

TI-BASIC

Dialeto para calculadoras Texas Instruments (TI-83, TI-84). Sintaxe visual com menus de teclas. Amplamente usado em educação matemática e como primeira experiência de programação.

Gambas

Visual Basic-like para Linux. IDE visual completa, orientação a objetos, acesso a Qt/GTK. Projeto maduro com componentes para GPU, OpenGL, sockets e banco de dados.

Primeiro Programa — Hello World

QBasic / QB64
PRINT "Hello, World!"
INPUT "Qual é o seu nome? ", nome$
PRINT "Bem-vindo, " + nome$ + "!"
FreeBASIC (moderno)
Dim nome As String
Print "Hello, World!"
Input "Qual é o seu nome? ", nome
Print "Bem-vindo, " & nome & "!"
Visual Basic .NET
Module HelloWorld
    Sub Main()
        Dim nome As String
        Console.Write("Qual é o seu nome? ")
        nome = Console.ReadLine()
        Console.WriteLine("Bem-vindo, {0}!", nome)
    End Sub
End Module
BASIC Clássico com Line Numbers (Dartmouth / GW-BASIC)
10  PRINT "Hello, World!"
20  INPUT "Qual é o seu nome? "; N$
30  PRINT "Bem-vindo, "; N$; "!"
40  END

Onde o BASIC é Usado Hoje

Bom para

  • Automação de Office com VBA (Excel, Word, Access)
  • Ensino de programação iniciante (QB64, TI-BASIC)
  • Protótipos rápidos com FreeBASIC no Linux
  • Aplicações desktop Windows com VB.NET
  • Resgatar e modernizar código legado BASIC dos anos 80/90
  • Calculadoras programáveis (TI-83/84)
  • Scripts de automação em ambientes corporativos antigos

Não recomendado para

  • Desenvolvimento web moderno (fora de VB.NET/ASP.NET)
  • Aplicações com alta concorrência e performance crítica
  • Mobile apps nativo (iOS/Android)
  • Projetos open-source colaborativos de grande escala
  • Machine Learning ou ciência de dados
