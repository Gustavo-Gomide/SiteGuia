LuaJIT — FFI

FFI (Foreign Function Interface)

Com FFI, LuaJIT conversa diretamente com bibliotecas C, reduzindo overhead de bindings tradicionais.

FFI e um dos recursos mais poderosos de LuaJIT para integrar performance nativa sem sair da linguagem.

FFI basico

LuaJIT — require ffi, cdef, load e chamada C
local ffi = require("ffi")

ffi.cdef[[
	int printf(const char *fmt, ...);
	double sqrt(double x);
]]

local C = ffi.C
C.printf("Ola %s!\n", "FFI")
print(C.sqrt(144.0))
LuaJIT — carregando biblioteca dinamica
local ffi = require("ffi")
ffi.cdef[[
	int sqlite3_libversion_number(void);
]]

local sqlite = ffi.load("sqlite3")
print(sqlite.sqlite3_libversion_number())

Structs e arrays C

LuaJIT — structs, arrays numericos e VLA
local ffi = require("ffi")

ffi.cdef[[
	typedef struct { int x; int y; } point_t;
]]

local p = ffi.new("point_t", { x = 10, y = 20 })
print(p.x, p.y)

local arr = ffi.new("double[5]", {1,2,3,4,5})
print(arr[0], arr[4])

local n = 1024
local vla = ffi.new("uint8_t[?]", n) -- VLA
for i = 0, n - 1 do vla[i] = i % 256 end

Integrar bibliotecas C

Em Linux/macOS geralmente carregamos .so/.dylib; no Windows, .dll.

LuaJIT + OpenSSL (exemplo de hashing simplificado)
local ffi = require("ffi")
ffi.cdef[[
	unsigned char *SHA256(const unsigned char *d, size_t n, unsigned char *md);
]]
local ssl = ffi.load("crypto")
local out = ffi.new("unsigned char[32]")
local msg = "site-guia"
ssl.SHA256(msg, #msg, out)
print("SHA256 calculado")

Callbacks C para Lua

LuaJIT — ffi.cast em callback_t
local ffi = require("ffi")

ffi.cdef[[
	typedef int (*callback_t)(int);
]]

local cb = ffi.cast("callback_t", function(x)
	return x * 2
end)

print(cb(21))

-- cuidado: mantenha referencia viva para evitar GC do callback
_G.keep_callback = cb

Se o callback for coletado pelo GC enquanto C ainda o usa, pode ocorrer crash.
