LuaJIT — Compatibilidade e Limitacoes

Onde LuaJIT brilha e onde exige cautela

Adotar LuaJIT envolve balancear velocidade, compatibilidade com ecossistema Lua e requisitos de plataforma.

Escolher runtime e decisao de arquitetura: performance imediata pode custar portabilidade futura.

Compatibilidade com Lua 5.1

LuaJIT e projetado para ser altamente compativel com Lua 5.1 e inclui parte de extensoes de 5.2.

AreaStatus no LuaJITObservacao
API base Lua 5.1CompletaFoco principal de compatibilidade
Bibliotecas padraoAmplamath/table/string/os/io disponiveis
Recursos 5.2+ParcialNem tudo de 5.3/5.4 existe
FFIExtensao propriaNao existe no Lua padrao

Limitacoes conhecidas

JIT e corrotinas

Alguns caminhos com coroutine podem sair do trace JIT.

32-bit pointer mode

Particularmente relevante em builds x86 e cenarios especificos.

Plataformas

Suporte forte em x86/x64/ARM, mas depende de versao e build.

Recursos NYI

Operacoes "not yet implemented" no JIT voltam ao interpretador.

Nem toda lentidao significa "LuaJIT ruim"; muitas vezes o codigo caiu do trace.

Quando escolher: LuaJIT vs Lua 5.4 vs MoonScript

OpcaoMelhor paraTrade-off
LuaJITBaixa latencia, embedding, OpenRestyBase 5.1 + limites de compatibilidade
Lua 5.4Atualizacao da linguagem, recursos modernosSem JIT nativo oficial
MoonScriptSintaxe alternativa para LuaCamada extra de toolchain/transpilacao

Lua usa double IEEE 754; inteiros exatos acima de 2^52 podem perder precisao em algumas operacoes.

LuaJIT em producao com OpenResty/nginx

OpenResty — cosocket e ngx.shared.DICT
# nginx.conf (trecho)
lua_shared_dict cache 50m;

server {
	listen 8080;
	location /ping {
		content_by_lua_block {
			local dict = ngx.shared.cache
			local hits = dict:incr("hits", 1, 0)
			ngx.say("pong; hits=", hits)
		}
	}
}

Em OpenResty, APIs como cosocket e ngx.shared.DICT permitem IO nao bloqueante e cache compartilhado, combinando throughput alto com baixa latencia.
