LuaJIT — Introducao

LuaJIT traz um compilador JIT para Lua 5.1, convertendo trechos "quentes" do codigo para maquina nativa em tempo de execucao.

Em workloads numericos e de rede, LuaJIT pode atingir desempenho proximo de C em varias rotinas, mantendo a simplicidade da linguagem Lua.

O que e LuaJIT

Pense no Lua tradicional como um tradutor que interpreta frase por frase. O LuaJIT observa quais frases se repetem muitas vezes (loops quentes) e cria uma versao "otimizada" para essas partes. Esse processo e chamado de tracing JIT.

Instalacao

Ha tres caminhos comuns: compilar do fonte, usar pacote do sistema e usar a variante integrada em OpenResty.

Shell — compilando do fonte e testando versao
# Linux/macOS
git clone https://luajit.org/git/luajit.git
cd luajit
make
sudo make install

# Ubuntu/Debian (pacote)
sudo apt update
sudo apt install luajit

# Homebrew
brew install luajit

# validar
luajit -v

OpenResty inclui LuaJIT otimizado para nginx e workloads HTTP de alta concorrencia.

Diferencas para Lua padrao

Compatibilidade base

LuaJIT e amplamente compativel com Lua 5.1.

Extensoes 5.2

Alguns recursos de Lua 5.2 existem, mas nao todos.

FFI nativo

Permite chamar C sem binding externo em muitos casos.

JIT tracing

Otimiza loops e traces quentes automaticamente.

Incompatibilidades

Alguns modulos/assuncos de 5.3+ nao se aplicam diretamente.

Foco

Performance em runtime + embedding.

Primeiro programa

Lua — hello.lua e modo interativo
-- hello.lua
print("Ola, LuaJIT!")

local soma = 0
for i = 1, 10 do
	soma = soma + i
end
print("Soma de 1..10:", soma)

-- executar arquivo
-- luajit hello.lua

-- modo interativo
-- luajit -i
