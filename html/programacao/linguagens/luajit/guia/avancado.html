Engenharia de desempenho com tracing, FFI e controle de memoria

Nesta camada, o foco nao e apenas "rodar rapido", mas manter performance estavel sob carga real de producao.

O ganho avancado no LuaJIT vem de diagnosticar side exits, reduzir churn de alocacao e estabelecer contratos claros entre Lua e C.

Internals: recorder, guards, side exits e trace stitching

Um trace e uma trilha linear otimizada com premissas de tipo e fluxo. Quando uma premissa falha, o runtime executa side exit e volta ao interpretador, podendo criar um trace secundario para aquele desvio. Em cenarios ideais, traces secundarios se conectam ao principal (stitching), reduzindo custo de retorno ao bytecode.

LuaJIT - instrumentacao de traces em laboratorio
local jit = require("jit")
local dump = require("jit.dump")

dump.start("tib")
jit.on()

local acc = 0
for i = 1, 2e6 do
  if i % 8192 == 0 then
    acc = acc + tonumber("1") -- forca desvio raro
  else
    acc = acc + i
  end
end

print(acc)

FFI avancado: ABI, alinhamento e ciclo de vida de ponteiros

Em cenarios de alta performance, FFI permite manipular estruturas C complexas, mas isso traz riscos de ABI: tamanho incorreto de struct, alinhamento invalido, callback coletado pelo GC e uso de ponteiro apos liberacao.

LuaJIT - callback ancorado e estrutura aninhada
local ffi = require("ffi")

ffi.cdef[[
  typedef struct { double x; double y; } vec2;
  typedef struct { vec2 p; vec2 v; } body;
  typedef int (*step_cb)(body* b, double dt);
]]

local b = ffi.new("body")
b.p.x, b.p.y = 10.0, -5.0
b.v.x, b.v.y = 1.25, 0.75

local cb = ffi.cast("step_cb", function(ptr, dt)
  ptr.p.x = ptr.p.x + ptr.v.x * dt
  ptr.p.y = ptr.p.y + ptr.v.y * dt
  return 0
end)

_G.__ffi_step_anchor = cb -- evita coleta do callback
cb(b, 0.5)
print(b.p.x, b.p.y)

Nunca exponha ponteiros para memoria temporaria quando a biblioteca C mantem referencia apos o retorno.

Otimizacoes avancadas: pressao de GC e layout de dados

Em sistemas de baixa latencia, alocacao transitiva costuma ser o primeiro inimigo. Pools de objetos, buffers FFI reutilizaveis e reducao de closures no hot loop sao tecnicas de alto impacto.

LuaJIT - pool de objetos para reduzir churn
local pool = {}

local function acquire()
  local obj = pool[#pool]
  if obj then
    pool[#pool] = nil
    return obj
  end
  return { a = 0, b = 0, c = 0 }
end

local function release(obj)
  obj.a, obj.b, obj.c = 0, 0, 0
  pool[#pool + 1] = obj
end

for i = 1, 1e6 do
  local v = acquire()
  v.a = i
  release(v)
end

Checklist de tuning avancado

  • Contar side exits por endpoint ou rotina.
  • Medir pausas de GC antes e depois de cada otimizaao.
  • Isolar chamadas FFI em fronteiras pequenas e testaveis.
  • Separar claramente caminho quente de caminho administrativo.
