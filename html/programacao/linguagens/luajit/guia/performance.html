LuaJIT — Performance e JIT

Como extrair o maximo do tracing JIT

Performance em LuaJIT nao depende so da linguagem, mas de como seu codigo gera traces estaveis e previsiveis.

A regra pratica: mantenha os hot paths simples, tipados de forma consistente e com baixa variacao de controle.

Como o JIT funciona

O LuaJIT observa execucao, identifica loops quentes, grava um trace e compila esse caminho para codigo nativo.

Lua — ligando/desligando JIT e inspecao
local jit = require("jit")

jit.on()        -- ativa JIT
jit.off()       -- desativa JIT
jit.flush()     -- limpa traces compilados

-- nyi/nyquil mode: recursos nao compilados caem para interpretador
-- para diagnosticar traces:
require("jit.dump").start("t")  -- dump de traces

Benchmarks e boas praticas

Loops numericos

for i=1,n tende a gerar traces mais estaveis que iteradores complexos.

Locais > globais

Cacheie funcoes e tabelas locais para reduzir lookup.

Evite closures em hot path

Alocacoes e upvalues podem atrapalhar traces.

Dados homogeneos

Misturar tipos frequentemente invalida otimizações.

FFI para performance maxima

LuaJIT FFI — arrays C e acesso direto
local ffi = require("ffi")
ffi.cdef[[
	typedef struct { double x; double y; } point_t;
]]

local n = 1e6
local arr = ffi.new("double[?]", n)
for i = 0, n - 1 do
	arr[i] = i * 0.5
end

local sum = 0.0
for i = 0, n - 1 do
	sum = sum + arr[i]
end
print(sum)

Exemplos praticos otimizados

LuaJIT — multiplicacao de matrizes e string scan
-- multiplicacao de matrizes N x N (versao simples otimizada)
local N = 128
local A, B, C = {}, {}, {}
for i = 1, N do
	A[i], B[i], C[i] = {}, {}, {}
	for j = 1, N do
		A[i][j] = (i + j) % 7
		B[i][j] = (i * j) % 11
		C[i][j] = 0
	end
end

for i = 1, N do
	local Ci = C[i]
	for k = 1, N do
		local aik = A[i][k]
		local Bk = B[k]
		for j = 1, N do
			Ci[j] = Ci[j] + aik * Bk[j]
		end
	end
end

-- string processing via FFI (ex.: bytes de um buffer)
local ffi = require("ffi")
local s = "LuaJIT-FAST-PATH"
local p = ffi.cast("const uint8_t*", s)
local upper = 0
for i = 0, #s - 1 do
	if p[i] >= 65 and p[i] <= 90 then upper = upper + 1 end
end
print("Maiusculas:", upper)

Benchmark real deve incluir aquecimento, varias iteracoes e isolamento de IO.
