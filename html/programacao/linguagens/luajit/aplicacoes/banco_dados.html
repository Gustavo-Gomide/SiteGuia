LuaJIT e Banco de Dados

Acesso SQL de alta eficiencia em runtime leve

Integre LuaJIT a SQLite, MySQL e PostgreSQL com foco em latencia, seguranca e consistencia.

Nao basta conectar e consultar: desempenho real depende de pool, prepared statements, transacoes e observabilidade de erro.

Drivers e estrategias de conexao

No ecossistema LuaJIT, voce encontra dois caminhos principais: wrappers SQL classicos (LuaSQL, LuaDBI) e drivers focados em OpenResty (resty.*). A escolha depende do ambiente de execucao.

AmbienteOpcao comumObservacao
Script/daemonLuaSQL, LuaDBIAPI simples, boa portabilidade
OpenRestylua-resty-mysql, lua-resty-postgresCosockets e pool no worker
Dados locaislsqlite3Zero servidor, ideal para edge

Trade-off central

Drivers blocking sao mais simples em scripts de lote. Em servidores concorrentes, prefira drivers nao bloqueantes para nao degradar throughput.

Prepared statements e transacoes

Prepared statements reduzem overhead de parse e mitigam SQL injection. Transacoes garantem consistencia quando varias operacoes devem ser atomicas.

Exemplo com LuaDBI (conceito portavel)
local DBI = require("DBI")
local conn = DBI.Connect("PostgreSQL", "appdb", "app", "segredo")

conn:autocommit(false)
local st = conn:prepare("INSERT INTO pedidos(cliente_id, total) VALUES(?, ?)")
local ok, err = st:execute(42, 199.90)

if ok then
  conn:commit()
else
  conn:rollback()
  error(err)
end

Nunca concatene parametros de usuario em SQL. Mesmo em scripts internos, use bind de parametros.

Pooling, timeout e retry

Conectar para cada requisicao custa caro. Pool de conexoes reduz handshake e melhora estabilidade sob carga.

Pseudo-config de pool em contexto OpenResty
local db = require("resty.mysql"):new()
db:set_timeout(2000)

local ok, err = db:connect({
  host = "127.0.0.1",
  port = 3306,
  user = "app",
  password = "segredo",
  database = "appdb",
  pool = "main_rw",
  pool_size = 100
})

if not ok then
  ngx.log(ngx.ERR, err)
end
