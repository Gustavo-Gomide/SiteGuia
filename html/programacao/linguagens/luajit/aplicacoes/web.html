LuaJIT para Web

Edge e API de baixa latencia com OpenResty

Use LuaJIT dentro do Nginx para autenticar, rotear, proteger e observar trafego em tempo real.

OpenResty permite escrever logica critica na borda da aplicacao, reduzindo hops e aumentando throughput sem perder legibilidade.

Conceitos e arquitetura

Em vez de tratar LuaJIT como "apenas script", pense nele como um motor de decisao no pipeline HTTP do Nginx. Cada fase da requisicao pode executar codigo Lua: autenticacao, roteamento, manipulacao de headers e resposta.

Handlers e roteamento com ngx

Uma estrategia comum e usar uma tabela de rotas com funcoes Lua. O handler decide rapidamente e responde sem passar por uma camada extra de app server.

content_by_lua_block - roteador minimo
local routes = {
  ["/health"] = function()
    ngx.header["content-type"] = "application/json"
    ngx.say('{"ok":true}')
  end,
  ["/v1/ping"] = function()
    ngx.say("pong")
  end
}

local h = routes[ngx.var.uri]
if h then
  h()
else
  ngx.status = 404
  ngx.say("not found")
end

Esse padrao e excelente para gateways e endpoints tecnicos. Para sistemas de dominio extenso, combine com separacao por modulos para evitar handlers gigantes.

Rate limit, cache e resiliencia

Em producao, use estruturas compartilhadas (lua_shared_dict) para limites por IP/chave e cache de respostas curtas.

Rate limit simples por janela fixa
local dict = ngx.shared.ratelimit
local key = ngx.var.binary_remote_addr
local now = ngx.time()
local bucket = key .. ":" .. tostring(now - (now % 60))
local n, err = dict:incr(bucket, 1)
if not n then
  dict:set(bucket, 1, 65)
  n = 1
end
if n > 120 then
  ngx.status = 429
  ngx.say("too many requests")
  return ngx.exit(429)
end

Janela fixa e facil de operar, mas pode gerar burst no limite de virada do minuto. Para fairness maior, prefira token bucket.

Observabilidade e operacao
