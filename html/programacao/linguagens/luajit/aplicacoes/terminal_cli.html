Parsing de argumentos e UX de comando

Uma CLI madura precisa de comandos claros, help consistente e codigos de saida previsiveis. Bibliotecas como argparse simplificam esse design.

Estrutura de subcomando com flags
local argparse = require("argparse")
local parser = argparse("devtool", "Automacao de deploy")

local deploy = parser:command("deploy", "Publica versao")
deploy:option("--env", "Ambiente alvo"):default("staging")
deploy:flag("--dry-run")

local args = parser:parse()

Regra de ouro

Se um comando pode causar dano, ofereca modo simulacao (--dry-run) e confirmacao explicita.

Automacao DevOps e subprocessos

CLIs em LuaJIT podem orquestrar ferramentas existentes (git, docker, kubectl), capturando saida e aplicando regras antes de prosseguir.

Execucao de comando com captura de saida
local function run(cmd)
  local h = io.popen(cmd .. " 2>&1")
  local out = h:read("*a")
  local ok = h:close()
  return ok, out
end

local ok, out = run("git rev-parse --short HEAD")
if not ok then
  io.stderr:write("falha no git\n")
  os.exit(2)
end

Logs, codigos de saida e observabilidade

CLI profissional precisa ser previsivel para humanos e para pipelines CI/CD.

stdout para dados

Saida principal consumivel por pipe.

stderr para erros

Separacao clara para troubleshooting.

Exit codes

0 sucesso, codigos nao-zero por classe de falha.

Formato estruturado

JSON lines para integracao com observabilidade.

Logger simples com niveis
local function log(level, msg)
  io.stderr:write(os.date("!%Y-%m-%dT%H:%M:%SZ"), " [", level, "] ", msg, "\n")
end

log("INFO", "inicio do processo")
log("ERROR", "timeout no servidor")

Robustez avancada

Ferramentas de automacao quebram em horarios criticos. Prepare rollback, timeout e comportamento idempotente.

Idempotencia significa poder repetir o comando sem corromper estado. Esse principio e essencial em operacao automatizada.
