Delphi / Object Pascal — Introdução

Delphi é um ambiente de desenvolvimento integrado (IDE) e linguagem criado pela Borland em 1995. Baseado em Object Pascal, combina tipagem estática forte, compilação nativa ultrarrápida e o paradigma RAD (Rapid Application Development) — arrastando componentes visuais, configurando propriedades no inspetor e gerando aplicações completas em minutos.

Delphi gerou toda uma era de desenvolvimento Windows nos anos 1990-2000. Hoje, com o FireMonkey, o mesmo código Pascal compila para Windows, macOS, iOS e Android — sem alteração. A base instalada inclui sistemas bancários, industriais e médicos rodando em produção há 25 anos.

Primeiro Programa

Console — program HelloWorld.dpr
program HelloWorld;

{$APPTYPE CONSOLE}       // diretiva compilador — app console
{$R *.res}               // incluir arquivo de recursos

uses
  System.SysUtils;       // unit padrão — IntToStr, TryStrToInt, etc.

var
  nome: string;          // variável string Unicode

begin
  Writeln('Olá, mundo!');
  Write('Digite seu nome: ');
  Readln(nome);
  Writeln('Bem-vindo, ' + nome + '!');
  Writeln('1 + 2 = ', 1 + 2);

  try
    raise Exception.Create('Erro de demonstração');
  except
    on E: Exception do
      Writeln('Exceção: ', E.Message);
  end;

  Readln;  // aguardar Enter antes de fechar
end.
Compilar e executar via linha de comando (DCC32/DCC64)
# Compilar (DCC32 ou DCC64 — incluso no Delphi)
dcc64 HelloWorld.dpr -U"C:\Program Files (x86)\Embarcadero\Studio\23.0\Lib\Win64\release"

# Ou via MSBuild
msbuild HelloWorld.dproj /p:Configuration=Release /p:Platform=Win64

# Executar
.\Win64\Release\HelloWorld.exe

Estrutura de um Projeto Delphi

Unit — bloco fundamental de código
unit MinhaUnit;

interface   // seção pública — declarações visíveis de fora

uses
  System.SysUtils, System.Classes;

type
  TPessoa = class         // declaração de classe
  private
    FNome:  string;
    FIdade: Integer;
  public
    constructor Create(const ANome: string; AIdade: Integer);
    procedure Apresentar;
    property  Nome:  string  read FNome  write FNome;
    property  Idade: Integer read FIdade write FIdade;
  end;

implementation  // seção privada — implementações

constructor TPessoa.Create(const ANome: string; AIdade: Integer);
begin
  FNome  := ANome;
  FIdade := AIdade;
end;

procedure TPessoa.Apresentar;
begin
  Writeln(Format('Olá! Sou %s e tenho %d anos.', [FNome, FIdade]));
end;

end.   // toda unit termina com end.

Por que Delphi?

Compilação Nativa

Gera executáveis nativos .exe / .dylib sem VM ou JIT. Binários Win64 de 2-5 MB standalone — sem instalação de runtime.

RAD — Velocidade

Arrastar um TButton, dar duplo-clique, escrever o handler. Uma aplicação com banco de dados em horas, não dias. VCL tem 1000+ componentes.

Cross-Platform

FireMonkey compila o mesmo código para Windows, macOS, iOS e Android. Compartilhe 90% do código de negócio entre plataformas.

Tipo Estático Forte

Detecta erros em compile time. Integer ≠ string, conversões explícitas obrigatórias. Refactoring seguro em bases de código grandes.

Legado e Longevidade

Código Delphi 5 (1999) ainda compila em Delphi 12 (2024) com modificações mínimas. Sistemas críticos com 25+ anos de produção contínua.

VCL + FireMonkey

VCL: framework nativo Win32/64 ultra-maduro. FMX: UI vetorial cross-platform. Dois mundos, mesma linguagem.

Quando Usar e Quando Evitar

Use Delphi para

  • Aplicações Windows desktop empresariais com banco de dados
  • Migrar / manter sistemas legados Delphi existentes
  • Apps mobile (iOS/Android) com código compartilhado com desktop
  • Embutir em hardware com interfaces gráficas industriais
  • Times que precisam de produtividade RAD alta e prazos curtos

Evite Delphi quando

  • O projeto é web — use linguagens com ecossistema web nativo
  • Precisa de grande ecossistema open-source (npm, PyPI)
  • O time não tem experiência Pascal e o prazo é curto
  • Licenciamento Embarcadero é inviável (IDE é proprietária e cara)
  • Microservices / containers — binários nativos grandes para Docker
