Delphi — Ecossistema e Tooling

O ecossistema Delphi gira em torno do RAD Studio IDE da Embarcadero — um ambiente completo com editor de código, designer visual, Object Inspector, depurador integrado, profiler e gerenciador de pacotes GetIt. O compilador DCC64 gera binários nativos sem dependências de runtime externo.

A alternativa gratuita é o Lazarus IDE com o compilador Free Pascal (FPC) — compatível com Object Pascal, open-source, e que suporta mais de 20 plataformas incluindo Linux. Para projetos novos sem orçamento para licença Embarcadero, Lazarus é a escolha.

Compiladores e Linha de Comando

DCC32/DCC64 — compilar da linha de comando
# Compilar para Win32
dcc32 MeuApp.dpr

# Compilar para Win64 com mensagens
dcc64 MeuApp.dpr -W    # -W = warnings

# Flags úteis
dcc64 MeuApp.dpr ^
  -O+                  # otimizações ligadas
  -$D-                 # sem info de debug (menor binary)
  -$L-                 # sem link de símbolos locais
  -E"output\"          # pasta de output
  -N"obj\"             # pasta de objetos .dcu
  -U"path1;path2"      # search path de units

# MSBuild — build profissional
msbuild MeuApp.dproj /p:Config=Release /p:Platform=Win64
msbuild MeuApp.dproj /p:Config=Debug   /p:Platform=Win32

# Free Pascal (Lazarus)
fpc meuapp.lpr -O2 -Xs -XX   # -O2 otimizar, -Xs strip, -XX executável

GetIt — Gerenciador de Pacotes

GetIt e pacotes essenciais
// GetIt está integrado ao IDE: Tools → GetIt Package Manager
// Ou via linha de comando (GetIt CLI):

getit install delphi-mocks
getit install dunitx
getit install mormot2

// Pacotes populares disponíveis no GetIt ou GitHub:
//   DUnitX       — framework de testes unitários
//   mORMot2      — ORM, REST, JSON, SQLite ultra-rápido
//   TMS FNC      — componentes cross-platform
//   Indy         — Internet Direct — TCP/HTTP/SMTP/FTP
//   Spring4D     — IoC, coleções, RTTI avançado
//   DByteBuffer  — manipulação binária
//   ZeosLib      — acesso a banco multi-driver
//   FireDAC      — DA nativo Embarcadero (incluso no RAD)

Depuração e Profiling

Breakpoints, watches e técnicas de depuração
// ── Breakpoints avançados ─────────────────────────────────────
// Breakpoint condicional — parar somente se condição verdadeira
// Clicar na barra lateral → right-click → "Breakpoint Properties"
// Condition: i = 50  → parar apenas quando i = 50
// Pass count: 10     → parar somente na 10ª passagem

// ── Debug watches ─────────────────────────────────────────────
// Ctrl+F5 ou Debug → Add Watch
// Expressões: lista.Count, lista.Items[5], pessoa.Nome, @ptr

// ── Logging com OutputDebugString ────────────────────────────
uses Winapi.Windows;

OutputDebugString(PChar('Debug: valor = ' + IntToStr(x)));
// Visual em "Debug Events" no IDE, ou com DebugView (Sysinternals)

// ── ReportMemoryLeaksOnShutdown ──────────────────────────────
program MeuApp;
{$APPTYPE CONSOLE}
begin
  ReportMemoryLeaksOnShutdown := True;   // detectar vazamentos
  // ... código ...
end.

// ── AQTime / Sampling profiler ────────────────────────────────
// Run → Sampling Profiler → Start — mostra hotspots por tempo gasto
// Run → AQTime — profiler completo com memória, GDI, threads

Free Pascal e Lazarus — Alternativa Open Source

Instalar e criar projeto Lazarus
# Ubuntu/Debian
sudo apt install fpc lazarus

# macOS
brew install fpc
# Baixar Lazarus.app de lazarus-ide.org

# Windows — instalador em lazarus-ide.org

# Compilar Console sem IDE
fpc HelloWorld.pas

# Cross-compile para Linux ARM (ex: Raspberry Pi)
fpc -Parm -Tlinux -O2 blink.pas

# Compilar com Lazarus IDE
# File → New Project → Application (LCL)
# Run → Build (F9) → gera binário na pasta project/

ReportMemoryLeaksOnShutdown := True é a primeira linha de defesa contra vazamentos. No modo debug, o Delphi Manager de FastMM4 (Memory Manager) detecta objetos não liberados e exibe a classe, o tamanho, e até a stack trace de onde foram alocados. Sem custo algum em produção, pois é zerado automaticamente em $IFDEF DEBUG.
