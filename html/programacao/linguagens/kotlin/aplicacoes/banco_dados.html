Kotlin — Banco de Dados

O acesso a banco de dados em Kotlin é idiomático com Exposed: um ORM que oferece tanto DSL SQL type-safe quanto camada DAO orientada a entidades. HikariCP gerencia o pool de conexões e Flyway cuida das migrações estruturadas.

Kotlin transforma o acesso a banco de dados em algo expressivo: blocos transaction { }, queries type-safe via objetos Kotlin e data classes mapeadas sem reflexão ou anotações excessivas.

Exposed DSL — SQL Type-Safe

Na API DSL do Exposed, tabelas são objetos Kotlin singleton. As queries são expressas em Kotlin puro e verificadas em tempo de compilação — sem strings SQL soltas e sem risco de injeção.

Kotlin — Exposed DSL CRUD
import org.jetbrains.exposed.sql.*
import org.jetbrains.exposed.sql.transactions.transaction

object Produtos : Table("produtos") {
    val id = integer("id").autoIncrement()
    val nome = varchar("nome", 256)
    val preco = double("preco")
    val estoque = integer("estoque").default(0)
    override val primaryKey = PrimaryKey(id)
}

fun main() {
    Database.connect("jdbc:postgresql://localhost:5432/app",
        driver = "org.postgresql.Driver",
        user = "admin", password = "senha")

    transaction {
        SchemaUtils.create(Produtos)

        // INSERT
        Produtos.insert {
            it[nome] = "Teclado Mecânico"
            it[preco] = 299.90
            it[estoque] = 50
        }

        // SELECT com filtro e ordenação
        val caros = Produtos
            .selectAll()
            .where { Produtos.preco greaterThan 100.0 }
            .orderBy(Produtos.preco to SortOrder.DESC)
            .map { row -> row[Produtos.nome] to row[Produtos.preco] }
        println(caros)

        // UPDATE
        Produtos.update({ Produtos.estoque less 10 }) {
            it[estoque] = 100
        }

        // DELETE
        Produtos.deleteWhere { preco less 0.0 }
    }
}

Exposed DAO — Entidades

A API DAO do Exposed mapeia linhas de tabela em objetos Kotlin Entity. Relacionamentos (hasMany, belongsTo) são lazy por padrão, evitando N+1 com with() de eager loading.

Kotlin — Exposed DAO
import org.jetbrains.exposed.dao.*
import org.jetbrains.exposed.dao.id.IntIdTable
import org.jetbrains.exposed.dao.id.EntityID

object UserTable : IntIdTable("users") {
    val nome = varchar("nome", 128)
    val email = varchar("email", 256).uniqueIndex()
}

class User(id: EntityID<Int>) : IntEntity(id) {
    companion object : IntEntityClass<User>(UserTable)
    var nome by UserTable.nome
    var email by UserTable.email
}

transaction {
    SchemaUtils.create(UserTable)

    // CREATE
    val u = User.new {
        nome = "Beatriz"
        email = "bea@email.com"
    }

    // FIND
    val todos = User.all().toList()
    val porId = User.findById(1)
    val porEmail = User.find { UserTable.email eq "bea@email.com" }.firstOrNull()

    // UPDATE
    porEmail?.nome = "Beatriz Lima"
}

HikariCP — Pool de Conexões

HikariCP é o pool de conexões JDBC mais rápido. Em produção, conecte o Exposed ao HikariCP em vez de strings diretas para gerenciar conexões com timeouts, pool size e health checks.

Kotlin — HikariCP + Exposed
import com.zaxxer.hikari.HikariConfig
import com.zaxxer.hikari.HikariDataSource

fun criarDataSource(): HikariDataSource {
    val config = HikariConfig().apply {
        jdbcUrl = "jdbc:postgresql://localhost:5432/appdb"
        driverClassName = "org.postgresql.Driver"
        username = "admin"
        password = System.getenv("DB_PASSWORD")
        maximumPoolSize = 10
        minimumIdle = 2
        connectionTimeout = 30_000
        idleTimeout = 600_000
        maxLifetime = 1_800_000
    }
    return HikariDataSource(config)
}

// Conectar Exposed ao pool
val ds = criarDataSource()
Database.connect(ds)

// A partir daqui, todas as transactions usam o pool
transaction {
    // queries normais
}

Flyway — Migrações de Banco

Flyway gerencia o histórico de evolução do schema. Scripts SQL nomeados (V1__descricao.sql) são aplicados em ordem, acompanhados de uma tabela de histórico. Integra com Ktor via hook de inicialização.

SQL — V1__create_users.sql
-- src/main/resources/db/migration/V1__create_users.sql
CREATE TABLE users (
    id    SERIAL       PRIMARY KEY,
    nome  VARCHAR(128) NOT NULL,
    email VARCHAR(256) NOT NULL UNIQUE,
    criado_em TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_email ON users(email);
Kotlin — Flyway integrado ao Ktor
import org.flywaydb.core.Flyway

fun Application.configurarBancoDados() {
    val ds = criarDataSource()

    // Executar migrações pendentes no startup
    Flyway.configure()
        .dataSource(ds)
        .locations("classpath:db/migration")
        .baselineOnMigrate(true)
        .load()
        .migrate()

    Database.connect(ds)
}
