Kotlin — GUI Desktop

Compose Desktop traz o paradigma declarativo do Jetpack Compose para macOS, Windows e Linux — a mesma API usada no Android. TornadoFX oferece uma alternativa baseada em JavaFX para aplicações legadas na JVM.

Compose Desktop elimina o modelo imperativo de UI: estado muda, a UI re-renderiza automaticamente. Com @Composable e remember, componentes encapsulam estado local sem controladores externos.

Compose Desktop — Introdução

Uma aplicação Compose Desktop começa com a função main que cria uma Window. O conteúdo é uma árvore de @Composable que recompõe automaticamente quando o estado muda.

Kotlin — Compose Desktop básico
import androidx.compose.desktop.ui.tooling.preview.Preview
import androidx.compose.foundation.layout.*
import androidx.compose.material.*
import androidx.compose.runtime.*
import androidx.compose.ui.Alignment
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp
import androidx.compose.ui.window.Window
import androidx.compose.ui.window.application

@Composable
fun App() {
    var contador by remember { mutableStateOf(0) }
    var texto by remember { mutableStateOf("") }

    MaterialTheme {
        Column(
            modifier = Modifier.fillMaxSize().padding(16.dp),
            horizontalAlignment = Alignment.CenterHorizontally,
            verticalArrangement = Arrangement.spacedBy(8.dp)
        ) {
            Text("Contador: $contador", style = MaterialTheme.typography.h5)
            Button(onClick = { contador++ }) { Text("Incrementar") }
            OutlinedTextField(value = texto, onValueChange = { texto = it }, label = { Text("Nome") })
            if (texto.isNotBlank()) Text("Olá, $texto!")
        }
    }
}

fun main() = application {
    Window(onCloseRequest = ::exitApplication, title = "Minha App Kotlin") { App() }
}

Layouts e Listas em Compose

LazyColumn renderiza listas longas de forma eficiente, semelhante ao RecyclerView. Modifier aplica tamanho, padding, bordas e click handlers de forma encadeada.

Kotlin — Compose lista e navegação
import androidx.compose.foundation.clickable
import androidx.compose.foundation.lazy.LazyColumn
import androidx.compose.foundation.lazy.items
import androidx.compose.material.Divider
import androidx.compose.material.ListItem

data class Item(val id: Int, val titulo: String, val desc: String)

@Composable
fun ListaItens(itens: List<Item>, aoClicar: (Item) -> Unit) {
    LazyColumn {
        items(itens, key = { it.id }) { item ->
            ListItem(
                text = { Text(item.titulo) },
                secondaryText = { Text(item.desc) },
                modifier = Modifier.clickable { aoClicar(item) }
            )
            Divider()
        }
    }
}

// Mestre-detalhe com estado local
@Composable
fun TelaLista(itens: List<Item>) {
    var selecionado by remember { mutableStateOf<Item?>(null) }
    Row(Modifier.fillMaxSize()) {
        Box(Modifier.weight(1f)) { ListaItens(itens) { selecionado = it } }
        Box(Modifier.weight(1f).padding(16.dp)) {
            selecionado?.let { Text(it.desc, style = MaterialTheme.typography.body1) }
                ?: Text("Selecione um item")
        }
    }
}

TornadoFX — JavaFX com Kotlin DSL

TornadoFX envolve JavaFX com DSLs Kotlin idiomáticas. Embora Compose Desktop seja a recomendação atual, TornadoFX é útil para manter aplicações JavaFX existentes com código mais conciso.

Kotlin — TornadoFX App
import tornadofx.*

class MinhaApp : App(MinhaView::class)

class MinhaView : View("Olá TornadoFX") {
    private val contador = SimpleIntegerProperty(0)
    private val nome = SimpleStringProperty("")

    override val root = vbox(spacing = 8.0) {
        padding = Insets(16.0)
        label { bind(contador.stringBinding { "Contador: $it" }) }
        button("Incrementar") { action { contador.set(contador.get() + 1) } }
        textfield(nome) { promptText = "Digite seu nome" }
        label { bind(nome.stringBinding { if (it!!.isNotBlank()) "Olá, $it!" else "" }) }
    }
}

fun main() = launch<MinhaApp>()

Distribuição Nativa

Compose Desktop gera instaladores nativos (MSI, DMG, DEB) via o bloco nativeDistributions no Gradle. Conveyor é uma alternativa que facilita auto-update e assinatura de código.

Kotlin — build.gradle.kts distribuição
import org.jetbrains.compose.desktop.application.dsl.TargetFormat

plugins {
    kotlin("jvm") version "2.1.0"
    id("org.jetbrains.compose") version "1.7.1"
}

compose.desktop {
    application {
        mainClass = "MainKt"
        nativeDistributions {
            targetFormats(TargetFormat.Dmg, TargetFormat.Msi, TargetFormat.Deb)
            packageName = "MeuApp"
            packageVersion = "1.0.0"
            macOS { iconFile.set(file("icon.icns")) }
            windows { iconFile.set(file("icon.ico")) }
            linux { iconFile.set(file("icon.png")) }
        }
    }
}

// Build: ./gradlew packageReleaseMsi  (Windows)
//        ./gradlew packageReleaseDmg  (macOS)
//        ./gradlew packageReleaseDeb  (Linux)
