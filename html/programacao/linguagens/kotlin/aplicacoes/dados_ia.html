Kotlin — Dados e IA

Kotlin opera com dados através de sequences funcionais eficientes, ONNX Runtime para inferência de modelos pré-treinados e DL4J para treinamento na JVM. É uma escolha natural quando o sistema já é JVM e precisa de coleta de dados, enriquecimento e IA embarcada.

A tipagem estática e as collections funcionais de Kotlin tornam pipelines de dados expressivos e seguros. Combinados com ONNX, permitem inferência sem dependência de Python em produção.

Pipelines de Dados Funcionais

Kotlin Sequences processam dados de forma lazy — ideal para CSVs grandes onde não queremos carregar tudo na memória. Operações map, filter, groupBy e fold compõem pipelines declarativos.

Kotlin — Pipeline CSV com Sequences
import java.io.File

data class Venda(val produto: String, val regiao: String, val valor: Double)

fun lerVendas(caminho: String): Sequence<Venda> =
    File(caminho).useLines { linhas ->
        linhas
            .drop(1) // cabeçalho
            .map { linha ->
                val (produto, regiao, valor) = linha.split(",")
                Venda(produto.trim(), regiao.trim(), valor.trim().toDouble())
            }
            .toList() // materializar dentro do useLines
            .asSequence()
    }

val relatorio = lerVendas("vendas.csv")
    .filter { it.valor > 0 }
    .groupBy { it.regiao }
    .mapValues { (_, vendas) ->
        mapOf(
            "total" to vendas.sumOf { it.valor },
            "media" to vendas.map { it.valor }.average(),
            "count" to vendas.size
        )
    }

relatorio.forEach { (regiao, stats) -> println("$regiao: $stats") }

ONNX Runtime — Inferência de Modelos

ONNX Runtime tem binding Java/Kotlin oficial. Permite executar modelos exportados do PyTorch ou Scikit-learn diretamente na JVM, sem servidor Python — ideal para microsserviços de inferência de baixa latência.

Kotlin — ONNX inferência
import ai.onnxruntime.*

fun inferir(modeloPath: String, entrada: FloatArray): FloatArray {
    val env = OrtEnvironment.getEnvironment()
    val sessao = env.createSession(modeloPath, OrtSession.SessionOptions())

    // Preparar tensor de entrada
    val shape = longArrayOf(1L, entrada.size.toLong())
    val tensor = OnnxTensor.createTensor(env, floatArrayOf(*entrada), shape)

    // Inferência
    val nomeEntrada = sessao.inputNames.first()
    val resultado = sessao.run(mapOf(nomeEntrada to tensor))

    // Extrair saída
    val saida = resultado[0].value as Array<FloatArray>
    tensor.close(); resultado.close(); sessao.close()
    return saida[0]
}

// Uso:
val features = floatArrayOf(0.5f, 1.2f, 3.4f)
val predicao = inferir("modelo.onnx", features)
println("Classe predita: ${predicao.indices.maxByOrNull { predicao[it] }}")

DL4J — Deep Learning na JVM

Deeplearning4j (DL4J) é a principal biblioteca de deep learning para a JVM. Kotlin se integra naturalmente, usando a API builders do DL4J com lambdas para configurar arquiteturas de redes neurais.

Kotlin — DL4J rede neural simples
import org.deeplearning4j.nn.conf.NeuralNetConfiguration
import org.deeplearning4j.nn.conf.layers.DenseLayer
import org.deeplearning4j.nn.conf.layers.OutputLayer
import org.deeplearning4j.nn.multilayer.MultiLayerNetwork
import org.nd4j.linalg.activations.Activation
import org.nd4j.linalg.lossfunctions.LossFunctions

val conf = NeuralNetConfiguration.Builder()
    .seed(42)
    .list()
    .layer(DenseLayer.Builder()
        .nIn(4).nOut(16)
        .activation(Activation.RELU)
        .build())
    .layer(OutputLayer.Builder(LossFunctions.LossFunction.MCXENT)
        .nIn(16).nOut(3)
        .activation(Activation.SOFTMAX)
        .build())
    .build()

val modelo = MultiLayerNetwork(conf)
modelo.init()

Integração com Python e Serviços ML

Quando o modelo vive em Python (FastAPI/Flask), Kotlin atua como cliente via Ktor HttpClient. Alternativamente, GraalVM Polyglot permite executar scripts Python diretamente no processo JVM.

Kotlin — Cliente de serviço ML Python
import io.ktor.client.*
import io.ktor.client.call.*
import io.ktor.client.engine.cio.*
import io.ktor.client.plugins.contentnegotiation.*
import io.ktor.client.request.*
import io.ktor.http.*
import kotlinx.serialization.Serializable

@Serializable
data class PredictRequest(val features: List<Double>)
@Serializable
data class PredictResponse(val label: String, val confianca: Double)

suspend fun classificar(features: List<Double>): PredictResponse {
    val client = HttpClient(CIO) { install(ContentNegotiation) { json() } }
    return client.post("http://ml-service:5000/predict") {
        contentType(ContentType.Application.Json)
        setBody(PredictRequest(features))
    }.body()
}
