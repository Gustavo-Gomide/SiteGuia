Kotlin — Desenvolvimento Web

Kotlin no backend combina a maturidade do ecossistema JVM com a expressividade da linguagem. Ktor oferece um servidor web assíncrono e idiomático; Spring Boot traz convenção enterprise. Ambos se integram naturalmente com coroutines.

Com tipos nulos tratados em tempo de compilação, coroutines nativas e DSLs expressivas, Kotlin torna a construção de APIs REST ou serviços de streaming muito menos propensa a erros do que Java ou JavaScript tipado opcionalmente.

Servidor Ktor Básico

Ktor usa coroutines para tratar requisições de forma não bloqueante. O setup mínimo requer apenas embeddedServer, um bloco routing e a engine de sua escolha (Netty, CIO ou Jetty).

Kotlin — Ktor Hello World
// build.gradle.kts (dependências essenciais)
// implementation("io.ktor:ktor-server-netty:2.3.12")
// implementation("io.ktor:ktor-server-content-negotiation:2.3.12")
// implementation("io.ktor:ktor-serialization-kotlinx-json:2.3.12")

import io.ktor.server.application.*
import io.ktor.server.engine.*
import io.ktor.server.netty.*
import io.ktor.server.routing.*
import io.ktor.server.response.*
import io.ktor.server.plugins.contentnegotiation.*
import io.ktor.serialization.kotlinx.json.*

fun main() {
    embeddedServer(Netty, port = 8080, host = "0.0.0.0", module = Application::module)
        .start(wait = true)
}

fun Application.module() {
    install(ContentNegotiation) { json() }
    routing {
        get("/") { call.respondText("Servidor Ktor rodando!") }
    }
}

API REST Completa

Rotas organizadas em funções de extensão de Route mantêm o código coeso. StatusPages centraliza o tratamento de erros. Data classes com @Serializable fazem a serialização JSON automaticamente.

Kotlin — Ktor REST API
import io.ktor.http.*
import io.ktor.server.plugins.statuspages.*
import io.ktor.server.request.*
import kotlinx.serialization.Serializable

@Serializable
data class User(val id: Long, val nome: String, val email: String)

@Serializable
data class CriarUserReq(val nome: String, val email: String)

@Serializable
data class ErroPagina(val mensagem: String)

val db = mutableListOf<User>()

fun Application.moduleApi() {
    install(ContentNegotiation) { json() }
    install(StatusPages) {
        exception<IllegalArgumentException> { call, cause ->
            call.respond(HttpStatusCode.BadRequest, ErroPagina(cause.message ?: "Erro"))
        }
        exception<NoSuchElementException> { call, _ ->
            call.respond(HttpStatusCode.NotFound, ErroPagina("Recurso não encontrado"))
        }
    }
    routing { userRoutes() }
}

fun Route.userRoutes() {
    route("/users") {
        get { call.respond(db) }
        post {
            val req = call.receive<CriarUserReq>()
            require(req.email.contains("@")) { "Email inválido" }
            val user = User(db.size + 1L, req.nome, req.email)
            db.add(user)
            call.respond(HttpStatusCode.Created, user)
        }
        get("/{id}") {
            val id = call.parameters["id"]?.toLongOrNull()
                ?: throw IllegalArgumentException("ID inválido")
            val user = db.find { it.id == id }
                ?: throw NoSuchElementException()
            call.respond(user)
        }
    }
}

Autenticação JWT

O plugin ktor-server-auth-jwt integra JSON Web Tokens. Rotas protegidas ficam dentro de blocos authenticate("jwt") { }. O token é extraído do header Authorization: Bearer ....

Kotlin — Ktor JWT
import io.ktor.server.auth.*
import io.ktor.server.auth.jwt.*
import com.auth0.jwt.JWT
import com.auth0.jwt.algorithms.Algorithm

const val SECRET = "meu-segredo"
const val ISSUER = "meu-app"

fun Application.configuraSecurity() {
    install(Authentication) {
        jwt("jwt") {
            realm = "App Kotlin"
            verifier(
                JWT.require(Algorithm.HMAC256(SECRET))
                    .withIssuer(ISSUER)
                    .build()
            )
            validate { credential ->
                if (credential.payload.getClaim("userId").asString() != null)
                    JWTPrincipal(credential.payload)
                else null
            }
        }
    }

    routing {
        post("/login") {
            val token = JWT.create()
                .withIssuer(ISSUER)
                .withClaim("userId", "123")
                .sign(Algorithm.HMAC256(SECRET))
            call.respond(mapOf("token" to token))
        }
        authenticate("jwt") {
            get("/perfil") {
                val principal = call.principal<JWTPrincipal>()
                val userId = principal!!.payload.getClaim("userId").asString()
                call.respond(mapOf("userId" to userId))
            }
        }
    }
}

Ktor HttpClient

O mesmo ecossistema Ktor oferece um cliente HTTP assíncrono com suporte a JSON, timeout, retry e WebSockets — reutilizando os mesmos plugins do servidor.

Kotlin — Ktor HttpClient
import io.ktor.client.*
import io.ktor.client.call.*
import io.ktor.client.engine.cio.*
import io.ktor.client.plugins.*
import io.ktor.client.plugins.contentnegotiation.*
import io.ktor.client.request.*
import io.ktor.client.statement.*
import io.ktor.http.*

val client = HttpClient(CIO) {
    install(ContentNegotiation) { json() }
    install(HttpTimeout) {
        requestTimeoutMillis = 10_000
        connectTimeoutMillis = 5_000
    }
    defaultRequest { header("X-App", "kotlin-client") }
}

suspend fun buscarUsers(): List<User> =
    client.get("https://api.exemplo.com/users").body()

suspend fun criarUser(req: CriarUserReq): User =
    client.post("https://api.exemplo.com/users") {
        contentType(ContentType.Application.Json)
        setBody(req)
    }.body()
