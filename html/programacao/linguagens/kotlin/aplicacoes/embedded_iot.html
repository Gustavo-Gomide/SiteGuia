Kotlin — Embedded e IoT

Kotlin na JVM roda em dispositivos como Raspberry Pi com acesso a GPIO via Pi4J. Coroutines simplificam a leitura contínua de sensores e a publicação MQTT de forma não bloqueante, enquanto Room e WorkManager garantem operação offline confiável.

A combinação de Kotlin Coroutines com Flow é natural para IoT: streams de sensores são modelados como Flow<Leitura>, permitindo backpressure, transformação e coleta assíncrona sem callbacks aninhados.

Pi4J — GPIO no Raspberry Pi

Pi4J v2 oferece uma API limpa para controle de GPIO em Kotlin. O contexto é criado via Pi4J.newAutoContext() e os pins são configurados com builders type-safe.

Kotlin — Pi4J LED blink
import com.pi4j.Pi4J
import com.pi4j.io.gpio.digital.*
import kotlinx.coroutines.*

fun main() = runBlocking {
    val pi4j = Pi4J.newAutoContext()

    val ledConfig = DigitalOutput.newConfigBuilder(pi4j)
        .id("led")
        .name("LED Vermelho")
        .address(18) // GPIO 18
        .shutdown(DigitalState.LOW)
        .initial(DigitalState.LOW)
        .build()

    val led = pi4j.create(ledConfig)

    val btnConfig = DigitalInput.newConfigBuilder(pi4j)
        .id("botao").address(17)
        .pull(PullResistance.PULL_DOWN)
        .build()
    val botao = pi4j.create(btnConfig)
    botao.addListener { e -> led.toggle() }

    // Piscar LED enquanto aguarda
    repeat(10) {
        led.high(); delay(500)
        led.low();  delay(500)
    }
    pi4j.shutdown()
}

MQTT com Eclipse Paho

Eclipse Paho é a implementação MQTT mais usada na JVM. Em Kotlin, wrappers com coroutines transformam os callbacks em código suspendível limpo.

Kotlin — MQTT Publish e Subscribe
import org.eclipse.paho.client.mqttv3.*
import kotlinx.serialization.json.Json
import kotlinx.serialization.Serializable

@Serializable
data class LeituraSensor(val id: String, val valor: Double, val ts: Long)

val cliente = MqttClient("tcp://broker.mqtt.local:1883", "kotlin-device-1")
val opcoes = MqttConnectOptions().apply {
    isAutomaticReconnect = true
    isCleanSession = false
    connectionTimeout = 10
}
cliente.connect(opcoes)

// Publicar leitura
fun publicar(leitura: LeituraSensor) {
    val payload = Json.encodeToString(leitura).toByteArray()
    cliente.publish("sensores/temperatura", payload, 1, false)
}

// Subscrever tópico
cliente.subscribe("atuadores/#", 1) { topico, msg ->
    println("[$topico] ${String(msg.payload)}")
}

Coroutines para Streams de Sensores

Flow modela leituras de sensores como streams reativos. conflate() descarta valores acumulados quando o consumidor é lento; StateFlow mantém sempre o valor mais recente.

Kotlin — Flow de sensor
import kotlinx.coroutines.*
import kotlinx.coroutines.flow.*

data class Leitura(val temp: Double, val umidade: Double)

// Simula leituras periódicas de sensor
fun fluxoSensor(): Flow<Leitura> = flow {
    while (true) {
        val leitura = Leitura(
            temp = 20.0 + Math.random() * 10,
            umidade = 40.0 + Math.random() * 20
        )
        emit(leitura)
        delay(500)
    }
}

val estadoAtual = MutableStateFlow(Leitura(0.0, 0.0))

suspend fun monitorar() = coroutineScope {
    launch {
        fluxoSensor()
            .conflate() // descarta se consumidor for lento
            .filter { it.temp > 28.0 } // apenas alertas
            .collect { leitura ->
                estadoAtual.emit(leitura)
                println("ALERTA TEMPERATURA: ${leitura.temp}°C")
            }
    }
}

Padrões Edge com Android

Para dispositivos Android embarcados (quiosques, painéis industriais), WorkManager garante execução periódica de tarefas, Room persiste dados localmente e a sincronização ocorre apenas quando há conectividade.

Kotlin — WorkManager para sincronização IoT
import androidx.work.*
import java.util.concurrent.TimeUnit

class SincronizarWorker(ctx: Context, params: WorkerParameters) : CoroutineWorker(ctx, params) {
    override suspend fun doWork(): Result {
        return try {
            val leiturasLocal = roomDao.obterNaoSincronizadas()
            apiService.enviarLeituras(leiturasLocal)
            roomDao.marcarSincronizadas(leiturasLocal.map { it.id })
            Result.success()
        } catch (e: Exception) {
            Result.retry()
        }
    }
}

// Agendar sincronização periódica com restrição de rede
val restricao = Constraints.Builder()
    .setRequiredNetworkType(NetworkType.CONNECTED)
    .build()

val req = PeriodicWorkRequestBuilder<SincronizarWorker>(15, TimeUnit.MINUTES)
    .setConstraints(restricao)
    .setBackoffCriteria(BackoffPolicy.EXPONENTIAL, 30, TimeUnit.SECONDS)
    .build()

WorkManager.getInstance(ctx).enqueueUniquePeriodicWork(
    "sincronizar", ExistingPeriodicWorkPolicy.KEEP, req
)
