Kotlin — Terminal e CLI

Kotlin é excelente para ferramentas de linha de comando: Clikt oferece um DSL idiomático para argumentos e subcomandos, Mordant adiciona output colorido com tabelas no terminal, e o Shadow JAR gera executáveis fat-jar portáteis.

Com null safety, sealed classes para modelar estados de saída e coroutines para progresso assíncrono, CLIs em Kotlin são mais seguras e expressivas do que equivalentes em Java ou scripts Shell.

Clikt — Framework CLI

Clikt usa delegates Kotlin (by argument(), by option()) para declarar os argumentos de forma type-safe. Subcomandos são classes aninhadas, e o help é gerado automaticamente.

Kotlin — Clikt CLI
import com.github.ajalt.clikt.core.*
import com.github.ajalt.clikt.parameters.arguments.*
import com.github.ajalt.clikt.parameters.options.*
import com.github.ajalt.clikt.parameters.types.*

class MeuCli : CliktCommand(help = "Ferramenta de processamento de arquivos") {
    val arquivo by argument(help = "Arquivo de entrada").file(mustExist = true)
    val saida by option("-o", "--saida", help = "Arquivo de saída").default("resultado.txt")
    val verbose by option("-v", "--verbose").flag()
    val limite by option("-n").int().default(100)

    override fun run() {
        if (verbose) echo("Processando: ${arquivo.name} → $saida")
        // processamento real aqui
        echo("Linhas processadas: $limite")
    }
}

// Subcomandos
class App : NoOpCliktCommand() {
    init { subcommands(MeuCli(), OutroComando()) }
}

fun main(args: Array<String>) = App().main(args)

Output Colorido com Mordant

Mordant (da mesma equipe do Clikt) adiciona renderização de texto rich, tabelas, progress bars e detecção automática de capacidades do terminal (cores 16/256/TrueColor, Unicode).

Kotlin — Mordant terminal rico
import com.github.ajalt.mordant.rendering.TextColors.*
import com.github.ajalt.mordant.rendering.TextStyles.*
import com.github.ajalt.mordant.table.table
import com.github.ajalt.mordant.terminal.Terminal

val t = Terminal()

t.println(bold(green("Sucesso!")))
t.println(red("Erro crítico"))
t.println(yellow("Aviso: recurso depreciado"))
t.println(cyan(italic("Info adicional")))

val tabela = table {
    header { row("Nome", "Versão", "Status") }
    body {
        row("Kotlin", "2.1.0", green("Estável"))
        row("Ktor", "2.3.12", green("Estável"))
        row("Exposed", "0.56", yellow("Beta"))
    }
}
t.println(tabela)

Processos Externos

Kotlin usa a API Java ProcessBuilder para executar comandos do sistema. Extension functions tornam a chamada mais concisa. Para pipelines de comandos, coroutines evitam bloqueio da thread principal.

Kotlin — ProcessBuilder idiomático
import java.io.File

// Extension function para executar shell command
fun String.runCommand(workDir: File? = null): String {
    val proc = ProcessBuilder("bash", "-c", this)
        .apply { workDir?.let { directory(it) } }
        .redirectErrorStream(true)
        .start()
    val saida = proc.inputStream.bufferedReader().readText()
    val codigo = proc.waitFor()
    if (codigo != 0) throw RuntimeException("Comando falhou ($codigo): $saida")
    return saida.trim()
}

// Uso direto
val arquivos = "ls -la /tmp".runCommand()
val gitLog = "git log --oneline -5".runCommand(File("."))
val versao = "docker --version".runCommand()

println(versao) // Docker version 28.0.1, ...

Empacotamento — Shadow JAR

O plugin Shadow Gradle produz um "fat JAR" com todas as dependências embutidas — distribuível como um único arquivo executável com java -jar. Para binários nativos, use GraalVM Native Image.

Kotlin — build.gradle.kts com Shadow
plugins {
    kotlin("jvm") version "2.1.0"
    application
    id("com.github.johnrengelman.shadow") version "8.1.1"
}

application {
    mainClass.set("MainKt")
}

tasks.shadowJar {
    archiveBaseName.set("minha-cli")
    archiveClassifier.set("")
    archiveVersion.set("")
    mergeServiceFiles()
}

// Build: ./gradlew shadowJar
// Run:   java -jar build/libs/minha-cli.jar --help

Dica: adicione um wrapper shell script que invoca java -jar minha-cli.jar "$@" para criar um executável transparente em Linux/Mac.
