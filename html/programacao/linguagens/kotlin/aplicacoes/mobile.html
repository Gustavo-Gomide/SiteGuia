Kotlin — Mobile Android

Kotlin é a linguagem oficial do Android desde 2019. Jetpack Compose substitui o XML por UI declarativa idiomática em Kotlin. ViewModel + StateFlow gerenciam estado de forma reativa, enquanto Retrofit e Room conectam a aplicação à rede e persistência local.

Android moderno é Kotlin-first em todos os níveis: da UI (Compose) ao gerenciamento de estado (Flow/StateFlow), da camada de rede (Retrofit) ao banco local (Room) — com coroutines unificando o modelo assíncrono.

Jetpack Compose — UI Declarativa

Compose elimina o inflating de XML. @Composable define componentes reutilizáveis. Scaffold monta a estrutura Material3 com TopAppBar, FAB e Snackbar. Navegação é gerenciada via NavController.

Kotlin — Compose tela principal
import androidx.compose.material3.*
import androidx.compose.foundation.lazy.LazyColumn
import androidx.compose.foundation.lazy.items
import androidx.compose.runtime.*
import androidx.navigation.NavController

@Composable
fun TelaLista(navController: NavController, viewModel: UserViewModel = viewModel()) {
    val uiState by viewModel.uiState.collectAsStateWithLifecycle()

    Scaffold(
        topBar = { TopAppBar(title = { Text("Usuários") }) },
        floatingActionButton = {
            FloatingActionButton(onClick = { navController.navigate("criar") }) {
                Icon(Icons.Default.Add, "Adicionar")
            }
        }
    ) { padding ->
        when (uiState) {
            is UiState.Loading -> CircularProgressIndicator()
            is UiState.Success -> LazyColumn(contentPadding = padding) {
                items((uiState as UiState.Success).users) { user ->
                    ListItem(headlineContent = { Text(user.nome) })
                }
            }
            is UiState.Error -> Text("Erro: ${(uiState as UiState.Error).mensagem}")
        }
    }
}

ViewModel + StateFlow

ViewModel sobrevive a recomposições e rotações de tela. MutableStateFlow é a fonte de verdade do estado da UI — coletado com collectAsStateWithLifecycle para respeitar o ciclo de vida.

Kotlin — ViewModel com StateFlow
import androidx.lifecycle.ViewModel
import androidx.lifecycle.viewModelScope
import kotlinx.coroutines.flow.*
import kotlinx.coroutines.launch

sealed class UiState {
    object Loading : UiState()
    data class Success(val users: List<User>) : UiState()
    data class Error(val mensagem: String) : UiState()
}

class UserViewModel(private val repo: UserRepository) : ViewModel() {
    private val _uiState = MutableStateFlow<UiState>(UiState.Loading)
    val uiState: StateFlow<UiState> = _uiState.asStateFlow()

    init { carregarUsers() }

    fun carregarUsers() {
        viewModelScope.launch {
            _uiState.emit(UiState.Loading)
            try {
                val users = repo.buscarTodos()
                _uiState.emit(UiState.Success(users))
            } catch (e: Exception) {
                _uiState.emit(UiState.Error(e.message ?: "Erro desconhecido"))
            }
        }
    }
}

Retrofit + Room — Rede e Persistência

Retrofit faz requisições HTTP com interfaces anotadas. Room é o ORM oficial do Android com suporte direto a coroutines e Flow. O padrão Repository encapsula ambos, expondo uma interface única para a ViewModel.

Kotlin — Retrofit + Room + Repository
import retrofit2.http.*
import androidx.room.*

// Retrofit interface
interface UserApi {
    @GET("/users") suspend fun getUsers(): List<UserDto>
    @POST("/users") suspend fun createUser(@Body user: UserDto): UserDto
}

// Room Entity e DAO
@Entity(tableName = "users")
data class UserEntity(
    @PrimaryKey val id: Long,
    val nome: String,
    val email: String,
    val sincronizado: Boolean = false
)

@Dao
interface UserDao {
    @Query("SELECT * FROM users") fun observarTodos(): Flow<List<UserEntity>>
    @Insert(onConflict = OnConflictStrategy.REPLACE) suspend fun inserir(users: List<UserEntity>)
    @Query("SELECT * FROM users WHERE sincronizado = 0") suspend fun naoSincronizados(): List<UserEntity>
}

// Repository — combina API e Room
class UserRepository(private val api: UserApi, private val dao: UserDao) {
    suspend fun buscarTodos(): List<User> {
        val remoto = api.getUsers()
        dao.inserir(remoto.map { it.toEntity() })
        return remoto.map { it.toDomain() }
    }
}

Kotlin Multiplatform (KMP)

KMP permite compartilhar código Kotlin entre Android e iOS (e Desktop/Web) via commonMain. A lógica de negócio, repositórios e modelos ficam no módulo compartilhado; UI fica em cada plataforma com actual/expect.

Kotlin — KMP shared module
// commonMain/kotlin/com/app/shared/UserRepository.kt
expect fun criarHttpClient(): HttpClient  // implementado por plataforma

class UserRepository {
    private val client = criarHttpClient()

    suspend fun buscarUsers(): List<User> =
        client.get("https://api.exemplo.com/users").body()
}

// androidMain/kotlin/com/app/shared/HttpClient.kt
actual fun criarHttpClient() = HttpClient(Android) {
    install(ContentNegotiation) { json() }
}

// iosMain/kotlin/com/app/shared/HttpClient.kt
actual fun criarHttpClient() = HttpClient(Darwin) {
    install(ContentNegotiation) { json() }
}
