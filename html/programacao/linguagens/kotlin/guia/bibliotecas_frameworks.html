Kotlin — Bibliotecas e Frameworks

O ecossistema Kotlin oferece ferramentas modernas para cada camada de uma aplicação: Ktor para servidores web idiomáticos, Spring Boot para aplicações enterprise, Arrow para programação funcional robusta e Exposed como ORM DSL nativo.

Kotlin herda todo o ecossistema JVM (Maven Central, Gradle) e adiciona bibliotecas projetadas especialmente para seu modelo de coroutines, null safety e DSLs expressivas.

Ktor — Framework Web Idiomático

Ktor é o framework web oficial da JetBrains, construído sobre coroutines Kotlin. Segue o modelo de plugins compostos, sem reflexão nem anotações — tudo é configurado em código Kotlin puro.

Kotlin — Ktor servidor básico
import io.ktor.server.application.*
import io.ktor.server.engine.*
import io.ktor.server.netty.*
import io.ktor.server.routing.*
import io.ktor.server.response.*
import io.ktor.server.request.*
import io.ktor.server.plugins.contentnegotiation.*
import io.ktor.serialization.kotlinx.json.*

fun main() {
    embeddedServer(Netty, port = 8080) {
        install(ContentNegotiation) { json() }
        routing {
            get("/hello") {
                call.respondText("Olá, Ktor!")
            }
            get("/users/{id}") {
                val id = call.parameters["id"]
                call.respond(mapOf("id" to id, "nome" to "Ana"))
            }
            post("/users") {
                val body = call.receive<Map<String, String>>()
                call.respond(body)
            }
        }
    }.start(wait = true)
}

HttpClient Ktor: o mesmo framework serve como cliente HTTP. Reutilize a engine CIO e os plugins de serialização tanto no servidor quanto no cliente.

Kotlin — Ktor HttpClient
val client = HttpClient(CIO) {
    install(ContentNegotiation) { json() }
    install(HttpTimeout) { requestTimeoutMillis = 5_000 }
}

val users: List<User> = client.get("https://api.exemplo.com/users").body()
client.close()

Spring Boot com Kotlin

Spring Boot funciona muito bem com Kotlin. Data classes substituem POJOs verbosos, extension functions aprimoram a API e coroutines têm suporte nativo no Spring WebFlux e Spring MVC desde Spring 5.

Kotlin — Spring Boot REST
import org.springframework.web.bind.annotation.*
import org.springframework.stereotype.Service

data class User(val id: Long, val nome: String, val email: String)

// DTO sem construtores extras ou getters/setters
data class CriarUserDto(val nome: String, val email: String)

@Service
class UserService {
    private val users = mutableListOf<User>()
    fun criar(dto: CriarUserDto): User {
        val u = User(users.size + 1L, dto.nome, dto.email)
        users.add(u)
        return u
    }
    fun listar() = users.toList()
}

@RestController
@RequestMapping("/users")
class UserController(private val service: UserService) {
    @GetMapping fun listar() = service.listar()
    @PostMapping fun criar(@RequestBody dto: CriarUserDto) = service.criar(dto)
}

Arrow — Programação Funcional

Arrow é a biblioteca de programação funcional de referência para Kotlin. Traz tipos como Either, Option, Raise e extensões para modelar erros sem exceções, tornando falhas explícitas no tipo de retorno.

Kotlin — Arrow Either
import arrow.core.*

// Either<Erro, Sucesso>
sealed class AppError
data class UsuarioNaoEncontrado(val id: Long) : AppError()
data class EmailInvalido(val email: String) : AppError()

fun validarEmail(email: String): Either<AppError, String> =
    if ("@" in email) email.right()
    else EmailInvalido(email).left()

fun processarEmail(email: String): String =
    validarEmail(email)
        .map { it.lowercase() }
        .getOrElse { err -> "Erro: $err" }

// Composição com flatMap / bindEither (Arrow DSL)
fun pipeline(email: String): Either<AppError, String> = either {
    val validado = validarEmail(email).bind()
    validado.trim().lowercase()
}

Raise DSL (Arrow 1.2+): substitui Either.bind() por um escopo de contexto mais limpo com raise(erro) e recover { }. Integra-se bem com coroutines.

Exposed — ORM DSL Kotlin

Exposed é o ORM da JetBrains para Kotlin. Oferece duas APIs: DSL (SQL type-safe via objetos Kotlin) e DAO (mapeamento por entidades). Ambas operam dentro de blocos transaction { }.

Kotlin — Exposed DSL
import org.jetbrains.exposed.sql.*
import org.jetbrains.exposed.sql.transactions.transaction

// Definição da tabela
object Users : Table("users") {
    val id = integer("id").autoIncrement()
    val nome = varchar("nome", 128)
    val email = varchar("email", 256).uniqueIndex()
    override val primaryKey = PrimaryKey(id)
}

Database.connect("jdbc:h2:mem:test", driver = "org.h2.Driver")

transaction {
    SchemaUtils.create(Users)

    Users.insert { row ->
        row[nome] = "Ana"
        row[email] = "ana@exemplo.com"
    }

    val todos = Users.selectAll().map {
        it[Users.nome] to it[Users.email]
    }
    println(todos)

    Users.update({ Users.email eq "ana@exemplo.com" }) {
        it[nome] = "Ana Silva"
    }
}

kotlinx.serialization e Demais kotlinx

As bibliotecas kotlinx.* são a família oficial de extensões Kotlin-first: serialização multiplataforma, coroutines, datetime e IO — todas projetadas para funcionar em JVM, JS e Native.

Kotlin — kotlinx.serialization
import kotlinx.serialization.*
import kotlinx.serialization.json.*

@Serializable
data class Produto(
    val id: Int,
    val nome: String,
    val preco: Double,
    @SerialName("em_estoque") val emEstoque: Boolean = true
)

val json = Json { prettyPrint = true; ignoreUnknownKeys = true }

val produto = Produto(1, "Teclado", 159.90)
val texto = json.encodeToString(produto)
// {"id":1,"nome":"Teclado","preco":159.9,"em_estoque":true}

val p2: Produto = json.decodeFromString(texto)

// Lista serializada
val lista = listOf(produto, produto.copy(id = 2, nome = "Mouse"))
val jsonLista = json.encodeToString(lista)
