C em Sistemas Embarcados e IoT

Microcontroladores, bare-metal, RTOS, ARM Cortex-M, comunicação serial, GPIO, ADC e IoT com ESP32 e STM32.

C é a língua nativa dos microcontroladores. Com restrições severas de memória (KB de RAM, não GB), sem sistema operacional e com acesso direto ao hardware via registradores, C é essencialmente a única linguagem praticável para programação bare-metal em microcontroladores de 8, 16 e 32 bits.

Por que C domina o mundo embarcado

Vantagens de C em embarcados

  • Código gerado mínimo — cabe em 16KB de flash
  • Sem garbage collector, sem runtime surpresas de latência
  • Acesso direto a registradores via ponteiros e volatile
  • Determinístico: tempo de execução previsível
  • Toolchains maduras para ARM, AVR, RISC-V, MIPS
  • Suporte de HAL (Hardware Abstraction Layer) de fabricantes

Desafios

  • Sem MMU na maioria dos MCUs — overflow de stack mata o sistema
  • Depuração complexa: JTAG/SWD, sem printf fácil
  • Gestão manual de memória ainda mais crítica que em PCs
  • Fabricantes usam C com extensões não-padrão (__attribute__)
  • Portabilidade entre arquiteturas requer cuidado com endianness

Acesso a Registradores de Hardware

Em bare-metal C, periféricos são controlados escrevendo em endereços de memória fixos. Estruturas de bit fields e ponteiros para endereços absolutos são o padrão da indústria.

C — GPIO em ARM Cortex-M (STM32 style)
#include <stdint.h>

/* Registradores GPIO do STM32F4 (endereços da memória mapeada) */
#define RCC_BASE    (0x40023800UL)
#define GPIOA_BASE  (0x40020000UL)

typedef struct {
    volatile uint32_t MODER;   /* Mode register */
    volatile uint32_t OTYPER;
    volatile uint32_t OSPEEDR;
    volatile uint32_t PUPDR;
    volatile uint32_t IDR;      /* Input data */
    volatile uint32_t ODR;      /* Output data */
    volatile uint32_t BSRR;     /* Bit set/reset */
} GPIO_TypeDef;

typedef struct {
    volatile uint32_t CR[2];
    volatile uint32_t AHB1ENR;  /* Clock enable para GPIO */
} RCC_TypeDef;

#define GPIOA ((GPIO_TypeDef*) GPIOA_BASE)
#define RCC   ((RCC_TypeDef*)  RCC_BASE)

/* LED no PA5 (Nucleo-64 board) */
int main(void) {
    /* Habilitar clock do GPIOA */
    RCC->AHB1ENR |= (1U << 0);

    /* PA5 como saída (MODER bits [11:10] = 01) */
    GPIOA->MODER &= ~(3U << (5 * 2));
    GPIOA->MODER |=  (1U << (5 * 2));

    while (1) {
        /* Setar bit 5 (ligar LED) */
        GPIOA->BSRR = (1U << 5);

        /* Delay simples */
        for (volatile int i = 0; i < 100000; i++);

        /* Resetar bit 5 (apagar LED) */
        GPIOA->BSRR = (1U << (5 + 16));

        for (volatile int i = 0; i < 100000; i++);
    }
}

RTOS — Real-Time Operating System

Quando um projeto embarcado cresce, um RTOS fornece gerenciamento de tarefas, semáforos, filas de mensagem e timers — tudo com garantias de tempo real determinístico.

FreeRTOS

RTOS mais popular do mundo. Suporta 40+ arquiteturas. Código aberto (MIT). Tarefas, semáforos, mutexes, filas e timers. Base do Amazon FreeRTOS (AWS IoT).

Zephyr RTOS

RTOS moderno da Linux Foundation. Suporte nativo a Bluetooth, Wi-Fi, Thread, Zigbee. Usado no Arduino Pro e em dispositivos médicos.

RT-Thread

RTOS popular na Ásia/China. Componentes opcionais: sistema de arquivos, pilha TCP/IP, shell, GUI. Ativo e bem mantido.

mbed OS

RTOS da ARM para Cortex-M. Abstrai CMSIS e periféricos. Suporte nativo a TLS (mbed TLS) e protocolos IoT (MQTT, CoAP).

C — FreeRTOS: duas tarefas e uma fila
#include "FreeRTOS.h"
#include "task.h"
#include "queue.h"
#include <stdio.h>

static QueueHandle_t fila;

void tarefa_produtor(void *params) {
    int valor = 0;
    while(1) {
        xQueueSend(fila, &valor, portMAX_DELAY);
        valor++;
        vTaskDelay(pdMS_TO_TICKS(500));  /* 500ms */
    }
}

void tarefa_consumidor(void *params) {
    int recebido;
    while(1) {
        if (xQueueReceive(fila, &recebido, portMAX_DELAY) == pdPASS)
            printf("Consumidor recebeu: %d\n", recebido);
    }
}

int main(void) {
    fila = xQueueCreate(10, sizeof(int));

    xTaskCreate(tarefa_produtor,  "Prod", 1024, NULL, 1, NULL);
    xTaskCreate(tarefa_consumidor, "Cons", 1024, NULL, 1, NULL);

    vTaskStartScheduler();  /* nunca retorna */
    return 0;
}
