C para Mobile

Android NDK/JNI, iOS via bridging e SDL2 mobile — como levar o desempenho máximo do C para smartphones e tablets.

O kernel Linux do Android é C. O runtime ART do Android usa código nativo C/C++. Codecs de vídeo (libvpx, libaom), engines de criptografia (OpenSSL no Android) e engines de jogos como o motor do Clash of Clans são escritos em C/C++. JNI (Java Native Interface) é a ponte que conecta o mundo Java/Kotlin do Android com o código nativo.

Android NDK + JNI: Código C Chamado do Kotlin/Java

JNI permite que código Kotlin/Java chame funções C/C++ de forma direta. O NDK cross-compila o código C para as arquiteturas alvo (armeabi-v7a, arm64-v8a, x86_64) e empacota as .so no APK.

C — implementação JNI (native-lib.c)
#include <jni.h>
#include <string.h>
#include <stdlib.h>
#include <math.h>

/*
 * Convenção de nomes JNI:
 * Java_<nome.do.pacote>_<Classe>_<método>
 * underscores duplos substituem maiúsculas no pacote
 */

/* Retorna uma string para o Kotlin */
JNIEXPORT jstring JNICALL
Java_br_dev_exemplo_MainActivity_stringNativa(
    JNIEnv *env, jobject this_obj)
{
    (void)this_obj;
    return (*env)->NewStringUTF(env, "Olá do C via JNI!");
}

/* Processar array de floats — ex: normalização de vetor */
JNIEXPORT jfloatArray JNICALL
Java_br_dev_exemplo_MainActivity_normalizarVetor(
    JNIEnv *env, jobject this_obj,
    jfloatArray entrada)
{
    (void)this_obj;
    jsize len = (*env)->GetArrayLength(env, entrada);
    jfloat *arr = (*env)->GetFloatArrayElements(env, entrada, NULL);

    double mag = 0.0;
    for (jsize i = 0; i < len; i++) mag += (double)arr[i] * arr[i];
    mag = sqrt(mag);

    jfloatArray saida = (*env)->NewFloatArray(env, len);
    jfloat *out = (*env)->GetFloatArrayElements(env, saida, NULL);

    for (jsize i = 0; i < len; i++)
        out[i] = (mag > 0.0) ? (jfloat)(arr[i] / mag) : 0.0f;

    (*env)->ReleaseFloatArrayElements(env, entrada, arr, JNI_ABORT);
    (*env)->ReleaseFloatArrayElements(env, saida,   out, 0);
    return saida;
}

/* Fibonacci recursivo — benchmark típico JNI vs Kotlin */
JNIEXPORT jlong JNICALL
Java_br_dev_exemplo_MainActivity_fibonacci(
    JNIEnv *env, jobject this_obj, jint n)
{
    (void)env; (void)this_obj;
    if (n <= 1) return (jlong)n;
    long long a = 0, b = 1;
    for (int i = 2; i <= n; i++) {
        long long t = a + b; a = b; b = t;
    }
    return (jlong)b;
}
Kotlin — chamando os métodos nativos (MainActivity.kt)
class MainActivity : AppCompatActivity() {

    external fun stringNativa(): String
    external fun normalizarVetor(entrada: FloatArray): FloatArray
    external fun fibonacci(n: Int): Long

    companion object {
        init { System.loadLibrary("native-lib") }
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        binding.tvSaudacao.text = stringNativa()

        val vetor = floatArrayOf(3.0f, 4.0f, 0.0f)
        val norm  = normalizarVetor(vetor)
        // norm ≈ [0.6, 0.8, 0.0]

        val fib = fibonacci(50)  // muito mais rápido que Kotlin puro
        binding.tvFib.text = "fib(50) = $fib"
    }
}

CMakeLists.txt para Android NDK

CMake — build de biblioteca nativa no NDK
# app/CMakeLists.txt
cmake_minimum_required(VERSION 3.22)
project(MeuProjeto)

add_library(
    native-lib      # nome da .so
    SHARED          # biblioteca compartilhada
    native-lib.c    # fontes
    vetor.c
)

# Bibliotecas NDK disponíveis
find_library(log-lib  log)          # __android_log_print
find_library(android-lib android)   # AAssetManager, ANativeWindow

target_link_libraries(
    native-lib
    PRIVATE
        ${log-lib}
        ${android-lib}
        m       # libm: sqrt, sin, cos
)

target_compile_options(native-lib PRIVATE
    -O2 -Wall -Wextra
    -fvisibility=hidden    # esconder símbolos não-JNI
)

SDL2 em Android e iOS

Android via SDL2

SDL2 inclui uma Activity Java (SDLActivity.java) que inicializa a JVM, carrega a libSDL2.so e chama SDL_main() do seu código C. Toda a lógica da aplicação permanece em C puro; o Java serve apenas como wrapper.

Passos: Copiar o template Android do SDL2, colocar seus fontes C em app/jni/src/, configurar o CMakeLists, fazer ./gradlew assembleDebug.

iOS via SDL2

No iOS, SDL2 usa o template Xcode fornecido (Org.libsdl.app). Seu main() em C substitui o main() padrão do Objective-C. O Xcode compila como .ipa com suporte a Retina, multitoque e Game Controller.

Restrição: App Store proíbe JIT; não use mmap(PROT_WRITE|PROT_EXEC). SDL2 é totalmente compatível com as regras da App Store.

Cross-platform com SDL2

O mesmo código C compila para Windows, Linux, macOS, Android e iOS. Apenas o CMakeLists.txt muda. Ideal para jogos indie, emuladores e ferramentas de processamento de mídia.

Performance: Em Android, SDL2 usa OpenGL ES 3.x ou Vulkan como backend de renderização. Em iOS, usa Metal via camada de compatibilidade OpenGL ES.
