C — Bibliotecas & Frameworks

Bibliotecas e Frameworks para C

Do padrão ISO à criptografia de nível bancário — o ecossistema de bibliotecas C cobre 50 anos de desenvolvimento e bilhões de linhas em produção.

C não tem gerenciador de pacotes oficial, mas possui um ecossistema maduro de bibliotecas que evoluiu ao longo de décadas. Da biblioteca padrão onipresente ao libevent que alimenta o Memcached, conhecer estas bibliotecas permite construir praticamente qualquer sistema.

Biblioteca Padrão C (libc)

Toda implementação C deve fornecer a biblioteca padrão definida pelo ISO. Ela cobre I/O, strings, matemática, memória, localização, tempo e muito mais — através de headers incluídos com #include.

C — I/O de arquivo com stdio.h
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    /* Escrever arquivo */
    FILE *f = fopen("dados.txt", "w");
    if (!f) { perror("fopen"); return 1; }

    for (int i = 1; i <= 5; i++)
        fprintf(f, "Linha %d\n", i);
    fclose(f);

    /* Ler arquivo linha por linha */
    f = fopen("dados.txt", "r");
    if (!f) { perror("fopen"); return 1; }

    char linha[256];
    while (fgets(linha, sizeof(linha), f))
        fputs(linha, stdout);

    fclose(f);
    return 0;
}

API POSIX

POSIX (Portable Operating System Interface) define APIs para Unix/Linux/macOS. Fornece chamadas de sistema para processos, threads, sockets, arquivos e sinais — disponíveis em Linux, macOS e WSL no Windows.

unistd.h / sys/stat.h

Operações de arquivo POSIX: open, read, write, close, mkdir, stat, unlink. Mais controle que stdio, zero buffering.

pthread.h

POSIX Threads: cria e gerencia threads com pthread_create, pthread_join, mutex, condition variables e read-write locks.

sys/socket.h

Berkeley Sockets: base de toda comunicação de rede em Unix. socket, bind, listen, accept, connect, send, recv.

sys/mman.h

Mapeamento de memória: mmap para mapear arquivos diretamente em memória virtual — extremamente eficiente para arquivos grandes.

C — servidor TCP mínimo com POSIX sockets
#include <stdio.h>
#include <string.h>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>

#define PORT 8080

int main(void) {
    int srv = socket(AF_INET, SOCK_STREAM, 0);

    struct sockaddr_in addr = {
        .sin_family      = AF_INET,
        .sin_addr.s_addr = INADDR_ANY,
        .sin_port        = htons(PORT)
    };
    bind(srv, (struct sockaddr*)&addr, sizeof(addr));
    listen(srv, 5);
    printf("Escutando na porta %d\n", PORT);

    int cli = accept(srv, NULL, NULL);
    const char *resp =
        "HTTP/1.1 200 OK\r\nContent-Length: 13\r\n\r\nHello, World!";
    write(cli, resp, strlen(resp));
    close(cli);
    close(srv);
    return 0;
}
/* gcc -o srv srv.c && ./srv
   Em outro terminal: curl http://localhost:8080 */

Bibliotecas de Terceiros

OpenSSL — criptografia e TLS

Usada em Apache, nginx, curl, OpenSSH e na maioria dos softwares que precisam de TLS/SSL. Fornece algoritmos simétricos (AES, ChaCha20), hash (SHA-256, SHA-3), chave assimétrica (RSA, ECDSA) e o protocolo TLS completo.

C — SHA-256 com OpenSSL
#include <stdio.h>
#include <string.h>
#include <openssl/sha.h>

int main(void) {
    const unsigned char *msg = (unsigned char*)"Hello, C!";
    unsigned char hash[SHA256_DIGEST_LENGTH];

    SHA256(msg, strlen((char*)msg), hash);

    printf("SHA-256: ");
    for (int i = 0; i < SHA256_DIGEST_LENGTH; i++)
        printf("%02x", hash[i]);
    puts("");
    return 0;
}
/* gcc -o sha sha.c -lssl -lcrypto */

SQLite — banco de dados embutido

Um único arquivo .c de ~200 mil linhas. Usado em todos os iPhones/Androids, Firefox, Chrome, Dropbox e no sistema de arquivos do Airbus A350. A biblioteca C mais usada do mundo por dispositivos únicos.

C — SQLite básico
#include <stdio.h>
#include <sqlite3.h>

int callback(void *NotUsed, int argc, char **argv, char **col) {
    for (int i = 0; i < argc; i++)
        printf("%s = %s\n", col[i], argv[i] ? argv[i] : "NULL");
    return 0;
}

int main(void) {
    sqlite3 *db;
    char *err = NULL;

    sqlite3_open(":memory:", &db);  /* banco em memória */

    sqlite3_exec(db,
        "CREATE TABLE usuarios (id INT, nome TEXT);"
        "INSERT INTO usuarios VALUES (1, 'Ana');"
        "INSERT INTO usuarios VALUES (2, 'Bruno');",
        NULL, NULL, &err);

    sqlite3_exec(db,
        "SELECT * FROM usuarios;",
        callback, NULL, &err);

    sqlite3_close(db);
    return 0;
}
/* gcc -o db db.c -lsqlite3 */

libcurl — transferência de dados de rede

Suporta HTTP, HTTPS, FTP, SMTP, SCP, WebDAV e mais de 20 protocolos. Usada em PHP, Python requests por baixo, wget, e incontáveis aplicações embarcadas.

C — GET HTTP com libcurl
#include <stdio.h>
#include <curl/curl.h>

size_t write_cb(void *data, size_t sz, size_t nmemb, void *userp) {
    return fwrite(data, sz, nmemb, (FILE*)userp);
}

int main(void) {
    curl_global_init(CURL_GLOBAL_DEFAULT);
    CURL *curl = curl_easy_init();

    curl_easy_setopt(curl, CURLOPT_URL, "https://httpbin.org/get");
    curl_easy_setopt(curl, CURLOPT_WRITEFUNCTION, write_cb);
    curl_easy_setopt(curl, CURLOPT_WRITEDATA, stdout);

    CURLcode res = curl_easy_perform(curl);
    if (res != CURLE_OK)
        fprintf(stderr, "curl: %s\n", curl_easy_strerror(res));

    curl_easy_cleanup(curl);
    curl_global_cleanup();
    return 0;
}
/* gcc -o req req.c -lcurl */

Alternativas à libc padrão

A implementação padrão da libc varia por plataforma (glibc no Linux, musl, MSVCRT no Windows). Para sistemas embarcados e containers, alternativas leves são preferidas.

musl libc

  • Implementação clean-room do padrão POSIX
  • Binários estáticos minúsculos — ideal para containers Docker
  • Padrão em Alpine Linux — base de muitas imagens Docker
  • Suporte completo a Unicode
  • Licença MIT — embarcados comerciais sem royalties

Newlib

  • libc para sistemas embarcados sem SO (bare-metal)
  • Usada com ARM GCC toolchain para microcontroladores
  • Suporte a ARM Cortex-M, RISC-V, AVR
  • Implementação de syscalls customizável pelo hardware alvo
