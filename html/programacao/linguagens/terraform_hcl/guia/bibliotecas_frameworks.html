Terraform e HCL - Bibliotecas e Frameworks

Ecossistema de Components para IaC

Providers, modulos e ferramentas de plataforma

Terraform/HCL nao usa bibliotecas no sentido classico de runtime, mas possui um ecossistema robusto de providers, modulos e ferramentas de composicao.

Dominar o ecossistema Terraform significa saber escolher blocos reutilizaveis com governanca, seguranca e baixo acoplamento.

Conceitos Fundamentais

Em Terraform, o equivalente a "biblioteca" e o provider (plugin que fala com API da plataforma) e o equivalente a "framework" e o conjunto de padroes, modulos e wrappers que estruturam ambientes.

Elementos centrais

  • Providers: AWS, AzureRM, Google, Kubernetes, Helm, Vault.
  • Modulos: blocos reutilizaveis para VPC, banco, cluster, observabilidade.
  • Tooling: Terragrunt, TFLint, Checkov, tfsec, Infracost.

Como o Ecossistema se Conecta

Categorias de Ferramentas

Providers oficiais

Conectam Terraform a APIs de nuvem e servicos de plataforma.

Modulos reutilizaveis

Encapsulam padroes de arquitetura para acelerar provisionamento.

Wrappers

Terragrunt e similares ajudam a reduzir repeticao entre ambientes.

Qualidade e seguranca

TFLint, tfsec e Checkov bloqueiam configuracoes arriscadas no pipeline.

Exemplo de Composicao Provider + Modulo

hcl - provider, modulo e policy checks
# Provider principal da AWS
provider "aws" {
  region = "us-east-1"
}

# Reuso de modulo de rede padronizado
module "network" {
  source = "terraform-aws-modules/vpc/aws"
  name   = "plataforma-core"
  cidr   = "10.40.0.0/16"
}

# Recurso de aplicacao usando output do modulo
resource "aws_security_group" "app_sg" {
  name   = "app-sg"
  vpc_id = module.network.vpc_id
}

Esse padrao evita duplicacao de configuracao e facilita evolucao com versionamento de modulo.

Quando Usar Cada Bloco

Use modulos quando

  • ha padroes repetidos em varios ambientes.
  • o time precisa de interface declarativa unica.
  • voce quer governanca por versao.

Use recursos diretos quando

  • a necessidade e pontual e simples.
  • o modulo adicionaria sobrecarga desnecessaria.
  • ha experimento tecnico de curto prazo.

Padroes Avancados de Plataforma

Catalogo interno de modulos

Empresas maduras mantem registry privado com modulos aprovados por seguranca. Isso padroniza tags, criptografia, backup e politicas de rede.

Armadilha comum: copiar modulo publico sem revisar defaults de seguranca e custos.
