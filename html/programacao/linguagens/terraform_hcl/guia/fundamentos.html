Terraform e HCL: Fundamentos

Base Solida para Terraform

Recursos, variaveis, outputs e modulos na pratica

Aprenda os blocos que aparecem em praticamente todo projeto HCL.

Se voce dominar recursos, variaveis, dependencias e modulos, ja consegue montar stacks reais de infraestrutura.

Recursos e Atributos

O bloco resource e a unidade principal do Terraform: ele representa algo que sera criado, alterado ou removido.

hcl - recurso basico
# Recurso EC2 basico para ambiente inicial
			resource "aws_instance" "web" {
	ami           = "ami-0c02fb55956c7d316"
	instance_type = "t3.micro"
	tags = {
		Name = "web-dev-01"
	}
}

O primeiro texto identifica o tipo do recurso e o segundo e o nome logico local dentro da configuracao.

Variaveis e Outputs

Variaveis permitem parametrizar configuracoes entre ambientes. Outputs expõem informacoes uteis para consumo humano ou de outros sistemas/pipelines.

hcl - variable
variable "region" {
	type        = string
	description = "Regiao AWS"
	default     = "us-east-1"
}
hcl - output
output "public_ip" {
	value       = aws_instance.web.public_ip
	description = "IP publico da instancia web"
}

Data Sources e Leitura de Estado Externo

Data sources consultam informacoes existentes sem gerenciar o ciclo de vida do recurso. Sao essenciais para integrar com infraestrutura ja provisionada.

hcl - data source
data "aws_ami" "ubuntu" {
	most_recent = true
	owners      = ["099720109477"]

	filter {
		name   = "name"
		values = ["ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*"]
	}
}

Modulos Basicos e Reuso

Modulo e um conjunto de arquivos Terraform reutilizavel. O modulo raiz chama modulos filhos para organizar responsabilidades.

hcl - chamada de modulo
module "network" {
	source      = "./modules/network"
	project     = var.project
	environment = var.environment
	cidr_block  = "10.20.0.0/16"
}

Boas interfaces de modulo possuem poucas variaveis obrigatorias, defaults coerentes e outputs claros.

Dependencias e Ordem de Execucao

Terraform infere dependencias quando um recurso referencia atributo de outro. Use depends_on apenas quando a dependencia nao puder ser inferida.

hcl - dependencia explicita
resource "aws_instance" "app" {
	ami           = data.aws_ami.ubuntu.id
	instance_type = "t3.micro"

	depends_on = [aws_security_group.app_sg]
}

Dependencias artificiais em excesso aumentam tempo de deploy e dificultam manutencao.
