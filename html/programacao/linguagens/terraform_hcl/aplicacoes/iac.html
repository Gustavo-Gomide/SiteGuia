Terraform e HCL: Aplicacao IaC

Provisionamento Completo de Stack

Rede + Compute + Storage com pipeline reproduzivel

Exemplo pratico de IaC para uma aplicacao web de medio porte.

A meta aqui e transformar arquitetura em codigo auditavel, do subnet ao bucket, com entrega controlada em CI/CD.

Arquitetura do Exemplo

O cenario inclui VPC, sub-redes publicas/privadas, grupo de seguranca, instancias de aplicacao e bucket para artefatos/logs.

Exemplo Real de Configuracao

hcl - stack simplificada
# Composicao de stack por modulos de dominio
			module "network" {
	source      = "./modules/network"
	cidr_block  = "10.40.0.0/16"
	environment = var.environment
}

module "compute" {
	source          = "./modules/compute"
	subnet_ids      = module.network.private_subnet_ids
	security_groups = [module.network.app_sg_id]
	instance_count  = 2
}

resource "aws_s3_bucket" "artifacts" {
	bucket = "acme-${var.environment}-artifacts"
	tags   = local.common_tags
}

Em projetos reais, cada modulo fica em pasta separada com variables.tf, outputs.tf e main.tf.

Pipeline de Entrega para IaC

Pipeline recomendada por ambiente: validacao obrigatoria em PR e apply controlado por aprovacao.

yaml - CI/CD com aprovacao
stages:
	- lint
	- plan
	- approve
	- apply

lint:
	script:
		- terraform fmt -check -recursive
		- terraform validate

plan:
	script:
		- terraform plan -out=tfplan

apply:
	script:
		- terraform apply tfplan

Nunca rode apply automatico em producao sem gate humano e trilha de auditoria.
