Ecossistema e Tooling TypeScript

tsx e ts-node para desenvolvimento, tsup para bundling de bibliotecas, Vitest com tipagem, type-check em CI e publicar pacotes com declarations corretas.

O maior custo de TypeScript e o passo de compilacao. Ferramentas modernas como tsx (esbuild) e Bun executam TS diretamente sem transpilacao observavel. tsup usa esbuild para gerar ESM + CJS em menos de 100ms com .d.ts automatico.

Executando TypeScript

tsc (oficial)

Compilador oficial da Microsoft. Faz type-checking completo. Mais lento, mas o mais correto. Use em CI para tsc --noEmit.

tsx

Executa arquivos .ts/.tsx via esbuild sem type-check. Ideal para scripts e desenvolvimento. npx tsx src/index.ts.

Bun

Runtime que executa TypeScript nativamente. Zero config. Performance muito superior ao Node.js em muitos benchmarks.

ts-node

O original, usa swc internamente na versao 10+. Mais configuravel. Use com Node.js --loader ts-node/esm.

Vitest com TypeScript

TypeScript + Vitest — Testes tipados com expectTypeOf
// src/__tests__/usuario.test.ts
import { describe, it, expect, expectTypeOf, vi, beforeEach } from "vitest";
import { UsuarioService } from "../services/usuario.js";
import { z } from "zod";

const UsuarioSchema = z.object({ id: z.string(), nome: z.string() });
type Usuario = z.infer<typeof UsuarioSchema>;

describe("UsuarioService", () => {
  let service: UsuarioService;

  beforeEach(() => {
    service = new UsuarioService();
  });

  it("buscarPorId retorna usuario correto", async () => {
    const usuario = await service.buscarPorId("1");
    expect(usuario).toBeDefined();
    expect(usuario?.nome).toBe("Alice");

    // Verificar tipo em tempo de compilacao (não em runtime)
    expectTypeOf(usuario).toEqualTypeOf<Usuario | null>();
  });

  it("criar lanca erro com dados invalidos", async () => {
    await expect(service.criar({ nome: "" })).rejects.toThrow();
  });

  // Mock de modulo
  it("chama repositorio ao criar", async () => {
    const spy = vi.spyOn(service["repo"], "salvar").mockResolvedValue({
      id: "novo-id", nome: "Bob",
    });
    await service.criar({ nome: "Bob" });
    expect(spy).toHaveBeenCalledOnce();
    expect(spy).toHaveBeenCalledWith({ nome: "Bob" });
  });
});

CI e Publicacao de Pacotes

GitHub Actions — CI com type-check, testes e publish
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main]
  pull_request:

jobs:
  verificar:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [20, 22]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: npm

      - run: npm ci
      - run: npm run check   # tsc --noEmit
      - run: npm run test --coverage
      - run: npm run build  # tsup

  publicar:
    needs: verificar
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    permissions:
      contents: write
      id-token: write  # npm provenance
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          registry-url: https://registry.npmjs.org
      - run: npm ci
      - run: npm run build
      - run: npm publish --provenance --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
