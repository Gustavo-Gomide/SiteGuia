Prolog para Web

Visao Geral

APIs Declarativas com SWI-Prolog HTTP

Regras de negocio explicitas, validacao logica e servicos REST com serializacao JSON.

Em aplicacoes web, Prolog se destaca quando a regra de negocio muda rapido e precisa ser auditavel: voce troca ifs complexos por uma base de conhecimento formal.

Arquitetura Web Declarativa

Em uma stack tradicional, controladores imperativos fazem validacao, autorizacao e composicao de resposta. Em Prolog, a rota vira ponto de entrada e a decisao de negocio vive em predicados de regra.

Analogia: o servidor HTTP e a recepcao; o motor logico e o especialista que decide segundo normas. Isso reduz codigo duplicado quando multiplos endpoints compartilham as mesmas politicas.

Servidor HTTP, Rotas REST e JSON

O pacote library(http/http_dispatch) organiza rotas; library(http/http_json) le e escreve JSON; o corpo da regra implementa validacao e inferencia.

Endpoint REST com SWI-Prolog 
:- use_module(library(http/thread_httpd)).
:- use_module(library(http/http_dispatch)).
:- use_module(library(http/http_json)).

:- http_handler(root(api/avaliar), avaliar_handler, []).

server(Porta) :-
  http_server(http_dispatch, [port(Porta)]).

avaliar_handler(Request) :-
  http_read_json_dict(Request, DictIn),
  autoriza(DictIn.usuario, DictIn.acao, Resultado),
  reply_json_dict(_{permitido:Resultado}).

autoriza("admin", _, true).
autoriza(_, "deletar", false).
autoriza(_, _, true).

Regras de Negocio e Validacao Logica

Validacoes complexas (combinar perfil, horario, estado da entidade, limite de tentativas) ficam mais legiveis em clausulas declarativas. Alem de decidir, Prolog pode explicar por que negou uma operacao.

Padrao recomendado

1. Predicados puros para regra.

2. Predicados de infraestrutura para HTTP e serializacao.

3. Predicados de auditoria para justificar resultado.

Autenticacao Basica e Token

Para cenarios simples, o SWI-Prolog suporta autenticacao HTTP basica. Em APIs modernas, o caminho mais comum e validar token JWT em camada externa (Nginx, API gateway, servico auth) e passar claims para o motor Prolog.

Validacao simplificada de token (conceitual) 
permitido(Claims, Acao) :-
  member(role=admin, Claims),
  acao_critica(Acao).

permitido(Claims, Acao) :-
  member(scope=Scope, Claims),
  scope_aceita(Scope, Acao).

Deploy, Escalabilidade e Limitacoes

Deploy tipico: container Docker com SWI-Prolog atras de proxy reverso. Prolog lida bem com APIs de regra, mas para workloads massivos de streaming e CPU numerica pesada, uma arquitetura hibrida costuma ser mais eficiente.

Evite acoplar logica de autenticao criptografica complexa ao mesmo modulo de inferencia: separe responsabilidade para reduzir risco operacional.
