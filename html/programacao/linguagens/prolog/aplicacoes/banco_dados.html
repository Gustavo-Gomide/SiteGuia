Prolog com Banco de Dados

Visao Geral

SQL + Inferencia Logica em Arquitetura Hibrida

Use o banco relacional para persistencia e Prolog para regras, consistencia semantica e decisao.

Integrar Prolog com banco nao significa substituir SQL; significa elevar o nivel da regra de negocio com um motor declarativo que consulta e deduz.

Modelo de Integracao

O banco relacional continua fonte de verdade transacional. Prolog atua como camada de raciocinio: recebe dados, aplica regras e devolve conclusoes ou validacoes.

Analogia: SQL organiza e recupera registros; Prolog interpreta o significado dessas relacoes para responder perguntas de mais alto nivel.

ODBC e PostgreSQL no SWI-Prolog

A biblioteca library(odbc) permite conectar em PostgreSQL (via driver ODBC), executar consultas e mapear linhas para termos Prolog.

Consulta SQL com ODBC (exemplo didatico) 
% Exemplo didatico de consulta parametrizada via ODBC.
:- use_module(library(odbc)).

abrir_conexao(Conn) :-
  odbc_connect(meu_dsn, Conn, [alias(pg), open(once)]).

buscar_pedidos(Conn, ClienteId, Linhas) :-
  odbc_query(
    Conn,
    'SELECT id, status, total FROM pedidos WHERE cliente_id = ?',
    Linhas,
    [parameters([ClienteId])]
  ).

Mapeamento Fato/Regra para Consultas

Depois de buscar dados SQL, converta linhas em fatos temporarios para inferencia. Esse padrao simplifica a leitura da regra e reduz acoplamento ao formato bruto da tabela.

Mapeando linhas para fatos de dominio 
% Normaliza a linha SQL em fatos de dominio para inferencia.
carrega_pedido(row(Id, Status, Total)) :-
  assertz(pedido(Id, Status, Total)).

pedido_risco(Id) :-
  pedido(Id, atrasado, Total),
  Total > 10000.

Em producao, prefira escopo controlado (modulos, transacoes ou contexto de requisicao) para evitar fato residual entre chamadas concorrentes.

Transacoes e Consistencia

Regras de consistencia podem ser avaliadas antes do commit SQL. Se a inferencia detectar violacao, a transacao e revertida, preservando integridade semantica alem das constraints do schema.

Fluxo robusto

1. Iniciar transacao no banco.

2. Ler estado relevante e inferir regras em Prolog.

3. Aplicar mudancas SQL apenas se as regras forem satisfeitas.

4. Commit ou rollback com log de justificativa.

Abordagem Hibrida: SQL para Dados, Prolog para Decisao

SQL deve continuar responsavel por filtros massivos, joins e agregacoes de alto volume. Prolog entra depois, sobre um subconjunto de dados relevante, para inferir politicas, classificacoes e recomendacoes.

Essa divisao melhora desempenho e manutenibilidade: cada tecnologia atua onde possui maior expressividade e custo operacional menor.

Padroes Avancados

Projetos maduros adotam cache de resultados inferidos, versionamento de regras e testes de regressao logica acoplados ao pipeline de migracoes SQL.

Outra pratica util e armazenar explicacoes da decisao (predicados disparados) junto ao evento de negocio para auditoria e compliance.
