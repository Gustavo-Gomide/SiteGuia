Scala - Fundamentos

Fundamentos de Scala

Base conceitual para escrever codigo tipado, expressivo e seguro na JVM.

Dominar tipos, inferencia, pattern matching e tratamento funcional de erros e o caminho mais rapido para produzir Scala de alta qualidade.

Tipos e Inferencia

Scala infere tipos em muitos contextos, mas sem abrir mao de verificacao estatia rigorosa. O compilador deduz tipos locais e ainda garante contratos entre modulos.

scala - tipos basicos e inferencia
val idade = 27 // Int inferido
val nome: String = "Lia"
val ativo = true

def dobro(x: Int): Int = x * 2
println(dobro(idade))

case class e Pattern Matching

case class reduz boilerplate e combina naturalmente com match, formando a base para modelagem de dominio com ADTs.

scala - ADT simples
sealed trait Pagamento
case class Pix(chave: String) extends Pagamento
case class Cartao(finalNumero: String) extends Pagamento

def descricao(p: Pagamento): String = p match {
	case Pix(ch) => s"PIX: $ch"
	case Cartao(f) => s"Cartao final $f"
}

Com sealed, o compilador avisa quando um caso nao foi coberto, prevenindo bugs de logica.

Colecoes Imutaveis

No pacote padrao, List, Vector e Map imutaveis favorecem paralelismo e previsibilidade. Operacoes retornam novas estruturas em vez de alterar estado global.

Regra pratica: comece com colecoes imutaveis; use mutaveis apenas em trechos quentes de performance, isolados e medidos.

Option, Either e Try

Em vez de nulos e excecoes como fluxo normal, Scala incentiva tipos explicitos para sucesso/falha.

scala - erros sem null
import scala.util.Try

def parseInt(s: String): Either[String, Int] =
	Try(s.toInt).toEither.left.map(_ => "Numero invalido")

val resultado = parseInt("42").map(_ + 1)
println(resultado) // Right(43)

Use Option para ausencia simples; Either quando for importante explicar o erro.
