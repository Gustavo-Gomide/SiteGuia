Scala - Avancado

Scala Avancado

Abstracoes de alto nivel para APIs seguras, composicao poderosa e alto desempenho.

No nivel avancado, Scala deixa de ser apenas sintaxe elegante e vira uma ferramenta de modelagem de dominio com verificacao em tempo de compilacao.

Givens, Using e Typeclasses

Scala 3 reorganizou implicitos em uma sintaxe mais explicita: given para prover instancias e using para consumir dependencias contextuais.

scala - typeclass com given/using
trait Show[A]:
	def render(a: A): String

given Show[Int] with
	def render(a: Int): String = s"Int($a)"

def imprimir[A](a: A)(using sh: Show[A]): String = sh.render(a)

Resolucao de Implicitos

O compilador busca instancias no escopo local, importacoes, companions e contexto lexical. Entender essa ordem evita ambiguidade e comportamentos inesperados.

Armadilhas comuns

1) duas instancias do mesmo tipo no mesmo escopo; 2) imports globais com givens conflitantes; 3) uso excessivo de conversoes implicitas.

Prefira givens locais por modulo de dominio e evite "magic imports" em codigo critico.

Macros e Metaprogramacao no Scala 3

Com Quotes e Splices, e possivel gerar codigo em compilacao de forma tipada, reduzindo reflexao em runtime e melhorando seguranca.

scala - ideia de macro inline
inline def assertInline(inline cond: Boolean): Unit =
	if !cond then
		throw new RuntimeException("Falha de assert")

// Em cenarios reais, usa-se quoted.Expr para inspecao de AST.

Concorrencia: Futures, Cats Effect e ZIO

A biblioteca padrao oferece Future; ecossistemas FP adicionam controle de efeito, cancelamento, retries e composicao deterministica.

scala - Future com composicao
import scala.concurrent.Future
import scala.concurrent.ExecutionContext.Implicits.global

val f1 = Future(40)
val f2 = Future(2)
val soma = for
	a <- f1
	b <- f2
yield a + b

soma.foreach(v => println(s"Resultado: $v"))
