Scala - Ecossistema e Tooling

Ferramentas do Dia a Dia em Scala

Build, dependencia, formato, refatoracao, IDE e testes em uma esteira produtiva.

Produtividade em Scala depende de uma cadeia bem configurada: resolver dependencias rapido, padronizar codigo automaticamente e validar qualidade no CI.

sbt, scala-cli e coursier

sbt segue padrao para projetos grandes; scala-cli acelera prototipagem e scripts; coursier gerencia JVM, Scala e apps de forma leve.

bash - fluxo inicial
# instalar scala-cli via coursier
cs install scala-cli

# criar e executar exemplo
scala-cli run Main.scala

# para projeto robusto
sbt test

Scalafmt e Scalafix

Scalafmt normaliza estilo automaticamente. Scalafix aplica regras semanticas para migracoes e remocao de anti-patterns.

hocon - .scalafmt.conf
version = "3.8.2"
maxColumn = 100
runner.dialect = scala3

// padrao minimamente estrito para times

Boa pratica: rode format e fix no pre-commit e bloqueie PR sem conformidade no CI.

Metals e Suporte de IDE

Metals fornece LSP completo: autocompletacao, jump-to-definition, diagnostics e code actions. Em VS Code, ele e o caminho padrao para projetos Scala modernos.

Testes: ScalaTest, MUnit, Weaver

Em equipes, e comum combinar testes unitarios (rapidos), de propriedade (ScalaCheck) e integracao (containers, banco real). A piramide de testes continua valida em Scala.

scala - teste com MUnit
class SomaSuite extends munit.FunSuite:
	test("soma basica") {
		val res = 2 + 2
		assertEquals(res, 4)
	}
