Scala - Projetos e Boas Praticas

Projetos Scala em Producao

Como organizar codigo, reduzir riscos e manter evolucao sustentavel em sistemas reais.

Boas praticas em Scala comecam na modelagem de dominio e terminam no pipeline de entrega: arquitetura, testes, observabilidade e governanca tecnica caminham juntos.

Arquitetura por Modulos

Separar modulos por dominio (ex.: faturamento, identidade, catalogo) reduz acoplamento e melhora ownership. Use multiplos subprojetos sbt para isolar dependencias.

scala - exemplo conceitual de modulo
trait PedidoRepo:
	def buscar(id: Long): Option[Pedido]

class PedidoService(repo: PedidoRepo):
	def totalComImposto(id: Long): Either[String, BigDecimal] = ??? // regra de dominio isolada

Clean Code com Estilo Funcional

Evite efeitos colaterais escondidos. Prefira funcoes puras para regra de negocio, mantendo IO na borda da aplicacao.

  • Modelagem explicita com ADTs.
  • Funcoes pequenas, com nomes de negocio.
  • Dependencias injetadas via construtor ou contexto controlado.

Tratamento de Erros e Resiliencia

Erros previsiveis devem ser modelados em tipos (Either, ADTs de erro). Excecoes ficam para falhas nao recuperaveis ou integracao inesperada.

Nunca "engula" excecoes em camada de infraestrutura sem log estruturado e contexto de correlacao.

scala - erro tipado por dominio
sealed trait ErroPagamento
case object LimiteExcedido extends ErroPagamento
case object SaldoInsuficiente extends ErroPagamento

def autorizar(valor: BigDecimal): Either[ErroPagamento, Unit] = ???

Observabilidade

Logs estruturados, metricas e tracing distribuidos sao obrigatorios em microsservicos. Em JVM, e comum combinar SLF4J + OpenTelemetry + Prometheus.

CI e Qualidade Continua

Esteira minima recomendada: formatar, analisar, testar, empacotar e publicar artefato versionado.

yaml - pipeline essencial
steps:
	- run: sbt scalafmtCheckAll
	- run: sbt test
	- run: sbt package
