Scala - Bibliotecas e Frameworks

Ecossistema de Bibliotecas Scala

Do backend HTTP ao processamento distribuido, Scala oferece um stack maduro para sistemas complexos.

A escolha da biblioteca define estilo arquitetural, modelo de concorrencia e custo de manutencao. Entender o papel de cada peca evita acoplamentos errados.

Play Framework e http4s

Play favorece produtividade em aplicacoes web completas; http4s privilegia composicao funcional e tipagem forte em APIs.

scala - rota simples em http4s
import org.http4s.dsl.io._
import cats.effect.IO

val rotas = HttpRoutes.of[IO] {
	case GET -> Root / "health" => Ok("ok")
}

Akka/Pekko, ZIO e FS2

Akka/Pekko oferecem atores e streams para sistemas reativos; ZIO entrega runtime de efeitos com cancelamento e supervisao; FS2 foca em streams puros e composicao declarativa.

Akka/Pekko

Modelo actor para isolamento de estado e processamento concorrente distribuido.

ZIO

Efeitos tipados, camadas de dependencia e forte ergonomia para resiliencia.

FS2

Pipelines de stream com backpressure, recurso seguro e interop com Cats Effect.

Cats e Typelevel

Cats padroniza typeclasses funcionais (Functor, Monad, Applicative) e permite composicao de codigo generico com leis matematicas verificaveis.

scala - EitherT com Cats
import cats.data.EitherT
import cats.effect.IO

type AppResult[A] = EitherT[IO, String, A]

def buscarUsuario(id: Long): AppResult[String] =
	EitherT.right(IO.pure(s"user-$id"))

Apache Spark

Spark em Scala e o caminho mais direto para jobs distribuidos com DataFrame, Dataset e MLlib, com forte integracao no ecossistema Hadoop/Kubernetes.

scala - transformacao com Spark
val df = spark.read.option("header", "true").csv("dados.csv")
val resumo = df.groupBy("categoria").count()
resumo.show()

Comparacao Rapida

  • Play: full-stack web MVC, rapido para times heterogeneos.
  • http4s: APIs funcionais minimalistas.
  • Akka/Pekko: concorrencia orientada a atores.
  • ZIO: efeito tipado e runtime robusto.
  • Spark: processamento de dados em cluster.
