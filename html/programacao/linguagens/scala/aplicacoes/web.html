APIs REST tipadas e performaticas com Play e http4s.

Scala e especialmente forte em APIs de dominio complexo, onde tipagem e concorrencia segura reduzem incidentes em producao.

Play vs http4s

Play acelera projetos full-stack; http4s prioriza composicao funcional minimalista.

  • Play: roteamento convencional, integracao facil com teams JVM classicos.
  • http4s: alto controle funcional, melhor alinhamento com Cats Effect.

APIs REST e JSON

scala - endpoint REST http4s
case class Produto(id: Long, nome: String)

val routes = HttpRoutes.of[IO] {
	case GET -> Root / "produtos" / LongVar(id) =>
		Ok(s"{\"id\":$id,\"nome\":\"Teclado\"}")
}

Para JSON tipado, use circe, play-json ou zio-json com codecs derivadas.

Autenticacao e Middleware

Autenticacao baseada em JWT e comum em APIs Scala. Middleware aplica concerns transversais: logging, correlacao e limitacao de taxa.

scala - middleware conceitual
def auth(token: String): Boolean = token.nonEmpty

def withAuth(token: String)(action: => IO[String]): IO[String] =
	if auth(token) then action
	else IO.pure("401 unauthorized")

Nunca valide JWT apenas no gateway; a API tambem deve validar assinatura e expiracao.

Escalabilidade e Observabilidade

Use cache, circuit breakers e tracing distribuido para resiliencia. Em alto throughput, prefira pools de conexao ajustados e serializacao eficiente.
