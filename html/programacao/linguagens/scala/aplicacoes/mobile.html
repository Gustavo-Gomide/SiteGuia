Scala - Aplicacoes Mobile

Scala e Mobile

Uso pragmático: interop com Android/JVM e lógica compartilhada em camadas de dominio.

Scala no mobile funciona melhor como componente de lógica e domínio, enquanto UI nativa normalmente permanece em Kotlin/Swift.

Android com Kotlin + Scala

A interoperabilidade JVM permite chamar código Scala em módulos Android, especialmente para regras de negócio e validações complexas.

scala - modulo de dominio reutilizavel
object Pricing:
	def calcularTotal(base: BigDecimal, taxa: BigDecimal): BigDecimal =
		base + (base * taxa)

KMM e Logica Compartilhada no Backend

Em cenarios multiplataforma, Kotlin Multiplatform cuida da camada cliente compartilhada, enquanto Scala pode atuar em servicos e regras centrais no backend.

Padrão comum: Mobile em Kotlin/Swift, APIs em Scala, contratos OpenAPI e testes de contrato automatizados.

Limitacoes Praticas

Tooling Android historicamente favorece Kotlin/Java. Em equipes mobile puras, Scala pode aumentar curva de aprendizagem sem ganho claro na camada de UI.

Use Scala no mobile apenas quando houver ganho objetivo de reuso de dominio ou padrao arquitetural JVM consolidado.
