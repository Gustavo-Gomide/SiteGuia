C++ — header com include guards, forward declarations e separacao de API
// include/meu-projeto/calculadora.hpp
#pragma once              // equivalente a include guard, mais simples

#include <string_view>
#include <optional>
#include <stdexcept>

// Forward declaration — evita incluir header inteiro quando possivel
class CalculadoraImpl;    // implementacao interna (pimpl)

namespace mproj {

class Calculadora {
public:
  Calculadora();
  ~Calculadora();

  // Operacoes sem-excecao quando possivel
  [[nodiscard]] double somar(double a, double b) const noexcept;
  [[nodiscard]] double subtrair(double a, double b) const noexcept;

  // std::optional — alternativa idiomatica a throw para erros esperados
  [[nodiscard]] std::optional<double> dividir(double a, double b) const noexcept;

  // Evalua expressao matematica em string
  [[nodiscard]] double eval(std::string_view expr);

private:
  std::unique_ptr<CalculadoraImpl> impl_;  // Pimpl: isola detalhes
};

}  // namespace mproj

Idiomas e Padroes C++ Modernos

C++ — CRTP (Curiously Recurring Template Pattern)
// CRTP — polimorfismo estatico sem overhead de virtual
template<typename Derived>
class Serializable {
public:
  void salvar(std::string_view path) const {
    // Chama metodo da classe derivada sem vtable
    auto data = static_cast<const Derived*>(this)->serializar();
    // ... escreve data em path
  }
};

class Config : public Serializable<Config> {
public:
  std::string serializar() const {
    return "{ hostname: local }";
  }
};

// Type erasure — armazena qualquer callable sem std::function overhead
class AnyCallable {
  struct Base { virtual void call() = 0; virtual ~Base() = default; };
  template<typename F>
  struct Impl : Base {
    F fn_;
    explicit Impl(F&& f) : fn_{ std::move(f) } {}
    void call() override { fn_(); }
  };
  std::unique_ptr<Base> impl_;
public:
  template<typename F>
  AnyCallable(F&& f) : impl_{ std::make_unique<Impl<F>>(std::forward<F>(f)) } {}
  void operator()() { impl_->call(); }
};
C++ — Pimpl idiom, [[nodiscard]] e GSL utilitarios
#include <memory>
#include <gsl/gsl>   // Microsoft GSL (Guidelines Support Library)

// gsl::not_null — contrato: ponteiro nunca nulo
void processar(gsl::not_null<int*> ptr) {
  *ptr *= 2;
}

// gsl::span — view nao-proprietaria sobre array contiguo
void imprimir(gsl::span<const int> arr) {
  for (int x : arr) std::print("{} ", x);
}

// [[nodiscard]] — compilador avisa se retorno nao for usado
[[nodiscard]] bool conectar(std::string_view host, int port);

// [[likely]] / [[unlikely]] — hint de previsao de branch (C++20)
void processar_evento(int tipo) {
  if (tipo == 0) [[likely]] {
    // caminho comum — otimizado pelo CPU
  } else [[unlikely]] {
    // caminho raro — erro ou evento especial
  }
}

// gsl::Expects / gsl::Ensures — pre/pos-condicoes (contratos em runtime)
double raiz_quadrada(double x) {
  gsl::Expects(x >= 0.0);     // pre-condicao
  double result = std::sqrt(x);
  gsl::Ensures(result >= 0.0);  // pos-condicao
  return result;
}

C++ Core Guidelines — Regras Essenciais

R.1 — Use RAII

Todo recurso deve ser gerenciado por um objeto. Use unique_ptr para propriedade exclusiva, shared_ptr para compartilhada. Nunca delete manualmente em codigo de aplicacao.

F.6 — noexcept

Funcoes que nao lancam excecoes devem ser marcadas noexcept. Isso permite otimizacoes (move em vez de copy em containers STL) e documenta o contrato.

ES.5 — Escopo minimo

Declare variaveis no menor escopo possivel, o mais perto do uso. Inicialize sempre na declaracao. Prefira const e constexpr sempre que o valor nao mudar.

C.45 — Rule of Zero

Se possivel, nao declare construtor de copia, operador de atribuicao, move ou destrutor. Use membros que ja implementam a semantica correta (smart pointers, containers STL).

I.23 — Parametros

Maximize legibilidade: in-params por const& ou value, out-params por ponteiro ou retorno, in/out por referencia nao-const. Prefira retornar valores a parametros de saida.

P.4 — Asserts

Use gsl::Expects/Ensures ou assert para invariantes. Use excecoes para erros recuperaveis. Use std::terminate para erros irrecuperaveis. Nao use valores de retorno magicos como -1 ou nullptr.
