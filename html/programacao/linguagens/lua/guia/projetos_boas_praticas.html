Estrutura de Modulo (return table)

O padrao mais usado em Lua e retornar uma tabela com as funcoes publicas do modulo.

math_utils.lua
local M = {}

function M.dobro(x)
	return x * 2
end

return M

Documentacao com LuaDoc

Use comentarios estruturados para gerar documentacao automatica de API, reduzindo duvidas de uso e custo de onboarding.

Boas praticas de docs

Descreva parametros, retorno e pre-condicoes. Mantenha exemplos minimos executaveis.

Evitar Globals: _ENV e Strict Mode

Variaveis globais acidentais sao fonte frequente de bugs. Prefira local e ambientes controlados com _ENV.

Em projetos grandes, adote um modo estrito para detectar escrita/leitura de globals nao declaradas.

Tratamento de Erros

Use pcall e xpcall para capturar falhas sem derrubar o processo. Gere mensagens com contexto ao chamar error.

erros.lua
local ok, resultado = pcall(function()
	error("falha ao conectar no servico X")
end)

if not ok then
	print("erro tratado:", resultado)
end

Performance na Pratica

  • Use local aliasing para funcoes globais chamadas muitas vezes.
  • Pre-aloque tabelas quando tamanho esperado for conhecido.
  • Evite concatenacao de string dentro de loops; acumule em tabela e finalize com table.concat.
