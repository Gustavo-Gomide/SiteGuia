Metatables e Metametodos

Metatables alteram o comportamento de tabelas. Metametodos como __index, __newindex, __tostring e __add controlam leitura, escrita, impressao e operadores.

metatable.lua
local Vetor = {}
Vetor.__index = Vetor

function Vetor.new(x, y)
	return setmetatable({x = x, y = y}, Vetor)
end

function Vetor.__add(a, b)
	return Vetor.new(a.x + b.x, a.y + b.y)
end

function Vetor.__tostring(v)
	return "(" .. v.x .. ", " .. v.y .. ")"
end

Heranca Prototipal com Metatables

Em Lua, em vez de classes formais, geralmente usamos prototipos: um objeto base serve de modelo para objetos derivados via __index.

Padrao comum

Classe.__index = Classe e setmetatable(instancia, Classe). Esse modelo e simples e suficiente para a maioria dos casos.

Corrotinas

Corrotinas permitem cooperacao entre tarefas com coroutine.create, resume, yield e status.

corrotina.lua
local co = coroutine.create(function()
	print("inicio")
	coroutine.yield("pausa 1")
	print("retomada")
	return "fim"
end)

print(coroutine.resume(co))
print(coroutine.status(co))

Strings e Pattern Matching

Atencao: Lua nao usa regex completa por padrao; usa um sistema proprio de patterns em string.find, string.match, string.gmatch e string.gsub.

FuncaoAplicacao
string.findEncontrar posicao de padrao.
string.matchExtrair primeira ocorrencia.
string.gmatchIterar por todas as ocorrencias.
string.gsubSubstituir padroes por texto ou funcao.

Modulos e require

O carregamento modular usa require, resolucao por package.path e cache em package.loaded.

Evite efeitos colaterais ao carregar modulos; prefira retornar uma tabela com API publica bem definida.
