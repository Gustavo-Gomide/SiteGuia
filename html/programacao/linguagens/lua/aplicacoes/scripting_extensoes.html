Lua para Scripting e Extensões

Lua embarcada em engines e apps

Modelo classico para permitir customizacao sem recompilar o software principal.

Quando um app embute Lua, ganha um motor de regras flexivel para plugins, scripts de usuario e automacao de runtime.

Modelo de Scripting Embarcado

O host em C/C++ cria um estado Lua, registra funcoes e executa scripts para delegar comportamentos variaveis sem alterar o nucleo da aplicacao.

Vantagens

Atualizacao rapida de regras, sandbox parcial e separacao entre engine e conteudo.

C API Essencial

Funcoes centrais: lua_State, lua_pcall, lua_pushstring, lua_tostring, luaL_newlib.

embedding.c (trecho)
/* fluxo simplificado de chamada Lua */
lua_State* L = luaL_newstate();
luaL_openlibs(L);
lua_pushstring(L, "ola");
if (lua_pcall(L, 0, 0, 0) != 0) {
	/* tratar erro */
}

Bindings e Sistema de Plugins

Ferramentas como tolua++ e SWIG automatizam binding entre C/C++ e Lua. Um sistema de plugin pode carregar scripts com load e loadfile.

Valide assinaturas e limite APIs expostas para reduzir riscos de scripts maliciosos em plugins de terceiros.

Exemplos em Jogos e Apps

  • LÖVE2D: arquivo conf.lua para configurar janela, vsync e comportamento inicial.
  • World of Warcraft: AddOns em Lua para UI e automacao de interface permitida pela API.
