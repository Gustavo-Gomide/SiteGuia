Lua - Banco de Dados

Visao Geral

Persistencia de dados com Lua em servidores e aplicacoes

Lua pode trabalhar com SQLite, PostgreSQL, MySQL, Redis e filas de dados com boa performance, especialmente em contextos como OpenResty, automacao e sistemas embarcados.

Se voce domina conexao, transacao e pooling em Lua, consegue construir APIs robustas e de baixa latencia mesmo em ambientes com recursos limitados.

Fundamentos de Persistencia em Lua

Imagine o banco como um "arquivo vivo" compartilhado entre processos. Em Lua, as bibliotecas de acesso variam conforme o ambiente: LuaSQL e LuaDBI em apps tradicionais, resty.* no OpenResty e drivers dedicados como pgmoon para PostgreSQL.

Conceitos obrigatorios: conexao, statement parametrizado, transacao, isolamento e estrategia de reconexao. Esses pilares evitam SQL injection, corrupcao de estado e gargalos em horarios de pico.

Stack tipica por cenario

  • Desktop/local: SQLite (lsqlite3) para simplicidade e offline.
  • API web: PostgreSQL (pgmoon/lua-resty-postgres) com pool.
  • Cache e fila: Redis (lua-resty-redis) para dados volateis.
  • Scripting legado: LuaSQL/LuaDBI para integracao rapida.

Fluxo Interno: da Requisicao ao Commit

Uma API em Lua tipicamente recebe request HTTP, valida payload, abre (ou reutiliza) conexao, executa query parametrizada, trata erro e responde JSON. Em cargas altas, a diferenca entre "abrir conexao por request" e "usar pool" define a escalabilidade.

Fluxo de operacao com tratamento de erro
-- Pseudocodigo realista para camada de repositorio
local repo = {}

function repo.criar_usuario(db, nome, email)
  if nome == "" then
    return nil, "nome invalido"
  end

  local sql = "INSERT INTO usuarios(nome, email) VALUES(?, ?)"
  local ok, err = db:execute(sql, nome, email)

  if not ok then
    return nil, err
  end

  return { status = "criado" }
end

Exemplos Praticos: SQL, Transacao e API

Exemplo com LuaDBI em PostgreSQL usando prepared statement e controle de transacao.

LuaDBI com transacao segura
local DBI = require("DBI")

local db = DBI.Connect(
  "PostgreSQL",
  "appdb",
  "usuario",
  "senha",
  "localhost",
  5432
)

local ok, err = db:autocommit(false)
if not ok then
  error(err)
end

local stmt = db:prepare("INSERT INTO contas(nome, saldo) VALUES(?, ?)")
stmt:execute("Ana", 1000)

local sucesso = true
if sucesso then
  db:commit()
else
  db:rollback()
end

db:close()

Para API server-side com OpenResty, combine pool de conexoes, timeout agressivo e fallback de cache em Redis para reduzir pressao no SQL primario.

Consulta com redis como cache
local redis = require("resty.redis")
local r = redis:new()
r:set_timeout(100)

local ok = r:connect("127.0.0.1", 6379)
if not ok then
  ngx.say("erro redis")
  return
end

local cache = r:get("user:42")
if cache ~= ngx.null then
  ngx.say(cache)
  return
end

ngx.say("cache miss -> buscar no SQL")

Quando Usar e Quando Evitar

Use Lua para banco quando

  • Seu backend usa OpenResty e precisa de baixa latencia.
  • A aplicacao e embarcada e pede footprint reduzido.
  • Voce precisa automacao de ETL e scripts de integracao rapidos.
  • Ha necessidade de cache inteligente com Redis e fallback SQL.

Evite quando

  • Equipe depende de ORM avançado e migracoes muito automatizadas.
  • Ha forte dependencia de tooling empresarial fora do ecossistema Lua.
  • Processamento analitico pesado exige stack columnar especializada.

Concorrencia, Paralelismo e Confiabilidade

Lua usa corrotinas para concorrencia cooperativa. Em OpenResty, voce ganha multiplexacao eficiente por worker. Para paralelismo real, escale por processos/workers e use fila/stream para desacoplar escrita pesada.

Padroes avancados

  • Bulk insert em batches para reduzir round-trips.
  • Outbox pattern para consistencia entre DB e eventos.
  • Idempotencia em endpoints de escrita (chave de requisicao).
  • Read replicas para leitura e primario exclusivo para escrita.
  • Backpressure com fila quando o banco atinge limite.

O maior gargalo raramente e a linguagem. Normalmente e query mal indexada, pool mal configurado e ausencia de limites de concorrencia.
