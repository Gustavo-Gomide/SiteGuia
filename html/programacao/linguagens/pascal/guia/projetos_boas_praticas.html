Pascal — Projetos e Boas Práticas

Estrutura de units, convenções de nomenclatura, documentação com PasDoc, organização de projetos Lazarus e migração de código legado.

Código Pascal de qualidade não é apenas código correto — é código que outros programadores conseguem ler, manter e estender. As convenções do ecossistema Object Pascal (especialmente as herdadas do Delphi) são maduras e amplamente adotadas. Segui-las é a diferença entre um projeto profissional e um projeto difícil de manter.

Estrutura de Units

A unit é a unidade de modularização do Pascal. Ela tem duas partes: interface (declarações públicas, equivalente ao arquivo .h do C) e implementation (implementação privada). O código de inicialização fica na seção initialization.

GeometriaPlanar.pas — Unit bem estruturada
{
  GeometriaPlanar.pas — Formas geométricas 2D
  Autor: Alice Silva
  Versão: 1.0.0
  Licença: MIT
}
unit GeometriaPlanar;

{$mode objfpc}     // modo Object Pascal (FPC)
{$H+}              // usa AnsiString em vez de ShortString
{$interfaces corba} // interfaces no estilo CORBA (FPC/Delphi-compatível)

interface

uses
  SysUtils, Math;

type
  { Interface pública — qualquer forma deve implementar }
  IForma = interface
    function Area: Double;
    function Perimetro: Double;
    function Descricao: String;
  end;

  { Classe base — não instanciar diretamente }
  TFormaBase = class(TObject, IForma)
  public
    function Area: Double; virtual; abstract;
    function Perimetro: Double; virtual; abstract;
    function Descricao: String; virtual;
  end;

  TCirculo = class(TFormaBase)
  private
    FRaio : Double;               { F = campo privado (Field) }
  public
    constructor Create(raio: Double);
    function  Area: Double; override;
    function  Perimetro: Double; override;
    function  Descricao: String; override;
    property  Raio: Double read FRaio;
  end;

  TRetangulo = class(TFormaBase)
  private
    FLargura, FAltura : Double;
  public
    constructor Create(largura, altura: Double);
    function  Area: Double; override;
    function  Perimetro: Double; override;
    function  Descricao: String; override;
  end;

{ Função de fábrica — retorna a forma mais eficiente para uma área }
function FormaComMaiorArea(formas: array of IForma): IForma;

implementation

function TFormaBase.Descricao: String;
begin
  Result := Format('%s: Área=%.2f Perímetro=%.2f',
    [ClassName, Area, Perimetro]);
end;

constructor TCirculo.Create(raio: Double);
begin
  if raio <= 0 then
    raise EArgumentException.Create('Raio deve ser positivo');
  FRaio := raio;
end;

function TCirculo.Area: Double;
begin
  Result := Pi * FRaio * FRaio;
end;

function TCirculo.Perimetro: Double;
begin
  Result := 2 * Pi * FRaio;
end;

function TCirculo.Descricao: String;
begin
  Result := Format('Círculo(r=%.2f): Área=%.4f', [FRaio, Area]);
end;

constructor TRetangulo.Create(largura, altura: Double);
begin
  FLargura := largura;
  FAltura  := altura;
end;

function TRetangulo.Area: Double;
begin
  Result := FLargura * FAltura;
end;

function TRetangulo.Perimetro: Double;
begin
  Result := 2 * (FLargura + FAltura);
end;

function TRetangulo.Descricao: String;
begin
  Result := Format('Retângulo(%gx%g): Área=%.4f', [FLargura, FAltura, Area]);
end;

function FormaComMaiorArea(formas: array of IForma): IForma;
var
  i : Integer;
begin
  Result := formas[0];
  for i := 1 to High(formas) do
    if formas[i].Area > Result.Area then
      Result := formas[i];
end;

end.

Convenções de Nomenclatura

O ecossistema Object Pascal/Delphi usa um conjunto de prefixos e sufixos padronizados. Segui-los torna o código instantaneamente reconhecível por qualquer desenvolvedor Pascal experiente.

Documentação com PasDoc

PasDoc gera documentação HTML, LaTeX ou CHM a partir de comentários especiais no código fonte. O formato usa /** */ ou {** } com tags @param, @returns, @raises, @seealso.

documented.pas — Comentários PasDoc
unit DocumentedUnit;

interface

type
  {** Representa um ponto no espaço 2D.
    @author Alice Silva
    @created 2024-01-15
  }
  TPonto = class
  private
    FX, FY : Double;
  public
    {** Cria um ponto nas coordenadas especificadas.
      @param x Coordenada horizontal
      @param y Coordenada vertical
    }
    constructor Create(x, y: Double);

    {** Calcula a distância euclidiana até outro ponto.
      @param outro O ponto de destino
      @returns Distância não-negativa em unidades do espaço
      @seealso DistanciaManhattan
    }
    function DistanciaEuclidiana(outro: TPonto): Double;

    {** Distância de Manhattan (L1).
      @seealso DistanciaEuclidiana
    }
    function DistanciaManhattan(outro: TPonto): Double;

    {** Coordenada X do ponto. }
    property X: Double read FX;
    {** Coordenada Y do ponto. }
    property Y: Double read FY;
  end;

implementation

uses Math;

constructor TPonto.Create(x, y: Double);
begin
  FX := x; FY := y;
end;

function TPonto.DistanciaEuclidiana(outro: TPonto): Double;
begin
  Result := Hypot(outro.X - FX, outro.Y - FY);
end;

function TPonto.DistanciaManhattan(outro: TPonto): Double;
begin
  Result := Abs(outro.X - FX) + Abs(outro.Y - FY);
end;

end.

Migração de Turbo Pascal para Free Pascal

Muito código Turbo Pascal (MS-DOS) ainda existe em produção. FPC pode compilá-lo com alterações mínimas usando {$mode turbo}, mas é recomendável modernizar o código progressivamente.

migracao.pas — Código Turbo Pascal e equivalente moderno
{ ── TURBO PASCAL (código antigo) ─────────────────────────────────── }
program Antigo;

uses Crt;   { unit DOS/CRT — não existe no FPC moderno }

var
  ch : Char;

begin
  ClrScr;           { limpar tela — Crt }
  GotoXY(10, 5);    { mover cursor — Crt }
  TextColor(Yellow); { cor — Crt }
  Write('Pressione tecla: ');
  ch := ReadKey;    { leitura de tecla — Crt }
  Sound(440);        { beep — Crt/DOS }
  Delay(500);        { esperar ms — Crt/DOS }
  NoSound;          { parar som — Crt/DOS }
end.

{ ── EQUIVALENTE MODERNO (FPC + SysUtils) ─────────────────────────── }
program Moderno;

uses
  SysUtils,
  { Crt ainda existe no FPC — use apenas se necessário }
  Crt;    { fpc -dUSECRT para habilitar }

begin
  { Alternativas modernas: }
  WriteLn(#27 + '[2J');         { ANSI escape — limpar tela }
  WriteLn(#27 + '[5;10H');      { ANSI — mover cursor (5,10) }
  WriteLn(#7);                  { BEL — beep }
  Sleep(500);                   { SysUtils: esperar 500ms }
end.

Organização de Projetos Lazarus

Estrutura de projeto Lazarus recomendada
MeuSistema/
├── MeuSistema.lpi          { projeto Lazarus (XML) }
├── MeuSistema.lpr          { programa principal (gerado) }
├── src/
│   ├── forms/
│   │   ├── MainForm.pas    { unit da janela principal }
│   │   ├── MainForm.lfm    { design da janela (XML) }
│   │   ├── ConfigForm.pas
│   │   └── ConfigForm.lfm
│   ├── data/
│   │   ├── PessoaDAO.pas   { acesso a dados }
│   │   └── ConexaoDB.pas
│   ├── models/
│   │   ├── Pessoa.pas      { entidades/modelos }
│   │   └── Produto.pas
│   └── utils/
│       ├── Validacao.pas
│       └── FormatUtils.pas
├── tests/
│   ├── TestRunner.pas
│   ├── PessoaTest.pas
│   └── ValidacaoTest.pas
├── docs/                   { PasDoc output }
└── resources/
    ├── icons/
    └── strings/
MeuSistema.lpr — Arquivo de projeto gerado pelo Lazarus
program MeuSistema;

{$mode objfpc}{$H+}

uses
  { Qt5 no Linux, Win32 no Windows, Cocoa no macOS }
  { o Lazarus seleciona automaticamente conforme a plataforma }
  Interfaces,
  Forms,
  MainForm,
  PessoaDAO,
  ConexaoDB;

begin
  RequireDerivedFormResource := True;
  Application.Initialize;
  Application.Title := 'Meu Sistema';
  Application.CreateForm(TMainForm, MainFormInstance);
  Application.Run;
end.
