Pascal — Ecossistema e Tooling

Free Pascal Compiler, Lazarus IDE, unidades padrão, testes com FPCUnit, depuração com GDB e cross-compilação para múltiplas plataformas.

O ecossistema moderno do Pascal gira em torno do Free Pascal Compiler (FPC) e do Lazarus IDE. Juntos, eles oferecem uma cadeia de ferramentas completa — da compilação à depuração — sem custo de licença, suportando dezenas de plataformas de destino.

Free Pascal Compiler (fpc)

Free Pascal Compiler é um compilador nativo que gera código de máquina otimizado para i386, x86_64, ARM, AArch64, MIPS, RISC-V, WebAssembly e outros. É compatível com Turbo Pascal 7 e Delphi 7.

bash — Uso do compilador fpc na linha de comando
# Compilar básico
fpc programa.pas

# Otimização agressiva
fpc -O3 programa.pas

# Incluir informações de debug (para GDB)
fpc -g programa.pas

# Debug completo com DWARF (melhor suporte em GDB moderno)
fpc -gw3 programa.pas

# Aviso extra (todos os avisos habilitados)
fpc -Wall programa.pas

# Verificações de range/overflow em runtime
fpc -Cr -Co programa.pas

# Saída em diretório específico
fpc -FE./bin -FU./obj programa.pas

# Cross-compilar para Windows 64-bit a partir do Linux
fpc -Tcross -Tos=win64 -Tcpu=x86_64 programa.pas

# Cross-compilar para ARM Linux (Raspberry Pi)
fpc -Tcross -Tos=linux -Tcpu=arm programa.pas

# Modo de compatibilidade Delphi
fpc -Mdelphi programa.pas

# Ver todas as flags
fpc -h
FlagEfeito
-O1/O2/O3Nível de otimização (0=nenhuma, 3=máximo)
-gInformações para depurador (formato nativo)
-gw3Debug DWARF v3 (GDB, LLDB)
-WallAtiva todos os avisos
-CrVerificação de range em runtime
-CoVerificação de overflow em runtime
-MfpcModo FPC padrão
-MdelphiCompatibilidade Delphi
-Tos=win64SO alvo (cross-compilation)
-Tcpu=armCPU alvo (cross-compilation)
-FEDiretório para executável
-FUDiretório para arquivos .ppu/.o
-lListar unidades disponíveis

Lazarus IDE

Lazarus é um IDE open-source para Free Pascal com editor code-aware (autocompletar, refatoração, navegação), debugger integrado (GDB), designer visual de formulários (RAD), e gerenciador de pacotes. É a alternativa gratuita ao Delphi.

Layouts de Projeto

.lpi — arquivo de projeto Lazarus (XML). .lpr — arquivo Pascal principal do projeto (gerado automaticamente). .lfm — formulário visual serializado. .pas — unit do formulário ou módulo.

Atalhos Essenciais

F9 Compilar e executar | F8 Step over | F7 Step into | F5 Breakpoint | Ctrl+F11 Compilar | Ctrl+Shift+F11 Opções do projeto | Ctrl+Alt+P Paleta de componentes

Gerenciador de Pacotes

Lazarus tem um sistema de pacotes (.lpk) para distribuir componentes. Pacotes podem ser instalados na IDE para aparecer na paleta de componentes. OPM (Online Package Manager) permite instalar desde a internet.

Cross-Platform

Lazarus usa a LCL (Lazarus Component Library) que abstrai widgets nativos. O mesmo código GUI compila para GTK3 (Linux), Win32/Win64 (Windows) e Cocoa (macOS) sem alterações.

Unidades Padrão Mais Importantes

unidades.pas — Demonstração das principais unidades
program UnidadesPadrao;

uses
  SysUtils,        { utilidades de sistema }
  Classes,         { TStringList, TStream, TComponent... }
  Math,            { funções matemáticas avançadas }
  StrUtils,        { manipulação de strings }
  DateUtils,       { manipulação de datas }
  Contnrs,         { TObjectList, TStack, TQueue (não-genérico) }
  Generics.Collections;  { TList, TDictionary... }

var
  lista   : TStringList;
  hoje    : TDateTime;
  x       : Double;
  s, s2   : String;

begin
  { ── SysUtils ───────────────────────────────────────────── }
  s := IntToStr(42);
  s := FloatToStr(3.14);
  s := Format('%d itens: %.2f', [5, 19.99]);
  s := UpperCase('olá');
  s := Trim('  espaços  ');
  s := StringReplace('abc-abc', '-', '/', [rfReplaceAll]);
  WriteLn(FileExists('/etc/hosts'));
  WriteLn(DirectoryExists('/tmp'));
  WriteLn(GetCurrentDir);
  WriteLn(ExtractFileName('/home/user/arquivo.pas'));  { arquivo.pas }
  WriteLn(ExtractFileExt('arquivo.pas'));              { .pas }

  { ── Classes — TStringList ──────────────────────────────── }
  lista := TStringList.Create;
  try
    lista.Add('linha um');
    lista.Add('linha dois');
    lista.Add('linha três');
    lista.Sort;
    WriteLn('Contagem: ', lista.Count);
    WriteLn('Índice 0: ', lista[0]);
    WriteLn('Texto: '#13#10, lista.Text);
    lista.SaveToFile('saida.txt');

    { Carregar de arquivo }
    lista.LoadFromFile('saida.txt');

    { Uso como dicionário (chave=valor) }
    lista.Add('nome=Alice');
    lista.NameValueSeparator := '=';
    WriteLn('nome: ', lista.Values['nome']);
  finally
    lista.Free;
  end;

  { ── Math ────────────────────────────────────────────────── }
  x := Power(2.0, 32.0);       { 4294967296.0 }
  x := Log2(1024.0);           { 10.0 }
  x := Log10(1000.0);          { 3.0 }
  x := Hypot(3.0, 4.0);        { 5.0 — hipotenusa }
  x := MaxValue([1.0, 5.0, 3.0]);  { 5.0 }
  x := Mean([1.0, 2.0, 3.0]);   { 2.0 }
  x := StdDev([2.0, 4.0, 4.0, 4.0, 5.0, 5.0, 7.0, 9.0]);
  WriteLn(Format('StdDev = %.4f', [x]));   { 2.0 }

  { ── DateUtils ───────────────────────────────────────────── }
  hoje := Now;
  WriteLn(FormatDateTime('dd/mm/yyyy hh:nn:ss', hoje));
  WriteLn('Dia da semana: ', DayOfWeek(hoje));  { 1=Dom, 7=Sab }
  WriteLn('Ano: ', YearOf(hoje));
  hoje := IncDay(hoje, 30);   { hoje + 30 dias }
  WriteLn('Daqui 30 dias: ', FormatDateTime('dd/mm/yyyy', hoje));

  { ── StrUtils ────────────────────────────────────────────── }
  s := AnsiContainsStr('Hello World', 'World').ToString;
  s2 := PadLeft('42', 5, '0');    { '00042' }
  s2 := ReverseString('Pascal');  { 'lacsaP' }
  WriteLn(WrapText('Uma string muito longa que vai ser quebrada em linhas', 20));
end.

Testes com FPCUnit

FPCUnit é o framework de testes unitários padrão do Free Pascal, baseado no padrão JUnit. Testes são métodos publicados em uma classe que herda de TTestCase.

Estrutura de projeto de testes FPCUnit
meu_projeto/
├── src/
│   ├── matematica.pas        { módulo a testar }
│   └── strings_util.pas
├── tests/
│   ├── matematica_test.pas   { arquivo de testes }
│   └── test_runner.pas       { programa principal }
└── Makefile
matematica.pas — Unidade sendo testada
unit Matematica;

interface

function Fatorial(n: Integer): Int64;
function EhPrimo(n: Integer): Boolean;
function MDC(a, b: Integer): Integer;

implementation

function Fatorial(n: Integer): Int64;
begin
  if n <= 1 then Result := 1
  else Result := n * Fatorial(n - 1);
end;

function EhPrimo(n: Integer): Boolean;
var i: Integer;
begin
  if n < 2 then begin Result := False; Exit; end;
  Result := True;
  i := 2;
  while i * i <= n do
  begin
    if n mod i = 0 then begin Result := False; Exit; end;
    Inc(i);
  end;
end;

function MDC(a, b: Integer): Integer;
begin
  while b <> 0 do
  begin
    a := a mod b;
    a := b; b := a;
  end;
  Result := a;
end;

end.
matematica_test.pas — Arquivo de testes FPCUnit
unit MatematicaTest;

interface

uses
  FPCUnit, TestRegistry, Matematica;

type
  TMatematicaTest = class(TTestCase)
  published   { métodos published são descobertos automaticamente }
    procedure TestFatorial_Zero;
    procedure TestFatorial_Um;
    procedure TestFatorial_Dez;
    procedure TestEhPrimo_Dois;
    procedure TestEhPrimo_Quatro;
    procedure TestEhPrimo_NumerosNegativo;
    procedure TestMDC_Basico;
  end;

implementation

procedure TMatematicaTest.TestFatorial_Zero;
begin
  AssertEquals('0! deve ser 1', Int64(1), Fatorial(0));
end;

procedure TMatematicaTest.TestFatorial_Um;
begin
  AssertEquals('1! deve ser 1', Int64(1), Fatorial(1));
end;

procedure TMatematicaTest.TestFatorial_Dez;
begin
  AssertEquals('10! deve ser 3628800', Int64(3628800), Fatorial(10));
end;

procedure TMatematicaTest.TestEhPrimo_Dois;
begin
  AssertTrue('2 deve ser primo', EhPrimo(2));
  AssertTrue('7 deve ser primo', EhPrimo(7));
  AssertTrue('97 deve ser primo', EhPrimo(97));
end;

procedure TMatematicaTest.TestEhPrimo_Quatro;
begin
  AssertFalse('4 não deve ser primo', EhPrimo(4));
  AssertFalse('100 não deve ser primo', EhPrimo(100));
end;

procedure TMatematicaTest.TestEhPrimo_NumerosNegativo;
begin
  AssertFalse('Números negativos não são primos', EhPrimo(-7));
  AssertFalse('0 não é primo', EhPrimo(0));
  AssertFalse('1 não é primo', EhPrimo(1));
end;

procedure TMatematicaTest.TestMDC_Basico;
begin
  AssertEquals('MDC(12,8) = 4', 4, MDC(12, 8));
  AssertEquals('MDC(100,75) = 25', 25, MDC(100, 75));
end;

initialization
  RegisterTest(TMatematicaTest);   { registra para descoberta automática }
end.
test_runner.pas — Programa principal de testes
program TestRunner;

uses
  FPCUnit, FPCUnitRunner, TestRegistry, ConsoleFPCTestRunner,
  MatematicaTest;   { importar para forçar a inicialização }

var
  App : TTestRunner;

begin
  App := TTestRunner.Create(nil);
  try
    App.Initialize;
    App.Run;
  finally
    App.Free;
  end;
end.

Depuração com GDB

bash — Sessão de depuração com GDB
# 1. Compilar com informações de debug
fpc -gw3 programa.pas

# 2. Iniciar GDB
gdb ./programa

# No prompt (gdb):
(gdb) break programa.pas:42       # breakpoint na linha 42
(gdb) break TAnimal.Falar         # breakpoint em método
(gdb) run                         # executar até o breakpoint
(gdb) print n                     # ver valor de variável
(gdb) print lista^.Valor          # desreferenciar ponteiro
(gdb) next                        # próxima linha (step over)
(gdb) step                        # entrar na função (step into)
(gdb) backtrace                   # pilha de chamadas
(gdb) continue                    # continuar até próximo breakpoint
(gdb) watch variavel              # parar quando variável mudar
(gdb) quit                        # sair

O Lazarus IDE integra o GDB com interface gráfica. Basta pressionar F5 para adicionar breakpoint e F9 para executar em modo debug. A janela "Watches" permite inspecionar variáveis em tempo real.

Cross-Compilação — Múltiplas Plataformas

FPC é um dos compiladores com melhor suporte a cross-compilação. Você escreve código uma vez e compila para dezenas de alvos.

bash — Instalação de cross-compilers e compilação
# Instalar cross-compilers no Ubuntu
sudo apt install fpc-cross-win64 fpc-cross-arm fpc-cross-aarch64

# ── Targets comuns ───────────────────────────────────────── #

# Windows 64-bit (de Linux)
fpc -Tos=win64 -Tcpu=x86_64 programa.pas

# Windows 32-bit
fpc -Tos=win32 -Tcpu=i386 programa.pas

# Raspberry Pi (Linux ARM 32-bit)
fpc -Tos=linux -Tcpu=arm -Tabi=gnueabihf programa.pas

# Linux ARM 64-bit (AArch64)
fpc -Tos=linux -Tcpu=aarch64 programa.pas

# macOS (de Linux, requer SDK)
fpc -Tos=darwin -Tcpu=x86_64 programa.pas

# DOS (para saudade ou retrocomputação)
fpc -Tos=dos -Tcpu=i386 programa.pas

# Android (via NDK)
fpc -Tos=android -Tcpu=arm programa.pas

# WebAssembly (experimental)
fpc -Tos=wasi -Tcpu=wasm32 programa.pas

# ── Usando Makefile ──────────────────────────────────────── #
# Makefile
# all: linux win64 arm
# linux: fpc -O3 programa.pas
# win64: fpc -Tos=win64 -Tcpu=x86_64 programa.pas
# arm:   fpc -Tos=linux -Tcpu=arm programa.pas
