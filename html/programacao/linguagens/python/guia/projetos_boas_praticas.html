Projetos e Boas Práticas Python

Estruture projetos escaláveis, escreva código seguro e Pythonico, gerencie configuração sem segredos expostos e colabore usando padrões da comunidade.

Boas práticas não são regras arbitrárias — são lições destiladas de projetos que cresceram e sobreviveram. O Guia de Estilo PEP 8, o pyproject.toml unificado e o padrão src layout existem para facilitar manutenção a longo prazo.

Estrutura de Projeto: src Layout

Shell — estrutura src layout recomendada
meu-projeto/
├── src/
│   └── meuapp/
│       ├── __init__.py
│       ├── core/
│       │   ├── __init__.py
│       │   ├── config.py       # Settings via pydantic-settings
│       │   └── exceptions.py
│       ├── domain/
│       │   ├── models.py       # Entidades de negócio (dataclasses/pydantic)
│       │   └── services.py     # Lógica de negócio pura (sem I/O)
│       ├── infra/
│       │   ├── db.py           # SQLAlchemy engine + session factory
│       │   └── cache.py        # Redis client
│       └── api/
│           ├── main.py         # FastAPI app factory
│           └── routes/
│               ├── usuarios.py
│               └── produtos.py
├── tests/
│   ├── conftest.py
│   ├── unit/
│   └── integration/
├── pyproject.toml
├── README.md
└── .env.example               # NUNCA commitar .env com segredos
src layout vs flat layout: o src/ layout evita que o Python importe o pacote diretamente do diretório raiz durante testes, forçando que a versão instalada seja usada. Isso detecta erros de empacotamento mais cedo.

Segurança e Gestão de Segredos

Práticas seguras

  • Usar pydantic-settings + variáveis de ambiente
  • HashiCorp Vault, AWS Secrets Manager para produção
  • secrets.token_urlsafe(32) para gerar segredos
  • Validar toda entrada de usuário com Pydantic
  • Usar parameterized queries no SQLAlchemy (sem f-strings em SQL)

Evitar sempre

  • Hardcoded passwords ou API keys no código
  • Commitar arquivos .env no git
  • eval() ou exec() com input do usuário
  • pickle.loads() em dados não confiáveis (RCE)
  • subprocess.shell=True com input do usuário (command injection)
Python — gestão segura de configuração
from pydantic_settings import BaseSettings, SettingsConfigDict
from pydantic import SecretStr
import secrets


class Settings(BaseSettings):
    # SecretStr: valor nunca aparece em __repr__ nem logs
    database_url: SecretStr
    secret_key:   SecretStr = SecretStr(secrets.token_urlsafe(32))
    debug:        bool       = False
    allowed_hosts: list[str] = ["localhost"]

    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        case_sensitive=False,
    )

settings = Settings()
db_url = settings.database_url.get_secret_value()   # desembrulha
print(settings.database_url)  # SecretStr('**********') — seguro em logs

# .gitignore deve conter:
# .env
# .env.*
# !.env.example
