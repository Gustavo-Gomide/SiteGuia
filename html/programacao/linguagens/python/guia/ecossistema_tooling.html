Python — Ecossistema e Tooling

Ecossistema e Tooling Python

Gerenciadores de pacotes, ambientes virtuais, testes, linting, type checking e CI/CD: o arsenal completo do desenvolvedor Python moderno.

Um projeto Python profissional usa muito mais do que o interpretador. Conhecer o ecossistema de ferramentas é o que separa um script de um produto de software confiável, testável e colaborativo.

Ambientes Virtuais e Gerenciadores de Pacote

venv (stdlib)

Cria ambientes isolados sem dependências externas. Padrão para projetos simples.

pip

Gerenciador oficial. Use pip install -r requirements.txt e pip freeze.

poetry

Gerencia dependências, builds e publicação no PyPI com pyproject.toml.

uv

Substituto ultrarrápido de pip + venv escrito em Rust. 10–100× mais rápido que pip.

pipx

Instala CLI tools Python em ambientes isolados globalmente (ex.: ruff, httpie).

conda / mamba

Ambiente de ciência de dados. Suporte nativo a pacotes compilados (numpy, scipy, etc.).

Shell — venv + pip vs uv (workflow comparativo)
# --- venv + pip (tradicional) ---
python -m venv .venv
# Linux/macOS:
source .venv/bin/activate
# Windows PowerShell:
.venv\Scripts\Activate.ps1

pip install fastapi uvicorn pydantic
pip freeze > requirements.txt

# --- uv (moderno, recomendado) ---
uv venv                          # cria .venv em <1s
uv pip install fastapi uvicorn   # 10x mais rápido
uv pip freeze > requirements.txt

# --- poetry (projetos de biblioteca) ---
poetry new meu-pacote
poetry add requests pydantic
poetry add --group dev pytest ruff mypy
poetry build    # gera dist/meu-pacote-0.1.0.tar.gz
poetry publish  # envia ao PyPI

pytest: Testes Automatizados

Python — pytest: fixtures, parametrize, mock e cobertura
# tests/conftest.py
import pytest
from sqlalchemy import create_engine
from sqlalchemy.orm import Session
from app.models import Base

@pytest.fixture(scope="session")
def engine():
    eng = create_engine("sqlite:///:memory:")
    Base.metadata.create_all(eng)
    yield eng
    Base.metadata.drop_all(eng)

@pytest.fixture
def db(engine):
    with Session(engine) as session:
        yield session
        session.rollback()  # isola cada teste


# tests/test_usuario.py
import pytest
from unittest.mock import patch, MagicMock
from app.services import criar_usuario, UsuarioJaExiste

@pytest.mark.parametrize("email,esperado", [
    ("ok@exemplo.com", True),
    ("invalido", False),
    ("@semuser.com", False),
])
def test_email_valido(email: str, esperado: bool):
    from app.utils import email_valido
    assert email_valido(email) == esperado


def test_criar_usuario_duplicado(db):
    criar_usuario(db, "João", "joao@ex.com")
    with pytest.raises(UsuarioJaExiste):
        criar_usuario(db, "João2", "joao@ex.com")


@patch("app.services.enviar_email")
def test_email_enviado_no_cadastro(mock_email: MagicMock, db):
    criar_usuario(db, "Ana", "ana@ex.com")
    mock_email.assert_called_once_with("ana@ex.com")
Shell — pytest com cobertura e relatório HTML
# Executar testes com cobertura
pytest --cov=app --cov-report=term-missing --cov-report=html

# Executar testes em paralelo (pytest-xdist)
pytest -n auto

# Rodar apenas testes marcados como rápidos
pytest -m "not slow" -v

# pyproject.toml — configuração centralizada
[tool.pytest.ini_options]
testpaths = ["tests"]
addopts   = "-ra -q --strict-markers"

[tool.coverage.run]
source = ["app"]
omit   = ["*/migrations/*"]

[tool.coverage.report]
fail_under = 80

Qualidade de Código: ruff, mypy e pre-commit

TOML — pyproject.toml com ruff, mypy e black configurados
# pyproject.toml
[tool.ruff]
line-length    = 88
target-version = "py312"
select = ["E", "F", "I", "UP", "B", "SIM"]
# E=pycodestyle, F=pyflakes, I=isort, UP=pyupgrade,
# B=flake8-bugbear, SIM=simplify

[tool.ruff.lint.isort]
force-single-line   = true
known-first-party   = ["app"]

[tool.mypy]
python_version         = "3.12"
strict                 = true
ignore_missing_imports = true
disallow_untyped_defs  = true

[tool.black]
line-length     = 88
target-versions = ["py312"]

CI/CD com GitHub Actions

YAML — .github/workflows/ci.yml completo
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v1
      - run: uv venv
      - run: uv pip install ruff mypy
      - run: ruff check .
      - run: mypy src/

  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.11", "3.12"]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      - run: pip install uv
      - run: uv pip install -e ".[dev]"
      - run: pytest --cov=src --cov-report=xml -q
      - uses: codecov/codecov-action@v4
        with:
          token: ${{ secrets.CODECOV_TOKEN }}

  publish:
    needs: [lint, test]
    if: startsWith(github.ref, 'refs/tags/')
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install build twine
      - run: python -m build
      - run: twine upload dist/*
        env:
          TWINE_USERNAME: __token__
          TWINE_PASSWORD: ${{ secrets.PYPI_TOKEN }}
