Haskell — Ecossistema e Tooling

O ecossistema Haskell é centrado no GHC (compilador), Cabal (build tool oficial) ou Stack (build tool com snapshots, reprodutibilidade garantida), Hackage (repositório de pacotes), HLS (Language Server) e ferramentas de qualidade como QuickCheck, HSpec e Ormolu.

GHCup é o instalador oficial recomendado — gerencia versões de GHC, Cabal, Stack e HLS de forma isolada, similar ao rustup do Rust. Não instale GHC pelo package manager do sistema; use GHCup para controle de versão.

Cabal e Stack — Projetos e Dependências

Comandos essenciais para build, test e run
-- Cabal — build tool oficial do GHC
cabal init meu-projeto           -- criar projeto interativo
cabal build                      -- compilar
cabal run                        -- compilar e executar
cabal test                       -- rodar testes
cabal repl                       -- GHCi com o projeto carregado
cabal install hlint ormolu       -- instalar ferramenta global

-- meu-projeto.cabal
cabal-version:      3.0
name:               meu-projeto
version:            0.1.0.0

executable meu-projeto
    main-is:          Main.hs
    hs-source-dirs:   src
    ghc-options:      -Wall -O2
    build-depends:
        base ^>=4.19,
        text ^>=2.1,
        aeson ^>=2.2,
        containers ^>=0.7

library
    exposed-modules:  Lib, Lib.Types
    hs-source-dirs:   lib
    build-depends:    base, text, containers

test-suite tests
    type:             exitcode-stdio-1.0
    main-is:          Spec.hs
    hs-source-dirs:   test
    build-depends:    base, meu-projeto, hspec, QuickCheck

-- Stack — build tool com snapshots LTS (Stackage)
stack new meu-projeto              -- criar projeto com template
stack build                        -- compilar
stack run                          -- executar
stack test                         -- testar
stack ghci                         -- REPL com projeto
stack exec -- meu-projeto          -- executar binário

-- stack.yaml
resolver: lts-22.28    -- snapshot LTS do Stackage — versões compatíveis garantidas
packages:
  - .
extra-deps:
  - some-package-1.2.3    -- dependência fora do LTS

HSpec e QuickCheck — Testes

Testes unitários expressivos e property-based testing
-- HSpec — BDD-style tests
import Test.Hspec
import Test.QuickCheck

-- Funções sendo testadas
reverter :: [a] -> [a]
reverter []     = []
reverter (x:xs) = reverter xs ++ [x]

somaLista :: [Int] -> Int
somaLista = foldl (+) 0

-- Suíte de testes HSpec
main :: IO ()
main = hspec $ do
    describe "reverter" $ do
        it "lista vazia retorna vazia" $
            reverter ([] :: [Int]) `shouldBe` []

        it "inverte lista simples" $
            reverter [1, 2, 3] `shouldBe` [3, 2, 1]

        it "reverter duas vezes = identidade" $
            reverter (reverter [1..5]) `shouldBe` [1..5]

    describe "somaLista" $ do
        it "soma lista vazia = 0" $
            somaLista [] `shouldBe` 0

        it "soma lista conhecida" $
            somaLista [1..100] `shouldBe` 5050

-- QuickCheck — property-based testing
propReverterDuasVezes :: [Int] -> Bool
propReverterDuasVezes xs = reverter (reverter xs) == xs

propReverterLength :: [Int] -> Bool
propReverterLength xs = length (reverter xs) == length xs

propSomaComutativa :: [Int] -> [Int] -> Bool
propSomaComutativa xs ys = somaLista (xs ++ ys) == somaLista (ys ++ xs)

testesQuickCheck :: IO ()
testesQuickCheck = do
    quickCheck propReverterDuasVezes
    quickCheck propReverterLength
    quickCheck propSomaComutativa
-- +++ OK, passed 100 tests.

-- QuickCheck com HSpec (combinado)
main2 :: IO ()
main2 = hspec $ do
    describe "reverter" $ do
        it "reverter duas vezes = identidade" $
            property propReverterDuasVezes
        it "preserva length" $
            property propReverterLength

GHC Extensions — habilitar no .cabal ou por arquivo
-- Por arquivo (pragma no topo)
{-# LANGUAGE OverloadedStrings #-}    -- "literais" :: Text (não só String)
{-# LANGUAGE RecordWildCards   #-}    -- let Pessoa{..} = p; usar nome, idade
{-# LANGUAGE LambdaCase        #-}    -- \case Nothing -> ...; Just x -> ...
{-# LANGUAGE TupleSections     #-}    -- (,3) como função (x -> (x,3))
{-# LANGUAGE ScopedTypeVariables #-}  -- forall a. em assinaturas locais
{-# LANGUAGE DerivingStrategies #-}   -- deriving (Show) via SomeNewtype
{-# LANGUAGE MultiParamTypeClasses #-} -- class Convert a b where ...
{-# LANGUAGE FlexibleInstances  #-}   -- instance MyClass [Int] where ...
{-# LANGUAGE TypeApplications   #-}   -- read @Int, fromIntegral @Int @Double

-- No .cabal (default-extensions aplicado a todos os módulos)
-- default-extensions:
--     OverloadedStrings
--     LambdaCase
--     RecordWildCards

-- Exemplos de uso das extensions
-- LambdaCase:
descricao :: Maybe Int -> String
descricao = \case
    Nothing -> "ausente"
    Just 0  -> "zero"
    Just n  -> "valor: " ++ show n

-- OverloadedStrings: usa "texto" com tipo Text diretamente
import Data.Text (Text)
titulo :: Text
titulo = "Haskell"    -- sem fromString "Haskell" necessário

Governanca Tecnica e Padroes de Time

Para manter consistencia entre projetos e linguagens, defina uma baseline de engenharia: formato obrigatorio, regras de review, gates de CI, politica de versao e checklist de release. Em Haskell, essa disciplina reduz muito a variabilidade entre modulos e facilita onboarding.

  • Qualidade minima: -Wall, testes automatizados e lint sem warnings criticos.
  • Operacao: logs estruturados, metricas por fluxo e alarmes orientados a SLO.
  • Seguranca: auditoria de dependencias e controle de atualizacao de snapshots.
  • Evolucao: migracoes pequenas, reversiveis e com plano de rollback.

QuickCheck mudou para sempre o que significa testar software. Em vez de escrever casos manualmente, você descreve propriedades que devem ser verdadeiras para qualquer input, e a biblioteca gera centenas de casos automaticamente — incluindo casos de borda que você jamais pensaria. Originou toda uma categoria de ferramentas: Hypothesis (Python), fast-check (JS), PropTest (Rust).
