Haskell — Web

Servant representa APIs REST como tipos Haskell — a API completa é uma expressão de tipo, e o compilador garante que handlers correspondem exatamente ao tipo especificado. Scotty oferece uma API simples monádica similar ao Express/Flask. Warp é o servidor HTTP de alta performance subjacente.

Servant é único: a API é um tipo de dados que pode ser usado para gerar automaticamente documentação Swagger, clientes JavaScript, clientes Haskell tipo-safe, e backends — tudo a partir da mesma definição de tipo. Se a API muda, tudo que depende dela não compila até ser atualizado.

Servant — REST APIs Type-Safe

API como tipo, handlers gerados pelo compilador
{-# LANGUAGE DataKinds         #-}
{-# LANGUAGE TypeOperators     #-}
{-# LANGUAGE OverloadedStrings #-}

import Servant
import Network.Warp (run)
import Data.Aeson (FromJSON, ToJSON, Generic)

-- Tipos de domínio
data Produto = Produto
    { produtoId   :: Int
    , produtoNome :: String
    , produtoPreco :: Double
    } deriving (Show, Generic, FromJSON, ToJSON)

data NovoProduto = NovoProduto
    { novoNome  :: String
    , novoPreco :: Double
    } deriving (Show, Generic, FromJSON, ToJSON)

-- Definição da API como TIPO
type ProdutosAPI
     = "produtos" :> Get '[JSON] [Produto]
  :<|> "produtos" :> Capture "id" Int :> Get '[JSON] Produto
  :<|> "produtos" :> ReqBody '[JSON] NovoProduto :> Post '[JSON] Produto
  :<|> "produtos" :> Capture "id" Int :> DeleteNoContent
  :<|> "produtos" :> QueryParam "busca" String :> Get '[JSON] [Produto]

-- Handlers — devem corresponder EXATAMENTE ao tipo
listarProdutos :: Handler [Produto]
listarProdutos = return
    [ Produto 1 "Teclado" 150.0
    , Produto 2 "Mouse" 80.0
    ]

buscarProduto :: Int -> Handler Produto
buscarProduto pid
    | pid == 1  = return (Produto 1 "Teclado" 150.0)
    | otherwise = throwError err404 { errBody = "Produto não encontrado" }

criarProduto :: NovoProduto -> Handler Produto
criarProduto novo = return $ Produto 99 (novoNome novo) (novoPreco novo)

deletarProduto :: Int -> Handler NoContent
deletarProduto _ = return NoContent

buscarPorNome :: Maybe String -> Handler [Produto]
buscarPorNome Nothing    = listarProdutos
buscarPorNome (Just nome) = return
    [p | p <- [Produto 1 "Teclado" 150.0], nome `isInfixOf` produtoNome p]

-- Server — combina handlers na ordem da API
server :: Server ProdutosAPI
server = listarProdutos
    :<|> buscarProduto
    :<|> criarProduto
    :<|> deletarProduto
    :<|> buscarPorNome

-- Executar com Warp
main :: IO ()
main = do
    putStrLn "Servidor em http://localhost:8080"
    run 8080 (serve (Proxy :: Proxy ProdutosAPI) server)

Scotty — API Simples e Rápida

Scotty: API expressiva similar ao Sinatra/Express
-- cabal install scotty aeson
import Web.Scotty
import Data.Aeson (object, (.=))

main :: IO ()
main = scotty 3000 $ do

    -- Middleware
    middleware logStdoutDev  -- requer wai-extra

    -- GET com path param
    get "/hello/:nome" $ do
        nome <- pathParam "nome"
        text $ "Olá, " <> nome <> "!"

    -- GET com query param
    get "/buscar" $ do
        q <- queryParam "q" :: ActionM Text
        json $ object ["busca" .= q]

    -- POST com JSON body
    post "/usuarios" $ do
        usuario <- jsonData :: ActionM Usuario
        status created201
        json usuario

    -- Tratamento de erro
    get "/produto/:id" $ do
        pid <- pathParam "id" :: ActionM Int
        produto <- liftIO $ buscarProduto pid
        case produto of
            Nothing -> do
                status notFound404
                json $ object ["erro" .= ("não encontrado" :: Text)]
            Just p  -> json p

    -- Rota não encontrada (catch-all)
    notFound $ json $ object ["erro" .= ("endpoint não existe" :: Text)]

Aeson — JSON em Haskell

Serialização e deserialização JSON type-safe
{-# LANGUAGE DeriveGeneric #-}
import Data.Aeson
import GHC.Generics (Generic)

-- Derivação automática via Generic
data Usuario = Usuario
    { usuarioId    :: Int
    , usuarioNome  :: String
    , usuarioEmail :: String
    } deriving (Show, Generic, FromJSON, ToJSON)

-- Customizar nomes dos campos JSON
data Config = Config
    { configHost :: String
    , configPort :: Int
    } deriving (Show, Generic)

instance ToJSON Config where
    toJSON = genericToJSON defaultOptions
        { fieldLabelModifier = \s -> drop 6 s  -- remover prefixo "config"
        }

instance FromJSON Config where
    parseJSON = genericParseJSON defaultOptions
        { fieldLabelModifier = \s -> "config" ++ s
        }

-- Parser manual para controle total
instance FromJSON Usuario where
    parseJSON = withObject "Usuario" $ \v ->
        Usuario
            <$> v .:  "id"
            <*> v .:  "nome"
            <*> v .:? "email" .!= "sem-email@exemplo.com"  -- campo opcional com default

-- Encode/decode
exemplo :: IO ()
exemplo = do
    let u = Usuario 1 "João" "joao@ex.com"

    -- Serializar para ByteString
    let bs = encode u
    putStrLn $ "JSON: " ++ show bs

    -- Deserializar
    case decode bs :: Maybe Usuario of
        Nothing -> putStrLn "Erro no parse"
        Just u' -> print u'

    -- eitherDecode para mensagem de erro detalhada
    case eitherDecode bs :: Either String Usuario of
        Left err -> putStrLn $ "Erro: " ++ err
        Right u' -> print u'

Arquitetura Recomendada e Trade-offs

Em producao, uma API Haskell costuma seguir o fluxo: rota tipada -> validacao de dominio pura -> camada de servico -> repositorio -> resposta serializada. Essa separacao permite testes rapidos no dominio e testes de integracao focados na infraestrutura.

DecisaoBeneficioCusto/limite
Servant em contratos complexosGarantia de tipo fim a fimCurva inicial maior
Scotty em MVPsEntrega rapidaMenos estrutura para crescimento
STM/async para concorrenciaMenos race conditionsExige modelagem cuidadosa
JSON com aeson derivadoProdutividade altaAtencao a compatibilidade de schema

Exemplos reais de uso

Backends de fintech, gateways de autenticacao, sistemas de auditoria e plataformas de dados com regras densas se beneficiam bastante desse modelo.

Servant demonstra o superpoder dos tipos em Haskell: a API é um tipo de nível de tipo (DataKinds) — não um valor ou string em runtime. Isso significa que o compilador pode provar que seu servidor implementa corretamente todos os endpoints com os tipos corretos. Impossível implementar um endpoint POST que retorne o tipo errado, ou esquecer de implementar um endpoint: não compila.
