Haskell — Dados e IA

Haskell combina pureza funcional com lazy evaluation para processamento de dados: hmatrix para álgebra linear, Cassava para CSV type-safe, pipes e conduit para streaming eficiente de grandes datasets, e funções puras compostas para transformações de dados.

O estilo funcional de Haskell é naturalmente expressivo para análise de dados: pipelines são composições de função. Uma transformação de dados é literalmente processar = filtrar . mapear . agrupar. A lazy evaluation permite que sequências infinitas de dados sejam processadas linha a linha sem carregar tudo na memória.

Cassava — CSV Type-Safe

Ler e escrever CSV com tipos derivados automaticamente
-- cabal install cassava vector
{-# LANGUAGE DeriveGeneric #-}
import Data.Csv
import qualified Data.ByteString.Lazy as BL
import qualified Data.Vector as V
import GHC.Generics

data Venda = Venda
    { vendaId      :: Int
    , vendaProduto :: String
    , vendaValor   :: Double
    , vendaRegiao  :: String
    } deriving (Show, Generic)

instance FromRecord Venda     -- derivação automática por posição
instance ToRecord   Venda

instance FromNamedRecord Venda  -- derivação por nome da coluna
instance ToNamedRecord   Venda
instance DefaultOrdered  Venda  -- ordem dos campos no header

-- Ler CSV por posição (sem header)
lerVendas :: FilePath -> IO (V.Vector Venda)
lerVendas path = do
    conteudo <- BL.readFile path
    case decode NoHeader conteudo of
        Left err     -> error err
        Right vendas -> return vendas

-- Ler CSV com header (por nome)
lerVendasNamed :: FilePath -> IO (V.Vector Venda)
lerVendasNamed path = do
    conteudo <- BL.readFile path
    case decodeByName conteudo of
        Left err          -> error err
        Right (_, vendas) -> return vendas

-- Análise com funções puras
analise :: V.Vector Venda -> IO ()
analise vendas = do
    let lista = V.toList vendas
    let total  = sum (map vendaValor lista)
    let count  = length lista
    let media  = total / fromIntegral count
    let maxVal = maximum (map vendaValor lista)

    putStrLn $ "Total vendas: " ++ show count
    putStrLn $ "Valor total: R$ " ++ show total
    putStrLn $ "Média: R$ " ++ show media
    putStrLn $ "Máximo: R$ " ++ show maxVal

    -- Agrupar por região (Map das somas)
    let porRegiao = foldl (\m v -> Map.insertWith (+) (vendaRegiao v) (vendaValor v) m)
                         Map.empty lista
    mapM_ (\(regiao, total) -> putStrLn $ regiao ++ ": R$ " ++ show total)
           (Map.toDescList porRegiao)

hmatrix — Álgebra Linear

Matrizes, vetores e operações numéricas
-- cabal install hmatrix
import Numeric.LinearAlgebra

-- Criar matrizes e vetores
a :: Matrix Double
a = (2><2) [1, 2,     -- 2x2 matrix from list
             3, 4]

b :: Matrix Double
b = (2><2) [5, 6,
             7, 8]

-- Operações matriciais
produto  = a <> b         -- multiplicação de matrizes
soma     = a + b
escalar  = 2 * a
transposta = tr a         -- transposta
det      = det a          -- determinante (-2.0)
inv      = inv a          -- inversa

-- Vetores
v :: Vector Double
v = fromList [1, 2, 3, 4, 5]

media = mean v             -- 3.0
std   = stdDev v           -- 1.58...
norm  = norm_2 v           -- norma euclidiana

-- Regressão linear simples
regressaoLinear :: Matrix Double -> Vector Double -> Vector Double
regressaoLinear x y = x <\> y    -- mínimos quadrados: x\y = inv(x^T x) x^T y

-- Exemplo de uso
exemplo :: IO ()
exemplo = do
    let n = 100
    let xs = fromList [1..fromIntegral n]
    -- X matrix com coluna de 1s (intercepto) e xs
    let xMat = (fromIntegral n><2) (concatMap (\x -> [1, x]) [1..fromIntegral n])
    let ys   = fromList $ map (\x -> 2*x + 3 + ruido x) [1..fromIntegral n]
    let coefs = regressaoLinear xMat ys
    print coefs    -- aprox: [3.0, 2.0]
  where ruido x = fromIntegral (mod (round x) 5) - 2

pipes — Streaming e Processamento Lazy

Processar arquivos gigantes linha a linha com O(1) memória
-- cabal install pipes pipes-csv
import Pipes
import qualified Pipes.Prelude as P
import qualified Pipes.ByteString as PB
import System.IO

-- Producer: gera valores para o pipeline
lerLinhas :: FilePath -> Producer String IO ()
lerLinhas path = do
    h <- liftIO $ openFile path ReadMode
    loop h
    liftIO $ hClose h
  where
    loop h = do
        eof <- liftIO $ hIsEOF h
        if eof then return ()
        else do
            linha <- liftIO $ hGetLine h
            yield linha
            loop h

-- Pipe: transforma Producer em Producer
parsearCSV :: Pipe String [String] IO ()
parsearCSV = P.map (splitOn ',')  -- transformar linha em campos
  where splitOn d s = case break (==d) s of
            (x, [])    -> [x]
            (x, _:rest) -> x : splitOn d rest

filtrarValidos :: Pipe [String] [String] IO ()
filtrarValidos = P.filter (\cols -> length cols >= 4)

-- Consumer: consome e processa
contarLinhas :: Consumer [String] IO Int
contarLinhas = P.length

-- Pipeline completo
processarArquivo :: FilePath -> IO ()
processarArquivo path = do
    n <- runEffect $
        lerLinhas path
        >-> P.drop 1              -- pular header
        >-> parsearCSV
        >-> filtrarValidos
        >-> P.map (!! 2)          -- pegar terceira coluna
        >-> P.map (read :: String -> Double)
        >-> P.filter (> 0)
        >-> P.fold (+) 0 id       -- somar acumulando
    putStrLn $ "Soma da coluna 3: " ++ show n

Arquitetura de Dados e IA em Producao

Em contexto corporativo, Haskell atua muito bem na camada de preparacao, validacao e orquestracao de dados. Um desenho comum e: ingestao streaming -> validacao tipada -> transformacoes puras -> persistencia/serving. A inferencia de modelos costuma ficar em frameworks especializados, enquanto Haskell garante consistencia de pipeline e observabilidade.

Exemplos reais

ETL de dados financeiros, saneamento de eventos de telemetria, feature engineering reprodutivel e servicos de inferencia com regras de negocio acopladas.

Trade-off central: Haskell oferece alta confiabilidade em processamento e contratos, mas o ecossistema de treino de modelos e menor que Python. Estrategia recomendada: integrar, nao substituir cegamente.

O paradigma de streaming com pipes/conduit em Haskell é matematicamente fundamentado em categoria: um Producer é uma co-rotina que produz valores, um Pipe é uma co-rotina que consome e produz, um Consumer consome. A composição >-> conecta co-rotinas em pipeline lazy garantindo que apenas o necessário é processado por vez — processamento de arquivos de 100 GB com KB de memória é o resultado natural.
