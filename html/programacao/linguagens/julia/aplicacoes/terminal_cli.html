Julia — Terminal e Linha de Comando

Julia funciona muito bem para ferramentas CLI, automação e utilitários de terminal. O caminho vai de scripts simples com ARGS até interfaces completas com ArgParse.jl, REPL.TerminalMenus, saída colorida e barras de progresso.

Em Julia, ferramentas de terminal são especialmente úteis quando a CLI também executa lógica numérica, análise ou transformação de dados. A mesma linguagem usada no pipeline científico pode virar a interface operacional que aciona esse pipeline.

ARGS e ArgParse.jl

Scripts simples e parser robusto de opções
# script básico com ARGS
if length(ARGS) < 2
		println("uso: julia app.jl <origem> <destino>")
		exit(1)
end

origem = ARGS[1]
destino = ARGS[2]
println("Copiando de $origem para $destino")

# Parser profissional com ArgParse.jl
using ArgParse

function build_parser()
		s = ArgParseSettings(description = "Ferramenta de processamento")

		@add_arg_table! s begin
				"--entrada", "-i"
						help = "arquivo de entrada"
						arg_type = String
						required = true
				"--saida", "-o"
						help = "arquivo de saída"
						arg_type = String
						default = "saida.csv"
				"--limite", "-n"
						help = "máximo de linhas"
						arg_type = Int
						default = 1000
				"--verbose", "-v"
						help = "ativa logs detalhados"
						action = :store_true
		end

		return s
end

args = parse_args(ARGS, build_parser())
println(args)

# ArgParse gera help, valida tipos e exige flags obrigatórias.

Menus Interativos no Terminal

REPL.TerminalMenus e fluxo guiado
using REPL.TerminalMenus

opcoes = ["Importar dados", "Treinar modelo", "Gerar relatório", "Sair"]
menu = RadioMenu(opcoes, pagesize = 4)

escolha = request("Escolha uma ação:", menu)

if escolha == 1
		println("Importando...")
elseif escolha == 2
		println("Treinando...")
elseif escolha == 3
		println("Gerando relatório...")
else
		println("Encerrando")
end

# Também existem menus multi-seleção para escolher vários itens.
Saída colorida com Crayons.jl
using Crayons

verde = Crayon(foreground = :green, bold = true)
vermelho = Crayon(foreground = :red)
azul = Crayon(foreground = :cyan)

println(verde("OK: operação concluída"))
println(vermelho("ERRO: arquivo não encontrado"))
println(azul("INFO: iniciando processamento"))

# Cores ajudam a distinguir status, warnings e falhas no terminal.

ProgressMeter e Comandos Externos

Barras de progresso e automação de shell
using ProgressMeter

@showprogress for i in 1:100
		sleep(0.02)
end

pb = Progress(50, desc = "Treinando")
for _ in 1:50
		sleep(0.05)
		next!(pb)
end

# Executar comandos externos
cmd = `git status --short`
run(cmd)                           # imprime na saída padrão

saida = read(`julia --version`, String)
println(saida)

# Pipeline entre comandos
run(pipeline(`echo hello`, `sort`))

# Comandos podem ser compostos com objetos Cmd, sem concatenar strings.
Tratamento de status e shell scripting
try
		run(`python --version`)
catch err
		println("Falha ao executar processo: $err")
end

cmd = Cmd(`ls`, ignorestatus = true)
resultado = read(cmd, String)

# Scripts Julia frequentemente substituem Bash/PowerShell quando:
# 1. há parsing complexo de dados
# 2. é preciso tipagem e testes
# 3. a automação chama rotinas científicas internas

Empacotamento e Distribuição

De script para ferramenta distribuível
ProjetoCLI/
├── Project.toml
├── src/
│   └── ProjetoCLI.jl
├── bin/
│   └── app.jl
└── test/
		└── runtests.jl

# bin/app.jl
using ProjetoCLI

function main()
		println("Executando CLI")
end

abspath(PROGRAM_FILE) == @__FILE__ && main()

# Reduzir startup com sysimage
using PackageCompiler
create_sysimage([:ArgParse, :ProgressMeter], sysimage_path = "cli.so")

# Executar:
# julia --sysimage cli.so bin/app.jl --entrada dados.csv

Quando Julia é ótima para CLI

  • Ferramentas internas de dados, ETL e automação científica
  • Utilitários que processam tabelas, modelos ou arquivos pesados
  • Pipelines que já dependem do ecossistema Julia

Limitações práticas

  • Startup JIT pode ser perceptível em CLIs muito curtas
  • Distribuição standalone exige mais cuidado que linguagens compiladas AOT puras
  • Ecossistema de TUI é menor que Rust ou Go
