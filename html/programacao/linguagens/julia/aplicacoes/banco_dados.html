Julia — Banco de Dados

Julia acessa bancos relacionais e analíticos por meio de drivers que seguem a interface comum de DBInterface.jl. Na prática, os pacotes mais usados são LibPQ.jl para PostgreSQL, SQLite.jl para SQLite, além de integração natural com DataFrames.jl e Tables.jl.

Em Julia, banco de dados e análise não ficam separados. Uma consulta SQL pode virar diretamente um DataFrame, passar por transformação vetorizada, alimentar um modelo estatístico e voltar ao banco no mesmo pipeline, sem sair da linguagem.

DBInterface.jl e Prepared Statements

Interface comum entre drivers
using DBInterface
using SQLite
using DataFrames

conn = SQLite.DB("app.db")

DBInterface.execute(conn, """
CREATE TABLE IF NOT EXISTS produtos (
    id INTEGER PRIMARY KEY,
    nome TEXT NOT NULL,
    preco REAL NOT NULL
)
""")

stmt = DBInterface.prepare(conn,
    "INSERT INTO produtos (nome, preco) VALUES (?, ?)")

DBInterface.execute(stmt, ("Teclado", 199.9))
DBInterface.execute(stmt, ("Mouse", 89.5))
DBInterface.close!(stmt)

resultado = DBInterface.execute(conn,
    "SELECT id, nome, preco FROM produtos WHERE preco > ?", (100.0,))

df = DataFrame(resultado)
println(df)

DBInterface.close!(conn)

# Prepared statements evitam SQL injection e reduzem parse repetido.
# A mesma ideia vale para PostgreSQL, MySQL e outros drivers aderentes.

PostgreSQL com LibPQ.jl

Conexão, consultas e transações
using LibPQ
using DataFrames
using DBInterface

conn = LibPQ.Connection("host=localhost port=5432 dbname=loja user=app password=senha")

DBInterface.execute(conn, "BEGIN")

try
    DBInterface.execute(conn,
        "INSERT INTO contas (nome, saldo) VALUES ($1, $2)",
        ("Ana", 1000.0)
    )
    DBInterface.execute(conn,
        "UPDATE contas SET saldo = saldo - $1 WHERE nome = $2",
        (150.0, "Ana")
    )
    DBInterface.execute(conn,
        "UPDATE contas SET saldo = saldo + $1 WHERE nome = $2",
        (150.0, "Carlos")
    )
    DBInterface.execute(conn, "COMMIT")
catch err
    DBInterface.execute(conn, "ROLLBACK")
    rethrow(err)
end

res = DBInterface.execute(conn,
    "SELECT id, nome, saldo FROM contas ORDER BY saldo DESC")

df = DataFrame(res)
println(first(df, 5))
close(conn)

# PostgreSQL usa placeholders $1, $2, $3...
# Transações devem proteger operações com consistência múltipla.
COPY para carga em lote
using LibPQ
using Tables
using DataFrames

conn = LibPQ.Connection("dbname=analytics")

lote = DataFrame(
    nome = ["A", "B", "C"],
    preco = [10.0, 20.0, 30.0],
)

copy_query = "COPY produtos (nome, preco) FROM STDIN (FORMAT CSV)"
open(LibPQ.CopyIn(conn, copy_query)) do io
    for row in eachrow(lote)
        println(io, "$(row.nome),$(row.preco)")
    end
end

close(conn)

# COPY é preferível a milhares de INSERTs unitários.

SQLite, DataFrames e Fluxo Analítico

Banco embutido com ida e volta para DataFrame
using SQLite
using DBInterface
using DataFrames
using Statistics

conn = SQLite.DB("metricas.db")

DBInterface.execute(conn,
    "CREATE TABLE IF NOT EXISTS vendas (dia TEXT, valor REAL, categoria TEXT)")

for linha in [("2026-01-01", 120.0, "A"), ("2026-01-02", 80.0, "B"), ("2026-01-03", 140.0, "A")]
    DBInterface.execute(conn,
        "INSERT INTO vendas (dia, valor, categoria) VALUES (?, ?, ?)", linha)
end

vendas = DataFrame(DBInterface.execute(conn, "SELECT * FROM vendas"))
resumo = combine(groupby(vendas, :categoria),
    :valor => mean => :media,
    :valor => sum => :total,
    nrow => :quantidade)

SQLite.load!(resumo, conn, "resumo_vendas", ifnotexists = false)
close(conn)

# SQLite é excelente para apps desktop, protótipos e pipelines locais.
# DataFrame permite análise antes de persistir resultados derivados.

ODBC, Cache e Boas Práticas

ODBC para sistemas legados e cache com Redis
using ODBC
using DataFrames

conn = ODBC.Connection("DSN=MeuERP")
clientes = DBInterface.execute(conn, "SELECT id, nome FROM clientes") |> DataFrame
close(conn)

# Para cache, pacotes Redis permitem padrão cache-aside.
# Ideia geral:
# 1. tentar ler chave no Redis
# 2. se não existir, consultar SQL
# 3. serializar e armazenar com TTL
# 4. invalidar após escrita

# Boas práticas:
# - nunca concatenar SQL com input do usuário
# - use transações em operações compostas
# - converta resultados para DataFrame só quando precisar análise tabular
# - para grandes volumes, itere resultados em streaming
# - documente migrações e esquema fora do código ad hoc
