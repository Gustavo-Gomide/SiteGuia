Julia — Web

Julia — Desenvolvimento Web

Julia não é conhecida principalmente por web, mas possui stack suficiente para APIs, painéis interativos e serviços backend com bom desempenho. A base é HTTP.jl; acima dele surgem camadas como Oxygen.jl e Genie.jl.

O ponto forte de Julia na web aparece quando o backend precisa fazer cálculo pesado, otimização, estatística ou inferência numérica dentro da própria requisição. Em vez de separar um backend web em outra linguagem e um motor científico separado, Julia permite unir os dois no mesmo processo.

HTTP.jl — Base do Servidor

Servidor HTTP puro, roteamento e respostas JSON
using HTTP
using JSON

function handler(req::HTTP.Request)
    if req.method == "GET" && req.target == "/health"
        body = JSON.json(Dict("status" => "ok", "tempo" => time()))
        return HTTP.Response(200, ["Content-Type" => "application/json"], body)
    elseif req.method == "GET" && req.target == "/"
        html = "<h1>Olá, Julia Web!</h1>"
        return HTTP.Response(200, ["Content-Type" => "text/html; charset=utf-8"], html)
    end

    return HTTP.Response(404, "rota não encontrada")
end

HTTP.serve(handler, "0.0.0.0", 8080)

# Handler recebe HTTP.Request e retorna HTTP.Response.
# A resposta pode ser texto, HTML, JSON, bytes ou stream.
# HTTP.jl também suporta routers e middleware compostos.
Router com parâmetros de rota e middleware
using HTTP
using JSON

router = HTTP.Router()

function logging_middleware(handler)
    return function(req::HTTP.Request)
        inicio = time()
        resp = handler(req)
        println("$(req.method) $(req.target) - $(round(time() - inicio, digits=4))s")
        resp
    end
end

HTTP.register!(router, "GET", "/api/status") do req
    HTTP.Response(200, JSON.json(Dict("ok" => true)))
end

HTTP.register!(router, "GET", "/api/usuarios/{id}") do req
    id = HTTP.getparams(req)["id"]
    body = JSON.json(Dict("id" => parse(Int, id), "nome" => "Usuário $id"))
    HTTP.Response(200, ["Content-Type" => "application/json"], body)
end

HTTP.serve(logging_middleware(router), ip"127.0.0.1", 8081)

Oxygen.jl — Microframework Moderno

Rotas declarativas com macros
using Oxygen
using JSON

@get "/" function(req)
    return html("<h1>Bem-vindo ao Oxygen</h1>")
end

@get "/api/produtos/{id}" function(req, id::Int)
    return Dict(
        "id" => id,
        "nome" => "Produto $id",
        "preco" => 99.9,
    )
end

@post "/api/somar" function(req)
    dados = json(req)
    a = dados["a"]
    b = dados["b"]
    return Dict("resultado" => a + b)
end

serve(host = "0.0.0.0", port = 8080)

# Vantagens do Oxygen:
# 1. menos boilerplate que HTTP.jl puro
# 2. JSON automático para Dicts e structs
# 3. extração tipada de parâmetros
# 4. ótimo para APIs pequenas e médias
AbordagemQuando usar
HTTP.jl puroMáximo controle, middleware manual, dependências mínimas
Oxygen.jlAPIs REST rápidas, sintaxe mais enxuta e didática
Genie.jlApps maiores com MVC, views, ORM e convenções

Genie.jl — MVC e Aplicações Maiores

Estrutura típica com rotas, controller e view
using GenieFramework
using Genie.Router
using Genie.Renderer.Json

route("/api/ping") do
    json(Dict("mensagem" => "pong"))
end

route("/usuarios/:id") do
    id = params(:id)
    json(Dict("id" => id, "perfil" => "analista"))
end

up()

# Em projetos reais Genie costuma organizar:
# app/resources/ controllers, models, views
# config/ para ambiente e rotas
# db/ para migrações e persistência
# SearchLight como ORM opcional

Bom encaixe de Genie

  • Painéis internos e CRUDs estruturados
  • Aplicações com muitas rotas e views server-side
  • Projetos que se beneficiam de convenções MVC

Pontos de atenção

  • Mais abstração e mais convenções para aprender
  • Menos minimalista que HTTP.jl ou Oxygen
  • Nem sempre vale a pena para APIs pequenas

JSON, WebSockets e Deploy

JSON, StructTypes e WebSockets
using HTTP
using JSON3
using StructTypes

struct Usuario
    id::Int
    nome::String
end

StructTypes.StructType(::Type{Usuario}) = StructTypes.Struct()

u = Usuario(1, "Ana")
payload = JSON3.write(u)
obj = JSON3.read(payload, Usuario)

# WebSocket com HTTP.jl
HTTP.WebSockets.listen("0.0.0.0", 9000) do ws
    send(ws, "conectado")
    while !eof(ws)
        msg = receive(ws)
        send(ws, "echo: $msg")
    end
end

# Deploy típico
# 1. executar Julia atrás de Nginx ou Caddy
# 2. usar PackageCompiler.jl para reduzir startup
# 3. separar configuração por ambiente
# 4. monitorar latência da primeira execução JIT
PackageCompiler para produção
using PackageCompiler

create_sysimage(
    [:HTTP, :JSON3, :Oxygen],
    sysimage_path = "webapp.so"
