Testes, Benchmark e Profiling

Test.jl, BenchmarkTools e Profile
using Test
using MeuPacote

@testset "soma" begin
		@test somar(2, 3) == 5
		@test somar(-1, 1) == 0
		@test_throws DomainError raiz_segura(-1)
end

@testset "aproximações numéricas" begin
		@test sin(π / 2) ≈ 1.0 atol = 1e-12
end

# Executar:
# pkg> test

using BenchmarkTools

dados = rand(10^6)
@benchmark sum($dados)
@btime sort($dados)

# Profiling
using Profile
Profile.clear()
@profile begin
		for _ in 1:100
				sort(rand(10_000))
		end
end
Profile.print()

# Memory allocation
@time sum(rand(10^6))
@allocated sum(rand(10^6))

Documentação e IDEs

Docstrings, Documenter.jl, VS Code e notebooks
# Docstring padrão Markdown antes da função
"""
		distancia(x1, y1, x2, y2)

Calcula a distância euclidiana entre dois pontos 2D.

# Exemplos
```julia
distancia(0, 0, 3, 4)
```
"""
function distancia(x1, y1, x2, y2)
		sqrt((x2 - x1)^2 + (y2 - y1)^2)
end

Fluxo recomendado

  • Desenvolver em ambiente local com Project.toml isolado
  • Usar Revise.jl para ciclo rápido
  • Benchmarkar com BenchmarkTools, não com @time isolado
  • Documentar API pública com docstrings e exemplos

Problemas comuns

  • Trabalhar no ambiente global e poluir dependências
  • Versionar código sem Manifest quando a reprodutibilidade importa
  • Comparar performance com primeira execução JIT
  • Esquecer compat bounds no Project.toml
