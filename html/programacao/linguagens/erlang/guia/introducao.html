Erlang — Introdução

Erlang

Erlang foi criado em 1986 pela Ericsson para resolver um problema real: como construir software de telecomunicações que nunca pode parar. A resposta foi uma linguagem funcional com processos leves, gerenciamento de memória por processo, passagem de mensagens e a filosofia "let it crash" — deixar processos falharem e reiniciar limpos, em vez de tentar recuperar estado corrompido.

O switch AXD301 da Ericsson, escrito em Erlang, atingiu 99,9999999% de disponibilidade (nine nines) — menos de 31 milissegundos de downtime por ano. A plataforma WhatsApp usou Erlang para suportar 2 bilhões de usuários com menos de 50 engenheiros.

História e Filosofia

1986 — Criação

Joe Armstrong, Robert Virding e Mike Williams criam Erlang internamente na Ericsson para substituir linguagens como PLEX e CHILL em switches de telefonia.

1998 — Open Source

A Ericsson bane brevemente o Erlang internamente (por ser open source), Joe Armstrong demite-se. A empresa reverte a decisão e libera o código publicamente no mesmo ano.

2007 — Erlang sob pressão

Com multi-core CPUs tornando-se ubíquas, Erlang ganha atenção por seu modelo de concorrência nativo. Rich Hickey cria Clojure inspirado em parte pelo modelo de atores Erlang.

2012 — Elixir

José Valim cria Elixir, uma linguagem que compila para a BEAM (VM do Erlang), trazendo metaprogramação e sintaxe moderna sem abrir mão das garantias de Erlang.

Hello, World! e Sintaxe Básica

hello.erl — módulo, função, compilação e execução
% hello.erl
-module(hello).          % nome do módulo (mesma que o arquivo)
-export([main/0,         % funções públicas: nome/aridade
         saudar/1]).

main() ->
    io:format("Hello, World!~n"),
    io:format("Erlang ~s~n", [erlang:system_info(otp_release)]),
    saudar("Mundo").

saudar(Nome) ->
    io:format("Olá, ~s!~n", [Nome]).

% Compilar e executar no terminal:
% $ erlc hello.erl        % gera hello.beam
% $ erl -noshell -s hello main -s init stop
%
% Ou no shell interativo (erl):
% 1> c(hello).            % compilar
% 2> hello:main().
%

Características-Chave

O que torna Erlang único

CaracterísticaDescriçãoBenefício
Processos levesNão são threads do SO — schedulados pela BEAMMilhões por nó, ~300 bytes cada
Imutabilidade totalVariáveis são vinculadas uma vez (single assignment)Sem data races, sem locks
Passagem de mensagensProcessos comunicam por send/receive assíncronoSem memória compartilhada
Let it crashNão trate erros — deixe crashar e supervisionarCódigo limpo, recuperação automática
Hot code reloadAtualizar código sem parar o sistemaDeployments zero-downtime nativos
GC por processoCada processo tem seu próprio heapSem STW global, latências previsíveis
Links e monitorsProcessos detectam morte de outros processosÁrvores de supervisão composíveis
Pattern matchingMatching em toda a linguagem (args, receive, case)Código declarativo e seguro

Quando Usar Erlang

Use Erlang quando:

  • Precisar de alta disponibilidade (>99.99%) — telecom, fintech, infraestrutura
  • Tiver muitas conexões simultâneas de longa duração (WebSocket, TCP, IoT)
  • Precisar de hot code reload em produção sem downtime
  • Construir sistemas distribuídos com múltiplos nós
  • O domínio for naturalmente orientado a mensagens (filas, protocolos)

Prefira outra ferramenta quando:

  • A tarefa for computação numérica intensiva (use C/Rust/Julia)
  • Precisar de UI desktop ou mobile nativa
  • O problema for simples e não precisar de concorrência massiva
  • A equipe não tiver experiência com programação funcional
  • Precisar de ecossistema de bibliotecas muito amplo (prefira Python, Java)

Joe Armstrong disse: "A maioria das linguagens foi projetada para programas que rodam em um único computador. Erlang foi projetado para programas que precisam continuar rodando mesmo quando partes do sistema falham." Esta filosofia, nascida de requisitos reais de telecomunicação, resultou na linguagem mais adequada para sistemas onde parar não é uma opção.
