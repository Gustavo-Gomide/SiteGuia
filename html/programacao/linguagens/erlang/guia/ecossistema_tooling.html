Erlang — Ecossistema e Tooling

O ecossistema Erlang gira em torno do rebar3 como gerenciador de projetos, do shell interativo erl para exploração e depuração, Dialyzer para análise estática de tipos, Observer para visualização gráfica de processos e o sistema de releases para empacotamento de aplicações autossuficientes.

Dialyzer usa Success Typings — em vez de exigir anotações de tipo completas, ele infere o que pode funcionar e aponta inconsistências. Diferente de sistemas de tipos tradicionais, nunca causa falso positivo: se Dialyzer reclama, é um bug real.

rebar3 — Gerenciador de Projetos

Criação, compilação, testes e release
# Criar novo projeto
rebar3 new app meu_app          # aplicação OTP
rebar3 new lib minha_lib        # biblioteca
rebar3 new release meu_sistema  # release completo
rebar3 new escript meu_script   # executável standalone

# Compilar
rebar3 compile

# Testes com EUnit
rebar3 eunit                    # rodar todos os testes
rebar3 eunit --module=contador  # só um módulo

# Testes com Common Test
rebar3 ct                       # testes de integração mais completos

# Análise estática
rebar3 dialyzer                 # rodar Dialyzer (lento na 1ª vez — cria PLT)

# Shell interativo com dependências carregadas
rebar3 shell

# Release
rebar3 release                  # gera _build/default/rel/
rebar3 tar                      # gera tarball para deploy

# Executável escript
rebar3 escriptize               # gera binário standalone

# Dependências (rebar.config)
rebar.config — configuração do projeto
{erl_opts, [debug_info]}.

{deps, [
    {cowboy, "2.10.0"},
    {jsx, "3.1.0"},
    {lager, "3.9.2"}
]}.

{relx, [
    {release, {meu_app, "1.0.0"}, [meu_app, sasl]},
    {mode, dev},               % dev = symlinks; prod = cópia
    {extended_start_script, true}
]}.

{profiles, [
    {prod, [{relx, [{mode, prod}]}]},
    {test, [{deps, [{proper, "1.4.0"}]}]}
]}.

Shell erl — REPL e Depuração

Comandos essenciais do shell Erlang
% Iniciar o shell
erl
erl -name no1@localhost -setcookie myCookie  % com nome de nó distribuído

% Comandos do shell
help().          % lista comandos disponíveis
q().             % sair (quit)
halt().          % parar VM imediatamente

% Compilar e carregar módulos
c(modulo).       % compila modulo.erl e carrega
c(modulo, [debug_info, {outdir, "ebin"}]).
l(modulo).       % carregar .beam sem compilar
m().             % listar módulos carregados
m(modulo).       % informações de um módulo

% Histórico de expressões
v(1).            % valor da expressão 1
e(3).            % re-executar expressão 3
f().             % limpar todas as bindigs
f(X).            % limpar binding de X

% Listar e inspecionar processos
i().             % list processos ativos (como top)
i(Pid).          % info do PID
regs().          % processos com nome registrado

% Documentação inline
h(lists, sort).  % doc da função (OTP 23+)
h(io).           % doc do módulo

% Conectar ao nó remoto
net_adm:ping('outro@host').     % conectar
nodes().                         % listar nós conectados
rpc:call('outro@host', io, format, ["olá~n"]).

Dialyzer e Especificações de Tipo

-spec e -type para análise estática
-module(calculadora).
-export([somar/2, dividir/2]).

%% Definir tipos customizados
-type positivo() :: pos_integer().
-type resultado(T) :: {ok, T} | {erro, atom()}.

%% -spec: especificação de tipo da função
-spec somar(number(), number()) -> number().
somar(A, B) -> A + B.

-spec dividir(number(), number()) -> resultado(float()).
dividir(_, 0) -> {erro, divisao_por_zero};
dividir(A, B) -> {ok, A / B}.

%% Tipos compostos
-type usuario() :: #{
    id    := pos_integer(),
    nome  := binary(),
    email := binary(),
    ativo := boolean()
}.

-spec criar_usuario(pos_integer(), binary(), binary()) -> usuario().
criar_usuario(Id, Nome, Email) ->
    #{id => Id, nome => Nome, email => Email, ativo => true}.

Observer — Monitor Visual em Tempo Real

Iniciar Observer e navegar pelo sistema vivo
%% Iniciar Observer (GUI wxWidgets)
observer:start().

%% Abas disponíveis:
%% System   — CPU, memória, processos ativos, uptime
%% Load chart — uso de cada scheduler em tempo real
%% Memory   — heap, binaries, ETS, código, sistema
%% Applications — árvore de supervisão visual interativa
%% Processes — tabela ordenável (memória, reductions, msg queue)
%% Ports    — sockets, arquivos, drivers C abertos
%% ETS      — todas as tabelas com size e memory
%% Mnesia   — tabelas e fragmentos (se Mnesia rodando)
%% Trace overview — interface gráfica para dbg

%% Clicar em um processo: ver estado, dicionário, stack trace,
%% links, monitors, heap, binaries — tudo sem reiniciar

%% Alternativa textual — etop (como top do Unix)
etop:start([{output, text}, {interval, 2}, {lines, 20}]).
etop:stop().

%% Métricas via código
erlang:memory().                 % [{total, N}, {processes, N}, ...]
erlang:statistics(run_queue).    % processos aguardando CPU
erlang:statistics(wall_clock).   % tempo real desde start
erlang:statistics(reductions).   % reduções desde start

Principais Bibliotecas do Ecossistema

Cowboy

Servidor HTTP/WebSocket puro Erlang. Base do Phoenix (Elixir). Suporta HTTP/1.1, HTTP/2 e WebSocket com handlers roteados.

RabbitMQ

Broker AMQP escrito em Erlang. Um dos sistemas de mensageria mais usados em produção, aproveitando a robustez do OTP.

jiffy / thoas

Parsers JSON. jiffy usa NIF C para máxima velocidade; thoas é pure Erlang, sem dependência de compilador C.

hackney

Cliente HTTP assíncrono Erlang com suporte a streaming, connection pools e SSL — muito usado via HTTPoison no Elixir.

lager / logger

lager é o logger histórico Erlang. OTP 21+ inclui o módulo logger nativo com formatters e handlers configuráveis.

PropEr / QuickCheck

Property-based testing: geram casos de teste aleatórios baseados em propriedades — muito mais cobertura que testes manuais.
