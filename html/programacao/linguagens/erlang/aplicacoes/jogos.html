Erlang — Jogos

Erlang não é uma linguagem de engine gráfica ou renderização 3D, mas é extremamente competente nos bastidores de jogos online: matchmaking, presença, chat, sessões multiplayer, gateways de evento e serviços que precisam sustentar muitos jogadores simultâneos com alta disponibilidade.

Em game backend, a pergunta mais cara não é “quantos polígonos eu desenho?”, mas “como mantenho milhares de jogadores conectados, sincronizados e recuperáveis quando partes do sistema falham?”. É aí que Erlang aparece com força.

Onde a linguagem entrega valor

Matchmaking

filas concorrentes, balanceamento por região e criação de salas sob demanda.

Presença e chat

conexões longas, canais, status online e broadcast com isolamento de falhas.

Serviços persistentes

guildas, lobby, ranking, inventário e telemetria de sessão.

Arquitetura típica de backend multiplayer

Essa ideia funciona bem porque partidas, lobbies e canais são unidades naturais de concorrência. Cada uma pode ser tratada como processo com estado local, timeout, reinício e observabilidade próprios.

Exemplo de sala como processo

Loop simplificado de sala multiplayer
loop(Estado = #{jogadores := Jogadores, eventos := Eventos}) ->
    receive
        {entrar, PlayerPid} ->
            monitor(process, PlayerPid),
            loop(Estado#{jogadores => [PlayerPid | Jogadores]});
        {acao, PlayerPid, Acao} ->
            broadcast(Jogadores, {evento_jogo, PlayerPid, Acao}),
            loop(Estado#{eventos => [Acao | Eventos]});
        {'DOWN', _, process, PlayerPid, _} ->
            loop(Estado#{jogadores => lists:delete(PlayerPid, Jogadores)})
    end.

Não é um servidor de jogo completo, mas ensina o essencial: o domínio multiplayer pode ser descrito como troca de mensagens, monitoramento de participantes e evolução explícita de estado.

Erlang é excelente para o backend social e distribuído do jogo. Para física, renderização e frame loop em tempo real estrito, a engine continua sendo a ferramenta certa.
