Erlang — Terminal e CLI

Erlang não é lembrado primeiro como linguagem de linha de comando, mas oferece dois caminhos bastante úteis: escript para ferramentas pequenas e releases administráveis para utilitários operacionais capazes de interagir com sistemas BEAM vivos.

A melhor CLI em Erlang geralmente não é a que imprime mais cores no terminal, e sim a que automatiza tarefas operacionais com segurança, códigos de saída corretos e integração limpa com processos e releases.

escript: scripts executáveis de forma simples

escript permite empacotar um arquivo Erlang executável diretamente pela shell. Para tarefas administrativas, transformação de dados, checagens de ambiente e automação de operações, isso costuma ser suficiente.

CLI mínima com escript
#!/usr/bin/env escript
%%! -smp enable -noshell

main(["status"]) ->
    io:format("status=ok~n"),
    halt(0);
main(["versao"]) ->
    io:format("otp=~s~n", [erlang:system_info(otp_release)]),
    halt(0);
main(_) ->
    io:format("uso: ferramenta <status|versao>~n"),
    halt(1).

Esse padrão já ensina algo importante: uma CLI de produção precisa definir interface, retorno e erro com a mesma disciplina que qualquer serviço.

CLIs para operar releases e clusters

Erlang também é forte quando a ferramenta de terminal precisa conversar com um nó já em execução: consultar estado, pedir mudança de configuração, acionar manutenção e coletar diagnósticos. Nesse cenário, a CLI vira uma extensão do ambiente operacional.

Boas práticas operacionais

Faça

  • retorne códigos de saída claros
  • trate timeout e falha de nó explicitamente
  • registre ações críticas e contexto mínimo
  • mantenha comandos pequenos e previsíveis

Evite

  • scripts que dependem de estado implícito do ambiente
  • comandos interativos demais para automação
  • acoplar lógica de operação com parsing ad hoc em muitos módulos
  • assumir que o nó remoto sempre está saudável

Em times de infraestrutura, uma CLI pequena, idempotente e bem distribuída costuma valer mais do que um painel web pesado para tarefas recorrentes.
