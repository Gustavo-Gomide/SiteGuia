Ferramentas e Infraestrutura para Desenvolver em Common Lisp

O ecossistema Lisp combina compiladores maduros, gerenciadores de pacotes pragmáticos e fluxo interativo de desenvolvimento centrado no REPL.

A produtividade em Lisp vem da integração entre compilador, editor e runtime vivo, não apenas de sintaxe.

Quicklisp e ASDF

Quicklisp simplifica descoberta e instalação de bibliotecas. ASDF define, constrói e carrega sistemas.

Lisp/Common Lisp
; carregar biblioteca
(ql:quickload :hunchentoot)

; pesquisar sistemas por termo
(ql:system-apropos "json")

; exemplo de .asd
(asdf:defsystem "meu-projeto"
	:version "0.1.0"
	:depends-on ("alexandria" "cl-ppcre")
	:components ((:file "package")
							 (:file "core")))

Boa prática: manter dependências mínimas e versionamento semântico explícito no sistema.

SLIME e SLY no Emacs

SLIME e SLY oferecem IDE interativa para Common Lisp com compilação incremental, inspeção de objetos e depuração em tempo real.

Fluxo clássico com SLIME
  • C-c C-c: compila forma/função atual
  • Inspector para navegar estruturas complexas
  • Debugger interativo com restarts
Alternativa moderna com SLY
  • Interface mais modular e atualizada
  • Integração robusta com múltiplos REPLs
  • Extensões ativas para workflows recentes
Lisp/Common Lisp
; exemplo mínimo no Emacs init
(setq inferior-lisp-program "sbcl")
(require 'slime)
(slime-setup '(slime-fancy))

; alternativa com SLY
; M-x package-install RET sly RET
; M-x sly

Bibliotecas Relevantes

Hunchentoot

Servidor web Common Lisp usado para APIs e aplicações HTTP.

CL-PPCRE

Biblioteca de expressões regulares com API madura e eficiente.

Alexandria

Conjunto de utilitários gerais que evita reescrita de funções recorrentes.

Bordeaux-Threads

Abstração portátil para multithreading entre implementações Lisp.

Lisp/Common Lisp
(ql:quickload '(:hunchentoot :cl-ppcre :alexandria :bordeaux-threads))

(cl-ppcre:scan "^ab" "abc")
(alexandria:flatten '((1 2) (3 (4))))
