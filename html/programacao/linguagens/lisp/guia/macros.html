Lisp - Macros

Código Que Escreve Código

Macros expandem formas antes da avaliação normal, permitindo introduzir novas construções sem alterar o compilador.

Aprender macros é cruzar a fronteira entre usar a linguagem e moldar a linguagem para o seu domínio.

defmacro

defmacro recebe formas Lisp e retorna outras formas Lisp. A expansão ocorre em compile-time (ou load-time), antes da execução da forma final.

Lisp/Common Lisp
; macro básica
(defmacro incf2 (var)
	`(setf ,var (+ ,var 2)))

; backquote (`) monta template
; unquote (,) injeta expressão
; splice (,@) injeta lista de elementos

(defmacro my-progn (&body body)
	`(progn ,@body))

Regra mental útil: função calcula valores; macro calcula código.

Macros Utilitárias

Reimplementar formas conhecidas ajuda a entender expansion e hygiene.

Lisp/Common Lisp
; when reimplementado
(defmacro my-when (condicao &body corpo)
	`(if ,condicao
			 (progn ,@corpo)
			 nil))

; unless reimplementado
(defmacro my-unless (condicao &body corpo)
	`(if (not ,condicao)
			 (progn ,@corpo)
			 nil))

; macro fictícia de domínio
(defmacro com-macro-personagem ((nome hp) &body corpo)
	`(let ((personagem (list :nome ,nome :hp ,hp)))
		 ,@corpo))

; hygiene com gensym
(defmacro with-temp ((expr) &body body)
	(let ((tmp (gensym "TMP-")))
		`(let ((,tmp ,expr))
			 ,@body)))

Macros Avançadas

Ferramentas além de defmacro
  • Reader macros para alterar leitura sintática (nível de reader)
  • define-symbol-macro para expansão de símbolos em formas
  • Cálculo em compile-time para eliminar custo em runtime
Lisp/Common Lisp
; symbol macro
(define-symbol-macro pi2 6.283185307)

(defun circunferencia (r)
	(* pi2 r))

; compile-time computation simplificada
(defmacro const-fold-soma (a b)
	(if (and (numberp a) (numberp b))
			(+ a b)
			`(+ ,a ,b)))

Evite reader macros em excesso: elas elevam expressividade, mas podem reduzir legibilidade para quem não conhece o dialeto interno do projeto.

Exemplos Práticos

Lisp/Common Lisp
; macro de logging
(defmacro with-log ((tag) &body body)
	`(progn
		 (format t "[~a] inicio~%" ,tag)
		 (prog1 (progn ,@body)
			 (format t "[~a] fim~%" ,tag))))

; mini DSL de rotas
(defmacro defroutes (&body routes)
	`(list
		,@(mapcar
			 (lambda (r)
				 `(list ,(first r) #',(second r)))
			 routes)))

(defparameter *rotas*
	(defroutes
		("/" home-handler)
		("/users" users-handler)))

Esse estilo de DSL reduz boilerplate e permite que o domínio de problema apareça diretamente na sintaxe.
