Lisp - Introdução

Lisp

A família Lisp introduziu ideias que moldaram quase toda a computação moderna: coleta de lixo, código como dados e metaprogramação em alto nível.

Se você entende Lisp, você passa a enxergar linguagens não como sintaxe bonita, mas como modelos de avaliação e transformação de estruturas simbólicas.

História

Lisp foi criado em 1958 por John McCarthy, no MIT, inicialmente como notação prática para manipulação simbólica em IA. Desde então, diversos dialetos surgiram para necessidades diferentes.

Linha do tempo resumida
AnoMarco
1958McCarthy propõe Lisp e formaliza processamento de listas simbólicas
1970sConsolidação de múltiplos dialetos acadêmicos e industriais
1984Common Lisp começa a unificar implementações incompatíveis
1994ANSI Common Lisp é padronizado (X3.226-1994)
1975-1998Scheme evolui com foco minimalista e semântico (R5RS)
2007Clojure surge como Lisp funcional na JVM

Dialetos centrais para estudo hoje: Common Lisp (engenharia geral), Scheme (didática e semântica) e Clojure (produção na JVM).

Características

Homoiconicidade

Programas são representados por estruturas da própria linguagem, tornando macros extremamente poderosas.

Listas Como Código

Uma expressão é literalmente uma lista: operador na cabeça e argumentos no restante.

Avaliação Uniforme

O avaliador percorre s-expressions de modo previsível, reduzindo "mágicas" sintáticas.

REPL Interativo

Desenvolvimento incremental, exploratório e orientado a feedback imediato.

Metaprogramação

Macros permitem criar mini linguagens e abstrações de domínio com segurança.

Garbage Collection

Gerência de memória automática desde as origens da família Lisp.

Instalação

Para um ambiente moderno de Common Lisp: instale SBCL, adicione Quicklisp para dependências e use SLIME no Emacs para integração com REPL.

Lisp/Common Lisp
; 1) Instalar SBCL (Windows com winget)
winget install SBCL.SBCL

; 2) Baixar e instalar Quicklisp
curl -O https://beta.quicklisp.org/quicklisp.lisp
sbcl --load quicklisp.lisp

; 3) No REPL do SBCL
(quicklisp-quickstart:install)
(ql:add-to-init-file)

; 4) Emacs + SLIME (MELPA)
M-x package-install RET slime RET

No Emacs, após configurar o caminho do SBCL, use M-x slime para abrir sessão interativa e compilar funções diretamente do buffer.

Primeiro Programa

Lisp/Common Lisp
; hello world
(format t "Hello, Lisp!~%")

; fatorial recursivo
(defun factorial (n)
	(if (<= n 1)
			1
			(* n (factorial (- n 1)))))

(factorial 5) ; => 120

; sessão REPL
CL-USER> (+ 10 20)
30
CL-USER> (mapcar #'1+ '(1 2 3))
(2 3 4)

Perceba o ciclo: escrever pequena função, avaliar no REPL, inspecionar saída, ajustar e evoluir. Esse fluxo é o coração da produtividade em Lisp.
