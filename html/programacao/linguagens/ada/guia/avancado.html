Ada Avançado

OOP, Concorrência, Generics e Verificação Formal

Os mecanismos que tornam Ada a escolha para sistemas onde a falha não é uma opção.

Ada vai muito além de um sistema de tipos robusto. Seu modelo de objetos com tagged types, a concorrência unificada por tasks e protected objects, os generics com restrições formais de gênero e o subconjunto SPARK para verificação matemática formam um conjunto de ferramentas sem paralelo em engenharia de software crítico. Esta página cobre os tópicos que separam programadores Ada básicos dos profission ais que escrevem software para aviões e satélites.

1. Orientação a Objetos com Tagged Types

Ada não tem "classes" como Java ou C++. Em vez disso, usa tagged types: records que podem ser estendidos por herança e ter operações primitivas (métodos virtuais). Ada 95 foi a primeira linguagem ISO com OOP completo padronizado.

ada — hierarquia de tipos tagged
package Animais is

   -- Tipo base abstrato (como classe abstrata Java)
   type Animal is abstract tagged record
      Nome : access constant String;
      Peso : Float := 0.0;
   end record;

   -- Operação primitiva abstrata (método virtual puro)
   procedure Emitir_Som (A : Animal) is abstract;
   function  Descricao  (A : Animal) return String is abstract;

   -- Subtipo derivado: herda + adiciona campos
   type Cao is new Animal with record
      Raca : String (1 .. 30);
   end record;

   overriding
   procedure Emitir_Som (A : Cao);
   overriding
   function  Descricao  (A : Cao) return String;

   type Gato is new Animal with null record;

   overriding
   procedure Emitir_Som (A : Gato);
   overriding
   function  Descricao  (A : Gato) return String;

end Animais;
ada — dispatch dinâmico via class-wide types
with Animais; use Animais;
with Ada.Text_IO; use Ada.Text_IO;

procedure OOP_Demo is
   -- Animal'Class é o "class-wide type" — referência polimórfica
   type Animal_Ref is access all Animal'Class;

   Zoo : array (1 .. 3) of Animal_Ref;
begin
   Zoo (1) := new Cao  '(Nome => new String'("Rex"), Peso => 30.0, Raca => (others => ' '));
   Zoo (2) := new Gato '(Nome => new String'("Mimi"), Peso => 4.5);

   for I in Zoo'Range loop
      if Zoo (I) /= null then
         Emitir_Som (Zoo (I).all);  -- dispatch dinâmico!
         Put_Line   (Descricao (Zoo (I).all));
      end if;
   end loop;
end OOP_Demo;

O marcador overriding é um contrato com o compilador: ele garante que você está sobrescrevendo uma operação da classe pai. Em C++, esquecer virtual ou usar o nome errado introduz bugs silenciosos; Ada detecta isso em compilação.

2. Tasks e Concorrência Nativa

Ada tem concorrência como parte da linguagem, não de uma biblioteca. Tasks são threads de execução declaradas como tipos ou objetos. Protected objects são monitores: objetos com dados compartilhados protegidos por mutexes gerenciados implicitamente.

ada — task simples com rendezvous
with Ada.Text_IO; use Ada.Text_IO;

procedure Concorrencia_Demo is

   -- Declaração de task: especificação com "entries" (pontos de sincronização)
   task Trabalhador is
      entry Iniciar (Tarefa : String);
      entry Parar;
   end Trabalhador;

   task body Trabalhador is
      Trabalho_Atual : String (1 .. 50);
   begin
      loop
         select
            accept Iniciar (Tarefa : String) do
               Trabalho_Atual (1 .. Tarefa'Length) := Tarefa;
               Put_Line ("Iniciando: " & Tarefa);
            end Iniciar;
         or
            accept Parar;
            Put_Line ("Trabalhador parando.");
            exit;
         end select;
      end loop;
   end Trabalhador;

begin
   Trabalhador.Iniciar ("Processar relatório");  -- rendezvous (sincronização)
   Trabalhador.Parar;
end Concorrencia_Demo;
ada — protected object (monitor seguro)
protected Contador_Global is
   procedure Incrementar;
   function  Valor return Natural;
   entry     Aguardar_Minimo (Min : Natural);
private
   V : Natural := 0;
end Contador_Global;

protected body Contador_Global is
   procedure Incrementar is
   begin  V := V + 1;  end Incrementar;

   function Valor return Natural is
   begin  return V;  end Valor;

   -- entry com "barrier" — a task espera até a condição ser verdadeira
   entry Aguardar_Minimo (Min : Natural) when V >= Min is
   begin  null;  end Aguardar_Minimo;
end Contador_Global;

3. Generics (Programação Genérica)

Os generics de Ada são análogos aos templates de C++, mas com verificação de tipo completa na instanciação: o compilador verifica os parâmetros formais antes de instanciar. Isso evita os famosos erros "template instantiation" de C++ com mensagens incompreensíveis.

ada — generic stack (pilha genérica)
generic
   Capacidade : in Positive;          -- parâmetro genérico de valor
   type Elemento (<>) is private;   -- parâmetro genérico de tipo (<> = indefinido = qualquer tipo)
package Pilha_Generica is

   type Pilha is limited private;

   procedure Empilhar  (P : in out Pilha; Item : Elemento);
   procedure Desempilhar (P : in out Pilha; Item : out Elemento);
   function  Esta_Vazia  (P : Pilha) return Boolean;
   Pilha_Cheia : exception;
   Pilha_Vazia : exception;

private
   type Indice is range 0 .. Capacidade;
   type Vetor  is array (1 .. Capacidade) of Elemento;
   type Pilha  is record
      Dados : Vetor;
      Topo  : Indice := 0;
   end record;
end Pilha_Generica;

-- Instanciação: cria tipos concretos
package Pilha_Inteiros  is new Pilha_Generica (Capacidade => 100, Elemento => Integer);
package Pilha_Strings   is new Pilha_Generica (Capacidade => 50,  Elemento => String);

4. SPARK Ada — Verificação Formal

SPARK é um subconjunto de Ada projetado para verificação formal. Com a ferrramenta GNATprove, é possível provar matematicamente que um programa SPARK não tem buffer overflows, divisões por zero, acesso a ponteiros nulos, nem violações de pré/pós-condições — sem executar o código.

ada spark — contratos formais com Pre/Post
package Conta_Bancaria
   with SPARK_Mode => On
is
   type Saldo_T is range 0 .. 1_000_000_00;  -- em centavos

   type Conta is private;

   function Saldo (C : Conta) return Saldo_T;

   procedure Depositar (C : in out Conta; Valor : Saldo_T)
      with
         Pre  => Valor > 0,
         Post => Saldo (C) = Saldo (C)'Old + Valor;
         -- Pós-condição: saldo aumentou exatamente pelo valor depositado

   procedure Sacar (C : in out Conta; Valor : Saldo_T)
      with
         Pre  => Valor > 0 and Saldo (C) >= Valor,
         Post => Saldo (C) = Saldo (C)'Old - Valor;
         -- GNATprove PROVA que saldo nunca fica negativo

private
   type Conta is record
      Saldo_Interno : Saldo_T := 0;
   end record;
end Conta_Bancaria;
terminal — executar GNATprove
# Provar ausência de erros em tempo de execução (RTE)
gnatprove -P meu_projeto.gpr --mode=prove --level=2

# Saída esperada para código correto:
# conta_bancaria.adb:12: info: precondition proved
# conta_bancaria.adb:15: info: postcondition proved
# conta_bancaria.adb:22: info: overflow check proved

Nível Proof

--mode=prove: verifica pré/pós-condições formalmente. Exige que cada subprograma tenha contratos completos. Usado no desenvolvimento de sistemas DO-178C DAL-A (mais crítico).

Nível Flow

--mode=flow: analisa fluxo de dados — detecta variáveis não inicializadas, efeitos colaterais não declarados e dependências incorretas. Mais rápido que prova completa.

SPARK vs Rust

Rust garante segurança de memória estaticamente. SPARK vai além: prova propriedades funcionais arbitrárias (cálculos corretos, saldos nunca negativos, buffers nunca fora dos limites).

Casos Industriais

SPARK é usado no kernel do sistema operacional da Muen (separation kernel certificado), no MULTOS (smartcards) e em componentes do sistema ATC britânico (NATS).

5. Perfil Ravenscar — Tempo Real Certificável

O Ravenscar Profile é um subconjunto das tasks Ada projetado para sistemas de tempo real estrito (HRT) onde é necessário calcular o WCET. Ele restringe algumas construções de concorrência para tornar o comportamento temporal analisável.

ada — ativação periódica com Ravenscar
pragma Profile (Ravenscar);   -- ativa restrições do perfil

with Ada.Real_Time; use Ada.Real_Time;
with Ada.Text_IO;

procedure Sistema_Tempo_Real is

   -- Task periódica com period fixo — padrão Ravenscar
   task Sensor_Task
      with Priority => 10;  -- priority ceiling

   task body Sensor_Task is
      Proximo_Ciclo : Time     := Clock;
      Periodo       : constant Time_Span := Milliseconds (100);  -- 100ms
   begin
      loop
         Proximo_Ciclo := Proximo_Ciclo + Periodo;
         delay until Proximo_Ciclo;  -- dormir até próximo ciclo
         Ada.Text_IO.Put_Line ("Lendo sensor...");
         -- processar leitura
      end loop;
   end Sensor_Task;

begin
   null;  -- tasks iniciam automaticamente
end Sistema_Tempo_Real;
