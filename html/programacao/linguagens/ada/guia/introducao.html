Introdução ao Ada

Ada — A Linguagem da Segurança Crítica

Projetada pelo Departamento de Defesa dos EUA para sistemas onde falhas custam vidas.

Ada é uma linguagem de programação de alto nível fortemente tipada, criada na virada dos anos 1980 por mandato do DoD norte-americano para unificar as dezenas de linguagens usadas em sistemas militares e tornar o software de missão crítica mais seguro, confiável e manutenível. Hoje, Ada é a escolha padrão em aviação civil, defesa, sistemas ferroviários e espaçonaves, pois oferece verificação de tipos em tempo de compilação, suporte nativo a concorrência, tratamento estruturado de exceções e contratos formais por design.

O que é Ada?

Ada é uma linguagem de programação de uso geral, mas com foco declarado em correção, legibilidade e manutenibilidade em detrimento de conveniências sintáticas de curto prazo. Diferente de C ou Python, Ada foi projetada de cima para baixo por um comitê de especialistas para satisfazer requisitos formais de engenharia de software.

O nome homenageia Ada Lovelace (1815–1852), matemática britânica considerada a primeira programadora da história por ter escrito, em 1843, um algoritmo para a Máquina Analítica de Charles Babbage calcular os Números de Bernoulli.

Fortemente Tipada

Ada exige que todos os tipos sejam explicitamente declarados. Conversões implícitas entre tipos numéricos distintos são proibidas — erros de tipo são capturados em tempo de compilação, não em produção.

Concorrência Nativa

Ada inclui tasks e protected objects como construções da própria linguagem. Não depende de bibliotecas externas para programação concorrente segura.

Design by Contract

Ada 2012 introduziu aspects para pré-condições (Pre), pós-condições (Post) e invariantes de tipo — contratos formais verificados em tempo de execução ou em análise estática com SPARK.

Verificação Formal

O subconjunto SPARK Ada permite provar matematicamente a ausência de buffer overflows, divisões por zero, fluxo de dados indevido e outros defeitos — sem executar o código.

História e Origem

Na década de 1970 o Departamento de Defesa dos EUA descobriu que seus sistemas militares usavam mais de 450 linguagens de programação diferentes. Isso tornava impossível a interoperabilidade, aumentava os custos de manutenção e multiplicava os riscos de bugs. Em 1975, o DoD criou o High Order Language Working Group (HOLWG) para especificar uma linguagem única.

Linha do Tempo Ada

  • 1975 — DoD cria o HOLWG; documento Strawman define requisitos iniciais.
  • 1977 — Documento Ironman estabelece requisitos formais detalhados.
  • 1978 — Quatro equipes competem; Jean Ichbiah (CII Honeywell Bull, França) vence.
  • 1980 — Ada 80 publicada como linguagem militar MIL-STD-1815.
  • 1983 — Ada 83 torna-se padrão ANSI/ISO; primeira versão oficial.
  • 1995 — Ada 95 adiciona OOP, hierarquias de tipos e interfaces — primeira grande linguagem OO padronizada pela ISO.
  • 2005 — Ada 2005 traz interfaces múltiplas e melhorias de container.
  • 2012 — Ada 2012 introduz contratos formais (Pre, Post, Type_Invariant).
  • 2022 — Ada 2022 adiciona 'Image para qualquer tipo, declare expressions e melhorias em SPARK.

Por que Ada não "dominou" o mundo? Ada foi adotada obrigatoriamente pelo DoD, mas o setor civil preferiu C e depois C++ pela velocidade de escrita e pela grande base de código existente. Ada permaneceu como referência de qualidade em setores onde vidas dependem do software.

Primeiro Programa em Ada

O programa mais simples em Ada segue uma estrutura de procedure principal, com o pacote Ada.Text_IO para saída de texto. Ada usa with para importar pacotes e use para trazer os identificadores ao escopo atual.

ada — ola_mundo.adb
with Ada.Text_IO;
use  Ada.Text_IO;

procedure Ola_Mundo is
begin
   Put_Line ("Olá, Mundo!");
end Ola_Mundo;

Anatomia do programa:

  • with Ada.Text_IO — importa ("contextualiza") o pacote de I/O padrão.
  • use Ada.Text_IO — permite usar Put_Line sem o prefixo Ada.Text_IO..
  • procedure Ola_Mundo is — declara a procedure principal. O nome do arquivo deve ser ola_mundo.adb (em minúsculas).
  • begin ... end — bloco executável da procedure.
  • O compilador Ada converte automaticamente o identificador para o caso correto — Ada não é case-sensitive mas é case-preserving.
terminal — compilar e executar com GNAT
# Compilar com GNAT (GNU Ada Compiler)
gnatmake ola_mundo.adb

# Executar
./ola_mundo

Estrutura Básica de um Programa

Ada organiza o código em packages (equivalentes a módulos ou namespaces). Todo programa tem uma procedure principal. A estrutura mais completa usa pacotes separados em arquivos de especificação (.ads) e corpo (.adb).

ada — calculadora.ads (especificação do pacote)
package Calculadora is

   -- Tipos públicos
   type Numero is new Float;

   -- Funções públicas
   function Somar   (A, B : Numero) return Numero;
   function Dividir (A, B : Numero) return Numero;

end Calculadora;
ada — calculadora.adb (corpo do pacote)
with Ada.Exceptions;

package body Calculadora is

   function Somar (A, B : Numero) return Numero is
   begin
      return A + B;
   end Somar;

   function Dividir (A, B : Numero) return Numero is
   begin
      if B = 0.0 then
         raise Constraint_Error with "Divisão por zero";
      end if;
      return A / B;
   end Dividir;

end Calculadora;

Em Ada, todo bloco deve ser fechado com um identificador explícito: end Calculadora; em vez de apenas end;. Isso parece verboso, mas elimina ambiguidades em programas grandes e facilita revisões de código sem IDE.

Onde Ada é Usado Hoje

Domínios Críticos

  • Aviação: sistemas de controle de voo do A380, B777, F-22, F-35.
  • Espacial: Ariane 5/6, sistemas do Orion (NASA), James Webb Space Telescope.
  • Ferroviário: sistemas de controle ETCS/ERTMS em trens de alta velocidade europeus.
  • Defesa: mísseis Patriot, sistemas de radar, submarinos nucleares (US Navy).
  • Nuclear: sistemas de controle de reatores sob padrões IEC 61513.
  • Médico: dispositivos de radioterapia, pacemakers certificados FDA.

Onde Raramente se Usa

  • Aplicações web de propósito geral (onde Python, Go ou JavaScript dominam).
  • Scripts de automação rápida (shell script, Python).
  • Startups e prototipagem ágil (curva de aprendizado e verbosidade são barreiras).
  • Jogos comerciais (salvo simuladores militares).

Quando Aprender Ada

Ada não é a linguagem mais popular no mercado geral de TI, mas entender Ada transforma a maneira como você pensa sobre qualidade de software. Todo programador tem a ganhar ao estudar Ada, mesmo que nunca a use profissionalmente.

Sistemas Embarcados Críticos

Se você pretende trabalhar com avionics, ferrovias, defesa ou dispositivos médicos, Ada é frequentemente um requisito. Padrões como DO-178C (aeronáutica) e EN 50128 (ferroviário) recomendam ou exigem Ada/SPARK.

Verificação Formal

Se você quer provar a correção de software com ferramentas como GNATprove (SPARK), Ada oferece o melhor suporte industrial disponível para verificação formal de programas.

Aprender Boas Práticas

Ada força você a ser explícito sobre tipos, escopo e efeitos colaterais. Programadores Ada que migram para outras linguagens tendem a escrever código mais limpo e defensivo.

Projetos Governamentais

Contratos governamentais de defesa nos EUA, Reino Unido, França e Brasil (FAB, MB) frequentemente exigem Ada para componentes de software de missão crítica.

Ada vs C — Uma Comparação Rápida

Ada e C foram criadas na mesma época e são ambas usadas em sistemas embarcados, mas têm filosofias completamente opostas sobre segurança vs. flexibilidade.

c — erro silencioso de overflow
/* C: sem verificação de range — overflow silencioso */
unsigned char x = 255;
x = x + 1;  /* x vira 0 silenciosamente — Undefined Behavior */
ada — exceção em overflow
-- Ada: verificação em tempo de execução
type Byte is range 0 .. 255;
X : Byte := 255;
X := X + 1;  -- lança Constraint_Error em execução (ou provado incorreto pelo SPARK)
