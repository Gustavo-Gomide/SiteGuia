Ada — Concorrência

Concorrência em Ada

Tasks, Protected Objects e Comunicação por Rendezvous

O modelo de concorrência mais seguro e analisável de qualquer linguagem mainstream.

Ada integra concorrência na linguagem, não em uma biblioteca. Isso garante que o modelo de threading seja verificável formalmente e analisável estaticamente. O mecanismo de rendezvous (encontro) permite comunicação segura entre tasks sem memória compartilhada. Os protected objects implementam monitores com políticas de escalonamento definidas para análise de tempo real. Esta é a combinação mais robusta para sistemas embarcados críticos.

1. Tasks — Threads da Linguagem

Uma task Ada é uma unidade de execução concorrente declarada com a mesma sintaxe de qualquer outra entidade Ada. Tasks podem ser declaradas como tipos (para criar múltiplas instâncias) ou como objetos singulares.

ada — task type: múltiplas instâncias
with Ada.Text_IO;           use Ada.Text_IO;
with Ada.Integer_Text_IO;   use Ada.Integer_Text_IO;

procedure Produtor_Consumidor is

   -- Buffer compartilhado com protected object
   protected Buffer is
      entry    Produzir (Item : in  Integer);
      entry    Consumir (Item : out Integer);
   private
      Dado    : Integer := 0;
      Cheio   : Boolean := False;
   end Buffer;

   protected body Buffer is
      entry Produzir (Item : in Integer) when not Cheio is
      begin
         Dado  := Item;
         Cheio := True;
      end Produzir;

      entry Consumir (Item : out Integer) when Cheio is
      begin
         Item  := Dado;
         Cheio := False;
      end Consumir;
   end Buffer;

   -- Task Produtor
   task Produtor;
   task body Produtor is
   begin
      for I in 1 .. 5 loop
         Buffer.Produzir (I);
         Put_Line ("Produzido: " & Integer'Image (I));
      end loop;
   end Produtor;

   -- Task Consumidor
   task Consumidor;
   task body Consumidor is
      Item : Integer;
   begin
      for I in 1 .. 5 loop
         Buffer.Consumir (Item);
         Put_Line ("Consumido: " & Integer'Image (Item));
      end loop;
   end Consumidor;

begin
   null;  -- Tasks iniciam automaticamente ao entrar no escopo
          -- Aguarda término de todas as tasks antes de sair
end Produtor_Consumidor;

2. Prioridades e Escalonamento

Ada define um modelo de escalonamento por prioridade com suporte ao protocolo de teto de prioridade (Priority Ceiling Protocol) para eliminar inversões de prioridade — problema crítico em sistemas de tempo real.

ada — prioridades e priority ceiling protocol
with System;  -- Para prioridades: System.Priority
with Ada.Real_Time; use Ada.Real_Time;

procedure Sistema_RT is

   -- Protected object com teto de prioridade
   protected Sensor_Temp
      with Priority => 20  -- ceiling = prioridade mais alta das tasks que acessam
   is
      function  Ler       return Float;
      procedure Atualizar (V : Float);
   private
      Temp : Float := 0.0;
   end Sensor_Temp;

   protected body Sensor_Temp is
      function  Ler           return Float is (return Temp; end Ler;
      procedure Atualizar (V : Float) is    begin Temp := V; end Atualizar;
   end Sensor_Temp;

   -- Task de alta prioridade (controle)
   task Controle
      with Priority => 20;

   task body Controle is
      Prox : Time := Clock;
   begin
      loop
         Prox := Prox + Milliseconds (10);  -- 100Hz
         delay until Prox;
         -- Ler sensor e atuar
      end loop;
   end Controle;

   -- Task de baixa prioridade (monitoração)
   task Monitor
      with Priority => 5;

   task body Monitor is
      Prox : Time := Clock;
   begin
      loop
         Prox := Prox + Seconds (1);  -- 1Hz
         delay until Prox;
         -- Logar temperatura
      end loop;
   end Monitor;

begin
   null;
end Sistema_RT;

O Priority Ceiling Protocol de Ada garante que uma task de baixa prioridade que detém um protected object nunca bloqueia uma task de alta prioridade por mais tempo que o corpo do objeto protegido. Isso torna o WCET analisável — essencial para especificação DO-178C.

3. Select Assíncrono e Timeouts

ada — timed entry call e delay alternativo
-- Timed entry call: aguarda atendimento com timeout
select
   Servidor.Processar (Pedido);  -- tenta fazer rendezvous
or
   delay 2.0;  -- desiste após 2 segundos
   Put_Line ("Timeout: servidor não respondeu");
end select;

-- Conditional entry call: tenta sem esperar
select
   Buffer.Consumir (Item);
else
   Put_Line ("Buffer vazio — pular esta iteração");
end select;

-- Asynchronous Transfer of Control (ATC)
select
   delay 5.0;  -- abortador: dispara após 5s
then abort
   Calcular_Resultado_Lento;  -- abortado se demorar > 5s
end select;
