Ruby em Mobile

Desenvolvimento Mobile com Ruby

Historico, estrategias atuais e limites tecnicos no uso de Ruby para iOS e Android.

Ruby no mobile e um nicho: ainda util para prototipos e compartilhamento de logica de negocio, mas com menos tracao que stacks nativas e multiplataforma modernas.

RubyMotion e Titanium

RubyMotion permitiu escrever apps iOS/Android em Ruby compilado para nativo. Titanium SDK tambem ofereceu rota multiplataforma em periodos de maior adocao.

Essas abordagens foram importantes historicamente para mostrar viabilidade de Ruby fora do servidor.

Core Ruby + Interface Nativa

Uma estrategia realista e manter regras de negocio em Ruby (ou mruby) e expor por FFI/bridge para camadas nativas Swift/Kotlin, principalmente em bibliotecas compartilhadas de dominio.

core.rb
module Pricing
  def self.calcular(valor, taxa)
    (valor * taxa).round(2)
  end
end

MRuby e Bibliotecas Compartilhadas

Com cross-compilation em MRuby, e possivel produzir componentes reutilizaveis em C/Ruby para apps nativos, desde que a equipe tenha dominio de toolchain e constraints de runtime.

Limitacoes Atuais e Alternativas

Limitacoes recorrentes: menor comunidade mobile, atualizacoes de SDK nativo e ecosistema de plugins menos abrangente. Alternativas frequentes hoje: Flutter, React Native e stacks nativas.

Ruby no mobile faz mais sentido quando reaproveitamento de dominio pesa mais que adocao de ecossistema amplo.
