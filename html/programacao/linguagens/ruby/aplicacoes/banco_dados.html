Ruby em Banco de Dados

Persistencia com Ruby

Modelagem, consultas e performance de acesso a dados em projetos modernos.

ActiveRecord simplifica CRUD e modelagem relacional, mas desempenho real exige dominio de joins, includes, indices e planos de execucao.

ActiveRecord: Migrations, Associations e Validacoes

ActiveRecord oferece schema versionado por migrations, associações como has_many/belongs_to/has_many :through, validacoes e callbacks para invariantes de negocio.

model.rb
class Pedido < ApplicationRecord
  has_many :itens
  belongs_to :cliente
  validates :total, presence: true
end

Query Interface, Scopes e Explain

Use joins, includes, scopes e atributos virtuais com criterio. Verifique SQL gerado e plano com EXPLAIN para validar indices e custo.

Ponto de atencao

Scopes devem ser composiveis e previsiveis. Evite callbacks de consulta com efeitos colaterais.

Sequel, PG, Redis e MongoDB

  • Sequel: alternativa ao ActiveRecord com DSL SQL robusta.
  • pg gem: acesso direto ao PostgreSQL para casos de baixo nivel.
  • redis-rb: cache, filas e estruturas em memoria.
  • Mongoid: ODM para MongoDB em projetos orientados a documentos.
