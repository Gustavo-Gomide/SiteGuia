Ruby: Avancado

Ruby Avancado

Tecnicas para criar codigo extensivel, performatico e expressivo em sistemas reais.

No nivel avancado, Ruby deixa de ser apenas sintaxe elegante e vira uma plataforma de design de linguagens internas, composicao dinamica e concorrencia controlada.

Mixins com Modules

include adiciona metodos de instancia; extend injeta metodos na singleton class; prepend altera a cadeia de lookup antes da classe original.

mixins.rb
module Auditavel
  def salvar
    puts "auditando"
    super
  end
end

class Pedido
  prepend Auditavel
end

Metaprogramacao na Pratica

Recursos como method_missing, define_method, send e respond_to? permitem construir APIs dinâmicas, DSLs e adapters.

Armadilha Comum

Se usar method_missing, implemente respond_to_missing? para manter introspeccao e ferramentas de IDE funcionando corretamente.

Eigenclass e DSLs Internas

A eigenclass (singleton class) permite definir comportamento especifico por objeto. DSLs como ActiveRecord e Rake exploram essa flexibilidade para criar APIs proximas da linguagem natural.

dsl.rb
class Tarefa
  def self.descricao(texto)
    @descricao = texto
  end
end

Tarefa.descricao("Gerar relatorio diario")

Proc vs Lambda vs Block

  • Block: argumento implicito de metodo, nao e objeto por si.
  • Proc: objeto chamavel, aridade flexivel e retorno nao estrito.
  • Lambda: aridade estrita e semantica de retorno similar a metodo.

Essa diferenca e crucial para APIs funcionais seguras em bibliotecas.

Fibers, Lazy Enumerators, GC e Ractors

Fiber permite cooperacao de tarefas leves; Enumerator::Lazy processa fluxos grandes sem materializar tudo em memoria; o GC evoluiu para reduzir pausas; e Ractor oferece paralelismo com isolamento de estado.

Ractors exigem dados compartilhaveis imutaveis ou copiados. Planeje arquitetura para evitar gargalos de comunicacao.
