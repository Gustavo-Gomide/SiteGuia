Assembly — Introdução

Assembly (ou linguagem de montagem) é a representação textual legível do código de máquina. Cada instrução corresponde diretamente a uma operação do processador — sem camadas de abstração. É a linguagem mais próxima do hardware que ainda é escrita por humanos, sendo essencial para desenvolvimento de sistemas operacionais, drivers, firmware, otimizações críticas e engenharia reversa.

Diferente de linguagens de alto nível, em Assembly o programador controla diretamente registradores, memória, flags e o fluxo de execução do processador. Assemblers como NASM, GAS e MASM traduzem o texto assembly em bytecode binário executável.

História e Contexto

Anos 1940–50

Primeiros computadores (ENIAC, UNIVAC) eram programados diretamente em código de máquina binário. Assembly surgiu como mnemônicos para facilitar a escrita — ADD em vez de 00000011.

Anos 1960–70

IBM System/360 padronizou arquiteturas. Assembly era a linguagem principal para sistemas operacionais, BIOS e software de alto desempenho antes de C surgir.

Anos 1980–90

DOS e Windows 3.x usavam extensamente assembly. Jogos como Wolfenstein 3D e Doom tinham kernels de renderização em ASM x86 para máxima performance no 386/486.

Hoje

Compiladores modernos geram assembly excelente, mas ASM ainda é indispensável em: bootloaders, kernels de SO, firmware embarcado, criptografia de baixo nível, SIMD/AVX e engenharia reversa/malware.

Sintaxes: Intel vs AT&T

Assembly x86 tem duas sintaxes principais. Intel (usada por NASM, MASM) é mais legível: dst, src. AT&T (usada por GAS/gcc) inverte a ordem: src, dst com prefixos obrigatorios.

Assembly x86-64 — Intel (NASM) vs AT&T (GAS)
; ===== Sintaxe Intel (NASM) =====
; dst ← src  (destino à esquerda)
mov   rax, 42          ; rax = 42
add   rax, rbx          ; rax = rax + rbx
mov   rax, [rbp - 8]    ; rax = *(rbp - 8)  — desreferencia memória
mov   [rsi], rdx        ; *rsi = rdx
lea   rdi, [rip + msg]  ; rdi = endereço de msg (RIP-relative)

; ===== Sintaxe AT&T (GAS / gcc -S) =====
// src, dst  (origem à esquerda — invertido)  sufixos de tamanho
movq  $42,    %rax       // rax = 42   (q = quadword = 64-bit)
addq  %rbx,   %rax       // rax += rbx
movq  -8(%rbp), %rax     // rax = *(rbp - 8)
movq  %rdx,   (%rsi)     // *rsi = rdx
leaq  msg(%rip), %rdi    // rdi = &msg

; Sufixos AT&T: b=byte(8), w=word(16), l=long(32), q=quad(64)
movb  $0x41, %al          // al = 'A'  (8-bit)
movw  $1000, %cx          // cx = 1000 (16-bit)
movl  $0,    %edi         // edi = 0   (32-bit — zera bits 32..63 de rdi)

Estrutura de um Programa NASM

NASM x86-64 Linux — Hello World com syscall
; Compilar:  nasm -f elf64 hello.asm -o hello.o
;            ld hello.o -o hello
;            ./hello

section .data
    msg     db  "Hello, World!", 10  ; string + newline (LF)
    msg_len equ $ - msg            ; tamanho calculado em tempo de montagem

    numero  dq  42                  ; quadword (64-bit) = 42
    pi      dd  3.14159             ; float 32-bit

section .bss                         ; variáveis não inicializadas
    buffer  resb 64                  ; reserva 64 bytes
    count   resq 1                   ; reserva 1 quadword

section .text
    global _start                   ; ponto de entrada para o linker

_start:
    ; syscall write(1, msg, msg_len)
    ; Linux x86-64: syscall number em RAX, args em RDI, RSI, RDX...
    mov  rax, 1           ; syscall: sys_write
    mov  rdi, 1           ; fd: 1 = stdout
    mov  rsi, msg         ; ponteiro para string
    mov  rdx, msg_len     ; quantidade de bytes
    syscall               ; executa a syscall

    ; syscall exit(0)
    mov  rax, 60          ; syscall: sys_exit
    mov  rdi, 0           ; exit code 0
    syscall
NASM x86-64 — Declaracoes de dados (db/dw/dd/dq)
section .data
    ; Bytes individuais e strings
    byte_val    db  0xFF                    ; define byte
    char_a      db  'A'                     ; caractere ASCII = 65
    texto       db  "Ola", 0               ; string null-terminated
    array_bytes db  1, 2, 3, 4, 5         ; array de 5 bytes

    ; Word (16-bit), Dword (32-bit), Qword (64-bit)
    w           dw  1000                   ; 2 bytes
    d           dd  100000                 ; 4 bytes
    q           dq  9999999999             ; 8 bytes

    ; Constantes com EQU (sem alocação de memória)
    STDIN       equ 0
    STDOUT      equ 1
    SYS_WRITE   equ 1
    SYS_EXIT    equ 60

section .bss
    ; resb/resw/resd/resq N — reservar N unidades sem inicializar
    buf64   resb 64     ; 64 bytes
    matrix  resq 16     ; 16 quadwords = 128 bytes (ex: matriz 4x4 float64)

Importante: Ao escrever em um registrador de 32-bit (ex: EAX), os bits 32–63 de RAX são zerados automaticamente em x86-64. Mas ao escrever em 8-bit ou 16-bit, os bits superiores não são alterados — o que pode gerar bugs sutis.

Ferramentas e Primeiros Passos

Shell — Instalacao e compilacao com NASM no Linux
# Instalar NASM
sudo apt install nasm build-essential

# Montar (assemblar) para ELF64
nasm -f elf64 hello.asm -o hello.o

# Linkar com ld (sem libc)
ld hello.o -o hello

# Executar
./hello

# Montar com listagem (útil para aprender)
nasm -f elf64 -l hello.lst hello.asm -o hello.o
cat hello.lst   # mostra endereços, bytes e código

# Inspecionar o binário
objdump -d hello                  # disassembly
objdump -s -j .data hello         # conteúdo da seção .data
readelf -h hello                  # cabeçalho ELF

# Compilar com GCC (usa GAS / AT&T por padrão)
gcc -no-pie -o hello_c hello.c

# Ver assembly gerado pelo GCC (sintaxe Intel)
gcc -S -masm=intel -O2 hello.c -o hello.s
cat hello.s
