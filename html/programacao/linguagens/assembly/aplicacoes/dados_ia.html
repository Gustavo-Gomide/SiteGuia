Assembly — Dados e IA

O núcleo computacional de frameworks de IA como NumPy, PyTorch e TensorFlow é implementado em C/C++ com rotinas SIMD otimizadas para x86 e ARM. Operações de álgebra linear — multiplicação de matrizes (GEMM), dot products, ativações ReLU — exploram ao máximo AVX2, AVX-512 e VNNI para atingir TFLOPS de throughput em CPUs convencionais sem GPU.

NumPy usa OpenBLAS ou MKL como backend de BLAS — ambos implementados em assembly otimizado para cada microarquitetura. Quando você chama np.dot(a, b) em Python, estão sendo executadas dezenas de kilobytes de código assembly AVX-512 escritos à mão.

Matrix Multiplication (GEMM) com SIMD

NASM x86-64 — Micro-kernel GEMM com AVX2 (block 8x1)
; GEMM micro-kernel: C[m][n] += A[m][k] * B[k][n]
; Estratégia: processar 8 elementos de C de uma vez com AVX2 YMM (256-bit = 8 floats)
;             para cada linha de A × coluna de B (inner product)
;
; rdi = C (saída), rsi = A, rdx = B
; rcx = m (linhas A/C), r8 = n (colunas B/C), r9 = k (colunas A = linhas B)

section .text

; gemm_8x1: kernel de uma linha de C = 8 floats no mesmo índice da coluna B
gemm_kernel_8x1:
    ; Parâmetros: rdi=C_ptr, rsi=A_row_ptr, rdx=B_col_ptr, rcx=k
    vxorps   ymm0, ymm0, ymm0   ; acumulador = 0
    test     rcx, rcx
    jz       .done
.kloop:
    vbroadcastss ymm1, dword [rsi]  ; broadcast A[k] para todos os 8 slots
    vmovups  ymm2, [rdx]              ; carregar 8 floats de B[k][0..7]
    vfmadd231ps ymm0, ymm1, ymm2     ; acc += A[k] * B[k][0..7]  (FMA)
    add      rsi, 4                  ; avançar A para próximo elemento k
    add      rdx, 32                 ; avançar B para próxima linha (8 floats)
    dec      rcx
    jnz      .kloop
.done:
    ; Acumular em C (C += resultado)
    vmovups  ymm3, [rdi]
    vaddps   ymm0, ymm0, ymm3
    vmovups  [rdi], ymm0
    vzeroupper
    ret

; DGEMM (double precision) — 4 doubles por YMM
gemm_kernel_4x1_double:
    vxorpd   ymm0, ymm0, ymm0
.loop:
    vbroadcastsd ymm1, qword [rsi]
    vmovupd  ymm2, [rdx]
    vfmadd231pd ymm0, ymm1, ymm2
    add      rsi, 8
    add      rdx, 32
    dec      rcx
    jnz      .loop
    vmovupd  ymm3, [rdi]
    vaddpd   ymm0, ymm0, ymm3
    vmovupd  [rdi], ymm0
    vzeroupper
    ret

ReLU, Sigmoid e Softmax Vetorizados

NASM x86-64 — Funcoes de ativacao com AVX2
; ── ReLU: f(x) = max(x, 0) ────────────────────────────────
; rdi = array float, rsi = n elementos
relu_avx:
    vxorps  ymm7, ymm7, ymm7    ; ymm7 = 0.0 em 8 raias
    test    rsi, rsi
    jz      .fim
.loop:
    vmovups ymm0, [rdi]
    vmaxps  ymm0, ymm0, ymm7    ; max(x, 0) para 8 floats
    vmovups [rdi], ymm0
    add     rdi, 32
    sub     rsi, 8
    jnz     .loop
.fim:
    vzeroupper
    ret

; ── Leaky ReLU: f(x) = x >= 0 ? x : alpha*x  (alpha=0.01) ─
section .rodata
    align 32
    leaky_alpha dd 0.01, 0.01, 0.01, 0.01, 0.01, 0.01, 0.01, 0.01

section .text
leaky_relu_avx:
    vmovaps  ymm6, [leaky_alpha]
    vxorps   ymm7, ymm7, ymm7
.loop:
    vmovups  ymm0, [rdi]
    vmulps   ymm1, ymm0, ymm6      ; ymm1 = x * 0.01 (lado negativo)
    vcmpps   ymm2, ymm0, ymm7, 13  ; ymm2 = máscara: x >= 0 → 0xFFFFFFFF
    vblendvps ymm0, ymm1, ymm0, ymm2 ; blend: x>=0 → x, senão alpha*x
    vmovups  [rdi], ymm0
    add      rdi, 32
    sub      rsi, 8
    jnz      .loop
    vzeroupper
    ret

; ── Softmax numérico estável ─────────────────────────────
; Implementado em C com intrinsics (mais prático que .asm direto):
;   1. max = horizontal max de todos os elementos
;   2. x[i] = exp(x[i] - max)   (estabilidade numérica)
;   3. sum = horizontal sum
;   4. x[i] /= sum
; A função exp() em SIMD usa a aproximação: e^x ≈ 2^(x/ln2)
; via frexp, ldexp e tabela de lookup de 2^frac(x)

Quantizacao INT8 e VNNI

NASM x86-64 — Convolucao quantizada INT8 com VNNI (AVX-512)
; AVX-512 VNNI (Vector Neural Network Instructions) — disponível desde Ice Lake
; vpdpbusd: dot product de uint8 × int8 → int32 (4 elementos por iteração)
; Processa 4 operações de 8-bit multiply-add de uma vez em 32 bits de saída

; Quantização INT8: converter float32 → int8
; scale = 127.0 / max_abs_value
; quantized = clamp(round(x * scale), -128, 127)

quantize_int8_avx2:
    ; rdi = float32* input, rsi = int8* output, rdx = n, xmm0 = scale
    vbroadcastss  ymm7, xmm0       ; broadcast scale para 8 raias
.loop:
    vmovups   ymm0, [rdi]           ; 8 float32
    vmulps    ymm0, ymm0, ymm7      ; x * scale
    vcvtps2dq ymm0, ymm0            ; float → int32 (com arredondamento)
    vpackssdw ymm0, ymm0, ymm0     ; int32 → int16 saturado
    vpacksswb xmm0, xmm0, xmm0    ; int16 → int8 saturado
    movq      [rsi], xmm0           ; salvar 8 int8
    add       rdi, 32
    add       rsi, 8
    sub       rdx, 8
    jnz       .loop
    vzeroupper
    ret

; AVX-512 VNNI — dot product int8: zmm = 16×(a[u8] * b[i8]) acumulado em int32
; Esta instrução processa 16 pares uint8×int8 e acumula em 4 int32:
;   vpdpbusd zmm_acc, zmm_a, zmm_b
; Performance: ~4× mais throughput que SIMD float32 para redes quantizadas INT8

; OBS: VNNI requer CPU Intel Ice Lake ou posterior (ou AMD Zen4+)
; Verificar suporte: CPUID leaf 7, ECX bit 11 = AVXVNNI

VFMADD231PS (Fused Multiply-Add) faz em uma instrução: dst += src1 * src2, com apenas um arredondamento IEEE 754 vs dois separados. Para GPT-style attention (QKV dot products) em CPUs, usar FMA em kernel GEMM bem bloqueado pode atingir 70–85% do pico teórico de FLOPS da CPU — o mesmo que frameworks como OpenBLAS e MKL fazem internamente.
