Clojure — Introdução

Clojure

Clojure é um dialeto Lisp moderno e dinâmico criado por Rich Hickey em 2007, projetado para rodar na JVM (Java Virtual Machine). Combina o poder expressivo do Lisp com imutabilidade por padrão, estruturas de dados persistentes e suporte nativo a concorrência — tudo em um design orientado à simplicidade e praticidade industrial.

Clojure não é uma linguagem acadêmica: é usada em produção por Nubank (o maior banco digital do mundo), Walmart Labs, Puppet e Circle CI. Sua filosofia central é "Simple Made Easy" — distinguir o que é simples (sem entrelaçamento) do que é fácil (familiar). O código simples escala; o código fácil nem sempre.

Clojure — Hello World e primeiros conceitos
; Tudo em Clojure é uma expressão
(ns hello.core)   ; declarar namespace

(println "Hello, World!")

; Definir uma função
(defn saudar [nome]
  (str "Olá, " nome "!"))

(saudar "Clojure")   ; => "Olá, Clojure!"

; Estruturas de dados imutáveis
(def lista [1 2 3 4 5])
(def mapa  {:nome "Maria" :idade 30})

(map inc lista)          ; => (2 3 4 5 6)
(filter even? lista)     ; => (2 4)
(reduce + lista)         ; => 15

; Threading macro — leitura esquerda → direita
(->> lista
     (filter odd?)
     (map #(* % %))
     (reduce +))            ; => 1+9+25 = 35

Características Principais

Imutabilidade por Padrão

Toda estrutura de dados em Clojure é imutável. conj, assoc, dissoc retornam novas versões — persistência estrutural garante que dados antigos não são copiados, apenas compartilhados.

Lisp Homoiconico

Código é dado — a AST é representada como listas Clojure. Macros manipulam código antes de compilação, permitindo criar DSLs (Domain-Specific Languages) com poder total da linguagem.

STM e Concorrência

Software Transactional Memory nativa: ref + dosync para transações coordenadas; atom para estado independente; agent para ações assíncronas; var para estado thread-local.

Interoperabilidade JVM

Acesso direto a toda a JVM: (.method objeto args), (new Classe args). Todas as bibliotecas Java são acessíveis diretamente — JDBC, Hadoop, Kafka, Spring sem wrappers necessários.

REPL Interativo

O REPL (Read-Eval-Print Loop) é o coração do desenvolvimento Clojure. Conecta diretamente ao processo em produção via nREPL — alterações ao vivo sem reiniciar o serviço.

ClojureScript

O mesmo código (com ajustes mínimos) roda no browser via ClojureScript compilado para JavaScript. Frameworks como Re-frame e Reagent (wrapper de React) são usados em produção.

Quando Usar Clojure

Cenários Ideais
  • Sistemas de processamento de dados / ETL em grande escala
  • APIs e microsserviços que requerem alta concorrência
  • Aplicações onde imutabilidade evita bugs de estado compartilhado
  • Times que valorizam código expressivo e testável
  • Quando já existe infraestrutura JVM (aproveitar JARs Java)
  • Projetos de data science com Clojure + Apache Arrow / Spark
Quando Evitar
  • Times sem experiência com programação funcional — curva de aprendizado alta
  • Sistemas embarcados ou baixo nível (JVM tem overhead)
  • Projetos onde parênteses/S-expressions causam resistência cultural
  • Quando é necessário máximo desempenho CPU (preferir Rust/C++)
  • Recrutamento difícil — pool de desenvolvedores Clojure é pequeno
