Clojure — Fundamentos

A sintaxe do Clojure é radicalmente simples: tudo são S-expressions — listas entre parênteses onde o primeiro elemento é o operador. Não há palavras reservadas para aritmetíca, não há precedência de operadores, não há sintaxe especial para chamada de função vs. operador — apenas listas aninhadas.

Dominar os fundamentos do Clojure significa entender suas coleções imutáveis (vetor, mapa, set, lista), a diferença entre def/defn/let, como usar if/cond/case, e como a recursão com recur substitui loops imperativos.

Todos os tipos literais do Clojure
; ── Escalares ────────────────────────────────────────────────

Funções — defn, fn, #()

Definição e chamada de funções
; ── defn — definição de função nomeada ───────────────────────
(defn soma
  "Soma dois números. Docstring é o segundo argumento."
  [a b]
  (+ a b))

(soma 3 4)   ; => 7

; ── Aridad múltipla ──────────────────────────────────────────
(defn saudar
  ([nome]        (saudar nome "Olá"))
  ([nome msg]    (str msg ", " nome "!")))

(saudar "Ana")            ; => "Olá, Ana!"
(saudar "Ana" "Oi")       ; => "Oi, Ana!"

; ── Variadic com & ───────────────────────────────────────────
(defn log [nivel & msgs]
  (apply println (str "[" nivel "]") msgs))

(log "INFO" "servidor" "iniciado")  ; [INFO] servidor iniciado

; ── fn anônimo ───────────────────────────────────────────────
(fn [x] (* x x))
#(* % %)   ; shorthand — % é o primeiro arg; %2 o segundo

(map #(* % %) [1 2 3 4])  ; => (1 4 9 16)

let e Bindings Locais

let, let*, if-let, when-let
; let — bindings locais em pares [nome valor]
(let [x 10
      y 20
      soma (+ x y)]
  (println "soma =" soma)
  (* soma 2))   ; => 60 (último expr é o retorno)

; if-let — bind e testa se truthy
(if-let [usuario (buscar-usuario 42)]
  (println "Bem-vindo," (:nome usuario))
  (println "Usuário não encontrado"))

; when-let — bind e executa só se truthy
(when-let [resultado (calcular)]
  (salvar resultado)
  (notificar resultado))

; destructuring em let — extrair de vetores e mapas
(let [[primeiro segundo & resto] [1 2 3 4 5]]
  primeiro)   ; => 1

(let [{:keys [nome idade cidade]} {:nome "Carlos"
                                    :idade 25
                                    :cidade "SP"}]
  (str nome " tem " idade " anos e mora em " cidade))

Condicionais e Fluxo de Controle

if, cond, case, when
; if — ÚNICA expressão em cada branch
(if (> x 0)
  "positivo"
  "não positivo")

; when — if sem else; múltiplos forms no body (retorna nil se falso)
(when (valid? dados)
  (processar dados)
  (salvar dados))

; cond — múltiplos branches
(cond
  (< x 0)  "negativo"
  (= x 0)  "zero"
  (> x 0)  "positivo"
  :else      "impossível")    ; :else é sempre truthy

; case — match por valor (mais eficiente que cond)
(case dia-semana
  :seg "Segunda"
  :ter "Terça"
  :qua "Quarta"
  "Outro dia")   ; default (sem test)

; and / or — curto-circuito, retornam o valor
(and true 42)     ; => 42 (retorna último truthy)
(or  nil false 7) ; => 7 (retorna primeiro truthy)

Coleções — Operações Essenciais

Vetor, mapa, set, lista — CRUD imutável
; ── Vetor ────────────────────────────────────────────────────
(def v [10 20 30])
(conj v 40)       ; => [10 20 30 40] (append)
(get v 1)         ; => 20
(v 1)             ; => 20 (vetor é função de seu índice)
(assoc v 1 99)   ; => [10 99 30]
(count v)         ; => 3
(subvec v 1 3)   ; => [20 30]

; ── Mapa ─────────────────────────────────────────────────────
(def m {:nome "Ana" :idade 25})
(assoc m :email "ana@ex.com")   ; adicionar chave
(dissoc m :idade)               ; remover chave
(update m :idade inc)           ; => {:nome "Ana" :idade 26}
(merge m {:cidade "SP"})        ; unir mapas
(keys m)                         ; => (:nome :idade)
(vals m)                         ; => ("Ana" 25)

; ── Set ──────────────────────────────────────────────────────
(def s #{1 2 3})
(conj s 4)               ; => #{1 2 3 4}
(disj s 2)               ; => #{1 3}
(contains? s 2)          ; => true
(clojure.set/union s #{3 4})       ; => #{1 2 3 4}
(clojure.set/intersection s #{2 3}) ; => #{2 3}

Loop e Recur — Recursão Otimizada

loop/recur, doseq, dotimes, for
; recur — tail-call optimization (sem stack overflow)
(defn fatorial [n]
  (loop [n n, acc 1]
    (if (<= n 1)
      acc
      (recur (dec n) (* acc n)))))

(fatorial 10)   ; => 3628800

; doseq — side effects em coleção
(doseq [item ["a" "b" "c"]]
  (println "Item:" item))

; dotimes — repetir N vezes
(dotimes [i 5]
  (println "i =" i))

; for — list comprehension
(for [x [1 2 3]
      y [1 2]
      :when (not= x y)]
  [x y])
; => ([1 2] [2 1] [3 1] [3 2])

Interoperabilidade Java

Chamada de métodos e classes Java
; Criar instância
(new java.util.ArrayList)
(java.util.ArrayList.)   ; shorthand — ponto no final = construtor

; Chamar método de instância — (.metodo objeto args)
(def lista-java (java.util.ArrayList.))
(.add lista-java "item1")
(.add lista-java "item2")
(.size lista-java)         ; => 2
(.get lista-java 0)        ; => "item1"

; Método estático — (Classe/metodo args)
(java.lang.Math/sqrt 16.0)   ; => 4.0
(java.lang.Math/PI)          ; => 3.141592...
(System/currentTimeMillis)   ; timestamp

; String Java
(.toUpperCase "hello")          ; => "HELLO"
(.contains "hello world" "world") ; => true
(str "a" "b" "c")             ; => "abc" (Clojure native)

; Import de classes Java
(import 'java.util.Date 'java.io.File)
(Date.)    ; data atual
