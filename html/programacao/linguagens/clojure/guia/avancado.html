Clojure — Avançado

O nível avançado de Clojure revela os seus superpoderes: macros homoiconicas que transformam código em tempo de compilação, o modelo de concorrência baseado em identidade/valor, transducers para transformação sem alocação intermediária, e core.async para programação assíncrona com canais.

A frase de Rich Hickey resume o modelo de concorrência do Clojure: "State is a value at a time. An identity is an entity with a succession of values over time." — atom, ref e agent modelam diferentes políticas de atualização de identidade.

Macros — Metaprogramação Homoicônica

defmacro — criar novas formas sintáticas
; ── defmacro — código é dado, dado é código ──────────────────
(defmacro unless
  "Executa body apenas se cond for falso."
  [cond & body]
  `(when (not ~cond) ~@body))      ; ` = syntax-quote ; ~ = unquote

(unless (empty? lista)
  (println "lista tem elementos"))

; ── macroexpand — ver o código gerado ────────────────────────
(macroexpand '(unless false (println "ok")))
; => (clojure.core/when (clojure.core/not false) (println "ok"))

; ── Macro com gensym para evitar symbol capture ──────────────
(defmacro bench [expr]
  `(let [start# (System/nanoTime)
          result# ~expr
          elapsed# (- (System/nanoTime) start#)]
     (println "Tempo:" (/ elapsed# 1e6) "ms")
     result#))    ; start# gera nome único — sem conflito

(bench (reduce + (range 1000000)))

; ── Threading macros — composição legível ────────────────────
(; ->  threading: insere resultado como PRIMEIRO argumento)
(-> dados
    (assoc :ativo true)
    (update :nome clojure.string/upper-case)
    (dissoc :senha))

; ->> threading: insere resultado como ÚLTIMO argumento
(->> (range 100)
     (filter even?)
     (map #(* % %))
     (take 5)
     (reduce +))   ; => 0+4+16+36+64 = 120

Concorrência — atom, ref, agent

atom, ref/dosync e agent — três modelos de estado
; ── atom — estado independente, swap! é atômico ─────────────
(def contador (atom 0))

(swap! contador inc)         ; CAS — compare-and-swap seguro
(swap! contador + 10)       ; => aplica (+ @contador 10)
(reset! contador 0)         ; forçar valor
@contador                    ; deref — ler valor atual

; Múltiplas threads seguras via atom
(def cache (atom {}))
(defn buscar-com-cache [id]
  (or (get @cache id)
      (let [resultado (buscar-banco id)]
        (swap! cache assoc id resultado)
        resultado)))

; ── ref — transações coordenadas (STM) ───────────────────────
(def conta-a (ref 1000))
(def conta-b (ref 500))

(defn transferir [de para valor]
  (dosync                      ; transação atômica
    (alter de    - valor)
    (alter para  + valor)))    ; ambos ou nenhum

(transferir conta-a conta-b 200)
@conta-a  ; => 800
@conta-b  ; => 700

; ── agent — estado assíncrono ─────────────────────────────────
(def log-agent (agent []))
(send log-agent conj {:tipo :info :msg "servidor iniciado"})
(send log-agent conj {:tipo :info :msg "conexão aceita"})
(await log-agent)   ; bloquear até agente processar
@log-agent

Transducers — Transformação Sem Alocação

Transducers — composição eficiente de transformações
; Transducer = transformação sem contexto de coleção
; Sem xf: cada etapa cria uma coleção intermediária
(->> dados
     (filter pred?)        ; aloca nova seq
     (map transform)       ; aloca nova seq
     (take 10))            ; aloca nova seq

; Com xf: zero alocações intermediárias
(def xf
  (comp
    (filter pred?)        ; retorna transducer
    (map transform)       ; retorna transducer
    (take 10)))

; Aplicar ao contexto que quiser
(transduce xf conj [] dados)      ; => vector
(transduce xf + 0 dados)          ; => soma
(sequence xf dados)               ; => lazy seq
(into [] xf dados)                ; => vector eficiente

; Channel com transducer (core.async)
(chan 10 xf)   ; canal que aplica xf a cada valor

Protocolos e Multimethods — Polimorfismo

defprotocol, defrecord, defmulti/defmethod
; ── Protocol — interface abstrata ────────────────────────────
(defprotocol Serializable
  (to-json [this])
  (to-edn  [this]))

; ── defrecord — estrutura imutável com protocol ──────────────
(defrecord Usuario [id nome email]
  Serializable
  (to-json [this]
    (str "{\"id\":" id ",\"nome\":\"" nome "\",\"email\":\"" email "\"}"))
  (to-edn [this]
    (pr-str (into {} this))))

(def u (Usuario. 1 "Maria" "maria@ex.com"))
(to-json u)   ; => "{\"id\":1,\"nome\":\"Maria\",...}"

; ── defmulti — dispatch por função arbitrária ────────────────
(defmulti  area :tipo)   ; dispatch pela chave :tipo
(defmethod area :circulo  [{:keys [raio]}]  (* Math/PI raio raio))
(defmethod area :retangulo [{:keys [l a]}]   (* l a))
(defmethod area :default   [_]  0)

(area {:tipo :circulo   :raio 5})      ; => 78.539...
(area {:tipo :retangulo :l 4 :a 3})   ; => 12

core.async — Canais e Go Blocks

core.async — CSP (Communicating Sequential Processes)
(require '[clojure.core.async :as async :refer [chan go <! >! close!]])

; ── Canal simples ─────────────────────────────────────────────
(def ch (chan 10))   ; canal com buffer de 10

; go block — goroutine leve
(go
  (>! ch "mensagem 1")   ; park se canal cheio
  (>! ch "mensagem 2"))

(go
  (let [msg (<! ch)]    ; park se canal vazio
    (println "Recebido:" msg)))

; ── pipeline — processar com paralelismo controlado ───────────
(async/pipeline
  4                           ; 4 workers paralelos
  saida-ch
  (map processar)
  entrada-ch)

; ── timeout e select ─────────────────────────────────────────
(go
  (let [[v port] (async/alts! [ch (async/timeout 1000)])]
    (if (= port ch)
      (println "Recebido:" v)
      (println "Timeout após 1s"))))

Transducers resolvem um problema real de performance: um pipeline (->> dados (filter f) (map g) (take 10)) cria 3 coleções intermediárias. Com transducers — (comp (filter f) (map g) (take 10)) — o pipeline é executado em uma única passagem, com uma única alocação de resultado. Para processar arquivos de 1GB linha a linha, a diferença é entre caber na memória e não caber.
