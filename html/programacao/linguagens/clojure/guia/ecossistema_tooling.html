Clojure — Ecossistema e Tooling

O ecossistema Clojure é construído em torno do REPL (Read-Eval-Print Loop) como ferramenta central de desenvolvimento. Leiningen e a Clojure CLI gerem dependências e builds; CIDER e Calva integram o REPL ao editor; Kaocha executa testes especificados via deps.edn.

O desenvolvimento Clojure é orientado a REPL: você nunca para o servidor para reiniciar — você recarrega namespaces dentro do processo ao vivo. Todo o fluxo de trabalho gira em torno dessa premissa.

Leiningen

Instalar e Comandos Essenciais
# ── Instalação ───────────────────────────────────────────────
# Linux / macOS
curl -o lein https://raw.githubusercontent.com/technomancy/leiningen/stable/bin/lein
chmod +x lein
sudo mv lein /usr/local/bin/

# Windows — baixar lein.bat de leiningen.org e adicionar ao PATH

# ── Comandos comuns ───────────────────────────────────────────
lein new app  meu-projeto     # cria projeto com template "app"
lein new lib  minha-lib       # cria biblioteca

lein repl                     # inicia REPL nREPL na porta 7888
lein run                      # executa -main
lein test                     # roda todos os testes
lein test :only meu.ns/teste  # roda teste específico

lein uberjar                  # JAR standalone com deps incluídas
java -jar target/meu-projeto-0.1.0-standalone.jar

lein deps                     # baixar dependências
lein deps :tree               # árvore de dependências
lein clean                    # limpar target/

Clojure CLI e deps.edn

Instalação e uso da Clojure CLI moderna
# ── Instalação ───────────────────────────────────────────────
# Linux
curl -L -O https://github.com/clojure/brew-install/releases/latest/download/linux-install.sh
chmod +x linux-install.sh
sudo ./linux-install.sh

# macOS
brew install clojure/tools/clojure

# ── Comandos ─────────────────────────────────────────────────
clj                            # REPL interativo
clj -M -m meu.ns               # executar namespace com -main
clj -X meu.ns/funcao           # executar função de topo
clj -T:build uber              # build com tools.build
clj -A:dev                     # activar alias :dev

REPL — Desenvolvimento Interativo

Fluxo de trabalho orientado a REPL
; ── Recarregar namespace sem reiniciar ───────────────────────
(require '[meu.ns :as ns] :reload)
(require '[meu.ns :as ns] :reload-all)   ; deps também

; ── Introspection ────────────────────────────────────────────
(doc map)              ; docstring + aridade
(source map)           ; código-fonte
(dir clojure.string)   ; listar todas as funções do ns
(apropos "sort")        ; funções cujo nome contém "sort"

; ── ns-changer ───────────────────────────────────────────────
(in-ns 'meu.app.core)   ; mudar de namespace

; ── Depuração no REPL ─────────────────────────────────────────
(tap> valor)    ; enviar para tap handler (ex: portal)
(add-tap prn)   ; handler que printa
(remove-tap prn)

; ── tools.namespace — hot reload no dev ──────────────────────
(require '[clojure.tools.namespace.repl :refer [refresh]])
(refresh)   ; recarregar todos os ns modificados

Testes — clojure.test e Kaocha

clojure.test — framework padrão
(ns meu-app.calculadora-test
  (:require [clojure.test :refer [deftest testing is are]]
            [meu-app.calculadora :as calc]))

(deftest soma-test
  (testing "soma inteiros"
    (is (= 4 (calc/soma 2 2)))
    (is (= 0 (calc/soma -1 1))))
  (testing "soma com zero"
    (is (= 5 (calc/soma 5 0)))))

; are — múltiplos casos numa tabela
(deftest soma-tabela
  (are [a b esperado] (= esperado (calc/soma a b))
    1  1   2
    0  5   5
   -3  3   0))

; fixture — setup/teardown
(defn carregar-db [f]
  (setup-test-db!)
  (f)
  (teardown-db!))

(use-fixtures :each carregar-db)
kaocha — runner moderno com relatórios ricos
; tests.edn — declaração dos suites
#kaocha/v1
{:tests [{:id          :unit
          :test-paths  ["test"]
          :ns-patterns [".*-test$"]}
         {:id         :integration
          :test-paths ["integration-test"]}]
 :reporter kaocha.report/documentation
 :fail-fast? false}

# Executar
clj -M:test           # todos os suites
clj -M:test --suite unit
clj -M:test --watch   # modo watch — re-executa ao salvar

O segredo do workflow Clojure: nunca feche o REPL. Use clj -M:dev para iniciar um nREPL com CIDER, conecte seu editor, e passe o dia inteiro avaliando formas dentro do processo ao vivo. refresh cuida do hot-reload dos namespaces — você escreve código, avalia, vê o resultado, itera. SEM reiniciar o servidor.
