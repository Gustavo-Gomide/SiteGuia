Clojure: Aplicacoes Mobile

Mobile com ClojureScript e arquitetura orientada a eventos

O caminho mais comum para mobile com Clojure e ClojureScript + React Native, aproveitando ecossistema JavaScript sem perder modelagem funcional.

Com re-frame, cada acao do usuario vira evento; o estado global da app evolui de forma rastreavel, o que facilita debug e testes de fluxo.

Conceitos tecnicos

No mobile, restricoes de bateria, rede intermitente e memoria limitada exigem arquitetura previsivel. ClojureScript ajuda ao tratar estado como dados e padronizar transicoes.

  • Unidirectional data flow para reduzir efeitos colaterais.
  • Persistencia local para modo offline-first.
  • Sincronizacao eventual com backend por fila de operacoes.

Pipeline de eventos em re-frame

Esse fluxo deixa claro onde cada regra mora: evento dispara intencao, handler muda estado, subscriptions derivam dados para render.

Exemplo minimo em ClojureScript

(ns app.mobile.core
  (:require
    [re-frame.core :as rf]
    [reagent.core :as r]))

(rf/reg-event-db
  :init
  (fn [_ _]
    {:contador 0 :sync/status :idle}))

(rf/reg-event-db
  :inc
  (fn [db _]
    (update db :contador inc)))

(rf/reg-sub
  :contador
  (fn [db _] (:contador db)))

(defn main-screen []
  (let [c @(rf/subscribe [:contador])]
    [:<>
     [:Text (str "Valor: " c)]
     [:Button {:title "Incrementar"
               :on-press #(rf/dispatch [:inc])}]]))

Mesmo em app simples, ja e possivel testar handlers e subscriptions sem renderizar UI, acelerando feedback de qualidade.

Padroes avancados mobile

Offline-first e reconciliacao

Grave operacoes localmente com identificador idempotente. Ao reconectar, reenvie em ordem, trate conflitos por versao e atualize app-db via eventos de reconciliacao.

Nao acople componente de tela a chamadas HTTP diretas. Centralize efeitos em handlers para manter rastreabilidade e reduzir regressao.
