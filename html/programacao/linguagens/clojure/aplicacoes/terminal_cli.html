Clojure: Aplicacoes Terminal e CLI

Ferramentas de linha de comando com composicao funcional

CLI e uma das areas onde Clojure entrega alto retorno: parsing, transformacao de dados e automacao com feedback rapido via REPL ou scripts.

Entre Clojure JVM e babashka, voce pode escolher entre ecossistema completo ou startup ultrarrapido para scripts operacionais.

Conceitos de CLI bem projetada

Uma boa CLI separa interface de comando da logica de dominio. O parser interpreta argumentos, o dominio executa a regra e o formatter serializa saida (texto, JSON, EDN).

  • Comandos composicionais e previsiveis.
  • Codigos de saida claros para automacao em CI/CD.
  • Mensagens de erro acionaveis para o operador.

Fluxo de execucao de comando

Esse encadeamento facilita testes de contrato da CLI: dado um argv, espera-se saida e codigo de retorno deterministico.

Exemplo com clojure.tools.cli

(ns app.cli
  (:require
    [clojure.tools.cli :refer [parse-opts]]
    [clojure.string :as str]))

(def cli-options
  [["-n" "--name NAME" "Nome para saudacao"]
   ["-j" "--json" "Saida em JSON"]
   ["-h" "--help"]])

(defn run [{:keys [name json]}]
  (let [msg (str "Ola, " (or name "mundo") "!")]
    (if json
      (str "{\"message\":\"" msg "\"}")
      msg)))

(defn -main [& args]
  (let [{:keys [options errors summary]} (parse-opts args cli-options)]
    (cond
      (:help options) (do (println summary) (System/exit 0))
      (seq errors)    (do (doseq [e errors] (binding [*out* *err*] (println e))) (System/exit 2))
      :else           (do (println (run options)) (System/exit 0)))))

O retorno de System/exit e essencial para pipelines shell e jobs de automacao.

Quando usar babashka

Babashka executa scripts Clojure com startup muito rapido, ideal para tarefas de automacao e CLIs menores onde cold start da JVM seria um gargalo.

#!/usr/bin/env bb
(require '[babashka.fs :as fs])

(doseq [f (fs/glob "." "*.log")]
  (println "Arquivo:" f))

Padroes avancados para CLIs grandes

Subcomandos e plugin system

Modele comandos como mapa de handlers e metadata. Isso permite carregar subcomandos por namespace e habilitar extensao sem alterar o nucleo.

Cuidado com side effects no parse de args. Parsing deve ser puro; efeitos ficam no executor do comando para manter testes simples.
