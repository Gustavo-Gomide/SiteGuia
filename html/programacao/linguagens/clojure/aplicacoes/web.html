Clojure — Desenvolvimento Web

Clojure tem um ecossistema web maduro: Ring define a abstração HTTP como dados simples; Compojure e Reitit roteiam requisições; Hiccup gera HTML com vetores Clojure; e ClojureScript compila Clojure para JavaScript, permitindo usar Reagent (React) no front-end com o mesmo idioma.

A filosofia Ring: uma requisição HTTP é um mapa Clojure, um handler é uma função {:request} → {:response}, e um middleware é uma função que envolve um handler. Sem frameworks mágicos — só dados e funções.

Ring Handler e servidor Jetty
(ns meu-app.core
  (:require [ring.adapter.jetty :as jetty]
            [ring.middleware.params  :refer [wrap-params]]
            [ring.middleware.json    :refer [wrap-json-body
                                             wrap-json-response]]))

; Handler = função request → response
(defn handler [req]
  {:status  200
   :headers {"Content-Type" "application/json"}
   :body    "{\"ok\":true}"})

; Middleware = função que envolve um handler
(def app
  (-> handler
      (wrap-json-body   {:keywords? true})
      (wrap-json-response)
      (wrap-params)))

; Iniciar servidor
(defn -main [& _]
  (jetty/run-jetty app {:port 3000 :join? false})
  (println "Servidor em http://localhost:3000"))

Compojure — Roteamento

Definindo rotas com Compojure
(ns meu-app.routes
  (:require [compojure.core  :refer [defroutes GET POST PUT DELETE]]
            [compojure.route :refer [not-found]]
            [ring.util.response :refer [response]]
            [cheshire.core :as json]
            [meu-app.db :as db]))

(defroutes api-routes

  (GET "/usuarios" req
    (response (db/listar-usuarios)))

  (GET "/usuarios/:id" [id]
    (let [usuario (db/buscar (Long/parseLong id))]
      (if usuario
        (response usuario)
        {:status 404 :body {:error "não encontrado"}})))

  (POST "/usuarios" {body :body}
    (let [novo (db/criar-usuario! body)]
      {:status 201 :body novo}))

  (PUT "/usuarios/:id" {body :body {id :id} :params}
    (let [atualizado (db/atualizar! (Long/parseLong id) body)]
      (response atualizado)))

  (DELETE "/usuarios/:id" [id]
    (db/deletar! (Long/parseLong id))
    {:status 204 :body nil})

  (not-found {:status 404 :body {:error "rota não existe"}}))

Hiccup — HTML como Dados

Hiccup — vetores Clojure como HTML
(ns meu-app.views
  (:require [hiccup2.core :as h]))

; Sintaxe: [tag attrs? & children]
(defn layout [titulo & content]
  (h/html
    {:mode :html}
    [:html {:lang "pt-br"}
     [:head
      [:meta {:charset "UTF-8"}]
      [:title titulo]]
     [:body
      [:h1 titulo]
      content]]))

(defn lista-usuarios [usuarios]
  (h/html
    [:ul.user-list           ; shorthand para class
     (for [u usuarios]
       [:li {:key (:id u)}
        [:span.nome (:nome u)]
        [:span.email (:email u)]])]))

; :#main-id = id="main-id"
[:div#main-id.container.flex
 [:p "conteúdo"]]
; => <div id="main-id" class="container flex"><p>conteúdo</p></div>

ClojureScript — Front-end com Reagent

Reagent — React com sintaxe Hiccup
(ns meu-app.frontend
  (:require [reagent.core :as r]
            [reagent.dom  :as rdom]))

; Atom Reagent — estado reativo
(def contador (r/atom 0))

; Componente = função que retorna Hiccup
(defn botao-contador []
  [:div
   [:p "Contador: " @contador]
   [:button {:on-click #(swap! contador inc)}
    "+"]
   [:button {:on-click #(swap! contador dec)}
    "-"]])

; Montar na DOM
(rdom/render [botao-contador]
             (.getElementById js/document "app"))
Re-frame — arquitetura Flux/Redux para ClojureScript
(ns meu-app.events
  (:require [re-frame.core :as rf]))

; ── db — estado global ───────────────────────────────────────
(rf/reg-event-db
  ::init
  (fn [_ _] {:contador 0 :usuarios []}))

; ── event — como state muda ──────────────────────────────────
(rf/reg-event-db
  ::incrementar
  (fn [db _] (update db :contador inc)))

; ── subscription — derivar estado ───────────────────────────
(rf/reg-sub
  ::contador
  (fn [db _] (:contador db)))

; ── view — componente que lê subscription ────────────────────
(defn tela []
  (let [n (rf/subscribe [::contador])]
    (fn []
      [:div
       [:h2 "Contador: " @n]
       [:button {:on-click #(rf/dispatch [::incrementar])}
        "Incrementar"]])))

Ring transforma um conceito complexo (HTTP) em algo simples: requisição é um mapa, resposta é um mapa, middleware é uma função. Isso permite compor middlewares como funções: (-> handler wrap-auth wrap-json wrap-cors) — cada camada adiciona comportamento sem alterar a lógica central, seguindo literalmente o princípio de responsabilidade única.
