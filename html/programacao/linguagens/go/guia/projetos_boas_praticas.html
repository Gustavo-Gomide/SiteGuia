Go — Projetos e Boas Práticas

Boas práticas em Go cobrem a organização idiomática de pacotes e módulos, injeção de dependências com interfaces (não frameworks), tratamento de erros contextual com fmt.Errorf("%w"), padrões de configuração com variáveis de ambiente, Dockerfile multi-stage para imagens mínimas e pipeline de CI/CD.

Go não tem um padrão oficial de estrutura de projeto, mas a comunidade converge para o layout do repositório golang-standards/project-layout: cmd/ para main packages, internal/ para código privado do módulo, pkg/ para bibliotecas reutilizáveis. Simples, escalável e sem mágica.

internal/service/produto.go — injeção de dependências com interfaces
package service

import (
    "context"
    "fmt"
)

type Produto struct {
    ID    int
    Nome  string
    Preco float64
}

// Interface — define contrato, não implementação
type ProdutoRepository interface {
    FindByID(ctx context.Context, id int) (*Produto, error)
    Create(ctx context.Context, p *Produto) error
    List(ctx context.Context) ([]Produto, error)
}

// Service depende de interface — fácil de testar com mock
type ProdutoService struct {
    repo ProdutoRepository
}

func NewProdutoService(repo ProdutoRepository) *ProdutoService {
    return &ProdutoService{repo: repo}
}

func (s *ProdutoService) BuscarProduto(ctx context.Context, id int) (*Produto, error) {
    p, err := s.repo.FindByID(ctx, id)
    if err != nil {
        return nil, fmt.Errorf("ProdutoService.BuscarProduto: %w", err)
    }
    return p, nil
}

// Mock para testes — sem mockgen
type mockRepo struct {
    produtos map[int]*Produto
}

func (m *mockRepo) FindByID(ctx context.Context, id int) (*Produto, error) {
    if p, ok := m.produtos[id]; ok {
        return p, nil
    }
    return nil, ErrNaoEncontrado
}

Configuração com Variáveis de Ambiente

internal/config/config.go — 12-factor app
package config

import (
    "fmt"
    "os"
    "strconv"
)

type Config struct {
    ServerPort  int
    DatabaseURL string
    JWTSecret   string
    LogLevel    string
    Debug       bool
}

func Load() (*Config, error) {
    port, err := strconv.Atoi(getEnv("SERVER_PORT", "8080"))
    if err != nil {
        return nil, fmt.Errorf("SERVER_PORT inválida: %w", err)
    }

    dbURL := os.Getenv("DATABASE_URL")
    if dbURL == "" {
        return nil, fmt.Errorf("DATABASE_URL é obrigatória")
    }

    return &Config{
        ServerPort:  port,
        DatabaseURL: dbURL,
        JWTSecret:   getEnv("JWT_SECRET", "dev-secret-trocar-em-producao"),
        LogLevel:    getEnv("LOG_LEVEL", "info"),
        Debug:       getEnv("DEBUG", "false") == "true",
    }, nil
}

func getEnv(key, defaultVal string) string {
    if v := os.Getenv(key); v != "" {
        return v
    }
    return defaultVal
}

Dockerfile Multi-Stage

Imagem final minúscula: scratch + binário estático
# Dockerfile
# Stage 1: build
FROM golang:1.22-alpine AS builder

WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download

COPY . .
RUN CGO_ENABLED=0 GOOS=linux \
    go build -ldflags="-s -w" -o servidor ./cmd/server

# Stage 2: imagem final — sem go, sem compilador, sem shell
FROM scratch

COPY --from=builder /app/servidor /servidor
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/

EXPOSE 8080
ENTRYPOINT ["/servidor"]
# Imagem final: ~5-10MB em vez de 350MB do Go base

Makefile — Automação de Tarefas

Makefile idiomático para projetos Go
.PHONY: build test lint run docker-build docker-push migrate

BIN_NAME := servidor
VERSION  := $(shell git describe --tags --always --dirty)
LDFLAGS  := -ldflags="-s -w -X main.Version=$(VERSION)"

build:
	CGO_ENABLED=0 go build $(LDFLAGS) -o bin/$(BIN_NAME) ./cmd/server

test:
	go test -race -coverprofile=coverage.out ./...
	go tool cover -html=coverage.out -o coverage.html

test-fast:
	go test ./...

lint:
	golangci-lint run ./...

vet:
	go vet ./...

run:
	go run ./cmd/server

migrate-up:
	migrate -path migrations/ -database $$DATABASE_URL up

migrate-down:
	migrate -path migrations/ -database $$DATABASE_URL down 1

docker-build:
	docker build -t meuprojeto:$(VERSION) .

generate:
	go generate ./...

tidy:
	go mod tidy
	go fmt ./...

ci: tidy vet lint test build

O pacote internal/ é a feature de encapsulamento mais importante de Go para projetos crescentes. Qualquer código dentro de internal/ só pode ser importado por pacotes do mesmo módulo — o compilador rejeita imports externos. Isso cria uma API pública explícita (pkg/) e uma privada (internal/) sem annotations, sem access modifiers, sem javadoc. Simples e obrigatório pelo compilador.
