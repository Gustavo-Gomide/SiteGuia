Go — Ecossistema e Tooling

Go inclui tudo o que você precisa na instalação padrão: compilador, formatador (gofmt), analisador estático (go vet), gerenciador de módulos (go mod), framework de testes (testing) com benchmarks e race detector. Sem depender de ferramentas de terceiros para o ciclo básico de desenvolvimento.

gofmt é obrigatório — não uma sugestão. Todo código Go é formatado identicamente. Não há debate sobre estilo, não há configuração de linters para espaço vs tab. Qualquer desenvolvedor pode abrir qualquer projeto Go no mundo e ver o mesmo estilo. Pull requests de formatação não existem.

go.mod, go.sum e comandos de dependência
# Criar módulo
go mod init github.com/usuario/meuprojeto

# go.mod — define o módulo
module github.com/usuario/meuprojeto

go 1.22

require (
    github.com/gin-gonic/gin v1.9.1
    golang.org/x/text v0.14.0
)

require (
    // dependências indiretas (transitivas)
    github.com/bytedance/sonic v1.10.2 // indirect

# Adicionar dependência
go get github.com/pkg/errors@v0.9.1

# Atualizar para versão mais recente
go get -u github.com/gin-gonic/gin

# Remover dependências não usadas
go mod tidy

# Download de todas as dependências
go mod download

# Verificar integridade (go.sum)
go mod verify

# Substituir módulo local (para dev)
# replace github.com/usuario/lib => ../lib

# Vendor — copiar deps para /vendor (CI offline)
go mod vendor
go build -mod=vendor ./...

Testing — Testes, Benchmarks e Fuzzing

Testes unitários, table-driven, benchmarks e fuzz
// soma_test.go (sufixo _test.go — não vai no binário final)
package calc_test

import (
    "testing"
    "github.com/usuario/meuprojeto/calc"
)

// Table-driven test — idioma padrão Go
func TestSoma(t *testing.T) {
    casos := []struct {
        nome     string
        a, b     int
        esperado int
    }{
        {"positivo", 2, 3, 5},
        {"zero", 0, 0, 0},
        {"negativo", -1, 1, 0},
        {"grande", 1000, 999, 1999},
    }
    for _, tc := range casos {
        t.Run(tc.nome, func(t *testing.T) {
            resultado := calc.Soma(tc.a, tc.b)
            if resultado != tc.esperado {
                t.Errorf("Soma(%d,%d) = %d; quero %d", tc.a, tc.b, resultado, tc.esperado)
            }
        })
    }
}

// Benchmark
func BenchmarkSoma(b *testing.B) {
    for i := 0; i < b.N; i++ {
        calc.Soma(100, 200)
    }
}

// Fuzz testing (Go 1.18+)
func FuzzDividir(f *testing.F) {
    f.Add(10.0, 2.0)   // seed corpus
    f.Add(0.0, 1.0)
    f.Fuzz(func(t *testing.T, a, b float64) {
        resultado, err := calc.Dividir(a, b)
        if err == nil && b != 0 {
            epsilon := 1e-10
            if resultado * b - a > epsilon {
                t.Errorf("Dividir inconsistente: a=%v b=%v res=%v", a, b, resultado)
            }
        }
    })
}

Linters e Análise Estática

golangci-lint e staticcheck
# Instalar golangci-lint (aggrega 50+ linters)
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest

# Rodar
golangci-lint run ./...

# .golangci.yml — configuração
linters:
  enable:
    - staticcheck    # erros sutis de API e lógica
    - errcheck       # erros não verificados
    - govet          # análise do compilador ampliada
    - ineffassign    # atribuições nunca usadas
    - unused         # código não referenciado
    - gosec          # vulnerabilidades de segurança (OWASP)
    - gocyclo        # complexidade ciclomática

linters-settings:
  gocyclo:
    min-complexity: 15

# staticcheck standalone
go install honnef.co/go/tools/cmd/staticcheck@latest
staticcheck ./...

IDEs e Extensões

VS Code + Go Extension

Extensão oficial golang.go: completação via gopls (Language Server), import automático, debug via Delve, test runner integrado e snippets de struct/func/test.

GoLand (JetBrains)

IDE comercial com refatorações avançadas, análise de data flow, profiler integrado, suporte nativo a Docker/Kubernetes e debugger com watches de goroutines.

gopls — Language Server

LSP oficial do Go Team: hover, completação, rename, inlay hints, semantic tokens. Usado por VS Code, Neovim, Emacs, Helix e qualquer editor LSP-compatível.

Delve — Debugger

dlv debug — breakpoints, step into/over, inspect goroutines, avaliar expressões em runtime. Integrado no VS Code e GoLand. Funciona com remote debug em containers Docker.

O race detector (go test -race, go run -race) é uma das ferramentas mais poderosas da linguagem. Ele instrumenta o binário para rastrear todos os acessos a memória compartilhada e reporta exatamente onde duas goroutines acessam a mesma variável sem sincronização — com o stack trace de ambas. Não é uma análise estática: detecta races reais em tempo de execução. Indispensável em code review e CI.
