Go — Introdução

Go (ou Golang) é uma linguagem compilada, estaticamente tipada e com coleta de lixo, criada no Google em 2007 e lançada publicamente em 2009. Projetada para ser simples, eficiente e com suporte de primeira classe a concorrência, Go se tornou a linguagem dominante em infraestrutura de nuvem, sistemas distribuídos e ferramentas de backend modernas.

A filosofia de Go é radical na sua simplicidade: menos é mais. Nada de herança, sem generics por anos, sem sobrecarga de operadores, sem exceções — apenas interfaces implícitas, goroutines e channels. O resultado é código que qualquer desenvolvedor Go pode ler em dias, não semanas. Docker, Kubernetes, Terraform, etcd, Prometheus e CockroachDB são todos escritos em Go.

Hello World e Estrutura Básica

hello.go — primeiro programa
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}

// Compilar e executar
// go run hello.go
// go build -o hello hello.go ; ./hello
Instalação e primeiros passos
# Instalar Go (Linux/macOS)
# Baixar de https://go.dev/dl/
tar -C /usr/local -xzf go1.22.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

# Verificar instalação
go version                  # go version go1.22.x linux/amd64

# Criar novo projeto (Go Modules)
mkdir meuprojeto ; cd meuprojeto
go mod init github.com/usuario/meuprojeto
# Cria go.mod — define módulo e versão mínima de Go

# Adicionar dependência
go get github.com/gin-gonic/gin@latest
go get golang.org/x/text@v0.14.0

# Comandos go tool
go build ./...     # compila tudo
go test ./...      # roda todos os testes
go vet ./...       # análise estática
go fmt ./...       # formata código (obrigatório)
go doc fmt.Println # documentação inline
go env             # variáveis de ambiente Go

Características Principais

Compilação Rápida

Go compila centenas de milhares de linhas em segundos. O compilador é projetado para ser incremental e sem dependências circulares — garantia da linguagem, não do tooling.

Goroutines e Channels

Concorrência nativa: goroutines são threads cooperativas de ~2KB (vs ~1MB por thread OS). Channels são canais com tipos para comunicação segura entre goroutines — CSP (Communicating Sequential Processes).

Interfaces Implícitas

Um tipo satisfaz uma interface automaticamente se implementar seus métodos — sem implements. Permite desacoplar produtor e consumidor de interface sem coordenação.

Binário Estático

Go compila para um único binário estático sem dependências externas. Deploy de um arquivo. Containers sem camadas extras. Funciona offline. Ideal para CLIs e serviços em containers.

GC com Baixa Latência

Coletor de lixo com pausas sub-milissegundo. Desde Go 1.5, pausas < 10ms; desde Go 1.14, pausas < 1ms em workloads típicos. Adequado para APIs de baixa latência.

Tratamento de Erros Explícito

Sem exceções. Funções retornam (resultado, error). Cada erro é verificado explicitamente — sem surpresa de propagação. Verboso, mas auditável e previsível.

Quando Usar Go

Go é excelente para

  • Microsserviços e APIs REST/gRPC de alto throughput
  • Ferramentas CLI distribuídas como binários estáticos
  • Infraestrutura de nuvem (Kubernetes operators, Terraform providers)
  • Servidores de rede com alta concorrência (proxies, load balancers)
  • Projetos com equipe grande — simplicidade forçada facilita leitura
  • Serviços que precisam de tempo de build curto em CI/CD

Considere outra linguagem para

  • Computação científica e álgebra linear (Python/Julia/Fortran)
  • GUIs desktop ricas (Rust/C++/C#/Kotlin)
  • Sistemas embarcados com restrições de memória (C/Rust/Zig)
  • Programação funcional avançada (Haskell/OCaml/F#)
  • Processamento de linguagem natural e ML (Python)
  • Aplicações mobile (Kotlin/Swift/Dart)

Rob Pike, co-criador de Go, descreveu a motivação: "Começamos a projetar Go enquanto esperávamos uma compilação C++ de 45 minutos terminar." A frustração com a complexidade acidental de C++, Java e Python gerou uma linguagem que prioriza o tempo do engenheiro sobre o tempo de máquina. Go não é a linguagem mais rápida nem a mais expressiva — é a que torna equipes grandes mais produtivas.
