Elixir — Introdução

Introducao ao Elixir

Elixir

Linguagem funcional, concorrente e tolerante a falhas construída sobre a BEAM VM — a máquina virtual do Erlang, com mais de 30 anos de histórico em sistemas de telecomunicações de alta disponibilidade.

Elixir combina a robustez do Erlang com uma sintaxe moderna e expressiva, tornando fácil construir sistemas distribuídos escaláveis que permanecem online 24/7, mesmo diante de falhas parciais.

Sintaxe Inicial em Elixir

iex e primeiros comandos com Mix
# shell interativo
$ iex
iex> is_atom(:ok)
true
iex> "elixir" |> String.upcase()
"ELIXIR"

# criar projeto OTP
$ mix new hello_beam --sup
$ cd hello_beam
$ mix test

# executar funcao em runtime
$ mix run -e "IO.puts(:erlang.system_info(:otp_release))"

Filosofia e Princípios

Pilares do Elixir

PrincípioDescrição
ImutabilidadeDados nunca são modificados no lugar — cada transformação cria novo valor, eliminando bugs de estado compartilhado
Processos LevesErlang/BEAM suporta milhões de processos concorrentes com custo de ~2KB de memória cada; isolados por padrão
Let It CrashFilosofia OTP: deixe processos falharem e use Supervisores para reiniciá-los; código de erro fica separado do caminho feliz
Pattern MatchingCorrespondência estrutural de padrões em vez de condicionais verbosas; destructuring nativo em todas as expressões
Pipe OperatorO operador |> encadeia transformações de dados de forma legível, passando resultado de uma função como primeiro argumento da próxima
Tolerância a FalhasÁrvores de supervisão OTP garantem que falhas em componentes individuais não derrubem o sistema inteiro
DistribuiçãoMúltiplos nós BEAM se comunicam nativamente via mensagens; concorrência e distribuição são o mesmo modelo
José Valim sobre Elixir: "Elixir é sobre produtividade e extensibilidade. Queremos que desenvolvedores se sintam em casa, mas também que possam construir sistemas que nunca param." — A BEAM VM, subjacente ao Elixir, alimenta sistemas com 99,9999999% de uptime (nine nines) na infraestrutura de telecomunicações.

Primeiro Programa

hello_world.exs — Script Elixir

# hello_world.exs — execute com: elixir hello_world.exs

IO.puts("Olá, Mundo!")

# Variáveis e imutabilidade
nome = "Elixir"
versao = 1.17
IO.puts("Bem-vindo ao #{nome} #{versao}")

# Funções anônimas
saudar = fn nome -> "Olá, #{nome}!" end
IO.puts(saudar.("Mundo"))

# Pipe operator: encadeamento elegante
resultado =
  "  hello elixir  "
  |> String.trim()
  |> String.upcase()
  |> String.replace("ELIXIR", "🧪 ELIXIR")

IO.puts(resultado)  # HELLO 🧪 ELIXIR

# Pattern matching
{:ok, valor} = {:ok, 42}
IO.puts("Valor: #{valor}")

# Listas e Enum
numeros = [1, 2, 3, 4, 5]
soma = Enum.reduce(numeros, 0, fn x, acc -> x + acc end)
IO.puts("Soma: #{soma}")  # Soma: 15

Módulo Elixir — hello.ex

defmodule Hello do
  # @moduledoc documenta o módulo inteiro
  @moduledoc """
  Módulo de boas-vindas demonstrando estrutura básica de um módulo Elixir.
  """

  # @doc documenta a função pública seguinte
  @doc "Saúda uma pessoa pelo nome"
  def saudar(nome) do
    "Olá, #{nome}! Bem-vindo ao Elixir."
  end

  # defp = função privada
  defp formatar(texto) do
    "[#{texto}]"
  end

  def main do
    msg = saudar("Mundo")
    IO.puts(formatar(msg))
  end
end

# Executar: mix run -e "Hello.main()"
Hello.main()  # [Olá, Mundo! Bem-vindo ao Elixir.]

A BEAM VM

Como funciona a BEAM (Bogdan/Björn's Erlang Abstract Machine)

CaracterísticaDetalhe Técnico
ProcessosProcessos BEAM são isolados — sem memória compartilhada. Cada processo tem sua própria heap. GC ocorre por processo, sem pausas globais (stop-the-world)
SchedulerPreemptivo com time-slicing (redução de chamadas). Um scheduler por núcleo de CPU. Trabalho roubado entre schedulers para balanceamento
MensagensProcessos comunicam via passagem de mensagens (send/receive). Mensagens são copiadas entre heaps de processos garantindo isolamento
Supervisores OTPÁrvores de supervisão definem estratégias de reinício: :one_for_one, :one_for_all, :rest_for_one. Falhas são confinadas e recuperadas automaticamente
Hot Code ReloadCódigo pode ser atualizado em produção sem parar o sistema — dois módulos coexistem temporariamente durante a transição
DistribuiçãoNós BEAM na mesma rede se descobrem automaticamente. Node.connect/1 conecta nós; processos remote e local usam o mesmo send/2
ETS/DETSErlang Term Storage: tabelas hash/árvore ultrarrápidas em memória compartilhada por processos (exceção controlada ao isolamento)

Características Principais

Concorrência Real

Milhões de processos leves simultâneos. Cada conexão HTTP pode ser um processo isolado sem overhead de threads do SO. Sem callback hell, sem async/await.

Tolerância a Falhas

Filosofia "Let It Crash" + Supervisores OTP. Sistemas com 99,9999% de uptime. Falhas se propagam verticalmente pela árvore de supervisão, não horizontalmente.

Phoenix LiveView

UIs reativas em tempo real sem JavaScript. O servidor mantém estado da view, envia diff de HTML via WebSocket. Produtividade full-stack com Elixir puro.

Macros Poderosas

Sistema de macros homoicônico (AST manipulável em Elixir). Phoenix, Ecto e ExUnit são todos implementados em macros Elixir puras — extensibilidade máxima.

Ecossistema Hex

Gerenciador de pacotes Hex.pm com milhares de bibliotecas. Mix como ferramenta de build integrada. ExUnit para testes. Tooling de primeira classe desde o início.

Interop com Erlang

Acesso total a todas as bibliotecas Erlang/OTP: :ets, :crypto, :gen_tcp, :mnesia. Décadas de código battle-tested disponíveis gratuitamente.

Quando Usar Elixir

Use Elixir quando...

  • Construindo APIs web com alta concorrência (milhares de req/s simultâneas)
  • Sistemas de tempo real: chats, dashboards ao vivo, multiplayer, streaming
  • Aplicações que precisam de 99.9%+ de uptime (telecomunicações, fintech)
  • Microsserviços que precisam de comunicação entre nós distribuídos
  • IoT com Nerves: firmware Elixir para Raspberry Pi e dispositivos embarcados
  • Pipelines de processamento de dados assíncronos com GenStage/Broadway
  • Backend de WebSockets e canais em tempo real com Phoenix Channels
  • UIs reativas sem JavaScript com Phoenix LiveView

Evite Elixir quando...

  • Computação numérica intensiva (melhor: Python/C++/Julia)
  • Scripts rápidos e utilitários simples de linha de comando (curva de aprendizado é alta para pequenas tarefas)
  • Aplicações desktop GUI nativas (sem framework maduro equivalente ao Qt)
  • Equipe sem experiência em programação funcional e sem tempo para aprender
  • Projetos com budget zero para infraestrutura — BEAM tem overhead de memória base maior que Node.js para cargas simples
  • Machine learning/deep learning em produção (Nx está crescendo mas Python domina)
