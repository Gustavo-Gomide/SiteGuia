Elixir — Ecossistema e Tooling

O toolchain oficial do Elixir inclui mix (build, test, deps, release), IEx (REPL interativo), ExUnit (testes), e Dialyzer (análise estática de tipos). Juntos, formam um ambiente de desenvolvimento completo sem necessidade de ferramentas de terceiros para desenvolvimento básico.

Instale sempre o Elixir via asdf (version manager) — não pelo gerenciador de sistema. Isso permite múltiplas versões de Elixir e Erlang por projeto, reprodutibilidade total e atualizações sem sudo.

Instalação com asdf

Instalação multiplataforma via asdf
# ── Instalar asdf ────────────────────────────────────────────
git clone https://github.com/asdf-vm/asdf.git ~/.asdf --branch v0.14.0
echo '. "$HOME/.asdf/asdf.sh"' >> ~/.bashrc
source ~/.bashrc

# ── Erlang (pré-requisito) ───────────────────────────────────
asdf plugin add erlang
asdf install erlang 27.2
asdf global erlang 27.2

# ── Elixir ───────────────────────────────────────────────────
asdf plugin add elixir
asdf install elixir 1.17.3-otp-27
asdf global elixir 1.17.3-otp-27

# Verificar
elixir --version
# Erlang/OTP 27 [erts-15.1] ... Elixir 1.17.3

# ── macOS (alternativa) ──────────────────────────────────────
brew install elixir

# ── Windows ──────────────────────────────────────────────────
# Usar WSL2 (Ubuntu) + asdf, ou installer em elixir-lang.org

# ── .tool-versions por projeto ───────────────────────────────
# Criar arquivo na raiz do projeto:
erlang 27.2
elixir 1.17.3-otp-27
# asdf usa automaticamente as versões corretas no diretório

Mix — Build Tool Oficial

Comandos mix essenciais
# ── Criar projeto ────────────────────────────────────────────
mix new meu_projeto              # aplicação simples
mix new meu_app --sup            # com OTP Application + Supervisor
mix phx.new meu_site             # projeto Phoenix completo

# ── Dependências ─────────────────────────────────────────────
mix deps.get                     # instalar dependências do mix.exs
mix deps.update --all            # atualizar todas
mix deps.update jason            # atualizar uma específica
mix deps.clean --all             # limpar

# ── Compilação ───────────────────────────────────────────────
mix compile                      # compilar
mix compile --warnings-as-errors # tratar warnings como erros

# ── Testes ───────────────────────────────────────────────────
mix test                         # todos os testes
mix test test/meu_test.exs       # arquivo específico
mix test test/meu_test.exs:42    # linha específica
mix test --only integration      # apenas @tag :integration
mix test --cover                 # com cobertura de código

# ── Linting e qualidade ──────────────────────────────────────
mix format                       # formatar código (como gofmt)
mix format --check-formatted     # verificar sem modificar (CI)
mix credo                        # análise estática (instalar credo)
mix dialyzer                     # análise de tipos Dialyzer

# ── Release ──────────────────────────────────────────────────
MIX_ENV=prod mix release         # criar release produção
./_build/prod/rel/meu_app/bin/meu_app start

# ── Ecto (banco de dados) ────────────────────────────────────
mix ecto.create                  # criar banco
mix ecto.migrate                 # rodar migrations
mix ecto.rollback                # reverter última migration
mix ecto.gen.migration criar_usuarios

IEx — REPL Interativo

IEx — sessão interativa e helpers úteis
# Iniciar IEx
iex                  # REPL simples
iex -S mix           # com projeto mix carregado
iex -S mix phx.server  # Phoenix em modo interativo

# Helpers no IEx
h String             # docs do módulo String
h String.split       # docs da função split
i "hello"           # inspecionar tipo e metadados
v()                  # último valor avaliado
v(3)                 # 3º valor avaliado
r MeuModulo         # recarregar módulo sem reiniciar
c "lib/meu.ex"      # compilar arquivo no IEx
ls                   # listar diretório
pwd                  # diretório atual

# Conexão remota com nó em produção
iex --sname debug --cookie minha_cookie
Node.connect(:"app@servidor")
:observer.start()    # GUI do Observer (para Erlang)

# Breakpoints com IEx.pry
require IEx
IEx.pry()            # pausa execução e abre IEx no contexto

# Autocompletar com Tab
Enum.    # Tab → lista todas as funções de Enum

ExUnit — Framework de Testes

ExUnit — testes unitários e de integração
defmodule MeuApp.CalculadoraTest do
  use ExUnit.Case, async: true   # async: true = rodar em paralelo
  alias MeuApp.Calculadora

  # Setup — executar antes de cada teste
  setup do
    {:ok, calc: Calculadora.new()}
  end

  @tag :unit
  test "soma dois números", %{calc: calc} do
    assert Calculadora.soma(calc, 2, 3) == 5
  end

  test "divisão por zero retorna erro" do
    assert {:error, "divisão por zero"} == Calculadora.dividir(10, 0)
  end

  test "pattern matching em resultado" do
    assert {:ok, resultado} = Calculadora.dividir(10, 2)
    assert resultado == 5.0
  end

  # Doctest — testa exemplos da documentação
  doctest Calculadora
end

Dialyzer e Typespecs

@spec, @type e Dialyzer para análise estática
defmodule MeuApp.Pedido do

  @type status :: :pendente | :aprovado | :cancelado

  @type t :: %__MODULE__{
    id: integer(),
    valor: float(),
    status: status(),
    itens: list(String.t())
  }

  @spec criar(float(), list(String.t())) :: {:ok, t()} | {:error, String.t()}
  def criar(valor, itens) when valor > 0 do
    {:ok, %__MODULE__{id: gerar_id(), valor: valor, status: :pendente, itens: itens}}
  end
  def criar(_, _), do: {:error, "valor deve ser positivo"}

  @spec aprovar(t()) :: {:ok, t()} | {:error, String.t()}
  def aprovar(%__MODULE__{status: :pendente} = pedido) do
    {:ok, %{pedido | status: :aprovado}}
  end
  def aprovar(_), do: {:error, "pedido não está pendente"}

end

mix format é o formatador oficial do Elixir — opinativo como gofmt — e elimina qualquer discussão de estilo de código. O comando mix format --check-formatted no CI garante que nenhum código mal formatado chegue ao repositório. Adote desde o primeiro dia: é um hábito que economiza centenas de horas de revisão de código ao longo de um projeto.
