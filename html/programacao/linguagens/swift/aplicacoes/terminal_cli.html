Swift - Terminal e CLI

Swift para terminal e linha de comando

Ferramentas com ArgumentParser, distribuicao por binarios e automacao com Swift Package Manager.

Ferramentas CLI sao o laboratorio ideal para perceber o valor de Swift fora da interface grafica: compilacao rapida, binarios nativos, tipagem forte e uma ergonomia que ajuda a manter scripts pequenos evoluindo para utilitarios profissionais.

Por que escrever CLI em Swift

Uma ferramenta de terminal costuma nascer simples: automatizar um deploy, validar arquivos, migrar dados, gerar relatorios. Com o tempo, ganha subcomandos, configuracao, logs e testes. Swift funciona bem nesse crescimento porque oferece binario compilado, API organizada e menos risco de erro estrutural do que scripts improvisados.

ArgumentParser como base

O pacote swift-argument-parser e a forma idiomatica de construir comandos, flags, ajuda automatica e subcomandos. Em vez de manipular argumentos manualmente, o desenvolvedor modela a interface da CLI como tipos Swift.

swift - comando com ArgumentParser
import ArgumentParser

struct BuildReport: ParsableCommand {
    @Option(name: .shortAndLong, help: "Arquivo de entrada")
    var input: String

    @Flag(name: .long, help: "Nao grava arquivos, apenas simula")
    var dryRun = false

    mutating func run() throws {
        if dryRun {
            print("Simulando processamento de \(input)")
            return
        }

        print("Gerando relatorio a partir de \(input)")
    }
}

BuildReport.main()

Distribuicao e instalacao

Ferramentas CLI precisam chegar ao usuario final. No ecossistema Swift, a distribuicao pode ser feita como codigo-fonte via Swift Package Manager, como binario assinado ou via gerenciadores como Homebrew. O ponto importante e tratar versao, plataforma alvo e experiencia de instalacao como parte do produto.

  • SPM: ideal para desenvolvedores Swift.
  • Homebrew: excelente para distribuicao em macOS.
  • Artefatos binarios: uteis quando se quer reduzir tempo de setup.

Boas praticas para CLIs serias

Uma CLI madura deve ter help consistente, codigos de saida confiaveis, mensagens de erro legiveis e testes. Tambem vale separar parsing de argumentos, logica de dominio e acesso a sistema de arquivos para tornar o comando auditavel.

O teste essencial de uma CLI nao e so se ela compila. E se uma pessoa sob pressao consegue entender o help, corrigir um erro de uso e automatizar o comando em CI sem adivinhar comportamentos escondidos.

Comparacao com scripts shell e Python

Shell e excelente para colar utilitarios ja existentes; Python e otimo para automacao rica com bibliotecas vastas. Swift entra quando a ferramenta vai crescer, precisa de binario nativo ou sera mantida com padrao semelhante ao de uma aplicacao. Nao substitui tudo, mas reduz improviso quando o script vira produto interno.
