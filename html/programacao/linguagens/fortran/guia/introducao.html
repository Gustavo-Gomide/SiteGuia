Fortran — Introdução

Fortran — Formula Translation

Fortran é a primeira linguagem de programação de alto nível amplamente adotada, criada pela IBM em 1957. O nome vem de FORmula TRANslation — seu objetivo original era traduzir fórmulas matemáticas em código de máquina. Mais de 65 anos depois, continua sendo a linguagem dominante em simulação numérica, previsão climática, aerodinâmica e supercomputação.

Benchmarks consistentes mostram que código Fortran bem escrito compete — e frequentemente supera — C e C++ em computação numérica vetorizada. O compilador pode assumir que ponteiros nunca se sobrepõem (sem aliasing), resultando em vetorização SIMD mais agressiva. Os codes de física nuclear, modelos de clima e motores CFD mais rápidos do mundo são escritos em Fortran.

Primeiro Programa — Hello World

hello.f90 — Fortran moderno (free-form)
! Fortran moderno — free-form source (extensão .f90 ou .f95)
program hello
    implicit none   ! obrigatório em código moderno — sem variáveis implícitas

    print *, "Olá, Mundo!"
    write(*, *) "Escrita formatada"

end program hello
Compilar e executar com gfortran e ifort/ifx
# gfortran (GCC) — gratuito e multiplataforma
gfortran hello.f90 -o hello
./hello

# ifx (Intel Fortran Compiler — substituto moderno do ifort)
# Otimizações Intel AVX-512, MKL
ifx hello.f90 -o hello
./hello

# nvfortran (NVIDIA) — compilar para GPU
# Flags comuns de otimização
gfortran -O3 -march=native -ffree-line-length-none -o calc calc.f90

# Com OpenMP (paralelismo)
gfortran -O3 -fopenmp -o calc_par calc_par.f90

Características Essenciais

Arrays como cidadão de 1ª classe

Arrays multidimensionais com indexação Fortran (1-based), operações vetorizadas nativas (A = B + C), slicing (A(1:N:2)) e funções intrínsecas (matmul, dot_product, sum, maxval).

Column-major order

Arrays multidimensionais são armazenados por coluna (ao contrário de C que é por linha). Iterar na ordem correta pode ser 10× mais rápido por localidade de cache.

Sem aliasing por padrão

O padrão permite ao compilador assumir que argumentos de subrotinas não se sobrepõem na memória — possibilitando vetorização SIMD mais agressiva que C (onde restrict é necessário).

Coarrays (paralelismo)

Fortran 2008 introduziu coarrays como parte da linguagem — paralelismo por particionamento de dados sem biblioteca externa: REAL :: X[*] cria um array distribuído entre imagens SPMD.

Módulos e encapsulamento

Fortran 90 trouxe módulos com controle de visibilidade (public/private), interfaces explícitas, tipos derivados e herança (Fortran 2003).

Interoperabilidade com C

Fortran 2003 padronizou o módulo iso_c_binding: chamar funções C de Fortran e vice-versa com tipos exatamente correspondentes — fundamental para integrar com MPI, BLAS, CUDA.

Quando Usar Fortran

Use Fortran quando…

  • Física numérica: simulação de fluidos, elementos finitos, Monte Carlo
  • Previsão climática e modelos de circulação oceânica
  • HPC em supercomputadores (TOP500 usa muito Fortran)
  • Código legado FORTRAN 77/90 para manter ou estender
  • Cálculo de matrizes densas de alta performance
  • Integrar com BLAS/LAPACK nativamente

Evite Fortran quando…

  • Desenvolvimento web, APIs, aplicações de negócio
  • Processamento de strings, parsing de texto
  • GUIs e aplicações de desktop
  • Ecossistema de bibliotecas gerais (Python/JS são melhores)
  • Prototipagem rápida sem foco em performance numérica
