Fortran — Ecossistema e Tooling

O ecossistema Fortran moderno inclui o fpm (Fortran Package Manager) para gerenciar projetos e dependências, compiladores de classe mundial (gfortran livre, ifx/ifort Intel, nvfortran NVIDIA para GPU), e um conjunto de bibliotecas numéricas — BLAS, LAPACK, FFTW, ScaLAPACK — sem par em qualquer outra linguagem.

O compilador Intel Fortran (ifx) gera código que explora ao máximo as instruções AVX-512 dos processadores Intel, com otimizações automáticas para loops numéricos. Para código rodando em hardware Intel — que é a maioria dos supercomputadores do TOP500 — ifx muitas vezes supera gfortran em 20–40% em benchmarks LINPACK.

fpm — Fortran Package Manager

Criar projetos e gerenciar dependências com fpm
# Instalar fpm
# Via conda:        conda install -c conda-forge fpm
# Via macOS/Linux:  Download do binário em fpm.fortran-lang.org

# Criar novo projeto
fpm new meu_projeto
cd meu_projeto

# Estrutura gerada:
# meu_projeto/
#   fpm.toml          ← manifest do projeto
#   src/              ← código fonte
#     meu_projeto.f90
#   app/              ← programa principal
#     main.f90
#   test/             ← testes

# Compilar e executar
fpm run                          # compila e roda
fpm run -- arg1 arg2             # com argumentos
fpm build                        # só compila
fpm build --profile release      # build otimizado
fpm test                         # rodar testes

# fpm.toml — configuração do projeto
fpm.toml com dependências
name = "meu_projeto"
version = "0.1.0"
license = "MIT"
author = "Seu Nome"
maintainer = "voce@email.com"
copyright = "2025"

[build]
auto-executables = true
auto-tests = true
auto-examples = true

[install]
library = true

[[executable]]
name = "calcular"
source-dir = "app"
main = "main.f90"

[dependencies]
stdlib = { git = "https://github.com/fortran-lang/stdlib" }
json-fortran = { git = "https://github.com/jacobwilliams/json-fortran" }

[dev-dependencies]
test-drive = { git = "https://github.com/fortran-lang/test-drive" }

BLAS, LAPACK e FFTW

Álgebra linear de alto desempenho com BLAS/LAPACK
! BLAS — Basic Linear Algebra Subprograms
! Nível 1: operações vetor-vetor (dot, axpy, norm)
! Nível 2: operações matriz-vetor (gemv, trmv)
! Nível 3: operações matriz-matriz (gemm, trmm)
program blas_demo
    implicit none
    integer, parameter :: N = 1000, dp = kind(1.0d0)
    real(dp) :: A(N,N), B(N,N), C(N,N)
    real(dp) :: alpha = 1.0_dp, beta = 0.0_dp

    ! Inicializar A e B com valores aleatórios
    call random_number(A); call random_number(B)

    ! DGEMM: C = alpha*A*B + beta*C
    ! Muito mais rápido que loop triplo manual!
    call dgemm('N', 'N', N, N, N, alpha, A, N, B, N, beta, C, N)

    print *, "Trace(C) =", sum([(C(i,i), i=1,N)])

    ! Compilar: gfortran blas_demo.f90 -o blas_demo -lblas -llapack
    !           ou com OpenBLAS: -lopenblas
    !           ou com Intel MKL: -mkl
end program blas_demo
LAPACK — sistemas lineares e autovalores
! Resolver sistema linear A*x = b com DGESV
program lapack_demo
    implicit none
    integer, parameter :: N = 3, dp = kind(1.0d0)
    real(dp) :: A(N,N), b(N), x(N)
    integer  :: ipiv(N), info

    A = reshape([2.0_dp, 1.0_dp, 0.0_dp,   &
                 1.0_dp, 3.0_dp, 1.0_dp,   &
                 0.0_dp, 1.0_dp, 2.0_dp], [N,N])
    b = [1.0_dp, 2.0_dp, 3.0_dp]
    x = b   ! DGESV sobrescreve x com solução

    ! LU decomposition + solve
    call dgesv(N, 1, A, N, ipiv, x, N, info)

    if (info == 0) then
        print *, "Solução:", x
    else
        print *, "LAPACK erro:", info
    end if

end program lapack_demo

Principais Bibliotecas do Ecossistema

Fortran Stdlib

Biblioteca padrão da comunidade Fortran-lang: arrays dinâmicos, strings, algoritmos de sorting, funções especiais matemáticas, I/O de CSV, linalg de alto nível.

FFTW

Fastest Fourier Transform in the West — implementação de FFT mais rápida disponível. Interface Fortran nativa. Usado em oceanografia, processamento de sinais, solucionadores de EDP.

MPI (Message Passing Interface)

OpenMPI e MPICH têm bindings Fortran nativos. Comunicação entre processos em supercomputadores com milhares de cores — MPI_Send, MPI_Recv, MPI_Bcast, MPI_Reduce.

NetCDF / HDF5

Formatos de dados auto-descritivos para ciências da terra e física. NetCDF4 usa HDF5 internamente. Toda a meteorologia mundial usa NetCDF — com bindings Fortran completos.

json-fortran

Parse e geração de JSON em Fortran puro. Essencial para interoperar com APIs web, arquivos de configuração modernos e integrar código Fortran em pipelines de dados.

OpenACC / CUDA Fortran

OpenACC: diretivas para offload em GPU (similar ao OpenMP). CUDA Fortran (nvfortran): acessar CUDA diretamente de Fortran com kernels e mecanismo de cópia explícito.

BLAS e LAPACK — escritos originalmente em FORTRAN 77 nos anos 1970–80 — são a base oculta de toda computação numérica de alto desempenho: NumPy/SciPy chamam BLAS, R usa BLAS, MATLAB usa MKL (variante Intel do BLAS). Otimizar essas rotinas para cada arquitetura de CPU é um trabalho de décadas — e é por isso que código Fortran batendo chamadas Python/NumPy "ingênuas" quase nunca acontece: ambos chamam o mesmo LAPACK no final.
