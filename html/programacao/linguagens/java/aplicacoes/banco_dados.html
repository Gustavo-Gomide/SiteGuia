Java — Banco de Dados

Java e Bancos de Dados

Spring Data JPA com Hibernate, mapeamento de entidades e relacionamentos, JPQL/Criteria API, controle transacional, migracao com Flyway e otimizacoes para evitar o problema N+1.

Hibernate e o JPA provider padrao do Spring Boot. Ele traduz as entidades Java em tabelas SQL, mas exige atencao ao plano de execucao das queries e as estrategias de fetch para nao gerar dezenas de queries implicitas por requisicao.

Entidades e Relacionamentos

Java + JPA — @Entity com @OneToMany, @ManyToOne e @ManyToMany
@Entity
@Table(name = "pedidos")
public class Pedido {

  @Id
  @GeneratedValue(strategy = GenerationType.UUID)
  private UUID id;

  @Column(nullable = false)
  private LocalDateTime criadoEm = LocalDateTime.now();

  @Enumerated(EnumType.STRING)
  @Column(nullable = false, length = 20)
  private StatusPedido status = StatusPedido.AGUARDANDO;

  // Lado "muitos" — FK cliente_id na tabela pedidos
  @ManyToOne(fetch = FetchType.LAZY, optional = false)
  @JoinColumn(name = "cliente_id")
  private Cliente cliente;

  // Lado "um" — colecao mapeada pela entidade filha
  @OneToMany(mappedBy = "pedido", cascade = CascadeType.ALL,
             orphanRemoval = true, fetch = FetchType.LAZY)
  private List<ItemPedido> itens = new ArrayList<>();

  // Metodo auxiliar para manter consistencia bidirecional
  public void adicionarItem(ItemPedido item) {
    item.setPedido(this);
    itens.add(item);
  }
}

@Entity
@Table(name = "itens_pedido")
public class ItemPedido {

  @EmbeddedId
  private ItemPedidoId id;   // chave composta

  @ManyToOne(fetch = FetchType.LAZY)
  @MapsId("pedidoId")
  private Pedido pedido;

  @ManyToOne(fetch = FetchType.LAZY)
  @MapsId("produtoId")
  private Produto produto;

  @Column(nullable = false)
  private int quantidade;

  @Column(nullable = false, precision = 10, scale = 2)
  private BigDecimal precoUnitario;
}

@Embeddable
public record ItemPedidoId(UUID pedidoId, UUID produtoId) implements Serializable {}

// ManyToMany: produto <-> categoria (tabela associativa)
@ManyToMany(cascade = { CascadeType.PERSIST, CascadeType.MERGE })
@JoinTable(
  name = "produto_categoria",
  joinColumns        = @JoinColumn(name = "produto_id"),
  inverseJoinColumns = @JoinColumn(name = "categoria_id")
)
private Set<Categoria> categorias = new HashSet<>();

Nunca use FetchType.EAGER em colecoes. Torna imprevisivel o numero de queries geradas e e o principal causador do problema N+1. Use FetchType.LAZY e carregue joinando explicitamente com @EntityGraph ou JPQL JOIN FETCH somente quando necessario.

Repositories e Queries

Java + Spring Data JPA — Repository com JPQL, @EntityGraph e projecoes
public interface PedidoRepository extends JpaRepository<Pedido, UUID>,
    JpaSpecificationExecutor<Pedido> {

  // Derivado do nome — Spring gera a query automaticamente
  List<Pedido> findByClienteIdAndStatus(UUID clienteId, StatusPedido status);

  // JPQL parametrizado — sem risco de SQL injection
  @Query("SELECT p FROM Pedido p JOIN FETCH p.itens i JOIN FETCH i.produto "
       + "WHERE p.id = :id")
  Optional<Pedido> buscarComItens(@Param("id") UUID id);

  // EntityGraph — evita N+1 sem alterar a entidade
  @EntityGraph(attributePaths = { "itens", "itens.produto", "cliente" })
  Optional<Pedido> findWithGraphById(UUID id);

  // Projecao — carrega apenas os campos necessarios
  List<PedidoResumo> findByClienteId(UUID clienteId);

  // Count / existencia
  boolean existsByClienteIdAndStatus(UUID clienteId, StatusPedido status);

  // Bulk update com @Modifying — contorna o cache de 1o nivel
  @Modifying
  @Query("UPDATE Pedido p SET p.status = :novo WHERE p.status = :antigo")
  int atualizarStatusEmLote(@Param("antigo") StatusPedido antigo,
                             @Param("novo")   StatusPedido novo);

  // Paginacao nativa — conta total numa query separada
  @Query(value = "SELECT p FROM Pedido p JOIN FETCH p.cliente WHERE p.status <> 'CANCELADO'",
         countQuery = "SELECT COUNT(p) FROM Pedido p WHERE p.status <> 'CANCELADO'")
  Page<Pedido> listarAtivos(Pageable pageable);
}

// Projecao como interface — Spring Data cria o proxy automaticamente
public interface PedidoResumo {
  UUID           getId();
  LocalDateTime  getCriadoEm();
  StatusPedido   getStatus();
  NomeCliente    getCliente();   // nested projection

  interface NomeCliente { String getNome(); }
}

Controle Transacional

Java + Spring — @Transactional com propagacao e isolamento
@Service
public class PedidoService {

  // readOnly=true — Hibernate nao executa flush, conexao liberada antes
  @Transactional(readOnly = true)
  public Page<PedidoResumo> listar(Pageable pageable) {
    return repo.findAll(pageable).map(assembler::toResumo);
  }

  // Propagation.REQUIRED (padrao): reutiliza transacao externa, se houver
  @Transactional
  public Pedido criar(CriarPedidoRequest req) {
    var cliente = clienteRepo.getReferenceById(req.clienteId());
    var pedido  = new Pedido(cliente);
    req.itens().forEach(i -> {
      var produto = produtoRepo.getReferenceById(i.produtoId());
      pedido.adicionarItem(new ItemPedido(produto, i.quantidade()));
    });
    return repo.save(pedido);
  }

  // REQUIRES_NEW — suspende transacao externa e inicia outra independente
  // util para log de auditoria que deve persistir mesmo se a tx principal falhar
  @Transactional(propagation = Propagation.REQUIRES_NEW)
  public void registrarAuditoria(String evento) {
    auditoriaRepo.save(new Auditoria(evento));
  }

  // rollbackFor — Spring so faz rollback automatico em RuntimeException
  // Para exceptions checadas, e necessario declarar
  @Transactional(rollbackFor = IOException.class)
  public void importarCSV(InputStream csv) throws IOException { ... }
}

/* Niveis de isolamento:
  READ_UNCOMMITTED — le dados nao commitados (dirty read)
  READ_COMMITTED   — le apenas dados commitados (padrao Postgres)
  REPEATABLE_READ  — mesma leitura retorna mesmos dados (Postgres usa por padrao para serializable)
  SERIALIZABLE     — maximo isolamento, menor concorrencia
*/

Problema N+1 e Cache L2

Java + Hibernate — Diagnostico e resolucao do N+1
// application.properties — habilitar log de queries (APENAS dev)
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
logging.level.org.hibernate.SQL=DEBUG
logging.level.org.hibernate.orm.jdbc.bind=TRACE

// Exemplo de N+1: ao listar pedidos, Hibernate emite 1 query para a lista
// e mais 1 query POR pedido para carregar o cliente (FetchType.LAZY)
// Total: 101 queries para uma lista de 100 pedidos

// SOLUCAO 1: JOIN FETCH na query
@Query("SELECT p FROM Pedido p JOIN FETCH p.cliente")
List<Pedido> findAllComCliente();

// SOLUCAO 2: @BatchSize — Hibernate agrupa sub-queries em lotes
@BatchSize(size = 30)
@OneToMany(mappedBy = "pedido", fetch = FetchType.LAZY)
private List<ItemPedido> itens;

// SOLUCAO 3: cache de segundo nivel (Hibernate L2 + Caffeine)
// pom.xml:
// <dependency>
//   <groupId>com.github.ben-manes.caffeine</groupId>
//   <artifactId>caffeine</artifactId>
// </dependency>

@Entity
@Cache(usage = CacheConcurrencyStrategy.READ_WRITE)   // entidade mutavel
public class Categoria {
  @Id @GeneratedValue(strategy = GenerationType.UUID)
  private UUID id;
  private String nome;
}

spring.jpa.properties.hibernate.cache.region.factory_class=org.hibernate.cache.jcache.JCacheRegionFactory

Specification API (Criteria Dinamica)

Java + Spring Data JPA — Filtros dinamicos com Specification
public class PedidoSpecs {

  private PedidoSpecs() {}

  public static Specification<Pedido> comStatus(StatusPedido status) {
    return (root, query, cb) ->
      status == null ? cb.conjunction() : cb.equal(root.get("status"), status);
  }

  public static Specification<Pedido> doCliente(UUID clienteId) {
    return (root, query, cb) ->
      clienteId == null ? cb.conjunction()
        : cb.equal(root.get("cliente").get("id"), clienteId);
  }

  public static Specification<Pedido> criadoDepoisDe(LocalDateTime data) {
    return (root, query, cb) ->
      data == null ? cb.conjunction()
        : cb.greaterThanOrEqualTo(root.get("criadoEm"), data);
  }
}

// Uso no servico — combina specs dinamicamente
public Page<Pedido> filtrar(FiltrosPedido filtros, Pageable pageable) {
  var spec = Specification.where(PedidoSpecs.comStatus(filtros.status()))
    .and(PedidoSpecs.doCliente(filtros.clienteId()))
    .and(PedidoSpecs.criadoDepoisDe(filtros.desde()));
  return repo.findAll(spec, pageable);
}
