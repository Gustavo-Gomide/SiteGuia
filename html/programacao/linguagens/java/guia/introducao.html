Java — Introducao

Introducao ao Java

Java e uma linguagem orientada a objetos, compilada para bytecode e executada na JVM. Nasce na Sun Microsystems em 1995 com o lema Write Once, Run Anywhere e hoje pertence a Oracle — mantida tambem pela comunidade via OpenJDK.

A JVM nao executa o codigo-fonte diretamente: o compilador javac produz bytecode (.class), que e interpretado e JIT-compilado em tempo de execucao para o codigo de maquina nativo. Isso garante portabilidade sem sacrificar desempenho.

Java adotou cadencia de releases a cada 6 meses a partir da versao 9 (2017). Versoes LTS (Long-Term Support) recebem patches por 8 ou mais anos — use sempre a LTS mais recente em producao.

OpenJDK vs Oracle JDK

OpenJDK e a implementacao de referencia, open-source (GPL + CE). Oracle JDK oferece suporte comercial estendido. Distribuicoes populares: Eclipse Temurin (Adoptium), Amazon Corretto, Microsoft Build of OpenJDK, GraalVM.

JVM, JRE, JDK

JDK contem compilador + JRE. JRE contem a JVM + bibliotecas padrao (java.lang, java.util ...). JVM e a maquina virtual que interpreta e JIT-compila bytecode em codigo nativo para a plataforma atual.

GraalVM Native Image

Compila Java ahead-of-time para binario nativo sem JVM no runtime — startup em milissegundos, memoria muito menor. Usado por Quarkus e Micronaut para microsservicos serverless.

Primeiro Programa

Java — Hello World comentado linha a linha
// Toda classe publica deve estar em arquivo com mesmo nome: OlaMundo.java
public class OlaMundo {

  // main e o ponto de entrada: public (JVM acessa), static (sem instancia),
  // void (sem retorno), args (argumentos da linha de comando)
  public static void main(String[] args) {

    // System.out e um PrintStream conectado ao stdout
    System.out.println("Ola, Mundo!");

    // A partir do Java 21, instance main e programas sem classe sao previa
    // (JEP 463 — Unnamed Classes and Instance Main Methods)
  }
}
Shell — Compilar e executar
# Compilar: gera OlaMundo.class no mesmo diretorio
javac OlaMundo.java

# Executar: passa o nome da classe (sem .class)
java OlaMundo

# A partir do Java 11: rodar arquivo .java diretamente (single-file execution)
java OlaMundo.java

# Ver versao instalada
java -version
Java 21 — Unnamed Class (previa JEP 463)
// Arquivo: OlaSimples.java — nao precisa de declaracao de classe em Java 21+
void main() {
  System.out.println("Sintaxe simplificada para iniciantes!");
}

Autoboxing tem custo e armadilhas

Integer a = 127; Integer b = 127; a == b retorna true (cache do pool). Mas Integer a = 128; Integer b = 128; a == b retorna false. Sempre use .equals() para comparar objetos.

Arquitetura da JVM

Class Loader Subsystem

Carrega bytecode em memoria em tres fases: Loading (le .class), Linking (verifica, prepara, resolve simbolos), Initialization (executa blocos static initializer). Delegation model: Bootstrap → Extension → Application.

Runtime Data Areas

Heap: objetos e arrays — gerenciado pelo GC. Method Area: bytecode, constantes, metadados de classe (Metaspace no Java 8+). Stack: frames de chamada por thread. PC Register: ponteiro de instrucao por thread.

JIT Compiler

HotSpot detecta metodos chamados frequentemente ("hot spots") e os compila para codigo nativo. C1 compila rapido com otimizacoes leves; C2 (server) faz otimizacoes pesadas (inlining, loop unrolling, escape analysis).

Garbage Collector

Java 21 usa G1GC por padrao (equilibra latencia e throughput). Alternativas: ZGC (latencia < 1ms, Java 15+), Shenandoah (Red Hat), SerialGC (embeddings). Tuning: -Xmx2g -Xms512m -XX:+UseZGC.

Virtual Threads (Java 21)

Project Loom introduz threads gerenciadas pela JVM — nao pelo OS. Millhoes de threads com custo de memoria de KB cada. Compativel com java.util.concurrent e synchronized. Ideal para I/O-bound: servidores HTTP, acesso a banco.

JPMS — Module System

Java Platform Module System (Java 9). Modulos declaram requires e exports em module-info.java, eliminando o classpath caos. Habilita jlink para criar JREs customizados com apenas os modulos necessarios.

Roteiro de Evolucao Profissional em Java

Nivel 1 - Base solida

Domine sintaxe, orientacao a objetos, excecoes, Collections e testes com JUnit. O foco e ler codigo de outros projetos sem se perder em detalhes de framework.

Nivel 2 - Backend produtivo

Aprenda Spring Boot, SQL, JPA, cache e observabilidade. Aqui voce passa de "codigo que funciona" para servicos que escalam, monitoram erro e tem deploy previsivel.

Nivel 3 - Engenharia de plataforma

Atue em arquitetura, desempenho de JVM, seguranca e governanca de repositorio. O diferencial e decidir trade-offs de latencia, custo e produtividade para o time inteiro.
