Java Avancado

Generics com wildcards, Streams API para processamento de dados, Optional, lambdas e interfaces funcionais, Sealed Classes, Pattern Matching em switch e Virtual Threads do Project Loom.

A Streams API transformou Java de uma linguagem imperativa verbosa em uma plataforma com estilo funcional expressivo. Combinada com Optional e lambdas, permite pipelines de transformacao de dados legiveis e paralelizaveis com um simples .parallel().

Generics

Java — Classe generica, bounded types e wildcards
// Classe generica — T e um parametro de tipo (type parameter)
public class Caixa<T> {
  private T conteudo;

  public Caixa(T conteudo) { this.conteudo = conteudo; }
  public T abrir()          { return conteudo; }

  // Metodo que mapeia para outro tipo
  public <U> Caixa<U> transformar(Function<T, U> func) {
    return new Caixa<>(func.apply(conteudo));
  }
}

var caixaNome  = new Caixa<>("Java 21");
var caixaTamanho = caixaNome.transformar(String::length); // Caixa<Integer>

// Bounded type — T deve ser Comparable
public static <T extends Comparable<T>> T maximo(List<T> lista) {
  return lista.stream().max(Comparator.naturalOrder())
               .orElseThrow();
}

// Wildcards: tres sabores
// ? extends T — upper bound — leitura (produtor)
public double somarNumeros(List<? extends Number> nums) {
  return nums.stream().mapToDouble(Number::doubleValue).sum();
}

// ? super T — lower bound — escrita (consumidor)
public void adicionarNumeros(List<? super Integer> destino) {
  destino.add(42); // seguro — todo List<? super Integer> aceita Integer
}

// PECS: Producer Extends, Consumer Super
public static <T> void copiar(
    List<? extends T> fonte,
    List<? super T> destino) {
  destino.addAll(fonte);
}

Streams API

Java — Criacao, operacoes intermediarias e terminais
import java.util.stream.*;

// Fontes de Stream
Stream<String> s1 = Stream.of("a", "b", "c");
IntStream       s2 = IntStream.range(0, 10);        // 0..9
IntStream       s3 = IntStream.rangeClosed(1, 10); // 1..10
Stream<String> s4 = lista.stream();
Stream<String> s5 = lista.parallelStream();       // ForkJoinPool
Stream<String> s6 = Stream.generate(() -> "X").limit(5);
Stream<Integer> s7 = Stream.iterate(0, n -> n + 2).limit(5); // 0,2,4,6,8

// Pipeline de transformacao
List<String> resultado = Stream.of("banana", "abacate", "kiwi", "abacaxi")
  .filter(s -> s.startsWith("aba"))     // intermediaria — lazy
  .map(String::toUpperCase)              // intermediaria — lazy
  .sorted()                               // intermediaria — lazy
  .distinct()                             // intermediaria
  .limit(10)                              // intermediaria
  .toList();                              // terminal — dispara execucao (Java 16)

// Reducao
int soma = IntStream.rangeClosed(1, 100).sum();     // 5050
long qnt = lista.stream().filter(s -> s.length() > 3).count();
double media = lista.stream()
  .mapToInt(String::length).average().orElse(0);

// Collectors avancados
Map<boolean, List<String>> particionado =
  lista.stream().collect(Collectors.partitioningBy(s -> s.length() > 5));

Map<Integer, List<String>> grupoPorTamanho =
  lista.stream().collect(Collectors.groupingBy(String::length));

Map<Integer, long> contPorTamanho =
  lista.stream().collect(
    Collectors.groupingBy(String::length,
                          Collectors.counting()));

String frase = lista.stream().collect(Collectors.joining(", ", "[", "]"));

// flatMap — Stream de Streams → Stream plana
List<List<Integer>> aninhada = List.of(List.of(1, 2), List.of(3, 4));
List<Integer> plana = aninhada.stream()
  .flatMap(Collection::stream).toList(); // [1,2,3,4]

Optional — Eliminando NullPointerException

Java — Optional correto — criacao, encadeamento e extracoes
// Criacao
Optional<String> vazio      = Optional.empty();
Optional<String> presente  = Optional.of("Java");
Optional<String> nullable  = Optional.ofNullable(buscarNome()); // pode ser null

// Encadeamento funcional — sem if-null
Optional<Usuario> usuario = repositorio.buscarPorId(42L);

String email = usuario
  .map(Usuario::getEmail)
  .map(String::toLowerCase)
  .filter(e -> e.contains("@"))
  .orElse("sem-email@exemplo.com");

// orElseGet — lazy (funcao so executada se vazio)
String nome = usuario
  .map(Usuario::getNome)
  .orElseGet(() -> gerarNomeTemporario());

// orElseThrow — lanca excecao se vazio (Java 10+: sem argumento lanca NoSuchElement)
Usuario u = repositorio.buscarPorId(id)
  .orElseThrow(() -> new EntidadeNaoEncontradaException("Usuario", id));

// flatMap — quando o metodo ja retorna Optional
Optional<String> fone = usuario
  .flatMap(Usuario::getTelefone); // getTelefone() retorna Optional<String>

// ifPresentOrElse (Java 9)
usuario.ifPresentOrElse(
  u2 -> System.out.println("Bem-vindo, " + u2.getNome()),
  ()  -> System.out.println("Usuario nao encontrado")
);

// or (Java 9) — fallback para outro Optional
Optional<Usuario> resultado = usuario
  .or(() -> cacheLocal.buscar(id));

// ANTI-PADRAO: nunca use isPresent() + get() — use map/orElse
// if (opt.isPresent()) { return opt.get(); } // ruim — equivale a null check

Interfaces Funcionais e Lambdas

Java — @FunctionalInterface e method references
import java.util.function.*;

// As 4 interfaces funcionais principais
Supplier<LocalDate>    supplier = LocalDate::now;   // () -> T
Consumer<String>      consumer = System.out::println; // T -> void
Function<String, Integer> fn = String::length;       // T -> R
Predicate<Integer>    pred = n -> n > 0;               // T -> boolean

// Composicao de funcoes
Function<String, String> limpar    = String::trim;
Function<String, String> upper     = String::toUpperCase;
Function<String, String> limparUpper = limpar.andThen(upper);

// Composicao de Predicates
Predicate<String> naoVazio = s -> !s.isBlank();
Predicate<String> comprido = s -> s.length() > 5;
Predicate<String> valido   = naoVazio.and(comprido);

// @FunctionalInterface customizada
@FunctionalInterface
public interface Validador<T> {
  boolean validar(T valor);

  default Validador<T> e(Validador<T> outro) {
    return v -> this.validar(v) && outro.validar(v);
  }
}

Validador<String> emailValido =
  s -> s.contains("@") && s.contains(".");

// Closures — lambdas capturam variaveis efetivamente finais
String prefixo = "SR_";
Function<String, String> adicionar = nome -> prefixo + nome;
// prefixo = "OUTRO"; // erro de compilacao — efetivamente final

// Method references — 4 tipos
Function<String, Integer> r1 = Integer::parseInt;    // estatico
Consumer<String>          r2 = System.out::println;  // instancia especifica
Function<String, String>  r3 = String::toLowerCase;  // instancia arbitraria
Supplier<ArrayList>       r4 = ArrayList::new;         // construtor

Sealed Classes e Pattern Matching (Java 17-21)

Java 21 — Sealed, switch com pattern e guarded patterns
// Sealed class — lista fechada de subtipos (Java 17)
public sealed interface FormaGeometrica
  permits Circulo, Retangulo, Triangulo {}

public record Circulo(double raio)
  implements FormaGeometrica {}

public record Retangulo(double largura, double altura)
  implements FormaGeometrica {}

public record Triangulo(double base, double alturaT)
  implements FormaGeometrica {}

// Pattern matching em switch (Java 21 — GA)
// O compilador verifica exaustividade — sem default se todos os subtipos forem cobertos
public double calcularArea(FormaGeometrica f) {
  return switch (f) {
    case Circulo c    -> Math.PI * c.raio() * c.raio();
    case Retangulo r  -> r.largura() * r.altura();
    case Triangulo t  -> (t.base() * t.alturaT()) / 2;
  };
}

// Guarded patterns — when (Java 21)
public String descrever(FormaGeometrica f) {
  return switch (f) {
    case Circulo c when c.raio() > 100 -> "Circulo gigante";
    case Circulo c                       -> "Circulo r=" + c.raio();
    case Retangulo r when r.largura() == r.altura() -> "Quadrado";
    case Retangulo r -> "Retangulo %dx%d".formatted((int)r.largura(), (int)r.altura());
    case Triangulo t -> "Triangulo b=" + t.base();
  };
}

// Pattern matching instanceof (Java 16)
Object obj = obterValor();
if (obj instanceof String s && s.length() > 5) {
  System.out.println(s.toUpperCase()); // s ja e String, sem cast
}

Concorrencia e Virtual Threads

Java 21 — Virtual Threads (Project Loom)
// Thread virtual — custo de criacao ~microsegundos, nao KB de stack OS
var t = Thread.ofVirtual().start(() -> {
  System.out.println("Executando em thread virtual: " + Thread.currentThread());
});
t.join();

// Executor de threads virtuais — substitui pool
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
  List<Future<String>> futures = IntStream.range(0, 1000)
    .mapToObj(i -> executor.submit(() -> buscarDados(i)))
    .toList();
  // 1000 tarefas I/O-bound sem criar 1000 OS threads
}

// CompletableFuture — orquestracao assincrona
CompletableFuture<String> cf =
  CompletableFuture.supplyAsync(() -> buscarUsuario(1L))
    .thenApply(Usuario::getNome)
    .thenApply(String::toUpperCase)
    .exceptionally(ex -> "USUARIO_DESCONHECIDO");

String nome = cf.join(); // bloqueia ate completar

// combinar dois futures em paralelo
CompletableFuture<String> nomeF  = CompletableFuture.supplyAsync(() -> buscarNome(id));
CompletableFuture<String> emailF = CompletableFuture.supplyAsync(() -> buscarEmail(id));

CompletableFuture<String> combinado = nomeF.thenCombine(emailF,
  (nome2, email) -> nome2 + " <" + email + ">");

// StructuredTaskScope (previa Java 21 — JEP 453)
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
  var usuario  = scope.fork(() -> buscarUsuario(id));
  var pedidos  = scope.fork(() -> buscarPedidos(id));
  scope.join().throwIfFailed();
  return new PerfilCompleto(usuario.get(), pedidos.get());
}

Decisoes Avancadas em Producao

Streams vs loops

Streams melhoram legibilidade e composicao. Em trechos ultra-criticos de latencia, loops imperativos ainda podem ser mais previsiveis no perfil de CPU.

Virtual Threads vs reativo

Virtual Threads simplificam I/O bloqueante com modelo mental tradicional. Stack reativa ainda e valida para pipelines declarativos com backpressure extremo.

Optional e APIs publicas

Use Optional para retorno de ausencia de valor. Evite Optional em campos de entidade e serializacao de borda sem estrategia explicita de contrato.
