R — Ecossistema e Tooling

Ecossistema e Tooling R

CRAN, Bioconductor, RStudio IDE, gerenciamento de ambientes com renv e desenvolvimento de pacotes de qualidade.

O ecossistema de ferramentas R amadureceu enormemente na última década. renv resolveu a reprodutibilidade de ambientes; devtools e roxygen2 tornaram o desenvolvimento de pacotes acessível; testthat trouxe TDD para R; e GitHub Actions com r-lib/actions automatiza CI/CD de pacotes. Conhecer essas ferramentas é diferencial profissional.

CRAN e Bioconductor

O CRAN (Comprehensive R Archive Network) é o repositório central de pacotes R. Todo pacote submetido passa por um processo de revisão automatizado que verifica instalação em múltiplos sistemas operacionais e versões de R. Isso garante portabilidade excepcional.

pacotes.R — gerenciamento de pacotes
# Instalar do CRAN
install.packages("ggplot2")
install.packages(c("dplyr", "tidyr", "readr"))

# Instalar versão específica
remotes::install_version("ggplot2", version = "3.4.0")

# Instalar do GitHub (desenvolvimento)
remotes::install_github("tidyverse/ggplot2")

# Bioconductor
BiocManager::install("DESeq2")
BiocManager::valid()   # verifica consistência de versões

# Carregar e verificar
library("ggplot2")
packageVersion("ggplot2")   # "3.5.1"
citation("ggplot2")         # como citar o pacote

# Atualizar pacotes
update.packages(ask = FALSE)

# Listar instalados
installed.packages()[, c("Package", "Version")]

RStudio IDE

O RStudio é o IDE padrão para desenvolvimento R, com integração nativa de console, visualizador de dados, plots, help, Git e gerenciamento de projetos. Conhecer seus atalhos e funcionalidades avançadas aumenta consideravelmente a produtividade.

renv — Reprodutibilidade de Ambientes

renv (Reproducible Environments) resolve o problema clássico de R: "funciona na minha máquina mas não na sua". Cria um lockfile renv.lock que registra exatamente quais versões de pacotes estão instaladas, permitindo restaurar o ambiente em qualquer máquina.

renv-workflow.R
# Inicializar renv em um projeto
renv::init()
# Cria: renv.lock, renv/, .Rprofile com source("renv/activate.R")

# Instalar pacote e atualizar o lockfile
install.packages("ggplot2")   # instala na biblioteca isolada do projeto
renv::snapshot()              # registra estado atual em renv.lock

# Restaurar ambiente a partir do lockfile (em outra máquina / CI)
renv::restore()

# Verificar diferenças entre lockfile e estado atual
renv::status()

# Atualizar pacote e registrar
renv::update("dplyr")
renv::snapshot()

# Exemplo de renv.lock (fragmento JSON)
# {
#   "R": { "Version": "4.4.1", "Repositories": [...] },
#   "Packages": {

devtools e roxygen2 — Desenvolvimento de Pacotes

        

          
desenvolvimento-pacote.R

          
# Criar estrutura de pacote
usethis::create_package("meupacote")
# Cria: DESCRIPTION, NAMESPACE, R/, .gitignore, meupacote.Rproj

# Adicionar função documentada com roxygen2
# Arquivo: R/saudacao.R

#' Gera uma saudação personalizada
#'
#' @param nome Nome da pessoa a ser saudada
#' @param prefixo Prefixo do cumprimento (padrão: "Olá")
#' @return Uma string com a saudação
#' @export
#' @examples
#' saudacao("Ana")
#' saudacao("Beto", "Oi")
saudacao <- function(nome, prefixo = "Olá") {
  paste(prefixo, nome)
}

# Gerar documentação a partir dos comentários roxygen2
devtools::document()   # cria man/saudacao.Rd e atualiza NAMESPACE

# Workflow de desenvolvimento
devtools::load_all()   # carrega pacote sem instalar (rápido)
devtools::check()     # R CMD CHECK completo
devtools::install()   # instala o pacote localmente

# Adicionar dependência no DESCRIPTION
usethis::use_package("dplyr")           # Imports
usethis::use_package("ggplot2", "Suggests")  # Suggests

        

        

      


      

        
testthat — Testes Unitários

        

          
tests/testthat/test-saudacao.R

          
library("testthat")

# test_that() agrupa assertions relacionadas
test_that("saudacao retorna string correta", {
  expect_equal(saudacao("Ana"), "Olá Ana")
  expect_equal(saudacao("Beto", "Oi"), "Oi Beto")
})

test_that("saudacao aceita argumento por nome", {
  expect_equal(
    saudacao(nome = "Cris", prefixo = "Hey"),
    "Hey Cris"
  )
})

test_that("saudacao lança erro com entrada inválida", {
  expect_error(saudacao(NULL))
  expect_error(saudacao(123))
})

# Snapshots — testa saída textual complexa
test_that("formatacao de relatorio", {
  expect_snapshot(print_resumo(dados_teste))
})

# Executar testes
devtools::test()   # executa todos os testes do pacote
testthat::test_file("tests/testthat/test-saudacao.R")  # arquivo específico

        

        

      


      

        
Qualidade de Código: lintr e styler

        

          
qualidade.R

          
# lintr — análise estática (encontra problemas sem executar)
library("lintr")
lintr::lint("R/saudacao.R")     # arquivo único
lintr::lint_package()         # pacote inteiro
# Verifica: comprimento de linha, espaçamento, nomes, complexidade

# Configurar lintr (.lintr no root do projeto)
# linters: linters_with_defaults(
#   line_length_linter(120),
#   object_name_linter("snake_case")
# )

# styler — formata código automaticamente
library("styler")
styler::style_file("R/saudacao.R")   # formata um arquivo
styler::style_pkg()               # formata o pacote inteiro
styler::style_dir("R/")           # formata um diretório

# pkgdown — gerar site de documentação do pacote
library("pkgdown")
pkgdown::build_site()   # gera docs/ com site estático completo
# Configurar em _pkgdown.yml para personalizar estrutura
