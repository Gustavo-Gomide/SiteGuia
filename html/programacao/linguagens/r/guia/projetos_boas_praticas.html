Estrutura de Pacote R

Um pacote R é a unidade fundamental de compartilhamento de código. A estrutura de diretórios é padronizada; o R CMD CHECK valida cada componente antes da publicação.

Estilo de Código — Tidyverse Style Guide

O Tidyverse Style Guide (style.tidyverse.org) é o padrão de facto para código R. Seguir um estilo consistente melhora a leitura e facilita revisão de código em equipe.

RegraCorretoIncorreto
Nomes de objetosminha_variavel (snake_case)minhaVariavel, MnhVar
Nomes de funçõescalcular_media()CalcMedia(), calc.media()
Espaços em operadoresx <- 1 + 2x<-1+2
Comprimento de linhaMáximo 80 caracteresLinhas longas dificultem revisão
Vírgulasc(1, 2, 3)c(1,2,3)
Chavesif (cond) { na mesma linhaif (cond)
{ na linha seguinte
TRUE/FALSETRUE, FALSET, F (perigoso — podem ser sobrescritos)

Nunca use T e F como atalhos para TRUE e FALSE em código de produção. Eles são variáveis comuns que podem ser acidentalmente sobrescritas: T <- 42 é R válido e silenciosamente causará bugs.

Documentação com roxygen2

R/calcular_imc.R — documentação completa
#' Calcula o Índice de Massa Corporal (IMC)
#'
#' Calcula o IMC a partir de peso e altura, com classificação opcional
#' segundo os critérios da OMS.
#'
#' @param peso Peso em quilogramas (numérico, escalar ou vetor)
#' @param altura Altura em metros (numérico, escalar ou vetor)
#' @param classificar Lógico. Se `TRUE`, retorna data frame com IMC e
#'   classificação OMS. Padrão: `FALSE`.
#'
#' @return Se `classificar = FALSE`, vetor numérico com valores de IMC.
#'   Se `classificar = TRUE`, tibble com colunas `imc` e `categoria`.
#'
#' @export
#' @seealso [calcular_risco()] para avaliação de risco cardiovascular
#'
#' @examples
#' calcular_imc(70, 1.75)         # 22.86
#' calcular_imc(c(70, 90), c(1.75, 1.80))
#' calcular_imc(70, 1.75, classificar = TRUE)
calcular_imc <- function(peso, altura, classificar = FALSE) {
  stopifnot(is.numeric(peso), is.numeric(altura), all(altura > 0))
  imc <- peso / altura^2
  if (!classificar) return(imc)
  tibble::tibble(
    imc      = imc,
    categoria = cut(imc, breaks = c(-Inf, 18.5, 25, 30, Inf),
                    labels = c("Abaixo", "Normal", "Sobrepeso", "Obeso"))
  )
}

Relatórios Reprodutíveis com Quarto

Quarto é a plataforma de publicação científica de nova geração da Posit, sucessor do R Markdown. Documentos .qmd intercalam código R (ou Python, Julia) com texto Markdown, gerando PDFs, HTMLs e apresentações reprodutíveis.

relatorio.qmd — estrutura básica Quarto
---
title: "Análise de Vendas 2024"
author: "Equipe de Data Science"
date: last-modified
format:
  html:
    toc: true
    code-fold: true
    theme: flatly
  pdf:
    toc: true
execute:
  echo: true
  warning: false
  message: false
---

CI/CD com GitHub Actions

.github/workflows/R-CMD-check.yaml
on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

name: R-CMD-check

jobs:
  R-CMD-check:
    runs-on: ${{ matrix.config.os }}
    name: ${{ matrix.config.os }} (R ${{ matrix.config.r }})

    strategy:
      fail-fast: false
      matrix:
        config:
          - {os: ubuntu-latest, r: 'release'}
          - {os: windows-latest, r: 'release'}
          - {os: macOS-latest, r: 'release'}
          - {os: ubuntu-latest, r: 'devel'}

    steps:
      - uses: actions/checkout@v4

      - uses: r-lib/actions/setup-pandoc@v2

      - uses: r-lib/actions/setup-r@v2
        with:
          r-version: ${{ matrix.config.r }}
          use-public-rspm: true

      - uses: r-lib/actions/setup-r-dependencies@v2
        with:
          extra-packages: any::rcmdcheck
          needs: check

      - uses: r-lib/actions/check-r-package@v2
        with:
          upload-snapshots: true

Publicando no CRAN

O processo de submissão ao CRAN exige que o pacote passe no R CMD CHECK sem warnings nem erros em múltiplas plataformas. Use o rhub ou o win-builder para verificar em ambientes que você não tem localmente.

publicacao.R — checklist pré-CRAN
# 1. Verificar localmente
devtools::check()             # zero ERRORs, zero WARNINGs

# 2. Verificar no win-builder (Windows + R-devel)
devtools::check_win_devel()   # envia para cran-win-builder.r-project.org

# 3. Verificar no r-hub (múltiplas plataformas)
rhub::rc_submit()   # submete para rhub v2 via GitHub Actions

# 4. Atualizar NEWS.md e DESCRIPTION (version bump)
usethis::use_version("minor")   # 0.1.0 -> 0.2.0

# 5. Gerar documentação e site
devtools::document()
pkgdown::build_site()

# 6. Submeter ao CRAN
devtools::submit_cran()
# Redireciona para formulário web em https://cran.r-project.org/submit.html
# O mantenedor receberá email de confirmação
# Tempo de revisão: 2–5 dias úteis
