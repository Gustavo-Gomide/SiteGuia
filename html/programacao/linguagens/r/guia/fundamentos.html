R — Fundamentos

Fundamentos de R

Estruturas de dados essenciais, funções, controle de fluxo e programação funcional com a família apply.

Em R, tudo é um objeto e todo objeto tem uma classe. Compreender as cinco estruturas de dados fundamentais — vetores atômicos, matrizes, data frames, listas e fatores — é o alicerce para qualquer análise. A vetorização automática e a família apply substituem laços explícitos na maior parte dos casos.

Vetores Atômicos

Um vetor atômico é a estrutura de dados mais básica em R — uma sequência ordenada de elementos do mesmo tipo. Diferente de arrays em outras linguagens, um "escalar" em R é simplesmente um vetor de comprimento 1.

vetores.R — criação, indexação e operações
# Criação com c() — combine
inteiros   <- c(1L, 2L, 3L)
doubles    <- c(1.5, 2.7, 3.14)
strings    <- c("alfa", "beta", "gama")
logicos    <- c(TRUE, FALSE, TRUE)

# Sequências
1:10                          # 1 2 3 ... 10
seq(0, 1, by = 0.1)           # 0.0 0.1 0.2 ... 1.0
rep("x", times = 5)           # "x" "x" "x" "x" "x"
rep(c(1,2), each = 3)          # 1 1 1 2 2 2

# Indexação — base-1, múltiplos estilos
x <- c(10, 20, 30, 40, 50)
x[2]                          # 20 — positivo
x[-1]                         # 20 30 40 50 — exclui primeiro
x[c(1, 3, 5)]                 # 10 30 50 — índice múltiplo
x[x > 25]                     # 30 40 50 — lógico
x[c(TRUE,FALSE,TRUE,FALSE,TRUE)] # 10 30 50 — máscara lógica

# Nomes em vetores
notas <- c(Ana = 9.2, Beto = 7.8, Cris = 8.5)
notas["Ana"]                  # 9.2
names(notas)                  # "Ana" "Beto" "Cris"

# Coerção de tipos — hierarquia: logical < integer < double < complex < character
c(TRUE, 1L, 2.5)             # 1.0 1.0 2.5 — coerce para double
c(1, "dois")                  # "1" "dois" — coerce para character
as.integer("42")              # 42L — conversão explícita
as.logical(0)                 # FALSE

Matrizes e Arrays

Uma matriz é um vetor com dimensões. Internamente, R armazena matrizes em ordem coluna-maior (column-major), o que tem implicações de performance ao iterar.

matrizes.R
# Criação de matriz — preenchimento por coluna (padrão)
m <- matrix(1:12, nrow = 3, ncol = 4)
#      [,1] [,2] [,3] [,4]
# [1,]    1    4    7   10
# [2,]    2    5    8   11
# [3,]    3    6    9   12

# Indexação [linha, coluna]
m[2, 3]            # 8 — elemento na linha 2, coluna 3
m[1, ]             # 1 4 7 10 — toda a linha 1
m[, 2]             # 4 5 6 — toda a coluna 2

# Operações matriciais
A <- matrix(c(1,2,3,4), nrow = 2)
B <- matrix(c(5,6,7,8), nrow = 2)
A %*% B            # multiplicação matricial
t(A)               # transposta
solve(A)           # inversa
det(A)             # determinante: -2

# apply em matrizes — evita laços
apply(m, 1, sum)   # soma por linha (MARGIN=1)
apply(m, 2, mean)  # média por coluna (MARGIN=2)

# Arrays 3D (ou N-D)
arr <- array(1:24, dim = c(2, 3, 4))
arr[1, 2, 3]       # elemento [1,2,3]

Data Frames

O data frame é a estrutura de dados central de R para análise tabular. É uma lista de vetores de mesmo comprimento, onde cada vetor é uma coluna. Pense nele como uma planilha ou tabela SQL em memória.

data-frames.R
# Criação manual
df <- data.frame(
  nome  = c("Ana", "Beto", "Cris"),
  idade = c(28L, 34L, 22L),
  nota  = c(9.2, 7.8, 8.5),
  ativo = c(TRUE, TRUE, FALSE)
)

# Inspecção
str(df)              # estrutura completa
head(df, 2)          # primeiras 2 linhas
summary(df)          # estatísticas descritivas
nrow(df); ncol(df)   # 3 linhas, 4 colunas

# Acesso a colunas — três formas equivalentes
df$nome              # [1] "Ana" "Beto" "Cris"
df[, "nome"]         # mesmo resultado
df[["nome"]]         # vetor, sem metadados de data frame

# Indexação de linhas
df[df$idade > 25, ]  # filtra maior de 25 anos
df[2:3, c("nome", "nota")]  # linhas 2-3, colunas específicas

# Adicionar coluna
df$aprovado <- df$nota >= 8.0

# Combinar data frames
rbind(df, data.frame(nome="Davi", idade=30L, nota=7.1, ativo=TRUE, aprovado=FALSE))
merge(df, outro_df, by = "nome")  # equivalente a JOIN

Listas

Uma lista é um vetor que pode conter elementos de tipos diferentes e de tamanhos arbitrários, incluindo outras listas. São a estrutura de dados mais flexível de R e a base interna do data frame.

listas.R
# Lista heterogênea
pessoa <- list(
  nome   = "Ana Silva",
  idade  = 28L,
  notas  = c(8.5, 9.0, 7.5),
  ativo  = TRUE,
  endereco = list(cidade = "São Paulo", uf = "SP")  # lista aninhada
)

# Acesso — [[ ]] extrai o elemento; [ ] mantém a lista
pessoa[["nome"]]           # "Ana Silva" — elemento
pessoa["nome"]             # lista com 1 elemento nominado
pessoa$endereco$cidade    # "São Paulo" — acesso encadeado

# Modificar
pessoa$email <- "ana@exemplo.com"
pessoa[["notas"]][2] <- 9.5

# Iterar com lapply / for
lapply(pessoa, class)         # classe de cada elemento
lengths(pessoa)               # comprimento de cada elemento

Fatores

Fatores representam dados categóricos. Internamente são inteiros com um atributo levels (os rótulos possíveis). São essenciais para modelagem estatística — regressões e gráficos tratam fatores de forma especial.

fatores.R
# Fator simples (não-ordenado)
genero <- factor(c("M", "F", "M", "F", "M"))
levels(genero)           # "F" "M" — ordem alfabética por padrão
nlevels(genero)          # 2
table(genero)            # contagem: F=2, M=3

# Fator ordenado — representa escala ordinal
escolaridade <- factor(
  c("Médio", "Superior", "Fundamental", "Superior"),
  levels  = c("Fundamental", "Médio", "Superior"),
  ordered = TRUE
)
escolaridade[1] < escolaridade[2]   # TRUE — Médio < Superior

# forcats (tidyverse) — reordenação de fatores
# library(forcats)
forcats::fct_infreq(genero)   # ordena por frequência decrescente
forcats::fct_rev(genero)      # inverte ordem dos níveis

Funções em R

Funções são cidadãos de primeira classe em R — podem ser armazenadas em variáveis, passadas como argumentos e retornadas de outras funções. O mecanismo de escopo é lexical: uma função "vê" as variáveis do ambiente onde foi definida, não de onde foi chamada.

funcoes.R
# Definição básica
saudacao <- function(nome, prefixo = "Olá") {
  paste(prefixo, nome)
}
saudacao("Ana")           # "Olá Ana"
saudacao("Beto", "Oi")   # "Oi Beto"
saudacao(prefixo="Hey", nome="Cris")  # argumentos nomeados

# Retorno implícito — última expressão avaliada
quadrado <- function(x) x ^ 2
quadrado(7)              # 49

# ... (dot-dot-dot) — argumentos variádicos
minha_soma <- function(...) sum(c(...))
minha_soma(1, 2, 3, 4)   # 10

# Passagem para outra função via ...
minha_media <- function(x, ...) mean(x, ...)
minha_media(c(1,NA,3), na.rm = TRUE)   # 2 — na.rm repassado para mean()

# Escopo lexical — a função vê o ambiente de criação
potencia <- function(exp) {
  function(base) base ^ exp  # retorna uma função (closure)
}
quadrado2 <- potencia(2)
cubo      <- potencia(3)
quadrado2(5)   # 25
cubo(4)        # 64

# Funções anônimas — sintaxe tradicional e nova (R 4.1+)
sapply(1:5, function(x) x ^ 2)   # 1 4 9 16 25
sapply(1:5, \(x) x ^ 2)          # idem — sintaxe \(x) novo em R 4.1

Controle de Fluxo

controle.R
# if / else if / else
classificar <- function(nota) {
  if (nota >= 9.0) {
    "Excelente"
  } else if (nota >= 7.0) {
    "Aprovado"
  } else {
    "Reprovado"
  }
}

# ifelse() vetorizado — diferente do if/else escalar
notas <- c(6.5, 8.2, 9.1, 5.0)
ifelse(notas >= 7.0, "Aprovado", "Reprovado")

# switch() — múltiplos casos
switch("b",
  a = "Alfa",
  b = "Beta",  # retorna "Beta"
  c = "Gama"
)

# for — itera sobre qualquer vetor/lista
for (i in 1:5) {
  cat(i, " ")
}

# while e repeat
x <- 1
while (x < 10) {
  x <- x * 2
}
# x = 16

repeat {
  x <- x - 3
  if (x < 0) break
}

# next — equivalente ao continue
for (i in 1:10) {
  if (i %% 2 == 0) next   # pula pares
  cat(i, "")
}
# 1 3 5 7 9

Prefira vetorização a laços: em R, laços for têm overhead significativo. Sempre que possível, use funções vetorizadas (ifelse, cumsum, diff) ou a família apply. Laços fazem sentido quando há dependência de estado entre iterações.

Família apply

A família apply substitui laços explícitos para operações repetidas sobre estruturas de dados. Cada variante é otimizada para um caso de uso diferente.

FunçãoInputOutputUso Típico
apply()Matriz/arrayVetor ou listaLinhas/colunas de matriz
lapply()Lista/vetorListaAplicação sobre listas, sempre retorna lista
sapply()Lista/vetorVetor/matriz simplificadoAplicação onde simplificação é desejada
vapply()Lista/vetorTipo declaradoComo sapply, mas com tipo de retorno garantido
tapply()Vetor + fatorArrayEstatísticas por grupo (tabela de aplicação)
mapply()Múltiplos vetoresLista/vetorMúltiplos argumentos em paralelo
apply-family.R
# lapply — sempre retorna lista
resultado <- lapply(list(1:5, 1:10), mean)
# list(3, 5.5)

# sapply — simplifica para vetor quando possível
sapply(1:5, function(x) x^2)
# [1]  1  4  9 16 25

# vapply — type-safe: FUN.VALUE declara o tipo de retorno esperado
vapply(c("olá", "mundo"), nchar, integer(1))
#  olá mundo
#    3     5

# tapply — estatística por grupo
vendas <- c(100, 200, 150, 300, 250)
regiao <- c("Sul", "Norte", "Sul", "Norte", "Sul")
tapply(vendas, regiao, mean)
# Norte   Sul
#   250   167

# mapply — múltiplos vetores em paralelo
mapply(function(x, y) x + y, 1:4, 10:13)
# [1] 11 13 15 17

Importação de Dados

importacao.R
# Base R — CSV
df <- read.csv("dados.csv", stringsAsFactors = FALSE)
df <- read.csv2("dados.csv")   # separador ";" e decimal ","

# readr (tidyverse) — mais rápido, mensagens de parsing, tibble
# library(readr)
df <- readr::read_csv("dados.csv")    # separador vírgula
df <- readr::read_csv2("dados.csv")   # separador ponto-e-vírgula
df <- readr::read_tsv("dados.txt")    # separador tab
df <- readr::read_delim("dados.txt", delim = "|")

# Excel
# library(readxl)
df <- readxl::read_excel("planilha.xlsx", sheet = 1)

# RDS e RData — formatos nativos R (binário, preservam atributos)
saveRDS(df, "dados.rds")
df2 <- readRDS("dados.rds")

# feather / Parquet — formatos de alta performance e interoperabilidade
# library(arrow)
arrow::write_parquet(df, "dados.parquet")
df3 <- arrow::read_parquet("dados.parquet")
