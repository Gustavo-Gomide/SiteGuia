R — Bibliotecas e Frameworks

Bibliotecas e Frameworks R

O ecossistema de pacotes que transformou R na plataforma principal de ciência de dados e bioinformática.

Com mais de 21 000 pacotes no CRAN e 2 000 no Bioconductor, o ecossistema R cobre desde manipulação de tabelas a modelos de machine learning, de apps web reativos a análise de genomas. O Tidyverse unificou a API de manipulação de dados; o data.table fornece performance de nível C; e o Tidymodels padronizou o fluxo de modelagem preditiva.

Tidyverse

O Tidyverse é uma coleção de pacotes R desenvolvidos por Hadley Wickham e colaboradores na Posit (ex-RStudio), com uma filosofia comum: dados tidy (uma observação por linha, uma variável por coluna), pipes para composição e APIs consistentes. Um único library(tidyverse) carrega os oito pacotes principais.

tidyverse-pipeline.R — análise completa com pipe
library("tidyverse")

# Pipeline completo de análise
resultado <- read_csv("vendas.csv") |>
  filter(!is.na(valor), regiao != "Desconhecida") |>
  mutate(
    data      = ymd(data),
    trimestre = quarter(data),
    valor_k   = valor / 1000
  ) |>
  group_by(regiao, trimestre) |>
  summarise(
    total    = sum(valor_k),
    media    = mean(valor_k),
    n_vendas = n(),
    .groups  = "drop"
  ) |>
  arrange(desc(total))

# ggplot2 — gráfico publicável em uma pipeline
resultado |>
  ggplot(aes(x = factor(trimestre), y = total, fill = regiao)) +
  geom_col(position = "dodge") +
  scale_fill_brewer(palette = "Set2") +
  labs(
    title    = "Vendas por Trimestre e Região",
    subtitle = "Em R$ mil",
    x        = "Trimestre", y = "Total (R$ k)",
    fill     = "Região"
  ) +
  theme_minimal()

# tidyr — pivot para análise matricial
resultado |>
  select(regiao, trimestre, total) |>
  pivot_wider(names_from = trimestre, values_from = total)

data.table — Alta Performance

data.table é uma extensão do data.frame nativo com performance de nível C para operações sobre grandes conjuntos de dados. Suporta tabelas de bilhões de linhas em memória RAM — uma área onde o dplyr convencional teria dificuldades. A sintaxe DT[i, j, by] é compacta e muito expressiva.

data-table.R
library("data.table")

# Criar data.table (ou converter)
dt <- data.table(
  id     = 1:1e6,
  grupo  = sample(c("A","B","C"), 1e6, replace = TRUE),
  valor  = rnorm(1e6)
)
setDT(df_existente)   # converte data.frame NO LUGAR (sem cópia)

# Sintaxe DT[i, j, by]
# i = filtro de linhas, j = expressão de colunas, by = agrupamento
dt[valor > 0, .(mean(valor), .N), by = grupo]
#    grupo        V1      N
# 1:     A  0.7978...  250089

# Modificação POR REFERÊNCIA — sem cópia (eficiente)
dt[, valor_abs := abs(valor)]           # nova coluna
dt[grupo == "A", nota := "excelente"]  # condicional

# Chaves e joins rápidos
setkey(dt, grupo)                        # ordena e indexa por grupo
dt["A"]                                 # join ultra-rápido por chave

dt2 <- data.table(grupo = c("A","B"), descricao = c("Alta","Baixa"))
setkey(dt2, grupo)
dt[dt2]   # inner join automático por chaves

# Leitura de CSV de alta performance
grande <- fread("arquivo-gigante.csv")   # multi-thread, detecta tipos

Tidymodels — Machine Learning

tidymodels é um meta-pacote que unifica o fluxo de machine learning em R com a mesma filosofia tidy. Seus componentes principais são: recipes (pré-processamento), parsnip (interface unificada de modelos), workflows (une recipe + modelo), tune (hiperparâmetros) e rsample (cross-validation).

tidymodels-workflow.R
library("tidymodels")

# 1. Split treino/teste
splits <- initial_split(mtcars, prop = 0.8)
treino <- training(splits)
teste  <- testing(splits)

# 2. Receita de pré-processamento
receita <- recipe(mpg ~ ., data = treino) |>
  step_normalize(all_numeric_predictors()) |>
  step_nzv(all_predictors())   # remove variância próxima de zero

# 3. Especificação do modelo com parsnip
modelo_rf <- rand_forest(
  trees = tune(),   # será tunado
  mtry  = tune()
) |>
  set_engine("ranger") |>
  set_mode("regression")

# 4. Workflow = receita + modelo
wf <- workflow() |>
  add_recipe(receita) |>
  add_model(modelo_rf)

# 5. Tuning com cross-validation
cv_folds <- vfold_cv(treino, v = 5)
grid <- tune_grid(wf, resamples = cv_folds, grid = 10)

# 6. Selecionar melhor e finalizar
melhor <- select_best(grid, metric = "rmse")
wf_final <- finalize_workflow(wf, melhor)
fit_final <- last_fit(wf_final, splits)

# 7. Métricas no conjunto de teste
collect_metrics(fit_final)

Bioconductor — Bioinformática

Bioconductor (bioconductor.org) é o repositório especializado em ferramentas computacionais para bioinformática. Fundado em 2001 por Robert Gentleman, abriga mais de 2 000 pacotes para análise de RNA-seq, genômica, epigenômica e proteômica.

bioconductor.R — instalação e exemplo RNA-seq
# Instalação via BiocManager
if (!requireNamespace("BiocManager", quietly = TRUE))
  install.packages("BiocManager")
BiocManager::install(c("DESeq2", "GenomicRanges", "Biostrings"))

# DESeq2 — expressão diferencial de genes (RNA-seq)
library("DESeq2")

# Criar objeto DESeqDataSet
dds <- DESeqDataSetFromMatrix(
  countData = counts_matrix,   # matriz de contagens (genes x amostras)
  colData   = amostra_meta,    # metadados das amostras
  design    = ~ condicao       # fórmula do modelo
)

# Normalização e análise
dds <- DESeq(dds)
res <- results(dds, contrast = c("condicao", "tratamento", "controle"))

# Genes diferencialmente expressos
sig <- as.data.frame(res[res$padj < 0.05 & !is.na(res$padj), ])
plotMA(res, main = "Expressão Diferencial")   # volcano-like plot

Shiny — Apps Web Reativos

Shiny permite criar aplicações web interativas diretamente em R, sem escrever JavaScript. A UI é declarativa; o servidor é reativo. Qualquer análise R pode se tornar um app acessível por navegador.

app.R — Shiny mínimo funcional
library("shiny")

ui <- fluidPage(
  titlePanel("Histograma Interativo"),
  sidebarLayout(
    sidebarPanel(
      sliderInput("bins", "Número de bins:", min = 5, max = 100, value = 30),
      selectInput("variavel", "Variável:", choices = names(mtcars))
    ),
    mainPanel(plotOutput("histograma"))
  )
)

server <- function(input, output, session) {
  output$histograma <- renderPlot({
    hist(
      mtcars[[input$variavel]],
      breaks = input$bins,
      col    = "steelblue",
      main   = paste("Histograma de", input$variavel),
      xlab   = input$variavel
    )
  })
}

shinyApp(ui, server)

# plotly — converte ggplot2 em gráfico interativo HTML
library("plotly")
p <- ggplot(mtcars, aes(wt, mpg, color = factor(cyl))) +
  geom_point(size = 3)
ggplotly(p)   # converte para Plotly.js interativo

# plotly nativa
plot_ly(
  data   = mtcars,
  x      = ~wt, y = ~mpg,
  color  = ~factor(cyl),
  type   = "scatter", mode = "markers",
  text   = ~rownames(mtcars),
  hoverinfo = "text+x+y"
) |>
  layout(title = "Peso vs Consumo")

# highcharter — wrapper de Highcharts.js
library("highcharter")
hchart(mtcars, "scatter", hcaes(x = wt, y = mpg, group = factor(cyl))) |>
  hc_title(text = "Peso vs Consumo") |>
  hc_tooltip(pointFormat = "mpg: {point.y}")

# forecast — pacote de referência para séries temporais
library("forecast")
ts_vendas <- ts(vendas_mensais, start = c(2020, 1), frequency = 12)

autoplot(ts_vendas)           # gráfico automático
modelo <- auto.arima(ts_vendas)  # seleciona ARIMA automaticamente
previsao <- forecast(modelo, h = 12)   # 12 períodos à frente
autoplot(previsao)
accuracy(modelo)              # MAE, RMSE, MAPE

# prophet (Meta) — séries com tendências e sazonalidades
library("prophet")
df_prophet <- data.frame(
  ds = seq.Date(as.Date("2020-01-01"), by = "month", length.out = 48),
  y  = vendas_mensais
)
m <- prophet(df_prophet)
futuro <- make_future_dataframe(m, periods = 12, freq = "month")
prev <- predict(m, futuro)
plot(m, prev)
prophet_plot_components(m, prev)   # tendência + sazonalidade
