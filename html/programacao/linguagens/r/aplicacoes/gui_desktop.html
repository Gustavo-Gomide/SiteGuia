R para GUI Desktop

R para Interfaces Gráficas Desktop

Shiny como app local, empacotamento com electricShine/Electron, bindings GTK+ com RGtk2, tcltk nativo da instalação base e gWidgets2 portável.

Criar GUIs nativas não é o ponto forte de R — a linguagem foi projetada para análise interativa, não para software de desktop de uso geral. Contudo, há casos legítimos onde um analista precisa distribuir uma ferramenta de análise para usuários não-técnicos. As abordagens van da mais simples (Shiny local) à mais robusta (Electron wrapper).

Shiny como Aplicação Desktop

A abordagem mais pragmática: executar Shiny localmente com shiny::runApp() e abrir automaticamente no browser padrão. Para o usuário final, parece um aplicativo — mas é apenas uma página web servida em localhost.

launch.R — inicializador de app desktop
library("shiny")

# Detectar porta livre dinamicamente
porta_livre <- function() {
  con <- serverSocket(0)
  on.exit(close(con))
  sockopt(con)$port
}

# Abrir no browser assim que Shiny estiver pronto
shinyApp(
  ui     = source("ui.R", local = TRUE)$value,
  server = source("server.R", local = TRUE)$value,
  options = list(
    port    = porta_livre(),
    launch.browser = TRUE,     # abre o browser automaticamente
    quiet   = TRUE
  )
)
launch.bat — launcher para Windows (duplo-clique)
@echo off
:: Launcher para Windows — empacote com o R portátil
SET R_HOME=%~dp0R
SET PATH=%R_HOME%\bin;%PATH%

Rscript.exe "%~dp0launch.R"
if errorlevel 1 (
  echo Erro ao iniciar a aplicacao. Verifique os logs.
  pause
)

electricShine — Empacotamento com Electron

electricShine empacota uma aplicação Shiny junto com um runtime R portátil em um executável nativo usando Electron — o mesmo framework usado pelo VS Code e Slack. O resultado é um .exe (Windows) ou .dmg (macOS) que o usuário instala sem precisar ter R.

build-electron.R
# install.packages("electricShine")
library("electricShine")

electrify(
  app_name       = "AnáliseDados",
  product_name   = "Análise de Dados Corporativos",
  app_dir        = "./meu_shiny_app",   # pasta com app.R ou ui.R + server.R
  cran_like_url  = "https://cran.rstudio.com",
  r_version      = "4.4.1",
  build_path     = "./dist",
  package_json   = list(
    author      = "Equipe Análise",
    version     = "1.0.0",
    description = "Dashboard analítico para equipe comercial"
  )
)

# Resultado: ./dist/AnáliseDados-win32-x64/ com AnáliseDados.exe

electricShine gera instaladores grandes (200–600 MB) porque inclui R portátil completo. Para ferramentas internas corporativas isso é aceitável. Para distribuição pública, considere Shiny deploiado em servidor ou shinylive (WebAssembly).

tcltk — GUI Nativa Incluída no R Base

tcltk é parte da instalação padrão de R — sem instalar pacotes adicionais. Baseado no toolkit Tcl/Tk, cria janelas nativas mas com visual dos anos 2000. Útil para scripts simples que precisam de mínima interação do usuário.

tcltk-dialogo.R — seletor de arquivo com GUI
library("tcltk")

# Janela simples de seleção de arquivo
arquivo <- tkgetOpenFile(
  title      = "Selecione o arquivo CSV",
  filetypes  = list(c("Arquivos CSV", ".csv"),
                    c("Todos os arquivos", "*"))
)

if (nchar(arquivo) == 0) stop("Nenhum arquivo selecionado")

# Janela com input simples
janela <- tktoplevel()
tkwm.title(janela, "Configurar Análise")

# Label + Entry
tklabel(janela, text = "Número de clusters:") |> tkgrid()
k_var <- tclVar("3")
tkentry(janela, textvariable = k_var) |> tkgrid()

# Botão OK
ok_fn <- function() {
  k <<- as.integer(tclvalue(k_var))
  tkdestroy(janela)
}
tkbutton(janela, text = "OK", command = ok_fn) |> tkgrid()
tkwait.window(janela)

message("Arquivo: ", arquivo, " | k = ", k)

gWidgets2 — Toolkit Abstrato Portável

gWidgets2 abstrai o toolkit subjacente (RGtk2, tcltk). O mesmo código funciona com diferentes backends de GUI sem modificação.

gwidgets2-app.R
# Requer: install.packages(c("gWidgets2", "gWidgets2tcltk"))
library("gWidgets2")
options(guiToolkit = "tcltk")

# Janela principal
w <- gwindow("Calculadora Estatística", width = 400, height = 300)
g <- ggroup(container = w, horizontal = FALSE)

# Widgets
glabel("Insira os valores (separados por vírgula):", container = g)
entrada    <- gedit("", container = g)
resultado  <- glabel("Resultado aqui...", container = g)

# Botões em linha
g_botoes <- ggroup(container = g, horizontal = TRUE)
gbutton("Média", container = g_botoes, handler = function(...) {
  vals <- as.numeric(strsplit(svalue(entrada), ",")[[1]])
  svalue(resultado) <- paste("Média:", round(mean(vals), 4))
})
gbutton("Desvio Padrão", container = g_botoes, handler = function(...) {
  vals <- as.numeric(strsplit(svalue(entrada), ",")[[1]])
  svalue(resultado) <- paste("DP:", round(sd(vals), 4))
})

Quarto — Relatórios como "Aplicação"

Para muitos casos de uso, o que o usuário precisa não é uma GUI interativa mas sim um relatório HTML/PDF gerado automaticamente com inputs parametrizados. Quarto com parâmetros é frequentemente a solução mais adequada para distribuir análises para usuários não-técnicos.

relatorio.qmd — com parâmetros interativos
---
title: "Relatório de Vendas"
format:
  html:
    theme: cosmo
    toc: true
  pdf:
    toc: false
params:
  regiao: "Sul"
  ano: 2024
execute:
  echo: false
  warning: false
---

```{r}
#| label: setup
library(tidyverse)
dados <- read_csv("vendas.csv") |>
  filter(regiao == params$regiao, lubridate::year(data) == params$ano)
```

## Vendas em `r params$regiao` — `r params$ano`

Total de registros: `r nrow(dados)`

```{r}
#| label: grafico
#| fig-cap: "Evolução mensal de vendas"
dados |>
  count(mes = lubridate::floor_date(data, "month"), wt = valor) |>
  ggplot(aes(mes, n)) +
  geom_col(fill = "steelblue") +
  labs(x = NULL, y = "Valor Total (R$)")
```
Terminal — renderizar com parâmetros
# Renderizar para PDF com parâmetros específicos
quarto render relatorio.qmd -P regiao:Nordeste -P ano:2023 --to pdf

# Loop para gerar relatório para cada região
Rscript -e "
regioes <- c('Norte','Nordeste','Sul','Sudeste','Centro-Oeste')
for (r in regioes) {
  quarto::quarto_render('relatorio.qmd',
    output_file = paste0('relatorio-', r, '.pdf'),
    execute_params = list(regiao = r, ano = 2024))
}"
