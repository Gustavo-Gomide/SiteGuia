R para Jogos e Simulações

gganimate para visualizações animadas, Shiny para simulações interativas, grid para gráficos customizados frame a frame, e Monte Carlo para jogos probabilísticos.

R não foi projetado para jogos comerciais — sem motor de renderização em tempo real, sem suporte a entrada de teclado/mouse em loop, sem física. Porém, é uma plataforma excelente para simulações estatísticas, visualizações animadas e jogos probabilísticos educacionais. Cientistas de dados usam R para simular processos estocásticos, ensinar probabilidade e criar infográficos de dados animados.

Contexto — O que R pode (e não pode) fazer

R é bom para

Simulações Monte Carlo, visualizações animadas (gganimate), dashboards Shiny interativos, gráficos "corrida de barras" animados, autômatos celulares, passeios aleatórios, puzzles probabilísticos (Monty Hall, etc.)

R não é adequado para

Jogos com loop em tempo real (60 fps), detectar teclas pressionadas continuamente, renderização 3D, física de colisão, jogos multijogador em rede. Use Unity/Godot/Pygame para isso.

Simulações Monte Carlo — Probabilidade na Prática

Monte Carlo é onde R brilha em contextos de "jogo": simular resultados probabilísticos milhares de vezes para estimar distribuições. O problema de Monty Hall é o exemplo clássico.

monte-carlo.R — Monty Hall, dado viciado, roleta
# ---- Problema de Monty Hall ----
simular_monty_hall <- function(n_sim = 100000, estrategia = "trocar") {
  ganhos <- vapply(1:n_sim, function(...) {
    portas   <- 1:3
    premio   <- sample(portas, 1)
    escolha  <- sample(portas, 1)
    abertas  <- sample(setdiff(portas, c(escolha, premio)), 1)
    if (estrategia == "trocar") {
      nova_escolha <- setdiff(portas, c(escolha, abertas))
      nova_escolha == premio
    } else {
      escolha == premio
    }
  }, logical(1L))
  mean(ganhos)  # probabilidade estimada
}

simular_monty_hall(100000, "trocar")    # ≈ 0.667 (2/3)
simular_monty_hall(100000, "manter")   # ≈ 0.333 (1/3)

# ---- Teste do qui-quadrado para dado viciado ----
lançamentos <- sample(1:6, 1200, replace = TRUE,
                      prob = c(0.1, 0.1, 0.2, 0.2, 0.2, 0.2))
chisq.test(table(lançamentos))
# p-value < 0.001 → dado não é justo

# ---- Paridade da roleta européia ----
roleta_eu <- c(0, 1:36)   # 37 números, 1 zero
resultados <- replicate(1000000, sample(roleta_eu, 1))
mean(resultados %% 2 == 0 & resultados != 0)   # P(par) ≈ 0.486
# Casa leva 2.7% (1/37) em média por aposta

gganimate — Visualizações Animadas

gganimate estende ggplot2 com animações. O caso de uso mais famoso é o "racing bar chart" — barras que se movem ao longo do tempo. Tecnicamente é uma sequência de frames renderizados, não um jogo, mas visualmente é muito eficaz.

gganimate-corrida.R — racing bar chart
library("gganimate")
library("ggplot2")
library("dplyr")

# PIB histórico por país (dados de exemplo)
dados <- gapminder::gapminder |>
  filter(continent == "Americas") |>
  group_by(year) |>
  mutate(rank = rank(-gdpPercap), valor_rel = gdpPercap / max(gdpPercap)) |>
  filter(rank <= 10) |>
  ungroup()

p <- ggplot(dados, aes(rank, group = country)) +
  geom_tile(aes(y = gdpPercap / 2, height = gdpPercap, fill = country),
            width = 0.9, alpha = 0.85) +
  geom_text(aes(y = gdpPercap, label = paste(" $", round(gdpPercap))),
            hjust = 0, size = 4) +
  geom_text(aes(y = 0, label = country), hjust = 1, size = 4) +
  coord_flip(clip = "off", expand = FALSE) +
  scale_x_reverse() +
  labs(title = "PIB per capita — Américas",
       subtitle = "Ano: {closest_state}",  # interpolado pelo gganimate
       x = NULL, y = "PIB per capita (USD)") +
  theme_minimal() +
  theme(legend.position = "none") +
  transition_states(year, transition_length = 2, state_length = 1) +
  ease_aes("cubic-in-out")

animate(p, fps = 15, duration = 20, width = 800, height = 500,
        renderer = gifski_renderer("pib_corrida.gif"))

Simulações Interativas com Shiny

Shiny permite criar simulações onde o usuário controla parâmetros e observa resultados em tempo real. O jogo da vida de Conway (autômato celular) é um exemplo canônico que funciona bem com Shiny + invalidateLater().

jogo-da-vida.R — Conway em Shiny
library("shiny")

# Regras de Conway: 2 ou 3 vizinhos sobrevivem; 3 = nascimento
prox_geracao <- function(grid) {
  n <- nrow(grid)
  m <- ncol(grid)
  soma_vizinhos <- function(g) {
    dirs <- list(c(-1,-1), c(-1,0), c(-1,1), c(0,-1),
                  c(0,1),  c(1,-1),  c(1,0),  c(1,1))
    Reduce(`+`, lapply(dirs, \(d) {
      rbind(if(d[1] < 0) matrix(0,abs(d[1]),m) else NULL,
            g, if(d[1] > 0) matrix(0,d[1],m) else NULL)[
        (1+max(0,d[1])):(n+max(0,d[1])),
        pmax(1, pmin(m, (1:m)+d[2])), drop=FALSE]
    }))
  }
  viz <- soma_vizinhos(grid)
  (grid == 1 & viz %in% c(2,3)) | (grid == 0 & viz == 3)
}

ui <- fluidPage(
  titlePanel("Jogo da Vida de Conway"),
  sidebarLayout(
    sidebarPanel(
      sliderInput("tam", "Tamanho do grid", 20, 80, 40),
      sliderInput("vel", "Velocidade (ms)", 100, 1000, 200),
      actionButton("iniciar", "Iniciar"),
      actionButton("parar", "Parar"),
      actionButton("reset", "Reset")
    ),
    mainPanel(plotOutput("grid_plot", height = "500px", click = "click"))
  )
)

server <- function(input, output, session) {
  grid <- reactiveVal(matrix(0L, 40, 40))
  rodando <- reactiveVal(FALSE)

  observeEvent(input$iniciar, rodando(TRUE))
  observeEvent(input$parar,   rodando(FALSE))
  observeEvent(input$reset, {
    rodando(FALSE)
    n <- input$tam
    grid(matrix(sample(0:1, n*n, replace=TRUE, prob=c(0.7,0.3)), n, n))
  })

  # Tick automático
  observe({
    if (rodando()) {
      invalidateLater(input$vel, session)
      grid(prox_geracao(grid()))
    }
  })

  output$grid_plot <- renderPlot({
    image(grid(), col = c("white","steelblue"), axes = FALSE)
  })
}
shinyApp(ui, server)

Passeio Aleatório e Simulações Estocásticas

random-walk.R — n dimensões
library("ggplot2")

# Passeio aleatório 2D (n=5000 passos, n_walkers=20 caminhantes)
simular_passeio <- function(n = 5000, n_caminhantes = 20) {
  map_dfr(1:n_caminhantes, function(id) {
    passos <- matrix(sample(c(-1, 1), n * 2, replace = TRUE), ncol = 2)
    trajetoria <- rbind(c(0, 0), apply(passos, 2, cumsum))
    tibble(
      caminhante = id,
      passo = 0:n,
      x = trajetoria[, 1],
      y = trajetoria[, 2]
    )
  })
}

set.seed(42)
passeio <- simular_passeio(2000, 30)

ggplot(filter(passeio, passo == max(passo))) +
  geom_point(aes(x, y, color = factor(caminhante)), size = 3, alpha = 0.8) +
  geom_segment(aes(x = 0, y = 0, xend = x, yend = y, color = factor(caminhante)),
               alpha = 0.3) +
  theme_minimal() +
  labs(title = sprintf("Posição final após %d passos", 2000)) +
  theme(legend.position = "none")

# Distância esperada do ponto de partida: √n ≈ 44.7
passeio |>
  filter(passo == 2000) |>
  mutate(dist = sqrt(x^2 + y^2)) |>
  summarise(dist_media = mean(dist))   # ≈ 44.7
