R para Embedded e IoT

R na borda: análise de séries temporais de sensores, execução em ARM (Raspberry Pi), MQTT, detecção de anomalias e integração com Python para pipelines híbridos.

R não é uma linguagem nativa de sistemas embarcados — não controla GPIO, não interage diretamente com hardware de microcontroladores. Seu papel no ecossistema IoT é na camada de análise: processar, modelar e detectar padrões nos dados coletados pelos dispositivos. Em dispositivos mais potentes como Raspberry Pi, R pode ser executado diretamente na borda.

Posicionamento de R na Arquitetura IoT

Em uma arquitetura IoT típica, R atua nas camadas superiores da pilha, onde o poder computacional está disponível:

R no Raspberry Pi

O Raspberry Pi 4/5 tem processador ARM Cortex-A72/A76 de 64 bits e pode executar R satisfatoriamente para análises em tempo quase-real. O Raspberry Pi OS inclui R nos repositórios, mas a versão pode ser desatualizada.

MQTT — Recebendo Dados de Sensores

MQTT é o protocolo de mensageria padrão em IoT (leve, pub/sub, ideal para redes instáveis). O pacote mqtt (e rmqtt) permite que R se inscreva em tópicos MQTT e processe dados de sensores em tempo quase-real.

mqtt-subscriber.R — assinar tópico de sensor
# Instalar: install.packages("mqtt")  (wrapper do Paho MQTT)
library("mqtt")
library("jsonlite")

buffer_leituras <- list()

# Conectar ao broker e processar mensagens
mqtt_broker("localhost", 1883L) |>
  mqtt_subscribe("sensores/temperatura", qos = 1L) |>
  mqtt_subscribe("sensores/umidade", qos = 1L) |>
  mqtt_run(function(topico, mensagem, ...) {
    dados <- fromJSON(rawToChar(mensagem))

    # Acumular no buffer
    buffer_leituras[[length(buffer_leituras) + 1]] <<- data.frame(
      topico    = topico,
      timestamp = Sys.time(),
      valor     = dados$valor,
      dispositivo = dados$id
    )

    # Processar a cada 100 leituras
    if (length(buffer_leituras) >= 100) {
      df <- do.call(rbind, buffer_leituras)
      processar_lote(df)
      buffer_leituras <<- list()
    }
  })

Séries Temporais de Sensores

iot-series-temporais.R — análise e visualização
library("tidyverse")
library("lubridate")
library("tsibble")   # extensão tidy para séries temporais
library("feasts")    # Feature Extraction And Statistics for Time Series

# Carregar dados de sensor
leituras <- read_csv("sensor_temperatura.csv") |>
  mutate(timestamp = as_datetime(timestamp, tz = "America/Sao_Paulo")) |>
  as_tsibble(index = timestamp, key = dispositivo)

# Preencher gaps (timestamps faltantes com NA)
leituras <- fill_gaps(leituras, .full = TRUE)

# Decomposição STL (Seasonal and Trend decomposition using Loess)
decomp <- leituras |>
  model(STL(valor ~ trend() + season(period = "day"))) |>
  components()

autoplot(decomp)   # gráfico: tendência + saz. diária + resíduo

# Agregação por hora
leituras |>
  index_by(hora = ~floor_date(., "hour")) |>
  summarise(
    media   = mean(valor, na.rm = TRUE),
    max_val = max(valor, na.rm = TRUE),
    n_obs   = n()
  )

Detecção de Anomalias

anomalias.R — detecção estatística
library("anomalize")  # tibble-based anomaly detection
library("forecast")

# anomalize — método IQR ou GESD
leituras |>
  time_decompose(valor, method = "stl") |>
  anomalize(remainder, method = "iqr", alpha = 0.05) |>
  time_recompose() |>
  filter(anomaly == "Yes") |>
  select(timestamp, valor, anomaly)

# Detecção com z-score rolante (janela deslizante)
detectar_anomalias <- function(x, janela = 60, limiar = 3) {
  n    <- length(x)
  flag <- logical(n)
  for (i in (janela + 1):n) {
    janela_x <- x[(i - janela):(i - 1)]
    z <- (x[i] - mean(janela_x)) / sd(janela_x)
    flag[i] <- abs(z) > limiar
  }
  flag
}

leituras$anomalia <- detectar_anomalias(leituras$valor)

# Alerta por email se anomalia detectada
if (any(tail(leituras$anomalia, 5))) {
  # blastula::email_message(...) para envio de email com ggplot
  message("ALERTA: anomalia detectada em sensor")
}

Integração com Python via reticulate

Para pipelines IoT onde a coleta de dados é feita em Python (frameworks como paho-mqtt têm suporte melhor) e a análise em R, o pacote reticulate permite chamar código Python diretamente de dentro de sessões R.

reticulate-iot.R
library("reticulate")

# Usar ambiente Python específico (venv ou conda)
use_virtualenv("~/envs/iot-env", required = TRUE)

# Importar módulo Python
paho <- import("paho.mqtt.client")
serial <- import("serial")   # pyserial para comunicação serial

# Executar script Python e recuperar variável
py_run_file("coletar_sensores.py")
dados_py <- py$dados_coletados   # acessar variável Python
df_r     <- as.data.frame(dados_py)

# O inverso — passar dados R para Python
py$df_r <- df_r   # disponibiliza em py$

# Tensorflow para inferência de modelo treinado em Python
tf <- import("tensorflow")
modelo_tf <- tf$keras$models$load_model("modelo_sensor.h5")
X_new <- as.matrix(df_r[, features])
previsoes <- modelo_tf$predict(r_to_py(X_new))
