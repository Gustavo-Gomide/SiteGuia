R para Mobile

R para Dispositivos Móveis

Shiny como PWA, flexdashboard responsivo, shinymobile com Framework7, Rserve como backend analítico, e R no Android via Termux.

R não tem SDK nativo para iOS ou Android — você não compila código R em um .apk ou .ipa. A estratégia é diferente: R roda no servidor como back-end analítico, e a interface mobile é construída com tecnologias web (Shiny responsivo, PWA) ou com frameworks nativos (Flutter, React Native) que consomem APIs R via Plumber ou Rserve.

Shiny como PWA (Progressive Web App)

Uma PWA é instalável no homescreen do smartphones, funciona offline (parcialmente) e tem aparência próxima a um app nativo. Para transformar um app Shiny em PWA, basta adicionar um manifest.json e um Service Worker na pasta www/.

www/manifest.json — Web App Manifest
{
  "name": "Dashboard Analítico",
  "short_name": "Dashboard",
  "description": "Dashboard de análise de vendas",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#1a1a2e",
  "theme_color": "#3b7dd8",
  "orientation": "any",
  "icons": [
    { "src": "icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
app.R — adicionar manifest ao head do Shiny
library("shiny")

ui <- fluidPage(
  tags$head(
    # Link para o manifest
    tags$link(rel = "manifest", href = "manifest.json"),
    # Meta tags para iOS (Safari não suporta manifest plenamente)
    tags$meta(name = "apple-mobile-web-app-capable", content = "yes"),
    tags$meta(name = "apple-mobile-web-app-status-bar-style", content = "black"),
    tags$meta(name = "viewport",
              content = "width=device-width, initial-scale=1.0, maximum-scale=1.0"),
    tags$meta(name = "theme-color", content = "#3b7dd8")
  ),
  titlePanel("Dashboard Mobile"),
  # Detecção de orientação para layout adaptável
  tags$style(HTML("
    @media (max-width: 768px) { .sidebar { display: none; } }
    @media (orientation: landscape) { .main { flex-direction: row; } }
  ")),
  plotOutput("grafico")
)

shinymobile — Framework7 em Shiny

shinymobile integra o Framework7 ao Shiny, fornecendo componentes com visual nativo de iOS e Android: navbar, tabs, list views, cards, action sheets e side panels.

shinymobile-app.R
# remotes::install_github("RinteRface/shinymobile")
library("shiny")
library("shinymobile")

ui <- mobilePageLayout(
  mobileNavbar(title = "Análise de Campos"),

  mobileTabLayout(
    id = "tabs",

    mobileTab(
      tabName = "resumo",
      icon    = "chart_bar",
      label   = "Resumo",
      f7Card(title = "Totais do Dia",
             uiOutput("card_totais")),
      plotOutput("grafico_hoje")
    ),

    mobileTab(
      tabName = "entrada",
      icon    = "plus_circle",
      label   = "Entrada",
      f7Card(
        title = "Registrar Dado",
        f7Text("campo_id", "ID do Talhão:"),
        f7Slider("valor", "Leitura:", 0, 100, 50),
        f7Button("salvar", "Salvar", fill = TRUE)
      )
    )
  )
)

server <- function(input, output, session) {
  dados <- reactiveValues(registros = data.frame())
  observeEvent(input$salvar, {
    novo <- data.frame(id = input$campo_id, valor = input$valor, ts = Sys.time())
    dados$registros <- rbind(dados$registros, novo)
  })
  output$card_totais <- renderUI({
    p(strong(nrow(dados$registros)), " registros hoje")
  })
}

shinyApp(ui, server)

Rserve — Backend para Apps Nativos

Rserve é um servidor TCP que expõe uma sessão R. Apps nativos em Flutter, React Native ou Swift podem enviar código R e receber resultados. É uma arquitetura mais complexa mas oferece experiência totalmente nativa.

Comparação: Plumber API vs Rserve
# ---- Opção 1: Plumber API (REST) — recomendado ----
# + Protocolo HTTP padrão, fácil de consumir de qualquer linguagem
# + Deploy em containers Docker
# + Auth com Bearer Token, CORS configurável
# - Stateless (cada request é independente)

library("plumber")
#* @get /previsao
function(x = "1,2,3,4,5") {
  vals <- as.numeric(strsplit(x, ",")[[1]])
  list(proximo = forecast::forecast(ts(vals), h = 3)$mean)
}

# A partir do Flutter: GET http://meuservidor.com/previsao?x=1,2,3,4,5

# ---- Opção 2: Rserve (TCP) — legado/especializado ----
# + Sessão R persistente (estado entre chamadas)
# - Protocolo binário proprietário
# - Mais difícil de autenticar/escalar
library("Rserve")
Rserve(args = "--no-save --RS-port 6311")

Termux — R no Android

Termux é um emulador de terminal Linux para Android. Permite instalar R sem root, tornando o dispositivo um ambiente de análise de dados de campo.

Terminal Termux — instalação e uso
# Instalar pacotes no Termux
$ pkg update && pkg upgrade
$ pkg install r-base

# Iniciar R
$ R
R version 4.4.1 ...

# Instalar pacotes R
> install.packages("tidyverse", repos="https://cran.rstudio.com/")

# Executar script de campo
$ Rscript analise_campo.R --input dados_gps.csv --output relatorio.html

# Compartilhar resultados via Telegram (termux-api)
$ pkg install termux-api
$ Rscript gerar_relatorio.R
$ termux-share -a send relatorio.html

Termux junto com termux-api permite acessar GPS, câmera, microfone e sensor de aceleração diretamente de scripts R via chamadas de sistema — útil para cientistas fazendo coletas de campo que precisam de análise imediata no próprio dispositivo.
