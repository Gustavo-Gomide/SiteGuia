Rust para Web

APIs seguras e performaticas com Axum, Tower e stack async moderna.

Rust em web backend combina latencia baixa, uso eficiente de memoria e um modelo de concorrencia robusto para servicos de alta carga.

Axum: Router, Extractors, Middleware e State

axum base
// estrutura minima de API com estado compartilhado
			use axum::{extract::State, routing::{get, post}, Json, Router};
use serde::{Deserialize, Serialize};

#[derive(Clone)]
struct AppState { versao: &'static str }

#[derive(Deserialize)]
struct LoginReq { user: String, pass: String }

#[derive(Serialize)]
struct Health { ok: bool }

async fn health() -> Json<Health> { Json(Health { ok: true }) }
async fn login(State(_s): State<AppState>, Json(_req): Json<LoginReq>) -> Json<Health> {
		Json(Health { ok: true })
}

Com Tower middleware, adicione logging, timeout, compressao e rate limit sem acoplar regras ao handler.

JWT, Argon2, WebSocket e Rate Limiting

Autenticacao comum: senha com Argon2, emissao de JWT para sessao stateless e validacao por middleware. Para comunicacao em tempo real, Axum suporta WebSocket nativamente.

  • Hash de senha: argon2 com salt unico por usuario.
  • Token: jsonwebtoken com expiracao curta e refresh strategy.
  • Rate limiting: camada Tower para proteger endpoints sensiveis.
  • WebSocket: canais para notificacoes e eventos push.

Deploy com Docker, Health Checks e Benchmarks

Deploy padrao usa multi-stage Docker build. Exponha endpoint de health para orquestradores e rode benchmarks periodicos com criterion e testes de carga externos.

dockerfile simplificado
FROM rust:1.85 as builder
WORKDIR /app
COPY . .
RUN cargo build --release

FROM debian:bookworm-slim
COPY --from=builder /app/target/release/api /usr/local/bin/api
CMD ["api"]
