Rust em Embedded e IoT

Firmware seguro para microcontroladores com controle fino de hardware.

Em ambientes com memoria limitada e requisitos de confiabilidade, Rust oferece previsibilidade sem sacrificar desempenho.

no_std e embedded-hal

no_std remove dependencia da biblioteca padrao para bare metal. A crate embedded-hal define traits comuns para GPIO, I2C, SPI e UART.

piscar LED (conceitual)
// firmware bare metal sem std
			#![no_std]
#![no_main]

use embedded_hal::digital::OutputPin;

fn toggle<P: OutputPin>(mut led: P) {
		led.set_high().ok();
		led.set_low().ok();
}

RTIC, Embassy e Targets Cortex-M

RTIC organiza concorrencia por tarefas com prioridades e recursos compartilhados. Embassy traz async/await para embedded com executores leves. Targets comuns: STM32 e Raspberry Pi Pico (RP2040), em geral com familias Cortex-M.

  • RTIC: previsibilidade temporal e reduz race conditions em interrupcoes.
  • Embassy: programacao async em drivers e perifericos.
  • Target example: thumbv7em-none-eabihf.

Debugging com probe-rs e Ciclo de Build

probe-rs facilita flash e debug via SWD/JTAG. No fluxo diario, combine testes host-side para logica pura e testes em hardware para drivers reais.

comandos uteis
rustup target add thumbv7em-none-eabihf
cargo build --release --target thumbv7em-none-eabihf
probe-rs run --chip STM32F401RE target/thumbv7em-none-eabihf/release/app
