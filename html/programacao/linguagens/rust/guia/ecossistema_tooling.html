Rust - Ecossistema e Tooling

Ecossistema e Tooling

Pipeline completa de build, qualidade, documentacao e distribuicao em Rust.

Cargo e rustup formam o coracao operacional do ecossistema: um para projeto e dependencias, outro para toolchains e targets.

Cargo, crates.io e rustup

Cargo.toml define metadados, dependencias, features e perfis de build. Workspaces permitem organizar repositorios multi-crate com lockfile unico.

workspace + features
# Cargo.toml (raiz)
[workspace]
members = ["crates/api", "crates/core"]

[workspace.dependencies]
serde = { version = "1", features = ["derive"] }

# crates/api/Cargo.toml
[features]
default = ["http"]
http = []

Comandos-chave: cargo test, cargo bench, cargo doc --open. Para publicar crates, use cargo publish no crates.io.

rustup no fluxo diario

  • rustup toolchain install stable
  • rustup default stable
  • rustup component add rustfmt clippy
  • rustup target add x86_64-unknown-linux-musl

Qualidade, Editor e Documentacao

rustfmt padroniza estilo; clippy detecta anti-patterns e melhorias idiomaticas; rust-analyzer oferece autocomplete, go-to-definition e refactors de alta precisao.

pipeline local
cargo fmt --all
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all
cargo doc --no-deps

Com rustdoc, comentarios /// viram documentacao navegavel e exemplos compilaveis em doctests.

Cross-Compilation e Profiling

Cross-compilation permite gerar binarios para Linux, Windows, macOS e targets embarcados a partir do mesmo host. Para CPU hotspots, flamegraph ajuda a localizar gargalos reais.

cross + flamegraph
rustup target add x86_64-unknown-linux-musl
cargo build --release --target x86_64-unknown-linux-musl
cargo flamegraph --bin meu_app
