Rust - Introducao

Rust

Linguagem de sistemas com foco em seguranca de memoria, desempenho e concorrencia previsivel.

Rust combina controle de baixo nivel com garantias em tempo de compilacao, reduzindo classes inteiras de bugs sem depender de garbage collector.

Historia e Origem

Rust comecou em 2006 com Graydon Hoare como projeto pessoal. Em 2010, a Mozilla passou a apoiar oficialmente o desenvolvimento, e em 2015 saiu a versao 1.0 com compromisso de estabilidade sem quebrar codigo em releases menores.

Linha do tempo essencial

  • 2006: experimento inicial de linguagem por Graydon Hoare.
  • 2010: Mozilla Research assume apoio institucional.
  • 2015: Rust 1.0 e modelo de estabilidade semver para o compilador.
  • 2018 e 2021: edicoes da linguagem com melhorias de ergonomia.

Filosofia da Linguagem

Rust segue dois pilares: memory safety sem GC e fearless concurrency. Ownership e borrowing permitem que o compilador prove seguranca de acesso a memoria; o sistema de tipos e traits permite escrever codigo generico sem custo extra em runtime.

Em Rust, erro de compilacao frequentemente representa bug evitado em producao.

Casos de Uso Reais

Sistemas

Ferramentas de infraestrutura, bancos, proxies e componentes de alto desempenho.

Backend

APIs com Axum/Actix, servicos async com Tokio e observabilidade moderna.

WASM

Modulos leves para browser com foco em performance de CPU e binarios pequenos.

Embedded

Firmware no_std para microcontroladores com controle preciso de recursos.

Instalacao com rustup e Hello World

O caminho padrao e instalar via rustup, que gerencia toolchains e componentes. Depois, use cargo new para iniciar um projeto.

terminal
# instalar rustup (Windows pode usar rustup-init.exe)
rustup default stable
cargo new ola_rust
cd ola_rust
cargo run
src/main.rs
fn main() {
		println!("Ola, Rust!");
}

Toolchain essencial: rustc (compilador), cargo (build/deps), rustfmt (formatacao), clippy (lint).
