Rust - Avancado

Rust Avancado

Padroes de alto nivel sem abandonar desempenho e previsibilidade.

Os recursos avancados de Rust ampliam expressividade sem sacrificar o modelo de seguranca: abstrair melhor, sem custo oculto.

Trait Objects, Generics, Closures e Iteradores

dyn Trait habilita dispatch dinamico; generics usam monomorfizacao e costumam gerar codigo otimizado por tipo concreto.

dyn Trait vs generic
trait Render { fn render(&self); }

fn draw_dyn(itens: &[Box<dyn Render>]) {
		for i in itens { i.render(); }
}

fn draw_generic<T: Render>(itens: &[T]) {
		for i in itens { i.render(); }
}

Iteradores sao lazy: map, filter e collect compoem pipelines legiveis.

Async/Await, Macros e Smart Pointers

tokio async
use reqwest;

#[tokio::main]
async fn main() -> Result<(), reqwest::Error> {
		let body = reqwest::get("https://example.com").await?;
		println!("status={}", body.status());
		Ok(())
}

Macros declarativas (macro_rules!) removem boilerplate; procedurais geram codigo em compile-time para derives e atributos personalizados.

Mapa mental de ponteiros inteligentes

  • Box<T>: alocacao no heap com ownership unico.
  • Rc<T>/Arc<T>: referencia compartilhada single-thread/multi-thread.
  • RefCell<T>/Mutex<T>: mutabilidade interior com checks em runtime/sincronizacao.

Unsafe Rust e Zero-Cost Abstractions

unsafe e uma ferramenta de fronteira para FFI, raw pointers e otimizacoes especificas. O contrato e: voce assume manualmente as invariantes de seguranca.

unsafe minimo
fn ler_raw(p: *const i32) -> i32 {
		unsafe { *p } // caller deve garantir ponteiro valido
}

Use unsafe em blocos pequenos, documentados e com testes de fronteira.

Zero-cost abstractions significa que iteradores, traits genericos e padroes idiomaticos podem compilar para codigo tao eficiente quanto implementacoes manuais de baixo nivel.
