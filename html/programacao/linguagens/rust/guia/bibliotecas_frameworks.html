Rust - Bibliotecas e Frameworks

Bibliotecas e Frameworks em Rust

Crates essenciais para montar sistemas completos, de API a CLI.

O ecossistema Rust privilegia composicao: voce combina crates especializadas em vez de depender de um framework monolitico unico.

Runtime, Web e Dados

Tokio

Runtime async mais adotado para IO, timers, tasks e networking.

Axum e Actix Web

Frameworks HTTP com abordagens distintas: Axum mais composicional e Tower-first; Actix focado em throughput e ergonomia pragmatica.

Serde

Serializacao e desserializacao altamente performaticas para JSON, YAML, TOML e formatos binarios.

SQLx e Diesel

SQLx: queries verificadas em compile-time; Diesel: ORM fortemente tipado com DSL.

HTTP, Paralelismo, Erros, CLI e Observabilidade

reqwest

Cliente HTTP async/sync com TLS, JSON e middlewares via ecosystem crates.

Rayon

Paralelismo de dados com iteradores paralelos e work-stealing.

anyhow / thiserror

thiserror para erros de biblioteca; anyhow para apps executaveis com contexto rico.

clap e tracing

CLI robusta por derive e telemetria estruturada com spans e eventos.

exemplo de stack minima
# Cargo.toml
[dependencies]
tokio = { version = "1", features = ["full"] }
axum = "0.7"
serde = { version = "1", features = ["derive"] }
sqlx = { version = "0.8", features = ["runtime-tokio-rustls", "postgres"] }
tracing = "0.1"

Como Escolher no Dia a Dia

  • API web moderna: Tokio + Axum + Serde + SQLx + tracing.
  • Projeto enterprise com ORM forte: Diesel pode trazer previsibilidade de schema.
  • Ferramenta de linha de comando: clap + anyhow + tracing-subscriber.
  • CPU-bound em lote: rayon para acelerar transformacoes em colecoes grandes.
