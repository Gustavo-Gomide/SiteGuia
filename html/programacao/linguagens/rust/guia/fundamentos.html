Rust - Fundamentos

Fundamentos do Rust

A base conceitual que torna Rust seguro e rapido em producao.

Ownership, borrowing e lifetimes formam o nucleo da linguagem; dominar esses pilares simplifica qualquer topico avancado depois.

Ownership, Stack/Heap e Borrowing

Em Rust, cada valor tem um unico dono. Ao mover um valor, o antigo binding deixa de ser valido. Referencias imutaveis (&T) e mutaveis (&mut T) evitam copias e mantem seguranca de aliasing.

ownership e move
fn main() {
		let s1 = String::from("rust");
		let s2 = s1; // move: s1 deixa de ser valido
		println!("{}", s2);
}
borrowing
fn tamanho(s: &String) -> usize { s.len() }

fn anexar(s: &mut String) {
		s.push_str("!");
}

Lifetimes, Tipos, Structs e Enums

Na maioria dos casos, lifetimes sao inferidos. Anotacoes explicitas entram quando ha multiplas referencias e o compilador precisa de relacao temporal clara.

lifetime explicito
fn maior<'a>(x: &'a str, y: &'a str) -> &'a str {
		if x.len() > y.len() { x } else { y }
}

Tipos primitivos incluem inteiros, floats, bool e char. Structs modelam dados compostos; enums representam variantes fechadas, ideais para estado.

  • struct Usuario { id: u64, nome: String }
  • enum Status { Ativo, Bloqueado, Suspenso }
  • Option<T> e Result<T, E> sao enums da std cruciais.

Pattern Matching, Traits, Modulos e Erros

match, if let e while let expressam controle de fluxo orientado a tipos. Traits cumprem papel semelhante a interfaces, com dispatch estatico por padrao.

match + if let
enum Status { Ativo, Bloqueado }

fn mensagem(s: Status) -> &'static str {
		match s {
				Status::Ativo => "ok",
				Status::Bloqueado => "negado",
		}
}
Result, Option e ?
use std::fs;

fn ler_cfg() -> Result<String, std::io::Error> {
		let txt = fs::read_to_string("config.toml")?;
		Ok(txt)
}

Modulos organizam o codigo com mod e visibilidade por pub, facilitando encapsulamento e APIs limpas.
