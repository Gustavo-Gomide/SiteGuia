Projetos e Boas Praticas

Arquitetura, qualidade e manutencao de longo prazo em codigo Rust.

Codigo Rust sustentavel depende de estrutura de repositorio, politica de versao, testes efetivos e manejo consistente de erros.

Workspace e Error Handling na Pratica

Separe responsabilidades em crates: dominio, adapters, binarios. Em bibliotecas, prefira tipos de erro explicitos com thiserror; em binarios, agregue contexto com anyhow.

erro de biblioteca + erro de app
use thiserror::Error;

#[derive(Error, Debug)]
pub enum RepoError {
		#[error("registro nao encontrado: {0}")]
		NotFound(u64),
}

// no binario
use anyhow::Context;
let cfg = carregar_cfg().context("falha ao carregar config")?;

Newtypes, Builder Pattern e API Clara

Newtypes evitam confusao semantica entre valores do mesmo tipo base (ex.: String para email e user_id). Builder simplifica objetos com muitos campos opcionais sem construtores gigantes.

  • struct UserId(String); para type safety.
  • ConfigBuilder::new().timeout(...).retries(...).build() para configuracao legivel.
  • Expose o minimo necessario com pub; mantenha invariantes dentro do modulo.

Testes, CI, MSRV, Changelog e SemVer

Combine testes unitarios, integration tests e doc tests. Em CI, execute formatacao, lint e suite completa. Defina MSRV (Minimum Supported Rust Version) para previsibilidade de usuarios e times.

github actions (resumo)
name: ci
on: [push, pull_request]
jobs:
	check:
		runs-on: ubuntu-latest
		steps:
			- uses: actions/checkout@v4
			- uses: dtolnay/rust-toolchain@stable
			- run: cargo fmt --all -- --check
			- run: cargo clippy --all-targets --all-features -- -D warnings
			- run: cargo test --all

Documente MSRV no README e respeite semver: mudanca breaking requer major version.
