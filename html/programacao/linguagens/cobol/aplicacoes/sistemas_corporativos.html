COBOL — Sistemas Corporativos

Bancos, seguradoras, governo e previdência social são o habitat natural do COBOL. Sua aritmética decimal exata (sem erros de ponto flutuante binário), acesso nativo a dados SQL, e décadas de código testado em produção tornam o COBOL insubstituível onde erros de centavos têm consequências legais e financeiras.

O Banco Central Europeu, a New York Stock Exchange, e o INSS brasileiro rodam COBOL. Entender como o COBOL resolve problemas reais nestas indústrias — cálculo de pensão, processamento de pagamento, apólice de seguro — é entender por que ele sobreviveu 65 anos.

Sistema Bancário — Transferência Segura

Transferência interbancária com rollback em caso de erro
       IDENTIFICATION DIVISION.
       PROGRAM-ID. TRANSFERE.

       WORKING-STORAGE SECTION.
         01 WS-CONTA-ORIGEM    PIC X(10).
         01 WS-CONTA-DESTINO   PIC X(10).
         01 WS-VALOR           PIC 9(12)V99 COMP-3.
         01 WS-SALDO-ORIG      PIC S9(12)V99 COMP-3.
         01 WS-SALDO-DEST      PIC S9(12)V99 COMP-3.
         01 WS-STATUS-DB       PIC 9(4).
         01 WS-MSG-ERRO        PIC X(80).

       EXEC SQL INCLUDE SQLCA END-EXEC.

       PROCEDURE DIVISION.
       0000-INICIAR.
           MOVE '0001234567' TO WS-CONTA-ORIGEM.
           MOVE '0009876543' TO WS-CONTA-DESTINO.
           MOVE 1500.00 TO WS-VALOR.
           PERFORM 1000-VALIDAR-SALDO.
           IF WS-STATUS-DB = 0
               PERFORM 2000-EXECUTAR-TRANSFERENCIA
           END-IF.
           STOP RUN.

       1000-VALIDAR-SALDO.
           EXEC SQL
               SELECT SALDO INTO :WS-SALDO-ORIG
               FROM CONTAS
               WHERE NUM_CONTA = :WS-CONTA-ORIGEM
               FOR UPDATE OF SALDO     *> lock pessimista
           END-EXEC.
           IF SQLCODE NOT EQUAL 0
               MOVE SQLCODE TO WS-STATUS-DB
               MOVE 'Conta origem não encontrada' TO WS-MSG-ERRO
               PERFORM 9999-ERRO
           END-IF.
           IF WS-SALDO-ORIG < WS-VALOR
               MOVE 4001 TO WS-STATUS-DB
               MOVE 'Saldo insuficiente' TO WS-MSG-ERRO
               PERFORM 9999-ERRO
           END-IF.

       2000-EXECUTAR-TRANSFERENCIA.
           EXEC SQL
               UPDATE CONTAS SET SALDO = SALDO - :WS-VALOR
               WHERE NUM_CONTA = :WS-CONTA-ORIGEM
           END-EXEC.
           IF SQLCODE NOT EQUAL 0 PERFORM 9000-ROLLBACK.

           EXEC SQL
               UPDATE CONTAS SET SALDO = SALDO + :WS-VALOR
               WHERE NUM_CONTA = :WS-CONTA-DESTINO
           END-EXEC.
           IF SQLCODE NOT EQUAL 0 PERFORM 9000-ROLLBACK.

           EXEC SQL
               INSERT INTO HISTORICO_TRANS
               (DT_HORA, ORIGEM, DESTINO, VALOR, TIPO)
               VALUES (CURRENT TIMESTAMP,
                       :WS-CONTA-ORIGEM, :WS-CONTA-DESTINO,
                       :WS-VALOR, 'TRF')
           END-EXEC.
           EXEC SQL COMMIT END-EXEC.
           DISPLAY 'Transferência realizada: R$ ' WS-VALOR.

       9000-ROLLBACK.
           EXEC SQL ROLLBACK END-EXEC.
           DISPLAY 'ROLLBACK executado. Erro: ' SQLCODE.
           STOP RUN.

       9999-ERRO.
           DISPLAY WS-MSG-ERRO.
           STOP RUN.

Previdência — Cálculo de Benefício

Cálculo de aposentadoria com media salarial e fator previdenciário
       WORKING-STORAGE SECTION.
         01 WS-TABELA-CONTRIB.
           05 WS-CONTRIB OCCURS 1 TO 480 TIMES
                         DEPENDING ON WS-QT-CONTRIB
                         INDEXED BY WS-IDX-C.
             10 WS-COMP  PIC 9(6).         *> competência AAAAMM
             10 WS-SAL   PIC 9(7)V99 COMP-3.

         01 WS-QT-CONTRIB   PIC 9(3).
         01 WS-MEDIA-80     PIC 9(9)V99 COMP-3.
         01 WS-FATOR-PREV   PIC 9(2)V9(4) COMP-3.
         01 WS-BENEFICIO    PIC 9(9)V99 COMP-3.
         01 WS-TETO-INSS    PIC 9(7)V99 VALUE 7786.02.
         01 WS-PISO-INSS    PIC 9(7)V99 VALUE 1412.00.

         01 WS-SOMA-80      PIC 9(13)V99 COMP-3.
         01 WS-QT-80        PIC 9(3).
         01 WS-AUX-VALOR    PIC 9(9)V99 COMP-3.
         01 WS-INDICE-SAL   PIC 9(2)V9(6) COMP-3.

       PROCEDURE DIVISION.
       3000-CALCULAR-MEDIA-80.
      *> Regra dos 80%: calcular sobre os maiores 80% dos salários
           COMPUTE WS-QT-80 = FUNCTION INTEGER(WS-QT-CONTRIB * 0.8).
           MOVE ZEROS TO WS-SOMA-80.

      *> Ordenar contribuições DESC (SORT interno simplificado)
           PERFORM VARYING WS-IDX-C FROM 1 BY 1
               UNTIL WS-IDX-C > WS-QT-80
               ADD WS-SAL(WS-IDX-C) TO WS-SOMA-80
           END-PERFORM.

           COMPUTE WS-MEDIA-80 ROUNDED =
               WS-SOMA-80 / WS-QT-80.

      *> Aplicar teto e piso do INSS
           IF WS-MEDIA-80 > WS-TETO-INSS
               MOVE WS-TETO-INSS TO WS-MEDIA-80
           END-IF.
           IF WS-MEDIA-80 < WS-PISO-INSS
               MOVE WS-PISO-INSS TO WS-MEDIA-80
           END-IF.

       3100-APLICAR-FATOR-PREVIDENCIARIO.
      *> Fator simplificado: (TC × a) / Es × (1 + (Id + TC × a) / 100)
      *> TC = tempo contribuição, Id = idade, a = alíquota, Es = expectativa
           COMPUTE WS-BENEFICIO ROUNDED =
               WS-MEDIA-80 * WS-FATOR-PREV.

      *> Exibir com máscara monetária
           MOVE WS-BENEFICIO TO WS-AUX-VALOR.
           DISPLAY 'Benefício Calculado: R$ '
                   WS-AUX-VALOR.

CICS — Transações Online

CICS BMS — consulta de saldo em terminal 3270
       IDENTIFICATION DIVISION.
       PROGRAM-ID. CONSULSAL.

       WORKING-STORAGE SECTION.
         01 WS-CONTA         PIC X(10).
         01 WS-SALDO         PIC S9(12)V99 COMP-3.
         01 WS-SALDO-EDIT    PIC ZZZ,ZZZ,ZZ9.99-.

       EXEC SQL INCLUDE SQLCA END-EXEC.

       LINKAGE SECTION.
         01 DFHCOMMAREA      PIC X(100).    *> área de comunicação CICS

       PROCEDURE DIVISION.
       INICIO.
           IF EIBCALEN = ZEROS
      *>      Primeira vez — enviar mapa de entrada
               EXEC CICS SEND MAP('CONSMAPA')
                              MAPSET('CONSMAPS')
                              ERASE
               END-EXEC
               EXEC CICS RETURN
                         TRANSID('CSAL')
                         COMMAREA(WS-CONTA)
               END-EXEC
           ELSE
      *>      Segunda vez — receber dados do operador
               EXEC CICS RECEIVE MAP('CONSMAPA')
                                 MAPSET('CONSMAPS')
               END-EXEC
               PERFORM 1000-CONSULTAR-SALDO
               EXEC CICS SEND MAP('CONSMAPA')
                              MAPSET('CONSMAPS')
                              DATAONLY
               END-EXEC
               EXEC CICS RETURN END-EXEC
           END-IF.

       1000-CONSULTAR-SALDO.
           EXEC SQL
               SELECT SALDO INTO :WS-SALDO
               FROM CONTAS WHERE NUM_CONTA = :WS-CONTA
           END-EXEC.
           IF SQLCODE = 0
               MOVE WS-SALDO TO WS-SALDO-EDIT
           ELSE
               MOVE 'CONTA NAO ENCONTRADA' TO MSGO
           END-IF.

Seguradora — Sinistro e Provisão

Cálculo de prêmio de seguro com tabela de risco
       WORKING-STORAGE SECTION.
         01 WS-TB-RISCO.
           05 WS-FAIXA OCCURS 10 TIMES INDEXED BY WS-IF.
             10 WS-IDADE-MIN  PIC 9(3).
             10 WS-IDADE-MAX  PIC 9(3).
             10 WS-FATOR-RISCO PIC 9V9(4) COMP-3.

         01 WS-APOL.
           05 WS-SEGURADO-IDADE  PIC 9(3).
           05 WS-CAPITAL-SEGURADO PIC 9(10)V99 COMP-3.
           05 WS-COBERTURA-TIPO  PIC X(2).
             88 COB-VIDA        VALUE 'VD'.
             88 COB-AUTOM       VALUE 'AT'.
             88 COB-RESID       VALUE 'RS'.
           05 WS-PREMIO-PURO    PIC 9(9)V99 COMP-3.
           05 WS-PREMIO-COMERCIAL PIC 9(9)V99 COMP-3.
           05 WS-LOADING        PIC 9V9(3) COMP-3.

       PROCEDURE DIVISION.
       5000-CALCULAR-PREMIO.
      *> Buscar fator de risco por faixa etária (SEARCH)
           SET WS-IF TO 1.
           SEARCH WS-FAIXA
               AT END MOVE 2.5 TO WS-LOADING
               WHEN WS-SEGURADO-IDADE >= WS-IDADE-MIN(WS-IF)
                AND WS-SEGURADO-IDADE <= WS-IDADE-MAX(WS-IF)
                   CONTINUE
           END-SEARCH.

           COMPUTE WS-PREMIO-PURO ROUNDED =
               WS-CAPITAL-SEGURADO * WS-FATOR-RISCO(WS-IF) / 1000.

      *> Loading: carregamento de despesas e lucro
           EVALUATE TRUE
               WHEN COB-VIDA
                   MOVE 1.35 TO WS-LOADING
               WHEN COB-AUTOM
                   MOVE 1.45 TO WS-LOADING
               WHEN COB-RESID
                   MOVE 1.25 TO WS-LOADING
               WHEN OTHER
                   MOVE 1.40 TO WS-LOADING
           END-EVALUATE.

           COMPUTE WS-PREMIO-COMERCIAL ROUNDED =
               WS-PREMIO-PURO * WS-LOADING.

           DISPLAY 'Prêmio puro:      R$ ' WS-PREMIO-PURO.
           DISPLAY 'Prêmio comercial: R$ ' WS-PREMIO-COMERCIAL.

O COBOL usa aritmética decimal compactada (COMP-3): cada dígito ocupa 4 bits, então PIC 9(7)V99 COMP-3 usa apenas 5 bytes para representar valores como 1.234.567,89 — sem erro de ponto flutuante binário. Isso explica por que sistemas financeiros que lidam com centavos não podem usar double de C ou float de Python: 0.1 + 0.2 = 0.30000000000000004 em binário IEEE-754, mas COMP-3 sempre dá 0.30.
