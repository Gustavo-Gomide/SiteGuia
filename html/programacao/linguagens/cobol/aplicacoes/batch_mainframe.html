COBOL — Batch e Mainframe

O processamento batch em mainframe IBM z/OS é o núcleo da infraestrutura financeira mundial. Durante a madrugada, centenas de jobs JCL processam bilhões de transações — extratos bancários, pagamento de pensões, cobrança de impostos — usando COBOL com VSAM, SORT utility e DB2 em pipelines de alta confiabilidade.

Um único job batch bem escrito pode processar 50 milhões de registros por hora em um IBM z16. A chave: zero overhead de I/O (arquivos VSAM sequenciais em buffers), processamento record-by-record em memória constante, e ausência de garbage collector.

VSAM — Virtual Storage Access Method

IDCAMS — definir KSDS (Key-Sequenced Data Set)
//DEFVSAM  JOB ...
//DEFCLUS  EXEC PGM=IDCAMS
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  DEFINE CLUSTER                           -
         (NAME(BANCO.VSAM.CONTAS)          -
          INDEXED                          -
          KEYS(10 0)                       -  /* chave 10 bytes, offset 0 */
          RECSZ(200 200)                   -  /* tamanho fixo 200 bytes */
          CYLINDERS(50 10)                 -
          FREESPACE(20 10)                 -  /* 20% CI, 10% CA livre */
          SHAREOPTIONS(1 3))               -
         DATA                             -
         (NAME(BANCO.VSAM.CONTAS.DATA))    -
         INDEX                            -
         (NAME(BANCO.VSAM.CONTAS.INDEX))
COBOL lendo e atualizando VSAM KSDS
       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
         FILE-CONTROL.
           SELECT VSAM-CONTAS ASSIGN TO CONTAS
               ORGANIZATION IS INDEXED
               ACCESS MODE IS DYNAMIC
               RECORD KEY IS VC-NUM-CONTA
               ALTERNATE RECORD KEY IS VC-CPF-TITULAR
                   WITH DUPLICATES
               FILE STATUS IS WS-FS-CONTAS.

       DATA DIVISION.
       FILE SECTION.
         FD VSAM-CONTAS.
         01 REG-CONTA.
           05 VC-NUM-CONTA    PIC X(10).    *> chave primária
           05 VC-CPF-TITULAR  PIC X(11).    *> chave alternada
           05 VC-NOME         PIC X(40).
           05 VC-SALDO        PIC S9(12)V99 COMP-3.
           05 VC-STATUS       PIC X.
           05 FILLER          PIC X(76).

       PROCEDURE DIVISION.
      *> ── Leitura por chave (RANDOM) ───────────────────────────
           MOVE '0001234567' TO VC-NUM-CONTA.
           READ VSAM-CONTAS
               INVALID KEY DISPLAY 'Conta não encontrada'
               NOT INVALID KEY PERFORM PROCESSAR-CONTA
           END-READ.

      *> ── Atualizar (REWRITE) ─────────────────────────────────
           ADD WS-CREDITO TO VC-SALDO.
           REWRITE REG-CONTA
               INVALID KEY DISPLAY 'Erro REWRITE: ' WS-FS-CONTAS.

      *> ── Percorrer sequencialmente por chave alternada ────────
           MOVE '12345678901' TO VC-CPF-TITULAR.
           START VSAM-CONTAS KEY = VC-CPF-TITULAR
               INVALID KEY DISPLAY 'CPF não encontrado'.

           PERFORM UNTIL WS-FS-CONTAS = '10'
               READ VSAM-CONTAS NEXT
                   AT END MOVE '10' TO WS-FS-CONTAS
                   NOT AT END PERFORM EXIBIR-CONTA
               END-READ
           END-PERFORM.

GDG — Generation Data Groups

GDG — histórico rotativo de datasets
// Definir GDG Base com IDCAMS
//SYSIN DD *
  DEFINE GDG                               -
         (NAME(BANCO.RELATORIO.SALDOS)     -
          LIMIT(30)                        -  /* manter 30 gerações */
          NOEMPTY                          -  /* não deletar ao atingir limite */
          SCRATCH)                            /* deletar antigas automaticamente */

SORT Utility — Controle Avançado

Parâmetros SORT, INCLUDE e OUTFIL para relatórios
  SORT FIELDS=(1,4,ZD,A,        /* campo 1, 4 bytes, Zoned Decimal, ASC */
               5,10,CH,A),      /* campo 5, 10 bytes, Character, ASC */
       FILSZ=E10000000          /* estimativa: 10M registros */

  * Eliminar duplicatas pelo campo chave completo
  SORT FIELDS=(1,14,CH,A),EQUALS
  SUM FIELDS=NONE

  * Filtrar apenas registros válidos (INCLUDE)
  INCLUDE COND=(15,1,CH,EQ,C'A')    /* status = 'A'  */

  * Excluir registros de teste
  OMIT COND=(1,3,ZD,EQ,999)         /* agencia = 999 */

  * OUTFIL — gerar múltiplas saídas e reformatar
  OUTFIL FNAMES=SAIDA1,
         INCLUDE=(15,1,CH,EQ,C'A'),  /* só ativos */
         BUILD=(1,14,           /* copiar campos 1-14 */
                30,10,ZD,EDIT=(IIIIIIT.TT),   /* formatar saldo */
                80,8,CH)             /* copiar data */

O GDG (Generation Data Group) é um mecanismo único do z/OS: um dataset com histórico rotativo automático. (+1) cria a próxima geração, (0) acessa a mais recente e (-1) a anterior. Bancos usam GDGs para manter 30 dias de relatórios diários sem gerenciar nomes de arquivos manualmente — o sistema cataloga e deleta gerações antigas automaticamente.
