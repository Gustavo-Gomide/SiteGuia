COBOL — Avançado

No nível avançado, COBOL revela suas capacidades imbatíveis para processamento de dados corporativos: SORT/MERGE para ordenação de arquivos enormes sem carregar na memória, VSAM para acesso indexado (ISAM) e randômico, tabelas internas com OCCURS/SEARCH para lookup tables, e EXEC SQL para embedded SQL em DB2 e Oracle.

O COBOL-2002 adicionou OOP completa — classes, herança, factories, interfaces. Embora raramente usado em código novo de mainframe (COBOL legado raramente precisa de OOP), entender a sintaxe é essencial para modernizar sistemas.

Tabelas Internas (Arrays) — OCCURS

OCCURS, SEARCH, SEARCH ALL — tabelas e busca
       WORKING-STORAGE SECTION.
       01 WS-TABELA-ESTADOS.
         05 WS-ESTADO OCCURS 27 TIMES
                      INDEXED BY WS-IDX-ESTADO.
           10 WS-UF          PIC XX.
           10 WS-NOME-ESTADO PIC X(20).
           10 WS-POPULACAO   PIC 9(8).

       01 WS-TABELA-TRIBUTOS.
         05 WS-TRIBUTO OCCURS 1 TO 100 TIMES
                       DEPENDING ON WS-QTDE-TRIBUTOS
                       ASCENDING KEY IS WS-COD-TRIB
                       INDEXED BY WS-IDX-TRIB.
           10 WS-COD-TRIB    PIC 9(4).
           10 WS-ALIQUOTA    PIC V9(4).

       01 WS-QTDE-TRIBUTOS   PIC 9(3).
       01 WS-ESTADO-BUSCA    PIC XX.

       PROCEDURE DIVISION.
      *> ── Carregar tabela manualmente ──────────────────────────
           MOVE 'SP' TO WS-UF(1)
           MOVE 'São Paulo' TO WS-NOME-ESTADO(1)
           MOVE 46649132 TO WS-POPULACAO(1)

           MOVE 'RJ' TO WS-UF(2)
           MOVE 'Rio de Janeiro' TO WS-NOME-ESTADO(2)
           MOVE 17463349 TO WS-POPULACAO(2)

      *> ── Percorrer com PERFORM VARYING ────────────────────────
           PERFORM VARYING WS-IDX-ESTADO FROM 1 BY 1
               UNTIL WS-IDX-ESTADO > 27
               DISPLAY WS-UF(WS-IDX-ESTADO) SPACE
                       WS-NOME-ESTADO(WS-IDX-ESTADO)
           END-PERFORM.

      *> ── SEARCH — busca linear na tabela ─────────────────────
           MOVE 'MG' TO WS-ESTADO-BUSCA.
           SET WS-IDX-ESTADO TO 1.
           SEARCH WS-ESTADO
               AT END
                   DISPLAY 'Estado não encontrado'
               WHEN WS-UF(WS-IDX-ESTADO) = WS-ESTADO-BUSCA
                   DISPLAY 'Estado: ' WS-NOME-ESTADO(WS-IDX-ESTADO)
           END-SEARCH.

      *> ── SEARCH ALL — busca binária (tabela SORTED) ──────────
           SEARCH ALL WS-TRIBUTO
               AT END
                   DISPLAY 'Tributo não encontrado'
               WHEN WS-COD-TRIB(WS-IDX-TRIB) = 1234
                   DISPLAY 'Alíquota: ' WS-ALIQUOTA(WS-IDX-TRIB)
           END-SEARCH.

SORT e MERGE — Ordenação de Arquivos

SORT com INPUT PROCEDURE e OUTPUT PROCEDURE
       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
         FILE-CONTROL.
           SELECT ARQ-ENTRADA ASSIGN TO 'ENTRADA.DAT'
               ORGANIZATION IS SEQUENTIAL.
           SELECT ARQ-SAIDA ASSIGN TO 'SAIDA.DAT'
               ORGANIZATION IS SEQUENTIAL.
           SELECT ARQ-SORT ASSIGN TO SORT-WORK.

       DATA DIVISION.
       FILE SECTION.
         FD ARQ-ENTRADA.
         01 REG-ENTRADA.
           05 RE-CODIGO    PIC 9(6).
           05 RE-NOME      PIC X(40).
           05 RE-DATA      PIC 9(8).       *> AAAAMMDD
           05 RE-VALOR     PIC 9(10)V99.

         SD ARQ-SORT.                      *> SD = Sort Description (não FD)
         01 REG-SORT.
           05 RS-CODIGO    PIC 9(6).
           05 RS-DATA      PIC 9(8).
           05 RS-NOME      PIC X(40).
           05 RS-VALOR     PIC 9(10)V99.

       WORKING-STORAGE SECTION.
         01 WS-TOTAL-REGISTROS PIC 9(7) VALUE 0.
         01 WS-TOTAL-VALOR     PIC 9(12)V99 VALUE 0.

       PROCEDURE DIVISION.
       MAIN-SORT.
      *> Ordenar por DATA ASCENDING e CODIGO ASCENDING
      *> com filtros no INPUT PROCEDURE
           SORT ARQ-SORT
               ON ASCENDING KEY RS-DATA
               ON ASCENDING KEY RS-CODIGO
               INPUT PROCEDURE IS PREPARA-INPUT
               OUTPUT PROCEDURE IS PROCESSA-OUTPUT.

           DISPLAY 'Total: ' WS-TOTAL-REGISTROS
                   ' Valor: ' WS-TOTAL-VALOR.
           STOP RUN.

       PREPARA-INPUT.
           OPEN INPUT ARQ-ENTRADA.
           PERFORM UNTIL EOF-ENTRADA
               READ ARQ-ENTRADA AT END
                   MOVE 'S' TO WS-EOF
               NOT AT END
      *>          Filtrar apenas registros de 2024
                   IF RE-DATA(1:4) = '2024'
                       MOVE RE-CODIGO TO RS-CODIGO
                       MOVE RE-DATA   TO RS-DATA
                       MOVE RE-NOME   TO RS-NOME
                       MOVE RE-VALOR  TO RS-VALOR
                       RELEASE REG-SORT    *> enviar para sort
                   END-IF
               END-READ
           END-PERFORM.
           CLOSE ARQ-ENTRADA.

       PROCESSA-OUTPUT.
           OPEN OUTPUT ARQ-SAIDA.
           PERFORM UNTIL EOF-SORT
               RETURN ARQ-SORT AT END    *> RETURN = ler do stream ordenado
                   MOVE 'S' TO WS-EOF-SORT
               NOT AT END
                   ADD 1 TO WS-TOTAL-REGISTROS
                   ADD RS-VALOR TO WS-TOTAL-VALOR
                   WRITE REG-SAIDA FROM REG-SORT
               END-RETURN
           END-PERFORM.
           CLOSE ARQ-SAIDA.

Subprogramas — CALL

CALL estático e dinâmico — inter-program communication
      *> ── Programa chamador ────────────────────────────────────
       WORKING-STORAGE SECTION.
       01 WS-ENTRADA         PIC 9(7)V99.
       01 WS-RESULTADO       PIC 9(7)V99.
       01 WS-STATUS-CALL     PIC 9(2).
       01 WS-NOME-PROGRAMA   PIC X(8).

       PROCEDURE DIVISION.
      *> CALL estático — nome literal
           CALL 'CALCJUROS'
               USING BY REFERENCE WS-ENTRADA
                     BY CONTENT  1.5               *> taxa = 1.5%
                     BY REFERENCE WS-RESULTADO
               ON EXCEPTION
                   DISPLAY 'Programa CALCJUROS não encontrado'
           END-CALL.

      *> CALL dinâmico — nome em variável
           MOVE 'CALCJUROS' TO WS-NOME-PROGRAMA.
           CALL WS-NOME-PROGRAMA
               USING WS-ENTRADA WS-RESULTADO.

      *> ── Subprograma chamado (CALCJUROS.cob) ──────────────────
       IDENTIFICATION DIVISION.
       PROGRAM-ID. CALCJUROS.

       DATA DIVISION.
       LINKAGE SECTION.                    *> LINKAGE = parâmetros recebidos
         01 LS-PRINCIPAL  PIC 9(7)V99.
         01 LS-TAXA       PIC 9(3)V99.
         01 LS-RESULTADO  PIC 9(7)V99.

       PROCEDURE DIVISION USING LS-PRINCIPAL LS-TAXA LS-RESULTADO.
           COMPUTE LS-RESULTADO ROUNDED =
               LS-PRINCIPAL * (1 + LS-TAXA / 100).
           EXIT PROGRAM.                   *> retornar ao chamador

REDEFINES — acesso alternativo ao mesmo espaço de memória
       WORKING-STORAGE SECTION.
      *> REDEFINES permite ver o mesmo bytes de 2 formas diferentes

       01 WS-DATA-NUMERICA    PIC 9(8).          *> 20241225
       01 WS-DATA-FORMATADA REDEFINES WS-DATA-NUMERICA.
         05 WS-ANO           PIC 9(4).           *> 2024
         05 WS-MES           PIC 9(2).           *> 12
         05 WS-DIA           PIC 9(2).           *> 25

      *> Registro com layout variável
       01 WS-REGISTRO.
         05 WS-TIPO-REG      PIC X.
         05 WS-DADOS-COMUNS  PIC X(79).

       01 WS-REG-PESSOA REDEFINES WS-REGISTRO.
         05 FILLER           PIC X.
         05 WS-CPF           PIC X(11).
         05 WS-NOME-PESSOA   PIC X(40).
         05 WS-NASCIMENTO    PIC 9(8).
         05 FILLER           PIC X(20).

       01 WS-REG-EMPRESA REDEFINES WS-REGISTRO.
         05 FILLER           PIC X.
         05 WS-CNPJ          PIC X(14).
         05 WS-RAZAO-SOCIAL  PIC X(60).
         05 FILLER           PIC X(5).

       PROCEDURE DIVISION.
           MOVE 20241225 TO WS-DATA-NUMERICA.
           DISPLAY WS-ANO '-' WS-MES '-' WS-DIA.  *> 2024-12-25

           EVALUATE WS-TIPO-REG
               WHEN 'P'  PERFORM PROCESSAR-PESSOA
               WHEN 'E'  PERFORM PROCESSAR-EMPRESA
           END-EVALUATE.

Embedded SQL — EXEC SQL

COBOL + DB2 — SELECT, INSERT, cursor com FETCH
       WORKING-STORAGE SECTION.
       EXEC SQL INCLUDE SQLCA END-EXEC.      *> SQL Communication Area

       01 WS-ID-CLIENTE     PIC 9(6).
       01 WS-NOME-CLIENTE   PIC X(40).
       01 WS-SALDO          PIC 9(10)V99.

       EXEC SQL
           DECLARE C1 CURSOR FOR
           SELECT NOME, SALDO
           FROM CLIENTES
           WHERE STATUS = 'A'
             AND SALDO > :WS-SALDO
           ORDER BY SALDO DESC
       END-EXEC.

       PROCEDURE DIVISION.
      *> ── SELECT INTO (uma linha) ──────────────────────────────
           MOVE 12345 TO WS-ID-CLIENTE.
           EXEC SQL
               SELECT NOME, SALDO
               INTO :WS-NOME-CLIENTE, :WS-SALDO
               FROM CLIENTES
               WHERE ID = :WS-ID-CLIENTE
           END-EXEC.

           EVALUATE SQLCODE
               WHEN 0
                   DISPLAY 'Cliente: ' WS-NOME-CLIENTE
               WHEN 100
                   DISPLAY 'Não encontrado (SQLCODE=100)'
               WHEN OTHER
                   DISPLAY 'Erro SQL: ' SQLCODE
                   DISPLAY SQLERRM
           END-EVALUATE.

      *> ── Cursor para múltiplas linhas ─────────────────────────
           MOVE 5000.00 TO WS-SALDO.
           EXEC SQL OPEN C1 END-EXEC.

           PERFORM UNTIL SQLCODE = 100
               EXEC SQL
                   FETCH C1 INTO :WS-NOME-CLIENTE, :WS-SALDO
               END-EXEC
               IF SQLCODE = 0
                   DISPLAY WS-NOME-CLIENTE SPACE WS-SALDO
               END-IF
           END-PERFORM.

           EXEC SQL CLOSE C1 END-EXEC.

      *> ── INSERT / UPDATE ───────────────────────────────────────
           EXEC SQL
               INSERT INTO TRANSACOES (ID, CLIENT_ID, VALOR, DT_TRANSACAO)
               VALUES (SEQ_TRANS.NEXTVAL, :WS-ID-CLIENTE,
                       :WS-SALDO, CURRENT_DATE)
           END-EXEC.

           EXEC SQL COMMIT END-EXEC.

O SEARCH ALL exige que a tabela esteja ordenada (ASCENDING KEY declarado) e realiza busca binária em O(log n) — enquanto SEARCH faz busca linear O(n). Para tabelas de código fiscal com centenas de alíquotas ou tarifas, a diferença entre 100 comparações e 7 comparações importa quando o mesmo código executa milhões de vezes por dia em batch.
