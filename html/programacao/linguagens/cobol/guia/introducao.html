COBOL — Introdução

COBOL (Common Business-Oriented Language) foi criado em 1959 sob liderança de Grace Hopper e do comitê CODASYL, com objetivo de criar uma linguagem legível por humanos (não por matemáticos) para processamento de dados comerciais. Em 1960, rodou no mesmo programa em dois computadores diferentes — o primeiro programa portável da história.

Em 2024, estima-se que 800 a 900 bilhões de linhas de COBOL estejam em produção ativa globalmente, executando pelo menos 95% das transações de cartão de crédito, 80% das transações em pontos de venda e a grande maioria das transações bancárias mundiais. É a linguagem mais executada do planeta — invisível para o público, indispensável para o sistema financeiro.

Estrutura de um Programa COBOL

As 4 Divisões obrigatórias
      *> Colunas: 1-6 sequência, 7 indicador, 8-11 área A, 12-72 área B
      *> Formato livre (COBOL-2002+) não requer colunas fixas

       IDENTIFICATION DIVISION.
      *> ─── Identifica o programa ────────────────────────────────
       PROGRAM-ID.    MEU-PROGRAMA.
       AUTHOR.        JOAO SILVA.
       DATE-WRITTEN.  2024-01-15.
       SECURITY.      CONFIDENTIAL.

       ENVIRONMENT DIVISION.
      *> ─── Descreve hardware/SO e arquivos externos ─────────────
       CONFIGURATION SECTION.
         SOURCE-COMPUTER.  IBM-MAINFRAME.
         OBJECT-COMPUTER.  IBM-MAINFRAME.
       INPUT-OUTPUT SECTION.
         FILE-CONTROL.
           SELECT ARQUIVO-CLIENTES
             ASSIGN TO 'CLIENTES.DAT'
             ORGANIZATION IS SEQUENTIAL
             ACCESS MODE IS SEQUENTIAL.

       DATA DIVISION.
      *> ─── Declara todas as variáveis e estruturas de dados ──────
       FILE SECTION.
         FD ARQUIVO-CLIENTES
           RECORD CONTAINS 80 CHARACTERS.
         01 REG-CLIENTE          PIC X(80).

       WORKING-STORAGE SECTION.
         01 WS-CONTADOR          PIC 9(5) VALUE 0.
         01 WS-TOTAL-VALOR       PIC 9(10)V99 VALUE 0.
         01 WS-FIM-ARQUIVO       PIC X VALUE 'N'.
           88 FIM-ARQUIVO        VALUE 'S'.
           88 NAO-FIM-ARQUIVO    VALUE 'N'.

       PROCEDURE DIVISION.
      *> ─── Lógica do programa ─────────────────────────────────────
       MAIN-PARA.
           PERFORM INICIALIZAR
           PERFORM PROCESSAR-ARQUIVO UNTIL FIM-ARQUIVO
           PERFORM FINALIZAR
           STOP RUN.

       INICIALIZAR.
           OPEN INPUT ARQUIVO-CLIENTES.

       PROCESSAR-ARQUIVO.
           READ ARQUIVO-CLIENTES
             AT END MOVE 'S' TO WS-FIM-ARQUIVO
           END-READ.

       FINALIZAR.
           CLOSE ARQUIVO-CLIENTES
           DISPLAY 'Total processado: ' WS-CONTADOR.

Primeiro Programa — Hello World

Hello World — GnuCOBOL formato livre
      *> Compilar: cobc -x -free hello.cob -o hello
      *> Executar: ./hello

       IDENTIFICATION DIVISION.
       PROGRAM-ID. HELLO.

       DATA DIVISION.
       WORKING-STORAGE SECTION.
         01 WS-NOME     PIC X(40) VALUE SPACES.
         01 WS-MENSAGEM PIC X(60).

       PROCEDURE DIVISION.
       MAIN-PARA.
           DISPLAY 'Qual é o seu nome? ' WITH NO ADVANCING
           ACCEPT WS-NOME

           STRING 'Olá, ' DELIMITED BY SPACE
                  WS-NOME DELIMITED BY SPACE
                  '! Bem-vindo ao COBOL.' DELIMITED BY SIZE
                  INTO WS-MENSAGEM

           DISPLAY WS-MENSAGEM
           STOP RUN.

Por que COBOL ainda importa?

Volume de Transações

O Federal Reserve dos EUA processa cerca de 95 milhões de transações ACH por dia — quase todas em COBOL. O mercado de câmbio mundial (US$7 trilhões/dia) é dominado por sistemas COBOL em mainframes IBM zSeries.

Decimais Exatos

COBOL usa arithmetic decimal nativo (BCD — Binary Coded Decimal) em vez de ponto flutuante binário. PIC 9(10)V99 representa exatamente R$ 12345678901.23 sem o erro de arredondamento de double em C/Java.

Batch Processing

COBOL foi otimizado para processar lotes de milhões de registros sequencialmente (arquivos VSAM, JCL no mainframe). O modelo SORT/MERGE/REPORT é imbatível para relatórios financeiros de final de mês.

Legado e Emprego

Há déficit crítico de programadores COBOL. Bancos e governos pagam salários premium — em 2023 o banco New Jersey pagou US$100/hora por contratos de manutenção COBOL emergencial.

Certificações IBM

A IBM mantém o Enterprise COBOL para mainframes z/OS. Há certificação oficial IBM: C2090-612 e módulos zLearnings. O COBOL moderno suporta OOP, JSON, XML e interfaces REST.

GnuCOBOL (Open Source)

GnuCOBOL compila programas COBOL para C e gera binários nativos no Linux, Windows e macOS. É a escolha para aprender e para sistemas não-mainframe. Compatível com COBOL-85 e COBOL-2002.
