COBOL — Fundamentos

O coração do COBOL é o sistema de tipos da PICTURE clause — uma notação de máscara que define formato exato de dados: PIC 9(7)V99 representa um número de 7 dígitos inteiros e 2 decimais exatos (sem ponto flutuante). Este sistema tornou COBOL ideal para dados financeiros onde "erros de centavo" são inaceitáveis.

Ao contrário de outras linguagens que têm "variáveis", o COBOL tem data items organizados em níveis hierárquicos (01 a 49, mais especiais 66, 77, 88). Cada item descreve a estrutura física dos dados — quantos bytes, que charset, como são formatados ao exibir — de forma completamente explícita.

PICTURE Clause — Sistema de Tipos

Tipos de dados PICTURE — guia completo
       WORKING-STORAGE SECTION.

      *> ── Tipos numéricos ──────────────────────────────────────
       01 WS-INT-PEQUENO     PIC 9(3).           *> 0 a 999 (3 dígitos)
       01 WS-INT-GRANDE      PIC 9(9).           *> 0 a 999999999
       01 WS-SALDO           PIC 9(10)V99.       *> 9999999999.99 (V=vírgula decimal implícita)
       01 WS-SALDO-SINAL     PIC S9(10)V99.      *> com sinal (S=signed)
       01 WS-PERCENTUAL      PIC V9(4).          *> 0.0000 a 0.9999

      *> ── Tipos alfanuméricos ──────────────────────────────────
       01 WS-NOME            PIC X(40).          *> string de 40 chars
       01 WS-INICIAL         PIC X.              *> 1 char
       01 WS-CODIGO          PIC A(5).           *> somente letras (A=alphabetic)

      *> ── Edição numérica (para DISPLAY) ──────────────────────
       01 WS-EXIBIR-VALOR    PIC $ZZZ,ZZ9.99.    *> $ Z=suprimir zeros . , separadores
       01 WS-EXIBIR-MAIS     PIC $B(5)9(7).99-.  *> B=espaço, - = sinal se negativo
       01 WS-CPF             PIC 999B999B999B99. *> 000 000 000 00 formatado

      *> ── Nivelamento hierárquico ───────────────────────────────
       01 WS-ENDERECO.
         05 WS-RUA          PIC X(50).
         05 WS-NUMERO       PIC 9(5).
         05 WS-COMPLEMENTO  PIC X(20).
         05 WS-BAIRRO       PIC X(30).
         05 WS-CIDADE       PIC X(30).
         05 WS-UF           PIC XX.
         05 WS-CEP.
           10 WS-CEP-1      PIC 9(5).          *> 3 níveis de hierarquia
           10 WS-CEP-2      PIC 9(3).

      *> ── Condition Names (nível 88) ───────────────────────────
       01 WS-STATUS-CLIENTE  PIC X.
         88 CLIENTE-ATIVO    VALUE 'A'.
         88 CLIENTE-INATIVO  VALUE 'I'.
         88 CLIENTE-BLOQUEADO VALUE 'B'.

       PROCEDURE DIVISION.
           IF CLIENTE-ATIVO             *> testa: WS-STATUS-CLIENTE = 'A'
               DISPLAY 'Cliente ativo'
           END-IF.

MOVE, ADD, SUBTRACT, MULTIPLY, DIVIDE e COMPUTE

Operações aritméticas e atribuição COBOL
       WORKING-STORAGE SECTION.
       01 WS-A      PIC 9(5)V99 VALUE 1000.50.
       01 WS-B      PIC 9(5)V99 VALUE 200.75.
       01 WS-C      PIC 9(6)V99.
       01 WS-TAXA   PIC V9(4) VALUE .1500.   *> 15.00%
       01 WS-REM    PIC 9(3).

       PROCEDURE DIVISION.
       CALCULOS.
      *> ── MOVE — atribuição ────────────────────────────────────
           MOVE WS-A TO WS-C.              *> WS-C = WS-A
           MOVE 0 TO WS-C WS-B WS-A.      *> zerar múltiplos campos
           MOVE SPACES TO WS-ENDERECO.    *> preencher grupo com espaços
           MOVE ZEROS  TO WS-SALDO.       *> preencher numérico com zeros

      *> ── ADD ──────────────────────────────────────────────────
           ADD WS-A TO WS-C.              *> WS-C = WS-C + WS-A
           ADD WS-A WS-B TO WS-C.         *> WS-C = WS-C + WS-A + WS-B
           ADD WS-A TO WS-B GIVING WS-C.  *> WS-C = WS-A + WS-B (WS-B não muda)

      *> ── SUBTRACT ─────────────────────────────────────────────
           SUBTRACT WS-B FROM WS-A.       *> WS-A = WS-A - WS-B
           SUBTRACT WS-B FROM WS-A GIVING WS-C.

      *> ── MULTIPLY ─────────────────────────────────────────────
           MULTIPLY WS-A BY WS-TAXA GIVING WS-C ROUNDED.
      *> ROUNDED arredonda usando regra de negócio correta

      *> ── DIVIDE ───────────────────────────────────────────────
           DIVIDE 3 INTO WS-A GIVING WS-C REMAINDER WS-REM.

      *> ── COMPUTE — expressões complexas ──────────────────────
           COMPUTE WS-C ROUNDED =
               (WS-A + WS-B) * WS-TAXA / 100.

      *> ── ON SIZE ERROR — overflow aritmético ─────────────────
           COMPUTE WS-C = WS-A * WS-B
               ON SIZE ERROR
                   DISPLAY 'ERRO: overflow aritmético!'
                   MOVE 0 TO WS-C
               NOT ON SIZE ERROR
                   DISPLAY 'Resultado: ' WS-C
           END-COMPUTE.

Controle de Fluxo

IF/ELSE, EVALUATE (switch), PERFORM (loop)
      *> ── IF / ELSE ─────────────────────────────────────────────
           IF WS-SALDO > 1000
               DISPLAY 'Saldo alto'
           ELSE IF WS-SALDO > 0
               DISPLAY 'Saldo positivo'
           ELSE
               DISPLAY 'Saldo zerado ou negativo'
           END-IF.

      *> ── EVALUATE (switch/match) ───────────────────────────────
           EVALUATE WS-STATUS-CLIENTE
               WHEN 'A'
                   PERFORM PROCESSAR-ATIVO
               WHEN 'I'
                   PERFORM PROCESSAR-INATIVO
               WHEN 'B'
                   DISPLAY 'Cliente bloqueado'
               WHEN OTHER
                   DISPLAY 'Status desconhecido'
           END-EVALUATE.

      *> ── EVALUATE com múltiplas condições ─────────────────────
           EVALUATE TRUE
               WHEN WS-SALDO > 10000 AND WS-TIPO = 'P'
                   PERFORM CLIENTE-PREMIUM
               WHEN WS-SALDO > 0
                   PERFORM CLIENTE-PADRAO
               WHEN OTHER
                   PERFORM CLIENTE-NEGATIVO
           END-EVALUATE.

      *> ── PERFORM (loop e subprograma) ─────────────────────────
       *> Chamar parágrafo
           PERFORM INICIALIZAR.

       *> Loop com contador
           PERFORM VARYING WS-I FROM 1 BY 1
               UNTIL WS-I > 100
               ADD WS-I TO WS-SOMA
           END-PERFORM.

       *> Loop com condição
           PERFORM UNTIL FIM-ARQUIVO
               PERFORM LER-PROXIMO-REGISTRO
               ADD 1 TO WS-CONTADOR
           END-PERFORM.

       *> Inline perform (código direto sem nomear parágrafo)
           PERFORM
               DISPLAY 'Executando inline'
               ADD 1 TO WS-CONTADOR
           END-PERFORM.

Manipulação de Strings

STRING, UNSTRING, INSPECT, MOVE
       WORKING-STORAGE SECTION.
       01 WS-NOME-COMPLETO  PIC X(80).
       01 WS-PRIMEIRO-NOME  PIC X(20).
       01 WS-SOBRENOME      PIC X(40).
       01 WS-RESULTADO      PIC X(120).
       01 WS-PONTEIRO       PIC 9(3).
       01 WS-CONTAGEM       PIC 9(5).

       PROCEDURE DIVISION.
      *> ── STRING — concatenar campos ───────────────────────────
           MOVE 'João' TO WS-PRIMEIRO-NOME.
           MOVE 'Silva' TO WS-SOBRENOME.

           STRING WS-PRIMEIRO-NOME DELIMITED BY SPACE
                  ' ' DELIMITED BY SIZE
                  WS-SOBRENOME DELIMITED BY SPACE
                  INTO WS-NOME-COMPLETO.
      *> WS-NOME-COMPLETO = 'João Silva'

      *> ── UNSTRING — dividir campo ─────────────────────────────
           MOVE 'João Silva Santos' TO WS-NOME-COMPLETO.
           UNSTRING WS-NOME-COMPLETO DELIMITED BY SPACE
               INTO WS-PRIMEIRO-NOME
                    WS-SOBRENOME.
      *> WS-PRIMEIRO-NOME = 'João', WS-SOBRENOME = 'Silva'

      *> ── INSPECT — analisar e transformar ─────────────────────
           MOVE 'Hello, World!' TO WS-RESULTADO.

       *> Contar ocorrências
           INSPECT WS-RESULTADO
               TALLYING WS-CONTAGEM
               FOR ALL 'l'.
      *> WS-CONTAGEM = 3

       *> Substituir caracteres
           INSPECT WS-RESULTADO
               CONVERTING LOWER-ALPHA TO UPPER-ALPHA.
      *> WS-RESULTADO = 'HELLO, WORLD!'

           INSPECT WS-RESULTADO
               REPLACING ALL 'L' BY 'x'.
      *> WS-RESULTADO = 'HExxO, WORxD!'

      *> ── Funções intrínsecas de string ────────────────────────
           DISPLAY FUNCTION LENGTH(WS-NOME-COMPLETO).       *> 80 (tamanho PIC)
           DISPLAY FUNCTION TRIM(WS-NOME-COMPLETO TRAILING). *> sem espaços finais
           DISPLAY FUNCTION UPPER-CASE(WS-NOME-COMPLETO).
           DISPLAY FUNCTION LOWER-CASE(WS-NOME-COMPLETO).
           DISPLAY FUNCTION REVERSE(WS-NOME-COMPLETO).

A diferença entre PIC 9(5)V99 e PIC 9(5).99 é crítica: V indica ponto decimal implícito (não ocupa espaço na memória — é apenas para controle aritmético), enquanto . seria o ponto explícito inserido no dado. Use V para dados internos, e cláusulas de edição (ZZZ,ZZ9.99) apenas para campos de saída/relatório.
