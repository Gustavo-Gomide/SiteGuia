COBOL — Ecossistema e Tooling

O ecossistema COBOL tem dois mundos: o mainframe IBM com z/OS, Enterprise COBOL, JCL e VSAM; e o mundo open-source com GnuCOBOL rodando em Linux/Windows. Ferramentas modernas como VS Code + extensão COBOL e Zowe CLI estão conectando os dois mundos.

Dominar o toolchain COBOL — compilador, linker, JCL e debugger — é tão importante quanto a linguagem em si. A maioria dos empregos COBOL está em bancos, governo e seguradoras que operam mainframe IBM z/OS.

GnuCOBOL — Instalação e Compilação

Instalação e comandos cobc essenciais
# ── Instalação ──────────────────────────────────────────────
# Ubuntu/Debian
sudo apt install gnucobol

# Fedora/RHEL
sudo dnf install gnucobol

# macOS (Homebrew)
brew install gnucobol

# Windows — usar WSL2 (Ubuntu) ou gnucobol4-win
# Download: https://gnucobol.sourceforge.io

# ── Compilar executável (formato livre) ─────────────────────
cobc -x -free hello.cob -o hello
./hello

# ── Compilar executável (formato fixo – padrão IBM) ─────────
cobc -x hello.cob -o hello

# ── Módulo de objeto para linking posterior ─────────────────
cobc -c -free calcjuros.cob              # gera calcjuros.o
cobc -x -free main.cob calcjuros.o -o programa

# ── Compilação com otimização ────────────────────────────────
cobc -x -O2 -free main.cob -o programa

# ── Flags úteis ─────────────────────────────────────────────
cobc -x -free \
     -Wall \                 # todos os warnings
     -fdebugging-mode \      # habilitar USE FOR DEBUGGING
     -fsource-location \     # incluir info de linha no objeto
     main.cob -o programa
Flags do cobc
FlagSignificado
-xCompilar executável (link automático)
-cGerar arquivo objeto .o sem linkar
-freeFormato livre (sem colunas fixas)
-fixedFormato fixo IBM (padrão)
-WallTodos os diagnosticos de aviso
-O2Otimização nível 2
-gSímbolos de debug (para GDB)
-fmainGerar ponto de entrada main() para linking manual
-std=cobol2014Aderência ao padrão ISO 2014
-std=ibm-strictCompatibilidade com dialeto IBM

IBM Enterprise COBOL e z/OS

JCL — Job Control Language básico para mainframe
//COMPJOB  JOB (ACCT),'COMPILAR COBOL',CLASS=A,MSGCLASS=X
//
//* Compilar programa COBOL com IBM Enterprise COBOL
//
//COBRUN   EXEC IGYWCL                        // proc de compilação IBM
//COBOL.SYSIN DD DSN=MINHABI.SOURCE(MEUPROG),DISP=SHR
//LKED.SYSLMOD DD DSN=MINHABI.LOAD(MEUPROG),DISP=SHR
//
//* Executar o programa compilado
//
//EXECJOB  JOB (ACCT),'EXECUTAR',CLASS=A,MSGCLASS=X
//STEP01   EXEC PGM=MEUPROG
//STEPLIB  DD DSN=MINHABI.LOAD,DISP=SHR
//SYSOUT   DD SYSOUT=*
//ENTRADA  DD DSN=MINHABI.DATA.ENTRADA(0),DISP=SHR  // GDG base(0)=gen atual
//SAIDA    DD DSN=MINHABI.DATA.SAIDA,
//            DISP=(NEW,CATLG,DELETE),
//            SPACE=(CYL,(10,5),RLSE),           // 10 cilindros, extend 5
//            DCB=(RECFM=FB,LRECL=200,BLKSIZE=0)  // Fixed-Blocked, 200 bytes
JCL não é COBOL — é a linguagem de controle de jobs do z/OS. Todo programa COBOL em mainframe é submetido via JCL. DD (Data Definition) define datasets de entrada/saída. DISP=(NEW,CATLG,DELETE) significa: criar novo; se sucesso catalogar; se erro deletar.

Debugging — GDB, Trace e USE FOR DEBUGGING

Debugging com GDB e DISPLAY de diagnóstico
# ── Compilar com símbolos de debug ──────────────────────────
cobc -x -free -g -fsource-location programa.cob -o programa

# ── GDB básico ───────────────────────────────────────────────
gdb ./programa

(gdb) break PROCESSAR-CLIENTES     # breakpoint em parágrafo COBOL
(gdb) run
(gdb) print ws_nome_cliente        # vars ficam snake_case em C gerado
(gdb) next
(gdb) continue
USE FOR DEBUGGING — trace nativo COBOL
       PROCEDURE DIVISION.
       DECLARATIVES.
       DEBUG-SECTION SECTION.
           USE FOR DEBUGGING ON ALL PROCEDURES.
       DEBUG-PARAGRAPH.
           DISPLAY 'TRACE: ' DEBUG-NAME
                   ' LINE: ' DEBUG-LINE
                   ' CONTEUDO: ' DEBUG-CONTENTS.
       END DECLARATIVES.

       INICIO.
      *> Compilar com: cobc -x -free -fdebugging-mode prog.cob
      *> Todas as chamadas de parágrafo serão logadas
           PERFORM PROCESSAR-CLIENTES.
           PERFORM GERAR-RELATORIO.
           STOP RUN.

Makefile para Projetos COBOL

Makefile — compilar projeto multi-módulo
COBC    = cobc
COBFLGS = -x -free -Wall -O2
LDFLAGS =

SRCS    = src/main.cob src/utils.cob src/relatorio.cob
TARGET  = bin/meuprograma

.PHONY: all clean

all: $(TARGET)

$(TARGET): $(SRCS)
	mkdir -p bin
	$(COBC) $(COBFLGS) $(SRCS) -o $@

debug: COBFLGS = -x -free -Wall -g -fsource-location -fdebugging-mode
debug: $(TARGET)

clean:
	rm -f bin/* *.o
