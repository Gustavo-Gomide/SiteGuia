COBOL — Projetos e Boas Práticas

Programas COBOL corporativos vivem décadas. O código escrito em 1985 ainda processa transações bancárias hoje. As boas práticas estabelecidas pela indústria — copybooks compartilhados, naming conventions, parágrafos curtos — existem exatamente para que esse código seja compreendido e mantido por gerações de programadores.

A regra de ouro do COBOL corporativo: "If it's not broken, don't fix it." — mas quando é preciso mudar, o código precisa ser legível. Copybooks, nomes descritivos e parágrafos bem delimitados são a diferença entre manutenção simples e meses de investigação.

Usando COPY em programas
       WORKING-STORAGE SECTION.
       COPY CLIENTE.                       *> inclui WS-CLIENTE
       COPY SQLCA.                         *> Communication Area DB2 (IBM)
       COPY MENSAGENS.                     *> textos de erro/sucesso

      *> COPY com REPLACING — personalizar prefixos
       COPY CLIENTE REPLACING ==WS-CLI== BY ==HD-CLI==.
      *> Gera HD-CLI-ID, HD-CLI-CPF, etc — ideal para header de relatório

       01 WS-AREA-TRABALHO.
         05 WS-COD-RETORNO    PIC 9(4).
         05 WS-MSG-RETORNO    PIC X(80).

Parágrafos Curtos — Regra das 20 Linhas

Ruim vs. bom — parágrafo monolítico vs. decomposto
      *> ── RUIM: parágrafo de 100 linhas ───────────────────────
       PROCESSAR.
           OPEN INPUT ARQ-CLIENTES.
           PERFORM UNTIL EOF-CLI
               READ ARQ-CLIENTES AT END MOVE 'S' TO SW-EOF-CLI
               NOT AT END
                   IF FS-CLI-STATUS = 'A'
                       COMPUTE WS-IMPOSTO ROUNDED =
                           FS-CLI-RENDA * 0.275
                       IF WS-IMPOSTO > 50000
                           ... *> mais 80 linhas aqui
                       END-IF
                   END-IF
               END-READ
           END-PERFORM.
           CLOSE ARQ-CLIENTES.

      *> ── BOM: parágrafos delimitados, PERFORM claro ──────────
       1000-PROCESSAR-CLIENTES.
           OPEN INPUT ARQ-CLIENTES.
           PERFORM 1100-LER-E-PROCESSAR
               UNTIL SW-FIM-ARQUIVO = 'S'.
           CLOSE ARQ-CLIENTES.

       1100-LER-E-PROCESSAR.
           READ ARQ-CLIENTES
               AT END MOVE 'S' TO SW-FIM-ARQUIVO
               NOT AT END PERFORM 1200-VALIDAR-CLIENTE
           END-READ.

       1200-VALIDAR-CLIENTE.
           IF CLI-ATIVO
               PERFORM 1300-CALCULAR-IMPOSTO
               PERFORM 1400-GRAVAR-RESULTADO
           END-IF.

       1300-CALCULAR-IMPOSTO.
           COMPUTE WS-IMPOSTO ROUNDED =
               FS-CLI-RENDA * WS-ALIQUOTA-IRPF.
           IF WS-IMPOSTO > WS-LIMITE-ADICIONAL
               PERFORM 1310-CALCULAR-ADICIONAL
           END-IF.

       1400-GRAVAR-RESULTADO.
           MOVE WS-IMPOSTO TO FS-SAIDA-IMPOSTO.
           WRITE REG-SAIDA.
Boas Práticas
  • Parágrafos com no máximo 20 linhas
  • Nomes de parágrafos descritivos e numerados
  • PERFORM em vez de GOTO
  • Copybooks para todos os layouts de registro
  • Level-88 para todos os valores de switch
  • ON SIZE ERROR em todo COMPUTE/DIVIDE
  • FILE STATUS em todos os arquivos
  • Comentários de cabeçalho com versão e autor
Evitar
  • GOTO (cria spaghetti code inauditável)
  • ALTER GOTO (altera o destino de um GO TO — obsoleto)
  • Parágrafos com centenas de linhas
  • Nomes genéricos: PROCESS, LOOP, TEMP
  • Magic numbers sem nomes de nível-88
  • COMPUTE sem ON SIZE ERROR em campos críticos
  • Ignorar FILE STATUS após READ/WRITE
  • Copiar código (copy-paste) em vez de CALL

FILE STATUS — Tratamento de Erros de Arquivo

Verificação completa de FILE STATUS
       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
         FILE-CONTROL.
           SELECT ARQ-CLIENTES ASSIGN TO 'CLIENTES.DAT'
               ORGANIZATION IS SEQUENTIAL
               FILE STATUS IS WS-STATUS-CLIENTE.

       WORKING-STORAGE SECTION.
         01 WS-STATUS-CLIENTE PIC XX.

       01 WS-STATUS-TABLE.
         05 FILLER PIC XX VALUE '00'. 88 FS-OK VALUE '00'.
         05 FILLER PIC XX VALUE '10'. 88 FS-EOF VALUE '10'.
         05 FILLER PIC XX VALUE '23'. 88 FS-NOT-FOUND VALUE '23'.
         05 FILLER PIC XX VALUE '35'. 88 FS-NO-FILE VALUE '35'.
         05 FILLER PIC XX VALUE '39'. 88 FS-ATTR-CONFLICT VALUE '39'.
         05 FILLER PIC XX VALUE '41'. 88 FS-ALREADY-OPEN VALUE '41'.

       PROCEDURE DIVISION.
           OPEN INPUT ARQ-CLIENTES.
           EVALUATE WS-STATUS-CLIENTE
               WHEN '00' CONTINUE
               WHEN '35' DISPLAY 'Arquivo não existe'
                         STOP RUN
               WHEN OTHER DISPLAY 'Erro OPEN: ' WS-STATUS-CLIENTE
                          STOP RUN
           END-EVALUATE.

           READ ARQ-CLIENTES
               AT END MOVE 'S' TO SW-FIM-ARQUIVO.

           EVALUATE WS-STATUS-CLIENTE
               WHEN '00' PERFORM PROCESSAR-REGISTRO
               WHEN '10' MOVE 'S' TO SW-FIM-ARQUIVO
               WHEN OTHER DISPLAY 'Erro READ: ' WS-STATUS-CLIENTE
                          PERFORM 9000-ENCERRAR-ERRO
           END-EVALUATE.

O COPY ... REPLACING é uma das ferramentas mais poderosas do COBOL para reutilização: um único copybook de layout de cliente pode ser incluído com prefixos diferentes (WS-CLI, HD-CLI, LS-CLI) em diferentes seções do mesmo programa sem conflito de nomes — algo que linguagens modernas resolvem com escopos mas que o COBOL resolve de forma elegante e auditável.
