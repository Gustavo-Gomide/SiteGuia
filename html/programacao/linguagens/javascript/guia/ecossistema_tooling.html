JavaScript — Ecossistema e Tooling

Ecossistema e Tooling JavaScript

npm, Vite, ESLint, Prettier, Vitest, TypeScript e GitHub Actions: o fluxo profissional de desenvolvimento JavaScript.

O ecossistema JavaScript é rico e move-rápido. Vite substituiu Webpack e Create React App, Vitest substituiu Jest com API compatível, e o Biome promete unificar linting e formatação em um binário Rust ultra-rápido.

Gerenciadores de Pacotes

npm

Gerenciador oficial do Node.js. npm v10 com workspaces e overrides. node_modules hoist por padrao.

pnpm

Conteudo-endereçado: economiza disco com hard links. 3x mais rápido que npm. Strict mode evita phantom deps.

Yarn Berry

Plug'n'Play: sem node_modules, dependências como arquivos zip. Zero-installs via cache no repositório.

Bun install

100x mais rápido que npm segundo benchmarks. API compatível com npm. Lockfile próprio bun.lockb.

Terminal — Comandos essenciais npm e pnpm
# npm
npm init -y                     # criar package.json
npm install express zod         # instalar dependências
npm install -D vite vitest      # dev dependencies
npm run build                   # executar script "build"
npm audit fix                   # corrigir vulnerabilidades
npm outdated                    # listar pacotes desatualizados

# pnpm
pnpm install
pnpm add -D @types/node
pnpm dlx create-vite meu-app   # equivalente ao npx
pnpm workspaces foreach run build  # monorepo: rodar em todos

Vite: Build Moderno com HMR Instantaneo

vite.config.js — Configuracao completa para SPA + SSR + PWA
// vite.config.js
import { defineConfig } from "vite";
import react  from "@vitejs/plugin-react-swc";     // SWC: compilação mais rápida
import { VitePWA } from "vite-plugin-pwa";

export default defineConfig(({ command, mode }) => ({
  plugins: [
    react(),
    VitePWA({
      registerType: "autoUpdate",
      workbox: { globPatterns: ["**/*.{js,css,html,ico,png,svg}"] },
    }),
  ],

  // Aliases de importação
  resolve: {
    alias: { "@": "/src" },
  },

  // Proxy para API em dev
  server: {
    proxy: {
      "/api": { target: "http://localhost:3000", changeOrigin: true },
    },
  },

  // Otimização de build
  build: {
    target: "esnext",
    rollupOptions: {
      output: {
        // Code splitting manual
        manualChunks(id) {
          if (id.includes("node_modules/react")) return "react-vendor";
          if (id.includes("node_modules"))       return "vendor";
        },
      },
    },
  },

  // Variáveis de ambiente com tipagem
  envPrefix: "VITE_",
}));

Vitest: Testes Unitários Modernos

JavaScript + Vitest — describe, it, expect, vi.mock, coverage
// utils.test.js
import { describe, it, expect, vi, beforeEach, afterEach } from "vitest";
import { calcularDesconto, buscarPreco } from "./utils.js";

describe("calcularDesconto", () => {
  it("aplica 10% de desconto", () => {
    expect(calcularDesconto(100, 10)).toBe(90);
  });

  it.each([
    [100, 0,   100],
    [200, 50,  100],
    [150, 100,    0],
  ])("preço=%s desconto=%s% → %s", (preco, pct, esperado) => {
    expect(calcularDesconto(preco, pct)).toBe(esperado);
  });

  it("lança RangeError para desconto > 100", () => {
    expect(() => calcularDesconto(100, 101))
      .toThrow(RangeError);
  });
});

// Mockar módulo externo
vi.mock("./api.js", () => ({
  buscarPreco: vi.fn(),
}));

describe("buscarPreco", () => {
  beforeEach(() => { buscarPreco.mockResolvedValue(49.90); });
  afterEach(() => vi.clearAllMocks());

  it("retorna preço mockado", async () => {
    await expect(buscarPreco("prod-01")).resolves.toBe(49.90);
    expect(buscarPreco).toHaveBeenCalledWith("prod-01");
  });
});

// vitest.config.js — cobertura via V8
import { defineConfig } from "vitest/config";
export default defineConfig({
  test: {
    coverage: { provider: "v8", reporter: ["text", "lcov"] },
    environment: "jsdom",  // simular DOM
  },
});

// eslint.config.js — flat config (ESLint 9)
import js       from "@eslint/js";
import globals  from "globals";

export default [
  js.configs.recommended,
  {
    languageOptions: {
      globals: { ...globals.browser, ...globals.node },
      ecmaVersion: "latest",

GitHub Actions: CI/CD para Projetos JavaScript

.github/workflows/ci.yml — lint, test, build, publicar npm
name: CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [20, 22]

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: npm

      - run: npm ci
      - run: npm run lint
      - run: npm run check        # prettier
      - run: npm test -- --coverage

      - name: Upload coverage
        uses: codecov/codecov-action@v4
        with:
          token: ${{ secrets.CODECOV_TOKEN }}

      - run: npm run build

  publish:
    needs: ci
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          registry-url: https://registry.npmjs.org
      - run: npm ci
      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
