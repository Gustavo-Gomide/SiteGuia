Projetos e Boas Praticas JavaScript

Estrutura de projeto, package.json completo, modulos ESM, seguranca, OWASP Top 10 no Node.js e convencoes de nomenclatura.

Projetos JavaScript escaláveis seguem a disciplina de módulos ESM, evitam mutação descontrolada de estado, protegem segredos com variáveis de ambiente, e tratam entradas do usuário como não confiáveis por padrão.

Seguranca: OWASP no Node.js

Praticas Seguras

  • Validar toda entrada externa com Zod ou Joi antes de usar
  • Usar crypto.randomUUID() para IDs, nunca Math.random()
  • Guardar segredos em variáveis de ambiente, nunca hardcoded
  • Usar helmet no Express para headers de segurança
  • Parametrizar queries SQL — nunca concatenar strings
  • Definir Content-Security-Policy para prevenir XSS
  • Rate limiting com express-rate-limit em endpoints públicos
  • Executar npm audit em cada build de CI

Vulnerabilidades Comuns

  • Usar eval() ou new Function() com dados do usuário — RCE
  • Passar objetos não validados diretamente para queries de DB
  • Expor stack traces em respostas de erro em produção
  • Usar Math.random() para tokens de segurança
  • Desabilitar verificação SSL (NODE_TLS_REJECT_UNAUTHORIZED=0)
  • Depender de pacotes npm sem auditar (supply chain)
  • Guardar JWT secret em código-fonte versionado
Node.js (Express) — Helmet, rate-limit, sanitização e validation
import express   from "express";
import helmet    from "helmet";
import rateLimit from "express-rate-limit";
import { z }     from "zod";

const app = express();
app.use(helmet());             // X-Frame-Options, CSP, etc.
app.use(express.json({ limit: "100kb" })); // evitar payload enorme

// Rate limiting: 100 req/15min por IP
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 100,
  standardHeaders: true,
  legacyHeaders: false,
});
app.use("/api", limiter);

// Middleware de validação genérico
function validar(schema) {
  return (req, res, next) => {
    const parsed = schema.safeParse(req.body);
    if (!parsed.success) {
      return res.status(400).json({
        erro: "Dados inválidos",
        detalhes: parsed.error.flatten(),
      });
    }
    req.body = parsed.data; // só dados válidos passam
    next();
  };
}

const CadastroSchema = z.object({
  nome:  z.string().min(2).max(80),
  email: z.string().email(),
  senha: z.string().min(12),
});

app.post("/api/usuarios", validar(CadastroSchema), (req, res) => {
  // req.body já é tipado e validado
  res.status(201).json({ mensagem: "Criado" });
});
