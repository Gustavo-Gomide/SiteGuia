Solidity: Introducao

O que e Solidity e por que importa

Solidity e a base programavel da Ethereum Virtual Machine

Uma linguagem orientada a contratos para definir regras imutaveis de negocio em redes blockchain.

Se JavaScript popularizou apps web, Solidity popularizou aplicacoes descentralizadas auditaveis, onde regras de negocio rodam sem servidor central.

Conceitos essenciais e contexto historico

Solidity surgiu junto da plataforma Ethereum para resolver um problema claro: criar logica executavel em blockchain de forma geral, e nao apenas transacoes simples como no Bitcoin Script. O compilador traduz o codigo para bytecode da EVM, que e executado por milhares de nos.

A analogia mais util para iniciantes: pense em um contrato inteligente como uma API publica e imutavel. Qualquer cliente pode chamar funcoes expostas, mas o estado so muda se as regras codificadas permitirem.

Em blockchain, "confiar" sai do servidor e vai para o protocolo + codigo auditado.

Ethereum, EVM e ciclo de execucao

O fluxo tecnico e: desenvolvimento em Solidity, compilacao com `solc`, geracao de ABI e bytecode, deploy para um endereco de contrato e interacao via chamadas de funcao. Cada instrucao EVM custa gas, portanto design e eficiencia importam desde o inicio.

Pipeline minimo

  • Autor escreve contrato.
  • Compilador gera ABI (interface) e bytecode (codigo executavel).
  • Transacao de deploy cria o contrato on-chain.
  • Usuarios e dApps chamam funcoes via carteira ou SDK.

Primeiro contrato inteligente

Solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract Contador {
		uint256 public total;

		function incrementar() external {
				total += 1;
		}
}

Esse contrato expoe o estado `total` e permite incremento externo. O modificador `public` cria um getter automatico, um comportamento util para observabilidade em dApps.

Casos de uso reais

Financas descentralizadas

Emprestimos, pools de liquidez, stablecoins e automacao de mercado.

Tokenizacao

NFTs, ingressos, pontos de fidelidade e representacao de ativos digitais.

Governanca

Votacoes transparentes com regras auditaveis de quorum e execucao.

Infra de identidade

Credenciais verificaveis e registros de prova de existencia.
