Solidity em Banco de Dados

Dados imutaveis e indexacao inteligente

On-chain para verdade, off-chain para escala

Blockchain nao substitui totalmente bancos tradicionais, mas redefine o que precisa de prova criptografica.

A pergunta central nao e "onde guardar tudo", e sim "qual parte precisa de verificabilidade global".

On-chain vs off-chain

On-chain oferece integridade e consenso, mas custa caro e possui throughput limitado. Off-chain e flexivel e barato para consultas complexas. Projetos maduros combinam ambos.

On-chain

Hashes, ownership, saldos, checkpoints criticos.

Off-chain

Metadados extensos, busca textual, analytics e dashboards.

Eventos como trilha de dados

Eventos emitidos por contratos sao otimos para indexacao. Em vez de varrer estado completo a cada bloco, indexadores consomem logs e montam visoes consultaveis.

Solidity
// Emite evento para indexadores off-chain
event RegistroCriado(address usuario, bytes32 hash);

function registrar(bytes32 hash) external {
	require(hash != bytes32(0), "hash invalido");
	emit RegistroCriado(msg.sender, hash);
}

The Graph e consultas de alto nivel

Com subgraphs, eventos viram entidades consultaveis via GraphQL. Isso reduz carga no frontend e melhora latencia de leitura em dApps.

Evento bem desenhado e schema de indexacao coerente simplificam monitoramento e compliance.
