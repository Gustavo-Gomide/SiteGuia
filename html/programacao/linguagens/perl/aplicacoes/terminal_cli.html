Perl — Terminal & CLI

Perl no Terminal e CLI

Ferramentas de linha de comando profissionais: parsing de argumentos, entrada interativa, cores, relatórios e integração com o sistema operacional.

Perl é uma linguagem de terminal por natureza — nasce no Unix, conhece o shell, manipula pipes e lê arquivos com elegância. Com módulos como Getopt::Long e Term::ANSIColor, você constrói ferramentas CLI de qualidade profissional em poucas dezenas de linhas.

Getopt::Long e Getopt::Std

Getopt::Long processa argumentos de linha de comando com suporte a flags longas (--verbose), curtas (-v), opções com valor (--arquivo=dados.txt) e validação de tipos. É mais completo que Getopt::Std (apenas flags curtas de um caractere).

ferramenta.pl — CLI completa
#!/usr/bin/env perl
use strict;
use warnings;
use Getopt::Long qw(GetOptions HelpMessage);
use Pod::Usage;

# Valores padrão
my %opts = (
    verbose  => 0,
    output   => 'stdout',
    max      => 100,
    formatos => [],
);

GetOptions(
    'verbose|v'       => \$opts{verbose},     # flag booleana
    'output|o=s'      => \$opts{output},     # string obrigatória
    'max|m=i'         => \$opts{max},        # inteiro
    'formato|f=s@'    => $opts{formatos},    # string repetível (array)
    'help|h'          => sub { pod2usage(1) }, # exibe ajuda e sai
    'man'             => sub { pod2usage(-exitval => 0, -verbose => 2) },
) or pod2usage(2);  # erro de parsing: exibe uso

# Argumentos posicionais restantes em @ARGV
die "Forneça ao menos um arquivo\n" unless @ARGV;

say "Modo verbose: sim" if $opts{verbose};
say "Output: $opts{output}";
say "Máximo: $opts{max}";
say "Formatos: " . join(", ", @{$opts{formatos}})
    if @{$opts{formatos}};
say "Arquivos: " . join(", ", @ARGV);

=head1 NOME

ferramenta.pl - Processa arquivos

=head1 SYNOPSIS

  ferramenta.pl [opções] arquivo1 arquivo2 ...

  -v, --verbose       Modo detalhado
  -o, --output=DEST   Destino da saída (padrão: stdout)
  -m, --max=N         Número máximo de linhas (padrão: 100)
  -f, --formato=FMT   Formato (pode repetir)
  -h, --help          Esta ajuda
      --man           Manual completo

=cut

Term::ReadLine: Entrada Interativa

Term::ReadLine fornece um prompt interativo com histórico de comandos, edição de linha (Ctrl-A, Ctrl-E, seta para cima/baixo) e autocompletar — essencial para ferramentas REPL ou wizards de configuração.

readline_interativo.pl
use strict;
use warnings;
use Term::ReadLine;

my $rl = Term::ReadLine->new("MinhaCLI");
my $OUT = $rl->OUT // \*STDOUT;

# Configurar autocompletar
my @comandos = qw(sair ajuda listar processar exportar importar);
$rl->Attribs->{completion_function} = sub {
    my ($text) = @_;
    return grep { /^\Q$text\E/i } @comandos;
};

say $OUT "Bem-vindo à MinhaCLI. Digite 'ajuda' ou 'sair'.";

while (defined(my $entrada = $rl->readline("minhacli> "))) {
    chomp $entrada;
    next unless $entrada =~ /\S/;  # ignora linhas vazias
    
    $rl->addhistory($entrada);  # adiciona ao histórico
    
    if ($entrada =~ /^sair$/i) {
        say $OUT "Até logo!";
        last;
    } elsif ($entrada =~ /^ajuda$/i) {
        say $OUT "Comandos: " . join(", ", @comandos);
    } elsif ($entrada =~ /^processar\s+(\S+)/i) {
        say $OUT "Processando: $1";
    } else {
        say $OUT "Comando desconhecido: $entrada";
    }
}

Executar Comandos: Backtick, system() e Pipes

Perl tem três formas principais de executar comandos do sistema operacional, cada uma com características diferentes para captura de saída, controle de fluxo e interação bidirecional.

execucao_sistema.pl
use strict;
use warnings;
use autodie qw(open);

# 1. Backticks `...` — captura a saída como string
my $data  = `date`;
chomp $data;
say "Data: $data";

# qx{} — equivalente a backticks, mais legível
my @arquivos = qx{ls -la /tmp/*.log 2>/dev/null};
chomp @arquivos;
say "Logs em /tmp: " . scalar @arquivos;

# $? — status de saída do último comando (shift 8 bits)
my $exit_code = $? >> 8;

# 2. system() — executa e retorna código de saída
#    Herda stdout/stderr do processo pai (aparece na tela)
my $ret = system("gzip /tmp/arquivo.log");
if ($ret != 0) {
    warn "Falha: gzip retornou $ret\n";
}

# system() com lista — MAIS SEGURO: sem shell, sem injeção
system("rsync", "-avz", "/origem/", "usuario\@server:/destino/");

# 3. open() com pipe — comunicação bidirecional
# Leitura: captura linha por linha (memória eficiente)
open my $ps, '-|', 'ps', 'aux';   # sem shell
while (my $linha = <$ps>) {
    say $linha if $linha =~ /perl/i;
}
close $ps;

# Escrita em pipe — enviar dados para um programa
open my $mail, '|-', 'sendmail', '-t';
print $mail "To: admin\@empresa.com\n";
print $mail "Subject: Aviso automático\n\n";
print $mail "Script executado com sucesso.\n";
close $mail;

# IPC::Open3 — stdin + stdout + stderr independentes
use IPC::Open3;
my ($in, $out, $err);
my $pid = open3($in, $out, $err, "comando_interativo");
print $in "entrada para o processo\n";
close $in;
my @saida = <$out>;
waitpid($pid, 0);

Nunca interpole variáveis de usuário em backticks ou system("...") com string. Use a forma de lista de system() ou sanitize rigorosamente com String::ShellQuote. Isso previne ataques de command injection.

Term::ANSIColor: Saída Colorida

Term::ANSIColor adiciona cores ANSI à saída do terminal, melhorando a legibilidade de relatórios, mensagens de erro e dashboards.

colorido.pl
use strict;
use warnings;
use Term::ANSIColor qw(:constants colored);

# colored() — envolve texto com a cor especificada
print colored("ERRO",    "bold red"),     ": arquivo não encontrado\n";
print colored("AVISO",   "bold yellow"),  ": disco acima de 80%\n";
print colored("OK",      "bold green"),   ": serviço rodando\n";
print colored("INFO",    "bold cyan"),    ": versão 2.0.1\n";

# Constantes importadas com :constants
print BOLD, BLUE, "Cabeçalho importante", RESET, "\n";

# Desabilitar cores quando não é terminal (ex: pipe para arquivo)
use Term::ANSIColor qw(colorstrip);
my $sem_cor = colorstrip(colored("texto", "red"));

# Verificar se o terminal suporta cores
if (-t STDOUT) {   # -t: filetest "is tty"
    $ENV{ANSI_COLORS_DISABLED} = 0;
} else {
    $ENV{ANSI_COLORS_DISABLED} = 1;  # desativa em pipe/arquivo
}

# Função helper para mensagens padronizadas
sub log_ok    { say colored("[ OK ]", "bold green")  . " @_" }
sub log_erro  { say colored("[ERRO]", "bold red")    . " @_" }
sub log_info  { say colored("[INFO]", "cyan")         . " @_" }

log_ok("Deploy concluído");
log_info("3 serviços reiniciados");
log_erro("Banco de dados inacessível");

Geração de Relatórios Tabulares

Perl oferece duas formas principais de formatar relatórios tabulares: printf/sprintf com especificadores de largura, e o sistema format/write — uma sintaxe de template legado, mas ainda poderosa para relatórios fixos.

relatorio.pl — com printf
use strict;
use warnings;
use POSIX qw(strftime);

my @dados = (
    { nome => "Servidor Web",  status => "online",  cpu => 23.5, mem => 45.2 },
    { nome => "Banco de Dados", status => "online",  cpu => 67.1, mem => 78.9 },
    { nome => "Cache Redis",    status => "offline", cpu =>  0.0, mem =>  0.0 },
    { nome => "API Gateway",    status => "online",  cpu => 12.3, mem => 30.1 },
);

my $titulo = "Relatório de Status dos Serviços";
my $data   = strftime("%d/%m/%Y %H:%M", localtime);
my $sep    = "=" x 60;

printf "\n%s\n",         $sep;
printf "  %-38s %s\n",   $titulo, $data;
printf "%s\n",           $sep;
printf "  %-20s %-8s  %6s  %6s\n", "Serviço", "Status", "CPU%", "Mem%";
printf "  %s\n", "-" x 50;

foreach my $s (@dados) {
    my $alerta = $s->{cpu} > 60 ? " (!)" : "";
    printf "  %-20s %-8s  %5.1f  %5.1f%s\n",
        $s->{nome}, $s->{status}, $s->{cpu}, $s->{mem}, $alerta;
}
printf "%s\n\n", $sep;

# Sumário
my $online  = grep { $_->{status} eq "online"  } @dados;
my $offline = grep { $_->{status} eq "offline" } @dados;
printf "  Online: %d  |  Offline: %d\n\n", $online, $offline;

Arquivos, Diretórios e File::Path

Perl tem filetest operators —-e, -f, -d, -r, -s etc. — que inspecionam propriedades de arquivos sem comandos externos. Módulos como File::Path, File::Copy e File::Basename completam o toolkit portável.

filesystem.pl
use strict;
use warnings;
use autodie;
use File::Path    qw(make_path remove_tree);
use File::Copy    qw(copy move);
use File::Basename qw(basename dirname);
use File::Spec;
use POSIX qw(strftime);

my $arquivo = "/var/log/app.log";

# Filetest operators
say "Existe"        if -e $arquivo;   # exists
say "É arquivo"    if -f $arquivo;   # regular file
say "É diretório"  if -d $arquivo;   # directory
say "Legível"       if -r $arquivo;   # readable
say "Gravável"      if -w $arquivo;   # writable
say "Executável"   if -x $arquivo;   # executable
printf "Tamanho: %d bytes\n", -s $arquivo // 0;

# Informações do inode (stat)
my ($dev,$ino,$mode,$nlink,$uid,$gid,$rdev,$size,
    $atime,$mtime,$ctime) = stat($arquivo);
say "Modificado: " . strftime("%d/%m/%Y %H:%M", localtime($mtime));

# Decomposição de caminho
say basename($arquivo);          # app.log
say dirname($arquivo);           # /var/log
say basename($arquivo, '.log');  # app (sem extensão)

# Criar estrutura de diretórios (equivale a mkdir -p)
make_path('/tmp/app/uploads/2024');

# Copiar e mover
copy($arquivo, '/tmp/app.log.bak');
move('/tmp/app.log.bak', '/tmp/bak/');

# Caminho portável (Windows/Unix)
my $path = File::Spec->catfile('base', 'sub', 'arquivo.txt');

# Listar diretório com glob
my @logs = glob("/var/log/*.log");
say "$_ (" . (-s $_) . " bytes)" for @logs;
