Perl em Banco de Dados

DBI, drivers SQL, transacoes e performance para sistemas de missao critica.

Perl sempre teve relacao forte com dados. Com DBI, voce conecta MySQL, PostgreSQL, SQLite e outros bancos usando uma API estavel, ideal para migracoes graduais e manutencao de legados sem sacrificar qualidade de engenharia.

Conceitos fundamentais

A interface DBI separa a aplicacao do driver especifico (DBD::Pg, DBD::mysql, DBD::SQLite). O codigo de dominio conversa com DBI, e o driver conversa com o banco. Isso reduz acoplamento e melhora portabilidade.

Elementos centrais

  • dbh: handle de conexao.
  • sth: statement handle para SQL preparado.
  • placeholders: parametros com ?.
  • transacao: bloco atomico com commit/rollback.

Como funciona na pratica

repositorio.pl - acesso seguro com DBI
use DBI;

my $dbh = DBI->connect(
  'dbi:Pg:dbname=erp;host=localhost;port=5432',
  'app_user',
  'segredo',
  { RaiseError => 1, AutoCommit => 1, pg_enable_utf8 => 1 }
);

my $sth = $dbh->prepare('select id, nome from cliente where ativo = ?');
$sth->execute(1);

while (my $row = $sth->fetchrow_hashref) {
  # processa cada cliente ativo
  print $row->{id} . ' - ' . $row->{nome} . "\n";
}

Uso: transacoes e consistencia

Em operacoes financeiras ou de estoque, nunca execute multiplos updates sem transacao. Em caso de falha parcial, o estado do banco pode ficar incoerente.

transacao.pl - commit e rollback
eval {
  $dbh->begin_work;

  my $debito = $dbh->prepare('update conta set saldo = saldo - ? where id = ?');
  my $credito = $dbh->prepare('update conta set saldo = saldo + ? where id = ?');

  $debito->execute(250, 10);
  $credito->execute(250, 20);

  $dbh->commit;
};

if ($@) {
  # desfaz tudo se qualquer passo falhar
  $dbh->rollback;
  die "falha na transacao: $@";
}

Avancado: performance e governanca SQL

Batch processing

Use fetch em lotes para evitar consumo excessivo de memoria em tabelas grandes.

Indices e planos

Analise EXPLAIN e ajuste indices antes de culpar a linguagem por lentidao.

Retry idempotente

Para erros transientes, use retries com limite e operacoes idempotentes.

Evite SQL construido por concatenacao de string; use placeholders em todas as entradas externas.

Ecossistema relacionado

  • DBI para conexao e SQL generico.
  • DBIx::Class para modelagem ORM mais declarativa.
  • SQL::Abstract para gerar SQL dinamico com mais seguranca.
  • Mojolicious para expor camada de dados via API.
