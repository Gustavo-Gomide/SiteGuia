Perl — Ecossistema & Tooling

Ecossistema & Tooling

CPAN, gerenciamento de versões, dependências, análise estática, formatação de código, documentação e debugging.

O ecossistema Perl é um dos mais maduros do mundo: o CPAN existe desde 1995 e conta com mais de 200.000 módulos. Ferramentas como perlbrew, Carton e perlcritic profissionalizam o desenvolvimento sem reinventar a roda.

CPAN e cpanm

O CPAN (Comprehensive Perl Archive Network) é o repositório central de módulos Perl, criado em 1995 por Jarkko Hietaniemi e Tim Bunce. É acessível em cpan.org e tem interface melhorada em metacpan.org, com busca, documentação e análise de qualidade.

bash — instalação de módulos
# cpanm (cpan minus) — instalador moderno, sem perguntas
cpanm Moose
cpanm Try::Tiny
cpanm DBI DBD::SQLite

# Instalar múltiplos módulos de uma vez
cpanm Mojolicious LWP::UserAgent HTML::TreeBuilder

# Instalar versão específica
cpanm YAML@1.30

# Instalar no diretório local (sem sudo)
cpanm --local-lib ~/perl5 Moose

# Instalar cpanm se ainda não tiver
curl -L https://cpanmin.us | perl - App::cpanminus

# Comando original cpan (interativo)
cpan Moose

# Via perl direto (sem cpanm, bootstrap)
perl -MCPAN -e 'install "Moose"'

perlbrew: Gerenciamento de Versões

perlbrew permite instalar e alternar entre múltiplas versões do Perl no mesmo sistema, sem dependência do Perl do sistema operacional. Funciona de forma similar ao rbenv (Ruby) ou nvm (Node.js).

bash — perlbrew
# Instalar perlbrew
\curl -L https://install.perlbrew.pl | bash

# Adicionar ao shell (bash/zsh)
echo 'source ~/perl5/perlbrew/etc/bashrc' >> ~/.bashrc
source ~/.bashrc

# Listar versões disponíveis para instalar
perlbrew available

# Instalar uma versão específica
perlbrew install perl-5.38.2
perlbrew install perl-5.36.3

# Listar versões instaladas
perlbrew list

# Usar uma versão globalmente
perlbrew use perl-5.38.2

# Definir versão padrão
perlbrew switch perl-5.38.2

# Executar um script com versão específica
perlbrew exec --with perl-5.36.3 perl myscript.pl

# Instalar cpanm no perl ativo
perlbrew install-cpanm

# Criar biblioteca separada por projeto
perlbrew lib create meu-projeto
perlbrew use perl-5.38.2@meu-projeto

Não modifique o Perl do sistema (geralmente em /usr/bin/perl ou /usr/local/bin/perl). Muitos sistemas operacionais dependem dele internamente. Use perlbrew para isolamento.

Carton: Gerenciamento de Dependências

Carton é o gerenciador de dependências de projeto para Perl, inspirado no Bundler (Ruby). Ele lê um arquivo cpanfile com as dependências declaradas e cria um snapshot cpanfile.snapshot com versões exatas — garantindo builds reproduzíveis em qualquer máquina.

cpanfile — declaração de dependências
# cpanfile: declara dependências do projeto
requires 'Mojolicious',    '9.0';          # versão mínima
requires 'DBI',            '1.643';
requires 'DBD::Pg';                         # qualquer versão
requires 'Moo',            '2.0';
requires 'Try::Tiny';

# Dependências apenas para testes
on 'test' => sub {
    requires 'Test::More', '1.302';
    requires 'Test::Exception';
};

# Dependências de desenvolvimento
on 'develop' => sub {
    requires 'Perl::Critic';
    requires 'Perl::Tidy';
};
bash — comandos Carton
# Instalar Carton
cpanm Carton

# Instalar dependências do cpanfile no diretório local/
carton install

# Executar script com as dependências do Carton
carton exec perl app.pl

# Atualizar snapshot após mudar cpanfile
carton update

# Instalar apenas dependências de produção (não dev/test)
carton install --deployment

Perl::Critic: Análise Estática

Perl::Critic analisa código Perl e aponta violações de boas práticas, baseado no livro Perl Best Practices de Damian Conway. Organizado em níveis de severidade de 1 (crítico) a 5 (cosmético).

bash — perlcritic
# Instalar
cpanm Perl::Critic

# Análise com nível máximo (-5 = todos os avisos)
perlcritic --severity 5 meu_script.pl

# Nível padrão (3 = moderado)
perlcritic meu_script.pl

# Modo crítico (apenas erros graves)
perlcritic --brutal meu_script.pl

# Formatos de saída
perlcritic --verbose 9 meu_script.pl   # detalhado com política

# Verificar diretório inteiro
perlcritic lib/
.perlcriticrc — configuração do projeto
# .perlcriticrc no raiz do projeto
severity  = 4
verbose   = 9

# Desabilitar políticas específicas
[-InputOutput::RequireCheckedSyscalls]
[-RegularExpressions::RequireExtendedFormatting]

# Configurar política específica
[CodeLayout::ProhibitTrailingWhitespace]
severity = 1

Para suprimir um aviso em linha específica (use com parcimônia):

supressão inline
my $x = $y = $z = 0;  ## no critic (ProhibitChainedComparisons)

Perl::Tidy: Formatação de Código

Perl::Tidy (comando perltidy) é o formatador oficial de código Perl — equivalente ao gofmt (Go) ou prettier (JS). Reformata automaticamente indentação, espaçamento e quebras de linha.

bash — perltidy
# Instalar
cpanm Perl::Tidy

# Formatar arquivo — cria arquivo.pl.tdy
perltidy meu_script.pl

# Formatar in-place (sobrescreve o original)
perltidy -b meu_script.pl      # cria backup .bak

# Formatar múltiplos arquivos
perltidy -b lib/**/*.pm

# Verificar sem modificar (exit code 1 se há diff)
perltidy --check-syntax meu_script.pl
.perltidyrc — configuração
# .perltidyrc no raiz do projeto
--indent-columns=4
--maximum-line-length=100
--opening-brace-on-new-line
--closing-brace-indentation=0
--check-syntax

perldoc: Documentação no Terminal

perldoc é a ferramenta de consulta à documentação Perl — acessa tanto a documentação da linguagem quanto a documentação POD embutida em qualquer módulo instalado.

bash — perldoc
# Documentação de uma função built-in
perldoc -f print
perldoc -f open
perldoc -f sort

# Manual de um módulo CPAN instalado
perldoc DBI
perldoc Mojolicious
perldoc LWP::UserAgent

# Manuais da linguagem
perldoc perlsyn          # sintaxe
perldoc perlre           # expressões regulares
perldoc perldsc          # data structures cookbook
perldoc perltoot         # OOP tutorial
perldoc perlref          # referências
perldoc perlmod          # módulos
perldoc perlvar          # variáveis especiais ($_, $@, etc.)

# Busca por palavra-chave
perldoc -q "regex"

# Listar módulos instalados
perldoc perlmodlib

perl -d: Debugger Interativo

Perl tem um debugger interativo embutido, ativado com a flag -d. Funciona no terminal, suporta breakpoints, inspeção de variáveis, step-into/step-over e execução de código ad-hoc.

bash — debugger
# Iniciar script no debugger
perl -d meu_script.pl

# Com argumentos
perl -d meu_script.pl arg1 arg2

bash — ferramentas complementares de debug
# Data::Dumper — dump de qualquer estrutura
perl -MData::Dumper -e 'print Dumper({a => 1, b => [2,3]})'

# Devel::NYTProf — profiler de performance
cpanm Devel::NYTProf
perl -d:NYTProf meu_script.pl
nytprofhtml               # gera relatório HTML

# Compilar sem executar — checar sintaxe
perl -c meu_script.pl

# Modo aviso extremo
perl -w -Mstrict meu_script.pl

# One-liners de diagnóstico
perl -e 'use Moose; print Moose->VERSION'
