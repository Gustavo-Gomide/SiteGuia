Perl — Introdução

Perl

A linguagem prática da era Unix — poderosa em texto, onipresente em sistemas e ainda indispensável em bioinformática e automação.

Perl foi a cola que manteve a Internet unida nos anos 90. Criada por Larry Wall em 1987, ela combina o melhor de shell scripting, awk, sed e C em uma linguagem expressiva que acredita que existe mais de uma maneira de fazer as coisas — e está certa nisso.

História: Larry Wall e a Origem do Perl

Larry Wall, linguista e programador da Unisys, estava trabalhando como administrador de sistemas em 1987 quando sua ferramenta do momento — awk — não era expressiva o suficiente para um projeto de geração de relatórios. Em vez de adaptar uma ferramenta existente, Wall criou uma nova linguagem de propósito geral orientada ao processamento de texto e administração de sistemas.

A primeira versão pública do Perl foi liberada em 18 de dezembro de 1987 na Usenet (comp.sources.misc). O nome originou diversas retroacronímias: Practical Extraction and Report Language, Pathologically Eclectic Rubbish Lister — Wall prefere a segunda como brincadeira, mas a primeira é a definição oficial.

Perl 1 (1987)

Primeira versão: processamento de texto, regex básico, interação com sistema Unix. Substituía awk + sed + shell em um único binário.

Perl 2 (1988)

Melhorias no mecanismo de expressões regulares. Inicío da reputação de Perl como "a melhor linguagem de regex do mundo".

Perl 3 (1989)

Suporte a dados binários, variáveis de ambiente e capacidades de rede primitivas.

Perl 4 (1991)

Acompanhou o livro Programming Perl (O'Reilly, 1991) — o "Livro do Camelo". Consolidou Perl como padrão de scripts Web CGI.

Perl 5 (1994)

Reescrita por Larry Wall. Introduziu referências, módulos, OOP com bless, mecanismo use e o CPAN. Versão que define o Perl moderno.

Perl 5.36+ (2022)

Perl moderno com use v5.36 ativa por padrão strict, warnings, assinaturas em subroutinas e diversas melhorias ergonômicas.

Do Perl 6 ao Raku

Em 2000, Larry Wall anunciou o Perl 6 como uma reescrita completa da linguagem. O projeto levou décadas e resultou em uma linguagem tão diferente que, em 2019, foi oficialmente renomeada para Raku para evitar confusão com o Perl 5. Perl 5 e Raku são hoje linguagens irmãs, mas distintas — compartilham filosofia mas não compatibilidade de código. O Perl 5 continua em desenvolvimento ativo e é o que a indústria usa quando diz "Perl".

Filosofia: TMTOWTDI e a Cultura Perl

Larry Wall, sendo linguista antes de programador, desenhou Perl com a filosofia de que linguagens naturais são tolerantes e expressivas — e linguagens de programação deveriam ser também. O resultado é o mantra:

"There Is More Than One Way To Do It" — TMTOWTDI (pronunciado "Tim Toady")

Isso contrasta diretamente com Python ("There should be one — and preferably only one — obvious way to do it") e é precisamente o que divide opiniões sobre Perl. No mundo Perl, liberdade de expressão é um valor, não um problema.

As três virtudes do programador Perl, segundo Larry Wall: Preguiça (escrever código que faz mais com menos), Impaciência (automatizar tudo que é repetitivo) e Arrogância (escrever código bom o suficiente para defender).

Concisão

Operações complexas em poucas linhas. Perl é notório pela "golf" (código mínimo), mas também permite código legível quando o desenvolvedor assim o deseja.

Práticidade antes de pureza

Perl não é uma linguagem puramente funcional, OO ou procedural. Usa o que funciona para o problema em mãos. Pragmatismo é um valor central.

Texto como cidadão de primeira classe

Processamento de texto é onde Perl brilha. Regex integrada à linguagem, manipulação de strings trivial, parsing de logs e dados semi-estruturados são naturais.

Unix como ambiente nativo

Pipes, redirecionamentos, chamadas de sistema, manipulação de arquivos — tudo com sintaxe compacta. Perl é o sucessor espiritual do shell scripting.

Onde Perl é Usado Hoje

Bioinformática

O BioPerl é um dos maiores frameworks de bioinformática. Análise de sequências de DNA, alinhamento, parsing de formatos FASTA/GenBank, pipelines de NGS. Institutos como NCBI, Ensembl e Sanger usam Perl extensivamente.

Administração de Sistemas

Scripts de manutenção, automação de tarefas, parsing de logs de servidor, gerenciamento de usuários e permissões. Ferramenta clássica de sysadmins Unix/Linux.

Processamento de Texto e Dados

ETL (Extract, Transform, Load) de arquivos de texto, transformação de formatos (CSV, XML, JSON), relatórios automatizados, scraping de conteúdo.

Web CGI (Legado)

Nos anos 90 e 2000, a maioria dos servidores web rodava scripts Perl via CGI. Muitos sistemas críticos ainda dependem desse código. Compreender Perl é necessário para manutenção de legado.

Redes e Segurança

Muitas ferramentas de segurança e redes são escritas em Perl: Nessus (plugins), ferramentas de IDS, scanners de vulnerabilidade, análise de pacotes com Net::Packet.

Testes Automatizados

O framework TAP (Test Anything Protocol) nasceu em Perl. Perl é bastante utilizado como linguagem de testes de integração em ambientes Unix.

Primeiro Script Perl

Todo script Perl executável começa com uma linha shebang que diz ao sistema operacional qual interpretador usar. A linha #!/usr/bin/perl é a mais clássica, mas #!/usr/bin/env perl é mais portátil pois encontra o perl no PATH do usuário.

ola_mundo.pl — Primeiro script Perl
#!/usr/bin/env perl
use strict;
use warnings;

# print envia texto para STDOUT; \n é newline
print "Olá, Mundo!\n";

# say é como print mas adiciona \n automaticamente (use 5.010+)
use 5.010;
say "Olá com say!";

# variáveis: $ para escalares
my $nome = "Larry Wall";
my $ano  = 1987;

say "Perl foi criado por $nome em $ano.";

# interpolação de variáveis funciona dentro de duplas aspas
printf "Versão atual: Perl %vd\n", $^V;

Para executar o script no terminal Linux/macOS:

bash — executando o script
# Tornar executável e executar diretamente
chmod +x ola_mundo.pl
./ola_mundo.pl

# Ou passar explicitamente ao interpretador
perl ola_mundo.pl

# Verificar versão instalada
perl --version

# One-liner: executar código diretamente na linha de comando
perl -e 'print "Olá, one-liner!\n"'

# -n lê stdin linha por linha (sem imprimir), útil para processar arquivos
perl -ne 'print if /erro/i' sistema.log
estrutura_basica.pl — Estrutura de um script completo
#!/usr/bin/env perl
use strict;    # obriga declaração de variáveis
use warnings; # avisos sobre código potencialmente problemático
use 5.036;    # Perl moderno: ativa strict, warnings e assinaturas

# Declaração de variáveis
my $mensagem = "Site Guia";
my @linguagens = ("Perl", "Python", "Ruby");
my %info = (nome => "Perl", ano => 1987, autor => "Larry Wall");

# Iteração
foreach my $lang (@linguagens) {
    say "Linguagem: $lang";
}

# Hash access
say "$info{nome} foi criado em $info{ano} por $info{autor}";

# Retorno explícito do script: 0 = sucesso
exit 0;

Instalação e Configuração

Perl está disponível em praticamente todos os sistemas Unix/Linux por padrão. No macOS, vem pré-instalado. No Windows, a distribuição mais popular é o Strawberry Perl.

bash — instalação em diferentes sistemas
# Linux Ubuntu/Debian — Perl já está instalado, mas para perlbrew:
sudo apt install build-essential curl
\curl -L https://install.perlbrew.pl | bash
perlbrew install perl-5.38.2
perlbrew use perl-5.38.2

# macOS — via Homebrew (recomendado, evita modificar /usr/bin/perl)
brew install perl

# Windows — Strawberry Perl (inclui compilador C e cpanm)
# Baixar de: https://strawberryperl.com/

# Verificar versão
perl -v

# Instalar módulos via CPAN (gerenciador de pacotes)
cpan install Mojolicious
# Ou com cpanminus (mais rápido):
cpanm Mojolicious
