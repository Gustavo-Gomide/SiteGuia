Perl Avançado

Referências, estruturas de dados complexas, OOP, closures, regex avançado, DBI e tratamento de exceções.

O verdadeiro poder de Perl emerge quando você combina referências para construir estruturas aninhadas, closures para encapsular estado e o sistema de objetos baseado em bless — tudo isso sustentado pelo motor de regex mais sofisticado do ecossistema de linguagens dinâmicas.

Referências e Estruturas Complexas

Uma referência em Perl é um escalar que armazena o endereço de memória de outro dado — escalar, array, hash ou sub-rotina. Referências são o mecanismo que permite criar estruturas aninhadas como arrays de hashes ou hashes de arrays.

referencias.pl
use strict;
use warnings;

# Criar referência com \ (backslash)
my $escalar  = 42;
my $ref_esc  = \$escalar;   # referência para escalar

my @array    = (1, 2, 3);
my $ref_arr  = \@array;     # referência para array

my %hash     = (a => 1);
my $ref_hash = \%hash;      # referência para hash

# Desreferenciar
say $$ref_esc;               # 42 — $$ref
say $ref_arr->[0];           # 1  — seta: mais legível
say $$ref_arr[0];             # 1  — alternativa
say $ref_hash->{a};          # 1  — seta para hash

# Referência anônima — sem criar variável nomeada
my $aref = [10, 20, 30];     # [] cria arrayref anônimo
my $href = { x => 1, y => 2 }; # {} cria hashref anônimo

# Estrutura complexa: array de hashes (tabela)
my @usuarios = (
    { nome => "Alice", idade => 30, admin => 1 },
    { nome => "Bob",   idade => 25, admin => 0 },
    { nome => "Carol", idade => 35, admin => 1 },
);

foreach my $u (@usuarios) {
    printf "%-8s %3d %s\n",
        $u->{nome}, $u->{idade},
        $u->{admin} ? "admin" : "user";
}

# Hash de arrays — índice invertido
my %por_role;
foreach my $u (@usuarios) {
    my $role = $u->{admin} ? "admin" : "user";
    push @{ $por_role{$role} }, $u->{nome};
}
say "Admins: " . join(", ", @{ $por_role{admin} });

# ref() — descobre o tipo da referência
say ref($aref);   # ARRAY
say ref($href);   # HASH

Closures

Uma closure é uma sub-rotina que "captura" variáveis léxicas (my) do escopo em que foi criada, mesmo depois que esse escopo não está mais ativo. Closures são o mecanismo de encapsulamento mais elegante em Perl.

closures.pl
# Fábrica de contador — cada chamada retorna uma closure independente
sub make_counter {
    my $n = 0;                # variável capturada
    return sub { return ++$n };  # closure que incrementa $n
}

my $c1 = make_counter();
my $c2 = make_counter();  # $n independente de $c1

say $c1->();  # 1
say $c1->();  # 2
say $c2->();  # 1 — contador próprio
say $c1->();  # 3

# Closure como memoize manual
sub make_fibonacci {
    my %cache;
    my $fib;
    $fib = sub {
        my ($n) = @_;
        return $n if $n <= 1;
        return $cache{$n} if exists $cache{$n};
        $cache{$n} = $fib->($n-1) + $fib->($n-2);
        return $cache{$n};
    };
    return $fib;
}

my $fib = make_fibonacci();
say $fib->(40);   # 102334155 — rápido por causa do cache

# Closures como callbacks (dispatch table)
my %ops = (
    soma    => sub { $_[0] + $_[1] },
    subtr   => sub { $_[0] - $_[1] },
    produto => sub { $_[0] * $_[1] },
);
say $ops{soma}->(10, 5);     # 15
say $ops{produto}->(6, 7); # 42

OOP com bless e package

Perl implementa orientação a objetos de forma minimalista: um objeto é qualquer referência "abençoada" (bless) com o nome de um package. O package define as sub-rotinas que se tornam métodos.

oop_animal.pl
package Animal;
use strict;
use warnings;

# Construtor: convenção new()
sub new {
    my ($classe, %args) = @_;
    my $self = {
        nome  => $args{nome}  // "Desconhecido",
        som   => $args{som}   // "...",
    };
    return bless $self, $classe;  # associa $self ao package
}

# Accessor / getter
sub nome { return $_[0]->{nome} }

# Método de instância
sub falar {
    my ($self) = @_;
    printf "%s faz: %s\n", $self->{nome}, $self->{som};
}

# Herança via @ISA
package Cachorro;
use parent -norequire, 'Animal';  # herda de Animal

sub new {
    my ($classe, %args) = @_;
    $args{som} = "Au au!";
    return $classe->SUPER::new(%args);  # chama o pai
}

# Sobrescrita de método
sub falar {
    my ($self) = @_;
    print $self->{nome} . " late: " . $self->{som} . "\n";
}

# Uso no script principal
package main;

my $gato  = Animal->new(nome => "Mimi",  som => "Miau");
my $cao   = Cachorro->new(nome => "Rex");

$gato->falar();   # Mimi faz: Miau
$cao->falar();    # Rex late: Au au!

say ref($cao);                      # Cachorro
say $cao->isa("Animal") ? "sim" : "não";  # sim

Moose e Moo: OOP Moderno

O sistema bless/package é poderoso mas verboso — cria getters/setters manualmente, não valida tipos e não tem accessors automáticos. Moose (e seu filho leve Moo) resolvem isso, trazendo um sistema de metaobjetos declarativo.

Moose

Sistema de meta-objetos completo. Atributos tipados, roles (mixins), herança limpa, before/after/around (métodos de ordem). Mais lento para inicializar; use em aplicações longas.

Moo

Subconjunto do Moose, 95% menor, inicialização ultrarrápida. Compatível com Moose (upgrade transparente). Recomendado para a maioria dos projetos.

Mouse

Alternativa ainda mais leve. Útil em scripts de curta duração onde o overhead de startup importa.

moo_exemplo.pl
package Pessoa;
use Moo;
use Types::Standard qw(Str Int Bool);

# Atributos declarativos com tipo e padrão
has nome  => (is => 'ro', isa => Str,  required => 1);
has idade => (is => 'rw', isa => Int,  default  => 0);
has ativo => (is => 'ro', isa => Bool, default  => 1);

# Método calculado (lazy)
has descricao => (
    is      => 'lazy',
    builder => '_build_descricao',
);

sub _build_descricao {
    my ($self) = @_;
    return sprintf "%s, %d anos", $self->nome, $self->idade;
}

# Método de instância
sub cumprimentar {
    my ($self) = @_;
    say "Olá, sou " . $self->nome;
}

# Herança com Moo
package Funcionario;
use Moo;
extends 'Pessoa';                      # herda todos os atributos
has cargo => (is => 'ro', isa => Str, required => 1);

package main;

my $p = Pessoa->new(nome => "Ana", idade => 28);
$p->cumprimentar();
say $p->descricao;     # Ana, 28 anos (calculado na primeira chamada)
$p->idade(29);         # rw: pode modificar

my $f = Funcionario->new(nome => "Bob", cargo => "Dev");

Regex Avançado

Além das ferramentas básicas, o motor de regex do Perl suporta lookahead, lookbehind, capturas nomeadas, quantificadores possessivos e a flag /x que permite documentar expressões complexas.

regex_avancado.pl
use strict;
use warnings;

# Capturas nomeadas: (?<nome>...)
my $data = "Evento: 2024-12-31";
if ($data =~ m/(?<ano>\d{4})-(?<mes>\d{2})-(?<dia>\d{2})/) {
    say "Ano: $+{ano}, Mês: $+{mes}, Dia: $+{dia}";
}

# Lookahead positivo (?=...) — casa se seguido por
my $preco = "100 reais, 50 dólares, 200 euros";
my @em_reais = $preco =~ m/(\d+)(?=\s+reais)/g;  # (100)

# Lookahead negativo (?!...) — casa se NÃO seguido por
my @nao_reais = $preco =~ m/(\d+)(?!\s+reais)/g; # 50, 200

# Lookbehind positivo (?<=...) — casa se precedido por
my @apos_R = $preco =~ m/(?<=\s)\d+/g;  # números após espaço

# Flag /x — regex legível com espaços e comentários
my $email = "dev@example.com";
my $regex_email = qr/
    ^                       # início da string
    ([\w.+-]+)              # local part: letras, ponto, +, -
    @                       # arroba literal
    ([\w-]+                 # domínio: letras e hífens
        (?:\.[\w-]+)*)      # subdomínios opcionais
    \.                      # ponto antes do TLD
    ([a-zA-Z]{2,})          # TLD: 2+ letras
    $                       # fim da string
/x;

if ($email =~ $regex_email) {
    say "Email válido: $1 @ $2 . $3";
}

# s/// com /e — avalia o lado direito como código
my $template = "2 + 2 = {2+2}, PI ≈ {3.14159:.2f}";
$template =~ s/\{([^}]+)\}/eval($1)/ge;  # substitui com resultado

# Possessivo (++): sem backtrack — mais eficiente
# \w++ equivale a \w+ mas não devolve caracteres ao backtrack
$email =~ m/\w++@/;  # possessivo

DBI: Banco de Dados

O módulo DBI (Database Interface) é a interface padrão de Perl para bancos relacionais. Trabalha com qualquer banco via driver DBD específico: DBD::Pg para PostgreSQL, DBD::mysql para MySQL, DBD::SQLite para SQLite, etc.

dbi_exemplo.pl
use strict;
use warnings;
use DBI;

# Conectar ao banco (SQLite como exemplo portável)
my $dbh = DBI->connect(
    'dbi:SQLite:dbname=app.db',
    '', '',
    {
        RaiseError => 1,        # exceção em erro (em vez de return undef)
        AutoCommit => 1,        # commit automático
        PrintError => 0,
    }
) or die DBI::errstr;

# Criar tabela
$dbh->do('CREATE TABLE IF NOT EXISTS usuarios (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    nome  TEXT    NOT NULL,
    email TEXT    UNIQUE NOT NULL,
    ativo INTEGER DEFAULT 1
)');

# INSERT com placeholder — previne SQL injection
my $sth_ins = $dbh->prepare(
    'INSERT INTO usuarios (nome, email) VALUES (?, ?)'
);
$sth_ins->execute("Alice", "alice@ex.com");
$sth_ins->execute("Bob",   "bob@ex.com");

# SELECT — buscar todos
my $sth_sel = $dbh->prepare('SELECT id, nome, email FROM usuarios');
$sth_sel->execute();

while (my $row = $sth_sel->fetchrow_hashref()) {
    printf "%3d | %-10s | %s\n",
        $row->{id}, $row->{nome}, $row->{email};
}

# Transação manual
$dbh->{AutoCommit} = 0;
eval {
    $dbh->do('UPDATE usuarios SET ativo=0 WHERE nome=?', undef, "Bob");
    $dbh->commit();
};
if ($@) {
    $dbh->rollback();
    die "Transação falhou: $@";
}
$dbh->{AutoCommit} = 1;

# Desconectar
$dbh->disconnect();

Nunca concatene valores do usuário diretamente em SQL. Use sempre placeholders ? com prepare/execute. Isso impede ataques de SQL injection.

Exceções com eval/die

Perl não tem sintaxe try/catch nativa — usa eval { } como bloco de captura e die para lançar exceções. A variável $@ contém o erro capturado. Módulos como Try::Tiny e Feature::Compat::Try adicionam sintaxe mais limpa.

excecoes.pl
use strict;
use warnings;

# eval captura exceções — como try/catch
eval {
    die "algo deu errado\n";   # lança exceção
};
if ($@) {
    chomp my $err = $@;
    warn "Capturado: $err\n";
}

# Objetos de exceção — die com referência
package MinhaExcecao;
sub new {
    my ($class, %args) = @_;
    return bless { mensagem => $args{mensagem}, codigo => $args{codigo} }, $class;
}
sub mensagem { $_[0]->{mensagem} }
sub codigo   { $_[0]->{codigo}   }

package main;
eval {
    die MinhaExcecao->new(mensagem => "Arquivo não encontrado", codigo => 404);
};
if (my $e = $@) {
    if (ref($e) eq 'MinhaExcecao') {
        printf "Erro %d: %s\n", $e->codigo, $e->mensagem;
    } else {
        die $e;  # relança exceção desconhecida
    }
}

# Try::Tiny — interface mais limpa (CPAN)
use Try::Tiny;

try {
    open my $fh, '<', '/arquivo/inexistente'
        or die "Não abriu: $!\n";
}
catch {
    warn "Erro capturado: $_";
}
finally {
    say "Bloco finally sempre executa";
};

# Carp — erros com stack trace mais informativo
use Carp qw(croak confess carp cluck);
sub dividir {
    my ($a, $b) = @_;
    croak "Divisão por zero!" if $b == 0;  # die da perspectiva do chamador
    return $a / $b;
}
