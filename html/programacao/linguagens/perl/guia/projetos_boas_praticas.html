Perl — Projetos e Boas Práticas

Projetos e Boas Práticas

Estrutura de projetos profissionais: testes automatizados, logging, documentação POD e convenções que os módulos do CPAN seguem.

Escrever Perl de qualidade vai além de código que funciona. Strict, warnings, testes automatizados com TAP, documentação POD e logging configu­rável são os pilares que transformam scripts descartáveis em software manutenível.

strict e warnings: Prática Obrigatória

Todo arquivo Perl de produção deve começar com esses dois pragmas. Eles evitam classes inteiras de bugs silenciosos e são a primeira linha de defesa do desenvolvedor.

boilerplate_moderno.pl
#!/usr/bin/env perl

# Opção A: clássico, compatível com Perl 5.8+
use strict;
use warnings;
use autodie;           # exceções automáticas em open/close/read/write
use 5.010;             # habilita say, state, given/when

# Opção B: moderno (Perl 5.36+)
# use v5.36;  # ativa strict + warnings + unicode + assinaturas

# O que strict proíbe:

# 1. Variáveis não declaradas
# $x = 1;               # ERRO: Global symbol requires explicit package name
my $x = 1;             # correto: declaração léxica com my

# 2. Strings como nomes de sub-rotinas
# &{"minha_sub"}();     # ERRO com strict refs

# O que warnings detecta:
my $num = "abc" + 0;   # aviso: "abc" não é numérico
my $u;
print $u;              # aviso: uso de variável não inicializada

# Categorias de aviso podem ser controladas
no warnings 'uninitialized';  # silencia só esta categoria
print $u;                     # sem aviso agora
use warnings 'uninitialized'; # reativa

Testes com Test::More e Test::Simple

Perl é o criador do protocolo TAP (Test Anything Protocol). Todo módulo CPAN sério tem uma suíte de testes em t/. O framework padrão é Test::More, que fornece assertions declarativas e integração com o harness prove.

t/01_basico.t — arquivo de teste
#!/usr/bin/env perl
use strict;
use warnings;
use Test::More;         # importa ok, is, isnt, like, cmp_ok, etc.

# Testar uma função hipotética
use lib 'lib';           # adiciona lib/ ao @INC
use MeuModulo::Math;    # o módulo que estamos testando

# ok() — verifica que a expressão é verdadeira
ok(soma(2, 3) == 5, "soma(2,3) deve ser 5");

# is() — verifica igualdade com eq / ==
is(soma(0, 0), 0,     "soma(0,0) = 0");
is(soma(-1, 1), 0,   "soma negativo");

# isnt() — verifica desigualdade
isnt(soma(1, 1), 3, "1+1 não é 3");

# like() — verifica com regex
like(versao(), qr/^\d+\.\d+/, "versão tem formato x.y");

# cmp_ok() — comparação flexível
cmp_ok(fatorial(5), '==', 120, "5! = 120");

# Testar exceção com eval
eval { dividir(1, 0) };
like($@, qr/[Zz]ero/, "divisão por zero lança exceção");

# Testar estrutura de dados
my @arr = primeiros_n(3);
is_deeply(\@arr, [1, 2, 3], "primeiros_n(3) retorna [1,2,3]");

# done_testing() conta os testes automaticamente (recomendado)
done_testing();

# Alternativa: declarar count no topo
# use Test::More tests => 8;
bash — executar testes
# Executar um arquivo de teste
prove t/01_basico.t

# Executar todos os testes em t/
prove t/

# Modo verboso (mostra cada ok/not ok)
prove -v t/

# Executar em paralelo (mais rápido)
prove -j4 t/

# Integração com Makefile.PL
perl Makefile.PL
make
make test

Estrutura de Módulo CPAN

Todo módulo Perl bem estruturado segue uma convenção de diretórios que os instaladores (cpanm, carton) e a infraestrutura CPAN entendem. O utilitário Module::Starter gera o esqueleto automaticamente.

Layout de Diretórios

MeuModulo/
├── lib/
│   └── MeuModulo/
│       ├── Math.pm         # módulo principal
│       └── Utils.pm        # submódulo
├── t/
│   ├── 01_basico.t         # testes numerados por ordem
│   └── 02_avancado.t
├── bin/
│   └── minha_ferramenta    # executáveis
├── cpanfile                # dependências (Carton)
├── Makefile.PL             # ou Build.PL (Module::Build)
├── Changes                 # changelog
├── MANIFEST                # lista todos os arquivos
├── MANIFEST.SKIP           # arquivos a ignorar
└── README.md
lib/MeuModulo/Math.pm
package MeuModulo::Math;

use strict;
use warnings;
our $VERSION = '0.01';

use Exporter 'import';          # herda import()
our @EXPORT_OK = qw(soma fatorial dividir versao);  # exportação opt-in

sub soma      { $_[0] + $_[1] }
sub versao   { return $VERSION }

sub fatorial {
    my ($n) = @_;
    return 1 if $n <= 1;
    return $n * fatorial($n - 1);
}

sub dividir {
    my ($a, $b) = @_;
    die "Divisão por zero!\n" if $b == 0;
    return $a / $b;
}

# Módulos DEVEM retornar verdadeiro no final
1;
Makefile.PL — metadados do módulo

autodie: Exceções Automáticas

O módulo autodie faz com que funções de I/O e de sistema (open, close, read, write, mkdir, etc.) lançam exceção automaticamente em falha, eliminando o padrão verboso or die "$!".

autodie_exemplo.pl
use strict;
use warnings;
use autodie; # aplica a open, close, read, write, mkdir, etc.

# SEM autodie (padrão verboso)
# open(my $fh, '<', 'dados.txt') or die "Não abriu: $!";

# COM autodie (limpo)
open my $fh, '<', 'dados.txt';   # morre automaticamente se falhar
while (my $linha = <$fh>) {
    chomp $linha;
    say $linha;
}
close $fh;

# autodie com escopo — apenas em bloco específico
{
    use autodie qw(open close);    # só open e close no escopo
    open my $out, '>', '/tmp/saida.txt';
    print $out "dados\n";
    close $out;
}

# Capturar exceção de autodie
use autodie;
eval {
    open my $fh, '<', '/nao/existe';
};
if (my $e = $@) {
    if ($e->isa('autodie::exception')) {
        say "Falha em: " . $e->function;
        say "Motivo: "  . $e->errno;
    }
}

Log::Log4perl: Logging Profissional

Log::Log4perl é a implementação Perl do framework Log4j. Permite configurar níveis de log (DEBUG, INFO, WARN, ERROR, FATAL), múltiplos appenders (arquivo, STDOUT, syslog) e formatação personalizada — sem modificar o código da aplicação.

log4perl.conf — configuração
# log4perl.conf
log4perl.rootLogger = DEBUG, Screen, Arquivo

# Appender 1: tela (stdout)
log4perl.appender.Screen              = Log::Log4perl::Appender::Screen
log4perl.appender.Screen.layout       = Log::Log4perl::Layout::PatternLayout
log4perl.appender.Screen.layout.ConversionPattern = %d{HH:mm:ss} %-5p %c — %m%n

# Appender 2: arquivo rotacionado
log4perl.appender.Arquivo             = Log::Log4perl::Appender::File
log4perl.appender.Arquivo.filename    = /var/log/app/app.log
log4perl.appender.Arquivo.mode        = append
log4perl.appender.Arquivo.layout      = Log::Log4perl::Layout::PatternLayout
log4perl.appender.Arquivo.layout.ConversionPattern = %d %p %c — %m%n

# Logger específico por módulo — nível diferente
log4perl.logger.MeuApp.Database = WARN
app.pl — uso do Log4perl
use strict;
use warnings;
use Log::Log4perl;

# Inicializar com arquivo de configuração
Log::Log4perl->init('log4perl.conf');

# Obter logger nomeado (use o package name da classe)
my $log = Log::Log4perl->get_logger('MeuApp');

$log->debug("Início do processamento");
$log->info("Conectado ao banco de dados");

eval {
    my $resultado = processar();
    $log->info("Processado $resultado registros");
};
if ($@) {
    $log->error("Falha no processamento: $@");
    $log->logdie("Abortando");  # log + die
}

# Configuração inline (sem arquivo) — útil para testes
Log::Log4perl->easy_init({
    level  => $Log::Log4perl::INFO,
    layout => '%d %p %m%n',
    file   => '>>/tmp/app.log',
});

POD: Plain Old Documentation

POD (Plain Old Documentation) é o sistema de documentação embutido do Perl. Comentários POD são intercalados diretamente no código-fonte e processados por ferramentas como perldoc, pod2html e pod2man.

lib/MeuModulo/Math.pm — com POD completo
package MeuModulo::Math;
use strict;
use warnings;
our $VERSION = '1.00';

sub soma { $_[0] + $_[1] }
sub fatorial {
    my ($n) = @_;
    return 1 if $n <= 1;
    return $n * fatorial($n-1);
}

1;

__END__  # tudo abaixo é ignorado pelo compiler Perl

=head1 NAME

MeuModulo::Math - Funções matemáticas básicas

=head1 VERSION

Version 1.00

=head1 SYNOPSIS

  use MeuModulo::Math qw(soma fatorial);

  my $resultado = soma(3, 4);       # 7
  my $fat       = fatorial(5);      # 120

=head1 DESCRIPTION

Módulo que fornece funções matemáticas básicas para cálculos comuns.
Todas as funções são exportáveis opcionalmente via C<@EXPORT_OK>.

=head1 FUNÇÕES

=head2 soma($a, $b)

Retorna a soma de dois números.

  my $s = soma(10, 5);  # 15

=head2 fatorial($n)

Retorna o fatorial de C<$n>. Para C<$n <= 1>, retorna C<1>.

=head1 ERROS

Nenhum erro conhecido. Reporte via GitHub Issues.

=head1 AUTOR

Seu Nome, C<seu@email.com>

=head1 LICENÇA

Este módulo é software livre; você pode redistribuí-lo sob os
termos da Artistic License ou da GPL v1+.

=cut

POD entre =head1 e =cut pode aparecer em qualquer lugar do arquivo. A convenção é colocar após o código ou após __END__. Use perldoc lib/MeuModulo/Math.pm para visualizar.
