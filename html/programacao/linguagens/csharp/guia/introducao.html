C# — Introducao

Introducao ao C#

C# (C-Sharp) e uma linguagem de programacao multiparadigma, fortemente tipada, desenvolvida pela Microsoft como parte da plataforma .NET. Combina a expressividade de linguagens funcionais com a solidez de orientacao a objetos e componentes, sendo usada em desenvolvimento web, jogos, desktop, mobile e sistemas corporativos.

C# e a linguagem principal do ecossistema .NET — usado em ASP.NET Core para web de alta performance, Unity para jogos 3D, MAUI para aplicacoes cross-platform e Azure Functions para serverless. Com .NET 9 e C# 13, a linguagem continua evoluindo com performance de ponta e expressividade moderna.

Historia e Evolucao

2000 — C# 1.0

Anders Hejlsberg lidera o design da linguagem para a plataforma .NET 1.0. Influencias claras de Java e C++, mas com garbage collector integrado, propriedades nativas e delegates.

2005 — C# 2.0

Generics, iteradores com yield return, tipos anulaveis (int?), classes parciais e metodos anonimos. Base para o LINQ que viria a seguir.

2007 — C# 3.0 (LINQ)

Lambda expressions, LINQ (Language Integrated Query), tipos anonimos, var, metodos de extensao e expression trees. Revolucao na forma de manipular colecoes e dados.

2012 — C# 5.0 (async/await)

async/await torna programacao assincrona legivel como codigo sincrono. Task<T> se torna o padrao universal para operacoes I/O-bound.

2016 — .NET Core

Microsoft abre o codigo do .NET e lanca o .NET Core: cross-platform (Windows, Linux, macOS), alta performance, modular. ASP.NET Core reescrito do zero.

2020-2025 — C# 9-13

Records, pattern matching avancado, init-only, primary constructors, collection expressions, interceptors, params collections. .NET unificado a partir do .NET 5.

Instalacao e Primeiro Programa

Shell — instalar .NET SDK e criar projeto
# Instalar .NET SDK (Linux/macOS via Microsoft script)
# Windows: https://dot.net  ou  winget install Microsoft.DotNet.SDK.9
dotnet --version     # 9.0.xxx

# Criar projetos
dotnet new console -n MeuApp      # app de console
dotnet new webapi  -n MinhaApi    # ASP.NET Core API
dotnet new classlib -n MinhaLib   # biblioteca
dotnet new xunit   -n MeusTestes  # projeto de testes

cd MeuApp
dotnet run           # compilar e executar
dotnet build         # so compilar
dotnet publish -c Release -r linux-x64 --self-contained  # binario standalone
C# 13 — Hello World top-level e completo
// Program.cs — top-level statements (C# 9+): sem classe Main explicita
using System;

Console.WriteLine("Hello, World!");  // forma minima

// Interpolacao de strings
string nome = "Mundo";
Console.WriteLine($"Ola, {nome}!");      // interpolacao
Console.WriteLine($"2 + 2 = {2 + 2}");  // expresssao inline

// Versao completa com namespace e ponto de entrada explicito
namespace MeuApp;

class Program
{
  static void Main(string[] args)
  {
    Console.WriteLine("Argumentos recebidos:");
    foreach (var arg in args)
      Console.WriteLine($"  - {arg}");
  }
}

Filosofia e Paradigmas

Pontos Fortes do C#

  • Tipagem estatica expressiva: Generics, union types via discriminated unions, pattern matching com switch expressions
  • async/await nativo: Programacao assincrona de primeira classe, backpressure com IAsyncEnumerable<T>
  • LINQ: Query de colecoes com sintaxe declarativa identica a SQL mas type-safe em compile-time
  • Plataforma .NET: Garbage collector de geracao, JIT e AOT (Native AOT), BCL completa
  • Interoperabilidade: P/Invoke para C nativo, COM interop no Windows, Span<T> para acesso zero-copy
  • Unity: Engine de jogos mais popular do mundo usa C# como linguagem primaria

Desvantagens e Cuidados

  • Historico Windows-centrico: Muito do ecosistema e documentacao ainda assume Windows/Visual Studio
  • Startup time: Aplicacoes JIT tem startup lento vs Go ou execuciveis nativos (mitigado por AOT)
  • Verbosidade historica: Versoes antigas exigem boilerplate — classes, namespaces explicitos mesmo para scripts simples
  • memoria gerenciada: GC pode causar pausas em sistemas de tempo-real — exige unsafe e Span para controle fino
  • Ecossistema fechado de IDE: Visual Studio e pesado; VS Code + OmniSharp e alternativa mas menos completa

Onde C# e Usado

Web — ASP.NET Core

API REST, MVC, gRPC e Blazor (WASM). Benchmark TechEmpower: .NET consistentemente no top 5 de frameworks mais rapidos do mundo.

Jogos — Unity

70%+ dos jogos mobile e uma parcela significativa de PC/console usam Unity com C#. MonoBehaviour, ECS (Entities), Burst Compiler para performance nativa.

Desktop — MAUI e WPF

.NET MAUI para apps cross-platform (iOS, Android, Windows, macOS) com codigo compartilhado. WPF e WinForms para apps Windows legadas de produtividade.

Cloud — Azure e Serverless

Azure Functions, Durable Functions e Azure SDK em C#. Orleans para sistemas distribuidos tipo-actor. Aspire para orquestracao de microservicos locais.

IA e ML — ML.NET

ML.NET para machine learning nativo em .NET sem Python. TorchSharp para redes neurais. ONNX Runtime com bindings .NET para inferencia.

Ferramentas e Scripts

dotnet-script e csx para scripts inline. Roslyn para compilar e analisar C# em runtime. Source generators para metaprogramacao em compile-time.
