C# — Ecossistema e Tooling

Ecossistema e Tooling C# / .NET

O tooling do .NET e um dos mais maduros entre linguagens modernas: CLI dotnet para tudo, NuGet com 300k+ pacotes, Roslyn para analise e geracao de codigo, BenchmarkDotNet para performance, dotnet-trace para profiling, e suporte nativo em Visual Studio, Rider e VS Code.

Desde o .NET 5 (2020) a plataforma e unificada — nao mais .NET Framework vs .NET Core. O SDK inclui compilador (Roslyn), JIT (RyuJIT), GC, BCL, e a CLI. Uma instalacao, todas as plataformas.

dotnet CLI — Comandos Essenciais

Shell — dotnet CLI completo: criar, buildar, testar, publicar, ferramentas
# === Novos projetos ===
dotnet new list                     # listar templates disponíveis
dotnet new console -n App -f net9.0 # app console com .NET 9
dotnet new webapi -n Api --use-controllers=false  # Minimal API
dotnet new classlib -n Shared       # biblioteca
dotnet new xunit -n Testes
dotnet new sln -n Solucao           # arquivo de solucao
dotnet sln add App/App.csproj Testes/Testes.csproj

# === Build e Execucao ===
dotnet build                        # compilar projeto atual
dotnet run --project App            # compilar + executar
dotnet run -- --port 8080           # passar args para o programa
dotnet watch run                    # hot reload em mudancas de arquivo

# === Testes ===
dotnet test                         # executar todos os testes
dotnet test -v normal               # verboso
dotnet test --filter "FullyQualifiedName~PedidoTests"  # filtrar
dotnet test --collect:"XPlat Code Coverage"            # cobertura

# === Publicacao ===
dotnet publish -c Release -o ./out                          # publicar framework-dependente
dotnet publish -c Release -r linux-x64 --self-contained     # binario standalone
dotnet publish -c Release -r linux-x64 /p:PublishAot=true   # Native AOT
dotnet publish -c Release /p:PublishSingleFile=true         # arquivo unico

# === NuGet ===
dotnet add package Dapper                     # ultima estavel
dotnet add package MediatR --version 12.4.0
dotnet remove package AutoMapper
dotnet list package                           # pacotes instalados
dotnet list package --outdated               # desatualizados
dotnet restore                               # restaurar dependencias

# === Ferramentas globais ===
dotnet tool install -g dotnet-ef            # EF Core migrations CLI
dotnet tool install -g dotnet-trace         # profiling de producao
dotnet tool install -g dotnet-counters      # metricas em tempo real
dotnet tool install -g dotnet-dump         # analise de dumps de memoria
dotnet tool list -g                         # ferramentas instaladas

# Entity Framework migrations
dotnet ef migrations add IniciaProdutos --project Shared --startup-project Api
dotnet ef database update
dotnet ef dbcontext scaffold "ConnectionString" Npgsql.EntityFrameworkCore.PostgreSQL -o Models

<Project Sdk="Microsoft.NET.Sdk.Web">

  <PropertyGroup>

Performance e Profiling

C# + BenchmarkDotNet — micro-benchmark com memoryDiagnoser
using BenchmarkDotNet.Attributes;
using BenchmarkDotNet.Running;

[MemoryDiagnoser]
[SimpleJob(RuntimeMoniker.Net90)]
public class StringBenchmarks
{
  private const string Texto = "ola mundo como vai voce amigo querido";

  [Benchmark(Baseline = true)]
  public int ContarComSplit() => Texto.Split(' ').Length;

  [Benchmark]
  public int ContarComSpan()
  {
    int count = 0; bool emp = true;
    foreach (char c in Texto.AsSpan()) {
      if (c == ' ') { emp = true; }
      else if (emp) { count++; emp = false; }
    }
    return count;
  }
}

BenchmarkRunner.Run<StringBenchmarks>();
Shell — dotnet-trace, dotnet-counters e dotnet-dump
# Monitorar contadores em tempo real (CPU, GC, ThreadPool)
dotnet counters monitor --process-id 12345 System.Runtime

# Coletar trace de performance (10 segundos)
dotnet trace collect --process-id 12345 --duration 00:00:10

# Analisar trace no Speedscope (https://speedscope.app)
dotnet trace convert trace.nettrace --format Speedscope

# Criar dump de memoria para analise post-mortem
dotnet dump collect --process-id 12345

# Analisar dump — encontrar objetos que consomem mais memoria
dotnet dump analyze core.dump
# (no prompt do dump):
# dumpheap -stat
# gcroot <endereco_objeto>

CI/CD com GitHub Actions

YAML — GitHub Actions workflow: build, testes, cobertura e publicacao Docker
name: .NET CI

on:
  push:
    branches: [main, dev]
  pull_request:
    branches: [main]

jobs:
  build-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Configurar .NET 9
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '9.0.x'

      - name: Restaurar dependencias
        run: dotnet restore

      - name: Build Release
        run: dotnet build -c Release --no-restore

      - name: Executar testes com cobertura
        run: dotnet test -c Release --no-build --collect:"XPlat Code Coverage"

      - name: Publicar cobertura no Codecov
        uses: codecov/codecov-action@v4
        with:
          token: ${{ secrets.CODECOV_TOKEN }}

  docker:
    needs: build-test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      - name: Build imagem Docker
        run: docker build -t minha-api:${{ github.sha }} .
      - name: Push para registry
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USER }}" --password-stdin
          docker push minha-api:${{ github.sha }}

IDEs e Editores

Visual Studio 2022

IDE mais completa para .NET. IntelliSense avancado, debugger visual, designer WPF/MAUI, profiler integrado e Hot Reload. Apenas Windows (Community gratis).

JetBrains Rider

IDE cross-platform (Windows/Linux/macOS) com analise de codigo superior, refactoring poderoso e suporte a Unity. Pago mas consistentemente avaliado melhor que VS por muitos devs .NET.

VS Code + C# Dev Kit

Extensao oficial da Microsoft. IntelliSense via Roslyn, execucao de testes integrada, debugger, e Solution Explorer. Ideal para Linux/macOS ou ambientes leves.
