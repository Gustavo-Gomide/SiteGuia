Livewire — Componentes Reativos PHP sem JavaScript

Atualizacoes em tempo real do DOM direto do servidor PHP. Logica no PHP, resultado no browser.

Livewire e um framework de componentes reativos para Laravel, criado por Caleb Porzio. A ideia central: componentes PHP com propriedades public sao automaticamente sincronizados com o DOM no browser via Ajax. Quando o usuario interage com um input ou clica num botao, Livewire envia o estado ao servidor, re-renderiza o componente em PHP e aplica apenas o diff do HTML resultante no browser — sem escrever JavaScript manual. Livewire 3 introduziu sintaxe updated, hooks e integracao com Alpine.js para animacoes e interacoes client-side opcionais.

1. Componente Livewire

php — app/Livewire/ContadorBusca.php
<?php
namespace App\Livewire;

use Livewire\Component;
use App\Models\Produto;

class ContadorBusca extends Component
{
    // Propriedades public sao sincronizadas com wire:model no HTML
    public string $busca = '';
    public int $contador = 0;

    // Chamado automaticamente quando $busca muda (Livewire 3)
    public function updatedBusca()
    {
        $this->contador++;  // conta quantas vezes buscou
    }

    public function incrementar()
    {
        $this->contador++;
    }

    public function render()
    {
        return view('livewire.contador-busca', [
            'produtos' => Produto::where('nome', 'like', "%{$this->busca}%")
                                    ->get(),
        ]);
    }
}
html — resources/views/livewire/contador-busca.blade.php
<div>
    Buscas realizadas: <strong>{{ $contador }}</strong>

    <input type="text"
           wire:model.live="busca"
           placeholder="Buscar produto..." />
    <!-- wire:model.live sincroniza a cada keystroke via Ajax -->

    <ul>
        @foreach($produtos as $produto)
            <li>{{ $produto->nome }}</li>
        @endforeach
    </ul>

    <button wire:click="incrementar">+1</button>
</div>

2. Recursos do Livewire 3

wire:model.live / .lazy

Sincronizacao de inputs em tempo real (.live) ou apos blur (.lazy). Reduz roundtrips sem perder reatividade.

wire:loading

Mostra/esconde elementos durante requests Ajax. wire:loading.class="opacity-50" adiciona uma classe enquanto o servidor responde.

Alpine.js

Livewire e Alpine.js sao complementares. Alpine cuida de interacoes puras no cliente (modais, dropdowns) sem server roundtrip.

persist e url

#[Persist] salva o estado do componente no localStorage. #[Url] sincroniza propriedades com a URL (paginacao, filtros).

3. Quando Usar Livewire

Ideal Para

  • Apps Laravel que precisam de interatividade (tabelas reativas, formularios ao vivo) sem SPA completa
  • Times PHP que nao querem manter outro stack JS (Vue/React) separado
  • CRUDs e dashboards internos com forms de validacao e feedback ao vivo
  • Real-time com Livewire Polling ou Laravel Echo para broadcasts

Evitar Quando

  • App mobile ou SPA standalone — use Inertia.js + React/Vue ou API REST + app separado
  • Latencia de rede e critica — cada interacao e um roundtrip ao servidor PHP
  • Nao usa Laravel — Livewire e fortemente acoplado ao ecossistema Laravel/Blade
