Haxe OpenFL

OpenFL — A API Flash Ressuscitada em Haxe

O poder da DisplayList do Flash, sem o plugin. Jogos e apps 2D para web, desktop e mobile.

OpenFL (Open Flash Library) nasceu em 2013 para preservar o modelo de programacao do Adobe Flash/AS3 usando Haxe. A API cobre DisplayObject, Sprite, MovieClip, Stage, Event, bitmaps, texto rico, tweens e audio. O runtime pode ser OpenGL nativo (mais rapido) ou HTML5 Canvas/WebGL no browser. Jogos como Papers, Please foram portados usando esse pipeline.

1. O Modelo DisplayList

haxe — sprite e eventos na displaylist
import openfl.display.Sprite;
import openfl.events.MouseEvent;
import openfl.text.TextField;

class MeuQuadrado extends Sprite {
    var _label: TextField;

    public function new() {
        super();
        graphics.beginFill(0x3498DB);     // cor azul hex
        graphics.drawRoundRect(0, 0, 200, 80, 16);
        graphics.endFill();

        _label = new TextField();
        _label.text = "Clique aqui";
        _label.x = 50; _label.y = 25;
        addChild(_label);                   // adiciona a arvore de display

        buttonMode = true;
        addEventListener(MouseEvent.CLICK, onClick);
    }

    function onClick(e: MouseEvent) {
        _label.text = "Clicado!";
        // alpha, x, y, rotation sao propriedades nativas do DisplayObject
        alpha = 0.5;
    }
}

2. Ecossistema OpenFL

Actuate

Motor de tweens fluidos (animacoes de propriedades). Sintaxe como Actuate.tween(sprite, 0.5, {x: 100, alpha: 0}).

Away3D Haxe

Motor 3D que roda sobre OpenFL. Usa Stage3D (openfl.display3D) para renderizacao 3D acelerada por GPU.

OpenFL + Lime

Lime e a camada de sistema abaixo do OpenFL: janela, contexto GL, eventos nativos. Pode ser usada sozinha para apps muito baixo nivel.

HaxeFlixel

Engine de jogos 2D construida sobre OpenFL. E a opcao mais popular para jogos 2D em Haxe: FlxGame, FlxSprite, FlxGroup, tilemaps, cameras.

3. Quando Usar OpenFL

Ideal Para

  • Desenvolvedores com background em Flash/AS3 migrando para Haxe
  • Jogos 2D criados com HaxeFlixel (cross-platform including HTML5 e consoles)
  • Apps com UI rica baseada em DisplayObjects e tween animations
  • Portar projetos Flash antigos para plataformas modernas

Evitar Quando

  • Jogos 3D complexos — prefira Heaps + h3d ou Armory3D
  • Nao tem familiaridade com o modelo Flash DisplayList
  • Projetos de app nativo de desktop (sem UI similar a Flash)
