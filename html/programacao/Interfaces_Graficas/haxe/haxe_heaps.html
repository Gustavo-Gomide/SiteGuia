Heaps.io — Game Engine de Producao em Haxe

Dead Cells. Northgard. Evoland. Todos rodando no mesmo engine escrito em Haxe.

Heaps.io e a engine de jogos criada pela Motion Twin (estudio de Dead Cells) e usada comercialmente em titulos de sucesso no Steam. E uma engine 2D/3D que compila Haxe para C++ nativo (via HashLink VM ou HXCPP), HTML5/WebGL e outras plataformas. O foco e performance de producao: rendering GPU com shaders GLSL, gerenciamento de cena 3D, tilemap, audio e input unificados — tudo em Haxe com tipagem estatica forte.

1. Estrutura de um App Heaps

haxe — aplicacao heaps minima
class Main extends hxd.App {

    var texto: h2d.Text;
    var contador: Int = 0;

    override function init() {
        var font = hxd.res.DefaultFont.get();
        texto = new h2d.Text(font, s2d);
        texto.text = "Heaps.io: " + contador;
        texto.x = 50;
        texto.y = 50;
    }

    override function update(dt: Float) {
        contador++;
        texto.text = "Frame: " + contador;
    }

    static function main() {
        new Main();
    }
}

2. Camadas 2D e 3D

h2d — Cena 2D

Sprites (h2d.Bitmap), tilemaps (h2d.TileGroup), texto, UI interativa. Hierarquia de cena com h2d.Object como base.

h3d — Cena 3D

Meshes, materiais, luzes e cameras 3D via h3d.scene.Scene. Shaders HXSL (Haxe Shader Language) compilados para GLSL/HLSL automaticamente.

hxd.res — Assets

Sistema de recursos com acesso tipado: hxd.res.Res.myImage.toTile(). Recursos compilados no binario ou carregados de arquivo em tempo de execucao.

3. Quando Usar Heaps

Ideal Para

  • Jogos 2D de producao que precisam de performance nativa (Dead Cells compila para C++)
  • Times ja familiarizados com Haxe que querem uma engine integrada a linguagem
  • Jogos cross-platform: Windows, macOS, Linux e HTML5 do mesmo codigo
  • Prototipagem rapida de jogos com aceleracao GPU sem overhead de engine maior

Evitar Quando

  • Equipe sem experiencia em Haxe — curva de aprendizado dupla (linguagem + engine)
  • Ecossistema menor que Unity/Godot (menos tutoriais, assets, plugins)
  • Precisa de editor visual integrado — Heaps nao tem IDE/editor como Godot
