Kivy — Python para Mobile e Multi-Touch

Um Python roda no Android ou iOS? Com Kivy, sim. Renderizacao OpenGL ES em todas as plataformas.

Kivy e um framework open-source para criacao de aplicacoes multi-touch que roda em Windows, macOS, Linux, Android e iOS a partir do mesmo codigo Python. Diferente de Tkinter e PyQt, Kivy nao usa widgets do sistema operacional: renderiza tudo via OpenGL ES, o que garante aparencia consistente entre plataformas e permite graficos fluidos. Ideal para apps touchscreen, kiosks interativos e ferramentas que precisam rodar tanto no desktop quanto no celular.

1. KV Language

Kivy oferece uma linguagem declarativa propria chamada KV para descrever interfaces separadas da logica Python. Arquivos .kv descrevem a hierarquia de widgets, bindings de propriedades e estilos — similares a XAML (WPF) ou QML (Qt).

kv — layout declarativo
# app.kv
BoxLayout:
    orientation: "vertical"
    padding: 20
    spacing: 10

    Label:
        text: "Resultado: " + str(app.resultado)
        font_size: "20sp"

    TextInput:
        id: campo
        hint_text: "Digite algo"
        multiline: False
        on_text_validate: app.processar(self.text)

    Button:
        text: "Enviar"
        background_color: 0.2, 0.6, 1, 1
        on_press: app.processar(campo.text)

Binding automatico: a expressao text: str(app.resultado) cria um binding reativo — quando app.resultado muda, o Label atualiza automaticamente, sem callback manual.

2. Estrutura de um App Kivy

python — App minimo com KV
from kivy.app import App
from kivy.uix.boxlayout import BoxLayout
from kivy.properties import NumericProperty

class RootWidget(BoxLayout):
    pass  # layout definido em app.kv

class MeuApp(App):
    resultado = NumericProperty(0)  # propriedade observavel

    def build(self):
        return RootWidget()

    def processar(self, texto):
        try:
            self.resultado = eval(texto)  # exemplo didatico
        except Exception:
            self.resultado = -1

MeuApp().run()

3. Propriedades Observaveis

Kivy usa um sistema de propriedades reativas proprio em vez de variaveis Python normais. Quando o valor de uma Property muda, todos os bindings KV e callbacks registrados sao notificados automaticamente.

NumericProperty

Inteiro ou float observavel. Bindings KV atualizam automaticamente qualquer widget que referencie esta propriedade.

StringProperty

String observavel. Perfeita para sincronizar Labels, TextInputs e titulos de janela com o estado do app.

ListProperty / DictProperty

Coleções observaveis que disparam eventos ao adicionar, remover ou modificar itens — sem precisar chamar notify manualmente.

ObjectProperty

Referencia a qualquer objeto Python. Usada para expor widgets-filho ou modelos de dados ao sistema de binding do KV.

4. Eventos de Toque e Gestos

python — multi-touch e gestures
from kivy.uix.widget import Widget
from kivy.core.window import Window

class PainelTouch(Widget):
    def on_touch_down(self, touch):
        # touch.pos: (x, y) do toque
        # touch.uid: id unico para multi-touch
        print(f"Toque em {touch.pos}")
        with self.canvas:
            Color(1, 0, 0)
            Ellipse(pos=(touch.x-10, touch.y-10), size=(20, 20))
        return True  # consome o evento

    def on_touch_move(self, touch):
        print(f"Arrastando: {touch.pos}")

    def on_touch_up(self, touch):
        print(f"Solto: {touch.pos}")

5. Compilacao Mobile com Buildozer

Buildozer e a ferramenta oficial para empacotar apps Kivy para Android (.apk / .aab) e iOS. Funciona no Linux/macOS; para iOS e necessario macOS com Xcode.

shell — buildozer para android
# Inicializa buildozer.spec (arquivo de configuracao)
buildozer init

# buildozer.spec: configure title, package, requirements...
# requirements = python3,kivy,pillow

# Compila e deploya no dispositivo conectado via ADB
buildozer -v android debug deploy run

# Gera .aab para Google Play
buildozer android release

Buildozer rodando no Windows requer WSL2. A compilacao inicial baixa o Android NDK/SDK e pode demorar 30-60 minutos. Tenha paciencia na primeira execucao.

6. Quando Usar Kivy

Ideal Para

  • Apps Python que precisam rodar no Android e iOS
  • Aplicacoes multi-touch: paineis interativos, kiosks, visualizacoes
  • Jogos 2D simples em Python com animacoes fluidas
  • Prototipagem rapida de apps mobile sem aprender Dart ou Kotlin
  • Raspberry Pi com telas touchscreen (muito usado em instalacoes interativas)

Evitar Quando

  • Precisa de widgets 100% nativos iOS/Android (Kivy nao usa widgets do SO)
  • App store com aparencia padrao do sistema — use Flutter ou React Native
  • Equipe ja sabe PyQt/PySide e precisa de desktop apenas
  • Performance critica: Python + OpenGL nao compete com C++ Qt nem Flutter para apps pesados

7. Ecossistema

Buildozer

Ferramenta de build para empacotar apps Kivy em .apk/.aab (Android) e .ipa (iOS). Gerencia o toolchain NDK/SDK automaticamente.

KivyMD

Biblioteca de widgets Material Design 3 para Kivy. Botoes, cards, nav drawers, snackbars e dialogs com visual Google Material sobre o Kivy.

python-for-android

Runtime Python para Android usado internamente pelo Buildozer. Permite importar bibliotecas C (NumPy, Pillow, etc.) no APK.

Plyer

Abstrai APIs de plataforma (camera, GPS, vibrador, notificacoes) com uma interface Python unificada que funciona em Android, iOS e desktop.
