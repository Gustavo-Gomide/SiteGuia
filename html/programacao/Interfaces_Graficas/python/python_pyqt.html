1. Signals e Slots

O mecanismo de Signal & Slot e a espinha dorsal da comunicacao entre componentes no Qt. Um Signal e emitido quando algo acontece; um Slot e qualquer funcao ou metodo conectado a ele. Conexoes podem cruzar threads de forma segura.

python — signals e slots com PySide6
from PySide6.QtCore import QObject, Signal, Slot

class Contador(QObject):
    # Define um Signal que transporta int
    valor_mudou = Signal(int)

    def __init__(self):
        super().__init__()
        self._valor = 0

    def incrementar(self):
        self._valor += 1
        self.valor_mudou.emit(self._valor)  # emite o signal

# Receptor
@Slot(int)
def on_valor_mudou(novo_valor):
    print(f"Novo valor: {novo_valor}")

c = Contador()
c.valor_mudou.connect(on_valor_mudou)  # conecta signal ao slot
c.incrementar()  # imprime "Novo valor: 1"

Conexoes cross-thread funcionam automaticamente: se o sender e o receiver estao em threads distintas, Qt enfileira o signal na event queue da thread do receiver. Nao precisa de lock manual.

2. QMainWindow e Layouts

python — QMainWindow com layout e menu
from PySide6.QtWidgets import (
    QApplication, QMainWindow, QWidget,
    QVBoxLayout, QHBoxLayout, QLabel, QPushButton, QLineEdit
)
import sys

class JanelaPrincipal(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("App Qt")
        self.resize(640, 480)

        # Widget central com layout vertical
        central = QWidget()
        self.setCentralWidget(central)
        layout = QVBoxLayout(central)

        # Linha horizontal: label + campo
        linha = QHBoxLayout()
        linha.addWidget(QLabel("Buscar:"))
        self.campo = QLineEdit()
        linha.addWidget(self.campo)
        layout.addLayout(linha)

        btn = QPushButton("Buscar")
        btn.clicked.connect(self.buscar)
        layout.addWidget(btn)

        # Barra de status
        self.statusBar().showMessage("Pronto")

    def buscar(self):
        texto = self.campo.text()
        self.statusBar().showMessage(f"Buscando: {texto}")

if __name__ == "__main__":
    app = QApplication(sys.argv)
    app.setStyle("Fusion")
    janela = JanelaPrincipal()
    janela.show()
    sys.exit(app.exec())

3. Arquitetura Model/View

Qt implementa o padrao MVC de forma rigorosa. Model armazena e fornece os dados; View os exibe (QListView, QTableView, QTreeView); Delegate controla renderizacao e edicao de cada item. A vantagem: uma mudanca no model notifica automaticamente todas as views conectadas.

QAbstractItemModel

Classe base para modelos customizados. Implemente rowCount(), columnCount() e data(). dataChanged signal notifica a view de mudancas especificas.

QStandardItemModel

Model pronto para uso rapido sem subclasse. Ideal para tabelas simples onde voce adiciona linhas com appendRow() e le dados com item(row, col).text().

QSortFilterProxyModel

Intercepta um model existente e aplica filtro e ordenacao dinamicos. Voce nao modifica o model original — um proxy e inserido entre model e view para filtrar por texto ou criterio custom.

4. QThread e Tarefas em Background

python — QThread para tarefa pesada sem travar UI
from PySide6.QtCore import QThread, Signal

class Worker(QThread):
    progresso = Signal(int)
    concluido = Signal(str)

    def run(self):
        for i in range(100):
            processar_chunk(i)
            self.progresso.emit(i + 1)
        self.concluido.emit("Processamento completo!")

# Na janela principal:
worker = Worker()
worker.progresso.connect(progress_bar.setValue)
worker.concluido.connect(label.setText)
worker.start()  # inicia em thread separada; UI continua responsiva

5. Quando Usar PyQt/PySide

Ideal Para

  • Aplicacoes desktop profissionais com visual nativo e polido
  • Ferramentas cientificas e de engenharia (Qt prevalece nesses dominios)
  • Apps com tabelas complexas de dados usando Model/View
  • Interfaces que precisam de graficos com QChart ou WebEngine (browser embutido)
  • Projetos grandes onde a estrutura robusta do Qt compensa a curva inicial

Evitar Quando

  • Ferramentas simples e rapidas — Tkinter ou customtkinter bastam
  • Projeto com budget zero e preocupacao de licenca (PyQt GPL) — use PySide6
  • Precisa de mobile nativo — use Kivy ou Flutter
  • Bundle size critico — PyQt adiciona ~60MB ao installer

6. Topicos Avancados

Qt Designer

Editor visual drag-and-drop que gera arquivos .ui (XML). Carregue com uic.loadUi("janela.ui", self) ou compile previamente com pyside6-uic para um modulo Python.

QML (Qt Quick)

Linguagem declarativa para UIs fluidas e animadas. Ideal para interfaces estilo mobile/embedded. Acessa Python via QQmlApplicationEngine e propriedades expostas com @Property.

QtWebEngine

Chromium embutido via QWebEngineView. Permite exibir HTML/CSS/JS dentro de janelas Qt, criar dashboards com D3.js ou ate construir um navegador interno na sua aplicacao.

Internacionalizacao (i18n)

Qt tem suporte robusto via tr(), arquivos .ts (linguisticos) e lupdate/ lrelease. Troca de idioma em tempo de execucao sem reiniciar.

7. Ecossistema

PySide6 / PyQt6

Versoes modernas com Qt 6: melhor suporte HiDPI, novo modulo Qt Quick Controls 3, API widget limpa. PyQt6 mantem GPL; PySide6 mantem LGPL.

pyqtgraph

Biblioteca de graficos 2D/3D de alta performance construida sobre Qt. Usada em osciloscropios, analisadores de espectro e dashboards de dados em tempo real.

Qt-Material

Tema Material Design para PyQt5/PySide2/PySide6. Uma linha para aplicar cores e estilos Google Material a qualquer app Qt sem modificar o codigo de UI.

Nuitka / PyInstaller

Nuitka compila Python para C++ e gera executavel nativo com Qt embutido; PyInstaller empacota em pasta dist. Nuitka gera binarios menores e mais rapidos para distribuicao.
