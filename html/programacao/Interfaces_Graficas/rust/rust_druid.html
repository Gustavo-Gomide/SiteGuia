Rust Druid

Druid — GUI Data-Driven e Passagem para Xilem

Framework GUI Rust orientado a dados, com modelo Lens e a evolucao para Xilem.

Druid foi criado pelo Linebender (liderado por Raph Levien, autor do Ghostscript/Skia) como framework GUI native-Rust data-driven. A ideia central e o sistema de Lens: tipos que descrevem como acessar um campo de um estado imutavel maior. Druid usa Piet para renderizacao 2D e tem suporte experimental de acessibilidade. Hoje, Druid esta em modo de manutencao: o desenvolvimento ativo migrouAqui para Xilem, sua reimplementacao com uma arquitetura reativa mais moderna inspirada no SwiftUI e Jetpack Compose. Conhecer Druid ainda e util para entender os fundamentos que ele estabeleceu.

1. Modelo de Dados com Lens

rust — contador simples com AppLauncher e Lens
use druid::widget::{Button, Flex, Label};
use druid::{AppLauncher, Data, Lens, Widget, WidgetExt, WindowDesc};

// Estado da aplicacao
#[derive(Clone, Data, Lens)]
struct AppState {
    contador: i64,
}

fn ui_builder() -> impl Widget<AppState> {
    let label = Label::new(|data: &AppState, _env: &_| {
        format!("Valor: {}", data.contador)
    });

    let btn_inc = Button::new("+")
        .on_click(|_ctx, data: &mut AppState, _env| data.contador += 1);

    let btn_dec = Button::new("-")
        .on_click(|_ctx, data: &mut AppState, _env| data.contador -= 1);

    Flex::column()
        .with_child(label)
        .with_spacer(8.0)
        .with_child(Flex::row().with_child(btn_dec).with_child(btn_inc))
}

fn main() {
    let main_window = WindowDesc::new(ui_builder)
        .title("Druid Contador")
        .window_size((250.0, 120.0));

    AppLauncher::with_window(main_window)
        .launch(AppState { contador: 0 })
        .expect("Falha ao iniciar app");
}
Druid em modo de manutencao: desde 2023 o development ativo de novos recursos foi transferido para Xilem. O repositorio Druid ainda existe mas nao recebe novas features.

2. A Transicao para Xilem

Druid — Lens

Estado mutavel acessado via tipos Lens; widgets recebem &mut do dado diretamente. Modelo direto, mas dificulta compor visualizacoes complexas.

Xilem — View Tree

Arvore de Views descritiva (como SwiftUI) reconciliada a cada update. Separacao clara entre View (descricao) e Element (estado renderizador).

Masonry — Widget Layer

Camada de widgets de baixo nivel extraida do Druid. Xilem usa Masonry internamente; pode ser usada independentemente.

Piet → Vello

Druid usava Piet (2D abstraction over Direct2D/CoreGraphics/OpenGL). Xilem usa Vello: renderizador GPU-first via wgpu para alta performance.

rust — xilem — contador reativo (API moderna)
// Xilem usa uma abordagem de View tree, similar ao SwiftUI
use xilem::view::{button, flex, label};
use xilem::{EventLoop, WidgetView, Xilem};

#[derive(Default)]
struct Contador {
    valor: i32,
}

fn app_logic(data: &mut Contador) -> impl WidgetView<Contador> {
    flex((
        label(format!("Valor: {}", data.valor)),
        button("+", |data: &mut Contador| data.valor += 1),
        button("-", |data: &mut Contador| data.valor -= 1),
    ))
}

fn main() {
    let app = Xilem::new(Contador::default(), app_logic);
    app.run_windowed(EventLoop::new().unwrap(), "Xilem Contador").unwrap();
}

3. Quando Usar Druid / Xilem

Considerar Quando

  • Quer aprender fundamentos de GUI Rust-native antes de adotar um framework maduro
  • Estuda arquiteturas de datflow e lens para entender como Xilem evoluiu
  • Projetos academicos ou experimentais que exploram o ecosistema Linebender

Evitar Para

  • Producao: prefira Xilem (desenvolvimento ativo), Iced (maduro) ou Tauri (web frontend)
  • APIs instavel: tanto Druid quanto Xilem ainda fazem breaking changes
