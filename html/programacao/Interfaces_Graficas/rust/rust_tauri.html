Rust — Tauri

Rust Tauri

Tauri — Desktop com Rust + Frontend Web (~3 MB)

O antidoto ao Electron: usa a webview nativa do SO e backend Rust. Apps de 3 MB em vez de 150 MB.

Tauri une um frontend em qualquer framework web (React, Vue, Svelte, Solid) com um backend em Rust de alta performance. Em vez de empacotar o Chromium inteiro como o Electron, Tauri usa a webview nativa do SO: WKWebView no macOS, WebView2 no Windows e WebKitGTK no Linux. O resultado: apps de 3–10 MB, uso de RAM drasticamente menor e tempo de startup mais rapido. Tauri v2 (2024) adicionou suporte a mobile (iOS/Android) e redesenhou o sistema de plugins e permissoes.

1. Arquitetura e IPC

rust — comando tauri (backend src-tauri/src/lib.rs)
use tauri::Manager;

// Tauri command: expoe funcao Rust para o frontend JS
#[tauri::command]
fn ler_arquivo(caminho: String) -> Result<String, String> {
    std::fs::read_to_string(&caminho)
        .map_err(|e| e.to_string())
}

#[tauri::command]
async fn buscar_dados(url: String) -> Result<String, String> {
    // async nativo — nao trava a UI
    reqwest::get(&url).await
        .map_err(|e| e.to_string())?
        .text().await
        .map_err(|e| e.to_string())
}

pub fn run() {
    tauri::Builder::default()
        .invoke_handler(tauri::generate_handler![ler_arquivo, buscar_dados])
        .run(tauri::generate_context!())
        .expect("Erro ao iniciar app Tauri");
}
typescript — invocar comando rust do frontend
import { invoke } from '@tauri-apps/api/core';

// Chama o comando Rust sem escrever IPC manual
const conteudo = await invoke<string>('ler_arquivo', {
    caminho: '/home/user/notas.txt'
});
console.log(conteudo);

2. Modelo de Seguranca

json — capabilities em tauri.conf.json (Tauri v2)
{
  "app": {
    "security": {
      "csp": "default-src 'self'; script-src 'self'"
    }
  },
  "plugins": {
    "fs": {
      // Permite acesso apenas ao diretorio de dados do app
      "scope": { "allow": ["$APPDATA/**"] }
    },
    "http": {
      // Permite requests apenas para dominios autorizados
      "scope": { "allow": ["https://api.minhaapp.com/**"] }
    }
  }
}

Permissoes granulares: Tauri v2 exige declarar explicitamente quais permissoes o app tem (filesystem, http, notificacoes, etc). Por padrao tudo e negado — diferente do Electron onde o acesso ao Node.js e optado desativando flags. Isso reduz drasticamente a superficie de ataque.

3. Tauri vs Electron

Tamanho

Tauri: ~3–10 MB. Electron: ~150 MB. A diferenca e o Chromium embutido no Electron vs webview nativa do OS no Tauri.

Uso de RAM

Tauri usa significativamente menos RAM por nao instanciar um processo Chromium separado. Importante para apps de baixo consumo.

Consistencia de UI

Electron garante renderizacao identica em todas as plataformas (Chromium fixo). Tauri pode ter pequenas diferencas de CSS/JS entre plataformas por usar webviews diferentes.

Backend

Electron: backend em Node.js. Tauri: backend em Rust (performance, seguranca de memoria, sem GC). Tauri 2 tambem suporta Kotlin/Swift para plugins nativos.

4. Quando Usar Tauri

Ideal Para

  • Apps onde tamanho do binario importa (instaladores menores, mais rapido de baixar)
  • Apps com operacoes pesadas no backend onde Rust da vantagem real de performance
  • Equipes que conhecem Rust e web tech e querem evitar Node.js no backend
  • Apps mobile + desktop cross-platform com Tauri v2

Evitar Quando

  • Time nao sabe Rust e precisa de produtividade maxima — Electron tem ecossistema maior
  • UI deve ser identica pixel-perfect em todas as plataformas — webviews divergem
  • Muitos modulos Node.js existentes que ja funcionam em Electron
