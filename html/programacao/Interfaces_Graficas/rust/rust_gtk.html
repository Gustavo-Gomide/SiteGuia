GTK-rs — Bindings GTK4 para Rust

Apps desktop GNOME-native, estilizados com CSS, desenvolvidos em Rust. Com builder de UI declarativo.

GTK-rs sao os bindings Rust oficiais para GTK4, GLib, GDK, Gio e PangoCairo. O projeto e mantido pela comunidade com suporte do GNOME. As APIs Rust refletem o modelo de objetos GTK (GObject) de forma idiomatica usando traits como IsA<Widget> e o sistema de sinais com closures. A criacao de UI pode ser feita programaticamente ou via XML Blueprint / GTK UI Builder (arquivos .ui carregados com GtkBuilder). Projetos notaveis em GTK-rs: o app GNOME Calculator, alguns apps do ecosssistema Fractal (Matrix).

1. App GTK4 em Rust

rust — janela gtk4 com botao e label
use gtk4::prelude::*;
use gtk4::{Application, ApplicationWindow, Box, Button, Label, Orientation};

fn main() {
    let app = Application::builder()
        .application_id("org.exemplo.MeuApp")
        .build();

    app.connect_activate(|app| {
        let label = Label::builder()
            .label("Ola, GTK-rs!")
            .margin_top(16).margin_bottom(16)
            .build();

        let btn = Button::with_label("Clique");

        // Captura `label` na closure com clone (GTK usa Rc internamente)
        let label_clone = label.clone();
        btn.connect_clicked(move |_| {
            label_clone.set_label("Clicado!");
        });

        let vbox = Box::new(Orientation::Vertical, 8);
        vbox.append(&label);
        vbox.append(&btn);

        let window = ApplicationWindow::builder()
            .application(app)
            .title("App GTK-rs")
            .default_width(300).default_height(150)
            .child(&vbox)
            .build();
        window.present();
    });

    app.run();
}

2. UI Declarativa com Blueprint / .ui XML

xml — window.ui (GTK Builder XML)
<?xml version="1.0" encoding="UTF-8"?>
<interface>
  <object class="GtkApplicationWindow" id="main_window">
    <property name="title">App GTK-rs</property>
    <property name="default-width">400</property>
    <child>
      <object class="GtkBox">
        <property name="orientation">vertical</property>
        <property name="spacing">8</property>
        <child><object class="GtkLabel" id="minha_label">
          <property name="label">Ola do XML!</property>
        </object></child>
      </object>
    </child>
  </object>
</interface>
rust — carregando o arquivo .ui com GtkBuilder
let builder = gtk4::Builder::from_resource("/org/exemplo/window.ui");
let window: ApplicationWindow = builder.object("main_window").unwrap();
let label: Label = builder.object("minha_label").unwrap();
window.present();

3. Quando Usar GTK-rs

Ideal Para

  • Apps GNOME/Linux-native que precisam integration total com o ecossistema GTK
  • Projetos Rust onde a aparencia nativa GTK e requisito (distros Linux)
  • Desenvolvedores que ja conhecem GTK em C e querem Rust com seguranca de memoria

Evitar Quando

  • Foco em Windows/macOS: GTK tem aparencia menos nativa fora do Linux
  • Prefere uma API mais idiomatica Rust sem GObject internamente — use Iced ou Tauri

Referencias Bibliograficas

  • GTK-RS TEAM. gtk4-rs Book. Disponivel em: gtk-rs.org/gtk4-rs/stable/latest/book. Acesso em: 2026.
  • Iced — GUI Rust-native com Elm Architecture
