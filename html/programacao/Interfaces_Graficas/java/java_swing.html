Java — Swing

Java Swing

Swing — O Toolkit GUI Classico da JVM

O framework que dominou o desktop Java por mais de 20 anos. Ainda amplamente usado em sistemas legados e ferramentas corporativas.

Swing faz parte da JDK desde a versao 1.2 (1998). Todos os componentes sao desenhados em Java puro (nao sao wrappers de widgets nativos do SO), o que garante aparencia consistente entre plataformas ao custo de nao parecer exatamente nativa em cada OS. O EDT (Event Dispatch Thread) e o coracao do modelo de concorrencia: toda atualizacao de UI deve rodar nessa thread. IDEs como IntelliJ IDEA e NetBeans historicamente usaram Swing para sua propria UI.

1. JFrame e Estrutura Basica

java — janela swing com painel e botao
import javax.swing.*;
import java.awt.*;

public class MinhaJanela {
    public static void main(String[] args) {
        // SEMPRE criar UI na EDT (Event Dispatch Thread)
        SwingUtilities.invokeLater(() -> {
            JFrame frame = new JFrame("Minha App Swing");
            frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
            frame.setSize(400, 300);

            JPanel panel = new JPanel(new BorderLayout());

            JLabel label = new JLabel("Ola, Swing!", SwingConstants.CENTER);
            JButton btn = new JButton("Clique");

            btn.addActionListener(e ->
                label.setText("Clicado!")
            );

            panel.add(label, BorderLayout.CENTER);
            panel.add(btn,   BorderLayout.SOUTH);
            frame.add(panel);
            frame.setVisible(true);
        });
    }
}

EDT e obrigatorio: Qualquer modificacao de componente Swing feita fora da EDT pode causar bugs de estado visual impredem e race conditions. Use sempre SwingUtilities.invokeLater() para agendar operacoes de UI.

2. Layout Managers

BorderLayout

Divide o container em North, South, East, West e Center. Cada regiao aceita um componente. O mais usado para estrutura basica de janela.

FlowLayout

Componentes em linha horizontal, quebrando para a proxima linha quando necessario. Padrao dos JPanel. Bom para grupos de botoes.

GridLayout

Grade de linhas x colunas com tamanhos iguais. Todas as celulas tem o mesmo tamanho. Simples para teclados virtuais ou formularios pequenos.

GridBagLayout

O mais poderoso e complexo. Define GridBagConstraints para cada componente: posicao, span, pesos e insets. Para layouts finos de formularios.

BoxLayout

Empilha componentes vertical (Y_AXIS) ou horizontal (X_AXIS) respeitando preferred/min/max sizes. Alternativa simples ao GridBagLayout.

GroupLayout

Criado para GUI Builders (NetBeans Matisse). Define grupos horizontais e verticais independentemente. Bom para formularios gerados por ferramentas visuais.

3. Concorrencia com SwingWorker

java — tarefa demorada sem travar a EDT
import javax.swing.SwingWorker;

// T = tipo do resultado final, V = tipo do progresso intermediario
SwingWorker<String, Integer> worker = new SwingWorker<>() {
    @Override
    protected String doInBackground() throws Exception {
        // roda em background thread (NAO na EDT)
        for (int i = 0; i <= 100; i += 10) {
            Thread.sleep(100);
            publish(i);             // envia progresso
        }
        return "Concluido!";
    }

    @Override
    protected void process(java.util.List<Integer> chunks) {
        // roda na EDT com cada lote de progresso publicado
        progressBar.setValue(chunks.get(chunks.size() - 1));
    }

    @Override
    protected void done() {
        // roda na EDT quando doInBackground termina
        try {
            label.setText(get());  // get() retorna resultado
        } catch (Exception ex) { ex.printStackTrace(); }
    }
};
worker.execute();

4. Look and Feel

java — trocando aparencia com UIManager
// Usar L&F do sistema operacional (recomendado para apps desktop)
UIManager.setLookAndFeel(UIManager.getSystemLookAndFeelClassName());

// FlatLaf — look moderno, muito popular em projetos novos
// (dependencia externa: com.formdev:flatlaf)
com.formdev.flatlaf.FlatDarkLaf.setup();

// Theming de cores individuais
UIManager.put("Button.background", new java.awt.Color(0x2D2D2D));
UIManager.put("Button.foreground", java.awt.Color.WHITE);

A biblioteca FlatLaf tornou-se o padrao de fato para modernizar a aparencia de apps Swing — e usada pelo IntelliJ IDEA e muitas IDEs modernas da JetBrains.

5. Quando Usar Swing

Ideal Para

  • Manutencao e evolucao de sistemas legados que ja usam Swing
  • Ferramentas internas corporativas onde aparencia modernissima nao e prioridade
  • Aprendizado de conceitos de GUI (EDT, layouts, events) antes de JavaFX
  • Apps que precisam rodar em ambientes Java antigos (pre-Java 8)

Evitar Quando

  • Projetos novos de desktop Java — prefira JavaFX com FXML e CSS
  • UI complexa com animacoes, graficos vetoriais ou estilos modernos
  • Nao quer depender de bibliotecas como FlatLaf para visual aceitavel
