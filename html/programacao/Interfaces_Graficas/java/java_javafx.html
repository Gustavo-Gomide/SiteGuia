JavaFX — GUI Moderna Declarativa para Java

Scene Graph, FXML, CSS e propriedades reativas. O sucessor oficial do Swing para apps desktop Java.

JavaFX foi lancado em 2008 e recuperou o foco como plataforma desktop Java quando a Oracle transferiu o projeto para a comunidade open-source (OpenJFX) em 2018. O modelo central e o Scene Graph: uma arvore de nos Node que representam tudo que aparece na tela. O estado visual e controlado por CSS, o layout e declarado em FXML (XML parecido com HTML) e o comportamento e vinculado via Properties e Bindings reativos. E a escolha natural para novos projetos de desktop em Java ou Kotlin.

1. FXML e Controladores

xml — layout declarativo em hello.fxml
<?xml version="1.0" encoding="UTF-8"?>
<?import javafx.scene.layout.*?>
<?import javafx.scene.control.*?>

<VBox xmlns:fx="http://javafx.com/fxml"
      fx:controller="MeuController"
      spacing="16" padding="24">

    <Label fx:id="label" text="Ola, JavaFX!" />
    <TextField fx:id="campo" promptText="Digite algo..." />
    <Button text="Confirmar"
            onAction="#handleClick"
            styleClass="btn-primary" />
</VBox>
java — controlador injetado pelo FXMLLoader
public class MeuController {
    @FXML private Label label;
    @FXML private TextField campo;

    @FXML
    private void handleClick() {
        label.setText("Ola, " + campo.getText());
    }
}

// Na classe Application.start():
FXMLLoader loader = new FXMLLoader(getClass().getResource("hello.fxml"));
Scene scene = new Scene(loader.load());
primaryStage.setScene(scene);
primaryStage.show();

2. Properties e Bindings Reativos

java — property observavel e binding bidirecional
import javafx.beans.property.*;
import javafx.beans.binding.*;

// Property observavel — wrapper de valor com listeners
StringProperty nome = new SimpleStringProperty("Ana");
IntegerProperty idade = new SimpleIntegerProperty(25);

// Binding: label.text sempre espelha "nome + ' — ' + idade"
label.textProperty().bind(
    nome.concat(" — ").concat(idade.asString())
);

// Binding bidirecional: textField <-> property em sincronia
textField.textProperty().bindBidirectional(nome);

// Listener manual
nome.addListener((obs, oldVal, newVal) ->
    System.out.println("Nome mudou para: " + newVal)
);

// ObservableList — lista que dispara eventos de mudanca
ObservableList<String> itens = FXCollections.observableArrayList("A", "B");
listView.setItems(itens);
itens.add("C"); // ListView atualiza automaticamente

3. Estilizando com CSS

css — arquivo app.css para JavaFX
/* Seletor de tipo no JavaFX: usa o nome da classe sem .java */
.btn-primary {
    -fx-background-color: #2980B9;
    -fx-text-fill: white;
    -fx-font-size: 14px;
    -fx-padding: 8 20;
    -fx-border-radius: 8;
    -fx-background-radius: 8;
}

.btn-primary:hover {
    -fx-background-color: #1A6090;
}

/* Variaveis CSS do JavaFX (fx-look) */
.root {
    -fx-font-family: "Segoe UI", sans-serif;
    -fx-base: #1E1E1E;
}
java — aplicar CSS na scene
scene.getStylesheets().add(getClass().getResource("app.css").toExternalForm());

4. Task e Service para Background

java — Task com progresso e resultado
import javafx.concurrent.Task;

Task<String> tarefa = new Task<>() {
    @Override
    protected String call() throws Exception {
        for (int i = 0; i <= 100; i++) {
            Thread.sleep(20);
            updateProgress(i, 100);     // dispara progressProperty
            updateMessage("Etapa " + i); // dispara messageProperty
        }
        return "Pronto!";
    }
};

progressBar.progressProperty().bind(tarefa.progressProperty());
label.textProperty().bind(tarefa.messageProperty());

tarefa.setOnSucceeded(e -> label.setText(tarefa.getValue()));
new Thread(tarefa).start();

5. Quando Usar JavaFX

Ideal Para

  • Novos projetos desktop Java que precisam de UI moderna e estilavel
  • Apps rico-cliente em organizacoes que ja usam Java no backend
  • Kotlin desktop: TornadoFX usa JavaFX como base
  • GraalVM Native Image pode gerar executaveis standalone com JavaFX via Gluon

Evitar Quando

  • App mobile — JavaFX nao e adequado para Android/iOS nativos
  • Deploys no ambiente web (sem JVM no cliente) — use GWT ou frameworks JS
  • Cenarios de aprendizado de GUI basico — comece com Swing antes do JavaFX
