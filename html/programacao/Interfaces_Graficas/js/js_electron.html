Electron — Apps Desktop com HTML, CSS e JS

O framework que alimenta VS Code, Slack, Discord, Figma e Notion. Web tech para o desktop.

Electron (criado pela GitHub em 2013, originalmente como Atom Shell) combina um runtime Chromium para a UI com um runtime Node.js para acesso ao sistema. Isso permite que desenvolvedores web criem apps desktop completos usando as mesmas tecnologias que ja conhecem. O preco e o tamanho: um app Electron simples começa em torno de 150 MB descomprimido porque embute o Chromium inteiro. Apesar disso, o ecossistema e enorme e a produtividade e altissima, tornando-o uma das plataformas desktop mais usadas no mundo.

1. Processo Main e Processo Renderer

javascript — main.js (processo principal)
const { app, BrowserWindow, ipcMain } = require('electron');
const path = require('path');

function createWindow() {
    const win = new BrowserWindow({
        width: 900, height: 600,
        webPreferences: {
            preload: path.join(__dirname, 'preload.js'),
            // contextIsolation DEVE ser true — separacao de seguranca
            contextIsolation: true,
            nodeIntegration: false  // NUNCA true em apps reais
        }
    });
    win.loadFile('index.html');
}

app.whenReady().then(createWindow);
app.on('window-all-closed', () => app.quit());
javascript — preload.js e renderer.js com contextBridge
// preload.js — a ponte segura entre main e renderer
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('api', {
    getVersion: () => ipcRenderer.invoke('get-version'),
    saveFile: (data) => ipcRenderer.invoke('save-file', data)
});

// renderer.js — codigo da pagina web tem acesso apenas a window.api
const version = await window.api.getVersion();
document.getElementById('ver').textContent = version;

// main.js — handler IPC do lado do Node.js
ipcMain.handle('get-version', () => app.getVersion());

Seguranca critica: Nunca habilite nodeIntegration: true em apps que carregam conteudo externo. Com contextIsolation: true + preload, o renderer nao tem acesso direto ao Node.js, eliminando XSS que se torna RCE (Remote Code Execution).

2. APIs Nativas do Electron

dialog

Caixas de dialogo nativas do SO: dialog.showOpenDialog(), showSaveDialog(), showMessageBox(). Aspecto completamente nativo.

Tray e Menu

Icone na bandeja do sistema com Tray e menu de contexto com Menu.buildFromTemplate(). Permite apps que ficam rodando em segundo plano.

shell e clipboard

shell.openExternal(url) abre o browser padrao. shell.openPath() abre arquivos com o app padrao. clipboard.writeText() para a area de transferencia.

autoUpdater

Atualizacoes automaticas em background com electron-updater (do electron-builder). Integra com GitHub Releases, S3 ou servidores proprios.

globalShortcut

Atalhos de teclado globais (mesmo quando o app nao esta em foco): globalShortcut.register('Ctrl+Shift+K', fn). Comum em launchers e utilidades.

nativeTheme

Detecta e reage ao tema do SO (claro/escuro). nativeTheme.shouldUseDarkColors e mudancas emitidas via evento updated.

4. Quando Usar Electron

Ideal Para

  • Equipe ja conhece web (React, Vue, Svelte) e precisa de um app desktop rapidamente
  • Apps com UI complexa e rica que se beneficiam de um browser engine completo
  • Ferramentas de produtividade, editores de codigo, clientes de chat, IDEs
  • Compartilhar codigo entre web e desktop com pouco esforco extra

Evitar Quando

  • Tamanho do binario e critico (<10 MB) — considere Tauri (Rust + webview nativo)
  • Performance de CPU/GPU e crucial — o overhead do Chromium e real
  • Apps simples de sistema — Tauri ou Qt/GTK producem apps menores e mais eficientes
