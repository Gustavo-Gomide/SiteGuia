React Native

React Native — Apps iOS e Android em JavaScript

Da Meta. Componentes nativos reais, nao WebViews. Aprenda uma vez, escreva em qualquer lugar.

React Native foi lancado pela Meta (Facebook) em 2015. A premissa e poderosa: escrever componentes React que renderizam widgets nativos reais do iOS e Android, nao elementos HTML num WebView. Isso significa que um <View> em React Native se torna um UIView no iOS e um android.view.View no Android. Em 2024, a nova arquitetura com JSI (JavaScript Interface) substituiu a ponte assincronas antiga por chamadas sincronas e diretas, melhorando drasticamente performance e interoperabilidade nativa.

1. Componentes Fundamentais

jsx — tela basica com hooks e StyleSheet
import React, { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, StyleSheet }
       from 'react-native';

export default function TelaLogin() {
    const [email, setEmail] = useState('');

    return (
        <View style={styles.container}>
            <Text style={styles.titulo}>Entre na sua conta</Text>

            <TextInput
                style={styles.input}
                placeholder="email@exemplo.com"
                value={email}
                onChangeText={setEmail}
                keyboardType="email-address"
                autoCapitalize="none"
            />

            <TouchableOpacity style={styles.btn} onPress={() => console.log(email)}>
                <Text style={styles.btnTexto}>Entrar</Text>
            </TouchableOpacity>
        </View>
    );
}

const styles = StyleSheet.create({
    container: { flex: 1, padding: 24, justifyContent: 'center' },
    titulo:    { fontSize: 22, fontWeight: 'bold', marginBottom: 16 },
    input:     { borderWidth: 1, borderRadius: 8, padding: 12, marginBottom: 12 },
    btn:       { backgroundColor: '#2980B9', padding: 14, borderRadius: 8, alignItems: 'center' },
    btnTexto:  { color: 'white', fontWeight: 'bold' }
});

2. Expo e EAS Build

Expo Go

App que escaneia um QR code e roda seu projeto em qualquer dispositivo iOS/Android sem compilar. Ciclo de desenvolvimento instantaneo — ideal para aprendizado e prototipagem.

Expo SDK

Conjunto de modulos prontos: Camera, Notifications, FileSystem, Contacts, Location, Sensors. Evita ter que escrever codigo nativo (Swift/Kotlin) para funcionalidades comuns.

EAS Build

Servico de CI/CD na nuvem da Expo: compila APK/AAB/IPA em servidores gerenciados sem precisar de Mac local para builds iOS. Integrava com GitHub Actions.

EAS Update

OTA (Over-The-Air) updates: atualiza o bundle JavaScript do app sem precisar republitar na App Store ou Play Store. Apenas codigo JS pode ser atualizado via OTA.

3. Navegacao com React Navigation

jsx — stack navigator com expo-router (file-based routing)
// Com Expo Router (recomendado desde 2023), estrutura de arquivos define rotas:
// app/index.tsx   → tela "/"
// app/perfil.tsx  → tela "/perfil"
// app/[id].tsx    → tela "/123" com params.id = "123"

// app/index.tsx
import { Link } from 'expo-router';

export default function Home() {
    return (
        <Link href="/perfil">Ir para Perfil</Link>
    );
}

// app/perfil.tsx
import { useRouter } from 'expo-router';

export default function Perfil() {
    const router = useRouter();
    return <Button title="Voltar" onPress={() => router.back()} />;
}

4. Quando Usar React Native

Ideal Para

  • Equipe com experiencia em React querendo entrar no mobile sem aprender Swift/Kotlin
  • Apps de conteudo, e-commerce, redes sociais — onde a UI e majoritariamente listas e formularios
  • Compartilhar logica de negocio entre web (React) e mobile com NativeWind
  • MVP rapido para validar produto antes de investir em nativo

Evitar Quando

  • Jogos ou apps com graficos pesados — Flutter ou nativo sao melhores
  • Acesso intensivo a APIs nativas muito especificas — pode exigir modulos nativos extras
  • Precisao de pixels perfeita para imitar o design nativo de cada plataforma — Flutter tem mais controle
