NW.js

NW.js — O Predecessor do Electron

Chromium + Node.js em contexto unificado. Mais simples de comecar, menos isolado por design.

NW.js (Node-WebKit) foi criado por Roger Wang em 2011, tornando-se o primeiro framework a unificar Node.js e um browser engine para apps desktop. A diferenca principal em relacao ao Electron e o contexto compartilhado: no NW.js, o objeto window do browser e o contexto do Node.js sao o mesmo, permitindo chamar require('fs') diretamente no HTML. Isso simplifica o codigo mas reduz o isolamento de seguranca. O Electron surgiu depois com a arquitetura main/renderer mais segura. NW.js ainda recebe atualizacoes e tem uma base de usuarios fiel.

1. Estrutura de App NW.js

json — package.json de um app NW.js
{
  "name": "meu-app",
  "main": "index.html",
  "window": {
    "title":   "Meu App NW.js",
    "width":   800,
    "height":  600,
    "toolbar": false,
    "frame":   true
  },
  "chromium-args": "--disable-web-security"
}
html — index.html com Node.js direto no browser
<script>
  // No NW.js, require() esta disponivel diretamente no contexto do browser
  const fs = require('fs');
  const os = require('os');

  document.querySelector('#info').textContent =
    `Sistema: ${os.platform()} | CPU: ${os.cpus()[0].model}`;

  document.querySelector('#salvar').addEventListener('click', () => {
    fs.writeFileSync('saida.txt', 'Salvo pelo NW.js!');
  });
</script>

2. NW.js vs Electron

Contexto Compartilhado

NW.js: browser e Node no mesmo contexto. require() funciona diretamente no HTML sem preload. Electron: dois contextos separados com IPC explicito.

Seguranca

Electron com contextIsolation e mais seguro por padrao. NW.js com contexto compartilhado expoe mais superficie de ataque se o app carrega conteudo externo.

Ecossistema

Electron tem ecossistema muito maior (electron-builder, electron-forge, Spectron para testes). NW.js tem menos ferramentas mas e mais simples para comecar.

Ponto de Entrada

NW.js: ponto de entrada e uma pagina HTML. Electron: ponto de entrada e um arquivo JS (main.js). NW.js e mais intuitivo para desenvolvedores que pensam em HTML primeiro.

3. Quando Usar NW.js

Ideal Para

  • Projetos que ja usam NW.js para manutencao sem migracao
  • Prototipagem rapida onde o contexto compartilhado simplifica o codigo
  • Apps internos onde seguranca de contexto nao e a preocupacao principal

Evitar Quando

  • Novos projetos — Electron tem ecossistema maior e modelo de seguranca melhor
  • Apps que precisam carregar URLs externas — contexto compartilhado e risco
  • Tamanho do app importa — considere Tauri como alternativa muito mais leve
