GTK — O Toolkit do GNOME

O mesmo toolkit que roda GIMP, GNOME Shell e Inkscape — direto em C, sem abstrações.

GTK (GIMP Toolkit) nasceu em 1996 criado por Spencer Kimball, Peter Mattis e Josh MacDonald para o GIMP — frustrados com as licencas do Motif. Hoje, na versao GTK4, e o toolkit padrao do ambiente GNOME e de centenas de aplicacoes Linux. Construido sobre GObject — um sistema de objetos em C puro com heranca, signals e gerenciamento de memoria — GTK e tecnicamente orientado a objetos mesmo em C.

1. Hello World com GTK4

c — janela GTK4 minima
#include <gtk/gtk.h>

static void activate(GtkApplication *app, gpointer user_data)
{
    GtkWidget *window = gtk_application_window_new(app);
    gtk_window_set_title(GTK_WINDOW(window), "GTK4 Hello");
    gtk_window_set_default_size(GTK_WINDOW(window), 400, 300);

    GtkWidget *label = gtk_label_new("Ola, GTK4!");
    gtk_window_set_child(GTK_WINDOW(window), label);

    gtk_widget_set_visible(window, TRUE);
}

int main(int argc, char **argv)
{
    GtkApplication *app = gtk_application_new(
        "org.exemplo.MeuApp",
        G_APPLICATION_DEFAULT_FLAGS
    );
    g_signal_connect(app, "activate", G_CALLBACK(activate), NULL);
    int status = g_application_run(G_APPLICATION(app), argc, argv);
    g_object_unref(app);
    return status;
}
shell — compilar com pkg-config
gcc `pkg-config --cflags --libs gtk4` -o meu_app main.c

2. GObject e Sistema de Signals

GObject e o sistema de tipos em C que sustenta o GTK. Todo widget e um GObject; o sistema de signals do GObject e equivalente ao Observer pattern: emita um signal e todos os handlers conectados sao chamados.

c — conectar signal de botao
static void on_button_clicked(GtkButton *btn, gpointer user_data)
{
    GtkLabel *label = GTK_LABEL(user_data);
    gtk_label_set_text(label, "Clicado!");
}

/* dentro de activate(): */
GtkWidget *box   = gtk_box_new(GTK_ORIENTATION_VERTICAL, 10);
GtkWidget *btn   = gtk_button_new_with_label("Clique aqui");
GtkWidget *label = gtk_label_new("Aguardando...");

/* conecta o signal "clicked" ao callback */
g_signal_connect(btn, "clicked", G_CALLBACK(on_button_clicked), label);

gtk_box_append(GTK_BOX(box), label);
gtk_box_append(GTK_BOX(box), btn);
gtk_window_set_child(GTK_WINDOW(window), box);

3. Containers e Layouts

GtkBox

Container linear: GTK_ORIENTATION_HORIZONTAL ou GTK_ORIENTATION_VERTICAL. Use gtk_box_append() para adicionar filhos. Propriedade spacing controla o espaco entre filhos.

GtkGrid

Layout tabular com linhas e colunas variaveis. Suporta widgets com width e height em celulas (colspan/rowspan). Mais flexivel que GtkBox para formularios.

GtkScrolledWindow

Adiciona barras de rolagem a qualquer widget filho. Usado para conteudo maior que a janela: listas, editores de texto (GtkTextView), imagens grandes.

GtkHeaderBar

Barra de titulo customizavel do GNOME com botoes de acao integrados. Substituiu GtkToolbar no GTK4 para apps GNOME modernas.

4. GtkBuilder e Glade

GtkBuilder carrega interfaces definidas em arquivos XML (.ui). Glade e o editor visual que gera esses arquivos — equivalente ao Qt Designer para GTK. Isso separa UI de logica, facilita a colaboracao de designers e programadores no mesmo projeto.

c — carregar UI de arquivo XML
GtkBuilder *builder = gtk_builder_new_from_file("janela.ui");

/* obter widgets pelo id definido no XML */
GtkWidget *window = GTK_WIDGET(gtk_builder_get_object(builder, "main_window"));
GtkWidget *btn    = GTK_WIDGET(gtk_builder_get_object(builder, "btn_salvar"));

g_signal_connect(btn, "clicked", G_CALLBACK(on_salvar), NULL);
gtk_window_present(GTK_WINDOW(window));

g_object_unref(builder);

5. Quando Usar GTK em C

Ideal Para

  • Aplicacoes nativas Linux/GNOME com maxima integracao ao sistema
  • Apps que precisam de desempenho C puro com UI rica
  • Ferramentas que serao distribuidas como pacotes nativos no sistema
  • Projetos que ja usam GLib (daemons, servicos) e precisam adicionar GUI
  • Aprender o funcionamento interno de toolkits GUI em baixo nivel

Evitar Quando

  • App multiplataforma (Windows/macOS) — GTK no Windows e complicado de distribuir
  • Prototipagem rapida — C/GTK tem muito boilerplate; use Python+GTK ou C++/Qt
  • Equipe sem experiencia em C e gerenciamento de memoria
  • App mobile — GTK nao tem suporte mobile oficial

6. Ecossistema GTK

GLib / GIO

Bibliotecas base do GTK: tipos fundamentais (GList, GHashTable), operacoes de arquivo async (GFile), threads (GThread), mainloop (GMainLoop) e muito mais.

Glade

Editor visual de interfaces GTK que gera arquivos .ui (XML). Permite desenhar janelas sem escrever codigo de layout — semelhante ao Qt Designer.

Cairo / Pango

Cairo: renderizacao 2D vetorial (linhas, curvas, texto, imagens) usada internamente pelo GTK. Pango: layout e renderizacao de texto Unicode com suporte a RTL.

GTK Inspector

Debugger integrado (ativado com Ctrl+Shift+D): inspeciona a arvore de widgets em tempo real, mede FPS e exibe propriedades CSS — vital para depuracao de UI.
