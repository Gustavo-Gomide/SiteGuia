Flutter — Um Codigo, Seis Plataformas

Android, iOS, Web, Windows, macOS e Linux a partir do mesmo codigo Dart — com 60fps garantidos.

Flutter foi lancado pelo Google em 2018. A proposta e radical: nao usa nenhum widget do sistema operacional. Em vez disso, o motor Impeller (antes Skia) renderiza cada pixel da UI via Metal (iOS/macOS), Vulkan (Android/Windows) ou OpenGL ES. Isso garante aparencia perfeitamente consistente entre plataformas e eliminacoes de inconsistencias de estilo do SO. O resultado e que o mesmo app, compilado em Dart, gera binarios nativos para iOS, Android, Windows, macOS, Linux e aplicativos web.

1. Tudo e um Widget

A filosofia central do Flutter: tudo e um Widget. Layout, padding, animacao, temas, inputs, gestos — cada conceito e representado por um widget imutavel. A arvore de widgets descreve o que renderizar; o Flutter reconstroi eficientemente apenas o que mudou via mecanismo de reconciliacao (similar ao Virtual DOM do React).

StatelessWidget

Widget sem estado mutavel. Recebe dados via construtor e sempre produz a mesma UI para os mesmos dados. Exemplos: Text, Icon, layouts puros. E imutavel e performatico.

StatefulWidget

Widget com estado mutavel encapsulado em um objeto State. Chame setState() para notificar o Flutter de que a UI precisa ser reconstruida. Exemplos: formularios, contadores, sliders.

InheritedWidget / Provider

Mecanismo de heranca de dados pela arvore de widgets. Provider (pacote pub.dev) e a abstracao mais usada: injeta um objeto em qualquer descendente sem passar parametros manualmente.

2. Hello World e Layout

dart — app counter padrao do Flutter
import 'package:flutter/material.dart';

void main() => runApp(const MeuApp());

class MeuApp extends StatelessWidget {
  const MeuApp({super.key});

  @override
  Widget build(BuildContext context) => MaterialApp(
    title: 'Flutter Demo',
    theme: ThemeData(colorSchemeSeed: Colors.blue, useMaterial3: true),
    home: const PaginaInicial(),
  );
}

class PaginaInicial extends StatefulWidget {
  const PaginaInicial({super.key});
  @override
  State<PaginaInicial> createState() => _PaginaInicialState();
}

class _PaginaInicialState extends State<PaginaInicial> {
  int _contador = 0;

  @override
  Widget build(BuildContext context) => Scaffold(
    appBar: AppBar(title: const Text('Contador')),
    body: Center(
      child: Column(
        mainAxisAlignment: MainAxisAlignment.center,
        children: [
          Text('$_contador', style: Theme.of(context).textTheme.displayMedium),
          const SizedBox(height: 16),
          FilledButton.icon(
            onPressed: () => setState(() => _contador++),
            icon: const Icon(Icons.add),
            label: const Text('Incrementar'),
          ),
        ],
      ),
    ),
  );
}

3. Gerenciamento de Estado

setState (local)

Estado restrito a um StatefulWidget. Suficiente para widgets simples. Nao compartilha dados entre telas diferentes — use apenas para estado local e efemero.

Provider

A solucao recomendada pelo Google para estado compartilhado. ChangeNotifier + Consumer: qualquer widget na arvore abaixo do Provider se reconstroi quando o estado muda.

Riverpod

Evolucao do Provider, mais segura e testavel. Providers sao globais e imutaveis; os StateNotifierProviders e AsyncNotifierProviders encapsulam estado de forma previsivel.

BLoC / Cubit

Business Logic Component: streams de eventos e estados. Favorece separacao rigorosa de camadas. Padrao popular em apps Flutter corporativos e de escala.

5. Quando Usar Flutter

Ideal Para

  • Apps mobile que precisam estar no Android e iOS com uma base de codigo unica
  • Startups e MVPs mobile: produtividade alta com hot reload instantaneo
  • Apps com visual customizado e marca forte (Flutter tem controle total do pixel)
  • Apps multiplataforma real: mobile + desktop + web no mesmo projeto
  • Equipes que preferem linguagem tipada moderna (Dart) a JavaScript

Evitar Quando

  • App precisa de widgets 100% nativos iOS/Android — React Native tem acesso mais direto
  • Equipe ja tem base em JS/TS e nao quer aprender Dart
  • App principalmente web — Flutter web tem SEO e acessibilidade limitados
  • Biblioteca nativa muito especifica que nao tem plugin Flutter existente

6. Ecossistema

pub.dev

Repositorio oficial de pacotes Dart/Flutter. Mais de 50.000 pacotes. flutter_svg, go_router, dio (HTTP), freezed (imutabilidade), fl_chart (graficos).

go_router

Pacote de rotas declarativas para Flutter. URL-based routing que funciona em mobile e web com deep links, parametros e redirecionamentos condicionais (guards).

Flame Engine

Game engine 2D sobre Flutter. Componentes, colisoes, audio, parallax e ciclo de jogo integrado. Jogos como Magma Drift foram feitos com Flame.

FlutterFlow

No-code/low-code builder que gera codigo Flutter real. Util para prototipagem visual rapida e iteracao com clientes antes de partir para implementacao manual.
