Fyne — GUI Simples para Go

O toolkit Go mais popular: API simples, cross-platform, renderizacao OpenGL e zero CGo.

Fyne e o toolkit GUI mais popular para a linguagem Go. Lançado em 2018, usa OpenGL (via OpenGL ES 2) para renderizar widgets proprios — o que garante aparencia consistente em Windows, macOS, Linux e Raspberry Pi. O diferencial: nenhuma dependencia CGo obrigatoria para a maioria das funcionalidades; a API e intencional e limpa — adequada a filosofia Go de simplicidade.

1. Hello World

go — app fyne basico
package main

import (
    "fyne.io/fyne/v2/app"
    "fyne.io/fyne/v2/container"
    "fyne.io/fyne/v2/widget"
)

func main() {
    a := app.New()
    w := a.NewWindow("Fyne Hello")

    entrada := widget.NewEntry()
    entrada.SetPlaceHolder("Digite seu nome...")

    resultado := widget.NewLabel("")

    botao := widget.NewButton("Saudar", func() {
        if entrada.Text != "" {
            resultado.SetText("Ola, " + entrada.Text + "!")
        } else {
            resultado.SetText("Digite um nome.")
        }
    })

    w.SetContent(container.NewVBox(
        widget.NewLabel("Nome:"),
        entrada,
        botao,
        resultado,
    ))

    w.ShowAndRun()
}

2. Layouts e Containers

container.NewVBox / HBox

Empilhamento vertical ou horizontal de widgets. O mais simples e usado para a maioria dos layouts lineares em Fyne.

container.NewBorder

Widget central cresce para preencher o espaco disponivel; os outros ficam nas bordas (top, bottom, left, right). Ideal para janelas com conteudo central variavel.

container.NewGrid

Grid de N colunas iguais. Todos os widgets filhos recebem o mesmo tamanho. Util para paineis de botoes ou galeria de items.

container.NewScroll

Adiciona rolagem a qualquer widget filho. Necessario para listas longas, logs de texto ou formularios maiores que a janela.

3. Data Binding

go — binding reativo entre dados e widgets
import "fyne.io/fyne/v2/data/binding"

// cria uma string observavel
nome := binding.NewString()
nome.Set("Mundo")

// label atualiza automaticamente quando nome muda
rotulo := widget.NewLabelWithData(nome)

// entrada bidirecional: digitar muda nome; nome muda entrada
campo := widget.NewEntryWithData(nome)

// binding derivado via transformacao (string -> string)
saudacao := binding.NewSprintf("Ola, %s!", nome)
rotuloSaudacao := widget.NewLabelWithData(saudacao)

4. Quando Usar Fyne

Ideal Para

  • Ferramentas CLI Go que precisam de uma GUI simples sem mudar de linguagem
  • Apps utilitarios desktop multiplataforma em Go
  • Projetos Go onde a equipe nao quer aprender outra linguagem para a UI
  • Binary Go unico: Fyne compila estaticamente tudo no executavel

Evitar Quando

  • UI muito complexa com widgets avancados — ecossistema menor que Qt
  • App mobile — suporte Android/iOS experimental no Fyne
  • Performance grafica critica — a renderizacao OpenGL propria tem limitacoes
