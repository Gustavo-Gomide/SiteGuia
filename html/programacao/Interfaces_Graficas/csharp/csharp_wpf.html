WPF — Windows Presentation Foundation

XAML + MVVM + DirectX: o modelo de UI declarativa mais poderoso do ecossistema Windows.

WPF foi lancado em 2006 com o .NET Framework 3.0 como substituto moderno do WinForms. Renderiza toda a UI via DirectX (nao usa controles Win32 nativos), o que permite transforms, animacoes, efeitos de shader e templates de controles totalmente customizaveis. O XAML (Extensible Application Markup Language) descreve a UI de forma declarativa e separa rigorosamente apresentacao de logica de negocios via padrao MVVM — o mais poderoso mecanismo de binding de dados do mundo .NET.

1. XAML — UI Declarativa

xaml — janela com layout e binding
<Window x:Class="MeuApp.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        Title="WPF App" Height="350" Width="500">

  <Window.DataContext>
    <local:MainViewModel />
  </Window.DataContext>

  <StackPanel Margin="20">
    <TextBox Text="{Binding Nome, UpdateSourceTrigger=PropertyChanged}" />
    <Button  Content="Saudar"
              Command="{Binding SaudarCommand}"
              Margin="0,10" />
    <TextBlock Text="{Binding Mensagem}"
               FontSize="16" Foreground="SteelBlue" />
  </StackPanel>

</Window>

2. MVVM com INotifyPropertyChanged

csharp — ViewModel com INotifyPropertyChanged
using System.ComponentModel;
using System.Runtime.CompilerServices;
using System.Windows.Input;

public class MainViewModel : INotifyPropertyChanged
{
    private string _nome = "";
    private string _mensagem = "";

    public string Nome {
        get => _nome;
        set { _nome = value; OnPropertyChanged(); }
    }

    public string Mensagem {
        get => _mensagem;
        set { _mensagem = value; OnPropertyChanged(); }
    }

    public ICommand SaudarCommand => new RelayCommand(() =>
        Mensagem = string.IsNullOrWhiteSpace(Nome)
            ? "Digite um nome!"
            : $"Ola, {Nome}!"
    );

    public event PropertyChangedEventHandler? PropertyChanged;
    protected void OnPropertyChanged([CallerMemberName] string? prop = null)
        => PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(prop));
}

3. Recursos WPF Avancados

ControlTemplate

Redefine completamente a aparencia visual de qualquer controle (Button, ComboBox, Slider). Nao ha limite de customizacao: voce re-desenha tudo em XAML com shapes, gradientes e efeitos mantendo o comportamento original.

DataTemplate

Define como um objeto de dados e exibido visualmente. Usado em ItemsControl, ListBox, ComboBox para transformar POCOs em layouts ricos com imagens, badges e multiplos textos.

Storyboard e Animacoes

Animacoes declarativas no XAML: muda propriedades (largura, opacidade, cor) ao longo do tempo com easing functions. Triggers visuais disparam animacoes em eventos como MouseEnter, IsPressed, IsEnabled.

Resource Dictionary

Centraliza estilos, cores e templates num arquivo XAML reutilizavel. Funciona como CSS para WPF: defina um Style global para Button e todos os botoes do app herdam automaticamente.

4. Quando Usar WPF

Ideal Para

  • Apps Windows desktop com visual rico, animacoes e temas customizados
  • Arquiteturas MVVM com separacao rigorosa — testabilidade maxima
  • Dashboards de dados com ItemsControl, DataGrid e virtualizacao de widgets
  • Apps com branding corporativo (temas completos via ResourceDictionary)
  • Equipes .NET que ja conhecem C# e querem aprender UI declarativa antes do MAUI

Evitar Quando

  • Precisar de multiplataforma — WPF e Windows-only; use .NET MAUI
  • App mobile — WPF nao roda em telefones
  • Prototipagem ultra-rapida — WinForms ou Blazor desktop sao mais ageis
