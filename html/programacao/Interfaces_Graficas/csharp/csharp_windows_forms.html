Windows Forms — O RAD Classico .NET

Drag-and-drop no designer, double-click para criar evento — produtividade imediata.

Windows Forms (WinForms) foi lancado em 2001 com o .NET Framework 1.0 e e a forma mais direta de criar aplicativos desktop Windows com C#. O designer visual do Visual Studio permite arrastar controles para um formulario, configurar propriedades no painel e gerar o codigo de evento com um duplo clique. Desde o .NET 5+, WinForms e open-source e roda no .NET moderno — mas continua sendo exclusivo para Windows.

1. Estrutura de um Form

csharp — form basico com controles
using System;
using System.Windows.Forms;

public class FormPrincipal : Form
{
    private TextBox _campoNome;
    private Button  _btnSaudar;
    private Label   _lblResultado;

    public FormPrincipal()
    {
        Text = "Windows Forms";
        Size = new System.Drawing.Size(400, 250);

        _campoNome   = new TextBox   { Left = 20, Top = 30, Width = 200 };
        _btnSaudar   = new Button    { Left = 20, Top = 70, Text = "Saudar" };
        _lblResultado = new Label    { Left = 20, Top = 120, Width = 300 };

        _btnSaudar.Click += BtnSaudar_Click;

        Controls.AddRange(new Control[] {
            _campoNome, _btnSaudar, _lblResultado
        });
    }

    private void BtnSaudar_Click(object sender, EventArgs e)
    {
        var nome = _campoNome.Text.Trim();
        _lblResultado.Text = string.IsNullOrEmpty(nome)
            ? "Digite seu nome!"
            : $"Ola, {nome}!";
    }

    [STAThread]
    static void Main()
    {
        Application.EnableVisualStyles();
        Application.Run(new FormPrincipal());
    }
}

2. Controles Principais

DataGridView

Tabela interativa com edicao inline, ordenacao e filtragem. Pode ser ligada a BindingSource conectada a um DataTable ou lista de objetos via DataBinding.

TreeView e ListView

TreeView exibe hierarquias (explorador de arquivos). ListView exibe colecoes em modo icones, lista ou detalhes com colunas — como o Windows Explorer.

TabControl e Panel

TabControl cria interface com abas. Panel e GroupBox agrupam controles. FlowLayoutPanel e TableLayoutPanel fornecem layout responsivo sem posicao fixa.

MenuStrip e ToolStrip

Barra de menu nativa Windows com itens, submenus, atalhos e icones. ToolStrip cria barras de ferramentas com botoes icone e textos — o padrao Windows desktop.

3. Data Binding

csharp — vincular lista de objetos ao DataGridView
var produtos = new List<Produto> {
    new("Teclado", 150m),
    new("Mouse",   80m)
};

// BindingSource torna a lista observavel
var binding = new BindingSource { DataSource = produtos };
dataGridView1.DataSource = binding;

// Adicionar novo produto e atualizar a grid automaticamente
binding.Add(new Produto("Monitor", 1200m));

// Bind simples: campo de texto sempre igual a propriedade
txtNome.DataBindings.Add("Text", binding, "Nome");

4. Quando Usar Windows Forms

Ideal Para

  • Apps Windows internos e ERPs onde desenvolvimento rapido importa mais que visual
  • Manutencao de codebase WinForms legado — nao ha motivo para migrar
  • Ferramentas de administracao, paineis de controle e utilitarios desktop
  • Equipes .NET com pouco tempo disponivel para aprender WPF/XAML

Evitar Quando

  • App publico com visual moderno e animacoes — use WPF ou MAUI
  • Precisar de multiplataforma (macOS/Linux) — WinForms e Windows-only
  • Arquitetura MVVM ou separacao rigorosa UI/logica — WPF e mais adequado
