TornadoFX — JavaFX com DSL Kotlin Nativo

Builders declarativos, injecao de dependencias, MVC integrado e zero boilerplate de JavaFX.

TornadoFX e um framework leve que envolve JavaFX com uma DSL Kotlin de alta expressividade. Em vez de construir nodes JavaFX manualmente, o desenvolvedor usa builders aninhados que espelham a hierarquia da UI. O sistema de ViewModel e Injectable oferece DI simples sem Spring. A DSL usa extension functions e lambdas com receivers para tornar o codigo de UI mais idiomatico que Java puro. Nota: TornadoFX 2.x esta em desenvolvimento para suporte ao Kotlin mais recente; cheque a estabilidade da versao antes de usar em producao.

1. DSL Declarativa

kotlin — view com builders tornadofx
import tornadofx.*

class MinhaView : View("Meu App TornadoFX") {
    override val root = vbox(spacing = 16) {
        paddingAll = 24

        label("Usuario:")
        val campoNome = textfield {
            promptText = "Digite seu nome..."
        }

        button("Confirmar") {
            action {
                information(
                    header  = "Ola!",
                    content = "Bem-vindo, ${campoNome.text}"
                )
            }
        }
    }
}
kotlin — app entry point
class MinhaApp : App(MinhaView::class)

fun main() = launch<MinhaApp>()

2. ViewModel e Bindings

kotlin — viewmodel com validacao e commit
data class Pessoa(val nome: String, val email: String)

class PessoaModel(val pessoa: Pessoa) : ItemViewModel<Pessoa>(pessoa) {
    val nome  = bind(Pessoa::nome)
    val email = bind(Pessoa::email)
}

class EditarPessoaView : View() {
    val model: PessoaModel by inject()

    override val root = form {
        fieldset("Editar Pessoa") {
            field("Nome")  { textfield(model.nome) }
            field("Email") {
                textfield(model.email).validator {
                    if (it?.contains("@") == false)
                        error("Email invalido")
                    else null
                }
            }
        }
        button("Salvar") {
            enableWhen(model.valid)  // desabilitado se invalido
            action { model.commit() }
        }
    }
}

3. Quando Usar TornadoFX

Ideal Para

  • Projetos Kotlin desktop que querem MVC estruturado com DI sem Spring/Koin
  • Formularios complexos com validacao automatica e bindings bidirecionais
  • Equipes que preferem DSL Kotlin sobre FXML XML para a UI

Evitar Quando

  • Meta-parametro importante: TornadoFX 2.x ainda e pre-release em 2025
  • Prefere Compose Multiplatform para compartilhar UI com Android e Web
  • Time nao tem familiaridade com Kotlin — curva e maior que JavaFX direto
