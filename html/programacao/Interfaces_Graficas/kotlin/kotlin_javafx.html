JavaFX com Kotlin — GUI Moderna e Segura de Tipos

O poder do JavaFX + a expressividade de Kotlin: lambdas, null safety e coroutines para UI desktop.

JavaFX funciona perfeitamente bem em Kotlin, ja que ambos rodam na JVM. A interoperabilidade e total: todas as APIs JavaFX (FXML, Properties, ObservableLists, CSS, Task) ficam disponíveis com a sintaxe mais concisa e segura do Kotlin. Alguns destaques: lambdas substituem o EventHandler verboso; a null safety do Kotlin protege contra NPE em @FXML injetados; e coroutines substituem Task para background work de forma mais natural. Para projetos novos de desktop na JVM, Kotlin + JavaFX ou TornadoFX sao as melhores opcoes.

1. App JavaFX em Kotlin

kotlin — app javafx com lambdas e null safety
import javafx.application.Application
import javafx.scene.Scene
import javafx.scene.control.Button
import javafx.scene.control.Label
import javafx.scene.layout.VBox
import javafx.stage.Stage

class MinhaApp : Application() {
    override fun start(stage: Stage) {
        val label = Label("Ola, Kotlin + JavaFX!")
        val btn = Button("Clique")

        // Lambda em vez de EventHandler verboso do Java
        btn.setOnAction { label.text = "Clicado!" }

        val root = VBox(16.0, label, btn).apply {
            setPrefSize(300.0, 150.0)
            style = "-fx-padding: 24; -fx-alignment: center;"
        }

        stage.scene = Scene(root)
        stage.title = "App Kotlin"
        stage.show()
    }
}

fun main() = Application.launch(MinhaApp::class.java)

2. Coroutines para Background Work

kotlin — coroutine em vez de Task/SwingWorker
import javafx.application.Platform
import kotlinx.coroutines.*

// Dispatcher que executa na EDT (JavaFX Application Thread)
val FxDispatcher = object : CoroutineDispatcher() {
    override fun dispatch(context: CoroutineContext, block: Runnable) {
        if (Platform.isFxApplicationThread()) block.run()
        else Platform.runLater(block)
    }
}

// Uso: background pesado no IO dispatcher, update da UI no FxDispatcher
btn.setOnAction {
    CoroutineScope(FxDispatcher).launch {
        label.text = "Carregando..."
        val resultado = withContext(Dispatchers.IO) {
            fazerRequisicaoLenta()   // nao trava a UI
        }
        label.text = resultado  // de volta na FX thread
    }
}

3. Quando Usar Kotlin + JavaFX Direto

Ideal Para

  • Projetos Kotlin que precisam de UI desktop simples sem adicionar outra dependencia de DSL
  • Times que ja conhecem JavaFX em Java e estao migrando para Kotlin
  • Compatibilidade maxima: usa a API JavaFX oficial sem abstracao adicional

Evitar Quando

  • Quer uma DSL Kotlin mais expressiva para UI — use TornadoFX
  • Precisa de Compose Multiplatform para compartilhar UI entre Android e desktop
