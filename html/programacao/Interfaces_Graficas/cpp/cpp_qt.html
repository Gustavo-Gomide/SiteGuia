Qt — O Framework C++ para Tudo

KDE, VirtualBox, BMW iDrive, Autodesk, Maya — todos rodam em Qt. O padrao do mercado.

Qt foi criado em 1992 pelos noruegueses Haavard Nord e Eirik Chambe-Eng da Trolltech. Hoje mantido pela Qt Company, o framework e muito mais do que uma biblioteca GUI: inclui rede, banco de dados, XML, JSON, threads, graficos OpenGL, animacoes, localizacao e ate um motor JavaScript (QJSEngine). O Meta-Object Compiler (MOC) pre-processa as macros Q_OBJECT e gera o codigo de signals/slots em tempo de compilacao — sem overhead de runtime via RTTI.

1. Signals e Slots em C++

cpp — classe com signal e slot proprios
#include <QObject>

class Contador : public QObject {
    Q_OBJECT  // habilita MOC para esta classe

public:
    Contador() : m_valor(0) {}
    int valor() const { return m_valor; }

public slots:
    void incrementar() {
        ++m_valor;
        emit valorMudou(m_valor);
    }

signals:
    void valorMudou(int novoValor);  // declaracao apenas

private:
    int m_valor;
};

// Conectar signal a slot (lambda C++11 suportado)
Contador c;
QObject::connect(&c, &Contador::valorMudou, [](int v) {
    qDebug() << "Novo valor:" << v;
});
c.incrementar();  // imprime "Novo valor: 1"

2. QWidget e Layouts

cpp — janela com layout e widgets
#include <QApplication>
#include <QMainWindow>
#include <QVBoxLayout>
#include <QPushButton>
#include <QLabel>
#include <QLineEdit>

class MainWindow : public QMainWindow {
    Q_OBJECT
public:
    MainWindow() {
        setWindowTitle("App Qt");
        resize(480, 320);

        QWidget *central = new QWidget(this);
        setCentralWidget(central);

        QVBoxLayout *layout = new QVBoxLayout(central);
        m_label = new QLabel("Resultado:");
        m_edit  = new QLineEdit();
        QPushButton *btn = new QPushButton("Calcular");

        layout->addWidget(m_label);
        layout->addWidget(m_edit);
        layout->addWidget(btn);

        connect(btn, &QPushButton::clicked, this, &MainWindow::calcular);
    }

private slots:
    void calcular() {
        m_label->setText("Input: " + m_edit->text());
    }

private:
    QLabel    *m_label;
    QLineEdit *m_edit;
};

3. Qt Widget vs Qt Quick (QML)

Qt Widgets

API C++ tradicional com QWidget como base. Usa o estilo do SO (Fusion, Windows, macOS). Melhor para apps desktop com muitas janelas, menus e formularios. Suporte extenso de acessibilidade e localizacao.

Qt Quick / QML

Linguagem declarativa JSON-like para UIs fluidas e animadas. O renderer corre sobre OpenGL/Vulkan/Metal — ideal para apps mobile, embedded (HMI automotivo) e qualquer UI com animacoes complexas. Estado gerenciado via property binding.

Quando Misturar

QQuickWidget embute uma cena QML dentro de uma janela QWidget. Util para adicionar elementos animados a apps legados sem reescrever toda a UI em QML.

4. Model/View em C++

cpp — tabela com QAbstractTableModel
class ProdutoModel : public QAbstractTableModel {
    Q_OBJECT
public:
    int rowCount(const QModelIndex &) const override {
        return m_dados.size();
    }
    int columnCount(const QModelIndex &) const override { return 3; }

    QVariant data(const QModelIndex &index, int role) const override {
        if (role != Qt::DisplayRole) return {};
        const auto &p = m_dados[index.row()];
        switch (index.column()) {
            case 0: return p.nome;
            case 1: return p.preco;
            case 2: return p.estoque;
        }
        return {};
    }
private:
    QVector<Produto> m_dados;
};

5. Quando Usar Qt em C++

Ideal Para

  • Apps profissionais multiplataforma (Windows, macOS, Linux, mobile, embedded)
  • Performance maxima com UI rica: CAD, editores de video, IDEs
  • Sistemas embedded e automotivos (HMI) onde Qt e padrao de industria
  • Apps cientificos com graficos via QCustomPlot ou QChart
  • Projetos com equipe ja experiente em C++ que precisa de GUI completa

Evitar Quando

  • Prototipagem rapida — PyQt/PySide ou C# WPF sao mais agieis
  • App web — use frameworks web especializados
  • Bundle muito pequeno necessario — Qt adiciona ~30-50MB ao deploy
  • Projeto GPL exige evitar licenca comercial Qt (use PySide6 LGPL)

6. Build e Deploy

CMake (Recomendado)

Qt6 usa CMake como sistema de build oficial. Macros como qt_add_executable() e qt_add_qml_module() simplificam a configuracao.

windeployqt / macdeployqt

Ferramentas oficiais que copiam todas as DLLs/libs Qt necessarias para a pasta do executavel, tornando-o distribuivel sem Qt instalado no sistema alvo.

Qt Installer Framework

Cria instaladores profissionais para Windows, macOS e Linux com wizard, licenca, componentes opcionais e atualizacoes automaticas integradas.

Qt for Android/iOS

Qt6 suporta build para Android via NDK e iOS via Xcode. A mesma base de codigo C++ roda no desktop e no mobile com ajustes de layout para telas pequenas.
