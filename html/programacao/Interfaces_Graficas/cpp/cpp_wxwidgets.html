C++ wxWidgets

wxWidgets — Nativo em Todas as Plataformas

O unico toolkit C++ que usa os controles reais do sistema operacional — sem emular nada.

wxWidgets nasceu em 1992 como wxWindows criado por Julian Smart na Universidade de Edinburgh. A proposta central e usar os widgets nativos de cada plataforma: no Windows, usa Win32/MFC; no macOS, usa Cocoa; no Linux, usa GTK3/GTK4. O resultado e que apps wxWidgets tem aparencia 100% identica a apps nativos do SO — algo que Qt (que desenha os proprios widgets) nao oferece por padrao. FileZilla, wxMaxima e Code::Blocks sao exemplos bem conhecidos.

1. Hello World

cpp — app wxWidgets minimo
#include <wx/wx.h>

class MeuFrame : public wxFrame {
public:
    MeuFrame() : wxFrame(nullptr, wxID_ANY, "wxWidgets Hello",
                         wxDefaultPosition, wxSize(400, 300)) {
        wxPanel *painel = new wxPanel(this);
        wxBoxSizer *sizer = new wxBoxSizer(wxVERTICAL);

        wxStaticText *lbl = new wxStaticText(painel, wxID_ANY, "Ola, wxWidgets!");
        wxButton     *btn = new wxButton(painel, wxID_ANY, "Clique");

        sizer->Add(lbl, 0, wxALL | wxALIGN_CENTER, 20);
        sizer->Add(btn, 0, wxALL | wxALIGN_CENTER, 10);
        painel->SetSizer(sizer);

        btn->Bind(wxEVT_BUTTON, [this](wxCommandEvent &) {
            wxMessageBox("Ola!", "Info", wxOK | wxICON_INFORMATION);
        });
    }
};

class MeuApp : public wxApp {
public:
    bool OnInit() override {
        MeuFrame *frame = new MeuFrame();
        frame->Show();
        return true;
    }
};

wxIMPLEMENT_APP(MeuApp);

2. Sizers — Gerenciamento de Layout

wxWidgets usa Sizers para layout responsivo — como as caixas de layout do Qt. O container nao tem posicao fixa: o Sizer calcula o tamanho dos filhos e redistribui quando a janela e redimensionada.

wxBoxSizer

Linear horizontal (wxHORIZONTAL) ou vertical (wxVERTICAL). O proportion na chamada Add() define quanto espaco extra o widget ocupa.

wxGridSizer / wxFlexGridSizer

Layout tabular. wxFlexGridSizer permite linhas e colunas de tamanhos diferentes e expandiveis. Ideal para formularios com labels e campos alinhados.

wxStaticBoxSizer

Cria um grupo visual com borda e titulo (como um fieldset HTML). O Sizer interno organiza os widgets dentro dessa area agrupada.

3. Tabela de Eventos

wxWidgets usa tabelas de eventos (macros) ou o metodo moderno Bind() com lambdas C++11. A abordagem Bind() e mais segura e flexivel do que as macros de event table.

cpp — bind de eventos
// Bind com lambda (moderno, C++11)
btn->Bind(wxEVT_BUTTON,   [this](wxCommandEvent &e) { onClique(); });
edit->Bind(wxEVT_TEXT,     [this](wxCommandEvent &e) { onTexto(e.GetString()); });
frame->Bind(wxEVT_CLOSE_WINDOW, [this](wxCloseEvent &e) {
    if (confirmarSaida()) e.Skip(); // Skip propaga para fechar mesmo
});

menu->Bind(wxEVT_MENU, &MeuFrame::onSalvar, this, ID_SALVAR);

// Forma classica (macros) ainda funciona
wxBEGIN_EVENT_TABLE(MeuFrame, wxFrame)
    EVT_BUTTON(ID_BTN, MeuFrame::onClique)
    EVT_MENU(ID_SALVAR, MeuFrame::onSalvar)
wxEND_EVENT_TABLE()

4. Quando Usar wxWidgets

Ideal Para

  • Apps onde aparencia 100% nativa e mandatoria (auditoria, corporativo, utilities)
  • Projetos open-source sem restricao de licenca Qt (wxWidgets e LGPL-like)
  • Ferramentas tecnicas como FileZilla, Code::Blocks, Audacity
  • Equipes que valorizam widgets nativos sobre visual customizado

Evitar Quando

  • UI altamente customizada (cores, animacoes) — Qt/QML e mais apropriado
  • Mobile — wxWidgets nao tem suporte oficial Android/iOS
  • Ecossistema menor: menos modulos extras que Qt (sem rede, banco de dados, etc.)
  • Documentacao e comunidade menores comparadas ao Qt
