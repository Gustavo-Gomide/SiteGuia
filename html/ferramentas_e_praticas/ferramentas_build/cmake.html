CMake — Build System Multiplataforma

CMake

O padrão de facto para projetos C/C++ — gera Makefiles, projetos Visual Studio, Ninja e outros a partir de uma única configuração

CMake não compila código diretamente. É um meta-build system: lê o arquivo CMakeLists.txt e gera os arquivos de build nativos do ambiente alvo (Makefile no Linux, .vcxproj no Windows, etc.). Isso permite que um mesmo projeto seja construído em qualquer plataforma sem modificação.

Por que usar CMake?

Vantagens

  • Portabilidade — mesmo projeto compila no Linux, macOS e Windows
  • Integração com IDEs — VSCode, CLion, Visual Studio leem CMake nativamente
  • Gerenciamento de dependências — find_package() localiza bibliotecas; FetchContent baixa automaticamente
  • Builds out-of-source — artefatos em pasta separada, repositório limpo
  • Modern CMake (v3+) — modelo baseado em targets e propriedades de visibilidade

Contexto Histórico

O CMake foi criado em 1999 pelo Kitware para o projeto ITK, financiado pelo NIH. A versão 3.0 (2014) introduziu o paradigma Modern CMake, baseado em targets com propriedades de visibilidade (PUBLIC, PRIVATE, INTERFACE).

CMake é hoje o sistema de build mais usado em projetos C++ open-source. É o padrão do ecossistema CLion, VSCode C++ Extension e vcpkg.

Workflow CMake

Fluxo de uso — Terminal
# 1. Criar diretório de build (out-of-source)
mkdir build && cd build

# 2. Configurar — gera o sistema de build nativo
cmake .. -DCMAKE_BUILD_TYPE=Release

# 3. Compilar
cmake --build . --parallel 4

# 4. Instalar (opcional)
cmake --install . --prefix /usr/local

# Com Ninja (mais rápido que Make)
cmake .. -G Ninja -DCMAKE_BUILD_TYPE=Debug
ninja

CMakeLists.txt — Estrutura Completa

CMakeLists.txt 
cmake_minimum_required(VERSION 3.21)
project(MeuProjeto VERSION 1.0.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# Biblioteca estática
add_library(minha_lib STATIC src/utils.cpp src/parser.cpp)
target_include_directories(minha_lib
    PUBLIC  include/
    PRIVATE src/internal/
)
target_compile_options(minha_lib PRIVATE -Wall -Wextra)

# Executável
add_executable(programa main.cpp)
target_link_libraries(programa PRIVATE minha_lib)

# Dependência externa
find_package(OpenSSL REQUIRED)
target_link_libraries(programa PRIVATE OpenSSL::SSL)

# FetchContent — baixa automaticamente
include(FetchContent)
FetchContent_Declare(
    googletest
    GIT_REPOSITORY https://github.com/google/googletest.git
    GIT_TAG v1.14.0
)
FetchContent_MakeAvailable(googletest)

# Testes
enable_testing()
add_executable(testes tests/test_utils.cpp)
target_link_libraries(testes PRIVATE minha_lib GTest::gtest_main)
add_test(NAME testes COMMAND testes)

# Instalação
install(TARGETS programa DESTINATION bin)
install(DIRECTORY include/ DESTINATION include)

Modern CMake — Targets e Visibilidade

O paradigma do Modern CMake (v3.0+) substitui variáveis globais por propriedades de targets com visibilidade explícita:

Visibilidade de Propriedades

PRIVATE

Aplica somente a este target. Não vaza para quem linkar.

PUBLIC

Aplica a este target E é propagado para quem linkar com ele.

INTERFACE

Aplica somente a quem linkar — não ao próprio. Usado em header-only libs.

Variáveis CMake Importantes

VariávelDescriçãoValores Comuns
