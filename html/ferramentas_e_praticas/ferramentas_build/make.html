Make e Makefile

O utilitário de automação de build mais longevo da computação — criado em 1976 e presente em todo sistema Unix-like até hoje

Make é uma ferramenta que automatiza a compilação e construção de programas determinando quais partes de um projeto precisam ser recompiladas e executando os comandos necessários. Com base em um arquivo de regras chamado Makefile, o Make constrói apenas o que mudou — economizando tempo em projetos grandes.

História e Contexto

O Make foi criado por Stuart Feldman em 1976, enquanto trabalhava no Bell Labs. Originalmente escrito em uma tarde para resolver um problema concreto: garantir que apenas os arquivos modificados fossem recompilados, sem depender de scripts que recompilam tudo sempre.

Feldman ganhou o ACM Software System Award de 2003 pelo Make. Em entrevista, ele afirmou que a "regra dos 8 espaços" (indentação de receitas com TAB) foi um erro que não pôde ser corrigido por compatibilidade retroativa — um dos bugs mais famosos da história do software.

Conceitos Fundamentais

Estrutura de uma Regra Make

alvo : dependências
receita: comandos
Alvo (target)

Arquivo a ser criado ou nome lógico de ação (ex: all, clean).

Dependências (prerequisites)

Arquivos dos quais o alvo depende. O Make reconstrói o alvo se alguma dependência for mais nova.

Receita (recipe)

Comandos shell executados para construir o alvo. DEVEM ser indentados com TAB, não espaços.

Makefile — exemplo comentado 
# Variáveis
CC      = gcc
CFLAGS  = -Wall -Wextra -O2
TARGET  = programa
SRCS    = main.c utils.c parser.c
OBJS    = $(SRCS:.c=.o)

# Alvo padrão (primeiro listado) — constrói tudo
all: $(TARGET)

# Linkagem: depende de todos os .o
$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) -o $@ $^

# Compilação genérica: .c → .o (regra de padrão)
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

# Alvos phony (não criam arquivos)
.PHONY: all clean install test

# Remove os artefatos de build
clean:
	rm -f $(OBJS) $(TARGET)

# Instala o binário em /usr/local/bin
install: $(TARGET)
	install -m 755 $(TARGET) /usr/local/bin/

Variáveis Automáticas

O Make possui variáveis automáticas que são populadas automaticamente em cada receita. São essenciais para escrever regras genéricas e não-repetitivas:

VariávelSignificadoExemplo
$@O alvo da regra atualgcc -o $@ → gcc -o programa
$<A primeira dependênciagcc -c $< → gcc -c main.c
$^Todas as dependências (sem duplicatas)Usado na linkagem
$?Dependências mais novas que o alvoPara builds incrementais
$*O stem da regra de padrão%.o: %.c → $* = main
$(@D)Diretório do alvomkdir -p $(@D)
$(@F)Arquivo (sem diretório) do alvoÚtil para gerar nomes

Técnicas Avançadas

Funções de String

Makefile — funções
# wildcard — encontra arquivos
SRCS = $(wildcard src/*.c)

# patsubst — substitui padrões
OBJS = $(patsubst src/%.c, build/%.o, $(SRCS))

# subst — substituição simples
NEW = $(subst .c,.o,$(SRCS))

# notdir — remove o caminho
NAMES = $(notdir $(SRCS))

# addprefix — adiciona prefixo
FULL = $(addprefix obj/, $(NAMES))

Make Além de C/C++

Embora originalmente criado para compilação de C, o Make é um orquestrador de comandos genérico. É amplamente usado em:

Projetos Python

test:
	pytest tests/ -v
lint:
	ruff check .
format:
	black .
all: lint test

Documentação e LaTeX

PDF = relatorio.pdf
$(PDF): relatorio.tex

	pdflatex $<  # 2x para referências

Boas Práticas

  • Sempre declare .PHONY para alvos que não geram arquivos (all, clean, test). Sem isso, se existir um arquivo chamado clean, o Make pensará que já está atualizado.
  • Use variáveis para compilador e flags — facilita trocar de gcc para clang em uma linha.
  • Prefixe receitas com @ para suprimir o eco do comando: @echo "Compilando..."
  • Prefixe com - para ignorar erros: -rm *.o (não falha se nenhum .o existir).
  • Use make -j$(nproc) para paralelizar o build em múltiplos cores.
  • Estruture Makefiles grandes com include — um arquivo por módulo.
