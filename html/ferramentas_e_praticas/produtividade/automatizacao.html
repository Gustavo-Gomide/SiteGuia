Automação de Tarefas

Codifique seus processos repetitivos uma vez e deixe a máquina executar — builds, testes, deploys, formatação e qualidade automatizados

Um desenvolvedor produtivo é aquele que automatiza tarefas recorrentes. Cada processo manual é um candidato à automação: executar testes antes de commitar, formatar código ao salvar, fazer deploy após merge, enviar notificações quando uma tarefa termina. Esses ganhos, individualmente pequenos, são enormes em escala de meses e anos.

Shell Scripts — A Base da Automação

setup.sh — script de configuração de ambiente 
#!/usr/bin/env bash
set -euo pipefail                     # sair ao primeiro erro; var não definida = erro
IFS=$'\n\t'

LOG() { echo "[$(date '+%H:%M:%S')] $*"; }

LOG "Configurando ambiente de desenvolvimento..."

# Verificar pré-requisitos
command -v python3 &>/dev/null || { echo "Python 3 não encontrado"; exit 1; }
command -v node   &>/dev/null || { echo "Node.js não encontrado";  exit 1; }

# Criar ambiente virtual Python
if [ ! -d ".venv" ]; then
    LOG "Criando venv..."
    python3 -m venv .venv
fi

source .venv/bin/activate

# Instalar dependências
LOG "Instalando dependências Python..."
pip install -q -r requirements.txt

LOG "Instalando dependências Node.js..."
npm ci --silent

LOG "Configurando git hooks..."
npx husky install

LOG "Pronto. Ambiente configurado com sucesso."

Taskfile — Make Moderno

Taskfile (ou task) é uma alternativa moderna ao Make: sintaxe YAML legive, multi-plataforma (inclui Windows) e sem as peculiaridades do Makefile.

Taskfile.yml 
version: '3'

vars:
  APP: meu-app

tasks:
  default:
    desc: "Mostra as tarefas disponíveis"
    cmds: [task --list]

  build:
    desc: "Compila o projeto"
    cmds:
      - go build -o bin/{{.APP}} ./cmd/{{.APP}}

  test:
    desc: "Executa todos os testes"
    cmds:
      - go test ./... -v -race -coverprofile=coverage.out

  lint:
    desc: "Lint + format"
    cmds:
      - golangci-lint run
      - gofmt -w .

  docker:build:
    desc: "Build da imagem Docker"
    cmds:
      - docker build -t {{.APP}}:latest .

  dev:
    desc: "Inicia servidor de desenvolvimento"
    deps: [build]
    cmds:
      - air                          # live reload

Execute com task build, task test, etc. Lista de tarefas com task --list.

Git Hooks — Automação no Fluxo de VCS

Hooks mais úteis

HookQuando executaUso comum
pre-commitAntes do commitLinting, formatação, testes unitários
commit-msgApós digitar a mensagemValidar formato do commit (Conventional Commits)
pre-pushAntes de fazer pushTestes de integração, segurança
post-mergeApós merge/pullReinstalar dependências

pre-commit — a ferramenta

pre-commit é um gerenciador de hooks Git. Define os hooks em .pre-commit-config.yaml e eles são compartilhados com a equipe:

repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.1.6
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

Cron — Tarefas Agendadas

crontab — sintaxe e exemplos
# Formato: minuto hora dia-do-mes mes dia-da-semana comando
# min (0-59) | hora (0-23) | dom (1-31) | mês (1-12) | dow (0-7, dom=0 ou 7)

# Backup diário às 2h da manhã
0 2 * * * /usr/local/bin/backup.sh

# Limpar logs toda segunda-feira às 3h30
30 3 * * 1 find /var/log/app -name "*.log" -mtime +30 -delete

# Sincronizar dados a cada 15 minutos
*/15 * * * * /scripts/sync.sh >> /var/log/sync.log 2>&1

# Abrir o editor de crontab
crontab -e

# Ver crontabs atuais
crontab -l
