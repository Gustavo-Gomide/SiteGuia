Cargo — Build System e Gerenciador de Pacotes Rust

Cargo

A ferramenta oficial do Rust que unifica build, teste, documentação, publicação e gerenciamento de dependências em uma única interface

Cargo é amplamente considerado o melhor gerenciador de pacotes entre as linguagens modernas. Diferente de npm ou pip, o Cargo é o único sistema de build do Rust — não há Makefile separado. Builds são determinísticos, incrementais e multiplataforma por padrão.

Cargo.toml — Manifesto do Pacote

Cargo.toml — anotado
[package] name = "meu-projeto" version = "0.1.0" edition = "2021" # edição Rust (2015, 2018, 2021) authors = ["Dev <dev@exemplo.com>"] license = "MIT" # Dependências de produção [dependencies] serde = { version = "1.0", features = ["derive"] } tokio = { version = "1", features = ["full"] } reqwest = { version = "0.11", features = ["json"] } # Dependências somente de desenvolvimento [dev-dependencies] criterion = "0.5" # benchmarks mockall = "0.12" # mocks # Features opcionais [features] default = ["tls"] tls = ["reqwest/native-tls"] async = ["tokio"] # Otimizações por perfil [profile.release] opt-level = 3 lto = true codegen-units = 1

Comandos Essenciais

Cargo — referência rápida
# Criar projeto
cargo new meu-projeto        # binário
cargo new minha-lib --lib    # biblioteca

# Compilar
cargo build                  # debug (rápido)
cargo build --release        # otimizado (para produção)

# Executar
cargo run
cargo run -- arg1 arg2       # passando argumentos

# Testes
cargo test
cargo test nome_do_teste      # filtra por nome

# Documentação (gera e abre no browser)
cargo doc --open

# Ferramentas de qualidade
cargo fmt                    # formata o código (rustfmt)
cargo clippy                 # linter com sugestões avançadas
cargo fix                    # corrige avisos automaticamente

# Dependências
cargo add serde --features derive
cargo remove serde
cargo update                 # atualiza Cargo.lock

# Benchmarks
cargo bench

Workspaces — Monorepos em Rust

Cargo Workspaces permitem gerenciar múltiplos crates (pacotes) num único repositório, compartilhando o Cargo.lock e o diretório target/.

# Cargo.toml raiz do workspace
[workspace]
members = [
  "app",
  "library",
  "shared"
]
resolver = "2"

crates.io é o registro público de crates Rust — equivalente ao PyPI ou npm registry. Todos os pacotes são open-source e imutáveis após publicação.
