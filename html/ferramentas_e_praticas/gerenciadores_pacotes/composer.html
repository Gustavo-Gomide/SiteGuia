Composer — Gerenciador de Dependências PHP

Composer

O gerenciador de dependências padrão do PHP moderno — resolverá dependências, gerará o autoloader PSR-4 e integrará com o Packagist

Composer foi criado em 2011 por Nils Adermann e Jordi Boggiano, inspirado no npm e Bundler. Ele não instala pacotes globalmente — cada projeto tem seu próprio conjunto de dependências em vendor/. Isso resolveu o grande problema histórico do PHP: a ausência de um sistema de pacotes padrão.

composer.json — Manifesto do Projeto

composer.json — anotado
{ "name": "empresa/meu-projeto", "description": "Descrição do projeto", "type": "project", "license": "MIT", "require": { "php": "^8.2", "laravel/framework": "^10.0", "guzzlehttp/guzzle": "^7.8" }, "require-dev": { "phpunit/phpunit": "^10.5", "fakerphp/faker": "^1.23" }, "autoload": { "psr-4": { "App\\": "src/" } }, "autoload-dev": { "psr-4": { "Tests\\": "tests/" } }, "scripts": { "test": "phpunit", "lint": "phpcs src/", "format": "phpcbf src/" }, "config": { "optimize-autoloader": true, "preferred-install": "dist" } }

Comandos Essenciais

Comandos Composer
# Inicializar projeto interativamente
composer init

# Instalar dependências do composer.json
composer install

# Adicionar dependência
composer require guzzlehttp/guzzle
composer require --dev phpunit/phpunit  # somente dev

# Atualizar dependências
composer update
composer update guzzlehttp/guzzle    # só um pacote

# Remover dependência
composer remove guzzlehttp/guzzle

# Regenerar autoloader (após adicionar classes)
composer dump-autoload
composer dump-autoload --optimize    # otimizado para produção

# Executar scripts definidos no composer.json
composer run-script test

# Verificar problemas
composer diagnose
composer validate

Autoloading PSR-4

Como o Autoloader funciona

Após composer install, inclua uma única linha no seu script de entrada:

<?php
require_once __DIR__ . '/vendor/autoload.php';

// A partir daqui, qualquer classe em src/ é carregada automaticamente
use App\Services\UserService;

$service = new UserService();

O Composer mapeia App\ para src/ conforme o composer.json. A classe App\Services\UserService será carregada de src/Services/UserService.php.
