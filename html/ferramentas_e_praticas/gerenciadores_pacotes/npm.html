npm — Node Package Manager

npm

O maior registro de pacotes open-source do mundo e a ferramenta padrão de gerenciamento de dependências do ecossistema JavaScript/Node.js

npm (Node Package Manager) é simultaneamente: (1) um registro público com mais de 2 milhões de pacotes em npmjs.com; (2) uma ferramenta de linha de comando para instalar, publicar e gerenciar pacotes; (3) um executor de scripts que automatiza tarefas do projeto.

package.json — O Manifesto do Projeto

package.json — anotado 
{
  "name": "meu-projeto",
  "version": "1.0.0",
  "description": "Descrição do projeto",
  "main": "src/index.js",          // entry point para require()
  "type": "module",                 // ES Modules (import/export)

  // Scripts — executados com 'npm run <nome>'
  "scripts": {
    "start":   "node src/index.js",
    "dev":     "nodemon src/index.js",
    "build":   "tsc",
    "test":    "jest",
    "lint":    "eslint src/",
    "format":  "prettier --write ."
  },

  // Dependências de produção
  "dependencies": {
    "express": "^4.18.2",
    "dotenv":  "^16.3.1"
  },

  // Dependências somente de desenvolvimento
  "devDependencies": {
    "typescript": "^5.3.0",
    "jest":       "^29.7.0",
    "nodemon":    "^3.0.0"
  },

  // Engines: versões mínimas recomendadas
  "engines": {
    "node": ">=18.0.0",
    "npm":  ">=9.0.0"
  }
}

Versionamento Semântico (SemVer)

npm usa Semantic Versioning para especificar intervalos de versão aceitos. Compreender SemVer é fundamental para evitar quebras inadvertidas.

Operadores de Versão

^4.18.2

Compatible: aceita >=4.18.2 <5.0.0. Atualiza patches e minors.

~4.18.2

Tilde: aceita >=4.18.2 <4.19.0. Apenas patches.

4.18.2

Pinado: versão exata. Reprodutível mas sem correções automáticas.

>=4.0.0 <5.0.0

Range explícito. Máxima clareza sobre o intervalo aceito.

Comandos Essenciais

Comandos npm 
# Inicializar projeto
npm init
npm init -y                     # aceita todos os defaults

# Instalar dependências
npm install                     # instala tudo do package.json
npm install express             # adiciona em dependencies
npm install jest --save-dev     # adiciona em devDependencies
npm install express@4.18.2     # versão específica
npm ci                         # install limpo para CI (usa package-lock)

# Remover
npm uninstall express

# Atualizar
npm update
npm outdated                    # lista pacotes desatualizados

# Executar scripts
npm run dev
npm test                        # atalho para 'npm run test'
npm start                       # atalho para 'npm run start'

# npx: executar binário sem instalar
npx create-react-app meu-app
npx prettier --write .

package-lock.json e Reprodutibilidade

Por que o lockfile importa

O package-lock.json registra a versão exata de cada pacote e suas dependências transitivas. Isso garante que npm ci em qualquer máquina, em qualquer data, instale exatamente o mesmo grafo de dependências.

Sempre commit o package-lock.json no repositório. Nunca o ignore no .gitignore.
