Python para Data Science

Visao Geral

1. Por que Python?

  • Sintaxe simples e rápida para prototipar.
  • Bibliotecas maduras (NumPy, pandas, scikit-learn, PyTorch, etc.).
  • Integração com notebooks, APIs e pipelines.

2. Stack típica

  • NumPy: arrays e álgebra.
  • pandas: tabelas (DataFrame), joins, groupby.
  • Matplotlib/Seaborn: visualização.
  • scikit-learn: ML clássico e pipelines.
  • PyTorch/TensorFlow: deep learning.

3. Boas práticas

  • Use ambientes isolados (venv/conda) e fixe versões.
  • Padronize formatação e lint (black/ruff) em projetos reais.
  • Separe exploração (notebooks) de código de produção (módulos).

Referencias

  • VANDERPLAS, Jake. Python Data Science Handbook. O'Reilly, 2016.
  • MCKINNEY, Wes. Python for Data Analysis. O'Reilly, 2022.
  • GERON, Aurelien. Hands-On Machine Learning with Scikit-Learn, Keras, and TensorFlow. O'Reilly, 2022.
