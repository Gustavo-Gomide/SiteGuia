Limpeza de Dados

Transforme dados brutos em dados confiáveis.

1. Por que limpar?

Modelos e análises são sensíveis a dados errados. Alguns impactos comuns:

  • quebra de pipelines (tipos inválidos, campos vazios);
  • métricas distorcidas (duplicatas, inconsistência);
  • modelos com baixa generalização (outliers e ruído).

2. Problemas mais comuns

  • Valores faltantes (NaN/null): decidir remover, imputar ou investigar a causa.
  • Duplicatas: registros repetidos em coletas/joins.
  • Tipos e formatos: datas como texto, moeda com separadores, campos mistos.
  • Outliers: erro de digitação vs evento real (fraude, anomalia).
  • Inconsistência: “sp”, “S. Paulo”, “SP” representando o mesmo valor.

3. Estratégias de tratamento

  • Remoção (dropping): útil quando o dano é pequeno e bem entendido.
  • Imputação: média/mediana/moda ou modelos (ex.: KNN) dependendo do caso.
  • Padronização: normalizar categorias, corrigir formatos e unidades.
  • Regras de validação: limites plausíveis (idade < 120 etc.).

4. Exemplo rápido (Pandas)

import pandas as pd

df = pd.read_csv("dados.csv")

# nulos por coluna
print(df.isnull().sum())

# preencher nulos (ex.: idade) com a mediana
df["idade"] = df["idade"].fillna(df["idade"].median())

# remover duplicatas
df = df.drop_duplicates()

# regra simples para outliers
df = df[df["idade"] < 120]
