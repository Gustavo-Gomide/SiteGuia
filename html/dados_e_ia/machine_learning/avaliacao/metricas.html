Métricas de Avaliação

Como medir se um modelo está realmente bom.

1. Por que metricas importam

Métrica é a forma de traduzir acertou/errou para um número. A escolha depende do problema: fraude, saúde e segurança raramente aceitam o mesmo tipo de erro.

  • Classes desbalanceadas: acurácia pode enganar.
  • Custo do erro: falso positivo vs falso negativo podem ter impactos diferentes.
  • Objetivo: priorizar pegar todos (recall) ou errar pouco quando alerta (precisão).

2. Matriz de confusão (base)

Para classificação binária, a matriz de confusão organiza as previsões em quatro casos: TP (acerto positivo), TN (acerto negativo), FP (alarme falso) e FN (perdeu um positivo). 

# sklearn
from sklearn.metrics import confusion_matrix

y_true = [0, 1, 1, 0]
y_pred = [0, 1, 0, 0]

confusion_matrix(y_true, y_pred)

3. Métricas principais

  • Acurácia: fração total de acertos (boa quando classes são equilibradas).
  • Precisão: entre os positivos previstos, quantos eram positivos de fato.
  • Recall: entre os positivos reais, quantos o modelo encontrou.
  • F1: média harmônica entre precisão e recall.
from sklearn.metrics import classification_report

print(classification_report(y_true, y_pred))
