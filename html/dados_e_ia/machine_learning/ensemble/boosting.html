Boosting

Ensemble sequencial para reduzir vies e erro.

1. Ideia central

Em boosting, os modelos sao treinados em sequencia. Cada novo modelo tenta corrigir os erros do conjunto anterior. O resultado final e uma combinacao ponderada de varios modelos fracos.

  • Bagging: paralelo, reduz variancia.
  • Boosting: sequencial, tende a reduzir vies (mas pode overfitar se exagerar).

2. Familias comuns

  • AdaBoost: repondera exemplos errados (classico em classificacao).
  • Gradient Boosting: otimiza uma funcao de perda adicionando arvores que aproximam o gradiente.
  • Variantes modernas: ver XGBoost/LightGBM/CatBoost (principalmente dados tabulares).

3. Hiperparametros que mais mudam tudo

  • n_estimators: numero de estagios.
  • learning_rate: tamanho do passo (menor costuma exigir mais estagios).
  • max_depth / max_leaf_nodes: complexidade de cada arvore (controla overfitting).
  • subsample: amostragem por estagio (ajuda generalizacao).

Combine com validacao cruzada e, quando possivel, early stopping.

4. Exemplo (scikit-learn)

from sklearn.ensemble import GradientBoostingClassifier

model = GradientBoostingClassifier(
    n_estimators=300,
    learning_rate=0.05,
    max_depth=3,
    random_state=42,
)

model.fit(X_train, y_train)
print(model.score(X_test, y_test))
