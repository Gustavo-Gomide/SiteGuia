K-Means

Clustering por centroides com iteracao simples.

1. Ideia central

O K-Means tenta particionar os dados em $K$ grupos, cada um representado por um centroide. O algoritmo alterna entre:

  • Atribuicao: cada ponto vai para o centroide mais proximo.
  • Atualizacao: cada centroide vira a media dos pontos do seu grupo.

Funciona melhor quando clusters sao aproximadamente esfericos e de tamanho parecido.

2. Como escolher $K$

  • Elbow: rode para varios $K$ e procure o ponto de "joelho" na inercia (SSE).
  • Silhouette: busca separacao/coesao (quanto maior, melhor em geral).
  • Dominio: o numero de grupos pode vir de contexto (ex.: segmentos de clientes).

3. Cuidados

  • Escala: padronize/normalize features antes (distancia domina).
  • Inicializacao: prefira k-means++ e rode multiplas inicializacoes.
  • Outliers: pontos extremos podem puxar centroides.

4. Exemplo (scikit-learn)

from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

X_scaled = StandardScaler().fit_transform(X)

model = KMeans(
    n_clusters=3,
    init="k-means++",
    n_init=10,
    random_state=42,
)

labels = model.fit_predict(X_scaled)
print(model.cluster_centers_)
print(labels[:10])
