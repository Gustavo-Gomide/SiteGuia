Data Lake

Armazenamento central (geralmente barato) para dados brutos e processados.

1. Conceito

Um data lake guarda dados em formatos variados (estruturados, semi e não estruturados), frequentemente em object storage. A ideia é separar armazenamento de computação.

2. Camadas e organização

  • Raw/Bronze: dados como chegam (auditável).
  • Silver: dados limpos e conformados.
  • Gold: datasets prontos para consumo (BI/ML).

3. Formatos e tabelas

  • Preferência por formatos colunares (Parquet) para analytics.
  • Tabelas em lake (Delta/Iceberg/Hudi) ajudam com ACID, schema evolution e time travel.

4. Governança

  • Catálogo, lineage, qualidade e controle de acesso são essenciais.
  • Sem governança, o lake vira “data swamp”.

Referencias

  • DIXON, James. Data Lake and the Rise of Poly-Structured Data, 2010.
  • KLEPPMANN, Martin. Designing Data-Intensive Applications. O'Reilly, 2017.
  • PROVOST, Foster; FAWCETT, Tom. Data Science for Business. O'Reilly, 2013.

Um Data Lake sem catalogo, qualidade e controle de acesso tende a virar data swamp.

Governanca e qualidade definem se dados geram valor real.
