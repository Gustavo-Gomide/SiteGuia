Hadoop

Ecossistema clássico para armazenamento e processamento distribuído.

1. Componentes principais

  • HDFS: sistema de arquivos distribuído.
  • YARN: gerenciamento de recursos e agendamento.
  • MapReduce: modelo de processamento batch (mais antigo).

2. Onde ele aparece hoje

Em muitos contextos modernos, Hadoop foi parcialmente substituído por cloud storage + engines como Spark/Trino. Ainda é relevante para entender a evolução e alguns ambientes on-premise.

3. Cuidados

  • Operação/cluster on-premise exige observabilidade e administração.
  • Latência: MapReduce não é ideal para interativo.
  • Governança e segurança são essenciais em dados sensíveis.
