Backpropagation

Como a rede aprende com os seus próprios erros.

1. A Culpa é de Quem?

Imagine que a rede neural chutou que uma foto de cachorro era um "gato". O Backpropagation é o gerente irritado que entra na sala e pergunta: "Quem foi o responsável por esse erro?".

Ele percorre a rede de trás para frente, calculando a contribuição (gradiente) de cada neurônio para o erro final e ajustando os pesos para que o erro diminua na próxima vez.

2. O Fluxo do Erro (Backward Pass)

O erro (pulso rosa) sai do final e viaja para o início, atualizando as conexões.

Os pesos piscam quando são atualizados pelo gradiente.

3. A Regra da Cadeia (Chain Rule)

Como calculamos a "culpa" de um neurônio lá no início da rede? Usando cálculo derivativo composto.

∂Erro / ∂Peso = (Como o Erro muda com a Saída) × (Como a Saída muda com a Ativação) × (Como a Ativação muda com o Peso)

Basicamente, multiplicamos as derivadas locais de trás para frente até chegar no peso que queremos ajustar.

4. A Magia no PyTorch

Frameworks modernos escondem a matemática difícil.

import torch

# 1. Forward Pass (Previsão)
predicao = modelo(dados_entrada)
perda = calcular_erro(predicao, real)

# 2. Backward Pass (A mágica acontece aqui)
# O PyTorch calcula todos os gradientes automaticamente
perda.backward()

# 3. Atualizar Pesos (Otimizador)
# Dá um passinho na direção oposta ao erro
otimizador.step()
otimizador.zero_grad()
