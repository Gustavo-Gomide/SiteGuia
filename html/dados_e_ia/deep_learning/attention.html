Attention

O mecanismo que permite ao modelo “focar” em partes relevantes da entrada.

Attention

Visao Geral

1. Intuição

Em vez de processar uma sequência e “esquecer” o começo, attention calcula pesos que indicam quais tokens são mais importantes para produzir uma representação/saída naquele momento.

2. Query, Key e Value (Q/K/V)

  • Query (Q): o que estou procurando.
  • Key (K): o que cada token “oferece”.
  • Value (V): a informação que será combinada.

A similaridade $Q\cdot K$ define pesos que combinam os $V$.

3. Self-attention e multi-head

  • Self-attention: tokens “olham” para outros tokens da mesma sequência.
  • Multi-head: várias atenções em paralelo capturam relações diferentes.
  • Positional encoding: injeta noção de ordem (já que attention puro é “permutation-invariant”).

4. Custo computacional

A atenção completa costuma ter custo $O(n^2)$ em relação ao tamanho da sequência $n$. Por isso existem variações (sparse/linear attention) para sequências longas.
