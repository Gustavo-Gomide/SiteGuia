Redes Convolucionais (CNN)

Os olhos da Inteligência Artificial.

1. Por que não usar redes comuns?

Se tentarmos usar uma Rede Neural densa (MLP) para reconhecer imagens, temos dois problemas:

  • Explosão de Pesos: Uma imagem 1000x1000 tem 1 milhão de pixels. Conectar isso a uma camada oculta gera bilhões de parâmetros.
  • Perda Espacial: O MLP "achata" a imagem numa lista. Ele esquece que o pixel do olho está vizinho ao pixel do nariz.

A Solução: Escanear a imagem em pedaços, procurando padrões locais.

[Image of CNN architecture diagram]

2. A Operação de Convolução

Imagine uma "janelinha" (Kernel) deslizando sobre a imagem. Ela multiplica os valores e soma, criando um novo mapa de características.

O filtro (kernel) $3 \times 3$ desliza e gera um pixel de saída.

3. Pooling (O Resumo)

Depois de detectar características, precisamos reduzir o tamanho da imagem para diminuir o processamento e evitar overfitting.

Max Pooling

Pega o maior valor da região. "Existe um olho aqui? SIM (máximo)."

Flatten

No final, transforma as matrizes em uma linguiça (vetor) para classificar.

4. Estrutura no Keras

Uma CNN típica segue o padrão: Conv -> Pool -> Conv -> Pool -> Dense.

from tensorflow.keras import layers, models

model = models.Sequential([
# 1. Extração de Características (Visão)
# 32 filtros, tamanho 3x3
layers.Conv2D(32, (3, 3), activation='relu', input_shape=(28, 28, 1)),
layers.MaxPooling2D((2, 2)),

layers.Conv2D(64, (3, 3), activation='relu'),
layers.MaxPooling2D((2, 2)),

# 2. Classificação (Decisão)
layers.Flatten(),
layers.Dense(64, activation='relu'),
layers.Dense(10, activation='softmax')
])
