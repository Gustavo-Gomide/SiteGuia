Pipeline

Execução sobreposta de instruções: estágios, hazards e mitigações

O pipeline é a técnica mais importante para aumentar o throughput de instruções sem elevar a frequência de clock. Enquanto a instrução N está na etapa EX, a instrução N+1 está em ID e N+2 em IF, ao mesmo tempo — como uma linha de montagem.

Estágios Clássicos (5 estágios MIPS)

EstágioSiglaO que faz
Instruction FetchIFLê instrução da memória (cache L1i) usando PC; incrementa PC
Instruction DecodeIDDecodifica opcode; lê registradores fonte; gera sinais de controle
ExecuteEXALU calcula resultado ou endereço efetivo; testa branch
Memory AccessMEMPara LOAD/STORE: acessa cache de dados L1d; outros passam direto
Write BackWBEscreve resultado no registrador destino

Hazards e Mitigações

TipoCausaMitigação
Hazard de Dados (RAW)Instrução usa resultado de instrução anterior ainda não completaForwarding/bypassing; stall (bolha no pipeline)
Hazard de ControleBranch: próxima instrução desconhecida até EXBranch prediction; execução especulativa; delay slots (MIPS legado)
Hazard EstruturalDois estágios querem o mesmo recurso físicoDuplicar unidades (ex: L1i e L1d separados eliminam esse hazard para IF e MEM)

CPUs modernas têm pipelines muito mais profundos (14–20 estágios no Intel Skylake; ~31 no NetBurst/Prescott) e usam execução out-of-order para esconder latências de hazards.
