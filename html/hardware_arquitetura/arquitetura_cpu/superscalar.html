Execução Superescalar

Múltiplas instruções por ciclo, out-of-order execution e especulação

Uma CPU superescalar possui várias unidades de execução (ALUs, FPUs, unidades de carga/armazenamento) e pode despachar múltiplas instruções por ciclo de clock (IPC > 1). Combinado com execução out-of-order (OoO), executa instruções na ordem mais eficiente — não necessariamente a ordem do programa — respeitando dependências verdadeiras.

Componentes da Execução Out-of-Order

ComponenteFunção
FrontendFetch + decode + renomeação de registradores (elimina hazards WAR/WAW)
Renomeação (RAT)Mapeia registradores arquiteturais (ex: RAX) para registradores físicos internos (ex: P134)
Reorder Buffer (ROB)Mantém instruções em ordem programática para commit em ordem mesmo executadas fora de ordem
Reservation Station (RS)Fila de instruções aguardando operandos; despacha para unidades de execução quando pronta
Unidades de ExecuçãoALU inteira, FPU, unidade SIMD/AVX, unidade de branch, load/store units (múltiplas de cada)
Commit/RetireInstruções são retiradas em ordem do ROB; efeitos colaterais se tornam vistos

IPC — Instructions Per Cycle

O que define o IPC

  • Número de unidades de execução ("largura" do back-end)
  • Qualidade da predição de branches (miss = flush do pipeline)
  • Tamanho do ROB (janela de instruções visível para OoO)
  • Latência de memória (cache miss = bubbles)

IPCs reais (2024)

  • Intel Core Ultra 200 (Arrow Lake): ~5–6 IPC médio
  • AMD Ryzen 9000 (Zen 5): ~5–6 IPC médio
  • Apple M4: ~7–8 IPC (backend 16 wide)
  • ARM Cortex-X4: ~5–6 IPC
  • Nota: IPC varia muito por workload (inteiro vs. FP vs. memória)
