Processadores Multicore

Paralelismo de hardware: núcleos, threads, SMT, NUMA e lei de Amdahl

A partir de 2005, aumentar a frequência de clock deixou de ser viável por limites térmicos. A solução foi colocar múltiplos núcleos no mesmo chip, permitindo escalar desempenho com paralelismo. Hoje é normal ter CPUs consumer com 16–32 núcleos e servidores com 128+.

Lei de Amdahl

Speedup máximo com paralelismo
# Lei de Amdahl:
# S(n) = 1 / ( (1 - p) + p/n )
#   p = fração paralelizável do programa
#   n = número de processadores
#   S = speedup obtido

# Exemplo: p=0.9 (90% paralelizável), n=16 cores
S = 1 / ((1 - 0.9) + 0.9/16)
S = 1 / (0.1 + 0.05625)
S ≈ 6.4x   # não 16x!

# Limite teórico com n=infinito:
S_max = 1 / (1 - 0.9) = 10x

# Conclusão: a parte serial limita o speedup máximo

A lei de Amdahl explica por que paralelizar apenas partes do código tem retornos decrescentes. Minimizar a parte serial (I/O blocking, locks, algoritmos inerentemente seqüenciais) é essencial.
