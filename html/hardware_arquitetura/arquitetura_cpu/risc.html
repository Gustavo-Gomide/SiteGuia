RISC — Reduced Instruction Set Computer

Filosofia de ISA simples, pipeline eficiente e alto throughput

RISC é uma filosofia de design de ISA que prioriza instruções simples, de tamanho fixo, que executam em exatamente 1 ciclo de clock. Isso simplifica o pipeline, reduz a lógica de decodificação e permite frequências maiores. CISC (x86) tem instruções complexas e de tamanho variável, mas as CPUs modernas as traduzem para micro-op RISC internamente.

RISC vs. CISC

RISC

  • Instruções simples, tamanho fixo (4 bytes em ARM/RISC-V)
  • Load/Store: só LOAD e STORE acessam memória; ALU opera apenas em registradores
  • Muitos registradores de propósito geral (32 em ARM64/RISC-V)
  • Pipeline eficiente, decodificação trivial
  • Exemplos: ARM, RISC-V, MIPS, PowerPC, SPARC
  • Domina: mobile (Apple Silicon, Qualcomm Snapdragon), servidores, embarcados

CISC

  • Instruções complexas, tamanho variável (1–15 bytes em x86-64)
  • Instruções podem operar diretamente na memória
  • Menos registradores de propósito geral (16 em x86-64)
  • Decodificador complexo converte para micro-ops internos (RISC-like)
  • Exemplos: x86, x86-64 (AMD64)
  • Domina: desktops, laptops, servidores x86
