Hardware e Arquitetura

Arquitetura de Von Neumann

O modelo que define quase todo computador de propósito geral desde 1945

Proposta por John von Neumann em 1945, esta arquitetura estabelece que dados e instruções residem na mesma memória e são acessados pelo mesmo barramento. Essa decisão de design simplificou imensamente o hardware e abriu caminho para programas armazenados — mas também criou o "gargalo de Von Neumann" que a indústria trabalha para contornar até hoje.

Componentes do Modelo

Memória (unificada)

Armazena tanto o programa (instruções) quanto os dados sobre os quais ele opera. Endereçável por bytes ou words. O compartilhamento de barramento com dados é o núcleo do gargalo.

Unidade de Controle (UC)

Lê instruções da memória (fetch), decodifica-as (decode) e emite sinais para as demais unidades executarem a operação (execute). Mantém o Program Counter (PC).

Unidade Lógica e Aritmética (ALU)

Executa operações aritméticas (+, -, ×, ÷) e lógicas (AND, OR, XOR, NOT, SHL, SHR) sobre operandos lidos dos registradores.

Registradores

Memória ultra-rápida dentro da CPU: PC (próxima instrução), IR (instrução corrente), MAR (endereço de memória), MDR/MBR (dado a ler/escrever), registradores de propósito geral.

I/O

Periféricos (teclado, tela, disco) interagem com a CPU via endereços de I/O mapeados em memória (MMIO) ou via instruções específicas (PMIO, como in / out no x86).

Barramento Único

Dados e instruções compartilham o mesmo barramento memória–CPU. Isso causa o gargalo: CPU só pode buscar dado ou instrução num ciclo de clock, nunca os dois ao mesmo tempo.

Ciclo Fetch–Decode–Execute

Pseudocódigo do ciclo de instrução
loop:
    # FETCH
    MAR ← PC
    MDR ← Mem[MAR]      # lê instrução da memória
    IR  ← MDR
    PC  ← PC + N         # aponta para próxima instrucao

    # DECODE
    Op  ← IR[opcode]
    Operandos ← IR[campos]

    # EXECUTE
    switch Op:
        ADD:  ACC ← ACC + Operando
        LOAD: MAR ← Operando; ACC ← Mem[MAR]
        STORE: MAR ← Operando; Mem[MAR] ← ACC
        JUMP: PC ← Operando
        HALT: parar

Em CPUs modernas este ciclo é feito em pipeline — múltiplas instruções em etapas sobrepostas — resultando em throughput muito superior ao ciclo serial clássico.

O Gargalo de Von Neumann

O Problema

  • CPU e memória compartilham um único barramento
  • Velocidade da CPU cresce muito mais rápido que largura de banda da memória
  • CPU fica ociosa aguardando dados (stall) por dezenas a centenas de ciclos
  • Mesmo com DRAM DDR5, latência de acesso é ~70–100 ns vs. 0.3 ns do clock da CPU

Mitigações

  • Hierarquia de cáche (L1/L2/L3) que mantém dados próximos da CPU
  • Arquitetura Harvard modificada (barramentos separados de dados e instruções nos caches L1)
  • Prefetch de instruções e dados (hardware e software)
  • Multi-channel DDR (maior largura de banda agregada)
  • Processamento In-Memory (PIM) e HBM para IA
