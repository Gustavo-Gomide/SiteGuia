Hardware e Arquitetura

Arquitetura Harvard

Memórias e barramentos separados para dados e instruções

A arquitetura Harvard usa memórias físicas distintas para instruções e dados, cada uma com seu próprio barramento. Isso permite que a CPU busque a próxima instrução enquanto acessa um dado, eliminando o gargalo de baramento do modelo Von Neumann puro. Microcontroladores (PIC, AVR, ARM Cortex-M) usam Harvard; CPUs de desktop usam Harvard modificada (caches L1 separados).

Von Neumann vs. Harvard

Von Neumann

  • Memória unificada (dados + instruções)
  • Um único barramento compartilhado
  • Auto-modificar código é possível (e perigoso)
  • Simples de implementar e programar
  • Gargalo: não pode buscar dado e instrução simultaneamente
  • Exemplos: x86/x86-64, ARM AArch64 (nível de cache L2+)

Harvard

  • Memórias separadas: ROM/Flash para instruções, RAM para dados
  • Dois barramentos independentes
  • Fetch de instrução e acesso a dado simultâneos (sem stall extra)
  • Código autoimodificável é mais complexo ou impossível
  • Exemplos: AVR ATmega, PIC, DSPs dedicados

Harvard Modificada em CPUs Modernas

NívelTipoPor quê
Cache L1Harvard (L1i + L1d separados)Fetch e load/store simultâneos; L1i é somente leitura
Cache L2Von Neumann (unificado)Simplifica coerência de cache; menos área de silício
Cache L3Von Neumann (unificado, compartilhado)Compartilhado entre núcleos; facilita comunicação entre cores
RAM principalVon NeumannPrograma e dados no mesmo espaço de endereços

O resultado é que CPUs x86-64 e ARM AArch64 são tecnicamente Harvard no nível de cache L1, mas Von Neumann a partir de L2. Isso combina o melhor dos dois mundos.
