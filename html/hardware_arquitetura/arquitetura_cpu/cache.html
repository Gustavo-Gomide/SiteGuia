Cache de CPU

L1, L2, L3 — políticas de mapeamento, substituição e coerência

Cache é uma memória pequena e muito rápida situada fisicamente próxima (ou dentro) da CPU. Explora o princípio da localidade: se um dado foi acessado, ele ou seus vizinhos provavelmente serão acessados novamente em breve. Um hit de cache L1 custa ~4 ciclos; um miss até a RAM principal custa ~200–300 ciclos.

Políticas de Mapeamento e Substituição

Mapeamento

  • Direct Mapped: cada bloco de RAM mapeia para exatamente uma linha de cache (rápido, mas colisões frequentes)
  • Fully Associative: qualquer bloco pode ir para qualquer linha (sem colisão, mas lógica cara)
  • N-way Set Associative: compromisso — cada bloco vai para um conjunto de N linhas. CPUs modernas usam 8-way, 16-way

Substituição (quando cache está cheio)

  • LRU (Least Recently Used): remove a linha menos recentemente acessada — padrão e eficiente
  • FIFO: remove a linha mais antiga independente de uso recente
  • Random: simples, evita patologias de LRU em acessos ciclicos
  • PLRU (Pseudo-LRU): aproximação efíci de LRU com menos hardware

Escrevendo Código Cache-Friendly

C — row-major vs column-major (cache miss)
// RUIM: percorre coluna por coluna em matriz row-major
// Cada acesso pula uma linha de cache (64 bytes) → alto miss rate
for (int j = 0; j < N; j++)
    for (int i = 0; i < N; i++)
        sum += A[i][j];  // A[0][0], A[1][0], A[2][0]... pula N*4 bytes

// BOM: percorre linha por linha → localidade espacial
for (int i = 0; i < N; i++)
    for (int j = 0; j < N; j++)
        sum += A[i][j];  // A[0][0], A[0][1], A[0][2]... sequencial

O impacto pode ser 10x+ em matrizes grandes. Cache lines típicas: 64 bytes. Prefira estruturas de arrays (SoA) a arrays de estruturas (AoS) quando iterar sobre um campo.
