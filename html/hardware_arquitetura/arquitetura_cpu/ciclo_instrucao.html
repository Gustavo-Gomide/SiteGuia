Ciclo de Instrução

As etapas que a CPU executa para cada instrução: fetch, decode, execute, writeback

O ciclo de instrução é o processo fundamental repetido pela CPU bilhões de vezes por segundo. Cada instrução passa por etapas bem definidas que envolvem registradores específicos. Compreender esse ciclo é pré-requisito para entender pipeline, superescalaridade e anomalias de desempenho.

Etapas do Ciclo

EtapaRegistradores envolvidosO que acontece
1. Fetch (IF)PC, MAR, MDR, IRPC copiado para MAR; memória[MAR] copiada para MDR; MDR copiado para IR; PC incrementado
2. Decode (ID)IR, decodificadorUnidade de controle interpreta o opcode do IR; identifica tipo de instrução e operandos
3. Fetch de OperandosMAR, MDR, registradores geraisSe operando está em memória, calcula endereço efetivo e faz outra leitura de memória
4. Execute (EX)ALU, registradores gerais, flagsALU realiza a operação; resultado fica em acumulador ou registrador destino; flags atualizadas
5. Writeback (WB)MDR, MAR, registradores geraisPara instruções STORE: escreve resultado em memória[MAR]. Para REGISTER: resultado já está no registrador.

Exemplo: Instrução ADD R1, R2, R3

Ciclo de ADD R1, R2, R3 (R1 ← R2 + R3)
FETCH:
  MAR ← PC                  # ex: MAR = 0x1000
  MDR ← Mem[0x1000]          # MDR = 0x02120300 (encoding de ADD R1,R2,R3)
  IR  ← MDR                  # IR  = 0x02120300
  PC  ← PC + 4               # PC  = 0x1004

DECODE:
  opcode  ← IR[31:26] = ADD
  destino ← IR[25:21] = R1
  src1    ← IR[20:16] = R2
  src2    ← IR[15:11] = R3

EXECUTE:
  resultado ← R2 + R3         # ALU soma conteudo de R2 e R3
  flags: ZF, SF, CF, OF (overflow)

WRITEBACK:
  R1 ← resultado             # escreve no registrador destino
