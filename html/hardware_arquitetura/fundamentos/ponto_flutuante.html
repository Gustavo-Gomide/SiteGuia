Ponto Flutuante — IEEE 754

Como computadores representam números reais, precisão, NaN e armadilhas clássicas

Números reais são infinitos; bits são finitos. O padrão IEEE 754 define como aproximar números reais em 32 ou 64 bits de forma padronizada. Entender ponto flutuante é essencial para evitar bugs sutis em comparações, cálculos financeiros e cientifícos.

Estrutura de um float32
# float32 = (-1)^sinal × 1.mantíssa × 2^(expoente-127)

Bit 31: sinal     (0 = positivo, 1 = negativo)
Bits 30–23: expoente (8 bits, bias de 127)
Bits 22–0:  mantíssa  (23 bits, bit implícito "1." no início)

# Exemplo: 0.15625 como float32
0.15625 = 0.00101₂ = 1.01 × 2^⁻³
sinal=0, exp=-3+127=124=01111100, mant=01000000000000000000000
→ 0 01111100 01000000000000000000000

Armadilhas Comuns

Python — erros clássicos de ponto flutuante
# 0.1 não tem representação exata em binário
0.1 + 0.2 == 0.3   # False!
print(0.1 + 0.2)   # 0.30000000000000004

# Nunca compare floats com ==
import math
math.isclose(0.1 + 0.2, 0.3)      # True
abs(a - b) < 1e-9                  # tolerancia manual

# NaN não é igual nem a si mesmo
import math
x = float('nan')
x == x        # False
math.isnan(x) # True

# Infinito
float('inf') > 1e308   # True
float('inf') + 1       # inf

# Cancelamento catastrófico: perda de dígitos significativos
a = 1.000000001
b = 1.000000000
a - b   # ~1e-9, pode ter erro relativo alto

# Para dinheiro: nunca use float! Use decimal
from decimal import Decimal
Decimal('0.1') + Decimal('0.2') == Decimal('0.3')  # True
