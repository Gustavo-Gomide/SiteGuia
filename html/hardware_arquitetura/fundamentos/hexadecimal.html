Hexadecimal

Base 16: a notação compacta preferida por programadores e hardware

Hexadecimal (base 16) usa 16 dígitos: 0–9 e A–F. Cada dígito hex representa exatamente 4 bits, tornando a conversão binário↔hex trivial. É a notação padrão para endereços de memória, dumps de memória, cores CSS/HTML, valores de registradores e bytecode.

Usos Práticos em Programação

Exemplos de uso real
# Endereços de memória e registradores
0x7fff5fbff878   # endereço na pilha (x86-64)
0xCAFEBABE       # magic number de arquivos .class (Java)
0xDEADBEEF       # valor clássico de "memória não inicializada"

# Cores HTML/CSS
#FF5733  →  R=0xFF(255) G=0x57(87) B=0x33(51)
#00FF00  →  verde puro

# Máscaras bit a bit
0xFF000000 & ip  # extrair octeto mais significativo

# Permissões Unix
chmod 0755  →  octal 7=111 (rwx), 5=101 (r-x)

# Dump de memória (hexdump -C)
00000000  48 65 6c 6c 6f  |Hello|

# Literals em várias linguagens
0xFF     // C, C++, Java, C#, JS
0xFF     # Python, Ruby, Go
16#FF    % Erlang
$FF      ; Assembly 6502

Conversão Rápida Binário ↔ Hex

Método: grupos de 4 bits
# Binário → Hex: 4 bits = 1 dígito hex
1010 1111 0011 1100
   A    F    3    C
→ 0xAF3C

# Hex → Binário: expandir cada dígito em 4 bits
0x2B7E
   2    B    7    E
0010 1011 0111 1110

# Python: conversões fáceis
hex(255)        # '0xff'
int('FF', 16)   # 255
bin(0xFF)       # '0b11111111'
f'{255:08b}'    # '11111111'  (8 dígitos)
f'{255:02X}'    # 'FF'        (maiúsculo, 2 dígitos)
