Sistemas Numéricos

Binário, octal, decimal e hexadecimal — bases, conversão e aritmética

Computadores operam exclusivamente com dois estados elétricos (ligado/desligado), que mapeamos para os dígitos 0 e 1. Toda informação — números, texto, imagens, código — é codificada em binário. Hexadecimal e octal são apenas notações compactas para representar grupos de bits.

Exemplos de conversão
# Decimal → Binário: divisões sucessivas por 2
42 ÷ 2 = 21 r 0
21 ÷ 2 = 10 r 1
10 ÷ 2 =  5 r 0
 5 ÷ 2 =  2 r 1
 2 ÷ 2 =  1 r 0
 1 ÷ 2 =  0 r 1
→ lendo os restos de baixo para cima: 42₁₀ = 101010₂

# Binário → Hexadecimal: agrupar 4 bits por vez
1010 1111 0011
   A    F    3
→ 0xAF3

# Hexadecimal → Decimal: valor posicional
0x1F = 1×16¹ + 15×16⁰ = 16 + 15 = 31
0xFF = 15×16 + 15     = 255

Complemento de 2 — Inteiros com Sinal

Como representar números negativos
# Para negar um número em complemento de 2:
# 1. Inverter todos os bits (complemento de 1)
# 2. Somar 1

+5 em 8 bits = 0000 0101
Inverter bits = 1111 1010
Somar 1       = 1111 1011  ← representação de −5

# Verificar: 5 + (−5) deve ser 0
  0000 0101
+ 1111 1011
= 0000 0000  (overflow descartado) ✓

# Faixas com 8 bits:
# sem sinal (uint8):  0 a 255
# com sinal (int8):  −128 a +127

O bit mais significativo (MSB) é o bit de sinal em representação com sinal: 0 = positivo, 1 = negativo. Por isso 0x80 é −128 como int8 mas +128 como uint8.
