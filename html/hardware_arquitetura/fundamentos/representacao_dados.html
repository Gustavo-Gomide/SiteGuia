Representação de Dados

Inteiros, strings, endianness e alinhamento de memória

Toda informação armazenada ou processada por um computador é um conjunto de bits. A interpretação desses bits depende do tipo de dado: os mesmos 32 bits podem ser um inteiro, um float, quatro caracteres ASCII ou uma instrução de máquina.

Endianness

Little-Endian

Byte menos significativo armazenado no endereço mais baixo. Usado por x86, x86-64 e ARM (modo padrão).

0x12345678 em memória (addr baixo → alto):
[addr+0] = 0x78  (LSB)
[addr+1] = 0x56
[addr+2] = 0x34
[addr+3] = 0x12  (MSB)

Big-Endian

Byte mais significativo no endereço mais baixo. Usado em redes (TCP/IP é big-endian = "network byte order"), PowerPC, SPARC.

0x12345678 em memória (addr baixo → alto):
[addr+0] = 0x12  (MSB)
[addr+1] = 0x34
[addr+2] = 0x56
[addr+3] = 0x78  (LSB)

Funções htons() / ntohl() (POSIX) convertem entre byte order do host e da rede. Em Python: int.to_bytes(4, 'big').

Alinhamento e Padding de Structs

C — efeito do alinhamento
// CPUs exigem que dados estejam alinhados a múltiplos do seu tamanho
// O compilador insere "padding" automaticamente

struct A {
    char  a;    // 1 byte,  offset 0
    // 3 bytes de padding (para alinhar 'b' em 4)
    int   b;    // 4 bytes, offset 4
    char  c;    // 1 byte,  offset 8
    // 3 bytes de padding (para align total = 4)
};  // sizeof(A) = 12, não 6!

// Reordenar para economizar memória
struct B {
    int   b;    // 4 bytes, offset 0
    char  a;    // 1 byte,  offset 4
    char  c;    // 1 byte,  offset 5
    // 2 bytes de padding
};  // sizeof(B) = 8

// Desabilitar padding (cuidado: pode ser lento)
#pragma pack(1)
struct C { char a; int b; char c; }; // sizeof = 6
