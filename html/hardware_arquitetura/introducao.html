Hardware e Arquitetura de Computadores

Componentes físicos, camadas de abstração e como um computador funciona de dentro para fora

Hardware é o conjunto de componentes físicos que compõem um sistema computacional. Arquitetura de computadores define como esses componentes são organizados e interagem para executar instruções. Compreender essa base é fundamental para escrever software eficiente, diagnosticar problemas de desempenho e entender os limites do que uma máquina pode fazer.

Componentes Principais

CPU

Unidade central de processamento. Executa instruções, realiza cálculos e coordena os demais componentes. Contém ALU, unidade de controle, registradores e caches L1/L2/L3.

Memória RAM

Memória de acesso aleatório, volátil. Armazena dados e instruções em uso ativo. Muito mais rápida que discos; muito mais lenta que caches da CPU. Tipicamente DDR4 ou DDR5.

Armazenamento

HDDs e SSDs armazenam dados de forma persistente. SSDs NVMe oferecem GBs/s de largura de banda; HDDs giratórios oferecem alta capacidade por menor custo.

GPU

Processador com milhares de núcleos simples otimizados para operações paralelas em dados vetoriais e matriciais. Essencial para renderização e computação de alto desempenho.

Placa-mãe

Substrato que interconecta todos os componentes. Contém chipset, slots PCIe, soquete da CPU, slots de RAM, conectores SATA/M.2 e portas de I/O.

Periféricos e I/O

Dispositivos de entrada (teclado, mouse, sensores) e saída (monitor, impressora, áudio). Comunicam-se com a CPU via barramentos como USB, PCIe e I2C.

Hardware vs. Software

Hardware

  • Componentes físicos, tangíveis
  • Microarquitetura definida em fabricação
  • Não muda sem substituição física
  • Executa instruções na velocidade de GHz
  • Limitado por leis físicas: calor, velocidade da luz

Software

  • Instruções e dados digitais
  • Flexível: atualizado e substituído a qualquer hora
  • Define o comportamento do hardware
  • Inclui firmware, SO, drivers e aplicativos
  • A distinção se borra em FPGAs e microcode

Do Disco à Execução

Ciclo de vida dos dados (simplificado)
Disco (HDD/SSD)
  └─ bytes do programa carregados pelo SO para RAM
       └─ instrução buscada pelo prefetch buffer da CPU
            └─ decodificada pela unidade de controle
                 └─ operandos lidos de registradores / cache L1
                      └─ ALU executa a operação
                           └─ resultado em registrador / cache
                                └─ eventualmente gravado de volta em RAM / disco

Caches (L1/L2/L3) existem para evitar que a CPU espere pela RAM a cada acesso. A hierarquia de memória é um dos fatores mais críticos de desempenho.
