Introdução ao Assembly

A linguagem mais próxima do hardware: mnemonônicos, assemblers e o primeiro programa

Assembly é uma linguagem de representação textual das instruções de máquina. Cada mnemonônico (ex: MOV, ADD, JMP) corresponde diretamente a um opcode binário. Não existe abstração sobre o hardware — você controla registradores, endereços e flags diretamente.

Por que Aprender Assembly?

Depuração avanada

Entender o disassembly no GDB/LLDB/WinDbg é essencial para depurar crashes, buffer overflows e problemas de desempenho em código de baixo nível.

Segurança e Reverse Engineering

Análise de malware, exploits, engenharia reversa de binários e CTFs exigem leitura fluente de assembly x86-64 e ARM.

Otimização de Desempenho

Hot paths em kernels de OS, codecs de vídeo, DSP e simuladores físicos são escritos em assembly (ou SIMD intrinsics) para extrair o máximo do hardware.

Hello World — NASM x86-64 Linux

hello.asm — NASM, syscall Linux
section .data
    msg db "Hello, World!", 0x0A  ; string + newline
    len equ $ - msg               ; tamanho calculado pelo assembler

section .text
    global _start

_start:
    ; syscall write(fd=1, buf=msg, count=len)
    mov rax, 1        ; syscall number: write
    mov rdi, 1        ; fd: stdout
    mov rsi, msg      ; endereço do buffer
    mov rdx, len      ; número de bytes
    syscall           ; chamada ao kernel

    ; syscall exit(0)
    mov rax, 60       ; syscall number: exit
    xor rdi, rdi      ; exit code 0
    syscall

; Build:
; nasm -f elf64 hello.asm -o hello.o
; ld hello.o -o hello
; ./hello

No Windows o ponto de entrada e as syscalls são diferentes; é comum usar a CRT ou Windows API diretamente. Em macOS (ABI Mach-O), syscalls usam números diferentes e são prefixadas com 0x2000000.
