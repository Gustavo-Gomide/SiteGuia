Assembly x86 (IA-32)

A arquitetura CISC de 32 bits que dominou o PC por duas décadas

O x86 de 32 bits (também chamado IA-32 ou i386) foi a ISA dominante em desktops e servidores de 1985 a 2005. Ainda presente em kernels de SO (modo de compatibilidade), binários legados e aplicações embarcadas 32-bit. Entendê-lo é fundamental para estudar x86-64.

Convenções de Chamada 32-bit

cdecl (C padrão Linux)

  • Argumentos empilhados da direita para a esquerda
  • Caller limpa a pilha após a chamada
  • Retorno em EAX (e EDX para valores 64-bit)
  • EBX, ESI, EDI, EBP preservados pelo callee

stdcall (Win32 API)

  • Argumentos empilhados da direita para a esquerda
  • Callee limpa a pilha (RET N)
  • Retorno em EAX
  • Usado pela WinAPI (ex: MessageBoxA)

Exemplo: função em NASM i386

NASM i386 — soma de dois inteiros (cdecl)
; int soma(int a, int b)
; Stack layout após CALL + PUSH EBP:
;   [EBP+8]  = a
;   [EBP+12] = b
soma:
    push ebp
    mov  ebp, esp    ; cria frame

    mov  eax, [ebp+8]   ; a
    add  eax, [ebp+12]  ; a + b
    ; resultado em EAX (convenção cdecl)

    pop  ebp
    ret              ; caller limpa os args

Em 64-bit (x86-64 / AMD64), os primeiros 6 argumentos vão em registradores (RDI, RSI, RDX, RCX, R8, R9), eliminando a necessidade de empilhar argumentos simples.
