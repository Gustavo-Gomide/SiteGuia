HDD e SSD em Detalhe

Anatomia do HDD, NAND Flash, wear leveling e comparação NVMe vs. SATA

HDDs e SSDs são tecnologias fundamentalmente diferentes: HDDs usam discos magnéticos giratórios com cabeçotes móveis; SSDs usam células NAND Flash sem partes móveis. Cada tecnologia tem perfil próprio de desempenho, vida útil e casos de uso ideais.

HDD — Hard Disk Drive

ComponenteFunção
Prato (Platter)Disco magnético giratório; dados são registrados como domínios magnéticos nas superfícies
Cabeçote de Leitura/EscritaFlutua ~10 nm acima do prato; lê/grava dados magnéticos
Atuador (Actuator Arm)Braço que posiciona o cabeçote; movimentação mécanica é o principal gargalo
Motor SpindleGira os pratos a 5.400 ou 7.200 RPM (desktop); 10k–15k RPM (enterprise SAS)
Cache DRAMBuffer de 8–256 MB para agrupar escritas e pré-buscar dados sequenciais

O tempo de seek mécano (tempo para o cabeçote alcançar a trilha) é ~5–10 ms. Para acessos aleatórios pequenos, SSDs são 10.000x mais rápidos.

NVMe vs. SATA

SATA III

  • Interface máxima: 600 MB/s
  • Protocolo: ATA herdado (AHCI)
  • Latencia de comando: ~100 μs
  • Queue depth: até 32 comandos
  • Conector: SATA de dados + alimentação
  • Compatível com sistemas antigos

NVMe (PCIe)

  • PCIe 4.0 ×4: até ~7 GB/s
  • Protocolo: NVMe — criado para Flash
  • Latencia de comando: <20 μs
  • Queue depth: até 65.535 queues × 65.535 cmds
  • Conector: M.2 ou U.2 (enterprise)
  • Ideal para workloads com alto IOPS
