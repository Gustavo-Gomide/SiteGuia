Memória RAM

DDR1 ao DDR5: tipos, timings, dual channel e ECC

RAM (Random Access Memory) é a memória de trabalho volátil do sistema. Tudo que a CPU usa ativamente fica na RAM: código do programa, dados, pilha. Velocidade e capacidade de RAM têm impacto direto no desempenho, especialmente em aplicações que processam grandes volumes de dados.

Canais e Timings

Dual / Quad Channel

Usar 2 (ou 4) pentes em slots pareados dobra (ou quadruplica) a largura de banda de memória. Importante para GPUs integradas e workloads de alto throughput. Pentes devem ser idênticos ou muito similares.

# Verificar modo de channel (Linux)
sudo dmidecode -t memory | grep -i "channel"
# ou via lshw, cpuid, ou ferramentas como CPU-Z

Timings (CL-tRCD-tRP-tRAS)

Latência de CAS (CL): número de ciclos de clock até o dado estar disponível. Números menores = mais rápido, mas deve ser avaliado em relação à freqüência absoluta.

# DDR4-3200 CL16
Latencia real = CL / (freq MHz) × 2000
             = 16 / 3200 × 2000 = 10 ns

# DDR5-6000 CL30
Latencia real = 30 / 6000 × 2000 = 10 ns
# Mesma latência, mas 6000 tem >1.8x bandwidth

ECC RAM

O que é ECC

Error-Correcting Code: adiciona bits extras para detectar e corrigir erros de 1 bit e detectar (sem corrigir) erros de 2 bits. Usa SECDED (Single Error Correct, Double Error Detect). Requer CPU + placa-mãe com suporte.

Quando usar

Servidores, estações científicas e sistemas críticos onde corrupção silenciosa de dados seria catastófica. ECC tem ~2–3% de overhead de desempenho. Suportado por AMD Ryzen Pro, EPYC, Intel Xeon.
