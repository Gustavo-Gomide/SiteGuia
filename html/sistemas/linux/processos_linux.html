Processos no Linux

Ciclo de vida, monitoramento, sinais e controle de prioridade

No Linux, tudo é processo. Desde o primeiro programa executado pelo kernel (PID 1) até cada comando que você digita no terminal, o sistema operacional gerencia execução por meio de processos. Entender como listá-los, monitorá-los em tempo real, enviar sinais e controlar sua prioridade é habilidade essencial para administração de sistemas e debugging.

Listando Processos

O comando ps (process status) exibe um instantâneo dos processos em execução. A combinação ps aux é a mais usada no dia a dia.

ps — exemplos de uso
# Todos os processos de todos os usuários com terminal virtual
ps aux
# USER       PID  %CPU  %MEM    VSZ   RSS  TTY   STAT  START   TIME  COMMAND
# root         1   0.0   0.1  168656  9240  ?     Ss   jan01   0:12  /sbin/init

# Formato personalizado: PID, PPID, estado, nome e comando completo
ps -eo pid,ppid,stat,comm,cmd

# Ordenar por uso de CPU (decrescente)
ps aux --sort=-%cpu | head -20

# Ordenar por uso de memória
ps aux --sort=-%mem | head -20

# Exibir hierarquia de processos em árvore
pstree -p

# Ver processos de um usuário específico
ps -u www-data

# Ver threads de um processo
ps -eLf | grep nginx

Entender as colunas é fundamental para interpretar a saída do ps aux:

Coluna Significado Detalhe
USERProprietário do processoUID efetivo que executa o processo
PIDProcess IDIdentificador único; herdado pelo filho via fork()
%CPUUso de CPUMédia desde o início; pode ultrapassar 100% em multicore
%MEMUso de memória físicaRSS dividido pela RAM total
VSZVirtual Size (KB)Todo o espaco de endereço virtual reservado, incluindo mmap e bibliotecas
RSSResident Set Size (KB)Paginas realmente na RAM; exclui swap e pages ainda nao carregadas
TTYTerminal controlador? = sem terminal (daemon)
STATEstado do processoS=sleeping, R=running, Z=zombie, T=stopped, D=uninterruptible I/O
TIMETempo de CPU acumuladoTempo total de CPU consumido desde o inicio

VSZ vs RSS vs SHR: VSZ e o total de espaco virtual reservado (inclui regioes nao carregadas); RSS e o que esta fisicamente na RAM; SHR e a parcela compartilhada com outros processos (bibliotecas dinamicas). Um processo com VSZ alto mas RSS baixo apenas reservou espaco — nao consumiu RAM de fato.

top, htop e Monitoramento em Tempo Real

Ferramentas interativas oferencem uma visao ao vivo dos processos, atualizando periodicamente as metricas de CPU, memoria e I/O.

Ferramenta Recursos exibidos Interatividade Instalacao
topCPU, MEM, load average, processosBasica (k=kill, r=renice, q=sair)Incluso no procps-ng
htopCPU por nucleo, mem, swap, uptime, arvoreAvancada com mouse, menus, filtrosapt install htop / dnf install htop
atopCPU, mem, disco, rede, processos historicosMedia; permite rebobinar historicoapt install atop
glancesCPU, mem, disco, rede, containers, alertasWeb UI e API REST disponiveispip install glances

Atalhos importantes no htop:

F5 — Tree View

Exibe a hierarquia pai/filho dos processos, equivalente ao pstree mas com metricas ao vivo.

F9 — Kill

Abre um menu de sinais para enviar ao processo selecionado (SIGTERM, SIGKILL, etc.).

F7 / F8 — Nice

Diminui ou aumenta a prioridade nice do processo. F7 diminui (mais prioritario), F8 aumenta.

Barra de espaco — Filtro

Filtra processos por nome em tempo real. Util para focar em um servico especifico em servidores movimentados.

Na cabecalho do htop, as colunas de processos incluem VIRT (virtual), RES (resident), SHR (shared), %CPU, %MEM e TIME+ (tempo de CPU com resolucao de decimos de segundo).

Sinais e kill

Sinais sao notificacoes assincronas enviadas a processos. O comando kill envia um sinal ao processo identificado pelo PID. Apesar do nome, nao e exclusivo para encerrar processos.

Sinal Numero Capturavel? Descricao Uso comum
SIGHUP1SimHangup — terminal fechou ou pedido de reloadRecarregar configuracao de daemon sem reiniciar
SIGINT2SimInterrupt — Ctrl+C no terminalEncerrar processo interativo
SIGQUIT3SimQuit — Ctrl+\ gera core dumpDiagnostico de travamento
SIGKILL9NaoKill incondicional pelo kernelForcado quando SIGTERM falha; nao permite limpeza
SIGTERM15SimTerminate — encerramento graciosoPadrao de kill sem argumento
SIGSTOP19NaoSuspende execucao (Ctrl+Z)Pausar processo; retomar com SIGCONT
SIGCONT18SimContinua processo suspensoRetomar processo pausado por SIGSTOP
SIGUSR110SimSinal definido pelo usuarioLogrotate usa para pedir reabertura de arquivos de log
SIGUSR212SimSinal definido pelo usuarioNginx usa para atualizacao binaria a quente
kill, killall, pkill — exemplos
# Enviar SIGTERM (gracioso) a um PID
kill -SIGTERM 1234
kill -15 1234
kill 1234          # SIGTERM e o padrao

# Forcado — nao pode ser ignorado
kill -SIGKILL 1234
kill -9 1234

# Matar todos os processos com um determinado nome
killall nginx
killall -9 python3

# Matar por padrao de nome (suporta regex)
pkill -f "backup_script.py"
pkill -u www-data

# Listar sinais disponíveis
kill -l

# Matar janela grafica clicando nela
xkill

nice, renice e Prioridade

O valor nice vai de -20 (prioridade maxima) a +19 (prioridade minima). O valor padrao e 0. Apenas root pode definir valores negativos (aumentar prioridade acima do padrao).

nice e renice — exemplos
# Iniciar processo com prioridade reduzida (mais amigavel ao sistema)
nice -n 10 ./backup.sh

# Iniciar com prioridade elevada (requer root)
sudo nice -n -10 ./processo_critico.sh

# Alterar prioridade de processo em execucao
renice -n -5 -p 2345

# Alterar prioridade de todos os processos de um usuario
renice -n 5 -u www-data

# Ver prioridade atual com ps
ps -eo pid,ni,pri,comm | grep backup
# PID  NI PRI COMMAND
# 2345  10  10 backup.sh

# Prioridade de tempo real (requer root)
# SCHED_FIFO: nao preemptivel ate bloquear ou ceder
chrt -f 50 ./processo_rt

# SCHED_RR: round-robin de tempo real
chrt -r 20 ./processo_rt

# Ver politica de escalonamento de um processo
chrt -p 2345

A prioridade de escalonamento exibida pelo kernel (PRI no ps) relaciona-se ao nice mas inclui ajustes dinamicos. Para tarefas de tempo real (audio, controle industrial), use chrt com politicas SCHED_FIFO ou SCHED_RR que garantem preempcao deterministica.
