Comandos Básicos Linux

Navegação, arquivos, pipes, redirecionamento e os 50 comandos mais usados

Dominar a linha de comando é o que separa um usuário Linux de um profissional. Os comandos são as ferramentas; o shell é a bançada de trabalho. Uma vez que os dedos memorizam os atalhos e padrões, a produtividade no terminal supera qualquer interface gráfica para tarefas de sysadmin e desenvolvimento.

Navegação e Arquivos

bash — comandos essenciais de navegação e manipulação
# Onde estou?
pwd                          # /home/user/projetos

# Listar arquivos (detalhado, ocultos)
ls -la                       # -l: formato longo, -a: ocultos (.bashrc etc)
ls -lh /var/log              # -h: tamanhos humanos (KB, MB)

# Navegar
cd /etc                      # caminho absoluto
cd ../logs                   # caminho relativo
cd ~                         # home do usuario
cd -                         # diretorio anterior

# Criar diretorios (com pais intermediarios)
mkdir -p projetos/app/src

# Copiar
cp arquivo.txt backup.txt    # copiar arquivo
cp -r pasta/ destino/        # copiar diretorio recursivamente

# Mover / Renomear
mv antigo.txt novo.txt       # renomear
mv arquivo.txt /tmp/         # mover para /tmp

# Remover
rm arquivo.txt               # remover arquivo
rm -rf pasta/                # remover diretorio e todo conteudo (cuidado!)

# Buscar arquivos
find /var/log -name "*.log" -mtime -7   # .log modificados nos ultimos 7 dias
find /home -size +100M -type f          # arquivos > 100 MB

# Ver conteudo
cat /etc/hosts               # imprime tudo de uma vez
less /var/log/syslog         # pagina (q para sair, /termo para buscar)
head -20 /etc/passwd         # primeiras 20 linhas
tail -f /var/log/syslog      # acompanhar em tempo real (segue o arquivo)

# Estatisticas
wc -l arquivo.txt            # contar linhas
stat arquivo.txt             # metadados completos (tamanho, permissoes, datas)
du -sh pasta/                # tamanho do diretorio em formato humano
df -h                        # uso do disco por particao

Pipes e Redirecionamento

bash — pipes, redirecionamento e exemplos práticos
# PIPES: conectar saida de um comando a entrada do proximo
ps aux | grep nginx          # processos nginx em execucao
cat /etc/passwd | cut -d: -f1 | sort   # usuarios em ordem alfabetica

# REDIRECIONAMENTO DE SAIDA
echo "linha" > arquivo.txt   # sobrescreve
echo "mais" >> arquivo.txt   # acrescenta
ls /nao-existe 2> erros.txt  # redirecionar stderr
ls /etc > saida.txt 2>&1     # stdout e stderr para o mesmo arquivo

# ENTRADA
wc -l < arquivo.txt          # contar linhas sem echo do nome do arquivo

# DESCARTAR SAIDA
comando &> /dev/null         # descartar stdout e stderr

# EXEMPLOS COMPOSTOS
# Top 5 IPs mais frequentes no log do nginx
cat /var/log/nginx/access.log \
  | awk '{print $1}' \
  | sort \
  | uniq -c \
  | sort -rn \
  | head -5

# Contar arquivos por extensao
find . -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn

# Filtrar linhas de erro no log e salvar
grep -i "error" /var/log/syslog | tail -100 > erros_recentes.txt

Man Pages e Ajuda

O manual do Linux está embutido no próprio sistema. Antes de recorrer à internet, consulte:

man

man ls — manual completo. Navegue com as setas, busque com /termo, saia com q. Seções: NAME, SYNOPSIS, DESCRIPTION, OPTIONS, EXAMPLES, SEE ALSO.

apropos

apropos compress — busca por palavra-chave em todos os man pages. Útil quando você não sabe o nome exato do comando.

--help

comando --help — sumário rápido de opções. Mais curto que o man page; ideal para referência rápida das flags.

tldr

tldr tar — man pages simplificadas com exemplos práticos. Instale com npm i -g tldr ou apt install tldr.

bash — exemplos de help e man
# Ver as primeiras 50 linhas do manual do ls
man ls | head -50

# Buscar comandos relacionados a rede
apropos network | grep -i socket

# Help rapido do tar
tar --help 2>&1 | head -30

# Secao especifica (1=user, 2=syscall, 3=lib, 5=file, 8=admin)
man 5 passwd   # formato do arquivo /etc/passwd
man 2 open     # syscall open()
