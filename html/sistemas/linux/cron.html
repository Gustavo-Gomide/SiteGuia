Cron — Agendamento de Tarefas

cron é o daemon Unix/Linux responsável por executar comandos em horários programados. Lê os arquivos crontab e dispara cada tarefa no momento exato definido, independentemente de qualquer interação humana. É indispensável para backups automáticos, rotação de logs, sincronização de dados e qualquer tarefa recorrente.

O serviço é gerenciado como cron ou crond dependendo da distribuição. Os arquivos de crontab de usuários ficam em /var/spool/cron/crontabs/ e os do sistema em /etc/crontab e /etc/cron.d/.

Sintaxe do Crontab

Formato e caracteres especiais
# Formato de cada linha:
# MIN  HORA  DIA-DO-MÊS  MÊS  DIA-DA-SEMANA  COMANDO
#  │    │        │         │         │
#  │    │        │         │         └── 0-7 (0 e 7 = domingo; 1=seg...6=sab)
#  │    │        │         └──────────── 1-12 (ou jan,feb,...,dec)
#  │    │        └────────────────────── 1-31
#  │    └─────────────────────────────── 0-23
#  └──────────────────────────────────── 0-59

# Caracteres especiais:
#   *    → qualquer valor
#   ,    → lista: 1,3,5
#   -    → intervalo: 1-5
#   /    → passo: */5 (a cada 5)
#   @reboot  → uma vez ao iniciar
#   @daily   → equivale a 0 0 * * *
#   @weekly  → equivale a 0 0 * * 0
#   @monthly → equivale a 0 0 1 * *
#   @hourly  → equivale a 0 * * * *

Exemplos Práticos

Entradas comuns no crontab
# Editar crontab do usuário atual
crontab -e

# Listar crontab do usuário atual
crontab -l

# ------- Exemplos de entradas no crontab -------

# Backup do banco de dados a cada dia às 2h30
30 2 * * * /usr/local/bin/backup-db.sh >> /var/log/backup.log 2>&1

# Limpar arquivos temporários todo domingo às 4h
0 4 * * 0 find /tmp -mtime +7 -delete

# Reiniciar serviço nginx a cada hora
0 * * * * systemctl restart nginx

# Executar a cada 5 minutos
*/5 * * * * /usr/bin/python3 /home/user/monitor.py

# Backup incremental de segunda a sexta às 23h
0 23 * * 1-5 rsync -avz /dados/ /backup/

# Executar ao reiniciar (após boot)
@reboot /home/user/start-services.sh

# Rodar em múltiplos horários: 6h e 18h
0 6,18 * * * /opt/scripts/sync.sh

# Variáveis de ambiente no crontab
MAILTO="admin@empresa.com"
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin

Tipos e Localizações

Local Quem usa Descrição
crontab -e (usuário) Usuário corrente Tarefas pessoais; arquivo em /var/spool/cron/crontabs/USER
/etc/crontab root Crontab do sistema; tem campo extra para o usuário de execução
/etc/cron.d/ Pacotes do sistema Cada pacote instala seu próprio arquivo; mesmo formato de /etc/crontab
/etc/cron.daily/ root Scripts executados diariamente pelo run-parts
/etc/cron.hourly/ root Scripts executados a cada hora
/etc/cron.weekly/ root Scripts executados semanalmente
/etc/cron.monthly/ root Scripts executados mensalmente
anacron root Executa tarefas que foram perdidas (máquina desligada); não precisa de horário fixo

cron vs systemd timers

cron

Familiar, simples e amplamente suportado em qualquer distribuição Linux. Não captura saída automaticamente (necessário redirecionar para arquivo ou usar MAILTO). Não ativa máquinas desligadas e não possui dependências entre tarefas.

systemd timers

Integração com journald (logs automáticos via journalctl), suporte a dependências entre units, calendários flexíveis com OnCalendar=, ativação na inicialização e inspeção com systemctl status TIMER.

Veja mais em Serviços e systemd e Processos Linux.
