Usuários e Grupos no Linux

/etc/passwd, /etc/shadow, /etc/group, gerenciamento e contas de serviço

No Linux, cada processo roda como um usuário específico; cada arquivo pertence a um usuário e a um grupo. Entender como gerenciar usuários, grupos, UIDs reservados e contas de serviço é fundamental tanto para segurança quanto para administração de sistemas multi-usuário.

Arquivos de Usuários

ArquivoFormatoConteúdo
/etc/passwdlogin:x:uid:gid:GECOS:home:shellInformações públicas do usuário; legível por todos; hash da senha movido para shadow
/etc/shadowlogin:$hash:lastchg:min:max:warn:inactive:expireHashes de senha e política de expiração; legível apenas por root
/etc/groupnome:x:gid:membrosDefinição de grupos e lista de membros suplementares
/etc/gshadownome:hash:admins:membrosSenhas de grupo (raro em uso); protegido como shadow
bash — linhas reais dos arquivos de usuários
# /etc/passwd (campo a campo)
# login : x  : uid  : gid  : GECOS       : home        : shell
  root  : x  : 0    : 0    : root        : /root       : /bin/bash
  www-data:x : 33   : 33   : www-data    : /var/www    : /usr/sbin/nologin
  joao  : x  : 1001 : 1001 : Joao Silva  : /home/joao  : /bin/bash

# /etc/shadow (campo a campo)
# login : $hash       : lastchg : min : max : warn : inactive : expire
  joao  : $6$salt$... : 19700   : 0   : 90  : 7    :          :
# $6$ = SHA-512; $5$ = SHA-256; $y$ = yescrypt (moderno)

# /etc/group (campo a campo)
# nome    : x : gid : membros
  docker  : x : 999 : joao,deployer
  sudo    : x : 27  : joao,admin
  www-data: x : 33  :

# Consultar usuario via getent (funciona com LDAP tambem)
getent passwd joao
getent group docker

Gerenciamento de Usuários

ComandoFlags ImportantesDescrição
useradd-m (criar home), -s (shell), -G (grupos), -c (GECOS)Criar novo usuário
userdel-r (remover home e mail spool)Remover usuário do sistema
usermod-aG (adicionar a grupo), -L (bloquear), -U (desbloquear), -s (mudar shell)Modificar usuário existente
passwd-l (lock), -u (unlock), -e (expirar senha)Gerenciar senha de usuário
chage-l (listar), -M (max dias), -W (aviso), -E (data expiração)Política de expiração de senha
id—Exibir UID, GID e grupos do usuário atual ou especificado
groups—Listar grupos do usuário
bash — exemplos práticos de gerenciamento
# Criar usuario com home, shell bash e grupos adicionais
useradd -m -s /bin/bash -G sudo,docker -c "Desenvolvedora" ana

# Definir senha
passwd ana

# Adicionar usuario a grupo sem remover dos outros (-a eh crucial)
usermod -aG docker ana

# Bloquear acesso (sem remover o usuario)
usermod -L joao
passwd -l joao      # equivalente

# Ver info de expiração da senha
chage -l ana

# Forcar troca de senha no proximo login
chage -d 0 ana
# ou
passwd -e ana

# Ver UID/GID e grupos do usuario atual
id
id ana

# Remover usuario E home
userdel -r joao_antigo

Grupos e Gerenciamento

Grupos permitem compartilhar acesso a recursos entre vários usuários. Cada usuário tem um grupo primário (GID no /etc/passwd) e pode pertencer a múltiplos grupos suplementares.

bash — gerenciamento de grupos
# Criar grupo
groupadd desenvolvedores

# Criar grupo com GID especifico
groupadd -g 2000 deploy

# Modificar grupo
groupmod -n devs desenvolvedores    # renomear
groupmod -g 2001 deploy             # mudar GID

# Remover grupo (apenas se nao for primario de nenhum usuario)
groupdel antigo_grupo

# Trocar grupo primario na sessao atual (sem logout)
newgrp docker     # abre subshell com docker como GID primario

# Executar comando como membro de grupo
sg docker -c "docker ps"

# Listar todos os grupos via NSS (funciona com LDAP)
getent group | grep -i develop

# Ver grupos do usuario
groups ana       # ana : ana sudo docker desenvolvedores

Boa prática: criar um grupo por serviço (www-data, docker, nginx) e adicionar apenas os usuários autorizados, em vez de usar root ou conceder permissões globais.

UID/GID Reservados e Contas de Serviço

Faixa de UIDUsoExemplos
0Superusuário rootroot (UID 0, GID 0) — acesso irrestrito
1 – 99Contas do sistema basedaemon (1), bin (2), sys (3) — histórico Unix
100 – 999Contas de serviço criadas por pacoteswww-data (33), syslog (104), messagebus (105), nobody (65534)
1000+Usuários reais interativosO primeiro usuário criado na instalação (UID 1000)
65534nobody — usuário sem privilégiosNFS, processos que precisam rodar sem identidade

A configuração em /etc/login.defs controla os ranges: UID_MIN (padrão 1000), UID_MAX (padrão 60000), SYS_UID_MIN (101), SYS_UID_MAX (999).

Contas de serviço boas práticas: sem shell login (/usr/sbin/nologin ou /bin/false), sem home dir significativo, sem senha definida, escondem privilégios mínimos necessários ao serviço.

bash — criar conta de servico corretamente
# Criar conta de servico segura
useradd \
  --system \
  --no-create-home \
  --shell /usr/sbin/nologin \
  --comment "Conta de servico do app" \
  appuser

# Verificar
getent passwd appuser
# appuser:x:998:998:Conta de servico do app:/:/usr/sbin/nologin

# Tentar logar deve falhar
su - appuser
# This account is currently not available.

# Ainda e possivel rodar processos como esse usuario
sudo -u appuser /opt/app/bin/app --config /etc/app/config.ini
