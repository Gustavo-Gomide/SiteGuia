YUM e DNF — Gerenciadores RPM

DNF (Dandified YUM) é o gerenciador de pacotes moderno das distribuições baseadas em RPM: Fedora, RHEL, CentOS Stream, Rocky Linux, AlmaLinux e openSUSE. Substituiu o YUM como padrão a partir do Fedora 22. O YUM ainda é usado no RHEL 7 e derivados mais antigos. Ambos resolvem dependências automaticamente a partir de repositórios configurados em arquivos .repo.

O DNF utiliza a biblioteca libsolv como SAT solver para resolução de dependências, tornando-o mais rápido e preciso que o YUM. Os metadados dos repositórios ficam em cache em /var/cache/dnf/.

Repositórios — Arquivos .repo

/etc/yum.repos.d/ e gerenciamento de repositórios
# Localização dos repositórios
ls /etc/yum.repos.d/

# Estrutura de um arquivo .repo
cat /etc/yum.repos.d/fedora.repo
# [fedora]
# name=Fedora $releasever - $basearch
# baseurl=http://download.fedoraproject.org/pub/fedora/linux/releases/$releasever/...
# enabled=1
# gpgcheck=1
# gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-fedora-$releasever-$basearch

# Listar todos os repositórios configurados
dnf repolist
dnf repolist all     # inclui desabilitados
dnf repoinfo REPO    # detalhes de um repositório

# Habilitar/desabilitar repositório temporariamente
dnf install --enablerepo=crb nginx
dnf update --disablerepo=updates-testing

# Adicionar repositório EPEL (Extra Packages for Enterprise Linux)
sudo dnf install epel-release

# Adicionar repositório do Docker no RHEL/Rocky
sudo dnf config-manager --add-repo https://download.docker.com/linux/rhel/docker-ce.repo
sudo dnf install docker-ce

DNF vs YUM

DNF

Resolução de dependências com libsolv (SAT solver): mais rápido e preciso. Melhor gerenciamento de memória, suporte a módulos e streams via AppStream. Padrão no Fedora 22+ e RHEL 8+. Sintaxe idêntica ao YUM na maioria dos comandos.

YUM (legado)

Baseado no resolvedor antigo em Python 2. Ainda presente no RHEL 7 e derivados. Sintaxe praticamente idêntica ao DNF. Não suporta módulos AppStream nem streams. Plugins em Python 2.

RPM Diretamente

O RPM é a ferramenta de nível baixo que instala arquivos .rpm diretamente, sem resolver dependências automaticamente. Equivalente ao dpkg no mundo Debian.

rpm — operações de nível baixo
# RPM: nível baixo (sem resolver dependências)
rpm -ivh pacote.rpm        # instalar
rpm -Uvh pacote.rpm        # atualizar
rpm -e pacote              # remover
rpm -qa                    # lista todos os pacotes instalados
rpm -ql pacote             # lista arquivos do pacote
rpm -qf /usr/bin/python3   # qual pacote instalou este arquivo
rpm -qi pacote             # informações do pacote
rpm -V pacote              # verificar integridade dos arquivos do pacote

# Inspecionar RPM sem instalar
rpm -qpR pacote.rpm        # dependências do pacote
rpm -qpl pacote.rpm        # arquivos que seriam instalados
