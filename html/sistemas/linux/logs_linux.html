Logs no Linux

syslog, rsyslog, journald, logrotate e analise avancada de logs

Logs sao a memoria do sistema. Sem eles, diagnosticar problemas seria como depurar as cegas — voce saberia que algo deu errado, mas nao quando, por que, ou qual componente falhou. Entender onde os logs sao armazenados, como filtrar mensagens relevantes e como configurar retencao adequada e habilidade critica para qualquer administrador Linux.

rsyslog e Niveis de Severidade

O protocolo syslog define um sistema padronizado de facilities e niveis de severidade. O rsyslog e o daemon moderno que implementa esse protocolo na maioria das distros.

Nivel Codigo Palavra-chave Uso
Emergency0emergSistema inutilizavel — pane total
Alert1alertAcao imediata necessaria (disco cheio, RAID degradado)
Critical2critCondicoes criticas — falha de hardware ou software essencial
Error3errErros nao fatais que requerem atencao
Warning4warningAvisos — algo pode falhar em breve
Notice5noticeCondicoes normais mas significativas
Info6infoMensagens informativas de operacao normal
Debug7debugInformacoes detalhadas para depuracao

As facilities indicam a origem: kern (kernel), user (processos de usuario), mail, daemon, auth, cron, local0–local7 (uso personalizado). A regra no /etc/rsyslog.conf define para onde rotear cada combinacao facility.severity:

/etc/rsyslog.conf — exemplos de roteamento
# Gravar todos os erros e acima em /var/log/erros.log
*.err                          /var/log/erros.log

# Logs do kernel vao para kern.log
kern.*                         /var/log/kern.log

# Auth em arquivo separado
auth,authpriv.*                /var/log/auth.log

# Nivel info ou acima, exceto mail e auth, vao para syslog
*.info;mail.none;authpriv.none /var/log/syslog

# Encaminhar para servidor syslog remoto (UDP 514)
*.err                          @192.168.1.100:514

# Encaminhar com TCP (mais confiavel)
*.err                          @@192.168.1.100:514

Analise de Logs

A linha de comando Unix oferece um conjunto poderoso de ferramentas para filtrar, agregar e analisar logs sem instalar nada adicional.

Tecnicas de analise de logs no terminal
# Busca com expressao regular
grep -E "(ERROR|CRITICAL|WARN)" /var/log/syslog

# Mostrar contexto (3 linhas antes e depois do match)
grep -A 3 -B 3 "Out of memory" /var/log/kern.log

# Extrair campo especifico com awk (ex: coluna do IP de access.log)
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20

# Top 10 URLs mais acessadas
awk '{print $7}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -10

# Top 10 codigos de status HTTP
awk '{print $9}' /var/log/nginx/access.log | sort | uniq -c | sort -rn

# Seguir logs em tempo real
tail -f /var/log/syslog
tail -f /var/log/nginx/error.log

# Ler logs comprimidos sem descomprimir
zcat /var/log/syslog.2.gz | grep "ssh"
zgrep "Failed password" /var/log/auth.log*

# journalctl com filtros combinados
journalctl -u nginx --since today -p err

# Contar tentativas de login falhas por IP (auth.log)
grep "Failed password" /var/log/auth.log \
  | awk '{print $11}' \
  | sort | uniq -c | sort -rn | head -10

logrotate — Rotacao e Retencao

O logrotate gerencia a rotacao automatica de logs, evitando que arquivos cresçam indefinidamente. E executado diariamente pelo cron ou por um timer systemd.

/etc/logrotate.d/nginx — configuracao tipica
/var/log/nginx/*.log {
    daily                    # rotacionar diariamente
    rotate 14                # manter 14 arquivos rotacionados
    compress                 # comprimir com gzip
    delaycompress            # comprimir somente na proxima rotacao (nao o mais recente)
    missingok                # nao errro se o arquivo nao existir
    notifempty               # nao rotacionar se o arquivo estiver vazio
    create 0640 www-data adm # recriar com permissoes especificas
    sharedscripts            # executar postrotate uma so vez mesmo com multiplos arquivos
    postrotate
        # Avisar nginx para reabrir os arquivos de log
        [ -f /var/run/nginx.pid ] && kill -USR1 $(cat /var/run/nginx.pid)
    endscript
}

Frequencia de Rotacao

Use daily para logs de alto volume (web, app), weekly para logs de sistema moderados, monthly para logs de baixo volume ou de longa retencao.

Politica de Retencao

Balanceie juridico (alguns setores exigem retencao por anos), espaco em disco, e LGPD (dados pessoais em logs devem ter ciclo de vida definido e podem exigir anonimizacao ou delecao).

Testar Configuracao

Use logrotate -d /etc/logrotate.d/nginx para executar em modo debug (dry-run) sem alterar arquivos, e logrotate -f para forcar rotacao imediata.

Aprofunde-se em Servicos e systemd para entender o journald em detalhe, e em Processos no Linux para correlacionar logs com o comportamento dos processos.

"Um sistema bem configurado nao esconde seus logs — ele os organiza, rotaciona e torna acessiveis no momento certo."
