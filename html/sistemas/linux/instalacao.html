Sistemas Operacionais

Instalação do Linux

Pendrive bootável, particionamento, dual boot e configuração pós-instalação

Instalar Linux ficou trivial — o instalador moderno guia o processo em poucos cliques. O principal cuidado está no particionamento correto e, quando necessário, na configuração do dual boot com Windows sem corromper o bootloader existente.

Preparação

Requisitos mínimos (Ubuntu 24.04): 2 GB de RAM (4 GB recomendado), 25 GB de disco, processador de 64 bits, acesso à internet para atualizações.

Baixe a ISO no site oficial da distribuição e verifique a integridade com SHA256 antes de gravar:

bash — verificar hash da ISO
sha256sum ubuntu-24.04-desktop-amd64.iso
# Compare com o hash publicado no site oficial

Crie o pendrive bootável com uma das ferramentas abaixo:

Rufus (Windows)

Interface gráfica simples; suporta GPT/MBR e UEFI/Legacy. Recomendado para usuários Windows que ainda não têm Linux instalado.

Balena Etcher

Disponível em Windows, macOS e Linux. Arrastar-e-soltar a ISO e escolher o dispositivo USB. Verifica a gravação automaticamente.

dd (Linux/macOS)

Comando nativo, sem interface gráfica. O mais direto e confiável em sistemas Unix.

bash — gravar ISO com dd
# Identifique o pendrive (cuidado: nao use o disco do sistema)
lsblk

# Grave a ISO (substitua sdX pelo seu pendrive, ex: sdb)
sudo dd if=ubuntu-24.04-desktop-amd64.iso of=/dev/sdX bs=4M status=progress oflag=sync

# Aguarde a conclusao (pode demorar alguns minutos)

Após gravar, reinicie o computador e acesse a BIOS/UEFI (geralmente F2, F12, DEL ou ESC durante o boot) para definir o pendrive USB como primeira opção de boot.

Particionamento

Esquema recomendado para uso geral em sistema UEFI com GPT:

PartiçãoTamanhoFormatoFunção
/boot/efi500 MBFAT32 (ESP)EFI System Partition; armazena bootloader (GRUB, systemd-boot)
/boot1 GB (opcional)ext4Kernels e initramfs (necessário se /boot/efi usar ESP separada)
/50+ GBext4 / btrfs / xfsSistema raiz: binários, bibliotecas, configurações, logs
/homeRestante do discoext4 / btrfsDados do usuário; separar permite reinstalar sem perder dados
swap= RAM ou 8 GB mín.swap ou swapfileMemória virtual; obrigatório para hibernação

MBR vs GPT: GPT é obrigatório para discos maiores que 2 TB e para boot UEFI. MBR é legado, suporta no máximo 4 partições primárias. Use sempre GPT em hardware moderno.

LVM: Logical Volume Manager permite redimensionar partições sem reiniciar, criar snapshots e fazer thin provisioning — recomendado para servidores.

Dual Boot com Windows

Para rodar Linux e Windows no mesmo disco, siga esta ordem para evitar problemas:

1. Desativar Fast Startup

No Windows: Painel de Controle > Opcoes de Energia > Escolher funcao dos botoes > desmarcar "Inicializacao rapida". Sem isso, o Windows deixa o disco NTFS em estado sujo, impedindo acesso pelo Linux.

2. Secure Boot

Ubuntu, Fedora e outras distros principais suportam Secure Boot via MOK (Machine Owner Key). Desativar o Secure Boot é opcional nessas distros, mas necessário para kernels não assinados ou Arch Linux.

3. Reduzir Partição Windows

No Windows: clique com botão direito em "Este PC" > Gerenciar > Gerenciamento de Disco > clique com direito na partição C: > Reduzir Volume. Libere espaço para o Linux.

4. Instalar Linux

O instalador do Ubuntu, Fedora ou outros detecta automaticamente o Windows e oferece a opção "Instalar ao lado do Windows". O GRUB detectará o Windows na primeira reinicialização.

Ajuste do fuso horário — Windows usa localtime (hora local) no relógio de hardware, Linux usa UTC por padrão. Para evitar divergência de 3 horas após alternar entre sistemas:

bash — configurar Linux para usar localtime (compativel com Windows)
timedatectl set-local-rtc 1 --adjust-system-clock

Pós-Instalação Essencial

Após o primeiro boot no sistema instalado, execute estes passos:

bash — configuração pós-instalação Ubuntu/Debian
# 1. Atualizar todos os pacotes
sudo apt update && sudo apt upgrade -y

# 2. Instalar drivers proprietários automaticamente (Ubuntu)
sudo ubuntu-drivers autoinstall

# 3. Configurar fuso horario
timedatectl set-timezone America/Sao_Paulo
timedatectl status

# 4. Ativar firewall
sudo ufw enable
sudo ufw status

# 5. Habilitar e iniciar SSH (para acesso remoto)
sudo systemctl enable --now ssh

# 6. Instalar ferramentas essenciais
sudo apt install -y curl wget git vim htop tree unzip build-essential

# 7. Verificar integridade do sistema
sudo apt autoremove && sudo apt autoclean
