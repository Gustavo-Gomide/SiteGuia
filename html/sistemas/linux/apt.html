APT — Advanced Package Tool

APT (Advanced Package Tool) é o gerenciador de pacotes padrão das distribuições baseadas em Debian: Ubuntu, Linux Mint, Pop!_OS, Kali Linux e muitas outras. Ele resolve automaticamente dependências, baixa pacotes dos repositórios configurados e mantém o sistema atualizado com um conjunto simples de comandos.

O banco de dados local de pacotes instalados fica em /var/lib/dpkg/status e o cache de pacotes baixados em /var/cache/apt/archives/. A lista de repositórios disponíveis é mantida em /etc/apt/sources.list e nos arquivos dentro de /etc/apt/sources.list.d/.

Comandos Essenciais

Comando O que faz Exemplo
apt update Atualiza lista de pacotes dos repositórios sudo apt update
apt upgrade Instala versões mais novas de todos os pacotes sudo apt upgrade -y
apt install PKG Instala pacote e suas dependências sudo apt install nginx
apt remove PKG Remove pacote (mantém config) sudo apt remove nginx
apt purge PKG Remove pacote e arquivos de configuração sudo apt purge nginx
apt autoremove Remove dependências órfãs sudo apt autoremove
apt search TERM Busca pacotes pelo nome/descrição apt search "text editor"
apt show PKG Exibe informações detalhadas do pacote apt show nginx
apt list --installed Lista pacotes instalados apt list --installed | grep nginx
apt-cache depends PKG Mostra dependências do pacote apt-cache depends git
apt full-upgrade Upgrade com remoção se necessário sudo apt full-upgrade
apt-mark hold PKG Impede atualização de um pacote sudo apt-mark hold linux-image-generic

Repositórios e Fontes

O APT lê os repositórios configurados em /etc/apt/sources.list e em arquivos .list dentro de /etc/apt/sources.list.d/. Repositórios externos precisam de uma chave GPG para verificar autenticidade dos pacotes.

/etc/apt/sources.list e adição de repositórios
# Formato: deb [options] URI SUITE COMPONENTS
deb http://archive.ubuntu.com/ubuntu noble main restricted universe multiverse
deb http://archive.ubuntu.com/ubuntu noble-updates main restricted universe multiverse
deb http://security.ubuntu.com/ubuntu noble-security main restricted

# Adicionar PPA (Personal Package Archive)
sudo add-apt-repository ppa:deadsnakes/python
sudo apt update

# Adicionar repositório externo com chave GPG
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu noble stable" | sudo tee /etc/apt/sources.list.d/docker.list

# Listar repositórios configurados
cat /etc/apt/sources.list
ls /etc/apt/sources.list.d/

APT vs apt-get vs dpkg

apt

Interface moderna e amigável, saída colorida com barra de progresso. Combina funcionalidades de apt-get e apt-cache em um único comando. Recomendado para uso interativo no terminal.

apt-get

Interface estável para scripts e automação. Sem barra de progresso, saída consistente entre versões. Preferido em scripts de provisionamento onde a previsibilidade de saída é essencial.

dpkg

Nível baixo: instala arquivos .deb diretamente sem resolver dependências. Use para pacotes baixados manualmente: sudo dpkg -i pacote.deb.

aptitude

TUI interativa com resolver de dependências avançado. Excelente para resolver conflitos complexos. Uso: sudo aptitude install pacote.

Configuração e Cache

Gerenciamento de cache e diagnóstico
# Ver configuração do APT
apt-config dump | grep -i cache

# Limpar cache de pacotes (libera espaço)
sudo apt clean           # remove todos os .deb do cache
sudo apt autoclean       # remove .deb de versões antigas

# Verificar integridade dos pacotes instalados
sudo dpkg --audit

# Listar pacotes quebrados
sudo dpkg --list | grep -E "^(iF|iH|rc)"

# Reinstalar pacote
sudo apt install --reinstall nginx

# Verificar arquivos de um pacote
dpkg -L nginx
dpkg -S /etc/nginx/nginx.conf   # qual pacote instalou este arquivo

# Desativar um repositório temporariamente
sudo add-apt-repository --remove ppa:deadsnakes/python
