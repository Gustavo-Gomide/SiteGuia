Sistemas Operacionais

Introdução ao Linux

Kernel, distribuições, filosofia Unix e por que 96% dos servidores rodam Linux

Linux é o kernel criado por Linus Torvalds em 1991; sozinho ele não forma um sistema operacional completo — foi a parceria com as ferramentas GNU que tornou o sistema utilizável. Hoje 96% dos servidores web do mundo rodam Linux, assim como a totalidade dos supercomputadores do Top500 e a base do Android.

Kernel vs Distribuição

O kernel é o núcleo do sistema operacional: ele gerencia o hardware, aloca memória, escala processos na CPU e expõe chamadas de sistema (syscalls) para os programas. O kernel Linux sozinho não é utilizável pelo usuário final.

Uma distribuição empacota o kernel junto com tudo mais necessário para um sistema funcional:

Filosofia Unix

Tudo é Arquivo

Dispositivos (/dev/sda), processos (/proc/1234), sockets, pipes — tudo é acessível como arquivo. APIs uniformes simplificam programação e automação.

Faça Uma Coisa Bem

Cada programa tem responsabilidade única: grep busca texto, sort ordena, uniq remove duplicatas. Combinados, resolvem problemas complexos.

Comunique-se por Texto

Saída em texto simples (stdout) permite que qualquer ferramenta consuma a saída de outra. Sem formatos proprietários, sem dependências entre programas.

Pipelines Poderosos

O operador | conecta saída de um programa à entrada do próximo, criando fluxos de processamento de dados sem arquivos temporários.

Exemplo de pipeline real — listar usuários únicos de processos nginx em execução:

bash — pipeline Unix
ps aux | grep nginx | awk '{print $1}' | sort | uniq
