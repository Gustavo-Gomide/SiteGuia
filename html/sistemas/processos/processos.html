Sistemas Operacionais

Processos

O programa em execução — espaço de endereçamento isolado, recursos próprios e ciclo de vida gerenciado pelo kernel.

Um processo é a abstração central de qualquer sistema operacional moderno: trata-se de um programa em execução dotado de espaço de endereçamento privado, conjunto de registradores, descritores de arquivos abertos e credenciais de segurança. Ao contrário de um simples arquivo executável em disco, o processo existe apenas enquanto o kernel mantém sua estrutura de controle — o PCB — e o escalonador lhe concede ciclos de CPU.

Criação de Processos

No Unix/Linux todo processo nasce de outro processo por meio de fork(), que cria uma cópia quase idêntica do processo pai usando copy-on-write — as páginas de memória são compartilhadas até que um dos dois tente modificá-las. Em seguida, exec*() substitui a imagem do processo pelo novo programa sem criar outro PID.

C — fork() + exec() criando processo filho
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>

int main(void) {
    pid_t pid = fork();          /* duplica o processo */

    if (pid < 0) {
        perror("fork");
        return 1;
    }

    if (pid == 0) {
        /* filho: substitui imagem por /bin/ls */
        char *args[] = { "ls", "-l", "/tmp", NULL };
        execvp(args[0], args);
        perror("execvp");        /* só chega aqui se exec falhar */
        return 1;
    }

    /* pai: espera filho terminar */
    int status;
    waitpid(pid, &status, 0);
    printf("filho terminou com status %d\n", WEXITSTATUS(status));
    return 0;
}

Processo vs Thread

Threads são unidades de execução dentro de um processo. Compreender as diferenças é essencial para escolher o modelo correto de concorrência:

CritérioProcessoThread
Espaço de endereçamentoIsolado (privado)Compartilhado com as demais threads do mesmo processo
Troca de contextoPesada — flush de TLB, troca de page tableLeve — apenas salvar/restaurar registradores + stack pointer
CriaçãoLenta (fork copia estruturas do kernel)Rápida (pthread_create aloca apenas stack + TCB)
Falha isoladaSim — crash de um processo não afeta outrosNão — falha de uma thread pode derrubar todo o processo
ComunicaçãoExige IPC explícito (pipe, socket, shm)Direta via memória compartilhada (mas requer sincronização)
Custo de memóriaAlto (mínimo ~4 MB de espaço virtual)Baixo (tipicamente 8 KB–2 MB de stack por thread)
