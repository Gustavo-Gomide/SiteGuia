Sistemas Operacionais

Sincronização de Processos

Seção crítica, algoritmo de Peterson, primitivas de sincronização e os problemas clássicos de concorrência.

Quando múltiplos processos ou threads compartilham dados mutáveis, a sincronização garante que apenas um acesse a região crítica por vez — sem ela surgem race conditions e resultados não-determinísticos que são notoriamente difíceis de reproduzir e depurar. A teoria da sincronização define os requisitos formais e as ferramentas para satisfazê-los.

Seção Crítica e Requisitos Formais

Uma solução correta para o problema da seção crítica deve satisfazer exatamente três condições:

Exclusão Mútua

Se o processo Pi está executando na seção crítica, nenhum outro processo pode estar na seção crítica ao mesmo tempo. Esta é a condição fundamental — sem ela, dados são corrompidos.

Progresso

Se nenhum processo está na seção crítica e alguns desejam entrar, somente os processos que não estão na seção restante participam da decisão. A decisão não pode ser adiada indefinidamente.

Bounded Waiting

Existe um limite no número de vezes que outros processos podem entrar na seção crítica após um processo ter solicitado a entrada e antes de sua solicitação ser atendida — prevenindo starvation.

Algoritmo de Peterson

Uma solução puramente em software para dois processos, sem hardware especial. Usa duas variáveis compartilhadas: flag[2] (intenção de entrar) e turn (vez de quem):

C — Algoritmo de Peterson para 2 threads
#include <stdbool.h>
#include <stdatomic.h>

/* Variáveis compartilhadas */
static _Atomic bool flag[2] = { false, false }; /* flag[i]: processo i quer entrar */
static _Atomic int  turn    = 0;                /* de quem é a vez */

/* Executado pelo processo i (outro processo é j = 1 - i) */
void lock(int i) {
    int j = 1 - i;
    flag[i] = true;   /* anuncio intenção */
    turn    = j;      /* cedo a vez ao outro */
    /* Espero enquanto o outro quer entrar E é a vez dele */
    while (flag[j] && turn == j)
        ;  /* busy-wait */
}

void unlock(int i) {
    flag[i] = false;  /* libero seção crítica */
}

/* Uso:
   lock(0);
   // ... seção crítica do processo 0 ...
   unlock(0);
*/

/* LIMITAÇÕES:
   - Funciona apenas para 2 processos (existem generalizações)
   - Requer barreiras de memória em CPUs modernas que reordenam instruções
   - Busy-wait desperdiça CPU — use mutex do SO em produção            */

Problemas Clássicos de Concorrência

Produtor-Consumidor (Buffer Limitado)

Produtor insere itens num buffer de capacidade N; consumidor os retira. Sem sincronização, produtor pode sobrescrever items ou consumidor ler posição vazia. Solução: semáforo vazio (inicializado em N) e cheio (inicializado em 0).

Leitores-Escritores

Múltiplos leitores simultâneos são permitidos; escritores precisam de acesso exclusivo. Preferência para leitores pode causar starvation de escritores. Solução: contagem de leitores ativos com mutex + semáforo de escrita.

Jantar dos Filósofos

5 filósofos, 5 garfos, cada um precisa de dois garfos para comer. Pegar todos o garfo esquerdo ao mesmo tempo causa deadlock. Soluções: filósofo assimétrico, limite de 4 filósofos na mesa, ou adquirir ambos os garfos atomicamente.

C — Produtor-Consumidor com semáforos POSIX
#include <stdio.h>
#include <semaphore.h>
#include <pthread.h>

#define N 5   /* tamanho do buffer */
int buffer[N];
int in = 0, out = 0;

sem_t vazio;  /* slots disponíveis para produzir */
sem_t cheio;  /* itens disponíveis para consumir */
pthread_mutex_t mtx = PTHREAD_MUTEX_INITIALIZER;

void *produtor(void *arg) {
    for (int item = 0; item < 20; item++) {
        sem_wait(&vazio);          /* aguarda slot livre */
        pthread_mutex_lock(&mtx);
        buffer[in] = item;
        in = (in + 1) % N;
        pthread_mutex_unlock(&mtx);
        sem_post(&cheio);          /* sinaliza item disponível */
    }
    return NULL;
}

void *consumidor(void *arg) {
    for (int i = 0; i < 20; i++) {
        sem_wait(&cheio);          /* aguarda item disponível */
        pthread_mutex_lock(&mtx);
        int item = buffer[out];
        out = (out + 1) % N;
        pthread_mutex_unlock(&mtx);
        sem_post(&vazio);          /* libera slot */
        printf("consumido: %d\n", item);
    }
    return NULL;
}

int main(void) {
    sem_init(&vazio, 0, N);
    sem_init(&cheio, 0, 0);
    pthread_t tp, tc;
    pthread_create(&tp, NULL, produtor, NULL);
    pthread_create(&tc, NULL, consumidor, NULL);
    pthread_join(tp, NULL);
    pthread_join(tc, NULL);
    sem_destroy(&vazio);
    sem_destroy(&cheio);
}
