I/O — Software

Camadas de abstração, buffering, caching, page cache e modelos de I/O

O software de I/O organiza os dados entre hardware e aplicações com camadas de abstração que evitam que cada programa precise conhecer cada dispositivo. Buffering, caching, modelos assíncronos e a revolucionária interface io_uring definem como aplicações modernas exploram o máximo de desempenho de armazenamento e rede.

Buffering e Caching

Buffer e cache são conceitos distintos com objetivos diferentes: um buffer é uma área temporária para transferência de dados em trânsito (input está esperando a aplicação ler, ou output ainda não foi enviado ao dispositivo). Um cache é uma cópia de dados recentemente usados para evitar I/O repetido.

Page Cache do Linux

O VFS mantém um cache de páginas (4 KB cada) dos arquivos lidos do disco. Leituras subsequentes do mesmo arquivo são servidas da RAM sem I/O. O kernel usa toda RAM disponível como page cache — por isso free -h mostra "available" alta mesmo com "free" baixo.

Double Buffer

Dois buffers alternam: enquanto o kernel preenche um (aguarda o disco), a aplicação lê o outro. Evita que aplicação e kernel bloqueiem uma à outra. Usado em streaming de vídeo e captura de áudio.

Circular Buffer

Buffer em anel com ponteiros de leitura e escrita que avançam. Eficiente para produtores/consumidores a taxas diferentes. Usado em pipes Unix, buffers de log de kernel e filas de pacotes de rede.

Page Cache e Write-Back

Toda leitura de arquivo passa pelo page cache. Escritas com write() padrão são buffered: o dado vai para o page cache e o kernel decide quando fazer flush para o disco (write-back). Isso melhora performance mas cria um risco: dados em cache são perdidos em caso de queda sem flush.

fsync(fd)

Força flush imediato de todos os dados e metadados do arquivo para o dispositivo. Retorna somente quando o hardware confirma a escrita. Usado por bancos de dados antes de confirmar uma transação.

fdatasync(fd)

Como fsync, mas flush apenas dos dados (não atualiza mtime/atime desnecessariamente). Ligeiramente mais rápido e suficiente para garantir integridade dos dados de usuário.

O_DIRECT

Bypassa completamente o page cache. Dados vão direto do buffer do usuário para o dispositivo (e vice-versa). Exige alinhamento de buffer e tamanho de transferência. Usado por bancos de dados que gerenciam seu próprio cache (PostgreSQL, MySQL InnoDB).
