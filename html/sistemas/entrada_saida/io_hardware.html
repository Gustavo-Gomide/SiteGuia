Sistemas Operacionais

I/O — Hardware

Controladores, registradores, polling, interrupções, DMA e o ciclo completo de I/O

I/O é a parte mais lenta de um computador: um disco NVMe leva ~100 µs; um HDD, ~10 ms; uma requisição de rede pode demorar centenas de ms. Entender como hardware comunica com a CPU — via registradores, interrupções e DMA — é essencial para otimizar aplicações I/O-bound e projeto de drivers.

Controladores e Registradores de Dispositivo

Cada dispositivo de I/O tem um controlador — um chip que gerencia a operação do hardware e expõe uma interface de registradores para o processador. Os registradores básicos são:

Status Register

A CPU lê este registrador para saber se o dispositivo está ocupado (busy), pronto (ready), ou se ocorreu um erro. O kernel lê repetidamente no polling ou aguarda uma interrupção.

Control Register

A CPU escreve comandos aqui: iniciar leitura, iniciar escrita, reset, etc. O controlador executa a operação e atualiza o status register ao concluir.

Data-In / Data-Out

Registradores de dados. A CPU lê data-in para receber bytes do dispositivo; escreve em data-out para enviar bytes. Em transferências lentas, isso é feito byte a byte (programmed I/O). Em transferências rápidas, usa-se DMA.

DMA — Direct Memory Access

O controlador DMA transfere dados diretamente entre o dispositivo e a RAM sem envolver a CPU para cada byte. A CPU apenas configura a transferência (endereço de destino, tamanho, direção) e é interrompida somente ao final. Isso libera a CPU para executar outro processo durante a transferência.

pseudocódigo — setup de transferência DMA
// CPU configura o controlador DMA
dma_controller.source_address  = disk_buffer_phys_addr
dma_controller.dest_address    = kernel_page_phys_addr
dma_controller.transfer_count  = 512  // bytes (1 setor)
dma_controller.direction       = DEVICE_TO_MEMORY
dma_controller.control         = START_TRANSFER

// CPU fica livre — escalona outro processo
// ...
// Quando DMA terminar, dispositivo envia IRQ
// ISR (Interrupt Service Routine):
void dma_irq_handler() {
    if (dma_controller.status == TRANSFER_COMPLETE)
        wake_up(waiting_process)   // acorda o processo que fez o read()
}

// IOMMU: protege contra DMA attacks
// mapeia apenas o buffer autorizado no espaço de I/O virtual do dispositivo
