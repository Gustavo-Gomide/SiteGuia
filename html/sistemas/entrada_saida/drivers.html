Drivers de Dispositivo

A cola entre kernel genérico e hardware específico — módulos, tipos e debugging

Drivers são a cola entre o kernel genérico e hardware específico. Um driver traduz operações abstratas como read() e write() nos comandos concretos que o controlador do dispositivo entende. Um driver ruim rodando em kernel space pode travar o sistema inteiro com um kernel panic — por isso a qualidade e revisão de drivers são críticas.

Módulos do Kernel Linux

Drivers Linux são compilados como Kernel Objects (.ko) — módulos carregáveis e descarregáveis em tempo de execução sem precisar recompilar ou reiniciar o kernel. Isso mantém o kernel base pequeno e permite atualizar drivers sem boot.

bash — gerenciamento de módulos
# Listar módulos carregados (nome, tamanho, uso, dependentes)
lsmod

# Informações sobre um módulo específico
modinfo iwlwifi
# filename:       /lib/modules/6.8.0/kernel/drivers/net/wireless/intel/iwlwifi/iwlwifi.ko
# description:    Intel(R) Wireless WiFi driver for Linux
# depends:        cfg80211
# vermagic:       6.8.0-52-generic SMP preempt

# Carregar módulo com dependências resolvidas automaticamente
modprobe iwlwifi

# Descarregar módulo (falha se em uso)
modprobe -r iwlwifi

# Inserir módulo manualmente (sem resolver dependências)
insmod /path/to/meudriver.ko

# Remover módulo manualmente
rmmod meudriver

# Verificar módulos a carregar automaticamente no boot
cat /etc/modules

# Listar todos os módulos via /proc
cat /proc/modules | head -10

# Eventos de dispositivo em tempo real (plugar/desplugar USB)
udevadm monitor

Estrutura de um Driver de Char (simplificado)

C — esqueleto de driver de char device Linux
#include <linux/module.h>
#include <linux/fs.h>
#include <linux/uaccess.h>

#define DEVICE_NAME "meuchar"
#define MAJOR_NUM 240

// Implementação das operações de arquivo
static ssize_t dev_read(struct file *f, char __user *buf,
                        size_t len, loff_t *off) {
    char msg[] = "ola do driver\n";
    size_t msg_len = strlen(msg);
    if (*off >= msg_len) return 0;            // EOF
    if (copy_to_user(buf, msg + *off, msg_len))
        return -EFAULT;                        // erro ao copiar para user space
    *off += msg_len;
    return msg_len;
}

static ssize_t dev_write(struct file *f, const char __user *buf,
                         size_t len, loff_t *off) {
    printk(KERN_INFO "meuchar: %zu bytes recebidos\n", len);
    return len;
}

// Tabela de operações — VFS consulta esta struct para cada syscall
static struct file_operations fops = {
    .owner = THIS_MODULE,
    .read  = dev_read,
    .write = dev_write,
};

// Chamado ao carregar o módulo (insmod / modprobe)
static int __init meuchar_init(void) {
    int ret = register_chrdev(MAJOR_NUM, DEVICE_NAME, &fops);
    if (ret < 0) {
        printk(KERN_ALERT "Falha ao registrar char device: %d\n", ret);
        return ret;
    }
    printk(KERN_INFO "meuchar: driver carregado (major %d)\n", MAJOR_NUM);
    return 0;
}

// Chamado ao descarregar o módulo (rmmod)
static void __exit meuchar_exit(void) {
    unregister_chrdev(MAJOR_NUM, DEVICE_NAME);
    printk(KERN_INFO "meuchar: driver descarregado\n");
}

module_init(meuchar_init);
module_exit(meuchar_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Exemplo");
MODULE_DESCRIPTION("Driver de char device minimo");

Debugging e Problemas de Drivers

dmesg e journalctl -k

Logs do kernel com timestamps. Todo printk() do driver aparece aqui. dmesg -w monitora em tempo real. journalctl -k -f faz o mesmo com integração ao systemd.

Kernel Oops e Panic

Um oops é um erro recuperável no kernel (null pointer deref em driver): o kernel registra call stack e tenta continuar. Um panic é fatal: o kernel para. Ambos geram registros detalhados para post-mortem com decode_stacktrace.sh.

debugfs e /sys/kernel/debug

Drivers podem expor variáveis internas via debugfs, acessível em /sys/kernel/debug/. Permite inspecionar estados internos do driver em tempo real sem recompilar. Montado com mount -t debugfs none /sys/kernel/debug.

DKMS — Dynamic Kernel Module Support

Framework que recompila automaticamente módulos out-of-tree (drivers fora do kernel oficial) após atualização do kernel. Usado por drivers NVIDIA, VirtualBox, ZFS on Linux. Evita que uma atualização de kernel quebre drivers de terceiros.
