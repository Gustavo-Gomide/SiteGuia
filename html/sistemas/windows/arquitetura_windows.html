Arquitetura do Windows NT

O kernel do Windows NT é classificado como híbrido. O HAL isola diferenças de hardware, o Executive implementa todas as funções do kernel em modo privilegiado, e os subsistemas de usuário (Win32, WSL, POSIX) rodam em modo de usuário comunicando-se via LPC/ALPC.

Modelo de Objetos do Kernel

Objetos do Kernel NT (conceitual)
// Tudo no kernel NT é um objeto com:
// - tipo (OBJECT_TYPE)
// - header (OBJECT_HEADER) com refcount, handles, nome
// - corpo (tipo-específico)

// Tipos de objetos comuns:
// Process  — criado por CreateProcess()
// Thread   — criado por CreateThread()
// File     — criado por CreateFile()
// Event    — sincronização (Set/Wait/Pulse)
// Mutex    — exclusão mútua (wait é recursivo)
// Semaphore — semáforo contável
// Section  — memória mapeada em arquivo
// Token    — representação de identidade/privilege

// Ver objetos via Sysinternals WinObj ou Process Hacker
// Handles são referências ao Object Manager namespace
// Ex: \Device\HarddiskVolume3\Windows\notepad.exe

Segurança: Tokens e Integridade

Tokens de Acesso

Cada processo/thread tem um access token com SID do usuário, grupos, privilégios (SeDebugPrivilege, SeBackupPrivilege...) e nível de integridade.

Níveis de Integridade (IL)

Low (IE protected mode), Medium (apps normais), High (admin elevado), System (serviços), Trusted Installer; controle de escrita entre níveis.

UAC (User Account Control)

Split token: admin comum roda com token Medium; ações elevadas geram prompt pedindo token High.

LSASS

Local Security Authority Subsystem gerencia autenticação, Kerberos, NTLM e guarda hashes/tickets (alvo de mimikatz e credential dumping).
