Sistemas Operacionais

Active Directory

AD DS (Active Directory Domain Services) é o serviço de diretório da Microsoft para gerenciar identidades e recursos em redes corporativas. Define a estrutura lógica (Forest → Domain → OU) e fornece autenticação centralizada via Kerberos, aplicação de GPOs (Group Policy Objects) e resolução de nomes via DNS integrado.

Estrutura Lógica do AD

Active Directory — Hierarquia Lógica
Forest (ex: empresa.com)
└── Tree: empresa.com
    ├── Domain: empresa.com
    │   ├── OU: TI
    │   │   ├── OU: Servidores
    │   │   │   └── Computer: SRV-DC01
    │   │   └── OU: Workstations
    │   │       └── Computer: WKS-001
    │   ├── OU: RH
    │   │   └── User: joao.silva
    │   ├── OU: Financeiro
    │   │   └── User: maria.santos
    │   ├── Built-in Groups: Domain Admins, Enterprise Admins, Domain Users
    │   └── Site objects, GPOs, trusts
    └── Child Domain: filial.empresa.com (trust automático)

# Objetos principais:
# User    — conta de usuário com atributos (email, telefone, cargo...)
# Group   — Security groups (permissões) e Distribution groups (email)
# Computer — conta de máquina (trust channel com o domínio)
# OU      — unidade organizacional para delegar admin e aplicar GPO
# GPO     — Group Policy Object: políticas vinculadas a site/domain/OU

Autenticação Kerberos

Etapa Componente Descrição
1. AS-REQClient → KDC (AS)Cliente pede TGT usando hash da senha
2. AS-REPKDC (AS) → ClientKDC retorna TGT criptografado com chave do krbtgt
3. TGS-REQClient → KDC (TGS)Cliente pede Service Ticket para recurso específico
4. TGS-REPKDC (TGS) → ClientKDC retorna Service Ticket
5. AP-REQClient → ServiceCliente envia Service Ticket ao servidor de destino
6. AP-REPService → ClientServidor valida ticket e autentica o cliente

GPO — Group Policy

PowerShell — Group Policy
# GPOs são vinculadas a Sites, Domínios ou OUs
# Processamento: Local > Site > Domain > OU (filho sobrescreve pai)
# Refresh automático: 90 minutos +- random offset (usuário), 5 min (DC)

# Forçar atualização de GPO imediatamente
gpupdate /force

# Gerar relatório de GPO efetiva para um usuário/computador
gpresult /H C:\gpresult.html /User DOMAIN\joao.silva

# Via PowerShell (módulo GroupPolicy)
Get-GPO -All | Select-Object DisplayName, GpoStatus, ModificationTime
New-GPO -Name "Politica-TI-Segurança"
Get-GPOReport -Name "Politica-TI-Segurança" -ReportType HTML -Path "C:\gpo_relatorio.html"

# Configurações comuns via GPO:
# - Password Policy: Computer > Windows Settings > Security Settings > Account Policies
# - Software Restriction / AppLocker
# - Mapeamento de drives de rede
# - Configurações do Internet Explorer/Edge
# - Scripts de logon/logoff e startup/shutdown
# - Desktop background, wallpaper, screensaver

Gerenciamento com PowerShell

PowerShell — Active Directory
# Módulo: ActiveDirectory (RSAT ou Windows Server)
Import-Module ActiveDirectory

# Buscar usuários
Get-ADUser -Filter {Department -eq "TI"} -Properties Department, Manager |
  Select-Object Name, SamAccountName, Manager

# Criar usuário
New-ADUser -Name "Carlos Ferreira" -GivenName "Carlos" -Surname "Ferreira" `
  -SamAccountName "carlos.ferreira" -UserPrincipalName "carlos.ferreira@empresa.com" `
  -Path "OU=TI,DC=empresa,DC=com" -AccountPassword (Read-Host -AsSecureString) -Enabled $true

# Adicionar a grupo
Add-ADGroupMember -Identity "GRP-TI-Admins" -Members "carlos.ferreira"

# Encontrar usuários que não logam há 90 dias
$data = (Get-Date).AddDays(-90)
Get-ADUser -Filter {LastLogonDate -lt $data -and Enabled -eq $true} -Properties LastLogonDate |
  Select-Object Name, SamAccountName, LastLogonDate | Sort-Object LastLogonDate

# Desbloquear conta
Unlock-ADAccount -Identity "joao.silva"

# Ver todos os computadores no domínio
Get-ADComputer -Filter * -Properties OperatingSystem, LastLogonDate |
  Select-Object Name, OperatingSystem, LastLogonDate | Sort-Object LastLogonDate -Descending
