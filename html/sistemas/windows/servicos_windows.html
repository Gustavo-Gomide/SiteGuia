Sistemas Operacionais

Serviços do Windows

Serviços são processos de longa duração que rodam em segundo plano, geralmente sem interface gráfica. Podem ser iniciados automaticamente no boot, executados sob contas privilegiadas e gerenciados via SCM (Service Control Manager). São os equivalentes Windows dos daemons Unix.

Tipos e Estados de Serviços

Estado/Tipo Descrição
RunningServiço em execução normal
StoppedServiço parado; não está consumindo recursos
PausedSuspenso temporariamente (nem todos suportam)
StartPending / StopPendingTransição entre estados
AutomaticInicia no boot automaticamente
Automatic (Delayed Start)Inicia no boot depois de outros serviços críticos
ManualInicia apenas quando solicitado por app ou usuário
DisabledNão pode ser iniciado, mesmo manualmente
Win32OwnProcessProcesso dedicado ao serviço (svchost.exe ou processo próprio)
Win32ShareProcessCompartilha processo svchost.exe com outros serviços (economia de memória)
Kernel DriverDriver de dispositivo (ex: NTFS, Tcpip, filtros de segurança)

Gerenciamento via PowerShell

PowerShell — Gerenciamento de Serviços
# Listar todos os serviços
Get-Service | Sort-Object Status, Name | Format-Table -AutoSize

# Filtrar por estado
Get-Service | Where-Object {$_.Status -eq 'Running'} | Measure-Object

# Iniciar/parar/reiniciar serviço
Start-Service -Name "wuauserv"    # Windows Update
Stop-Service  -Name "wuauserv"
Restart-Service -Name "Spooler"   # Print Spooler

# Configurar tipo de inicialização
Set-Service -Name "wuauserv" -StartupType Manual

# Ver dependências
(Get-Service -Name "Spooler").DependentServices
(Get-Service -Name "Spooler").ServicesDependedOn

# Detalhes de um serviço (incluindo usuário e executável)
Get-CimInstance -ClassName Win32_Service -Filter "Name='Spooler'" |
  Select-Object Name, PathName, StartName, StartMode, State

# Módulo SC.EXE (legado, útil em scripts batch)
sc query type= all state= all
sc start wuauserv
sc config "wuauserv" start= disabled

Contas de Serviço

SYSTEM (LocalSystem)

Conta mais privilegiada; acesso irrestrito ao sistema local; não tem senha; não acessa rede como usuário de domínio; evitar para serviços de terceiros.

LOCAL SERVICE

Privilégios mínimos locais; acessa rede como conta anônima; equivale a conta de usuário comum para recursos locais.

NETWORK SERVICE

Similar ao LOCAL SERVICE, mas acessa rede com credenciais do computador (conta de máquina no domínio); usado por IIS, RPC.

Managed Service Account (gMSA)

Conta AD com senha rotacionada automaticamente pelo AD; ideal para serviços em cluster sem precisar gerenciar senhas manualmente.

Conta de domínio dedicada

Máximo controle sobre permissões; exige gerenciamento de senha manual ou via LAPS.

Troubleshooting de Serviços

PowerShell — Diagnóstico de Serviços
# Ver eventos relacionados a um serviço
Get-WinEvent -LogName System | Where-Object {$_.Message -like "*Spooler*"} | Select-Object -First 20 TimeCreated, Message

# Verificar por que um serviço falhou
sc qfailure Spooler    # política de recuperação em falha

# Configurar recuperação automática
sc failure Spooler reset= 86400 actions= restart/5000/restart/10000/reboot/20000

# Dependências que impedem inicialização
(Get-Service "MSDependentService").ServicesDependedOn | Get-Service

# Verificar EXE do serviço no Registro
Get-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Services\Spooler" | Select-Object ImagePath, ObjectName, Start

# Listar todos os serviços do svchost.exe (agrupamentos)
Get-CimInstance Win32_Service | Where-Object {$_.PathName -like "*svchost*"} | Group-Object {($_.PathName -split '-k ')[1]} | Sort-Object Count -Descending
