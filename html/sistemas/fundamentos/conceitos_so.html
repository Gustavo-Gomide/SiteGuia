Sistemas Operacionais

Conceitos de Sistemas Operacionais

Kernel, espaço de usuário, syscalls, multiprogramação e interrupções

Um sistema operacional é o software responsável por abstrair o hardware e fornecer uma interface segura e previsível para os programas. Ele se divide em duas camadas fundamentais: o kernel, que executa com privilégio total sobre a máquina, e o espaço de usuário, onde os processos comuns residem sem acesso direto ao hardware. Essa separação é aplicada pelo próprio processador por meio dos anéis de proteção, garantindo que uma aplicação falha não possa comprometer o sistema inteiro.

Kernel e Modo Protegido

A arquitetura x86 define quatro anéis de proteção (ring 0 a ring 3). Apenas o ring 0 tem acesso irrestrito a instruções privilegiadas, portas de I/O e gerenciamento de memória. O SO moderno tipicamente usa só ring 0 (kernel) e ring 3 (aplicações), deixando os ring 1 e 2 historicamente para drivers de alguns sistemas legados.

Nível Nome Pode fazer Exemplos
Ring 0 Kernel Mode Qualquer instrução; acesso direto a hardware, memória física, I/O Kernel Linux, Windows NT Kernel
Ring 1 Driver Legacy Instruções de E/S limitadas; sem acesso direto a memória física Drivers antigos OS/2, VMware ring-1
Ring 2 Driver Legacy Similar ao ring 1, maior restrição que ring 0 Raramente usado em SO modernos
Ring 3 User Mode Instruções não-privilegiadas; precisa de syscall para acessar recursos Todas as aplicações de usuário

A proteção de memória é aplicada pela MMU (Memory Management Unit): cada processo enxerga apenas seu próprio espaço de endereçamento virtual. Tentativas de acesso fora desse espaço geram uma page fault ou segmentation fault, que o kernel captura e normalmente encerra o processo ofensor.

Chamadas de Sistema (Syscalls)

Uma syscall é o mecanismo pelo qual um programa em ring 3 solicita um serviço ao kernel. O processador realiza um trap — transição controlada para ring 0 — via instrução syscall (x86-64) ou int 0x80 (x86 legado). Após executar o serviço, o kernel retorna ao espaço do usuário restaurando o estado dos registradores.

Fluxo de uma chamada read() — pseudocódigo
/* Userspace */
ssize_t n = read(fd, buf, 128);   // wrapper da libc

// A libc faz:
//   RAX = 0  (número da syscall read no Linux x86-64)
//   RDI = fd
//   RSI = buf
//   RDX = 128
//   syscall          ← trap para ring 0

/* Kernel */
// Salva registradores do processo
// Despacha para sys_read() via tabela syscall_table[]
// Copia dados do buffer do kernel para buf do usuário
// Restaura registradores, retorna para ring 3

/* De volta ao userspace */
// RAX contém bytes lidos (ou -errno em caso de erro)

write1Escreve bytes em um descritor de arquivo
open2Abre ou cria um arquivo, retorna fd
fork57Cria processo filho duplicando o pai

Multiprogramação e Multitarefa

Multiprogramação significa manter vários programas na memória ao mesmo tempo, aproveitando o tempo que um processo fica bloqueado em I/O para executar outro. Time-sharing vai além: divide o processador em fatias de tempo (time slices) e alterna entre processos tão rápido que o usuário percebe paralelismo. A diferença entre processo e thread é que um processo tem espaço de memória isolado, enquanto threads dentro do mesmo processo compartilham memória.

Modelo Descrição Preempção Exemplos
Batch Executa um job por vez até o fim Nenhuma IBM OS/360 Job Control
Multiprogramação Vários jobs na memória; CPU troca ao bloquear Por I/O block CTSS, OS/MFT
Multitarefa Cooperativa Processo voluntariamente cede a CPU Voluntária Windows 3.x, Mac OS 9
Multitarefa Preemptiva Kernel interrompe processo ao fim do time slice Por timer interrupt Linux, Windows NT, macOS

Interrupções e Exceções

Interrupções são sinais assíncronos que suspendem a execução corrente e desviam o processador para uma rotina de tratamento (Interrupt Service Routine — ISR). O kernel mantém uma tabela de descritores de interrupção (IDT em x86) que mapeia cada número de vetor para o endereço da ISR correspondente.

Interrupções de Hardware

Originadas por dispositivos físicos: teclado, disco, placa de rede, timer. São assíncronas em relação ao fluxo do programa. O controlador de interrupções (APIC) serializa e encaminha ao processador.

Interrupções de Software (Traps)

Geradas intencionalmente por instruções como int 0x80 ou syscall. São síncronas — acontecem exatamente no ponto onde a instrução é executada. Usadas para implementar chamadas de sistema.

Exceções

Condições de erro detectadas pelo próprio processador: divisão por zero, acesso inválido à memória (page fault), instrução ilegal. O kernel decide se o processo pode se recuperar ou deve ser encerrado (sinal SIGSEGV, SIGFPE, etc.).

IDT / IVT

A Interrupt Descriptor Table (modo protegido) ou Interrupt Vector Table (modo real) contém até 256 entradas, cada uma apontando para uma ISR. O kernel preenche essa tabela durante a inicialização e protege suas entradas de modificação pelo userspace.
