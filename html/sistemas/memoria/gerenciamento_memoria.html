Sistemas Operacionais

Gerenciamento de Memória

Como o sistema operacional controla, protege e distribui a memória entre processos

O gerenciador de memória garante que cada processo tenha memória suficiente, isolada e protegida. Sem ele, processos invadiriam o espaço uns dos outros, corrompendo dados e tornando o sistema instável. É um dos subsistemas mais críticos de qualquer SO moderno.

Espaço de Endereçamento Virtual

Cada processo acredita ter acesso a um espaço de endereçamento enorme — em sistemas de 64 bits, teoricamente 264 bytes (embora arquiteturas x86-64 usem apenas 48 bits, resultando em 256 TB de espaço virtual por processo). Esse espaço é mapeado para a RAM física pela MMU (Memory Management Unit), um componente de hardware do processador.

O espaço de endereçamento virtual de um processo típico é dividido em regiões bem definidas:

Segmento de Código (.text)

Contém as instruções executáveis do programa. Geralmente mapeado como somente leitura e executável para evitar modificação acidental.

Segmento de Dados (.data / .bss)

Variáveis globais e estáticas. .data contém variáveis inicializadas; .bss contém variáveis não inicializadas (zero-filled pelo kernel).

Heap

Memória alocada dinamicamente em tempo de execução via malloc(), new, etc. Cresce para cima (endereços maiores) conforme necessário.

Stack (Pilha)

Armazena variáveis locais, parâmetros de funções e endereços de retorno. Cresce para baixo (endereços menores). Tamanho limitado (tipicamente 8 MB por thread no Linux).

A distinção entre endereço virtual (o que o programa vê) e endereço físico (posição real na RAM) é fundamental. O programador trabalha exclusivamente com endereços virtuais; a MMU realiza a tradução em hardware a cada acesso à memória.

Técnicas de Gerenciamento

Ao longo da história dos sistemas operacionais, diversas técnicas foram desenvolvidas para gerenciar a memória física. A tabela abaixo resume as principais abordagens:

Técnica Descrição Fragmentação Proteção Complexidade
Partições Fixas Memória dividida em blocos de tamanho fixo na inicialização Interna (sobra dentro da partição) Por partição (rudimentar) Muito baixa
Partições Variáveis Partições alocadas sob demanda com tamanho exato do processo Externa (buracos entre partições) Registradores base/limite Baixa
Paginação Divide memória em páginas fixas (tipicamente 4 KB) Somente interna (mínima) Por página (R/W/X bits) Média (TLB mitiga)
Segmentação Divide em segmentos lógicos (código, dados, pilha) Externa Por segmento (DPL, tipo) Média
Segmentação + Paginação Combinação dos dois — usada no x86 original Somente interna Dupla (segmento + página) Alta

MMU e Tradução de Endereços

A MMU (Memory Management Unit) é o componente de hardware responsável pela tradução de endereços virtuais em físicos. Em x86-64, o registrador CR3 aponta para o diretório de páginas de nível superior (PML4) do processo atual. A cada troca de contexto, o kernel atualiza CR3 para apontar para a tabela do novo processo.

A TLB (Translation Lookaside Buffer) é uma cache interna da MMU que armazena traduções recentes. Em workloads típicos, a TLB tem taxa de acerto superior a 99%, tornando a tradução de endereços praticamente gratuita. Um TLB miss força um page table walk — o hardware percorre os 4 níveis da tabela de páginas na RAM, o que custa dezenas de ciclos.

Processo de tradução passo a passo (x86-64, 4 níveis):

Tradução de Endereço Virtual — x86-64 (pseudocódigo)
; Endereço virtual de 48 bits: bits [47:39] = PML4 index
;                               bits [38:30] = PDPT index
;                               bits [29:21] = PD index
;                               bits [20:12] = PT index
;                               bits [11: 0] = offset dentro da página (4 KB)

1. CR3 → endereço físico da tabela PML4 do processo atual
2. PML4[bits 47:39] → endereço físico da tabela PDPT
3. PDPT[bits 38:30] → endereço físico da tabela PD
4. PD[bits 29:21]   → endereço físico da tabela PT
5. PT[bits 20:12]   → PTE (Page Table Entry)
   - Se bit Present = 0  → Page Fault (página não está na RAM)
   - Se bit Present = 1  → endereço físico do frame
6. Endereço físico = frame_base + offset (bits 11:0)
7. Resultado armazenado na TLB para acessos futuros

Monitoramento de Memória no Linux

O Linux expõe informações detalhadas sobre uso de memória via sistema de arquivos /proc e ferramentas de linha de comando:

free -h

Exibe total, usado, livre, buffers/cache e swap em formato legível. O campo available mostra memória realmente disponível para novos processos (inclui cache recuperável).

vmstat

Estatísticas de memória, I/O e CPU em tempo real. Campos si (swap in) e so (swap out) indicam pressão de memória. Alto si/so sinaliza thrashing.

cat /proc/PID/maps

Lista todas as regiões de memória virtual de um processo: endereços, permissões, offset, dispositivo e arquivo mapeado. Essencial para debugging e análise de segurança.

pmap PID

Versão formatada de /proc/PID/maps com totais por região. pmap -x PID mostra RSS (páginas residentes na RAM) vs. tamanho virtual.

/proc/meminfo — campos anotados
