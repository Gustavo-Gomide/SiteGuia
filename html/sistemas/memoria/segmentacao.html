Sistemas Operacionais

Segmentação

Divisão lógica do espaço de endereçamento em regiões com significado semântico

Segmentação divide o espaço de endereçamento em regiões lógicas com semântica distinta — código separado de dados, pilha isolada do heap. Precede historicamente a paginação e ainda é visível nas entranhas do x86, embora a arquitetura x86-64 moderna a tenha essencialmente abandonado em favor do modelo flat com paginação pura.

Segmentos e Descritores

Em arquiteturas com segmentação, cada segmento é descrito por uma entrada na GDT (Global Descriptor Table) ou LDT (Local Descriptor Table). O processador x86 em modo protegido lê esses descritores para validar cada acesso à memória.

Os registradores de segmento do x86 (CS, DS, SS, ES, FS, GS) não armazenam endereços diretamente — eles carregam seletores, que são índices na GDT/LDT mais bits de privilégio.

Campo do Descritor Tamanho Descrição
Base 32 bits Endereço linear inicial do segmento. Em x86-64, ignorado (sempre 0 exceto FS/GS)
Limit 20 bits Tamanho do segmento menos 1. Se bit G=1, granularidade de 4 KB (limite máx ~4 GB)
Type 4 bits Código/dados, leitura/escrita, expand-down, acesso
DPL (Descriptor Privilege Level) 2 bits Nível de privilégio: 0 = kernel, 3 = usuário. Acesso negado se CPL > DPL
G (Granularidade) 1 bit 0 = limite em bytes; 1 = limite em páginas de 4 KB
P (Present) 1 bit 1 = segmento válido na memória, 0 = segment not present fault
L (Long Mode) 1 bit 1 = segmento de código 64-bit (x86-64)

Segmentação no x86: Real Mode vs Protected Mode

Real Mode (8086)

Endereço físico = segmento × 16 + offset. Nenhuma proteção — qualquer programa pode acessar qualquer endereço. Limite de 1 MB de memória endereçável. Ainda usado no boot do PC antes do kernel assumir.

Protected Mode (80286+)

Registradores de segmento carregam seletores na GDT. CPU verifica DPL em cada acesso — processo em anel 3 não pode acessar segmentos de anel 0. Proteção real entre processos e kernel.

Long Mode (x86-64)

Segmentação essencialmente desabilitada: base=0, limit=máximo para CS, DS, SS, ES. Apenas FS e GS mantêm base configurável — usados para TLS (Thread Local Storage) e acesso a estruturas PCB no kernel.

FS/GS como TLS

Em Linux x86-64, FS aponta para o bloco TLS da thread atual. GS aponta para o per-CPU data do kernel. Valores configurados via syscalls arch_prctl(ARCH_SET_FS) e wrmsrl(MSR_GS_BASE).

Segmentação vs Paginação

Critério Segmentação Paginação
Unidade de alocação Variável (tamanho do segmento) Fixa (tipicamente 4 KB)
Fragmentação Externa (buracos entre segmentos) Interna (sobra dentro da última página)
Proteção lógica Por região semântica (código, dados) Por página individual (R/W/X)
Visibilidade para o programador Explícita (segmento:offset) Transparente (endereço linear)
Suporte a disco (swap) Difícil — segmentos de tamanho variável Natural — páginas de tamanho fixo
Uso atual Quase abandonado (exceto FS/GS para TLS) Universal em SOs modernos

Segmentação + Paginação

O Intel x86 original (80386) combinava segmentação e paginação em um modelo de dois estágios. O endereçamento seguia esta pipeline:

Pipeline de Tradução de Endereços no x86 com Segmentação + Paginação
Estágio 1 — Segmentação:
  Endereço Lógico (seletor:offset)
    → CPU lê descritor da GDT usando seletor
    → Verifica privilégio (DPL vs CPL)
    → Verifica limite (offset ≤ limit)
    → Endereço Linear = base + offset

Estágio 2 — Paginação:
  Endereço Linear (32 bits)
    → PD[bits 31:22] → Page Directory Entry
    → PT[bits 21:12] → Page Table Entry
    → Verifica bits P, R/W, U/S
    → Endereço Físico = frame_base + bits[11:0]

No Linux (flat model):
  base = 0, limit = 0xFFFFFFFF para todos os segmentos
  → Estágio 1 é no-op: endereço lógico = endereço linear
  → Apenas paginação faz trabalho real

O Linux adotou o modelo flat — todos os segmentos com base 0 e limite máximo — tornando a segmentação transparente. Isso garante portabilidade para arquiteturas (ARM, RISC-V) que não possuem segmentação.

Compare com a abordagem de blocos fixos em Paginação e entenda como fragmentação afeta ambas as técnicas em Fragmentação de Memória.

"Segmentação foi a primeira tentativa de dar semântica à memória — paginação foi a tentativa de torná-la eficiente. O x86-64 sepultou o conflito escolhendo a eficiência."
