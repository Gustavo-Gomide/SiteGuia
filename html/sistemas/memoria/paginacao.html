Sistemas Operacionais

Paginação

A técnica que elimina a fragmentação externa e sustenta toda a memória virtual moderna

Paginação divide a memória em blocos fixos chamados páginas, eliminando a fragmentação externa que assolava as técnicas anteriores. É a base sobre a qual toda a memória virtual moderna é construída, permitindo que processos enxerguem um espaço de endereçamento contíguo mesmo que suas páginas estejam espalhadas por toda a RAM física.

Conceitos Fundamentais

Os termos essenciais da paginação formam um vocabulário preciso que todo desenvolvedor de sistemas deve dominar:

Página (Page)

Bloco de memória virtual de tamanho fixo. O tamanho padrão em x86-64 é 4 KB (4096 bytes), definido pelo hardware. Existe também suporte a huge pages de 2 MB e 1 GB.

Frame (Quadro de Página)

Bloco equivalente na RAM física. Um frame tem exatamente o mesmo tamanho que uma página. A tabela de páginas mapeia páginas virtuais para frames físicos.

Page Table (Tabela de Páginas)

Estrutura de dados mantida pelo kernel para cada processo. Cada entrada (PTE) mapeia uma página virtual a um frame físico e armazena bits de controle.

Bits de Controle da PTE

Present: página está na RAM. Dirty: página foi modificada. Accessed: página foi lida/escrita recentemente. Invalid: endereço ilegal — gera segfault.

Estrutura de um PTE em x86-64 (64 bits)
Bit  0     — Present (P): 1 = página na RAM, 0 = page fault ao acessar
Bit  1     — Read/Write (R/W): 0 = somente leitura, 1 = leitura e escrita
Bit  2     — User/Supervisor (U/S): 0 = apenas kernel, 1 = espaço de usuário
Bit  3     — Write-Through (PWT): política de cache write-through
Bit  4     — Cache Disable (PCD): desabilita cache para este frame
Bit  5     — Accessed (A): CPU seta em 1 quando a página é acessada
Bit  6     — Dirty (D): CPU seta em 1 quando a página é escrita
Bit  7     — Page Size (PS): 1 = huge page (2 MB quando no PD)
Bits 8-11  — Disponíveis para uso pelo SO (ex: copy-on-write flag)
Bits 12-51 — Physical Frame Number (PFN): endereço físico do frame (alinhado a 4 KB)
Bits 52-62 — Disponíveis para uso pelo SO
Bit  63    — No-Execute (NX): 1 = CPU não pode executar código desta página

Paginação Multinível (x86-64)

Uma tabela de páginas de nível único para um espaço de 64 bits seria impraticável — precisaria de terabytes de memória só para a tabela. A solução é usar 4 níveis de tabelas (ou 5 no modo LA57), onde cada nível só é alocado quando necessário.

PML4 — Page Map Level 4

Nível mais alto. 512 entradas, cada uma cobrindo 512 GB do espaço virtual. O registrador CR3 aponta para esta tabela. Há somente um PML4 por processo.

PDPT — Page Directory Pointer Table

512 entradas, cada uma cobrindo 1 GB. Alocada sob demanda (só criada se o processo mapear memória naquela faixa de 512 GB).

PD — Page Directory

512 entradas, cada uma cobrindo 2 MB. Uma entrada com bit PS=1 mapeia diretamente para uma huge page de 2 MB, pulando o próximo nível.

PT — Page Table

512 entradas (PTEs), cada uma mapeando uma página de 4 KB para um frame físico. É o nível folha da hierarquia para páginas normais.

Decomposição de endereço virtual de 48 bits (x86-64)
Endereço virtual: 0x00007FFE_ABCD1234

Bit layout (48 bits utilizados em modo 4-level):
  [63:48] — Sign extension (bits idênticos ao bit 47)
  [47:39] — PML4 index  = bits 47-39 = 0x00 (0)
  [38:30] — PDPT index  = bits 38-30 = 0x1FF (511)
  [29:21] — PD index    = bits 29-21 = 0x155 (341)
  [20:12] — PT index    = bits 20-12 = 0x1CD (461)
  [11: 0] — Page offset = bits 11-0  = 0x234 (564 bytes dentro da página)

Por que 4 níveis suportam 256 TB de espaço virtual:
  512 (PML4) × 512 (PDPT) × 512 (PD) × 512 (PT) × 4 KB = 256 TB

TLB — Translation Lookaside Buffer

A TLB é uma cache de tradução de endereços embutida no processador. Sem ela, cada acesso à memória exigiria 4 acesso adicionais à RAM (um por nível da tabela de páginas), tornando o sistema impraticável.

Conceito Detalhes
Hit rate típico ~99% em workloads comuns — tradução custa ~1 ciclo extra
TLB miss Hardware percorre todos os 4 níveis da tabela (~50-100 ciclos)
TLB flush Ocorre ao escrever em CR3 (troca de contexto de processo). Invalida todas as entradas — custo alto por isso processos compartilham kernel space com ASID (Address Space ID)
Huge Pages (2 MB) Uma entrada TLB cobre 512× mais memória — reduz drasticamente pressão na TLB para alocações grandes
Huge Pages (1 GB) Uma entrada TLB cobre 256.000× mais que páginas normais — ideal para bancos de dados e VMs

Para habilitar huge pages no Linux:

Huge Pages no Linux
# Ver configuração atual de huge pages
cat /proc/sys/vm/nr_hugepages

# Alocar 512 huge pages de 2 MB (= 1 GB reservado)
echo 512 | sudo tee /proc/sys/vm/nr_hugepages

# Verificar uso
cat /proc/meminfo | grep -i huge

# Transparent Huge Pages (THP) — habilitado por padrão no Linux
cat /sys/kernel/mm/transparent_hugepage/enabled
# [always] madvise never  → 'always' = THP ativo automaticamente

Proteção com Paginação

Cada PTE contém bits de proteção que o hardware verifica em todo acesso à memória. Violações geram uma exceção de proteção que o kernel converte em SIGSEGV para o processo:

Bit R/W (Read/Write)

Controla se a página pode ser escrita. Páginas de código (.text) são somente leitura. Fundamental para o mecanismo de Copy-on-Write no fork().

Bit U/S (User/Supervisor)

Separa espaço de usuário do espaço do kernel. Páginas do kernel têm U/S=0 — um processo em modo usuário que tentar acessá-las recebe imediatamente um fault.

Bit NX (No-Execute)

Impede a CPU de executar código em páginas de dados (heap, stack). Mitigação crucial contra ataques de buffer overflow que injetam shellcode. Implementado via bit 63 do PTE.

Granularidade por Página

Proteção aplicada com granularidade de 4 KB — muito mais fino que segmentação. Permite, por exemplo, marcar páginas individuais da stack como somente leitura para detectar stack overflow.

Entenda como as páginas são carregadas do disco sob demanda em Memória Virtual e como o SO decide quais páginas despejar em Algoritmos de Substituição.

"A TLB é a razão pela qual paginação a 4 níveis é eficiente — sem ela, cada acesso à memória custaria 5 acessos à RAM."
