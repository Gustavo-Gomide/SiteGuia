Sistemas Operacionais

NTFS

New Technology File System: MFT, journaling, ACLs, compressão e recursos avançados

NTFS (New Technology File System) é o sistema de arquivos padrão do Windows desde o Windows NT 3.1. Projetado como substituto robusto do FAT, ele oferece journaling via $LogFile, permissões ACL por arquivo, compressão transparente, criptografia EFS e suporte a volumes de vários terabytes. É significativamente mais complexo que o FAT, mas muito mais confiável e seguro.

Arquitetura NTFS — MFT

Todo o volume NTFS é dividido em clusters. O coração do NTFS é a MFT (Master File Table) — uma tabela que contém ao menos uma entry (record) para cada arquivo e diretório do volume, incluindo os próprios metaarquivos do sistema.

Cada MFT record (1 KB por padrão) é uma coleção de atributos:

$STANDARD_INFORMATION

Timestamps (criação, modificação, acesso, alteração MFT), flags (read-only, hidden, system), contador de hard links.

$FILE_NAME

Nome do arquivo em Unicode, tamanho alocado, referência para o diretório pai. Um arquivo pode ter múltiplos atributos $FILE_NAME (para nome longo e nome 8.3 legado).

$DATA

Conteúdo do arquivo. Se o arquivo for pequeno (menos de ~900 bytes), os dados ficam inline na própria MFT record (resident). Arquivos maiores usam runs (extent lists) apontando para clusters no disco.

$SECURITY_DESCRIPTOR

ACL do arquivo: lista de ACEs (Access Control Entries) com permissões por usuário/grupo. Centralizado em $Secure para deduplicação de descritores idênticos.

Metaarquivo NTFSFunção
$MFTA própria Master File Table (MFT record 0)
$MFTMirrBackup das primeiras 4 records da MFT (redundância)
$LogFileJournal de transações para recuperação em caso de falha
$VolumeNome do volume, versão NTFS, flags (dirty bit)
$AttrDefTabela de definições de tipos de atributos
$BitmapMapa de bits indicando clusters livres e ocupados
$BootSetor de boot do volume (BPB, bootstrap code)
$SecureBanco de dados de security descriptors (ACLs)

Journaling e Recuperação

O $LogFile é um journal circular que registra transações de metadados (alocação de clusters, atualização da MFT, atualização de diretorios). Antes de qualquer alteração estrutural, o kernel NTFS escreve a intenção no $LogFile. Após um crash, o chkdsk replaya ou desfaz transações incompletas em segundos — sem precisar varrer todo o volume.

O que o journaling protege

Falhas de metadados: MFT corrompida, diretório orphan, cluster bitmap inconsistente. Após um replay bem-sucedido, a estrutura do volume está garantidamente consistente.

O que o journaling nao protege

Os dados brutos do arquivo. Se uma escrita de dados foi interrompida no meio, o arquivo pode estar com conteúdo parcial — mas a estrutura do FS estará correta. Para garantia de dados, é necessário FlushFileBuffers() ou abertura com FILE_FLAG_WRITE_THROUGH.

Permissões NTFS e Herança

Permissões NTFS são definidas por ACEs (Access Control Entries) em uma ACL (Access Control List). Cada ACE especifica um SID (usuário ou grupo), um conjunto de direitos de acesso e se é Allow ou Deny.

Permissão Arquivos Diretórios
Full ControlLer, escrever, executar, alterar permissoes, deletarTodos os acima + apagar subpastas e arquivos
ModifyLer, escrever, executar, deletarLer, criar, deletar arquivos e subpastas
Read & ExecuteLer e executarListar conteudo e acessar subpastas
ReadLer conteudo e atributosListar conteudo
WriteCriar/sobrescrever, alterar atributosCriar arquivos e subpastas

Por padrão, permissões são herdadas do diretório pai. Uma ACE pode ser configurada como "não herdar" para quebrar a herança. Permissões SMB de compartilhamento de rede são avaliadas em conjunto com as permissões NTFS — o resultado é o mais restritivo dos dois.
