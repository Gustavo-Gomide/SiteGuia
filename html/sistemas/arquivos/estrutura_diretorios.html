Estrutura de Diretórios

FHS, pseudo-filesystems, montagem de volumes e comparação Linux vs Windows

A hierarquia de diretórios define onde cada tipo de dado deve residir no sistema de arquivos, facilitando administração, automação e portabilidade de scripts. O padrão FHS (Filesystem Hierarchy Standard) codifica essa convenção para sistemas Unix-like, enquanto o Windows usa uma estrutura centrada em letras de drive e no Registry.

/proc e /sys — Pseudo-Filesystems

/proc expoe processos e parametros do kernel como arquivos de texto. Cada processo tem um subdiretorio /proc/PID/ com arquivos como maps (mapa de memoria virtual), fd/ (file descriptors abertos), status e cmdline. Arquivos como /proc/meminfo e /proc/cpuinfo expõem hardware ao espaco de usuario sem syscalls dedicadas.

/sys (sysfs) expõe a arvore de dispositivos do kernel: barramento, drivers, e atributos configuráveis. A filosofia Unix — "tudo é arquivo" — significa que parametros de kernel podem ser lidos e escritos com cat e echo.

bash — lendo informações via /proc
# Informações de memória
cat /proc/meminfo | head -10

# CPUs disponíveis
cat /proc/cpuinfo | grep "model name" | uniq

# Mapa de memória do processo 1234
cat /proc/1234/maps

# File descriptors abertos pelo processo 1234
ls -la /proc/1234/fd

# Parâmetro do kernel: tamanho máximo de arquivo de core
cat /proc/sys/kernel/core_pattern

# Alterar limite de arquivos abertos via sysctl / /proc
echo 1000000 > /proc/sys/fs/file-max

Montagem de Volumes

No Linux, todos os sistemas de arquivos sao montados em um ponto da arvore unica. O comando mount anexa um volumeao diretorio especificado. O arquivo /etc/fstab define montagens persistentes que ocorrem no boot.

bash — mount e fstab
# Montar um disco manualmente
mount -t ext4 /dev/sdb1 /mnt/dados

# Ver todas as montagens ativas
mount | column -t

# /etc/fstab — formato: dispositivo ponto-de-montagem tipo opções dump pass
# Usando UUID (preferível a /dev/sdX, que pode mudar)
UUID=a1b2c3d4-...  /home  ext4  defaults,noatime  0  2

# Opções comuns de montagem
# ro         — somente leitura
# noexec     — impede execução de binários
# nosuid     — ignora bits setuid/setgid
# noatime    — não atualiza atime em leitura (desempenho)

# Bind mount — redireciona /origem para /destino
mount --bind /var/lib/docker/volumes /mnt/volumes

# Namespace de montagem (containers): cada namespace tem sua própria árvore
unshare --mount bash
