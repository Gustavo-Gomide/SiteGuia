Sistemas Operacionais

ext4

O sistema de arquivos padrão do Linux: extents, journaling e confiabilidade comprovada

ext4 é a quarta evolução da família ext, dominante em servidores Linux e desktops Ubuntu/Debian. Introduziu extents (blocos contíguos), delayed allocation, journaling aprimorado e suporte a volumes de até 1 exabyte, combinando simplicidade operacional com confiabilidade testada por décadas em sistemas de produção.

Extents — O Fim dos Blocos Espalhados

ext2 e ext3 mapeavam arquivos com arrays de ponteiros de bloco individual: blocos diretos, indireto simples, indireto duplo e triplo. Para um arquivo de 1 GB com blocos de 4 KB, isso significava 262.144 entradas de ponteiro. ext4 introduziu extents: uma run contígua de blocos descrita por (bloco inicial, comprimento). Um arquivo de 1 GB sem fragmentação usa apenas 1 extent.

bash — debugfs para inspecionar extents
# Montar filesystem read-only para inspeção
debugfs /dev/sda1

# Dentro do debugfs — ver extents de um arquivo
debugfs: stat /var/log/syslog
# ...
# EXTENTS:
# (0-1023):  524288-525311    <- 1 extent cobrindo 1024 blocos de 4KB = 4 MB

# Verificar fragmentação de um arquivo
filefrag -v /var/log/syslog
# ext: 0  logical: 0-1023  physical: 524288-525311  length: 1024  flags: last
# Total: 1 extent

Journaling em ext4

O journal ext4 opera em um dos três modos configuráveis. O modo ordered é o padrão e equilibra performance com segurança de dados.

Modo O que e journalado Performance Segurança de dados
writebackSomente metadadosMais rapidoDados podem aparecer fora de ordem apos crash
ordered (padrao)Metadados; dados flushed antes do commitBalanceadoDados consistentes com metadados
journalMetadados E dadosMais lento (dobra escritas)Maximo: dados e metadados sempre consistentes
bash — tune2fs e parâmetros do ext4
# Ver todos os parâmetros do filesystem
tune2fs -l /dev/sda1 | grep -E "Journal|Features|Last"

# Alterar modo de journaling (requer umount)
tune2fs -o journal_data_writeback /dev/sda1  # modo writeback
tune2fs -o journal_data /dev/sda1            # modo data journal

# Verificar e reparar filesystem (somente com FS desmontado)
fsck.ext4 -f /dev/sda1

# Após crash: replay do journal (automático no mount)
# dmesg mostra:
# EXT4-fs (sda1): recovery complete
# EXT4-fs (sda1): mounted filesystem with ordered data mode
