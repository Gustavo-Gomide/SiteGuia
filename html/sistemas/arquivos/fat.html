FAT — File Allocation Table

FAT12, FAT16, FAT32 e exFAT: estrutura, limitações e ahol de compatibilidade universal

FAT é um dos sistemas de arquivos mais simples e amplamente compatíveis já criados, presente em pendrives, cartões SD, partições de boot EFI e legado em sistemas embarcados. Sua simplicidade é sua força e também sua limitação: sem permissões, sem journaling, sem links simbólicos.

Estrutura FAT

Um volume FAT é dividido em três regiões principais: o Boot Record (informações do volume), as tabelas FAT (duplicadas para redundância) e a área de dados (onde os clusters de arquivos residem). O cluster é a unidade mínima de alocação — vários setores consecutivos. Cada entrada na tabela FAT aponta para o próximo cluster do arquivo, ou contém marcadores especiais.

Região Conteúdo
Boot Record (setor 0)BPB (BIOS Parameter Block): tamanho do setor, setores por cluster, número de FATs, tamanho do volume
Tabela FAT (x2)Array de entradas de 12/16/32 bits; cada índice corresponde a um cluster; valor = próximo cluster ou EOF (0xFFF/0xFFFF/0x0FFFFFFF) ou livre (0x000)
Diretório Raiz (FAT12/16)Área de tamanho fixo com entradas de 32 bytes por arquivo/subdiretório (nome 8.3, atributos, cluster inicial, tamanho)
Área de DadosClusters com os dados reais dos arquivos e subdiretórios
pseudocódigo — travessia de cadeia FAT
// Ler arquivo a partir da entrada de diretório
cluster_atual = entrada.cluster_inicial  // ex: 5

while (cluster_atual != EOF) {
    ler_dados_do_cluster(cluster_atual)
    cluster_proximo = FAT[cluster_atual]  // ex: FAT[5] = 8
    cluster_atual = cluster_proximo       //     FAT[8] = 3
}                                         //     FAT[3] = EOF

// Para arquivo fragmentado: 5 → 8 → 3 → EOF
// Leitura sequencial mas saltos físicos no disco

Limitações e Problemas do FAT32

Limite de 4 GB por arquivo

O campo de tamanho no diretório é um inteiro de 32 bits sem sinal, limitando arquivos a 4.294.967.295 bytes (4 GB - 1). Impossível armazenar um único arquivo de vídeo HD ou imagem de disco maior que isso no FAT32.

Sem permissões ou segurança

Não há conceito de owner, group ou bits rwx. Qualquer sistema que monte o volume tem acesso total a todos os arquivos. Inadequado para partições de sistema Linux ou Windows.

Sem journaling

Uma queda de energia durante uma escrita pode corromper a tabela FAT e orphanear clusters. O único recurso é o CHKDSK/fsck.fat, que varre o volume inteiro — demorado em volumes grandes.

Sem links simbólicos nem hard links

FAT não possui inodes nem o conceito de link count. Nao é possível ter um arquivo apontando para outro (symlink) ou múltiplos nomes para o mesmo dado (hard link).

exFAT — O Sucessor Moderno

O exFAT (Extended FAT) foi criado pela Microsoft em 2006 para superar as limitações do FAT32 em mídias flash, sem a complexidade do NTFS. Ele remove o limite de 4 GB por arquivo e o limite prático de tamanho de volume, mantendo compatibilidade ampla.

Suporte nativo universal

Linux 5.4+ inclui driver exFAT nativo (exfatprogs). Windows, macOS, Android, cameras digitais e consoles de videogame modernos leem exFAT sem drivers adicionais.

Padrão SD Association

A SD Association adotou exFAT como formato padrão para cartões SDXC (64 GB a 2 TB) e SDUC (acima de 2 TB). Câmeras e drones formatam cartões grandes em exFAT automaticamente.

Fragmentação e alocação

exFAT mantém o mapa de alocação mas adiciona um Allocation Bitmap para marcar clusters livres/ocupados de forma mais eficiente. Suporta extensões para contiguous allocation hint, reduzindo fragmentação em mídias flash.
