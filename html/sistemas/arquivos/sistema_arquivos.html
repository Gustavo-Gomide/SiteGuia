Sistemas Operacionais

Sistema de Arquivos

Como o kernel organiza, armazena e recupera dados em dispositivos de bloco

Um sistema de arquivos organiza dados em disco como arquivos e diretórios, gerenciando espaço livre, metadados e integridade. O VFS (Virtual File System) do Linux abstrai as diferenças entre implementações concretas, permitindo que uma chamada read() funcione identicamente sobre ext4, btrfs, NFS ou tmpfs.

VFS — Virtual File System

O VFS é uma camada de indireção no kernel Linux que define uma interface uniforme para todos os sistemas de arquivos concretos. Ele exporta quatro objetos principais:

superblock

Metadados do volume montado: tipo de FS, tamanho do bloco, número de inodes, ponteiro para o inode raiz. Criado em memória no momento do mount.

inode

Metadados de um arquivo individual: dono (UID/GID), permissões, timestamps (atime/mtime/ctime), tamanho, ponteiros para blocos de dados. O nome do arquivo não vive no inode.

dentry

Entrada de diretório que associa um nome a um inode. Mantida em cache (dcache) pelo kernel para evitar percursos repetidos de pathname lookup.

file

Instância de um arquivo aberto. Contém offset de leitura/escrita, flags O_RDONLY/O_WRONLY etc. e ponteiro para o dentry. Uma mesma dentry pode ter múltiplos objetos file (vários processos abrindo o mesmo arquivo).

O caminho de uma chamada read(fd, buf, n) percorre: VFS (namespace lookup, permissões) → FS específico (ext4, btrfs…) → block layer (elevator, plug/unplug) → driver de bloco → hardware.

Journaling

Journaling registra a intenção de uma operação antes de executá-la (Write-Ahead Log — WAL). Após uma queda de energia, o kernel replaya o journal em segundos, em vez de rodar um fsck completo que pode demorar horas em volumes grandes.

writeback

Apenas metadados são journaled. Dados podem aparecer no disco antes dos metadados, mas sem garantia de ordem. Mais rápido, menos seguro para dados de arquivo.

ordered (padrão ext4)

Dados de arquivo são escritos no disco antes dos metadados correspondentes serem commitados no journal. Garante que nenhum metadado aponte para blocos de dados corruptos.

data journal

Tanto dados quanto metadados são escritos no journal. Máxima proteção, mas dobra as escritas de dados. Indicado para sistemas críticos onde perda de dados é inaceitável.

Para verificar e alterar o modo de journal de um volume ext4:

bash — tune2fs
# Ver parâmetros atuais do journal
tune2fs -l /dev/sda1 | grep -i journal

# Alterar para modo data journal (requer remontagem)
tune2fs -o journal_data /dev/sda1

# Verificar modo ativo no kernel
dmesg | grep "EXT4-fs.*mounted filesystem"
