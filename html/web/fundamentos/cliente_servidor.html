Cliente-Servidor

Base de Comunicação da Web

O modelo em que clientes solicitam recursos e servidores processam e respondem.

Grande parte dos sistemas web usa cliente-servidor para separar interface e processamento. Essa divisão permite evolução independente, escalabilidade progressiva e melhor governança de dados.

Fluxo de Requisição e Resposta

  1. Cliente resolve domínio via DNS e abre conexão segura com TLS.
  2. Cliente envia requisição HTTP com método, cabeçalhos e corpo opcional.
  3. Servidor autentica, valida regras e consulta recursos necessários.
  4. Servidor retorna status, cabeçalhos e payload para renderização no cliente.

Estado e Escalabilidade

Sistemas sem estado no servidor (stateless) simplificam escala horizontal. Quando há estado de sessão, cache distribuído e políticas de consistência tornam-se necessários para evitar comportamento inconsistente.

Limites e Evoluções

Cliente-servidor não resolve tudo: latência de rede, indisponibilidade de serviços e acoplamento de contratos ainda exigem boas práticas de retry, observabilidade e versionamento de API.
