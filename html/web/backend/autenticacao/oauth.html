OAuth 2.0

Delegação Segura de Acesso entre Sistemas

OAuth permite que um cliente acesse recursos em nome do usuário sem expor credenciais primárias.

OAuth 2.0 é base de integrações modernas com provedores de identidade. A implementação correta depende de escolha de fluxo, proteção contra replay e validação rigorosa de redirecionamento.

Fluxos Principais

  • Authorization Code + PKCE: recomendado para apps web e mobile.
  • Client Credentials: comunicação serviço-a-serviço sem usuário final.
  • Refresh Token: renovação de sessão com menor exposição de credenciais.

Cuidados de Segurança

Valide estritamente redirect URI, use state/nonce, aplique escopos mínimos e monitore tentativas de abuso. Em autenticação de usuários, OAuth normalmente é combinado com OpenID Connect para identidade.

Referências

  • RFC 6749. The OAuth 2.0 Authorization Framework. IETF, 2012.
  • RFC 7636. Proof Key for Code Exchange (PKCE). IETF, 2015.
  • OPENID FOUNDATION. OpenID Connect Core. 2023.

OAuth bem implementado reduz exposição de credenciais e melhora governança de integrações externas.
