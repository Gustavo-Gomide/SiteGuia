JWT

Token Assinado para Autenticação e Autorização

JSON Web Token transporta claims assinadas para validar identidade e permissões.

JWT facilita autenticação distribuída, mas seu uso exige controle rigoroso de expiração, escopo, rotação de chaves e armazenamento seguro no cliente.

Estrutura do Token

Um JWT possui três partes: header, payload e assinatura. O payload não é criptografado por padrão, portanto não deve armazenar dados sensíveis sem proteção adicional.

Boas Práticas

  • Assinar com algoritmo forte e validar issuer/audience.
  • Definir expiração curta e uso de refresh token controlado.
  • Revogar tokens em incidentes e eventos críticos de conta.
  • Evitar armazenamento em local suscetível a XSS.

Referências

  • RFC 7519. JSON Web Token (JWT). IETF, 2015.
  • OWASP. JSON Web Token Cheat Sheet. 2025.

JWT é eficiente, mas sem governança vira um ponto cego de segurança distribuída.

"Token assinado não substitui política de sessão e resposta a incidente."
