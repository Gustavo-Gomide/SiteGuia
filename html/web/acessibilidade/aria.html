ARIA

Semântica Acessível para Componentes Dinâmicos

Use ARIA para complementar HTML, nunca para substituir semântica nativa sem necessidade.

ARIA é essencial em componentes ricos de interface, desde que aplicado com responsabilidade para não piorar a experiência de tecnologias assistivas.

Papéis e Estados

Defina role, aria-expanded, aria-controls, aria-live e outros atributos de acordo com comportamento real do componente.

Erros Comuns

  • Adicionar ARIA redundante em elementos já semânticos.
  • Não atualizar estados ARIA após interação.
  • Usar roles incompatíveis com comportamento visual.

Referências

  • W3C. WAI-ARIA 1.2. 2023.
  • MDN. ARIA: Accessibility Tree. 2025.

ARIA bem usado aumenta clareza para leitores de tela; mal usado cria ruído e confusão.
