Code Splitting

JavaScript Carregado Sob Demanda

Menos código inicial, melhor tempo de interação.

Code splitting reduz bundle inicial e distribui custo de download conforme o usuário navega pelos recursos do produto.

Estratégias

  • Split por rota.
  • Split por componente pesado.
  • Importação dinâmica de dependências raras.

Equilíbrio

Divisão excessiva aumenta número de requests e pode degradar UX. A estratégia ideal depende de padrões reais de navegação e cache.
