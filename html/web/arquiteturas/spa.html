SPA

Single-Page Application

Interface dinâmica com transições sem recarga completa de documento.

SPA melhora fluidez da experiência e permite interações ricas, mas demanda disciplina em bundle size, estado global e estratégia de renderização para SEO.

Como Funciona

Após o carregamento inicial, roteamento e renderização ocorrem no cliente. Dados são buscados via APIs, e a UI reage às mudanças de estado.

Riscos Comuns

  • Bundle inicial pesado.
  • SEO prejudicado sem SSR/SSG.
  • Complexidade de estado e cache.

Referências

  • GOOGLE. Web Fundamentals. 2025.
  • MDN. Client-side Web APIs. 2025.

SPA entrega velocidade percebida quando arquitetura e performance são tratadas desde o início.

"Sem governança de estado, SPA escala bugs com a mesma velocidade que escala features."
