MPA

Arquitetura Multi-Page Application

Navegação com recarga de documento a cada rota.

MPA prioriza simplicidade de entrega, ótima indexação orgânica e acoplamento menor no frontend, sendo comum em portais, e-commerce tradicional e sistemas institucionais.

Características

  • Renderização por página no servidor.
  • URLs diretas e semânticas por rota.
  • Menor dependência de JavaScript para conteúdo inicial.

Trade-offs

Cada navegação implica nova requisição de documento, o que pode reduzir fluidez percebida. Em contrapartida, o modelo simplifica caching HTTP e SEO.

Referências

  • MDN. Website Architecture. 2025.
  • RFC 9110. HTTP Semantics. IETF, 2022.

MPA continua sendo escolha técnica sólida quando previsibilidade de entrega e indexação são prioridades.

"Arquitetura boa é a que atende contexto, não tendência."
