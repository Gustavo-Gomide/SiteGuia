Promises

Contrato para Operações Assíncronas

Promise representa valor futuro resolvido ou rejeitado.

Promises evitam callback hell e padronizam composição de tarefas assíncronas no JavaScript moderno.

Estados

  • pending
  • fulfilled
  • rejected

Composição

Use Promise.all, Promise.allSettled e Promise.race conforme necessidade de sincronização e tolerância a erro.

Referências

  • MDN. Using Promises. 2025.
  • ECMA-262. Promise Objects.

Promise bem modelada evita fluxos assíncronos opacos e imprevisíveis.
