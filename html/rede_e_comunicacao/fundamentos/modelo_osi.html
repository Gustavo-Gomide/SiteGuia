Modelo OSI

Open Systems Interconnection — o modelo de referência que organiza toda a comunicação em rede em 7 camadas independentes

O modelo OSI (Open Systems Interconnection), criado pela ISO em 1984, é uma arquitetura de referência que divide a comunicação em rede em 7 camadas independentes. Cada camada tem função específica e se comunica apenas com as camadas imediatamente acima e abaixo dela. Embora o TCP/IP seja o modelo implementado na prática, o OSI é indispensável para diagnóstico de falhas, design de protocolos e comunicação técnica entre equipes.

Mnemônico — de cima para baixo (7→1): All People Seem To Need Data Processing (Aplicação, Apresentação, Sessão, Transporte, Rede, Enlace, Física). De baixo para cima: Please Do Not Throw Sausage Pizza Away.

As 7 Camadas em Detalhe

Camada 1 — Física

PDU: Bit • Função: transmitir bits brutos através do meio físico.

  • Define características elétricas, ópticas e de rádio (voltagem, frequência, timing)
  • Especifica conectores físicos: RJ-45 (Ethernet), SC/LC (fibra), BNC (coaxial)
  • Codificação de sinal: Manchester, NRZ, 4B/5B, 8B/10B
  • Modos de transmissão: simplex, half-duplex, full-duplex
  • Meios físicos: Par trançado (Cat5e, Cat6, Cat6A), fibra multimodo/monomodo, rádio
  • Dispositivos: Hub, repetidor, cabo, fibra óptica, transceptor (SFP)

Camada 2 — Enlace de Dados

PDU: Frame • Função: comunicação confiável entre nós adjacentes na mesma rede local.

  • Endereçamento MAC (48 bits): AA:BB:CC:DD:EE:FF — único por NIC
  • Detecção e correção de erros: CRC (Cyclic Redundancy Check)
  • Controle de acesso ao meio: CSMA/CD (Ethernet com fio), CSMA/CA (Wi-Fi)
  • Divide-se em sublayers: LLC (controle lógico) e MAC (acesso ao meio)
  • VLANs: segmentação lógica via IEEE 802.1Q (tag de 12 bits)
  • Spanning Tree Protocol (STP / RSTP): elimina loops em redes comutadas
  • Protocolos: Ethernet (802.3), Wi-Fi (802.11), PPP, HDLC, Frame Relay

Camada 3 — Rede

PDU: Pacote • Função: endereçamento lógico e roteamento entre redes distintas.

  • Endereços IP (lógicos): IPv4 (32 bits) e IPv6 (128 bits)
  • Roteamento: escolha do melhor caminho até o destino via tabela de rotas
  • Fragmentação de pacotes quando MTU da rede seguinte é menor
  • TTL (Time-To-Live): evita loops infinitos de pacotes
  • Protocolos de roteamento: OSPF (link-state), BGP (path-vector), RIP (distance-vector)
  • ICMP: controle e diagnóstico (ping, traceroute, "destino inalcançável")
  • Dispositivos: roteador, switch L3, firewall L3

Camada 4 — Transporte

PDU: Segmento (TCP) / Datagrama (UDP) • Função: comunicação fim-a-fim entre processos de aplicação.

  • Multiplexação via portas (0–65535): identifica qual processo recebe os dados
  • TCP: confiável, orientado a conexão, controle de fluxo e congestionamento, 3-way handshake
  • UDP: não confiável, sem conexão, latência mínima — para streaming, DNS, VoIP
  • Segmentação: divide dados grandes em segmentos e remonta no destino
  • SCTP: alternativa ao TCP com suporte a multistreaming (telecomunicações)
  • QUIC: protocolo moderno sobre UDP, base do HTTP/3, multiplexação sem HOL blocking

Camada 5 — Sessão

PDU: Dados • Função: estabelecimento, manutenção e encerramento de sessões entre aplicações.

  • Controla diálogos: simplex, half-duplex, full-duplex
  • Sincronização com checkpoints: permite retomada após falha sem reiniciar do zero
  • Gerencia múltiplas sessões simultâneas (ex: várias abas do navegador)
  • Exemplos: NetBIOS, RPC (Remote Procedure Call), NFS, SOCKS, SQL de sessão
  • No TCP/IP prático, esta função é absorvida pela camada de Aplicação (HTTP keep-alive, WebSocket)

Camada 6 — Apresentação

PDU: Dados • Função: "tradutora" entre formatos de dados diferentes — garante que o destinatário entenda os dados.

  • Criptografia e descriptografia: TLS/SSL (HTTPS), PGP
  • Compressão de dados: gzip, zlib, Brotli
  • Serialização e codificação: JSON, XML, ASN.1, MIME (email attachments)
  • Codificação de caracteres: ASCII, Unicode, UTF-8, Base64
  • Conversão de formatos de imagem/vídeo: JPEG, MP4, H.264
  • No TCP/IP, absorvida pela camada de Aplicação (ex: TLS é biblioteca do app)

Camada 7 — Aplicação

PDU: Dados • Função: interface direta com o software do usuário, fornecendo serviços de rede.

  • Web: HTTP/1.1, HTTP/2, HTTP/3 (QUIC)
  • Nomes e endereços: DNS (porta 53), mDNS, LLMNR
  • Email: SMTP (envio), IMAP/POP3 (recebimento), S/MIME (assinatura)
  • Acesso remoto: SSH (porta 22), RDP (porta 3389), Telnet (porta 23)
  • Transferência de arquivos: FTP, SFTP, SCP, TFTP
  • Gerência de rede: SNMP, NetFlow/IPFIX, Syslog
  • "Aplicação" não é o app em si (Firefox), mas o protocolo que ele usa (HTTP)

Encapsulamento e Desencapsulamento

Ao enviar dados, cada camada adiciona seu cabeçalho (e às vezes um rodapé) ao payload recebido da camada superior — isso é encapsulamento. O receptor faz o caminho inverso, removendo cada cabeçalho ao subir a pilha.

Fluxo completo de encapsulamento — envio de uma requisição HTTPS

Dados descendo a pilha OSI (emissor)
Camada 7 - Aplicação   : [GET /index.html HTTP/1.1...]            (Dados HTTP)
Camada 6 - Apresentação: [TLS Record hdr][HTTP dados cifrados]   (criptografia TLS)
Camada 5 - Sessão      : [Session token][TLS][HTTP]              (gerência de sessão)
Camada 4 - Transporte  : [TCP hdr src:52341 dst:443][Dados]      (Segmento TCP)
Camada 3 - Rede        : [IP hdr src:10.0.0.5 dst:93.184.216.34][TCP][Dados]  (Pacote IP)
Camada 2 - Enlace      : [ETH hdr src:AA:BB dst:CC:DD][IP][TCP][Dados][CRC]   (Frame Ethernet)
Camada 1 - Física      : 10101100 01010110 11001010 ... (bits serializados no cabo)
Subindo a pilha OSI (receptor) — desencapsulamento
Camada 1: recebe bits, reconstrói o frame
Camada 2: verifica CRC, lê MAC destino (é para mim?), extrai pacote IP
Camada 3: lê IP destino (é para mim?), decrementa TTL, extrai segmento TCP
Camada 4: verifica checksum, monta buffer, entrega dados à porta 443 (HTTPS)
Camadas 5/6: TLS descriptografa o payload
Camada 7: servidor HTTP interpreta "GET /index.html" e prepara resposta

Diagnóstico por Camada

A maior utilidade prática do OSI é isolar falhas. A abordagem padrão é bottom-up: confirme que a camada 1 funciona antes de culpar a camada 3.

CamadaSintoma típicoFerramentas de diagnósticoO que verificar
1 — Física Link down, sem sinal, velocidade errada ip link, LEDs da NIC/switch Cabo, conector, tipo de cabo (Cat5e vs Cat6), NIC, porta do switch, SFP transceptor
2 — Enlace Ping ao gateway falha, link up arp -a, ip neigh, Wireshark (ARP) Tabela ARP, VLAN configurada corretamente, STP bloqueando porta
3 — Rede Ping entre redes diferentes falha ping, traceroute, ip route Tabela de rotas, IP/máscara corretos, gateway padrão, ACLs no roteador
4 — Transporte Serviço inacessível, ping OK netstat, ss, telnet host porta, nmap Porta aberta e escutando, firewall bloqueando, MTU causando fragmentação
5–6 — Sessão/Apresentação TLS handshake falha, sessão cai openssl s_client -connect host:443, Wireshark (TLS) Certificado expirado, cipher suite incompatível, SNI errado
7 — Aplicação Erro HTTP 4xx/5xx, app não funciona mas rede OK curl -v, logs da aplicação, Postman Configuração do servidor, autenticação, headers, DNS resolving correto
