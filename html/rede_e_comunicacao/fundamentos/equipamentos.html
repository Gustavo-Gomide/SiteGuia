Equipamentos de Rede

Do hub ao firewall NGFW — entenda o que cada equipamento faz, em qual camada OSI opera e quando usar cada um

Cada equipamento de rede opera em uma camada específica da pilha OSI, o que define o que ele "enxerga" no tráfego e como toma decisões. Um hub apenas regenera o sinal elétrico (camada 1). Um switch lê endereços MAC (camada 2). Um roteador lê endereços IP (camada 3). Um firewall NGFW pode inspecionar até a camada 7. Compreender isso é fundamental para escolher o equipamento certo e diagnosticar problemas corretamente.

Cada Equipamento em Detalhe

Hub (Concentrador)

Camada 1 — Física

Simplesmente replica o sinal elétrico recebido em uma porta para todas as outras. Não lê nem endereços MAC nem IP — é "burro" por design.

  • Cria um único domínio de colisão: só um dispositivo pode transmitir por vez (CSMA/CD)
  • Toda comunicação é compartilhada: se A transmite para B, C e D também recebem os bits
  • Banda compartilhada: 10 dispositivos em hub de 100 Mbps = ~10 Mbps por dispositivo real
  • Obsoleto. Não existe motivo para usar hub em redes novas. Switches são praticamente do mesmo custo hoje
  • Ainda útil para: port mirroring manual em análise de tráfego legado (capturar pacotes de terceiros)

Switch L2 (Comutador)

Camada 2 — Enlace

Opera com endereços MAC. Aprende quais MACs estão em quais portas e encaminha frames apenas para a porta correta — não para todas.

  • Tabela CAM (Content Addressable Memory): aprende MAC → porta ao ver frames chegando
  • Full-duplex: cada porta tem seu próprio domínio de colisão — sem CSMA/CD
  • Flooding: quando o MAC de destino não está na tabela, envia para todas as portas (exceto origem)
  • VLANs (802.1Q): segmenta logicamente a rede em múltiplos domínios L2 isolados
  • STP/RSTP: evita loops de switching entre switches redundantes
  • PoE (Power over Ethernet): alimenta APs, câmeras e telefones VoIP pelo cabo de rede
  • Switch não gerenciável: plug-and-play, sem configuração, barato, uso doméstico
  • Switch gerenciável: CLI ou web interface, VLANs, QoS, SNMP, port mirroring, ACLs

Switch L3 (Switch com Roteamento)

Camadas 2 e 3

Um switch com capacidade de roteamento via hardware (ASICs). Roteia entre VLANs sem precisar de um roteador externo separado, com performance muito superior ao roteador em software.

  • SVIs (Switched Virtual Interfaces): interfaces virtuais com IP para cada VLAN — gateway da VLAN
  • Routing inter-VLAN nativo: pacotes entre VLANs são roteados dentro do switch
  • Suporta protocolos de roteamento: OSPF, BGP, EIGRP
  • Típico na camada de distribuição e núcleo de redes corporativas
  • Exemplos: Cisco Catalyst 9300, Juniper EX, Aruba 2930F

Roteador

Camada 3 — Rede

Opera com endereços IP. Encaminha pacotes entre redes diferentes consultando a tabela de rotas. É o elemento que conecta sua LAN à internet.

  • Separa domínios de broadcast — não propaga broadcasts entre interfaces
  • NAT: tradução de IPs privados para público (PAT)
  • Tabela de rotas: rotas estáticas (configuradas manualmente) ou dinâmicas (OSPF, BGP, RIP)
  • Firewall básico: ACLs (Access Control Lists) para filtrar pacotes por IP/porta
  • QoS: priorização de tráfego VoIP sobre navegação, por exemplo
  • WAN: conecta a links de operadora (fibra, 4G/5G, MPLS, xDSL)
  • Roteadores domésticos (TP-Link, D-Link) combinam roteador + switch + AP + NAT/firewall em um só

Firewall

Camada 3–7 (depende do tipo)

Controla e filtra o tráfego de rede com base em regras de segurança. O guardião que decide o que entra e sai da rede.

  • Packet Filter (stateless): filtra por IP origem/destino + porta + protocolo. Sem contexto de conexão. Camadas 3–4. Rápido, mas limitado
  • Stateful Inspection: mantém tabela de estado de conexões TCP/UDP. Permite resposta a conexões iniciadas internamente; bloqueia respostas a conexões não solicitadas. Camadas 3–4
  • ALG (Application Layer Gateway): inspeciona protocolos específicos como FTP, SIP, H.323 que precisam de tratamento especial com NAT
  • NGFW (Next-Generation Firewall): adiciona IPS (Intrusion Prevention), DPI (Deep Packet Inspection), identificação de aplicação (Layer 7), controle por usuário (integração AD), SSL inspection, sandboxing. Camadas 2–7
  • Fabricantes NGFW: Palo Alto Networks, Fortinet, Check Point, Cisco Firepower, Sophos

Access Point (AP) Wi-Fi

Camadas 1 e 2

Converte o sinal sem fio 802.11 em tráfego Ethernet com fio e vice-versa. É o "switch sem fio" que permite que dispositivos móveis acessem a rede cabeada.

  • Modos: AP autônomo (configuração individual), AP gerenciado por controlador (Cisco, Aruba, Ubiquiti UniFi)
  • SSID: nome da rede Wi-Fi que os clientes veem. Um AP pode transmitir múltiplos SSIDs
  • Mapeamento SSID → VLAN: cada SSID pode ser mapeado a uma VLAN diferente (ex: "Corporativo" → VLAN 10, "Visitantes" → VLAN 20)
  • Padrões: 802.11n (Wi-Fi 4, 2.4/5 GHz), 802.11ac (Wi-Fi 5, 5 GHz), 802.11ax (Wi-Fi 6, 2.4/5/6 GHz), 802.11be (Wi-Fi 7)
  • Alimentação: PoE (802.3af/at/bt) — sem precisar de tomada separada
  • Roaming: 802.11r (Fast BSS Transition) e 802.11k/v para roaming rápido entre APs

Modem

Camada 1 (e parcialmente 2)

MOdulator/DEModulator: converte o sinal digital da rede local em sinal analógico (ou digital do tipo certo) para o meio de transmissão do provedor, e vice-versa.

  • DSL (xDSL): modula dados sobre par de cobre telefônico (ADSL, VDSL2)
  • Cable modem (DOCSIS): modula sobre cabo coaxial de TV a cabo (DOCSIS 3.1: até 10 Gbps downstream)
  • Modem de fibra (ONU/ONT): converte sinal óptico (GPON, XGS-PON) em Ethernet para o roteador
  • Modem 4G/5G: converte sinal de rádio celular em Ethernet
  • Em instalações modernas, o modem e o roteador vêm em um único dispositivo (gateway da operadora)

Load Balancer

Camadas 4 a 7

Distribui requisições entre múltiplos servidores para maximizar throughput, minimizar latência e garantir alta disponibilidade. Fundamental em arquiteturas de produção.

  • L4 (TCP/UDP): distribui conexões por IP/porta. Mais rápido, sem inspecionar conteúdo (ex: HAProxy em modo TCP, AWS NLB)
  • L7 (HTTP/HTTPS): distribui por URL, header HTTP, cookie, método. Permite afinidade de sessão (sticky sessions), SSL offloading, health checks avançados (ex: Nginx, HAProxy HTTP, AWS ALB)
  • Algoritmos: Round Robin, Least Connections, IP Hash, Weighted, Resource-Based
  • Health checks: remove automaticamente servidores com falha do pool
  • SSL Termination: decripta HTTPS no LB, envia HTTP para os backends (reduz carga de CPU dos servidores)

Comandos Cisco IOS — Básico Prático

Configuração inicial de switch

Cisco IOS — Modos e configuração básica
Switch> enable                    ! entra no modo privilegiado (EXEC)
Switch# configure terminal        ! entra no modo de configuração global
Switch(config)# hostname SW-01   ! define o nome do switch
Switch(config)# no ip domain-lookup  ! desativa tentativa de resolver nomes
Switch(config)# service password-encryption  ! roda hash nas senhas

! Senha para modo privilegiado
Switch(config)# enable secret MinhaSenh@Forte

! Configurar acesso via VTY (SSH/Telnet)
Switch(config)# line vty 0 15
Switch(config-line)# transport input ssh
Switch(config-line)# login local
Switch(config-line)# exit

! Criar usuário local
Switch(config)# username admin privilege 15 secret MinhaSenh@Forte

! Salvar configuração
Switch# copy running-config startup-config
Switch# write memory    ! equivalente ao copy run start

Configuração de VLANs

Cisco IOS — VLANs, access e trunk
! Criar VLANs na base de dados do switch
Switch(config)# vlan 10
Switch(config-vlan)# name CORPORATIVO
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name VISITANTES
Switch(config-vlan)# exit

! Porta Access (dispositivo final, sem tag)
Switch(config)# interface fastEthernet 0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
Switch(config-if)# exit

! Porta Trunk (para outro switch ou AP com múltiplas VLANs)
Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport trunk allowed vlan 10,20
Switch(config-if)# exit

! Verificar VLANs
Switch# show vlan brief
Switch# show interfaces trunk

Diagnóstico de switching

Cisco IOS — Comandos de show e debug
! Ver tabela MAC (CAM table)
Switch# show mac address-table
Switch# show mac address-table dynamic

! Ver estado das interfaces
Switch# show interfaces status
Switch# show interfaces gigabitEthernet 0/1

! Ver configuração das interfaces
Switch# show running-config interface fa0/1

! Spanning Tree
Switch# show spanning-tree
Switch# show spanning-tree vlan 10

! Ver vizinhos CDP (Cisco Discovery Protocol)
Switch# show cdp neighbors
Switch# show cdp neighbors detail

! Limpar tabela MAC forçando re-aprendizado
Switch# clear mac address-table dynamic

Interface de gerência e roteamento inter-VLAN (L3)

Cisco IOS — SVI e roteamento
! Interface de gerência do switch (SVI VLAN 1 ou dedicada)
Switch(config)# interface vlan 1
Switch(config-if)# ip address 192.168.1.10 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit
Switch(config)# ip default-gateway 192.168.1.1

! Switch L3 — roteamento entre VLANs via SVI
Switch(config)# ip routing              ! ativa roteamento

Switch(config)# interface vlan 10
Switch(config-if)# ip address 10.10.10.1 255.255.255.0
Switch(config-if)# no shutdown

Switch(config)# interface vlan 20
Switch(config-if)# ip address 10.10.20.1 255.255.255.0
Switch(config-if)# no shutdown

! Verificar rotas
Switch# show ip route
