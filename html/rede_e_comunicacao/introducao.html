Redes de Computadores — Visão Geral

Introdução às Redes de Computadores

Da comutação de pacotes à internet global — os fundamentos que conectam o mundo digital

Uma rede de computadores é um conjunto de dispositivos interligados capazes de compartilhar recursos e trocar informações. O que começou com dois computadores conectados por um cabo na ARPANET (1969) hoje sustenta a maior obra de engenharia da história humana: a internet, com mais de 5 bilhões de usuários e trilhões de dispositivos conectados.

Por que estudar redes?

  • Todo software moderno depende de rede — APIs, banco de dados, autenticação, CDN
  • Diagnóstico de problemas requer entender onde na pilha a falha ocorre
  • Segurança começa com compreender como os dados trafegam
  • Cloud, microsserviços e contêineres são essencialmente problemas de rede

Marcos Históricos

  • 1969 — ARPANET: primeiro pacote trafega entre UCLA e Stanford
  • 1973 — TCP/IP conceituado por Vint Cerf e Bob Kahn
  • 1983 — ARPANET migra para TCP/IP (Flag Day)
  • 1991 — Tim Berners-Lee cria a World Wide Web no CERN
  • 2007 — iPhone populariza a internet móvel
  • 2020+ — 5G, Wi-Fi 6, SDN, redes quânticas experimentais

Modelos de Referência (OSI e TCP/IP)

Modelos de referência descrevem como as funcionalidades de rede são divididas em camadas independentes. Cada camada oferece serviços à camada superior e consome serviços da camada inferior — isso permite que fabricantes diferentes trabalhem juntos sem precisar se conhecer.

Modelo OSI — 7 Camadas

Criado pela ISO em 1984. Modelo teórico de referência — não é implementado diretamente, mas é essencial para diagnóstico e comunicação entre engenheiros.

Encapsulamento — dados descendo a pilha

Quando você acessa google.com, seus dados passam por cada camada, recebendo um cabeçalho novo a cada nível:

  1. Aplicação → gera a requisição HTTP GET
  2. Transporte → encapsula em segmento TCP (porta 80/443)
  3. Rede → encapsula em pacote IP (endereços IP origem/destino)
  4. Enlace → encapsula em frame Ethernet (MACs origem/destino)
  5. Físico → converte em bits e transmite no cabo/ar
Mnemônico OSI (cima para baixo): "All People Seem To Need Data Processing"
Application, Presentation, Session, Transport, Network, Data Link, Physical.

Topologias Físicas

  • Barramento: todos num único cabo. Simples, mas uma falha derruba tudo. Hoje obsoleto.
  • Estrela: todos conectados a um switch central. Padrão atual — falha em um nó isola só aquele nó.
  • Anel: dados circulam em sentido único (Token Ring, FDDI). Ordem determinística, mas falha num nó pode quebrar o anel.
  • Malha: cada nó conectado a múltiplos outros. Alta redundância — base do roteamento da internet.
  • Árvore (estrela estendida): switches conectados hierarquicamente: acesso → distribuição → núcleo. Padrão em redes empresariais.

Como Descobrir as Informações da Sua Rede

Antes de configurar, diagnosticar ou proteger uma rede, você precisa saber o que já está lá. Esta seção mostra como identificar cada elemento da sua rede usando comandos nativos do sistema operacional — sem instalar nada.

O que você precisa saber sobre a sua rede

  • IP local (privado): o endereço da sua máquina dentro da rede (ex: 192.168.1.105)
  • Máscara de sub-rede: define o tamanho da rede em que você está (ex: 255.255.255.0 = /24)
  • Gateway padrão: o IP do roteador — todo tráfego para fora da rede local passa por ele
  • DNS: o servidor que traduz nomes (google.com) em IPs (142.250.x.x)
  • IP público (externo): o endereço que o mundo vê — pertence ao seu provedor
  • MAC address: identificador físico único da placa de rede — não muda com reinicializações (por padrão)
  • Hosts ativos: quais outros dispositivos estão na mesma rede (computadores, roteadores, câmeras, etc.)

Identificar fabricante pelo MAC address

Os primeiros 3 octetos de um MAC formam o OUI (Organizationally Unique Identifier) — um código registrado pela IEEE que identifica o fabricante da interface de rede.

Exemplos de OUI comuns
FC:FB:FB:xx:xx:xx  → Cisco Systems
00:50:56:xx:xx:xx  → VMware (interface virtual)
B8:27:EB:xx:xx:xx  → Raspberry Pi Foundation
AC:DE:48:xx:xx:xx  → Apple
3C:22:FB:xx:xx:xx  → Apple
00:1A:2B:xx:xx:xx  → TP-Link
DC:A6:32:xx:xx:xx  → Raspberry Pi Trading

# Consultar OUI online: macvendors.com
# Via terminal:
curl https://api.macvendors.com/AA:BB:CC
MAC spoofing: é possível alterar o MAC via software (ip link set eth0 address AA:BB:CC:DD:EE:FF). Por isso filtro de MAC sozinho não é uma medida de segurança confiável.

traceroute / tracert — o caminho até um destino

Cada roteador que um pacote atravessa é chamado de hop (salto). O traceroute exibe cada hop com seu IP e tempo de resposta.

Rastrear rota até google.com
# Windows
tracert google.com

# Linux / macOS
traceroute google.com

# Saída típica:
#  1   1ms  192.168.1.1      ← seu roteador (gateway)
#  2   8ms  10.10.0.1        ← roteador do provedor
#  3  12ms  200.131.x.x      ← backbone do provedor
#  4  18ms  72.14.x.x        ← rede Google
#  5  19ms  142.250.x.x      ← destino final

# * * *  = roteador não responde (firewall ou ICMP bloqueado)

Ver o que está ouvindo/conectado na máquina

netstat -ano | findstr LISTENING

# Linux — moderno (ss substitui netstat)

Subnetting e CIDR — Cálculos Passo a Passo

Subnetting é a habilidade de dividir uma rede maior em sub-redes menores. É um dos pontos mais cobrados em certificações (CCNA, CompTIA Network+) e um dos mais temidos — mas a lógica é puramente matemática e repetível.

Conceitos Fundamentais

  • Endereço de rede: primeiro endereço do bloco — identifica a sub-rede (bits de host todos 0)
  • Broadcast: último endereço do bloco — enviado a todos os hosts (bits de host todos 1)
  • Hosts utilizáveis: endereços entre o de rede e o broadcast
  • CIDR: notação /n indicando quantos bits são de rede

Fórmulas Essenciais

N° de sub-redes = 2n  (n = bits emprestados do host)

Hosts utilizáveis por sub-rede = 2h − 2  (h = bits de host restantes)

Incremento de bloco = 256 − valor do último octeto da máscara

Exemplo 1 — Dividir 192.168.10.0/24 em 4 sub-redes iguais

Passo 1 — Entender o problema

Rede dada: 192.168.10.0/24

Objetivo: Criar 4 sub-redes de igual tamanho.

A rede /24 tem 32 − 24 = 8 bits de host, equivalendo a 256 endereços (254 hosts úteis).

Precisamos emprestar bits do campo de host para criar as sub-redes. Quantos bits?

Passo 2 — Calcular bits a emprestar

Precisamos de pelo menos 4 sub-redes. Fórmula: 2n ≥ 4

  • 21 = 2 → insuficiente
  • 22 = 4 → ✓ suficiente!

Emprestamos n = 2 bits do campo de host.

Nova máscara: /24 + 2 = /26 → 255.255.255.192

Passo 3 — Calcular o incremento de bloco

O incremento define de quanto em quanto os endereços de rede pulam.

Incremento = 256 − 192 = 64

O valor 192 vem do último octeto da máscara 255.255.255.192.

Cada sub-rede ocupa um bloco de 64 endereços.

Hosts úteis por sub-rede: 26 − 2 = 62 hosts

Passo 4 — Listar as 4 sub-redes

Sub-redeEndereço de RedeHosts UtilizáveisBroadcast
1ª192.168.10.0/26192.168.10.1 – .62192.168.10.63
2ª192.168.10.64/26192.168.10.65 – .126192.168.10.127
3ª192.168.10.128/26192.168.10.129 – .190192.168.10.191
4ª192.168.10.192/26192.168.10.193 – .254192.168.10.255
Cada endereço de rede = anterior + 64 (o incremento).

Passo 5 — Verificar a qual sub-rede um IP pertence

Dado 192.168.10.140 com máscara /26 — qual sub-rede?

Divida pelo incremento (64): 140 ÷ 64 = 2 (resto 12) → bloco começa em 2 × 64 = 128.

  • Rede: 192.168.10.128/26
  • Broadcast: 192.168.10.191
  • Host válido? 140 entre 129-190 → ✓ SIM
Regra geral: endereço de rede = (IP ÷ incremento, resultado inteiro) × incremento.

Exemplo 2 — Qual máscara usar para acomodar 30 hosts por sub-rede?

Problema inverso: você sabe quantos hosts precisa e quer encontrar a máscara correta.

Passo 1 — Definir o requisito

Requisito: cada sub-rede deve suportar ao menos 30 hosts.

Fórmula: hosts úteis = 2h − 2, onde h = bits de host.

A subtração de 2 elimina o endereço de rede (bits de host = 0) e o broadcast (bits de host = 1).

Passo 2 — Encontrar bits de host necessários

Preciso: 2h − 2 ≥ 30

  • h = 4 → 24 − 2 = 14 → insuficiente
  • h = 5 → 25 − 2 = 30 → ✓ exatamente!
  • h = 6 → 26 − 2 = 62 → funciona com folga (mais endereços desperdiçados)

Mínimo: h = 5 bits de host.

Passo 3 — Calcular a máscara

32 bits totais − 5 bits de host = 27 bits de rede → /27

Máscara /27 em binário
11111111.11111111.11111111.11100000
   255  .   255  .   255  .  224
Os últimos 3 bits do quarto octeto são 0 (campo de host). 11100000 = 224.

Passo 4 — Confirmar o resultado

Máscara encontrada: /27 → 255.255.255.224

  • Incremento: 256 − 224 = 32
  • Hosts úteis: 25 − 2 = 30 ✓
  • Sub-redes geradas numa /24: 0, 32, 64, 96, 128, 160, 192, 224 → 8 sub-redes
Se usar /26 (62 hosts) desperdiçamos endereços. /27 é o mínimo eficiente para 30 hosts.

Exemplo 3 — VLSM: sub-redes de tamanhos diferentes no mesmo espaço

VLSM (Variable Length Subnet Masking) evita o desperdício de endereços usando máscaras diferentes para cada segmento de acordo com a necessidade real.

Passo 1 — O problema

Rede disponível: 10.0.0.0/24

Requisitos:

  • Segmento A: 100 hosts
  • Segmento B: 50 hosts
  • Segmento C: 25 hosts
  • Link WAN D: 2 hosts (ponto-a-ponto entre roteadores)
Regra VLSM: alocar sempre do maior para o menor.

Passo 2 — Segmento A (100 hosts)

2h − 2 ≥ 100 → h = 7 → /25 → máscara 255.255.255.128

  • Rede: 10.0.0.0/25
  • Hosts: 10.0.0.1 – 10.0.0.126 (126 hosts)
  • Broadcast: 10.0.0.127
Próximo bloco disponível começa em 10.0.0.128

Passo 3 — Segmento B (50 hosts)

2h − 2 ≥ 50 → h = 6 → /26 → máscara 255.255.255.192

  • Rede: 10.0.0.128/26
  • Hosts: 10.0.0.129 – 10.0.0.190 (62 hosts)
  • Broadcast: 10.0.0.191
Próximo bloco livre: 10.0.0.192

Passo 4 — Segmento C (25 hosts)

2h − 2 ≥ 25 → h = 5 → /27 → máscara 255.255.255.224

  • Rede: 10.0.0.192/27
  • Hosts: 10.0.0.193 – 10.0.0.222 (30 hosts)
  • Broadcast: 10.0.0.223
Próximo bloco livre: 10.0.0.224

Passo 5 — Link WAN D (2 hosts) e resumo

2h − 2 ≥ 2 → h = 2 → /30 → máscara 255.255.255.252

  • Rede: 10.0.0.224/30
  • Hosts: 10.0.0.225 – 10.0.0.226 (2 hosts)
  • Broadcast: 10.0.0.227

Espaços utilizados:

  • A: 0–127 (128 endereços) • B: 128–191 (64) • C: 192–223 (32) • D: 224–227 (4)
Sobra: 10.0.0.228–255 (28 endereços) para expansão futura.

Hub × Switch × Roteador — diferença prática

Domínios de colisão e broadcast
Hub (4 portas)
  1 domínio de colisão   qualquer transmissão
  1 domínio de bcast     ocupa todas as portas

Switch L2 (4 portas)
  4 domínios de colisão  cada porta é isolada
  1 domínio de bcast     broadcast ainda alcança tudo

Roteador
  N domínios de colisão  cada interface é uma rede
  N domínios de bcast    broadcast não atravessa

VLANs — segmentação lógica

VLANs (IEEE 802.1Q) segmentam uma rede física em múltiplas redes virtuais independentes sem precisar de hardware adicional. Cada VLAN é um domínio de broadcast separado dentro do mesmo switch.

Configuração básica VLAN (Cisco IOS)
# Porta de acesso (um dispositivo, uma VLAN)
interface GigabitEthernet0/1
 switchport mode access
 switchport access vlan 10

# Porta trunk (transporta múltiplas VLANs entre switches)
interface GigabitEthernet0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30

Tendências e Futuro das Redes

SDN — Redes Definidas por Software

Separa o plano de controle (decisões de roteamento) do plano de dados (encaminhamento). Um controlador centralizado (OpenDaylight, ONOS) programa os switches via OpenFlow. Resultado: redes programáveis, automatizáveis e totalmente auditáveis.

5G e além

5G promete até 20 Gbps teóricos e latência <1 ms via network slicing. Habilita IoT massivo (1 milhão de dispositivos/km²), carros autônomos e cirurgias remotas. 6G em pesquisa com estimativa de implantação em 2030.

Zero Trust Networking

Substitui "confiar em quem está dentro do perímetro" por "nunca confie, sempre verifique". Cada acesso é autenticado e autorizado independentemente. Base da segurança moderna em ambientes cloud e híbridos.

Redes Quânticas

Comunicação baseada em quantum key distribution (QKD) — qualquer interceptação altera o estado dos qubits e é detectável. Ainda experimental, mas China opera a maior rede QKD do mundo (~4600 km).

RFCs Essenciais

  • RFC 791 — Internet Protocol (IPv4)

  • RFC 826 — Address Resolution Protocol (ARP)
  • RFC 1034/1035 — Domain Name System (DNS)
  • RFC 2460 — Internet Protocol Version 6 (IPv6)
  • RFC 4632 — CIDR (Classless Inter-Domain Routing)

Ferramentas Práticas

  • Wireshark — analisador de protocolos (wireshark.org)
  • GNS3 / Packet Tracer — simuladores de rede
  • Nmap — scanner de rede e descoberta de hosts
  • ipcalc / sipcalc — calculadoras de sub-rede no terminal
  • NetworkLessons.com — tutoriais aprofundados com labs
