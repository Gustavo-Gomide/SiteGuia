POP3 e IMAP

Protocolos de acesso a e-mail: download local (POP3) vs sincronização no servidor (IMAP)

Enquanto o SMTP envia e-mail, o POP3 (RFC 1939) e o IMAP (RFC 9051) permitem ao cliente acessar e-mails armazenados no servidor. A diferença central: POP3 foi projetado para baixar e remover do servidor; IMAP para sincronizar entre múltiplos dispositivos mantendo mensagens no servidor.

POP3 vs IMAP

CaracterísticaPOP3IMAP

Comandos Principales

POP3 — Sessão

POP3 via telnet (porta 110)
S: +OK POP3 server ready
C: USER user@example.com
S: +OK
C: PASS senha
S: +OK Logged in
C: STAT          # conta msgs e tamanho total
S: +OK 3 14478
C: LIST          # lista msgs com tamanhos
S: +OK 3 messages
C: RETR 1        # baixa mensagem 1
S: +OK 4859 octets
[mensagem...]
C: DELE 1        # marca para deleto (efetivo no QUIT)
C: QUIT
S: +OK

IMAP — Sessão

IMAP4 (simplificado)
S: * OK IMAP4rev2 server ready
C: a001 LOGIN user@example.com senha
S: a001 OK Logged in
C: a002 LIST "" "*"   # lista pastas
S: * LIST (\HasNoChildren) "/" INBOX
S: * LIST (\HasNoChildren) "/" Sent
S: a002 OK List completed
C: a003 SELECT INBOX   # abre caixa
S: * 42 EXISTS         # 42 mensagens
S: * 3 RECENT
C: a004 FETCH 1 BODY[] # baixa msg 1
C: a005 STORE 1 +FLAGS (\Seen)  # marca lida
C: a006 SEARCH UNSEEN  # busca não lidas
C: a007 LOGOUT

Continue: SMTP, TLS (portas 993/995).

"POP3 foi criado quando cada pessoa tinha um computador. IMAP foi criado para o mundo multidispositivo. O webmail popularizou o IMAP sem que a maioria soubesse."
