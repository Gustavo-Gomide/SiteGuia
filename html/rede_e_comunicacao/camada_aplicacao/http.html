HTTP — HyperText Transfer Protocol

O protocolo da web: request-response, stateless, que alimenta toda a internet

O HTTP (RFC 9110–9114) é um protocolo de camada de aplicação de modelo request–response e stateless. O cliente envia uma requisição e o servidor responde. Estado é mantido por cookies/sessões. HTTP/1.1 corre sobre TCP; HTTP/2 também; HTTP/3 (QUIC) usa UDP.

Códigos de Status

Classes

  • 1xx — Informacional: 100 Continue, 101 Switching Protocols (WebSocket upgrade)
  • 2xx — Sucesso: 200 OK, 201 Created, 204 No Content
  • 3xx — Redirecionamento: 301 Moved Permanently, 302 Found, 304 Not Modified (cache)
  • 4xx — Erro do cliente: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 405 Method Not Allowed, 429 Too Many Requests
  • 5xx — Erro do servidor: 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable, 504 Gateway Timeout

Requisição e Resposta

Exemplo HTTP/1.1
# Requisição
GET /index.html HTTP/1.1
Host: example.com
Accept: text/html
Accept-Encoding: gzip, deflate, br
Connection: keep-alive

# Resposta
HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Content-Length: 1234
Cache-Control: max-age=3600
Set-Cookie: session=abc123; HttpOnly; Secure

<!DOCTYPE html>...
