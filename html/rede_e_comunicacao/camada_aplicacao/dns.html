DNS — Domain Name System

A agenda de contatos da internet: traduz nomes de domínio em endereços IP

O DNS (RFC 1034/1035) é um banco de dados distribuído e hierárquico que resolve nomes de domínio (example.com) em endereços IP. Opera na porta 53 (UDP para consultas ≤512 bytes; TCP para zone transfers e respostas DNSSEC grandes). É o protocolo mais crítico da internet — sem DNS, quase nada funciona.

Segurança DNS

DNSSEC

Adiciona assinaturas criptográficas (RSA/ECDSA) aos registros DNS. O validador verifica a cadeia de confiança até o root. Protege contra cache poisoning (Kaminsky attack). Não criptografa — apenas autentica.

DoH e DoT

  • DoT (DNS over TLS, porta 853): encapsula consultas DNS em TLS; perceptivelmente últil no nível de rede (ISP vê que é DNS mas não o conteúdo)
  • DoH (DNS over HTTPS, porta 443): encapsula DNS em HTTPS; indiferenciável de tráfego web; maior privacidade; usado por Chrome, Firefox e Windows 11

DNS Cache Poisoning

Atacante injeta respostas falsas no cache do resolver para redirecionar tráfego. Mitigado por: port randomization (Djkstra, 2008), DNSSEC, 0x20 bit randomization.
