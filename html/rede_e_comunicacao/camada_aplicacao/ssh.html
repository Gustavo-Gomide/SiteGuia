SSH — Secure Shell

Terminal remoto seguro com criptografia forte, autenticação por chave e tunelamento

O SSH (RFC 4251–4254) substitui o Telnet, rsh e rlogin com criptografia end-to-end. Corre em TCP/22. Além do acesso à shell, SSH suporta transferência de arquivos (SCP/SFTP), tunelamento de portas (local, remoto, dinâmico) e multiplexamento de canais num único TCP.

Autenticação por Chave Pública

Gerar e distribuir chave

Gerando par de chaves SSH
# Gerar chave Ed25519 (recomendado)
ssh-keygen -t ed25519 -C "user@machine" -f ~/.ssh/id_ed25519

# Ou RSA 4096 (compatível com sistemas legados)
ssh-keygen -t rsa -b 4096 -C "user@machine"

# Copiar chave pública ao servidor
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server
# Equivalente: cat ~/.ssh/id_ed25519.pub >> ~/.ssh/authorized_keys (no server)

# Conectar usando a chave
ssh -i ~/.ssh/id_ed25519 user@server

Como funciona a autenticação

  1. Servidor verifica se a chave pública está em ~/.ssh/authorized_keys
  2. Servidor gera um desafio aleatório criptografado com a chave pública do cliente
  3. Cliente decifra o desafio com sua chave privada e envia o resultado
  4. Servidor verifica — autenticado sem a senha nunca ter trafegado pela rede

Port Forwarding (Tunelamento)

Local Forward (-L)

Encaminha uma porta local para um destino via servidor SSH. Acesse serviços internos da rede remota:

Local forward
# Acessar banco na rede remota localmente
ssh -L 5432:db-interno:5432 user@jump-server
# Agora: psql -h localhost -p 5432 ...

Remote Forward (-R)

Expõe uma porta local no servidor remoto. Permite acesso reverso:

Remote forward (reverse tunnel)
# Expor serviço local na porta 8080 no servidor
ssh -R 8080:localhost:3000 user@public-server
# No servidor: acessa localhost:8080 e chega ao client:3000

Dynamic (SOCKS proxy) (-D)

SOCKS5 proxy via SSH
# Cria proxy SOCKS5 local na porta 1080
ssh -D 1080 user@server
# Configurar navegador para usar SOCKS5 127.0.0.1:1080
# Todo o tráfego passa pelo SSH (VPN pobre)

Hardening do Servidor SSH

/etc/ssh/sshd_config — configurações recomendadas
# Desabilitar login root
PermitRootLogin no

# Desabilitar senha (apenas chave pública)
PasswordAuthentication no
ChallengeResponseAuthentication no

# Apenas IPv4/IPv6 específico
AddressFamily inet

# Porta não padrão (dificulta bots)
Port 2222

# Limite de tentativas de autenticação
MaxAuthTries 3

# Tempo máximo para autenticar
LoginGraceTime 30s

# Permitir só alguns usuários
AllowUsers deployer sysadmin

# Algoritmos modernos apenas
KexAlgorithms curve25519-sha256,diffie-hellman-group16-sha512
HostKeyAlgorithms ssh-ed25519,rsa-sha2-512
Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com
