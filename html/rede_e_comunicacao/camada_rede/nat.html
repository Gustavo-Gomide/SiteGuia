NAT — Network Address Translation

Tradução de endereços que permitiu a internet sobreviver ao esgotamento do IPv4

O NAT (RFC 3022) é executado pelo roteador de borda e traduz endereços IP (e portas) entre a rede privada e a internet. Possibilita que milhares de dispositivos com IPs privados (RFC 1918) compartilhem um único IP público. É o mecanismo que adiou o fim do IPv4 por duas décadas.

Tipos de NAT

NAT Estático (1:1)

Mapeia um IP privado fixo para um IP público fixo. Usado para servidores internos que precisam ser acessíveis de fora (web server, FTP server).

Cisco IOS — NAT Estático
ip nat inside source static 192.168.1.10 203.0.113.10

interface GigabitEthernet0/0
 ip nat inside
interface GigabitEthernet0/1
 ip nat outside

NAT Dinâmico (N:M)

Mapeia IPs privados para um pool de IPs públicos. Quando o pool acaba, novas conexões são bloqueadas.

Cisco IOS — NAT Dinâmico
ip nat pool MEUPOOL 203.0.113.1 203.0.113.10 prefix-length 28
ip nat inside source list 1 pool MEUPOOL
access-list 1 permit 192.168.1.0 0.0.0.255

PAT / Masquerade (N:1) — mais comum

Também chamado NAT Overload. Traduz múltiplos IPs privados para um único IP público, diferenciados por porta de origem. É o que ocorre em roteadores domésticos e corporativos.

Cisco IOS — PAT (overload)
ip nat inside source list 1 interface GigabitEthernet0/1 overload
access-list 1 permit 192.168.0.0 0.0.255.255

Problemas e Limitações

ALG — Application Layer Gateway

Protocolos que embutem IPs no payload (FTP ativa, SIP, H.323) quebram com NAT. O ALG no roteador inspeciona e reescreve o payload. Pode causar incompatibilidades com VoIP e conferências.

NAT Traversal (NAT-T)

IPsec em modo túnel (ESP) não carrega informação de porta — o NAT não consegue desmultiplexar. NAT-T (RFC 3948) encapsula o ESP dentro de UDP/4500 para atravessar NAT.

NAT64

Tradução entre IPv6 e IPv4: hosts IPv6-only acessam servidores IPv4-only. O gateway NAT64 usa o prefixo 64:ff9b::/96 + DNS64 que sintetiza registros AAAA para hosts sem IPv6.
