Camada de Rede

Roteamento IP

Como roteadores tomam decisões de encaminhamento: tabela de roteamento, LPM e distâncias administrativas

Roteamento é o processo pelo qual um roteador decide para qual interface encaminhar um pacote IP. A decisão é baseada na tabela de roteamento usando o algoritmo Longest Prefix Match (LPM): a rota com a máscara mais longa (mais específica) vence.

 S*   0.0.0.0/0 [1/0] via 10.0.0.1   <-- default route

Componentes de uma entrada

  • Cödigo: origem da rota (C=direta, S=estática, O=OSPF, etc.)
  • Prefixo/máscara: rede de destino
  • [AD/métrica]: AD = distância administrativa; métrica = custo interno ao protocolo
  • via NEXT-HOP: endereço IP do próximo salto
  • Interface de saída: interface local pelo qual o pacote sai

Distância Administrativa (AD)

Desempata rotas de origens diferentes para o mesmo destino — menor AD = mais confiável:

AD padrão Cisco
Direta (C)         0
Rota estática (S)  1
EBGP               20
EIGRP sumário      5
EIGRP interno      90
OSPF               110
IS-IS              115
RIP                120
EIGRP externo      170
IBGP               200

Tipos de Rota

Rota Direta (Connected)

Adicionada automaticamente quando uma interface é configurada com IP e está up/up. AD = 0; é sempre a mais confiável.

Rota Estática

Configurada manualmente pelo administrador. Útil para redes pequeñas ou para links de backup.

Cisco IOS
# Rota para rede específica
ip route 10.10.10.0 255.255.255.0 10.0.0.2

# Rota padrão (default route)
ip route 0.0.0.0 0.0.0.0 10.0.0.1

# Rota flutuante (backup, AD alta)
ip route 10.10.10.0 255.255.255.0 10.0.0.3 200

ECMP — Equal-Cost Multi-Path

Quando existem múltiplas rotas com o mesmo AD e mesma métrica para o mesmo destino, o roteador balanceia carga entre elas. Cisco usa até 4 caminhos por padrão (máx 16 com EIGRP/OSPF).

CEF — Cisco Express Forwarding

Mecanismo de hardware/software para encaminhamento rápido de pacotes:

  • FIB (Forwarding Information Base): versão pré-computada da tabela de roteamento, pronta para LPM instantâneo
  • Adjacency Table: armazena informações de Camada 2 (MAC de próximo salto, rewrite de header) para cada próximo salto
  • Elimina a necessidade de consultar a tabela ARP a cada pacote

Continue: Protocolos de Roteamento, NAT, ICMP.

"O roteador não rota pacotes — encaminha. Rotear é calcular o melhor caminho; encaminhar é movêr o pacote. São planos distintos: control-plane vs. data-plane."
