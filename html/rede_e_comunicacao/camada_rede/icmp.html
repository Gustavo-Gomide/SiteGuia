ICMP — Internet Control Message Protocol

O mensageiro do IP: relata erros de entrega e viabiliza diagnósticos (ping e traceroute)

O ICMP (RFC 792 para IPv4; RFC 4443 para ICMPv6) é encapsulado diretamente no IP (Protocol = 1). Não garante entrega de pacotes — seu papel é reportar erros e fornecer funções de diagnóstico. Firewalls frequentemente bloqueiam ICMP; isso pode quebrar PMTUD e dificultar o troubleshooting.

Ping e Traceroute

Ping

Envia ICMP Echo Request (Type 8) e espera Echo Reply (Type 0). Mede RTT (round-trip time) e perda de pacotes.

Exemplos de ping
# Linux — 5 pings com tamanho de 1400 bytes
ping -c 5 -s 1400 8.8.8.8

# Windows
ping -n 5 -l 1400 8.8.8.8

# Cisco IOS — extended ping
ping 8.8.8.8 size 1400 repeat 5 df-bit

Traceroute

Descobre o caminho até o destino incrementando o TTL: TTL=1 descobre hop 1; TTL=2 descobre hop 2; etc. Cada hop responde com ICMP Time Exceeded (Type 11).

Traceroute Linux (UDP) vs Windows (ICMP)
# Linux: usa UDP por padrão (portas altas > 33434)
traceroute 8.8.8.8

# Linux com ICMP (como Windows)
traceroute -I 8.8.8.8

# Windows usa ICMP Echo Request
tracert 8.8.8.8

# Cisco IOS usa UDP por padrão
traceroute 8.8.8.8

ICMPv6 e NDP

O ICMPv6 (RFC 4443) é muito mais rico que o ICMP do IPv4 e incorpora funções que eram feitas por ARP e IGMP no IPv4:

Tipos ICMPv6 importantes

  • Type 1 — Destination Unreachable
  • Type 2 — Packet Too Big (equivalente ao ICMP Type 3 Code 4 do IPv4; usado por PMTUD)
  • Type 3 — Time Exceeded (traceroute IPv6)
  • Type 128/129 — Echo Request/Reply (ping IPv6)
  • Type 133 — Router Solicitation (RS): host descobre roteadores ao iniciar
  • Type 134 — Router Advertisement (RA): roteador anuncia prefixo + gateway; habilita SLAAC
  • Type 135 — Neighbor Solicitation (NS): substitui ARP Request
  • Type 136 — Neighbor Advertisement (NA): substitui ARP Reply

PMTUD e ICMP Blocking

PMTUD depende do roteamento correto dos ICMP Type 3 Code 4 (IPv4) / Type 2 (IPv6). Se firewalls bloquearem esses ICMPs, o cliente nunca reduz o MSS e conexões TCP grandes ficam travadas (“black hole”). Solucionar: ip tcp adjust-mss 1452 nos roteadores de borda ou liberar ICMP Unreachable no firewall.
