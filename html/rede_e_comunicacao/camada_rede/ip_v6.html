IPv6 — Internet Protocol v6

128 bits, 340 undecilhões de endereços: o substituto do IPv4 projetado para durar séculos

O IPv6 (RFC 8200) usa endereços de 128 bits em vez de 32, resolvendo o esgotamento do IPv4. Além da quantidade de endereços, o IPv6 simplificou o cabeçalho, eliminou broadcasts, tornou o checksum de Camada 3 opcional e trouxe configuração automática (SLAAC) nativa. IPv6 é a norma de hoje em data centers modernos, mobile e redes de operadoras.

Notação e Tipos de Endereço

Notação Hexadecimal

8 grupos de 4 hex separados por :. Regras de compressão:

  • Zeros à esquerda de cada grupo podem ser omitidos: 0064 → 64
  • Um ou mais grupos consecutivos de 0000 podem ser substituídos por :: — só uma vez por endereço: 2001:0db8:0000:0000:0000:0000:0000:0001 → 2001:db8::1
Exemplos
::1                  # loopback (127.0.0.1 do IPv6)
::                   # endereço não especificado
fe80::1              # link-local
2001:db8::1          # GUA de documentação
fc00::1              # ULA
ff02::1              # all nodes (multicast link-local)
ff02::2              # all routers
ff02::1:ff00:1       # solicited-node multicast (NDP)

Tipos de Endereço IPv6

  • GUA (Global Unicast): 2000::/3; roteável globalmente; prefixo 2001::/16 (IANA) ou blocos atribuídos pelo ISP; equivalente ao IP público IPv4
  • LLA (Link-Local): fe80::/10; gerado automaticamente em toda interface; válido apenas no link local; usado para NDP, OSPF, protocolos de roteamento
  • ULA (Unique Local): fc00::/7 (incluindo fd00::/8); equivalente ao RFC 1918 do IPv4; roteável dentro da organização, não para a internet
  • Multicast: ff00::/8; substitui broadcast do IPv4. ff02::1 = all nodes; ff02::2 = all routers
  • Loopback: ::1/128 — equivalente a 127.0.0.1
  • Não especificado: ::/128 — equivalente a 0.0.0.0

EUI-64: Interface ID a partir do MAC

O SLAAC gera automaticamente o Interface ID de 64 bits a partir do MAC de 48 bits usando o método EUI-64:

EUI-64: MAC → Interface ID de 64 bits
MAC original:   AA:BB:CC:DD:EE:FF

1. Dividir ao meio:     AA:BB:CC  |  DD:EE:FF
2. Inserir FF:FE no meio: AA:BB:CC:FF:FE:DD:EE:FF
3. Inverter bit U/L (bit 6 do 1º octeto):
   AA = 1010 1010 → inverter bit 6 → 1010 1000 = A8

Interface ID resultante: A8BB:CCFF:FEDD:EEFF

Endereço completo (com prefixo fe80::/10):
fe80::A8BB:CCFF:FEDD:EEFF

Muitos SOs modernos usam Privacy Extensions (RFC 4941) que geram Interface IDs aleatórios temporarios para evitar rastreamento por EUI-64.

Cabeçalho IPv6 (40 bytes fixos)

Campos do Cabeçalho

  • Version (4b): sempre 6
  • Traffic Class (8b): equivalente ao DSCP/ECN do IPv4
  • Flow Label (20b): identifica fluxos para tratamento especial por roteadores (QoS, MPLS)
  • Payload Length (16b): tamanho do payload após o cabeçalho base (inclui extension headers)
  • Next Header (8b): tipo do próximo header (protocolo do payload ou extension header); funciona como o campo Protocol do IPv4
  • Hop Limit (8b): equivalente ao TTL do IPv4
  • Source / Destination (128b cada): endereços IPv6

O que foi removido vs IPv4

  • Sem Header Checksum: confiado às camadas inferiores (Ethernet FCS) e superiores (TCP/UDP checksum)
  • Sem Fragmentação em roteadores: apenas o host de origem pode fragmentar (via Extension Header); roteadores retornam ICMPv6 Packet Too Big. PMTUD obrigatório
  • Sem Options no header base: substituidas por Extension Headers encadeados (Hop-by-Hop, Routing, Fragment, Destination, AH, ESP)
  • Sem IHL: cabeçalho base é sempre 40 bytes fixos
  • Sem Broadcast: substituído por multicast específico (all nodes = ff02::1)

Transição IPv4 ↔ IPv6

Dual Stack

O host ou roteador possui ambos IPv4 e IPv6 ativos simultaneamente. Usa IPv6 quando possível, cai para IPv4 se necessário. Método preferido durante transição.

Tunneling

Encapsula IPv6 dentro de IPv4 para traversar redes que só entendem IPv4:

  • 6in4 / 6to4: manualmente configurado ou automático via anycast 2002::/16
  • Teredo: IPv6 via UDP/IPv4 (atravessa NAT); obsoleto em favor de dual-stack
  • ISATAP: intra-site automatic tunnel

NAT64 / DNS64

Permite que hosts IPv6-only se comuniquem com servidores IPv4-only:

  • DNS64: servidor DNS sintetiza registros AAAA fictícios para domínios que só têm registros A
  • NAT64: traduz pacotes IPv6 ↔ IPv4 no ponto de saída da rede
  • Prefixo NAT64 padrão: 64:ff9b::/96
