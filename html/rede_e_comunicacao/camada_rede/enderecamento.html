Endereçamento IP

Espaço de endereços IPv4: classes, privados, públicos, especiais e o esgotamento que incentivou o IPv6

Um endereço IPv4 é um número de 32 bits (4 bytes), escrito em notação decimal pontilhada (ex: 192.168.1.10). Ele identifica univocamente uma interface na rede e carrega em si a informação de qual rede o host pertence (bits de rede) e qual host dentro da rede ele é (bits de host). O total de endereços IPv4 é 232 ≈ 4,3 bilhões.

Classes Históricas (Classful, pré-CIDR)

Até 1993, o espaço IPv4 era dividido em classes pelos bits iniciais. Esse sistema classful foi substituído pelo CIDR (RFC 1519), mas ainda aparece em provas e sistemas legados:

Endereços Especiais

Loopback e Link-Local

  • 127.0.0.0/8 (Loopback): qualquer pacote enviado para esse bloco é devolvido imediatamente ao próprio host sem sair pela interface; 127.0.0.1 = localhost; útil para testar serviços localmente
  • 169.254.0.0/16 (APIPA — Link-Local): atribuído automaticamente pelo SO quando DHCP falha (Windows/Linux); não é roteável; permite comunicação limitada na mesma LAN; RFC 3927

Broadcast e Network

  • 255.255.255.255: broadcast limitado; enviado para todos na mesma rede local sem traversar roteadores (ex: DHCPDiscover)
  • Endereço de rede: todos os bits de host = 0; identifica a rede (não atribuído a hosts). Ex: 192.168.1.0/24
  • Broadcast direcionado: todos os bits de host = 1; endereça todos os hosts de uma rede específica. Ex: 192.168.1.255/24
  • 0.0.0.0: "this host, this network"; usado em DHCPDISCOVER como IP de origem antes de ter IP; também usado como rota padrão (default route) em tabelas de roteamento

Multicast (Classe D)

  • 224.0.0.0/4: endereços de multicast IPv4
  • 224.0.0.0/24 (link-local): não roteado; protocolos de rede local
  • 224.0.0.1 — todos os hosts do segmento
  • 224.0.0.2 — todos os roteadores
  • 224.0.0.5/6 — OSPF (todos os roteadores / DR)
  • 224.0.0.9 — RIPv2
  • 224.0.0.18 — VRRP
  • 239.0.0.0/8 (escopo administrativo): multicast local da organização, não roteado para internet
