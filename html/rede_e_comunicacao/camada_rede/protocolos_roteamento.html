Protocolos de Roteamento Dinâmico

RIP, OSPF, EIGRP e BGP: como roteadores aprendem rotas automaticamente

Protocolos de roteamento dinâmico permitem que roteadores troquem informações de alcance e calculem automaticamente os melhores caminhos. São divididos em IGP (Interior Gateway Protocol — dentro de um AS) e EGP (Exterior Gateway Protocol — entre ASes). BGP é o protocolo que faz funcionar a internet pública.

Algoritmos: Distância-Vetor vs Estado-de-Enlace

Distância-Vetor (Distance Vector)

  • Cada roteador conhece apenas a distância (métrica) e direção (próximo salto) para cada rede — "routing by rumor"
  • Envia tabela completa de rotas periodicamente para vizinhos diretamente conectados
  • Converge devagar; suscetível a loops (Bellman-Ford)
  • Mecanismos anti-loop: split horizon, route poisoning, hold-down timers
  • Exemplos: RIP, EIGRP (híbrido)

Estado-de-Enlace (Link State)

  • Cada roteador tem visão completa da topologia da rede (LSDB — Link-State Database)
  • Envia LSAs (Link State Advertisements) inundados por toda a área
  • Usa algoritmo Dijkstra (SPF) para calcular árvore de menor custo
  • Converge rápido; escalável com mecanismo de áreas
  • Exemplos: OSPF, IS-IS

OSPF em Detalhe

Conceitos fundamentais

  • LSDB: banco de dados de estado de enlace; idêntico em todos os roteadores da mesma área
  • Área 0 (backbone): todas as demais áreas devem se conectar ao backbone; evita loops de roteamento inter-área
  • DR/BDR: em redes Ethernet multi-acesso, eleição de Designated Router e Backup; reduz número de adjacências (n² vs n)
  • Tipos de LSA: Type 1 (router LSA), Type 2 (network LSA — gerado pelo DR), Type 3 (summary LSA gerado pelo ABR), Type 5 (external LSA gerado pelo ASBR)
