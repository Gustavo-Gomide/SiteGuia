Subnetting

Dividir uma rede maior em sub-redes menores: FLSM, VLSM e o método do magic number

O subnetting divide um bloco de endereços IP em sub-redes menores, permitindo organizar a rede por departamento, andar ou função, reduzir domínios de broadcast e aumentar segurança. O FLSM usa sub-redes de tamanho igual; o VLSM usa tamanhos diferentes para otimizar o uso do espaço de endereços.

Método do Magic Number

O Método

O magic number = 256 − valor do octeto da máscara no octeto de interesse. Ele define o incremento entre sub-redes consecutivas.

Magic Number
/25 → máscara 255.255.255.128
Octeto de interesse: 4º (128)
Magic = 256 - 128 = 128
Sub-redes: .0, .128

/26 → máscara 255.255.255.192
Magic = 256 - 192 = 64
Sub-redes: .0, .64, .128, .192

/27 → máscara 255.255.255.224
Magic = 256 - 224 = 32
Sub-redes: .0, .32, .64, .96, .128, .160, .192, .224

Passo a Passo FLSM

Problema: dividir 192.168.1.0/24 em 4 sub-redes iguais.

  1. Quantas sub-redes? 4 → precisa de 2 bits emprestados (2²=4)
  2. Novo prefixo: /24 + 2 = /26
  3. Magic = 256 - 192 = 64

VLSM — Variable Length Subnet Mask

O VLSM permite criar sub-redes de tamanhos diferentes a partir do mesmo bloco, alocando com eficiência endereços para redes de tamanhos variados.

Regra VLSM: sempre alocar primeiro a maior sub-rede. Isso garante alinhamento de blocos.
