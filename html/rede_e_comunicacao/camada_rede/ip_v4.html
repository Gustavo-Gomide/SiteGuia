Camada de Rede

IPv4 — Internet Protocol v4

O protocolo que move pacotes através da internet desde 1981 — RFC 791

O IPv4 (RFC 791) opera na Camada 3 e é responsável pelo endereçamento lógico e pelo encaminhamento de datagramas (pacotes) entre redes. É um protocolo best-effort — sem garantia de entrega, ordem ou tempo. Essas garantias ficam a cargo das camadas superiores (TCP).

Campos principais

  • Version (4b): sempre 4 para IPv4
  • IHL — Internet Header Length (4b): tamanho do cabeçalho em words de 32 bits; mínimo 5 (= 20 bytes); máximo 15 (= 60 bytes com opções)
  • DSCP (6b): Differentiated Services Code Point; classifica o pacote para QoS (ex: EF = Expedited Forwarding para VoIP)
  • ECN (2b): Explicit Congestion Notification; sinaliza congestionamento sem descartar pacotes
  • Total Length (16b): tamanho total do datagrama (header + data) em bytes; máx 65.535 B
  • Identification (16b): identifica fragmentos do mesmo datagrama original
  • Flags (3b): bit 0 = Reserved; bit 1 = DF (Don't Fragment); bit 2 = MF (More Fragments)
  • Fragment Offset (13b): posição do fragmento em unidades de 8 bytes
  • TTL (8b): decrementado em 1 por cada roteador; ao chegar a 0, pacote é descartado e ICMP Time Exceeded é enviado ao remetente
  • Protocol (8b): protocolo da camada acima encapsulado no payload
  • Header Checksum (16b): verificação da integridade do header apenas (não do payload); recalculado a cada hop (TTL muda)
  • Source / Destination (32b cada): endereços IP

Fragmentação

Como funciona

Quando um roteador precisa encaminhar um datagrama maior que a MTU do próximo link, ele o fragmenta em datagramas menores. O destino final remonta os fragmentos usando ID, MF bit e Fragment Offset.

  • DF bit = 1: "não fragmente"; se o pacote for maior que a MTU do link, o roteador descarta e envia ICMP Fragmentation Needed (type 3 code 4) ao remetente. Usado por PMTUD (Path MTU Discovery)
  • PMTUD: o remetente descobre a menor MTU no caminho e reduz seus segmentos; imprescindível para evitar fragmentação em redes com VPNs (overhead de encapsulação)
  • Fragmentação é ineficiente e problemática; preferir PMTUD + DF bit ao invés de fragmentar em routers

TTL e Traceroute

O TTL (Time to Live) foi originalmente pensado como um tempo em segundos, mas na prática funciona como hop count: cada roteador decrementa em 1.

  • Linux: TTL padrão = 64; Windows = 128; Cisco IOS = 255
  • Ao chegar a 0: roteador descarta o pacote e responde ao remetente com ICMP Time Exceeded (type 11)
  • Traceroute explora isso: envia pacotes com TTL=1, 2, 3… e recebe os ICMPs de cada hop, revelando o caminho até o destino
