Camada de Rede

CIDR — Classless Inter-Domain Routing

Notação /prefixo: o fim das classes A/B/C e a base do endereçamento moderno

O CIDR (RFC 4632, anteriormente RFC 1519) substituiu as classes históricas em 1993. Em vez de usar classes fixas com máscaras de 8, 16 ou 24 bits, o CIDR usa um prefixo de comprimento variável — a notação /N indica quantos bits são a parte de rede. Isso permite criar blocos de qualquer tamanho e agregar rotas no BGP, reduzindo o tamanho das tabelas de roteamento.

Cálculo de Rede, Broadcast e Hosts

Fórmulas

  • Máscara: N bits 1 seguidos de (32-N) bits 0
  • Endereço de rede: IP AND máscara (todos bits de host = 0)
  • Broadcast: IP OR ~máscara (todos bits de host = 1)
  • Hosts válidos: 2(32-N) − 2 (sem rede e broadcast)

Exemplo: 192.168.10.35/27

Cálculo passo a passo
/27 = máscara 255.255.255.224
224 = 1110 0000 → incremento = 256-224 = 32

Blocos /27 em .0: 0, 32, 64, 96, 128 ...
IP 35 → está no bloco que começa em 32

Rede:      192.168.10.32
Broadcast: 192.168.10.63
Hosts:     192.168.10.33 – .62  (30 hosts)
Mask:      255.255.255.224  (/27)

Agregação de Rotas (Supernetting)

Blocos CIDR contiguos podem ser agregados em um único prefixo maior, reduzindo o número de entradas nas tabelas de roteamento (especialmente no BGP):

Exemplo de agregação
# 4 redes /24 contigúas → 1 rota /22
192.168.0.0/24
192.168.1.0/24
192.168.2.0/24
192.168.3.0/24
→ agregado: 192.168.0.0/22  (4.094 hosts)

# Regra: blocos devem ser contigúos E
# o endereço de rede do bloco maior deve
# ser o mesmo que o do menor bloco.

# Exemplo inválido (não é bloco alinhado):
192.168.1.0/24  +  192.168.2.0/24
→ NÃO forma /23 válido (seria 192.168.0.0/23 ou 192.168.2.0/23)
