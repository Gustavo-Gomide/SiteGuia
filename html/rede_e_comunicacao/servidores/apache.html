Apache HTTP Server

MPMs, VirtualHost, módulos, .htaccess e performance tuning

O Apache HTTP Server (httpd) é o servidor web mais antigo ainda amplamente utilizado. Criado em 1995 pela Apache Software Foundation, destaca-se pela arquitetura modular e pelo sistema de configuração por diretório via .htaccess.

MPMs — Multi-Processing Modules

O Apache seleciona o modelo de processamento através dos MPMs. A escolha do MPM correto é crítica para performance e compatibilidade com o stack da aplicação.

MPMProcessosThreadsModeloUso Típico
preforkN (1/conn)1Process-per-requestPHP via mod_php (não thread-safe)
workerPoucosN/processoThread-per-requestConteúdo dinâmico sem mod_php
eventPoucosN + keep-alive handlerAsync keep-aliveAlta concorrência, PHP-FPM, py, ruby
/etc/apache2/mods-available/mpm_event.conf
<IfModule mpm_event_module>
    StartServers          2
    MinSpareThreads      25
    MaxSpareThreads      75
    ThreadLimit          64
    ThreadsPerChild      25
    MaxRequestWorkers   150
    MaxConnectionsPerChild 0
</IfModule>

Estrutura de Configuração

O Apache usa uma hierarquia de arquivos bem definida no Debian/Ubuntu. Entender essa estrutura é essencial para ativar módulos e virtual hosts corretamente.

Hierarquia de Arquivos (Debian)

/etc/apache2/
├── apache2.conf        # config principal
├── ports.conf          # Listen 80, 443
├── mods-available/     # módulos disponíveis
├── mods-enabled/       # links simbólicos
├── sites-available/    # virtual hosts
├── sites-enabled/      # links simbólicos
└── conf-available/     # configs extras

Diretivas Globais Importantes

ServerRoot      "/etc/apache2"
ServerName      example.com
Listen          80

KeepAlive       On
MaxKeepAliveRequests 100
KeepAliveTimeout     5

Timeout         60
LogLevel        warn

VirtualHost — Name-based

O Apache usa o bloco <VirtualHost> para hospedar múltiplos domínios. O padrão ativo é sempre redirecionar HTTP para HTTPS e separar a config de cada domínio em arquivos dedicados.

HTTP → HTTPS redirect

<VirtualHost *:80>
    ServerName  example.com
    Redirect permanent / https://example.com/
</VirtualHost>

HTTPS + proxy para app server

<VirtualHost *:443>
    ServerName  example.com
    DocumentRoot /var/www/html

    SSLEngine on
    SSLCertificateFile    /etc/letsencrypt/live/example.com/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/example.com/privkey.pem
    SSLProtocol all -SSLv3 -TLSv1 -TLSv1.1

    ProxyPreserveHost On
    ProxyPass        /app/ http://127.0.0.1:8000/app/
    ProxyPassReverse /app/ http://127.0.0.1:8000/app/

    ErrorLog  ${APACHE_LOG_DIR}/example_error.log
    CustomLog ${APACHE_LOG_DIR}/example_access.log combined
</VirtualHost>

.htaccess — Configuração por Diretório

O .htaccess permite sobrescrever configurações do servidor por diretório sem reiniciar o Apache. Exige AllowOverride All no VirtualHost. Em produção, prefira mover as regras para o VirtualHost e usar AllowOverride None para evitar overhead por requisição.

Redirect & Rewrite

Options -Indexes
RewriteEngine On

# Force HTTPS
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# SPA fallback
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.html [L]

Cabeçalhos de Segurança

Header always set X-Content-Type-Options "nosniff"
Header always set X-Frame-Options "SAMEORIGIN"
Header always set Referrer-Policy "strict-origin-when-cross-origin"
Header always set Strict-Transport-Security \
  "max-age=31536000; includeSubDomains"

Cache de Assets Estáticos

<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType image/webp  "access plus 1 year"
    ExpiresByType text/css    "access plus 6 months"
    ExpiresByType application/javascript "access plus 6 months"
</IfModule>

Comandos de Gerenciamento

Os comandos essenciais para administrar o Apache no dia a dia — sempre valide a configuração antes de recarregar.

Bash — administração Apache (systemd)
systemctl status apache2
systemctl reload apache2   # reload sem derrubar conexões existentes
apache2ctl configtest       # deve retornar "Syntax OK"
apachectl -t -D DUMP_VHOSTS # listar virtual hosts
a2ensite  meusite.conf  ;  a2dissite  000-default.conf
a2enmod   rewrite ssl   ;  a2dismod   status
tail -f /var/log/apache2/error.log
