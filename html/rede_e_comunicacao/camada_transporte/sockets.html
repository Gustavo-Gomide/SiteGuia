Camada de Transporte

Sockets

A interface entre aplicação e rede — a API que toda comunicação em rede usa

Um socket é um endpoint de comunicação identificado pela 5-tupla: {protocolo, IP-local, porta-local, IP-remoto, porta-remota}. A API Berkeley Sockets (BSD 4.2, 1983) é a interface padrão em praticamente todos os SOs e linguagens de programação para comunicação em rede.

Ciclo de Vida do Socket

Servidor TCP

Python — Servidor TCP
import socket

# 1. Criar socket (AF_INET=IPv4, SOCK_STREAM=TCP)
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# 2. Reutilizar porta após restart
server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)

# 3. Vincular à interface e porta
server.bind(('0.0.0.0', 8080))

# 4. Escutar conexões (backlog = fila)
server.listen(5)
print('Aguardando conexões...')

# 5. Aceitar conexão (bloqueia até chegar)
conn, addr = server.accept()
print(f'Conectado de {addr}')

# 6. Ler e responder
data = conn.recv(1024)       # buffer de 1024 bytes
conn.send(b'HTTP/1.1 200 OK\r\n\r\nHello!')

# 7. Fechar
conn.close()
server.close()

Cliente TCP

Python — Cliente TCP
import socket

client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# connect() = 3-way handshake TCP
client.connect(('example.com', 80))

client.send(b'GET / HTTP/1.1\r\nHost: example.com\r\n\r\n')

response = client.recv(4096)
print(response.decode())

client.close()

UDP Socket

Python — UDP (connectionless)
# Servidor UDP
udp_server = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
udp_server.bind(('0.0.0.0', 5005))
data, addr = udp_server.recvfrom(1024)
udp_server.sendto(b'OK', addr)

# Cliente UDP
udp_client = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
udp_client.sendto(b'Ping', ('127.0.0.1', 5005))
resp, _ = udp_client.recvfrom(1024)

IPv6 Socket

Python — IPv6
# Usar AF_INET6 para IPv6
server6 = socket.socket(socket.AF_INET6, socket.SOCK_STREAM)
server6.bind(('::', 8080, 0, 0))  # (host, port, flowinfo, scopeid)
server6.listen(5)

Diagnóstico: netstat e ss

Comandos úteis
# Listar todos os sockets TCP abertos (Linux)
ss -tnp
# -t = TCP, -n = números, -p = processo

# Ver LISTEN e ESTABLISHED
ss -tnp state established
ss -tnp state listening

# Windows
netstat -ano | findstr :443
# -a = todos, -n = números, -o = PID

# Ver TIME_WAIT (após encerramento TCP)
ss -tan | grep TIME-WAIT | wc -l
