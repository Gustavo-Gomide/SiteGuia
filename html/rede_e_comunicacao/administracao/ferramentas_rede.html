Administração de Redes

Ferramentas de Rede

ping, traceroute, nmap, tcpdump, Wireshark, ss, dig, curl, iperf3 e mtr

O kit de ferramentas de um administrador de rede abrange desde a simples verificação de conectividade com ping até a captura e análise de pacotes em tempo real com tcpdump e Wireshark.

ping — Conectividade e Latência

# Básico
ping google.com
ping -c 4 8.8.8.8        # 4 pacotes
ping -i 0.2 -c 100 host  # 100 pings a 0.2s intervalo
ping -s 1400 host        # pacote grande (MTU test)
ping -M do -s 1472 host  # Don't Fragment (MTU path discovery)

# IPv6
ping6 ::1
ping -6 2001:4860:4860::8888

traceroute / mtr

traceroute

traceroute google.com          # UDP (padrão Linux)
traceroute -I google.com       # ICMP echo (precisa root)
traceroute -T -p 80 google.com # TCP SYN na porta 80
traceroute -n google.com       # sem resolver nomes
tracert google.com             # Windows (ICMP)

mtr (My TraceRoute)

mtr google.com            # modo interativo
mtr --report google.com   # relatório e sai
mtr -n google.com         # sem DNS
mtr --tcp -P 443 host     # TCP na porta 443
mtr --report-cycles 50 host

mtr combina ping e traceroute, mostrando perda de pacotes por hop em tempo real.

nmap — Port Scanner

# Scan básico
nmap 192.168.1.1
nmap 192.168.1.0/24         # scan de sub-rede
nmap -p 80,443,8080 host    # portas específicas
nmap -p- host               # todas as 65535 portas
nmap -p 1-1024 host         # range

# Detecção de serviços e SO
nmap -sV host               # versão dos serviços
nmap -O host                # detecção de SO
nmap -A host                # agressivo: SO + versao + scripts + traceroute

# Tipos de scan
nmap -sS host               # SYN stealth (padrão root)
nmap -sT host               # TCP connect (sem root)
nmap -sU -p 53,67,161 host  # UDP
nmap -sn 192.168.1.0/24     # host discovery (sem port scan)

# Scripts NSE
nmap --script=http-headers host
nmap --script=vuln host     # verifica vulnerabilidades conhecidas

# Output
nmap -oN resultado.txt host  # texto
nmap -oX resultado.xml host  # XML

Legal: Scan em redes e hosts sem permissão do proprietário é ilegal. Use apenas em redes que você administra ou tem autorização explícita.

tcpdump — Captura de Pacotes

# Básico
tcpdump -i eth0               # capturar na interface eth0
tcpdump -i any                # todas as interfaces
tcpdump -n -i eth0            # sem resolver nomes
tcpdump -v -i eth0            # verbose

# Filtros (Berkeley Packet Filter — BPF)
tcpdump -i eth0 port 80
tcpdump -i eth0 host 192.168.1.100
tcpdump -i eth0 src host 10.0.0.1
tcpdump -i eth0 dst port 443
tcpdump -i eth0 'tcp port 80 or tcp port 443'
tcpdump -i eth0 'not arp and not icmp'

# Salvar para Wireshark
tcpdump -i eth0 -w captura.pcap
tcpdump -r captura.pcap       # ler arquivo

# Mostrar conteúdo ASCII do payload
tcpdump -A -i eth0 port 80
tcpdump -X -i eth0 port 80    # ASCII + hex

ss / netstat — Sockets

# ss (moderno, mais rápido que netstat)
ss -tulpn           # TCP+UDP listen, processo, números
ss -t state ESTABLISHED  # conexões TCP estabelecidas
ss -s               # sumário
ss -o state time-wait  # conexões em TIME_WAIT

# netstat (legado, ainda presente)
netstat -tulpn      # mesmo que ss -tulpn
netstat -an         # todas, sem resolver
netstat -rn         # tabela de rotas
netstat -i          # estatísticas de interface

# Windows
netstat -ano        # lista PID junto
netstat -b          # mostra o executável (requer admin)

dig — Consultas DNS

# Básico
dig google.com             # registro A
dig google.com AAAA        # IPv6
dig google.com MX          # mail servers
dig google.com NS          # name servers
dig google.com TXT         # registros TXT (SPF, DKIM...)
dig google.com ANY         # todos os registros

# Consultar servidor específico
dig @8.8.8.8 google.com
dig @1.1.1.1 google.com MX

# Formato limpo
dig +short google.com
dig +noall +answer google.com

# Reverse DNS (PTR)
dig -x 8.8.8.8

# DNSSEC
dig +dnssec google.com

# Trace resolution path
dig +trace google.com

curl — HTTP na Linha de Comando

# Básico
curl https://example.com
curl -I https://example.com      # só headers
curl -i https://example.com      # headers + body
curl -v https://example.com      # verbose (debug TLS)

# Métodos HTTP
curl -X POST https://api/endpoint \
  -H "Content-Type: application/json" \
  -d '{"key": "value"}'
curl -X DELETE https://api/item/1
curl -X PUT https://api/item/1 -d '{"name":"novo"}'

# Autenticação
curl -u user:pass https://api
curl -H "Authorization: Bearer TOKEN" https://api

# Upload
curl -F "file=@foto.jpg" https://upload
curl -T arquivo.txt ftp://servidor/

# Tempo e redirect
curl -L https://encurtador.com   # seguir redirects
curl --max-time 10 https://api  # timeout
curl -o saida.html https://page  # salvar para arquivo
curl -w "%{http_code} %{time_total}\n" -o /dev/null -s https://api/health

iperf3 — Largura de Banda

# Servidor (modo passivo, aguardando cliente)
iperf3 -s
iperf3 -s -p 5202  # porta alternativa

# Cliente — TCP
iperf3 -c SERVIDOR_IP           # teste padrão 10s TCP
iperf3 -c SERVIDOR_IP -t 30     # 30 segundos
iperf3 -c SERVIDOR_IP -P 4      # 4 streams paralelos
iperf3 -c SERVIDOR_IP -R        # sentido reverso (download)

# UDP
iperf3 -c SERVIDOR_IP -u -b 100M  # UDP 100 Mbps
# Mostra jitter e packet loss (essencial para VoIP/vídeo)

# Exemplo de saída
# [ ID] Interval           Transfer     Bandwidth
# [  4]  0.00-10.00 sec  1.10 GBytes   944 Mbits/sec   sender
