Segurança de Redes

Firewalls, iptables, nftables, ACLs, IDS/IPS, segmentação, VPN e Zero Trust

A segurança de redes é uma disciplina multicamada: controle de acesso no perímetro, segmentação interna, detecção de intrusões e proteção de comunicações. Não existe uma única solução — a defesa em profundidade combina múiltiplas camadas.

iptables — Configuração

Regras iptables (Linux)
# Ver regras ativas
iptables -L -v -n
iptables -t nat -L -v -n

# Política padrão restritiva (whitelist)
iptables -P INPUT   DROP
iptables -P FORWARD DROP
iptables -P OUTPUT  ACCEPT

# Permitir loopback
iptables -A INPUT -i lo -j ACCEPT

# Permitir conexões estabelecidas/relacionadas (stateful)
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# Permitir SSH (com rate limiting contra brute force)
iptables -A INPUT -p tcp --dport 22 \
  -m conntrack --ctstate NEW \
  -m recent --set --name SSH
iptables -A INPUT -p tcp --dport 22 \
  -m recent --update --seconds 60 --hitcount 4 --name SSH \
  -j DROP
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Permitir HTTP e HTTPS
iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

# ICMP (ping)
iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT

# Salvar regras (Debian)
iptables-save > /etc/iptables/rules.v4

# Restaurar
iptables-restore < /etc/iptables/rules.v4

iptables está sendo substituído pelo nftables nas distribuições modernas. nftables tem melhor performance, sintaxe unificada (sem tabelas separadas para IPv4/IPv6/ARP) e é o padrão no Debian 10+, RHEL 8+.

nftables — Substituto Moderno

/etc/nftables.conf
#!/usr/sbin/nft -f

flush ruleset

table inet filter {
  chain input {
    type filter hook input priority 0; policy drop;

    iif lo accept
    ct state established,related accept

    # ICMP
    ip protocol icmp accept
    ip6 nexthdr icmpv6 accept

    # SSH com rate limit
    tcp dport 22 ct state new \
      limit rate 4/minute accept

    # HTTP + HTTPS
    tcp dport { 80, 443 } accept
  }

  chain forward {
    type filter hook forward priority 0; policy drop;
  }

  chain output {
    type filter hook output priority 0; policy accept;
  }
}

# Aplicar
# nft -f /etc/nftables.conf
# nft list ruleset

Segmentação de Rede e DMZ

DMZ (Demilitarized Zone)

Sub-rede entre a internet e a rede interna que hospeda serviços públicos (web servers, servidores de email). Uma breach na DMZ não dá acesso direto à rede interna.

Internet → [Firewall ext] → DMZ
           Web / Mail / DNS
           [Firewall int] → Rede Interna
           App Servers / DB / AD

VLANs

Segmentação lógica na camada 2 (switch). Cada VLAN é um domínio de broadcast separado. O tráfego entre VLANs passa por roteamento (inter-VLAN routing) que pode ser inspecionado por firewall.

  • VLAN 10 — Servidores
  • VLAN 20 — Usuários
  • VLAN 30 — Wi-Fi
  • VLAN 40 — IoT
  • VLAN 99 — Gerenciamento

Micro-segmentação

Controle de tráfego leste-oeste (entre VMs/containers na mesma rede). Implementado por firewalls de host, SDN, ou Network Policies no Kubernetes. Limita movimento lateral de atacantes.

WireGuard — configuração mínima (servidor)
# Gerar chaves
wg genkey | tee privkey | wg pubkey > pubkey

# /etc/wireguard/wg0.conf
[Interface]
PrivateKey = <SERVER_PRIVATE_KEY>
Address    = 10.0.0.1/24
ListenPort = 51820

# Habilitar NAT para internet access
PostUp   = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]  # cliente
PublicKey  = <CLIENT_PUBLIC_KEY>
AllowedIPs = 10.0.0.2/32

# Iniciar
wg-quick up wg0
systemctl enable wg-quick@wg0

Zero Trust Network Access (ZTNA)

Zero Trust abandona o modelo “confiar em tudo dentro do perímetro”. Em vez disso:

  • Never trust, always verify: toda requisição deve ser autenticada, autorizada e criptografada — independente da origem
  • Least privilege: acesso mínimo necessário, revisado continuamente
  • Micro-segmentação: isolar workloads para limitar movimento lateral
  • Device trust: verificar postura do dispositivo (OS atualizado, antivirus, compliance)
  • Continuous monitoring: logar e analisar todo o tráfego

Exemplos: Cloudflare Access, Google BeyondCorp, Zscaler ZPA, HashiCorp Boundary
