Monitoramento de Redes

SNMP, Prometheus, Grafana, Zabbix, alertas e métricas de disponibilidade

O monitoramento de redes fornece visibilidade sobre disponibilidade, performance e problemas em tempo real. Combinar coleta de métricas, visualização e alertas é essencial para manter SLAs e diagnosticar incidentes rapidamente.

SNMP — Simple Network Management Protocol

SNMP é o protocolo padrão para monitorar e gerenciar dispositivos de rede (switches, roteadores, firewalls, servidores). Usa uma base de dados hierárquica chamada MIB (Management Information Base).

VersãoSegurançaAutenticidadeCriptografiaUso Atual
SNMPv1Community string (texto claro)NãoNãoLegado
SNMPv2cCommunity string (texto claro)NãoNãoAinda comum
SNMPv3USM (User-based Security)MD5/SHADES/AESRecomendado

snmpwalk / snmpget

# v2c
snmpget -v2c -c public 192.168.1.1 \
  sysDescr.0

snmpwalk -v2c -c public 192.168.1.1 \
  interfaces

# v3
snmpget -v3 -u admin \
  -l authPriv \
  -a SHA -A senhaSHA \
  -x AES -X senhaAES \
  192.168.1.1 sysUpTimeInstance

Prometheus + Grafana

Prometheus coleta métricas via HTTP de endpoints (exporters) em modelo pull. Grafana consome o Prometheus como datasource e cria dashboards visualizáveis.

Exporters comuns

node_exporterCPU, mem, disco, interfaces
blackbox_exporterICMP ping, HTTP check, TCP
snmp_exporterDispositivos SNMP
nginx-prometheus-exporterMétricas Nginx

Prometheus — prometheus.yml

global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'nodes'
    static_configs:
      - targets:
          - node1:9100
          - node2:9100

  - job_name: 'blackbox_http'
    metrics_path: /probe
    params:
      module: [http_2xx]
    static_configs:
      - targets:
          - https://example.com
          - https://api.example.com/health
    relabel_configs:
      - source_labels: [__address__]
        target_label: __param_target
      - target_label: __address__
        replacement: blackbox:9115

PromQL — Queries úteis

# Up/Down de instâncias
up
up{job="nodes"}

# Bandwidth (bytes/s)
rate(node_network_receive_bytes_total[5m])

# Packet loss ICMP
1 - avg_over_time(probe_success[5m])

# Latência HTTP
probe_duration_seconds{job="blackbox_http"}

# CPU usage
100 - (avg by(instance)(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

Zabbix — Monitoramento Enterprise

Arquitetura

Zabbix Server coleta dados via Zabbix Agent (instalado nos hosts), SNMP, ICMP polling, ou Zabbix Proxy (para redes remotas). Armazena histórico em banco de dados (PostgreSQL/MySQL).

Fontes de dados

  • Zabbix Agent: CPU, mem, disco, processos, logs
  • SNMP: Switches, roteadores, UPS
  • ICMP: Polls de disponibilidade
  • HTTP agent: Endpoints web/API
  • JMX: Aplicações Java

Alertas

Triggers baseados em expressões: {host:key.last()} < 10.
Ações automáticas: email, Slack, PagerDuty, script customizado, auto-remediation.
