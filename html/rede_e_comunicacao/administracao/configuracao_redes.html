Configuração de Redes

ip, ifconfig, netplan, NM, rotas estáticas e resolução DNS no Linux e Windows

A configuração correta de interfaces de rede, endereços IP, rotas e DNS é a base para qualquer comunicação em rede. Este guia cobre as ferramentas modernas (iproute2, netplan) e clássicas (ifconfig, route) no Linux, além do netsh/PowerShell no Windows.

Comandos ip — iproute2 (Linux moderno)

AçãoComando ip (moderno)Equivalente ifconfig/route (legado)
Listar interfacesip link showifconfig -a
Listar endereços IPip addr showifconfig
Adicionar IPip addr add 192.168.1.10/24 dev eth0ifconfig eth0 192.168.1.10 netmask 255.255.255.0
Remover IPip addr del 192.168.1.10/24 dev eth0ifconfig eth0 0.0.0.0
Ativar interfaceip link set eth0 upifconfig eth0 up
Desativar interfaceip link set eth0 downifconfig eth0 down
Ver tabela de rotasip route showroute -n
Gateway padrãoip route add default via 192.168.1.1route add default gw 192.168.1.1
Rota estáticaip route add 10.0.0.0/8 via 192.168.1.1 dev eth0route add -net 10.0.0.0/8 gw 192.168.1.1
Remover rotaip route del 10.0.0.0/8route del -net 10.0.0.0/8
Ver vizinhos ARPip neigh showarp -n
Ver estatísticasip -s link show eth0ifconfig eth0

As alterações com ip são voláteis (perdem ao reiniciar). Para persistência, use netplan ou NetworkManager.

Netplan (Ubuntu 18.04+)

Netplan é o sistema de configuração de rede padrão no Ubuntu. Gera configurações para NetworkManager (desktop) ou systemd-networkd (servidor) a partir de arquivos YAML.

IP Estático

/etc/netplan/01-netcfg.yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: no
      addresses:
        - 192.168.1.100/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        search: [example.com]
        addresses: [8.8.8.8, 1.1.1.1]

DHCP + DNS personalizado

/etc/netplan/01-netcfg.yaml
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: yes
      dhcp4-overrides:
        use-dns: false
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
    eth1:
      dhcp4: yes

Comandos netplan

# Testar config (gera arquivos temporários)
netplan try
# Aplica por 120s; reverter com Enter se algo falhar

# Aplicar permanentemente
netplan apply

# Gerar config (sem aplicar)
netplan generate

# Checar sintaxe
netplan info

NetworkManager (nmcli / nmtui)

nmcli — linha de comando

# Listar conexões
nmcli connection show
nmcli device status

# Criar conexão estática
nmcli con add type ethernet \
  con-name "eth0-static" \
  ifname eth0 \
  ipv4.addresses 192.168.1.50/24 \
  ipv4.gateway 192.168.1.1 \
  ipv4.dns "8.8.8.8 1.1.1.1" \
  ipv4.method manual

# Ativar / desativar
nmcli con up   "eth0-static"
nmcli con down "eth0-static"

nmtui — interface TUI

nmtui oferece uma interface de texto interativa — útil quando não há GUI disponível.

# Iniciar interface TUI
nmtui
# Opções: Edit connection / Activate / Set hostname

Configuração DNS

/etc/resolv.conf (legado)

# Atenção: pode ser sobrescrito pelo NetworkManager!
# Em Ubuntu moderno, é um symlink para systemd-resolved
nameserver 8.8.8.8
nameserver 1.1.1.1
search example.com interno.example.com
options ndots:5 timeout:2 attempts:3

systemd-resolved

# Ver DNS atual
resolvectl status
resolvectl dns

# Configurar DNS por link
resolvectl dns eth0 8.8.8.8 1.1.1.1
resolvectl domain eth0 example.com

# Consulta via systemd-resolved
resolvectl query google.com

Windows — netsh / PowerShell

# Ver interfaces
Get-NetAdapter
netsh interface show interface

# IP estático
netsh interface ip set address `
  name="Ethernet" static 192.168.1.50 255.255.255.0 192.168.1.1

# DNS
netsh interface ip set dns `
  name="Ethernet" static 8.8.8.8

# PowerShell moderno
New-NetIPAddress -InterfaceAlias "Ethernet" `
  -IPAddress 192.168.1.50 -PrefixLength 24 `
  -DefaultGateway 192.168.1.1

Rotas Estáticas Persistentes

Debian/Ubuntu — /etc/network/interfaces (legado)

# /etc/network/interfaces
iface eth0 inet static
  address 192.168.1.100
  netmask 255.255.255.0
  gateway 192.168.1.1
  post-up   ip route add 10.0.0.0/8 via 192.168.1.250
  pre-down  ip route del 10.0.0.0/8 via 192.168.1.250

RHEL/Rocky — route-eth0

/etc/sysconfig/network-scripts/route-eth0
10.0.0.0/8 via 192.168.1.250
172.16.0.0/12 via 192.168.1.250

Windows — rota persistente

# -p = persistente (sobrevive ao reboot)
route -p ADD 10.0.0.0 MASK 255.0.0.0 192.168.1.250
route PRINT
route DELETE 10.0.0.0
