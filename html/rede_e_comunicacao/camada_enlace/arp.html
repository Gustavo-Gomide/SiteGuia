Camada de Enlace

ARP — Address Resolution Protocol

A ponte entre IP (Camada 3) e MAC (Camada 2): "Quem tem esse IP? Me dê seu MAC."

Para enviar um frame Ethernet a um dispositivo na mesma rede, o remetente precisa do endereço MAC do destino — mas só conhece o IP. O ARP (RFC 826) resolve esse mapeamento: envia um broadcast perguntando "quem tem o IP X?" e o dono responde com seu MAC.

Processo ARP Request / Reply

Cache ARP

Após receber o reply, o solicitante armazena o mapeamento em seu cache ARP por um período configurável:

  • Linux: invalido após ~60s sem uso, removido em <60s adicionais; ip neigh show
  • Windows: 15s–45s dinâmico; arp -a
  • Cisco: 4 horas padrão; show arp, clear arp-cache
  • Entradas estáticas não expiram; usadas para servidores críticos

Gratuitous ARP (GARP)

Um host envia um ARP Request com seu próprio IP no campo de destino — sem esperar resposta. Usos:

  • Detecção de IP duplicado: se alguém responder, há conflito
  • Atualização de cache: após mudarna de interface (failover, HSRP/VRRP), o novo gateway anuncia seu MAC para atualizar todos os caches da rede imediatamente
  • Live migration (VM): VM migrada envia GARP para anunciar novo MAC ou mesmo MAC em porta diferente do switch

Proxy ARP e ARP Spoofing

  • Proxy ARP: roteador responde a ARPs em nome de hosts em outras redes; permite que hosts sem rota padrão alcancem endereços remotos através do roteador (perigoso: pode esconder erros de configuração; desabilitar com no ip proxy-arp)
  • ARP Spoofing / Poisoning: atacante envia GARP falso associando IP da vítima (ou gateway) ao seu próprio MAC → MITM (Man-in-the-Middle); todo o tráfego passa pelo atacante antes de chegar ao destino
  • Mitigação — DAI: Dynamic ARP Inspection no switch valida ARPs contra a tabela DHCP Snooping; descarta ARPs inválidos em portas untrusted

ARP no IPv6: NDP

No IPv6, o ARP foi substituído pelo NDP (Neighbor Discovery Protocol), parte do ICMPv6. Em vez de broadcasts, usa multicast de solicitação (Solicited-Node Multicast Address: ff02::1:ffxx:xxxx):

Mensagens NDP

  • NS (Neighbor Solicitation, ICMPv6 tipo 135): equivalente ao ARP Request; enviado para o multicast de solicitação do alvo
  • NA (Neighbor Advertisement, tipo 136): equivalente ao ARP Reply; responde com o endereço de camada 2
  • RS (Router Solicitation, tipo 133): host pergunta quais roteadores existem na rede
  • RA (Router Advertisement, tipo 134): roteador anuncia prefixo de rede, gateway, flags SLAAC/DHCPv6, MTU
  • DAD (Duplicate Address Detection): NS enviado para próprio endereço antes de usá-lo; substitui o GARP
