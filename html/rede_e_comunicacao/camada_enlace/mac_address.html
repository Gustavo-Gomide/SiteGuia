Camada de Enlace

Endereço MAC

Identidade única de cada interface de rede na Camada 2 — 48 bits gravados na placa, visíveis apenas na LAN local

O endereço MAC (Media Access Control) é um identificador de 48 bits atribuído a cada interface de rede pelo fabricante. Enquanto o IP é um endereço lógico configuravel, o MAC é um endereço físico gravado em hardware (embora possa ser sobrescrito por software). Switches usam os MACs de origem/destino para encaminhar frames dentro da mesma rede local (L2).

Estrutura do Endereço MAC

Notações: aa:bb:cc:dd:ee:ff (Linux), AA-BB-CC-DD-EE-FF (Windows), aabb.ccdd.eeff (Cisco). São a mesma coisa.

Tipos de Endereço MAC

  • Unicast: bit I/G = 0; destinado a uma única interface; aa:bb:cc:dd:ee:ff onde aa é par
  • Broadcast: ff:ff:ff:ff:ff:ff; entregue a todos os dispositivos no mesmo domínio de broadcast (VLAN)
  • Multicast IPv4: 01:00:5e:xx:xx:xx; mapeamento dos 23 bits inferiores do grupo multicast IP
  • Multicast IPv6: 33:33:xx:xx:xx:xx; 4 bytes inferiores do IPv6 multicast
  • Locally Administered (U/L=1): virtuais, VMware, MAC spoofing; bit 1 do primeiro byte = 1

Tabela CAM — Como o Switch Aprende

Processo de Aprendizado

Um switch L2 mantém uma tabela CAM (Content Addressable Memory) que mapeia MAC de origem → porta de entrada. Cada vez que um frame chega, o switch registra o MAC de origem + porta.

  1. Learning: frame entra pela porta X com MAC origem A → switch anota A = porta X
  2. Flooding: se MAC destino não está na tabela → envia para todas as outras portas (exceto a de entrada)
  3. Forwarding: MAC destino já na tabela → encaminha apenas para a porta correta
  4. Filtering: MAC destino na mesma porta de origem → descarta (já está no segmento certo)
  5. Aging: entradas que não foram usadas por X segundos são removidas (padrão Cisco: 300s)

Segurança e Ataques

  • MAC Flooding: atacante gera frames com MACs de origem aleatórios, enchendo a tabela CAM (limitada) → switch passa a inundar todos os frames (como um hub) → sniffing de tráfego. Mitigação: Port Security (limitar nº de MACs por porta)
  • MAC Spoofing: atacante altera seu MAC para se passar por outro dispositivo (válido para bypass de autenticação por MAC). Mitigação: 802.1X (autenticação real de usuário/dispositivo)
  • Port Security Cisco: 
    switchport port-security maximum 2
switchport port-security mac-address sticky
switchport port-security violation restrict
