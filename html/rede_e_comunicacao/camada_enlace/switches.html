Camada de Enlace

Switches L2

O coracão da LAN moderna: encaminham frames, evitam loops, distribuím energia e agregam links

Um switch de Camada 2 encaminha frames Ethernet com base no endereço MAC destino, criando domínios de colisão individuais por porta. Diferentemente do hub (que repete para todas as portas), o switch aprende os MACs e encaminha apenas para a porta correta, isolando o tráfego e permitindo comunicação full-duplex simultânea.

STP — Spanning Tree Protocol (IEEE 802.1D)

Redes redundantes criam loops que geram tempestades de broadcast e instabilidade da tabela CAM. O STP bloqueia logicamente links redundantes, mantendo uma topologia sem loops.

Eleição do Root Bridge

  1. Cada switch anuncia seu Bridge ID (BID) = Priority (padrão 32768) + MAC address
  2. O switch com o menor BID vira Root Bridge; desempate: menor MAC
  3. Para mudar o root: spanning-tree vlan 1 priority 4096 (números múltiplos de 4096)

Funções de Porta

  • Root Port (RP): porta com melhor caminho para o root bridge; uma por switch (exceto root)
  • Designated Port (DP): melhor porta em cada segmento para alcançar o root; fica em forwarding
  • Blocked Port: porta redundante colocada em blocking para evitar loop

Estados de Porta (802.1D)

  • Blocking: não encaminha frames; escuta BPDUs (20s)
  • Listening: participa da eleição STP (15s)
  • Learning: aprende MACs, não encaminha ainda (15s)
  • Forwarding: encaminha frames normalmente
  • Disabled: porta administrativamente desativada

Conversão total: até 50s em 802.1D. RSTP (802.1w) reduz para < 1s com propostu/agreement.

RSTP e Melhorias

  • PortFast: porta de acesso vai diretamente para forwarding (sem estados intermediários); spanning-tree portfast
  • BPDU Guard: se PortFast receber BPDU → porta em err-disabled; protege contra APs/switches não autorizados
  • Root Guard: impede que porta específica se torne root port
  • Loop Guard: previne loops quando BPDUs param de chegar (uni-directional link failure)

Recursos Avançados

EtherChannel / LACP (802.3ad)

Agrega múltiplos links físicos em um único link lógico, aumentando largura de banda e resiliência. 2 a 8 portas por bundle.

Cisco — LACP EtherChannel
interface range GigabitEthernet0/1-2
  channel-group 1 mode active   ! LACP ativo
!
interface Port-channel1
  switchport mode trunk

SPAN (Port Mirroring)

Copia tráfego de portas de origem para uma porta de destino onde fica conectado um analisador (Wireshark, IDS/IPS).

Cisco — SPAN local
monitor session 1 source interface Gi0/1 both
monitor session 1 destination interface Gi0/24
! RSPAN: entre switches (VLAN espelho)
! ERSPAN: encapsula em GRE para destino remoto

Comandos de Diagnóstico

Cisco IOS — show commands
show mac address-table           ! tabela CAM completa
show mac address-table dynamic   ! só entradas dinâmicas
show spanning-tree               ! estado STP por VLAN
show spanning-tree vlan 10 detail
show interfaces GigabitEthernet0/1 status
show etherchannel summary        ! EtherChannels
show port-security               ! Port Security
