Fundamentos de Dart para Flutter

Tipagem, null safety e assicronismo como base de apps confiaveis.

No Flutter, bugs de estado e concorrencia normalmente nascem de base fraca em Dart. Dominar os fundamentos reduz retrabalho e melhora previsibilidade do codigo.

Conceitos essenciais

  • Null safety: evita classes inteiras de erro de runtime.
  • Tipagem forte: aumenta refactoring seguro e qualidade de API interna.
  • Classes e mixins: composicao clara de comportamento.
  • Future/Stream: modelo assicrono principal para I/O e eventos.

Padrao de codigo no app

boas_praticas_dart.txt
1. Preferir final/const por padrao
2. Evitar dynamic sem necessidade
3. Modelar erro com tipos explicitos
4. Isolar acesso remoto em repository
5. Escrever testes para regras puras
