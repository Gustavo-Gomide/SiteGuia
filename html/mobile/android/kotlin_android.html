Kotlin no Android

Kotlin como Base da Engenharia Android Moderna

Null safety, coroutines e Flow para codigo mais seguro e previsivel em apps mobile.

Kotlin virou padrao Android porque reduz classes inteiras de erro (null pointer, boilerplate de callback) e melhora legibilidade sem sacrificar performance.

Pilares da linguagem para Android

  • Null safety: tipo nullable explicito reduz crash de null.
  • Data classes: modelos concisos para estado de UI e dominio.
  • Extension functions: utilitarios sem classes helper extensas.
  • Sealed classes: modelagem segura de estados de tela.

Coroutines e Flow

Coroutines substituem callbacks aninhados por codigo assicrono legivel. Flow permite stream reativo de dados locais e remotos, essencial para UI orientada a estado.

state_flow_basico.kt
val state: StateFlow<UiState> = repository
    .streamData()
    .map { UiState.Data(it) as UiState }
    .catch { emit(UiState.Error) }
    .stateIn(viewModelScope, SharingStarted.WhileSubscribed(), UiState.Loading)

Boas praticas de manutencao

Evite escopo global de coroutine para tarefas de tela. Prefira viewModelScope e cancelamento automatico com ciclo de vida. Centralize mapeamento de erro de rede para nao duplicar mensagens e estados inconsistentes.
