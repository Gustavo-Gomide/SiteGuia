O que é Wireshark?

Wireshark

Captura e disseção de pacotes em tempo real para análise de protocolos.

Wireshark é o analisador de protocolos de rede open source mais usado do mundo. Captura e disseca pacotes em tempo real, permitindo ver exatamente o que trafega pela rede — desde headers TCP até payloads de aplicação. Essencial para forense de rede, debugging e pentest.

Filtros Essenciais

Wireshark usa dois tipos de filtro: Capture filters (Berkeley Packet Filter — BPF, durante captura) e Display filters (após captura, para visualização).

# ----- Display Filters -----

# Por protocolo:
http
dns
ftp
smb
icmp
ssl or tls

# Por IP:
ip.addr == 192.168.1.10
ip.src == 10.0.0.1
ip.dst == 10.0.0.2

# Por porta:
tcp.port == 80
tcp.port == 443
udp.port == 53

# Por conteúdo:
http.request.method == "POST"
http contains "password"
dns.qry.name contains "evil"

# ----- Capture Filters (BPF) -----
# Capturar apenas HTTP e DNS da rede:
port 80 or port 53

# Excluir broadcast:
not broadcast and not multicast

# Apenas tráfego de/para IP:
host 192.168.1.10

Análises Comuns

Credenciais em texto claro

Protocolos como FTP, Telnet, HTTP Basic Auth, SMTP, POP3 transmitem credenciais sem criptografia.

# Filtrar POST requests:
http.request.method == "POST"

# Seguir TCP Stream:
# Click direito → Follow → TCP Stream
# Procurar campos: username, password, login

Análise DNS

Detectar DNS tunneling (payloads grandes em queries TXT), DGA (domínios aleatórios), respostas apontando para IPs suspeitos, C2 via DNS.

dns
dns.qry.name contains "dga"
# Statistics → DNS → por tipo de consulta

Análise de scans

Nmap scan gera padrão de SYN sem completar handshake. Detectável: muitas conexões TCP SYN de 1 IP para múltiplas portas em curto período.

# Ver todos os IPs que enviaram SYN:
tcp.flags.syn == 1 and tcp.flags.ack == 0

Exportar objetos HTTP

File → Export Objects → HTTP — extrai todos os arquivos transferidos via HTTP: executáveis, documentos, imagens. Útil em forense para recuperar arquivos baixados por malware.

tshark — Wireshark em linha de comando

# Capturar 100 pacotes na interface eth0:
tshark -i eth0 -c 100 -w captura.pcap

# Ler .pcap e filtrar:
tshark -r captura.pcap -Y "http" -T fields \
    -e ip.src -e ip.dst -e http.request.uri

# Extrair todas as queries DNS:
tshark -r captura.pcap -Y "dns.qry.name" -T fields \
    -e dns.qry.name | sort | uniq -c | sort -rn

# Credenciais FTP:
tshark -r captura.pcap -Y "ftp" -T fields \
    -e ftp.request.command -e ftp.request.arg

Análise de .pcap é categoria frequente em CTFs de forense. Pratique com capturas do Wireshark Sample Captures wiki. Para captura de rede em pentest, combine Wireshark com tcpdump para captura remota em servidores sem interface gráfica.

"Wireshark não mente. O que trafega pela rede está ali — basta saber filtrar para encontrar."
