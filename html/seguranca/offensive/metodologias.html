Metodologias de Pentest

PTES, OWASP, OSSTMM — frameworks que garantem cobertura sistemática.

Metodologia transforma pentest de "hackeamento aleatório" em processo profissional. Garante cobertura sistemática — você não esquece de testar uma área inteira. Facilita relatórios consistentes. E no mundo corporativo, clientes exigem que pentests sigam padrões reconhecidos.

PTES — Penetration Testing Execution Standard

O PTES é a metodologia mais usada em pentests de rede/sistema. Define 7 fases:

1. Pre-engagement

Definir escopo, cronograma, objetivos e obter autorização formal. Reunião com o cliente para alinhar expectativas e entregáveis. Assinatura de NDA e Rules of Engagement (RoE).

2. Intelligence Gathering

Coleta passiva e ativa de informações sobre o alvo — OSINT, DNS, WHOIS, shodan, Google Dorks, LinkedIn, Netcraft. Sem tocar nos sistemas ainda.

3. Threat Modeling

Com as informações coletadas, identificar os vetores de ataque mais prováveis e impactantes. Quais sistemas são mais críticos? Qual seria o objetivo de um atacante real?

4. Vulnerability Analysis

Enumerar serviços, versões e identificar vulnerabilidades — scanning com Nmap, Nessus, Nikto. Buscar CVEs aplicáveis. Identificar misconfigurations.

5. Exploitation

Tentar explorar as vulnerabilidades identificadas para ganhar acesso. Metasploit, exploits manuais, web exploits. Documentar cada passo com screenshots.

6. Post-exploitation

Com acesso obtido, aprofundar: escalação de privilégio, lateral movement, persistência, coleta de hashes, acesso a dados sensíveis. Demonstrar o impacto real.

7. Reporting

Relatório técnico detalhado + sumário executivo. Cada vulnerabilidade com: descrição, evidência (screenshot/PoC), risco (CVSS), recomendação de correção.

Estrutura do Relatório de Pentest

Um bom relatório é tão importante quanto o teste em si — é o produto que o cliente recebe e usa para tomar decisões. Deve ter duas partes distintas:

Sumário Executivo

  • Escrito para diretores e C-level — sem jargão técnico
  • Contexto do teste (o que foi testado, quando, como)
  • Descobertas em linguagem de negócio (riscos e impactos)
  • Rating geral de risco (ex: Alto, Médio, Baixo)
  • Top 3–5 recomendações prioritárias

Relatório Técnico

  • Metodologia utilizada e ferramentas
  • Cada vulnerabilidade encontrada:
  •   → Título, severidade (CVSS), descrição técnica
  •   → Passos para reproduzir (PoC)
  •   → Screenshots/evidências
  •   → Recomendação de remediação específica
  • Apêndice: output de ferramentas, configurações

Metodologia é o que separa o pentester profissional do script kiddie. Siga as fases, documente tudo, e lembre que o produto final é o relatório — não o acesso em si. Veja as fases em detalhe: Reconhecimento, Enumeração, Exploração.

"Um pentest sem metodologia é como uma cirurgia sem protocolo — você pode ter sorte, mas não é profissional."
