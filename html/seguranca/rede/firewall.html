O que é um Firewall?

Firewall

Controla o tráfego de rede permitindo ou bloqueando com base em regras.

Firewall é um sistema de controle de tráfego que permite ou bloqueia comunicações de rede com base em regras. É a fundação da segurança perimetral — mas sozinho não é suficiente. Existem vários tipos, cada um com capacidades e posicionamentos distintos.

Tipos de Firewall

TipoComo funcionaVantagensLimitações
Packet Filtering (Stateless)Avalia cada pacote isoladamente por 5-tupla (src/dst IP, src/dst port, protocolo)Rápido, baixo overheadNão rastreia sessões; não entende contexto
Stateful InspectionRastreia estado das conexões TCP (SYN/SYN-ACK/ACK); permite respostas legítimas automaticamenteContexto de sessão; mais inteligenteNão inspeciona payload de aplicação
Application Proxy / ALGAtua como intermediário na camada de aplicação (Layer 7); entende HTTP, FTP, DNSDeep inspection; pode filtrar conteúdoLatência maior; um proxy por protocolo
NGFW (Next-Gen Firewall)Stateful + IPS integrado + inspeção SSL/TLS + controle de apps + threat intelVisibilidade total; proteção multicamadaCusto; complexidade; CPU-intensivo
WAF (Web Application Firewall)Protege especificamente aplicações HTTP/S (XSS, SQLi, CSRF, bots)Específico para webApenas protocolo HTTP

iptables / nftables (Linux)

# iptables — estrutura: TABLES → CHAINS → RULES

# Ver regras atuais:
iptables -L -n -v --line-numbers

# Política padrão: DROP tudo, liberar explicitamente (whitelist):
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# Permitir loopback:
iptables -A INPUT -i lo -j ACCEPT

# Permitir conexões estabelecidas/relacionadas:
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# Liberar SSH apenas de IP específico:
iptables -A INPUT -p tcp --dport 22 -s 203.0.113.10 -j ACCEPT

# Liberar HTTP e HTTPS:
iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

# Bloquear IP específico:
iptables -A INPUT -s 1.2.3.4 -j DROP

# Salvar regras (Debian/Ubuntu):
iptables-save > /etc/iptables/rules.v4

# ----- nftables (substituto moderno do iptables) -----
nft list ruleset
nft add rule inet filter input tcp dport 22 accept

Boas Práticas

Default Deny

Comece bloqueando tudo e libere apenas o necessário. Política whitelist é mais segura que blacklist. Nunca use permit any any em produção.

Princípio do Menor Privilégio

Libere apenas as portas, protocolos e IPs estritamente necessários para cada serviço. Revise regras regularmente e remova as que não são mais usadas.

Logging

Registre ao menos as conexões bloqueadas (DROP + log). Logs de firewall são essenciais para detectar scanning, tentativas de exploração e movimentação lateral.

Inspeção SSL/TLS

NGFWs podem decriptar e inspecionar tráfego HTTPS para detectar malware e exfiltração. Requer certificado CA próprio distribuído nos endpoints. Atenção a questões de privacidade.

Firewalls são necessários mas não suficientes. Um atacante com credenciais válidas passa pelo firewall legitimamente. Combine firewall com segmentação de rede, monitoramento de comportamento (SIEM/NDR) e autenticação forte para criar defesa real em profundidade.

"Um firewall só é tão bom quanto as regras que o administrador configura — e a disciplina para mantê-las."
