IDS vs IPS

IDS e IPS

IDS alerta sobre intrusões; IPS bloqueia em tempo real — a diferença é a posição no fluxo.

IDS (Intrusion Detection System) monitora o tráfego e alerta sobre atividades suspeitas. IPS (Intrusion Prevention System) faz o mesmo mas também bloqueia em tempo real. A diferença é a posição: IDS fora do fluxo de tráfego (passivo/tap), IPS inline (ativo).

IDS — Detecção

  • Monitora e gera alertas
  • Modo passivo (cópia do tráfego via TAP/SPAN)
  • Sem impacto no tráfego em caso de falha
  • Maior taxa de falsos positivos aceitável
  • Ideal para visibilidade e análise forense

IPS — Prevenção

  • Bloqueia tráfego malicioso em tempo real
  • Modo inline — está no caminho do tráfego
  • Falso positivo bloqueia tráfego legítimo
  • Requer alta disponibilidade (fail-open/fail-closed)
  • Ideal para proteção automática ativa

Métodos de Detecção

Baseado em Assinaturas

Compara tráfego com um banco de padrões conhecidos (signatures). Muito eficaz para ameaças conhecidas, zero falsos positivos para assinaturas bem escritas. Não detecta ameaças novas (zero-days).

Baseado em Anomalia

Cria um baseline do comportamento normal e alerta desvios (ex: pico de tráfego DNS, novo protocolo). Detecta zero-days mas tem alta taxa de falsos positivos. Requer período de aprendizado.

Baseado em Política (Policy-based)

Define o que é permitido (whitelist de comportamentos). Qualquer coisa fora da política é alerta. Muito preciso, mas difícil de manter atualizado em ambientes dinâmicos.

HIDS vs NIDS

HIDS (Host-based) monitora dentro do sistema operacional — arquivos, processos, logs, chamadas de sistema. NIDS (Network-based) monitora o tráfego de rede. São complementares.

Snort e Suricata

# ----- Suricata (NIDS/NIPS open source, multi-thread) -----

# Rodar em modo IDS na interface eth0:
suricata -c /etc/suricata/suricata.yaml -i eth0

# Rodar contra um arquivo .pcap:
suricata -c /etc/suricata/suricata.yaml -r captura.pcap

# Ver alertas em tempo real:
tail -f /var/log/suricata/fast.log

# ----- Estrutura de uma regra Suricata/Snort -----
# action proto src_ip src_port direction dst_ip dst_port (options)

alert tcp any any -> any 22 (
    msg:"SSH Brute Force Attempt";
    threshold: type threshold, track by_src, count 5, seconds 60;
    sid:1000001;
    rev:1;
)

alert http any any -> any any (
    msg:"SQL Injection Attempt";
    content:"union select";
    nocase;
    http.uri;
    sid:1000002;
    rev:1;
)

# Drop (modo IPS inline):
drop tcp any any -> $HOME_NET 23 (
    msg:"Telnet Connection Blocked";
    sid:1000003;
    rev:1;
)

IDS/IPS são complementares a firewalls: enquanto o firewall filtra por porta/IP, o IDS/IPS analisa o conteúdo dos pacotes. Em ambientes modernos, NGFWs muitas vezes integram IPS embutido. Para redes críticas, use Suricata com regras atualizadas (Emerging Threats ruleset) e integre os alertas ao SIEM.

"Um IDS sem resposta é apenas ruído. Um IPS mal configurado derruba produção. Calibre com cuidado."
