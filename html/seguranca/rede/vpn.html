VPN — Virtual Private Network

O que é uma VPN?

VPN — Virtual Private Network

Túnel criptografado sobre a internet para acesso remoto e privacidade de tráfego.

VPN cria um túnel criptografado sobre uma rede pública (internet), permitindo que dispositivos se comuniquem como se estivessem na mesma rede privada. Usada para acesso remoto seguro, site-to-site entre filiais e privacidade de tráfego.

WireGuard — Configuração Básica

# ----- Servidor (wg0.conf) -----
[Interface]
PrivateKey = <SERVER_PRIVATE_KEY>
Address = 10.0.0.1/24
ListenPort = 51820
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]
PublicKey = <CLIENT_PUBLIC_KEY>
AllowedIPs = 10.0.0.2/32

# ----- Cliente (wg0.conf) -----
[Interface]
PrivateKey = <CLIENT_PRIVATE_KEY>
Address = 10.0.0.2/24
DNS = 1.1.1.1

[Peer]
PublicKey = <SERVER_PUBLIC_KEY>
Endpoint = vpn.exemplo.com:51820
AllowedIPs = 0.0.0.0/0   # full tunnel
# AllowedIPs = 10.0.0.0/24  # split tunnel — apenas rede interna

# Gerar chaves:
wg genkey | tee privatekey | wg pubkey > publickey

Split Tunneling vs Full Tunnel

Full Tunnel

AllowedIPs = 0.0.0.0/0 — todo o tráfego do cliente passa pelo servidor VPN. Máxima visibilidade e controle para SOC. Pode impactar velocidade e saturar bandwidth corporativo.

Split Tunnel

Apenas tráfego para redes internas vai pelo túnel; tráfego para internet sai direto. Reduz carga no gateway VPN. Risco: dispositivo comprometido pode exfiltrar dados via internet sem passar pela inspeção corporativa.

VPN vs Zero Trust Network Access

VPN Tradicional — Limitações

  • Acesso amplo à rede interna após autenticação
  • Usuário comprometido = acesso lateral irrestrito
  • Difícil granularidade por aplicação
  • Não verifica postura do dispositivo continuamente

ZTNA — Zero Trust Network Access

  • Acesso por aplicação, não por rede inteira
  • Verifica identidade + dispositivo + contexto a cada sessão
  • Microsegmentação nativa
  • Exemplos: Cloudflare Access, Zscaler ZPA, Google BeyondCorp

Para uso pessoal e pequenas empresas, WireGuard em VPS próprio é a melhor relação custo-benefício e privacidade. Para ambientes corporativos, avalie ZTNA — a tendência clara do mercado para substituir VPNs tradicionais. Sempre habilite MFA no gateway VPN.

"Uma VPN protege o canal, não o destino. Combine com autenticação forte e monitoramento."
