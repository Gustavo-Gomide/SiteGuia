O que é Hardening?

Hardening de Sistemas

Reduzir a superfície de ataque desabilitando o desnecessário e aplicando menor privilégio.

Hardening é o processo de reduzir a superfície de ataque de um sistema — desabilitando funcionalidades desnecessárias, aplicando configurações seguras, removendo serviços ociosos e aplicando o princípio do menor privilégio. Um sistema hardened é mais difícil de explorar mesmo quando vulnerabilidades existem.

Princípios Fundamentais

Redução de Superfície de Ataque

Cada serviço, porta, conta e software instalado é um vetor potencial. Remova ou desabilite tudo que não é necessário para a função do sistema.

Menor Privilégio

Processos e usuários devem operar com as permissões mínimas necessárias. Evita que comprometimento de um componente resulte em comprometimento total.

Defense-in-Depth

Múltiplas camadas de controles. Se uma camada falha, as outras ainda protegem. Hardening do SO + firewall host + EDR + segmentação de rede.

Secure by Default

Configuração padrão deve ser a mais restritiva. Usuários/admins habilitam explicitamente o que precisam em vez de desabilitar o que não querem.

CIS Benchmarks

O Center for Internet Security (CIS) publica benchmarks gratuitos com checklists detalhados para hardening de sistemas operacionais, servidores, bancos de dados e aplicações cloud. São o padrão de mercado.

Processo de Hardening

1. Inventário e Baseline

Documente o estado atual do sistema. Use ferramentas como Lynis (Linux) ou Microsoft Security Compliance Toolkit (Windows) para comparar contra benchmarks.

2. Aplicar Controles

Implemente as recomendações priorizando por criticidade. Teste em ambiente não-produção antes. Use scripts de automação (Ansible, PowerShell DSC).

3. Validar

Execute auditoria pós-hardening para confirmar que os controles foram aplicados e que o sistema ainda funciona conforme esperado.

4. Manter

Hardening não é único. Patches, novos serviços e mudanças de configuração podem reverter controles. Automatize scans de compliance regulares.

Hardening deve ser parte do ciclo de vida do sistema — desde o provisionamento (gold image) até o descomissionamento. Use automação (Ansible, Puppet, Chef) para garantir que todos os sistemas sigam o mesmo baseline e que desvios sejam detectados. Veja as páginas específicas de Hardening Linux e Hardening Windows para checklists detalhados.

"A segurança padrão é insuficiente. Hardening transforma uma instalação padrão em um sistema que resiste ao atacante."
