Hardening Windows

GPOs, CIS Benchmarks e STIG para endurecer servidores e clientes Windows.

Hardening Windows aplica configurações seguras segundo benchmarks CIS e STIG, restringindo serviços, configurando políticas de grupo e removendo funcionalidades desnecessárias para reduzir a superfície de ataque.

CIS Benchmarks para Windows

Políticas detalhadas para Windows Server 2019/2022 e Windows 10/11 nos níveis 1 e 2. Podem ser importadas como GPOs. Download gratuito em cisecurity.org.

Microsoft Security Compliance Toolkit

Conjunto de GPOs e scripts oficiais da Microsoft para hardening. Inclui baselines para Windows, Edge, Microsoft 365 e Defender. Usa Policy Analyzer para comparar.

LGPO.exe

Ferramenta da Microsoft para aplicar configurações de GPO localmente em máquinas fora de domínio — ideal para workstations standalone.

Attack Surface Reduction Rules

ASR Rules do Microsoft Defender bloqueiam técnicas de ataque específicas mesmo sem assinaturas de malware.

# Habilitar ASR Rule via PowerShell:
# Exemplo: Bloquear execução de conteúdo de macros Office
Add-MpPreference -AttackSurfaceReductionRules_Ids d4f940ab-401b-4efc-aadc-ad5f3c50688a `
                 -AttackSurfaceReductionRules_Actions Enabled

# Regras importantes:
# Block Office from creating child processes:
# d4f940ab-401b-4efc-aadc-ad5f3c50688a

# Block credential stealing from LSASS:
# 9e6c4e1f-7d60-472f-ba1a-a39ef669e4b3

# Block execution of potentially obfuscated scripts:
# 5beb7efe-fd9a-4556-801d-275e5ffc04cc

# Block abuse of exploited vulnerable signed drivers:
# 56a863a9-875e-4185-98a7-b882c64b5ce5

# Ver status atual:
Get-MpPreference | Select-Object -ExpandProperty AttackSurfaceReductionRules_Ids

Checklist Básico

Contas e Acesso

  • Desabilitar conta Administrator padrão (ou usar LAPS)
  • Renomear conta Guest
  • Política de senha: 14+ chars, complexidade obrigatória
  • Lockout: 5 tentativas → bloquear 30 min
  • MFA para RDP e VPN

Serviços e Rede

  • Desabilitar SMBv1 (Set-SmbServerConfiguration -EnableSMB1Protocol $false)
  • Desabilitar LLMNR e NetBIOS
  • Windows Firewall habilitado em todos os perfis
  • RDP via VPN apenas; habilitar NLA
  • Desabilitar serviços não usados (Print Spooler, etc.)

Atualizações e Monitoramento

  • Windows Update automático habilitado
  • Microsoft Defender atualizado
  • Audit Policy: Success+Failure para Logon, Account Management, Policy Change
  • Sysmon instalado e configurado
  • Logs encaminhados ao SIEM

Para ambientes de domínio, aplique hardening via GPOs do Microsoft Security Compliance Toolkit. Para workstations isoladas, use LGPO.exe. Combine com Defender for Endpoint (EDR) e Microsoft Sentinel (SIEM) para visibilidade completa. Documente toda exceção feita ao baseline.
