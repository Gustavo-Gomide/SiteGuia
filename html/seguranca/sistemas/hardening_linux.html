Hardening Linux

SSH, serviços, kernel e controles mandatórios — checklist completo para Linux.

Hardening Linux reduz a superfície de ataque do sistema operacional aplicando configurações seguras, desabilitando serviços desnecessários e endurecendo componentes críticos como SSH, cron e controles de acesso mandatórios.

# /etc/ssh/sshd_config — configurações críticas:

# Desabilitar login root direto:
PermitRootLogin no

# Apenas autenticação por chave:
PasswordAuthentication no
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys

# Limitar usuários com acesso SSH:
AllowUsers deploy admin

# Desabilitar X11 forwarding:
X11Forwarding no

# Timeout de sessão inativa (300 segundos):
ClientAliveInterval 300
ClientAliveCountMax 0

# Protocolo e ciphers modernos apenas:
Protocol 2
KexAlgorithms curve25519-sha256,diffie-hellman-group14-sha256
Ciphers aes256-gcm@openssh.com,chacha20-poly1305@openssh.com
MACs hmac-sha2-256-etm@openssh.com

# Alterar porta padrão (opcional, segurança por obscuridade):
Port 2222

systemctl restart sshd

Usuários e Senhas

# Política de senha (/etc/security/pwquality.conf):
minlen = 14
dcredit = -1    # mínimo 1 dígito
ucredit = -1    # mínimo 1 maiúscula
lcredit = -1    # mínimo 1 minúscula
ocredit = -1    # mínimo 1 caracter especial
maxrepeat = 3   # máximo 3 repetições
enforce_for_root

# /etc/login.defs — expiração de senha:
PASS_MAX_DAYS 90
PASS_MIN_DAYS 7
PASS_WARN_AGE 14

# Desabilitar contas sem senha:
awk -F: '($2 == "" ) { print $1 }' /etc/shadow

# Verificar contas com UID 0 (root indevido):
awk -F: '($3 == "0") { print }' /etc/passwd

# Remover shells de login de contas de serviço:
usermod -s /sbin/nologin www-data

Filesystem e Permissões

# Arquivos com SUID perigosos (auditoria):
find / -perm -4000 -type f 2>/dev/null

# Arquivos world-writable:
find / -perm -0002 -type f 2>/dev/null

# /etc/fstab — montar /tmp com noexec,nosuid,nodev:
tmpfs /tmp tmpfs defaults,noexec,nosuid,nodev 0 0

# Permissões críticas:
chmod 600 /etc/shadow
chmod 644 /etc/passwd
chmod 700 /root
chmod 600 /etc/ssh/sshd_config

# Verificar integridade de arquivos com AIDE:
aide --init
aide --check

SELinux / AppArmor

SELinux (RHEL/CentOS/Fedora)

# Verificar status:
getenforce

# Modo enforcing (bloqueia):
setenforce 1

# /etc/selinux/config:
SELINUX=enforcing

# Ver negações recentes:
ausearch -m avc -ts recent

# Gerar política customizada:
audit2allow -a -M mypolicy

AppArmor (Debian/Ubuntu)

# Ver status:
aa-status

# Colocar perfil em enforce:
aa-enforce /etc/apparmor.d/usr.bin.nginx

# Ver violações:
dmesg | grep apparmor

# Criar perfil:
aa-genprof /usr/bin/myapp

Auditoria e Logs

# auditd — regras de auditoria (/etc/audit/rules.d/audit.rules):

# Monitorar acesso a arquivos sensíveis:
-w /etc/passwd -p wa -k identity
-w /etc/shadow -p wa -k identity
-w /etc/sudoers -p wa -k sudoers

# Monitorar execução de comandos:
-a always,exit -F arch=b64 -S execve -k exec_log

# Monitorar falhas de autenticação:
-a always,exit -F arch=b64 -S open -F exit=-EACCES -k access_denied

# Monitorar modificações de rede:
-a always,exit -F arch=b64 -S sethostname -S setdomainname -k network

service auditd restart
ausearch -k identity | aureport -f

Use o Lynis para auditoria automatizada contra CIS Benchmarks: lynis audit system. Automatize hardening com Ansible roles (ex: dev-sec/ansible-hardening no GitHub) para aplicar o mesmo baseline em todos os servidores do inventário.
