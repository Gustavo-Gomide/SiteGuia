Princípio do Menor Privilégio

Gerenciamento de Privilégios

Menor privilégio, PAM, sudo e controle de contas — limitar blast radius.

Menor privilégio (Principle of Least Privilege — PoLP) é um dos princípios mais importantes de segurança: cada usuário, processo ou sistema deve ter apenas as permissões estritamente necessárias para realizar sua função, e nada mais. Limita o blast radius em caso de comprometimento.

sudo — Linux Privilege Escalation Controlada

# /etc/sudoers (sempre editar com visudo):

# Permitir usuário específico executar apenas um comando como root:
deploy ALL=(root) NOPASSWD: /usr/bin/systemctl restart nginx

# Grupo ops pode reiniciar qualquer serviço:
%ops ALL=(root) /usr/bin/systemctl restart *

# Nunca dar sudo irrestrito:
# RUIM: user ALL=(ALL) NOPASSWD: ALL
# Isso equivale a dar root sem senha

# Auditar uso do sudo:
tail -f /var/log/auth.log | grep sudo

PAM — Privileged Access Management

PAM é uma solução corporativa para gerenciar, monitorar e auditar o acesso de usuários privilegiados a sistemas críticos. É diferente do PAM do Linux (Pluggable Authentication Modules).

Funcionalidades PAM Corporativo

  • Cofre de senhas com rotação automática
  • Gravação de sessões privilegiadas
  • Workflow de aprovação para acesso
  • MFA obrigatório para acesso privilegiado
  • Auditoria completa de ações

JIT — Just-in-Time Access

Acesso privilegiado concedido apenas quando necessário, por tempo limitado. Elimina contas admin permanentes. Reduz risco de roubo de credenciais privilegiadas ociosas. Exemplos: Azure PIM, CyberArk JIT, HashiCorp Vault.

JEA — Just Enough Administration

Recurso PowerShell que cria endpoints de administração restrita — usuários podem executar apenas comandos específicos em um contexto elevado, sem receber privilégios completos de admin.

Ferramentas PAM

  • CyberArk Privileged Access Manager
  • BeyondTrust Password Safe
  • Delinea Secret Server
  • HashiCorp Vault (open source)
  • Azure AD Privileged Identity Management

Contas administrativas são o alvo final de qualquer ataque sofisticado. PAM com JIT reduce o tempo de exposição de credenciais privilegiadas. Para começar: audite quem tem admin, implemente LAPS para admins locais Windows, e introduza aprovação de workflow para acesso a sistemas críticos.

"Uma conta com privilégio excessivo não é um vetor de ataque — é uma porta escancarada."
