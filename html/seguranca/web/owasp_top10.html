O que é o OWASP Top 10?

OWASP Top 10 — 2021

As 10 categorias de risco mais críticas em aplicações web segundo o OWASP.

O OWASP Top 10 é a lista das 10 categorias de risco mais críticas em aplicações web, publicada pela Open Web Application Security Project. É atualizada a cada 3–4 anos com base em dados de vulnerabilidades reais. Serve como guia de priorização para desenvolvedores e pentesters.

A01 — Broken Access Control (Detalhado)

Tornou-se #1 em 2021 (era #5 em 2017). Abrange IDOR, privilege escalation horizontal/vertical e falta de verificação de autorização.

IDOR — Insecure Direct Object Reference

GET /api/orders/12345  # pedido de outro usuário
# App não verifica se pedido 12345 pertence ao usuário logado

Privilege Escalation

# Vertical: usuário comum acessa função de admin
POST /admin/users/delete { "userId": 42 }

# Horizontal: usuário A acessa dados do usuário B
GET /profile?userId=B

Mitigação

  • Verificar autorização server-side em toda requisição
  • Usar IDs internos opacos (UUID) em vez de IDs sequenciais
  • Testes automatizados de controle de acesso
  • Audit log de operações sensíveis

O OWASP Top 10 não é uma lista de todas as vulnerabilidades — é um guia de priorização. Para cada item, consulte o portal owasp.org/Top10 para detalhes, casos reais e materiais de teste. O OWASP também publica toolsets como ZAP (scanner de aplicações web gratuito).

"Conhecer o OWASP Top 10 não é opcional para quem desenvolve ou testa aplicações web — é o mínimo."
