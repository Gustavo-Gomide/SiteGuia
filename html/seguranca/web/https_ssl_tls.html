HTTPS, SSL e TLS

HTTP sobre TLS — confidencialidade, integridade e autenticidade na camada de transporte.

HTTPS = HTTP sobre TLS (Transport Layer Security). TLS é o sucessor do SSL (que foi depreciado com SSLv3 em 2015). Fornece confidencialidade (criptografia), integridade (HMAC) e autenticidade (certificados X.509) na camada de transporte. TLS 1.3 é o padrão atual — mais rápido, mais seguro e com menos etapas de handshake.

TLS 1.3 Handshake

# TLS 1.3 — Handshake simplificado (1 RTT):

Client → Server: ClientHello
  - supported versions: TLS 1.3
  - cipher suites: AES-128-GCM-SHA256, ChaCha20-Poly1305
  - key_share: parâmetros Diffie-Hellman (ECDHE public key)

Server → Client: ServerHello + EncryptedExtensions + Certificate + CertificateVerify + Finished
  - escolhe cipher suite
  - envia key_share (ECDHE public key)
  - deriva chaves de sessão via HKDF
  - envia certificado X.509 (identidade do servidor)
  - CertificateVerify: assina o handshake com chave privada
  - Finished: MAC sobre o handshake completo

Client → Server: Finished
  - verifica assinatura e certificado
  - deriva mesmas chaves de sessão

# A partir daqui: dados criptografados com symmetric keys derivadas ECDHE
# Cada sessão usa chaves diferentes → Perfect Forward Secrecy garantida

Cipher Suites

Uma cipher suite é uma combinação de algoritmos para troca de chave, autenticação, cifra e MAC.

Cipher Suite (TLS 1.3)CifraHash / AEADStatus
TLS_AES_256_GCM_SHA384AES-256-GCMSHA-384 Excelente
TLS_AES_128_GCM_SHA256AES-128-GCMSHA-256 Boa
TLS_CHACHA20_POLY1305_SHA256ChaCha20Poly1305 Boa (mobile/embedded)

Em TLS 1.3, cipher suites não incluem mais componentes de troca de chave (são sempre ECDHE) ou autenticação (separados do cipher). Isso eliminou centenas de combinações inseguras legadas presentes no TLS 1.2.

Certificados X.509 e PKI

Cadeia de Confiança

Certificados formam uma cadeia: Root CA → Intermediate CA → Leaf certificate. Browsers e OSes têm listas de Root CAs confiáveis. A validação percorre a cadeia até um Root CA confiável.

Tipos de Validação

  • DV (Domain Validation) — verifica controle do domínio; emissão automática (Let's Encrypt)
  • OV (Organization Validation) — verifica organização; processo manual
  • EV (Extended Validation) — verificação rigorosa; barra verde (legado)

Certificate Transparency (CT)

Todos os certificados emitidos publicamente devem ser registrados em CT logs públicos. Permite detectar emissões fraudulentas. Verificável em crt.sh.

Revogação

CRL (Certificate Revocation List) — lista baixada periodicamente. OCSP — verificação online em tempo real. OCSP Stapling — servidor inclui resposta OCSP no handshake (mais eficiente).

Configuração Segura

# Nginx — TLS 1.3 + configuração segura:
ssl_protocols TLSv1.2 TLSv1.3;              # desabilita 1.0 e 1.1
ssl_ciphers 'TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256:ECDHE-RSA-AES256-GCM-SHA384';
ssl_prefer_server_ciphers off;               # TLS 1.3 ignora; TLS 1.2 usa cipher do server
ssl_session_cache shared:SSL:10m;
ssl_session_timeout 1d;
ssl_session_tickets off;                     # desabilita session tickets (PFS)
ssl_stapling on;                             # OCSP Stapling
ssl_stapling_verify on;
add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" always;

# OpenSSL — verificar cipher suite de um servidor:
openssl s_client -connect site.com:443 -tls1_3
openssl s_client -connect site.com:443 | openssl x509 -noout -text

# testssl.sh — auditoria completa:
./testssl.sh --full site.com

Certificate Pinning

Aplicações mobile/desktop podem fixar o certificado ou chave pública esperada do servidor. Impede ataques MitM mesmo com CA comprometida. Risco: pin expirado = app quebra sem update.

Mixed Content

Recursos HTTP carregados em página HTTPS — browsers bloqueiam conteúdo ativo misto (scripts, iframes). Use upgrade-insecure-requests no CSP como rede de segurança.

TLS 1.3 eliminou décadas de problemas de design: sem RSA key exchange, sem ciphers não-AEAD, handshake mais rápido. Se você está começando hoje, configure apenas TLS 1.2 + 1.3 com ECDHE e AEAD ciphers. Use Let's Encrypt para certificados gratuitos e automatizados. Teste com ssllabs.com/ssltest para obter nota A+.

"TLS 1.3 representa décadas de aprendizado com falhas criptográficas — use-o como padrão, sem exceções."
