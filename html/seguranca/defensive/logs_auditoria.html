Logs e Auditoria de Segurança

Sem logs, não há como detectar intrusões nem investigar incidentes.

Logs são o registro de tudo que acontece em sistemas e redes. Sem logs, não há possibilidade de detectar intrusões, investigar incidentes ou provar conformidade com regulações. "Você só pode defender o que pode ver."

Os três usos principais de logs: Segurança (detectar ataques, responder a incidentes), Conformidade (LGPD, PCI-DSS, ISO 27001 exigem logs por período mínimo) e Operações (debugging, performance, rastreamento de erros de aplicação).

Integridade dos Logs

Logs locais são insuficientes

Um atacante que obtém root/SYSTEM pode apagar logs locais. Se os logs existem apenas no host comprometido, a evidência desaparece. Logs devem ser enviados para servidor centralizado imediatamente.

Log forwarding em tempo real

Syslog, Winlogbeat, Fluentd, Vector — enviar logs para SIEM ou storage seguro em tempo real. Um atacante teria que comprometer o servidor de logs também — muito mais difícil.

Imutabilidade

Uso de storage imutável (S3 Object Lock, Azure Immutable Blob) para arquivamento de logs. Logs não podem ser alterados ou deletados — garantia para auditoria e forense.

NTP — Sincronização de tempo

Todos os sistemas devem estar sincronizados com NTP. Diferença de 1 minuto entre sistemas torna a correlação de eventos extremamente difícil durante investigação forense.

O que NÃO logar

Dados sensíveis em logs são um risco

Logs são frequentemente armazenados em texto claro e acessíveis a mais pessoas que o sistema original. Nunca logue:

  • Senhas (mesmo em campos de erro)
  • Tokens de autenticação completos, JWTs, session cookies
  • Dados de cartão de crédito (CVV, PAN completo) — PCI-DSS proíbe
  • Dados pessoais sensíveis (CPF, passaporte) — prefira log só do ID interno
  • Chaves privadas, segredos de API

A qualidade dos seus logs define a qualidade da sua resposta a incidentes. Uma investigação forense que depende de logs pobres é como resolver um crime sem câmeras ou testemunhas. Explore SIEM para centralizar e correlacionar esses logs.

"O atacante que entrou ontem já saiu — mas se você tiver os logs certos, você ainda pode seguir o rastro e entender tudo que ele fez."
