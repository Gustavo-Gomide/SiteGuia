O que é Forense Digital?

Forense Digital

Identificar, preservar e analisar evidências digitais para investigações e perícias.

Forense digital é a disciplina de identificar, preservar, analisar e apresentar evidências digitais de forma que possa ser usada em investigações e, quando necessário, em processos legais. A integridade das evidências é tão importante quanto o que elas revelam.

DFIR: Digital Forensics and Incident Response — as duas disciplinas são frequentemente combinadas. Forense foca na coleta e análise de evidências; IR foca na resposta ao incidente. Na prática, um bom investigador faz as duas coisas, em paralelo, desde o início.

Chain of Custody — Cadeia de Custódia

Chain of custody é a documentação ininterrupta de quem coletou, transferiu e analisou cada evidência. Sem essa rastreabilidade, as evidências podem ser inadmissíveis em processos judiciais.

O que documentar

  • Quem coletou a evidência, quando, onde e como
  • Hash criptográfico (SHA-256) da evidência no momento da coleta
  • Cada transferência de custódia — quem entregou, quem recebeu
  • Armazenamento: acesso controlado, lacres físicos se necessário

Princípio da não-alteração

Nunca analise a evidência original. Sempre trabalhe com cópias forenses bit-a-bit (imagens). O hash verifica que a cópia é idêntica ao original — qualquer alteração invalida a evidência.

Forense de Disco

Inclui análise de sistemas de arquivos, arquivos deletados, metadados, artefatos do sistema operacional e timelines de atividade.

Aquisição de imagem

Use dd, dcfldd ou FTK Imager para criar cópia forense bit-a-bit. Conecte o disco via write blocker (hardware) para garantir que nada é escrito no original.

Artefatos Windows

Prefetch (programs executados), Jump Lists, Shellbags (pastas acessadas), MFT ($MFT — Master File Table), Event Logs, Registro (NTUSER.DAT), LNK files (atalhos), Recycle Bin.

Artefatos Linux

/var/log/, ~/.bash_history, /etc/cron.d/, /tmp/, /proc/ (processo ao vivo), systemd journal, SSH known_hosts, authorized_keys.

Arquivos deletados

Quando um arquivo é deletado, o conteúdo nem sempre é apagado — só a referência no sistema de arquivos. Ferramentas como photorec, Autopsy e Scalpel podem recuperar dados de espaço "livre".

# Criar imagem forense com verificação de hash:
# Conectar disco via write-blocker, identificar device:
fdisk -l

# Criar imagem bit-a-bit com hash simultâneo:
dcfldd if=/dev/sdb of=/evidence/disco.img \
    hash=sha256 \
    hashlog=/evidence/disco.sha256.log

# Verificar integridade da imagem:
sha256sum /evidence/disco.img

Forense de Memória (RAM)

A memória RAM contém dados que não existem em disco: processos em execução, conexões de rede, credenciais em memória, malware fileless. É a evidência mais volátil — deve ser coletada primeiro, antes do disco.

Coleta de RAM

  • Windows: WinPmem, Magnet RAM Capture, DumpIt, Belkasoft RAM Capturer
  • Linux: LiME (Linux Memory Extractor) — módulo de kernel
  • VMware: suspender VM cria arquivo .vmem automaticamente

Análise com Volatility

  • pslist / pstree — lista processos
  • netscan — conexões de rede ativas e fechadas
  • malfind — detecta código injetado em processos legítimos
  • lsadump — extrai hashes NTLM da memória (como Mimikatz)
  • cmdline / consoles — comandos PowerShell/CMD executados

Forense digital exige paciência e rigor metodológico. Um erro na coleta pode destruir evidências irreversíveis. Para praticar, use plataformas CTF com desafios forenses (HackTheBox, CyberDefenders, BlueTeamLabs Online) e o projeto DFIR.training.

"A evidência que você não coletou nas primeiras horas pode nunca mais existir. Na forense digital, a ordem de volatilidade decide a ordem de coleta."
