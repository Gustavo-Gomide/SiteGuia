Funções Hash Criptográficas

O que é uma Função Hash?

Funções Hash Criptográficas

Transforma qualquer entrada em digest de tamanho fixo — determinístico e irreversível.

Uma função hash criptográfica transforma uma entrada de qualquer tamanho em uma saída de tamanho fixo (digest) — de forma determinística, rápida e irreversível.

Diferentemente da criptografia, hashing não é reversível — não existe chave, não existe decifração. Um hash é uma impressão digital matemática do dado.

Exemplo prático

SHA-256("ola") = 672a038b2e86dc6e4bbf9f0b6c1a7ff9cc5db5ccc... (64 caracteres hexadecimais)

SHA-256("Ola") = 3b6b1a0e94e4b2c012478399a3e74a44aca9a... — completamente diferente (efeito avalanche)

SHA-256("ola") sempre retorna o mesmo hash — determinístico

Propriedades de Segurança

Pré-imagem resistente

Dado um hash h, é computacionalmente inviável encontrar qualquer mensagem m tal que hash(m) = h. Garante que não dá para reverter.

Segunda pré-imagem resistente

Dado m e hash(m), é inviável encontrar m' ≠ m tal que hash(m') = hash(m). Garante que não dá para forjar um documento com mesmo hash de outro.

Resistência a colisões

É inviável encontrar qualquer par (m, m') tal que hash(m) = hash(m'). A propriedade mais forte — e a mais difícil de manter.

Efeito avalanche

Mudança de 1 bit na entrada altera ~50% dos bits da saída. Garante que outputs visualmente não revelam similaridades entre inputs.

Aplicações de Hashing

Armazenamento de senhas

NUNCA armazenar senhas em texto claro ou com hashes rápidos (SHA-256). Use bcrypt, Argon2 ou scrypt — projetados para ser lentos e resistentes a ataques de GPU.

# Python — Argon2 (estado da arte):
from argon2 import PasswordHasher
ph = PasswordHasher()
hash = ph.hash("minha_senha")
ph.verify(hash, "minha_senha")  # True

HMAC — Autenticação de Mensagem

Hash + chave secreta. HMAC-SHA256 garante que mensagem não foi alterada E que veio de quem tem a chave. Usado em JWTs, APIs REST, cookies de sessão.

import hmac, hashlib
mac = hmac.new(b"chave_secreta",
               b"mensagem",
               hashlib.sha256).hexdigest()

Verificação de integridade

Downloads de software: compare SHA-256 fornecido pelo fabricante com o hash local. Se divergir, arquivo foi modificado ou corrompido.

Assinaturas digitais

Documentos grandes não são assinados diretamente — o hash do documento é assinado. RSA/ECDSA assinam SHA-256(documento).

Blockchain / Proof of Work

Bitcoin usa SHA-256 duplo (SHA-256(SHA-256(bloco))). Minerar = encontrar nonce que produz hash com N zeros iniciais — prova de trabalho computacional.

Deduplicação de dados

Sistemas de backup e armazenamento usam hashes como endereço de conteúdo (content-addressed storage). Git e IPFS identificam objetos por SHA-1/SHA-256.

Ataques contra Hashes

Rainbow Tables

Tabelas pré-computadas de hash(senha) para dicionários comuns. Resolvido por salt — valor aleatório único adicionado a cada senha antes de hash. bcrypt/Argon2 incluem salt automaticamente.

Ataque de Aniversário

Paradoxo do aniversário: para hash de n bits, encontrar colisão requer ~2^(n/2) operações. SHA-256 requer 2^128 — impraticável. MD5 requer 2^64 — acessível com hardware moderno.

Length Extension Attack

Vulnerabilidade da construção Merkle-Damgård (MD5, SHA-1, SHA-2). Dado hash(m), um atacante pode computar hash(m || extensão) sem conhecer m. Use HMAC ou SHA-3 para evitar.

Hashes são a cola da segurança criptográfica moderna: protegem senhas, verificam integridade e habilitam assinaturas digitais. Veja MD5 e SHA em detalhes e Assinaturas Digitais.

"Um bom hash é como uma impressão digital digital — única, rápida de verificar, impossível de falsificar. Com o hash errado, você tem um castelo de areia."
