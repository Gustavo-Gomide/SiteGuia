O que é PKI?

PKI — Infraestrutura de Chaves Públicas

O sistema que torna possível confiar em identidades digitais em escala global.

PKI (Public Key Infrastructure) é o sistema de tecnologias, políticas e pessoas que cria, gerencia, distribui, usa e revoga certificados digitais — tornando possível a confiança em identidades digitais em escala global.

Sem PKI, criptografia assimétrica não resolve o problema do "quem é essa chave pública?" Um atacante poderia distribuir sua própria chave pública fingindo ser seu banco. PKI usa uma terceira parte confiável — a CA — para resolver isso.

Componentes da PKI

CA — Certification Authority

Autoridade Certificadora: emite, assina e revoga certificados. É a âncora de confiança do sistema. Root CAs são altamente protegidas — frequentemente offline em HSMs em instalações seguras.

RA — Registration Authority

Autoridade de Registro: recebe e valida pedidos de certificados antes de encaminhar à CA. Separa a função de validação da emissão — distribuindo a carga e aumentando segurança operacional.

VA — Validation Authority

Autoridade de Validação: responde a consultas OCSP sobre o status de certificados. Serve as listas de revogação (CRL) e respostas OCSP em tempo real.

Certificate Repository

Repositório público de certificados. LDAP ou HTTP. Os browsers consultam para baixar intermediate CAs e verificar CRLs.

HSM — Hardware Security Module

Módulo de hardware dedicado para armazenar e usar chaves privadas de CA. Chaves nunca saem do HSM em texto claro. Requisito para CAs confiáveis e ICP-Brasil.

Trust Store

Conjunto de root CAs em que o sistema confia — embutido no SO (Windows, Linux) e navegadores. Mozilla NSS, Microsoft Root Program, Apple Root Program definem quais CAs são confiáveis.

ICP-Brasil

A Infraestrutura de Chaves Públicas Brasileira é gerenciada pelo ITI (Instituto Nacional de Tecnologia da Informação) e possui validade jurídica definida pela MP 2.200-2/2001.

PKI Privada — Uso Interno

Organizações frequentemente operam sua própria PKI para autenticação interna, mTLS entre microserviços e VPNs — sem pagar CAs públicas por uso interno.

HashiCorp Vault

PKI Secrets Engine — CA privada totalmente automatizada. Emite certificados com TTL curtos (horas/dias), integra com Kubernetes e Nomad. Altamente recomendado para microserviços.

CFSSL (CloudFlare)

Toolkit de PKI open source da CloudFlare. CLI e API para gerenciar CA privada. Usado pelo Kubernetes internamente.

OpenSSL CA

Solução manual para contextos simples. Controle total mas operacional mais trabalhoso. Adequado para PKI de desenvolvimento e laboratórios.

# PKI privada simples com OpenSSL:

# 1. Criar CA raiz:
openssl genrsa -out ca.key 4096
openssl req -new -x509 -days 3650 -key ca.key -out ca.crt \
    -subj "/CN=Minha CA Privada/O=Empresa/C=BR"

# 2. Criar certificado de servidor:
openssl genrsa -out server.key 2048
openssl req -new -key server.key -out server.csr \
    -subj "/CN=api.empresa.local/O=Empresa/C=BR"

# 3. Assinar com a CA privada:
openssl x509 -req -days 365 -in server.csr \
    -CA ca.crt -CAkey ca.key -CAcreateserial -out server.crt

Mutual TLS (mTLS)

No TLS padrão, apenas o servidor apresenta certificado ao cliente. No mTLS, ambos apresentam certificados — autenticação mútua. Muito usado em arquiteturas zero-trust e comunicação entre microserviços.

Casos de uso

  • Comunicação segura entre microserviços (service mesh: Istio, Linkerd)
  • APIs B2B onde o cliente precisa se autenticar com certificado
  • VPNs — OpenVPN, WireGuard com PKI
  • Zero Trust Networks — todo acesso exige certificado de dispositivo

Vantagens sobre senha/token

  • Não há credenciais na rede — apenas prova criptográfica
  • Revogação centralizada (revogar certificado = revogar acesso)
  • Auditável — cada cliente tem identidade única rastreável
  • Difícil de phishing — certificado vinculado ao dispositivo

PKI é o sistema invisível que mantém a internet confiável. Entender seus componentes é essencial para qualquer papel de segurança ou arquitetura de sistemas. Revise Certificados X.509 e HTTPS/TLS para aplicação prática.
