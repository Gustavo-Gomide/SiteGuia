Animacao 3D: Key Poses, Timing e Integracao com a Engine

Da pose de referencia ao state machine: como animacao de arquivo vira comportamento de personagem.

Animacao de jogo nao e animacao de filme. O objetivo nao e impressionar numa cena linear — e comunicar estado do personagem de forma legivel para o jogador em qualquer frame do loop, inclusive no frame 1 do idle.

Animacao para Jogo vs Animacao para Filme

Em filme, a narrativa controla quando cada animacao acontece e por quanto tempo. Em jogos, o jogador controla o tempo: uma animacao de corrida pode durar indefinidamente, loop de idle pode ser interrompido a qualquer frame e ataques precisam ser cancelaveis se o design pedir. Isso muda tudo: cada clip precisa de pose de entrada e saida previsiveis para o sistema de blending da engine.

A diferenca pratica: animador de filme polida spline. Animador de jogo polida loops de 4, 8 ou 16 frames com pose inicial igual a pose final, e cuida de transicoes entre estados (idle-to-run, run-to-attack, hit-reaction).

Key Poses e Timing

A base tecnica da animacao sao as key poses: contact, down, passing e high para um ciclo de caminhada; windup, strike e recovery para um ataque. In-betweens preenchem o espaco entre as keys, mas sao as keys que carregam a leitura do movimento.

Timing por Contexto

  • Idle: 60–120 frames a 30fps. Loop com drift leve para nao parecer robotico.
  • Run cycle: 8–16 frames. Cada 4 frames e um paso; passos simetricos permitem loop.
  • Ataque leve: 20–40 frames. Windup (4–8f), strike (4–6f), recovery (8–16f).
  • Hit reaction: 12–20 frames. Rapido para nao interruper demais o flow do player.

Root Motion

Root motion e quando o deslocamento do personagem vem do proprio clip de animacao (o bone root se move no espaco). Permite animacoes mais organicas, mas exige que a engine leia o delta de posicao do root a cada frame. Contrapartida: in-place animation nao mexe o root — o movimento e controlado pelo codigo de locomocao.

Tipos e Organizacao de Clips

Nomenclatura de Clips

  • CH_Player_Idle_01 (Character, entidade, estado, variacao).
  • CH_Player_Run_Forward
  • CH_Player_Attack_Light_01
  • CH_Player_Death_Front

Blend Tree e State Machine

Na engine, clips sao organizados em Animator Controller (Unity) ou Animation Blueprint (Unreal). Um blend tree mistura idle e run proporcionalmente a velocidade. O state machine define transicoes com condicoes (isGrounded, speed, isAttacking) e tempos de crossfade.

NLA, Export e Otimizacao

No Blender, o NLA Editor (Non-Linear Animation) organiza todas as actions em faixas independentes. Para exportar para engine, cada action do NLA vira um clip separado no FBX. A nomenclatura da action vira o nome do clip na engine — usar o padrao de nomenclatura no NLA desde o inicio evita renomeacao manual no Unity/Unreal.

Otimizacao de animacao: remover keyframes desnecessarios (curvatura plana entre dois keys iguais), usar Sample Rate de 30fps para jogos mobile (em vez de 60fps), e compressao de animacao na engine (Unity Anim Compression, Unreal Compression Settings). Animacao de personagem de jogo mobile bem comprimida ocupa menos de 100KB por clip simples.
