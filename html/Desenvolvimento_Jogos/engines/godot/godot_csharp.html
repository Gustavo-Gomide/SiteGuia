C# no Godot: .NET 6, Atributos e Interop com GDScript

Quando usar C# no lugar de GDScript, como o sistema de atributos funciona e quais as limitacoes reais por plataforma.

C# no Godot 4 roda em .NET 6 via Mono/CoreCLR. Nao e uma versao limitada de C# — e C# completo, com LINQ, async/await e bibliotecas NuGet. A limitacao e de plataforma, nao de linguagem: iOS e Web exigem restricoes de AOT que ainda impactam o workflow.

C# vs GDScript: Quando Usar Cada Um

Use C# quando...

  • O projeto ja tem equipe com experiencia C#/Unity.
  • Sistemas complexos precisam de tipagem estrita e refactoring seguro (IDE + Roslyn).
  • Integracao com bibliotecas .NET externas (redes, serialization, ML).
  • Exportacao apenas para PC/console — plataformas sem restricoes AOT.

Use GDScript quando...

  • Exportacao para Web (HTML5) ou iOS — C# tem limitacoes de IL2CPP/WASM.
  • Time pequeno ou solo dev sem background C#.
  • Iteracao rapida e prototipagem de mecanicas.
  • Scripts de UI e logica de cena simples.

Atributos e Interop com o Editor

No Godot 4, a ponte entre C# e o editor usa atributos. [Export] expoe variaveis no Inspector, [Signal] declara signals usaveis via editor e via codigo GDScript. Toda classe exposta ao Godot deve herdar de um tipo Godot (Node, Resource, etc.) — classes C# puras nao aparecem no editor.

Player.cs
using Godot;

[GlobalClass]
public partial class Player : CharacterBody2D
{
    [Export] public float Speed { get; set; } = 200f;
    [Export] public NodePath AnimPlayerPath { get; set; }

    [Signal] public delegate void PlayerDiedEventHandler();

    private AnimationPlayer _anim;

    public override void _Ready()
    {
        _anim = GetNode<AnimationPlayer>(AnimPlayerPath);
    }

    public override void _PhysicsProcess(double delta)
    {
        var dir = Input.GetAxis("move_left", "move_right");
        Velocity = new Vector2(dir * Speed, Velocity.Y);
        MoveAndSlide();
    }
}

Interop GDScript — C# e NuGet

GDScript pode chamar metodos de classes C# e vice-versa via Variant. Passsagem de tipos primitivos e automatica; tipos complexos precisam de Variant.From(). Para libraries NuGet, basta adicionar no .csproj normalmente — Godot compila o projeto .NET junto ao build do jogo.

Cuidado com async/await: o loop principal do Godot nao e thread-safe. Use await ToSignal() para aguardar signals sem sair da main thread, ou use CallDeferred() para chamar metodos de threads filhas de volta na main thread.
