GDScript: A Linguagem Nativa do Godot

Tipagem opcional, signals e ciclo de vida de nodes — a base para sistemas de jogo robustos.

GDScript nao e Python para jogos. E uma linguagem projetada em torno do modelo de nodes do Godot: cada script e um tipo, cada node e um objeto. Entender essa relacao e o que separa codigo que "funciona" de codigo que escala.

GDScript vs Python e Tipagem

GDScript tem sintaxe inspirada em Python mas nao e Python: nao ha pip, nao ha virtualenv, e o modelo de concorrencia e completamente diferente. No Godot 4, a tipagem estatica opcional foi amplamente melhorada — usar : int, : String e : Vector2 nos parametros e variaveis permite ao compilador gerar bytecode mais eficiente e ao Godot LSP oferecer autocomplete preciso.

A regra pratica: use tipagem em scripts de sistema (character controller, inventory, save/load). Em scripts de one-shot (tweens pontuais, UI callbacks simples), tipagem e opcional. Scripts nao tipados com Variant sao mais lentos em hot paths.

Ciclo de Vida, Nodes e Cena

No Godot, o script e sempre anexado a um Node. O ciclo de vida segue uma ordem deterministica: _init() ao construir o objeto, _ready() quando o node e toda sua arvore entram na cena, _process(delta) a cada frame de render e _physics_process(delta) no loop de fisica (fixo a 60 Hz por padrao).

player.gd
extends CharacterBody2D

const SPEED: float = 200.0

@onready var anim: AnimationPlayer = $AnimationPlayer

func _ready() -> void:
    # Chamado quando o node esta na arvore. Seguro para acessar filhos.
    anim.play("idle")

func _physics_process(delta: float) -> void:
    var dir := Input.get_axis("move_left", "move_right")
    velocity.x = dir * SPEED
    move_and_slide()
    if dir != 0:
        anim.play("run")
    else:
        anim.play("idle")

Signals e Comunicacao entre Nodes

Signals Customizados

Signals desacoplam emissor de receptor. O node que emite nao precisa saber quem ouve. Declare com signal nome_do_signal(parametro: Tipo) e emita com nome_do_signal.emit(valor). Conecte via editor (painel Node) ou via codigo com .connect(metodo).

Autoloads como Singletons

Autoloads sao nodes persistentes carregados antes de qualquer cena. Sao o Singleton do Godot: use para GameManager, AudioBus, SaveSystem, EventBus. Acesse globalmente pelo nome configurado em Project Settings > Autoload. Nao abuse — autoloads em excesso criam acoplamento global.

Performance e Boas Praticas

Em hot paths (physics process, grande numero de inimigos), evite alocacao de novos objetos a cada frame. Prefira @onready para cachear referencias a nodes em vez de chamar get_node() repetidamente. Use StringName (&"nome") para identificadores de animacao e input — comparacao de StringName e O(1), String e O(n).

Para sistemas com muitos objetos (particulas customizadas, projectiles em massa), considere usar MultiMeshInstance2D ou TileMap ao inves de instanciar centenas de nodes individuais. O profiler integrado (Debugger > Profiler) mostra o custo real de cada metodo.
