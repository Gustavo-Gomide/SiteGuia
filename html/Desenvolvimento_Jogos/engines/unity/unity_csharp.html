C# no Unity

Arquitetura de Gameplay em C#

Estruturas praticas para codigo limpo no Unity.

Menos script gigante, mais sistemas pequenos e testaveis.

Fundamentos

  • Entenda `Awake`, `Start` e `Update`.
  • Prefira componentes pequenos.
  • Use `SerializeField` com encapsulamento.

Padroes Praticos

  • Event bus para comunicacao entre sistemas.
  • State machine para personagem e IA.
  • Object pooling para objetos recorrentes.
unity_csharp_checklist.txt
cada script = 1 responsabilidade
sem logica pesada em Update
eventos para UI e HUD
pooling para projeteis e FX

Avancado

  • Reducao de alocacao e GC em runtime.
  • Profiling de CPU por sistema.
  • Separacao entre logica de dominio e visual.

Erros Comuns

  • Monobehaviour com responsabilidade demais.
  • Dependencia circular entre scripts.
  • Otimizar sem medir no profiler.

C# no Unity escala quando arquitetura vem antes da pressa.
