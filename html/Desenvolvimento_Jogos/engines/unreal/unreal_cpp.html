C++ no Unreal: Macros, Gameplay Framework e GAS

UCLASS, UPROPERTY, UFUNCTION — como o sistema de reflexao do Unreal conecta C++ ao editor e ao Blueprint.

C++ no Unreal nao e C++ puro. O Unreal Header Tool (UHT) pre-processa seus headers e gera codigo de reflexao. Entender esse sistema de macros e pre-requisito para tudo: de expor variaveis no editor a criar habilidades com GAS.

Macros UHT: UCLASS, UPROPERTY, UFUNCTION

Toda classe gerenciada pelo GC do Unreal deve ser decorada com UCLASS(). Variaveis expostas ao editor, Blueprint ou replicadas pela rede precisam de UPROPERTY(). Funcoes chamáveis por Blueprint ou via console usam UFUNCTION(). Sem essas macros, o objeto existe em C++ mas e invisivel ao resto do engine.

MyCharacter.h
UCLASS()
class MYGAME_API AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    // Exposta no editor, replicada pela rede
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Replicated, Category="Stats")
    float Health = 100.f;

    // Chamavel pelo Blueprint
    UFUNCTION(BlueprintCallable, Category="Combat")
    void TakeDamage(float Amount);

private:
    // Referencia a componente — nao exposta
    UPROPERTY()
    TObjectPtr<USkeletalMeshComponent> MeshComp;
};

Gameplay Framework

Hierarquia de Classes

  • AActor: objeto basico da cena — tem transform, pode ter componentes.
  • APawn: Actor possuivel por um Controller — tem input.
  • ACharacter: Pawn com CharacterMovementComponent — locomocao, gravidade, nocao de chao.
  • AController / APlayerController: logica de decisao separada da representacao fisica.
  • AGameMode / AGameState: regras da partida; replicacao de estado global.

Hot Reload vs Live Coding

Hot Reload (legado): recompila sem fechar o editor, mas tem bugs conhecidos com heranca e delegates — pode corromper o estado de objetos em memoria. Live Coding (UE5 padrao): mais estavel, usa patching de codigo em execucao. Para mudancas grandes (adicionar UPROPERTY, mudar hierarquia), sempre feche o editor antes.

Gameplay Ability System (GAS)

GAS e o sistema de habilidades do Unreal usado em jogos como Fortnite e Paragon. Centraliza atributos (AttributeSet), efeitos (GameplayEffect) e habilidades (GameplayAbility) num framework replicavel e data-driven. A curva de aprendizado e alta — GAS so vale para projetos que precisam de habilidades complexas, stacks e redes multiplayer.

Para projetos menores, um UActorComponent customizado com UPROPERTY(Replicated) resolve 80% dos casos sem a complexidade do GAS. O criterio e: quantas habilidades, quantos efeitos, quanta interacao entre eles?
