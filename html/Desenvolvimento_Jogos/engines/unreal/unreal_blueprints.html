Blueprints: Visual Scripting, Event Graph e Interop com C++

Do Event BeginPlay ao Function Library — como Blueprint se encaixa num pipeline profissional com C++.

Blueprint nao e "C++ simplificado". E uma linguagem visual completa com tipos, heranca, interfaces e delegates. A diferenca real em relacao a C++ e performance em hot paths e legibilidade de sistemas complexos — nao capacidade funcional.

Tipos de Blueprint

Blueprint Class

Blueprint Class herda de uma classe C++ (ou de outro Blueprint). E o tipo mais comum — define comportamento de Actors, Pawns e Characters. Pode ter componentes, variaveis, funcoes e event graph proprios. O editor visual mostra o grafo de execucao, a hierarquia de componentes e o painel de variaveis ao mesmo tempo.

Blueprint Function Library

Function Library e uma colecao de funcoes estaticas puras sem estado — acessiveis de qualquer outro Blueprint ou C++. Ideal para utilitarios matematicos, helpers de conversao, queries de mundo. Nao tem instancia, nao tem componentes. Equivale a uma classe static em C++.

Event Graph e Comunicacao entre Blueprints

O Event Graph e acionado por eventos (BeginPlay, Tick, input, overlap, delegates). A execucao flui pelos pinos brancos (exec pins). Tick e o evento mais perigoso: qualquer logica pesada no Tick de centenas de objetos derruba framerate rapidamente — prefira timers e delegates.

Para comunicacao entre Blueprints: Cast funciona mas cria dependencia direta — use com moderação. Blueprint Interface desacopla completamente, mas so passa dados via parametros. Event Dispatcher (delegate) permite que objetos se inscrevam sem o emissor saber quem ouve — o padrao correto para eventos de gameplay (morte, pickup, fim de fase).

Interop C++ — Blueprint e Migracao

O padrao de producao e: logica de sistema em C++ (UFUNCTION(BlueprintCallable)), logica de conteudo e scripting de nivel em Blueprint. C++ define a interface; Blueprint a usa sem ter acesso ao codigo. Isso permite que designers iterem sem recompilar o projeto inteiro.

Quando migrar Blueprint para C++: quando o grafo tem mais de ~50 nodes e se tornou ilegivel; quando esta em hot paths que o profiler mostra como gargalo; quando voce precisa de templates, delegates complexos ou integracao com bibliotecas externas. O Unreal tem ferramenta de Blueprint Nativization (legado, UE4) mas em UE5 o caminho recomendado e reescrever manualmente.
